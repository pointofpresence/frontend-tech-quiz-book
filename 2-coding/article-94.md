# Function Declaration vs Function Expression в JavaScript, в чем разница?

### Категории

Coding, Algorithm complexity, JavaScript, Async, Event loop, Debugging, Algorithms, Function declaration, Functions, Function expression, Console output, TypeScript, Frontend, Live coding

### Краткий ответ

**Function Declaration vs Function Expression в JavaScript**

- обе создают функции, но различаются объявлением и временем создания  
- **Function Declaration**: функция создаётся при загрузке скрипта (hoisting), доступна до объявления  
- **Function Expression**: функция создаётся при выполнении, доступна только после инициализации  
- Function Declaration имеет имя, Function Expression может быть анонимной или именованной  
- Function Expression часто используется для колбеков и IIFE  
- важная разница — доступность и поднятие (hoisting) функций в коде  
- практическое значение — выбор зависит от нужд и контроля порядка вызовов функций

### Подробный ответ
#### Основной ответ
**Function Declaration** — это классическое объявление функции с использованием ключевого слова `function` в начале, которое создает именованную функцию, доступную во всём блоке видимости. **Function Expression** — это объявление функции как выражения, обычно присваиваемое переменной. Отличие в поведении при hoisting: объявления функций поднимаются к началу их области видимости, а выражения — нет.

#### Ключевые моменты
- **Hoisting**: Function Declarations загружаются в память до выполнения кода, поэтому их можно вызывать до объявления. Function Expressions — просто значения переменных, и доступны только после инициализации.
- **Scope и анонимность**: Function Declarations всегда имеют имя, выражения могут быть анонимными или именованными. Это влияет на отладку и рекурсию.
- **Использование в практике**: Function Declarations обычно применяют для создания стандартных функций, Function Expressions — для коллбеков, замыканий и передачи функций как первого класса объектов.

#### Практический контекст
В React 18+ часто применяют Function Expressions, например, стрелочные функции для обработчиков, чтобы сохранить контекст `this`. Также в современных проектах Function Expressions легче использовать с замыканиями и функциональными паттернами. В отличие от этого Function Declarations удобны для определения утилитного функционала, доступного во всем модуле.
