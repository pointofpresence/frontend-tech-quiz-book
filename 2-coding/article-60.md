# Как работает авторизация через OAuth?

### Категории

Prefix tree, Coding, Security, Authorization, Backend, Data structures, Live coding, Networking

### Краткий ответ

**Как работает авторизация через OAuth?**

- протокол авторизации для безопасного доступа  
- отделяет **аутентификацию пользователя** и предоставление прав приложениям  
- клиент (приложение) получает **токен доступа** от сервера авторизации  
- пользователь подтверждает права приложению через **OAuth consent screen**  
- токен доступа используется для запросов к защищённому API  
- поддерживает разные **grant types** (authorization code, implicit, client credentials)  
- обеспечивает **минимальный доступ** без передачи паролей пользователя

### Подробный ответ
#### Основной ответ
**OAuth** — это протокол авторизации, который позволяет сторонним приложениям получать ограниченный доступ к ресурсам пользователя на другом сервисе без передачи его логина и пароля. Основной принцип — пользователь явно предоставляет доступ приложению через доверенный сервер авторизации, чтобы оно могло получать токены доступа и взаимодействовать с API от имени пользователя.

#### Ключевые моменты
- В процессе участвуют три стороны: **ресурсный сервер** (где хранятся данные), **сервер авторизации** (выдает токены) и **клиентское приложение** (которое запрашивает доступ).
- Основные шаги: клиент перенаправляет пользователя на сервер авторизации, где пользователь входит и соглашается дать доступ. Далее сервер авторизации выдает **authorization code**, который клиент меняет на **access token** — короткоживущий ключ для запросов API.
- Существует несколько типов потоков (grant types), например, Authorization Code (для серверных приложений), Implicit (для SPA), Client Credentials (для серверных коммуникаций без пользователя) и Refresh Token (для обновления access token без повторной авторизации).
- OAuth 2.0 не занимается аутентификацией напрямую — для этого используется OpenID Connect, расширение OAuth, которое подтверждает личность пользователя.

#### Практический контекст
Часто OAuth используют, чтобы позволить, например, приложению делиться фото или читать почту пользователя на Google, не требуя вводить пароль в самом приложении. В крупных системах (например, Facebook Login, Google Sign-In) применяют OAuth 2.0 с дополнительными мерами безопасности: PKCE для SPA, HTTPS для защиты token, ограничение scope доступа, и мониторинг злоупотреблений.
