# Какова временная сложность операций List: add, remove, contains?

### Категории

Algorithms, Coding, Data structures, Complexity

### Краткий ответ

**Временная сложность операций List: add, remove, contains**  
- Контекст: List — структура данных, динамический массив или связанный список  
- add (добавление):  
  - Для ArrayList: **амортизированно O(1)** (добавление в конец)  
  - При перераспределении памяти — O(n)  
  - Для LinkedList: **O(1)** при добавлении в начало/конец  
- remove (удаление):  
  - ArrayList: **O(n)** (поиск элемента + сдвиг)  
  - LinkedList: **O(n)** (поиск), удаление — O(1) после поиска  
- contains (поиск):  
  - Линейный поиск, **O(n)** для обоих типов  
- Отличия зависят от типа List и реализации  
- Практическое применение:  
  - ArrayList выгоден для частых добавлений в конец и редких удалений  
  - LinkedList лучше, если много удалений/вставок в середине  
- Ключевой момент: операции поиска и удаления по значению — **линейная сложность**  
- Добавление в конец у ArrayList — **почти всегда константа**  
Итог: **add: O(1) амортизированно, remove: O(n), contains: O(n)** (вариации зависят от реализации)

### Подробный ответ
#### Основной ответ
Временная сложность операций списка (**List**) зависит от конкретной реализации структуры данных, но если говорить о наиболее распространённой — **ArrayList** (динамический массив) и **LinkedList** (связанный список), то их характеристики различаются.

Для **ArrayList**:
- **add (добавление в конец)** — амортизированное O(1), так как иногда происходит расширение массива.
- **remove (удаление по индексу)** — O(n) из-за сдвига элементов после удаления.
- **contains (поиск элемента)** — O(n), поскольку требуется последовательный перебор элементов.

Для **LinkedList**:
- **add (в конец)** — O(1), если есть указатель на конец списка.
- **remove (по элементу или индексу)** — O(n), так как нужно найти элемент или позицию.
- **contains (поиск элемента)** — O(n), линейный поиск по узлам.

#### Ключевые моменты
- **Сложности зависят от реализации**: ArrayList опирается на массив с быстрым доступом по индексу, LinkedList — на узлы с указателями.
- **Добавление в середину:** в ArrayList это O(n) (из-за сдвига), в LinkedList — O(n) из-за поиска позиции (доступ к узлу по индексу линейный).
- **Различие между амортизированным и худшим случаем**: расширение массива в ArrayList — редкое событие, но при нем вставка занимает O(n).

#### Практический контекст
В проектах, где важна быстрая индексация и быстрые добавления в конец, выбирают **ArrayList** (например, в Java Collections Framework). Если часты вставки и удаления в середину списка, без быстрого доступа по индексу, более эффективен **LinkedList**. Обычно для поиска по значению лучше использовать хэш-структуры или специальные индексы, а не List с линийным временем.
