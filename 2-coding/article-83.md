# Чем отличаются IEnumerable и IQueryable? Где применяется отложенное выполнение запросов?

### Категории

Coding, Microservices, Deferred execution, Algorithms, Optimization, Live coding, Collections

### Краткий ответ

**IEnumerable vs IQueryable, отложенное выполнение**

- **IEnumerable** — интерфейс для последовательной итерации по коллекции в памяти  
- **IQueryable** — интерфейс для построения и исполнения запроса к удалённым источникам (БД)  
- IEnumerable выполняет запросы локально, IQueryable — делегирует преобразования на сервер  
- Оба поддерживают **отложенное выполнение** (вычисление запускается при перечислении)  
- Отложенное выполнение в **IQueryable** позволяет оптимизировать запрос в БД, комбинируя операции до отправки  
- IEnumerable подходит для работы с in-memory коллекциями, IQueryable — для LINQ к БД и ORM (Entity Framework)  
- Основное применение отложенного выполнения — **оптимизация производительности** и **минимизация данных**, загружаемых из БД  

В итоге, IEnumerable — для локальной обработки, IQueryable — для эффективных запросов к внешним хранилищам через выражения, с применением отложенного выполнения для оптимизации.

### Подробный ответ
#### Основной ответ
**IEnumerable** и **IQueryable** — это два интерфейса в .NET, которые служат для работы с последовательностями данных, но отличаются по способу выполнения и месту обработки запросов. **IEnumerable** предназначен для перебора коллекций в памяти (in-memory), тогда как **IQueryable** позволяет отложенно формировать и выполнять запросы к удалённым источникам данных (например, к базе данных), по сути переводя их в конкретный SQL или другой запрос под капотом.

#### Ключевые моменты
- **IEnumerable<T>**: работает с уже загруженными в память данными, реализует перебор элементов, поддерживает отложенное выполнение для LINQ-to-Objects, но все операции выполняются на клиенте.
- **IQueryable<T>**: расширяет IEnumerable и добавляет возможность формирования выражений (Expression Trees), которые провайдер (например, Entity Framework) трансформирует в запрос под целевой источник (SQL для баз, OData и т.п.), что означает отложенное выполнение запроса именно там, где данные хранятся.
- **Отложенное выполнение (deferred execution)** — ключевая особенность обеих, но особенно IQueryable. Запрос не выполняется при построении LINQ-выражения, а только при реальном переборе (ToList(), foreach и т.д.), что позволяет оптимизировать выборку данных и избежать преждевременного вычисления.

#### Практический контекст
- **IEnumerable** используется, когда данные уже в памяти, например, списки, массивы.
- **IQueryable** — в ORM типа Entity Framework Core 6+, когда важна оптимизация запросов к СУБД, минимизация трафика, и нужна поддержка сложных фильтров, сортировок и перевод их в SQL.
- Типичный пример: при работе с БД через EF, запрос к `DbSet<T>` возвращает IQueryable — SQL формируется и выполняется только после вызова ToList() или при enumeration, что снижает нагрузку и ускоряет приложение.

Таким образом, понимание разницы между IEnumerable и IQueryable критично для оптимального построения эффективных запросов и правильного использования отложенного выполнения в реальных проектах.
