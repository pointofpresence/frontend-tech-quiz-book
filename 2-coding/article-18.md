# Как еще, кроме DISTINCT, можно добиться уникальности записей в выборке?

### Категории

Database, Coding, Query, Optimization

### Краткий ответ

**Как добиться уникальности записей кроме DISTINCT?**

- агрегатные функции (GROUP BY) — группировка по ключам  
- использование оконных функций (ROW_NUMBER(), RANK()) с фильтрацией по порядку  
- применение подзапросов с EXISTS/IN для исключения дубликатов  
- использование уникальных индексов/ключей на уровне БД для контроля уникальности  
- фильтрация по уникальному идентификатору (например, по первичному ключу)  
- UNION вместо UNION ALL для объединения наборов данных с удалением дубликатов  
- DISTINCT оптимален для простого удаления дублей, остальные методы — для более сложной логики  
   
Эти методы дают гибкость управления уникальностью, зависят от задачи и объема данных, часто применяются для оптимизации производительности или специфичной фильтрации.

### Подробный ответ
#### Основной ответ  
Кроме использования **DISTINCT** для получения уникальных записей в выборке, можно добиться уникальности с помощью **группировки (GROUP BY)** и использования **аналитических функций** или оконных функций, таких как **ROW_NUMBER()**. Также применяют агрегатные функции, выбирая, например, первый или последний элемент из группы. 

#### Ключевые моменты  
- **GROUP BY** — группирует строки по уникальному набору полей, при этом можно агрегировать остальные колонки с помощью функций (SUM, MAX, MIN). При правильном выборе колонок группировки обеспечивается уникальность результата.  
- **ROW_NUMBER() OVER (PARTITION BY ... ORDER BY ...)** — эта оконная функция нумерует строки внутри групп, что позволяет отфильтровать только первую запись из каждой группы (например, уникальные по определенному атрибуту). Часто используется в подзапросах или CTE.  
- Альтернативный метод — **DISTINCT ON** в PostgreSQL, позволяющий получить уникальные записи по определённой колонке с выбором нужной строки по ORDER BY, что гибче, чем простой DISTINCT.  

#### Практический контекст  
В реальных проектах часто используют **ROW_NUMBER()** для дедупликации данных при сложных фильтрах или когда нужно получить по одной уникальной записи на группу с дополнительным условием сортировки (например, самая свежая запись). В PostgreSQL 12+ и других СУБД с оконными функциями это стандартный подход для более гибкой работы с уникальностью, чем простое DISTINCT, который иногда слишком грубый и не позволяет выбрать конкретную версию записи.
