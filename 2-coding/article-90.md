# Расскажите про все методы Promise?

### Категории

Coding, Algorithm complexity, Asynchronous programming, JavaScript, Promises, Async, Event loop, Validation, Promise, Debugging, TypeScript, Frontend, Sorting

### Краткий ответ

**Методы Promise**

- **Promise** — объект асинхронной операции в JS  
- Основные методы:  
  - **then(onFulfilled, onRejected)** — добавляет обработчики успешного/ошибочного результата, возвращает новый Promise  
  - **catch(onRejected)** — сокращение для then(undefined, onRejected), ловит ошибки  
  - **finally(onFinally)** — выполняет код после завершения, не меняет результат  
- Статические методы:  
  - **Promise.resolve(value)** — создаёт выполненный Promise с value  
  - **Promise.reject(reason)** — создаёт отклонённый Promise с reason  
  - **Promise.all(iterable)** — ждёт выполнения всех, отклонится при первой ошибке  
  - **Promise.race(iterable)** — резолвится или реджектится при первом завершении любого из iterable  
  - **Promise.allSettled(iterable)** — ждёт завершения всех, возвращает результаты со статусами  
  - **Promise.any(iterable)** — возвращает первый успешно выполненный Promise, отклоняется если все rejected  
- Методы организуют цепочки и обработку асинхронности с контролем ошибок  
- Используются для упрощения кода и повышения читаемости по сравнению с коллбэками  

Этот набор методов формирует основу работы с Promise, позволяя гибко управлять асинхронными задачами и их результатами.

### Подробный ответ
#### Основной ответ
**Promise** в JavaScript — это объект, представляющий результат асинхронной операции. Методы Promise помогают управлять этими асинхронными операциями и их комбинациями. Основные методы делятся на **инстанс-методы** и **статические методы**.

#### Ключевые методы

##### Инстанс-методы:
- **then(onFulfilled, onRejected)** — обрабатывает успешный (fulfilled) или ошибочный (rejected) результат. Возвращает новый Promise, что позволяет цепочки.
- **catch(onRejected)** — сокращение для обработки ошибок, эквивалентен `then(null, onRejected)`.
- **finally(onFinally)** — вызывается всегда после завершения (успешно или с ошибкой), не влияет на результат, но полезен для очистки ресурсов.

##### Статические методы:
- **Promise.resolve(value)** — возвращает Promise, который решается с переданным значением.
- **Promise.reject(reason)** — возвращает Promise, который отклоняется с причиной.
- **Promise.all(iterable)** — ожидает успешного выполнения всех промисов в массиве, или сразу отклоняется при ошибке одного из них.
- **Promise.allSettled(iterable) (ES2020+)** — ждет завершения всех промисов с их статусами без отклонения при ошибках.
- **Promise.race(iterable)** — возвращает первый пришедший (resolved или rejected) промис.
- **Promise.any(iterable) (ES2021+)** — ждет первого успешно выполненного промиса, отклоняется, если все промисы отклоняются.

#### Ключевые моменты
- Методы **then/catch/finally** дают цепочку, позволяя строить сложные последовательности асинхронных операций с обработкой ошибок.
- Статические методы позволяют параллельно работать с множеством промисов — выбор алгоритма зависит от задачи: ждать все, первый результат или первый успех.
- С версии ES2020/ES2021 появились методы **allSettled** и **any**, значительно расширяющие возможности для управления множеством промисов.

#### Практический контекст
В реальных проектах методы Promise — база для асинхронного кода: загрузки данных, сетевых запросов, работы с файлами. Например, `Promise.all` используют для параллельного получения данных из нескольких API, `finally` для скрытия индикаторов загрузки, `Promise.any` — для оптимистичного ожидания первого успешного ответа из нескольких источников.
