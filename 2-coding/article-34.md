# Как очистить массив или слайс в Go?

### Категории

Coding, Slices, Context, Concurrency, Memory management, Arrays, Backend, Channels, Data structures

### Краткий ответ

**Как очистить массив или слайс в Go?**

- массив — фиксированного размера, очистить — перезаписать нулями
- слайс — динамический, ссылается на массив
- сбросить длину: `slice = slice[:0]` — освобождает элементы, память остаётся
- обнулить слайс: `slice = nil` — освобождает и память, длина и емкость 0
- для повторного использования — важно понимать, нужен ли полный сброс или просто очистка
- массивы редко очищают, обычно работают со слайсами
- практично для освобождения памяти и предотвращения утечек при больших данных

### Подробный ответ
#### Основной ответ
В Go для очистки массива или среза (slice) нужно учитывать, что массивы имеют фиксированную длину, а срезы — динамическую. Очистка массива невозможна в классическом понимании, потому что его размер и память статичны. Для среза очистка обычно означает либо обнуление всех элементов, либо сброс длины среза до нуля, чтобы “удалить” все элементы логически.

#### Ключевые моменты
- Для **массива** можно "обнулить" элементы с помощью цикла или функции `copy` с нулевым массивом, но сам размер останется неизменным, так как он фиксирован.
- Для **очистки среза** существует несколько вариантов:
  - присвоение `s = s[:0]` — сбрасывает длину среза до 0, оставляя выделенную память для повторного использования, что эффективно и часто используется
  - присвоение `s = nil` — устанавливает срез в `nil`, полностью освобождая внутренний массив, если на него нет других ссылок
  - явный цикл с обнулением элементов `[for i := range s { s[i] = zeroValue }]` — полезно, когда нужно гарантировать очистку содержимого из памяти для безопасности
- Выбор способа зависит от целей: если важна минимизация аллокаций, используют `s = s[:0]`; если нужна полная очистка памяти и отсутствие ссылок — `s = nil`.

#### Практический контекст
В реальных проектах часто применяют `s = s[:0]` для очистки срезов при переработке буферов или кешей, чтобы сохранить выделенную память и снизить количество аллокаций, особенно в горячих циклах. Использование `s = nil` оправдано, когда срез больше не нужен, и важно освободить ресурсы. Обнуление элементов актуально при работе с чувствительными данными (например, пароли), чтобы исключить утечки памяти.
