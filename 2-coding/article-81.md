# Что такое Split Queries? Как они решают проблему производительности при загрузке связанных сущностей?

### Категории

Coding, Performance, Algorithms, Backend, Database, Optimization, Live coding

### Краткий ответ

**Что такое Split Queries? Как они решают проблему производительности при загрузке связанных сущностей?**

- Технический контекст: ORM, загрузка связанных данных (Eager Loading)  
- Split Queries — способ выполнения нескольких отдельных SQL-запросов вместо одного с JOIN  
- Избегают **проблемы "N+1"**, когда один запрос порождает множество дополнительных  
- Уменьшают размер и сложность **одного большого JOIN-запроса**, снижая нагрузку на базу  
- Позволяют эффективно загружать данные с несколькими связями, разделяя запросы по сущностям  
- Повышают производительность за счёт улучшения планов выполнения и кэширования результатов  
- Практически полезны при работе с большими объёмами или сложными связями, где JOIN тормозит  

Итог: Split Queries — это техника разбиения сложного запроса с JOIN на несколько простых, что снижает нагрузку на БД и улучшает отклик при загрузке связанных сущностей.

### Подробный ответ
#### Основной ответ
**Split Queries** — это техника оптимизации выполнения запросов в ORM (например, Entity Framework Core 5+) для загрузки связанных сущностей. Вместо одного сложного запроса с множеством JOIN, который может привести к дублированию данных и большим накладным расходам, Split Queries разбивают загрузку на несколько простых запросов. Это снижает объем повторяющихся данных и улучшает производительность при работе со сложными графами объектов.

#### Ключевые моменты
- В обычных **Single Query (Single SQL)** ORM строит один запрос с большим количеством JOIN, из-за чего результат содержит дублированные строки, что увеличивает трафик и нагрузку на память.
- **Split Queries** расщепляют выборку на отдельные запросы для каждой связанной коллекции или навигации, затем ORM агрегирует данные в память, устраняя дублирование и снижая общий объем передаваемых данных.
- Обычно Split Queries выигрывают на запросах с множеством связанных коллекций и большим количеством строк, где Single Query приводит к экспоненциальному росту результата. Однако они делают больше обращений к базе, поэтому злоупотреблять ими не стоит.

#### Практический контекст
В проектах на **Entity Framework Core 5 и выше** Split Queries часто применяются через метод `AsSplitQuery()`. Например, при загрузке заказов с большим числом позиций и клиентов, Single Query может вернуть огромный результат с повторяющимися данными, а Split Queries разбивают это на три легких запроса, снижая задержки и нагрузку на сеть. Такой подход улучшает общую отзывчивость API и снижает потребление памяти в приложении.
