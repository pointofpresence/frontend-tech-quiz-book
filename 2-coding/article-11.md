# В каком порядке выполняется условие WHERE с оператором AND?

### Категории

Coding, Database, Query, Optimization, Condition, SQL

### Краткий ответ

**Порядок выполнения WHERE с AND**

- SQL: фильтрация строк в запросе
- WHERE обрабатывает все условия одновременно
- оператор AND требует **истинности всех условий**
- оптимизатор может менять порядок проверки условий
- проверка условий идёт для каждой строки таблицы
- нет гарантированного жёсткого порядка условий
- важна логика, не порядок выполнения
- ускоряет индексация и статистика СУБД

### Подробный ответ
#### Основной ответ
В SQL условие в **WHERE с оператором AND** выполняется поэтапно, но с точки зрения логики они объединяются как конъюнкция, то есть все подусловия должны быть истинными для включения строки в результат. На самом деле порядок вычисления отдельных подусловий в WHERE не строго фиксирован и может зависеть от оптимизатора запросов СУБД — он решает, в каком порядке проверять условия для максимальной эффективности.

#### Ключевые моменты
- **Логика AND**: Все условия должны быть выполнены (TRUE), иначе строка отбрасывается. Это эквивалент логического "И" — `(cond1 AND cond2 AND ...)`.
- **Оптимизация порядка**: Оптимизатор может менять порядок проверки условий, учитывая статистику, индексы и предполагаемую стоимость вычисления, чтобы быстрее отфильтровывать строки.
- **Короткое замыкание**: В некоторых СУБД при исполнении применяется логика короткого замыкания — если первое условие было FALSE, следующие не проверяются, что повышает производительность.
- **Влияние побочных эффектов**: Если в условиях есть функции с побочными эффектами, полагаться на порядок нельзя, так как оптимизатор может изменить последовательность выполнения.

#### Практический контекст
При написании запросов лучше структурировать условия так, чтобы тяжелые или ресурсоемкие проверки стояли после более простых, поскольку оптимизатор обычно это учитывает. Например, сначала фильтровать по индексируемым полям (например, `id = 5`), затем применять более сложные вычисления или подзапросы. В PostgreSQL 14+ и MySQL 8+ оптимизаторы позволяют эффективно переставлять условия для снижения latency запросов.
