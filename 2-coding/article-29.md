# Что такое курсорная пагинация и чем она лучше OFFSET?

### Категории

Coding, Pagination, Algorithms, Backend, Database, Optimization

### Краткий ответ

**Что такое курсорная пагинация и чем она лучше OFFSET?**  
- пагинация данных в БД/API  
- курсорная использует **уникальный маркер** последней записи (cursor)  
- OFFSET смещается по индексам: **OFFSET + LIMIT**  
- курсорная быстрее на больших наборах (нет пропуска строк)  
- OFFSET медленен при больших значениях (нужна полная фильтрация)  
- курсорная исключает дублирование и пропуски при изменениях данных  
- практично для API и потоковых данных (плавная навигация без лагов)

### Подробный ответ
#### Основной ответ
**Курсорная пагинация** — это метод постраничного вывода данных, при котором для перехода к следующему набору результатов используется уникальный идентификатор последней строки (курсор) текущей страницы, а не номер страницы или смещение. Это позволяет эффективно и точно «продвинуться» по большому набору данных.

Отличие от классического подхода с OFFSET в SQL (например, `LIMIT OFFSET`) заключается в том, что курсорная пагинация работает с устойчивыми позициями в наборе данных, а OFFSET пересчитывает и пропускает записи при каждом запросе, что при больших объемах данных вызывает деградацию производительности и несогласованность результатов (например, при параллельных вставках/удалениях).

#### Ключевые моменты
- **Производительность:** OFFSET заставляет СУБД просчитывать и пропускать все записи до смещения, что при больших страницах (например, OFFSET 100000) тяжелая операция. Курсорная пагинация использует индекс по уникальному полю (например, ID или timestamp) и быстро начинается с нужного места, минимизируя сканирование.
- **Консистентность:** при динамически изменяющейся базе OFFSET может пропускать или дублировать записи, если данные меняются во время пагинации. Курсорная пагинация надежнее в таких сценариях, потому что курсор является маркером позиции, что предотвращает пропуски.
- **Идемпотентность и UX:** курсор позволяет получить следующий набор данных без «пропусков» или «повторов», что улучшает пользовательский опыт при бесконечной прокрутке (infinite scroll) и real-time обновлениях.

#### Практический контекст
В реальных системах с большими объемами (например, ленты новостей, таблицы отчетов в PostgreSQL 14+, или API в GraphQL) курсорная пагинация — стандартный паттерн. Например, Facebook и Twitter используют курсоры для плавной загрузки бесконечного списка постов, а API GitHub с начала 2020-х перешел с OFFSET на курсоры для более масштабируемого и корректного решения.
