# Как работают map и filter? Изменяют ли исходный массив?

### Категории

Coding, Live coding, Functional programming, Map, Filter, Immutability, Array

### Краткий ответ

**Как работают map и filter? Изменяют ли исходный массив?**

- map и filter — методы работы с массивами в JavaScript  
- **map** создаёт новый массив, преобразуя каждый элемент функции  
- **filter** создаёт новый массив, отбирая элементы по условию  
- исходный массив **не изменяется**, методы **иммутабельны**  
- оба возвращают новый массив той же длины (**map**) или меньше (**filter**)  
- часто используются для **чистой обработки данных** без мутаций  
- поддерживают цепочки вызовов для удобства композиции  

---

Данный ответ показывает чёткое понимание сути методов, их иммутабельности и типовых сценариев использования — ключевые моменты для интервью.

### Подробный ответ
#### Основной ответ
**Методы `map` и `filter` в JavaScript** — это функции высшего порядка, которые работают с массивами, создавая на их основе новые массивы без изменения исходных данных. Метод **`map`** применяется для трансформации каждого элемента массива по заданной функции, возвращая новый массив с результатами этой трансформации. Метод **`filter`** возвращает новый массив, состоящий только из тех элементов исходного массива, для которых функция-предикат возвращает `true`.

Оба метода **не изменяют исходный массив**, они всегда возвращают новый массив, что помогает сохранять иммутабельность данных — важный аспект современного программирования.

#### Ключевые моменты
- **`map`** — проходит по каждому элементу исходного массива и создает новый массив того же размера, трансформируя элементы согласно переданной функции, например, увеличивая каждый элемент на 1.
- **`filter`** — выбирает подмножество элементов массива по условию (predicate), новые элементы могут быть равны исходным, но количество элементов может уменьшиться.
- Важная особенность — оба метода **не изменяют оригинальный массив**, тем самым поддерживают предсказуемость и упрощают отладку.
- В React (начиная с версий 16-18) такие методы активно используются для рендеринга списков, где изменение исходных данных крайне нежелательно.

#### Практический контекст
Использование этих методов типично для функционального программирования и реактивного кодирования: например, для отрисовки списка пользователей с трансформацией (map) или фильтрации списка задач по статусу (filter) в UI. Такой подход снижает количество ошибок, связанных с изменением состояния и улучшает поддержку кода.
