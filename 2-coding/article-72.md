# В чем разница между структурой и классом в C#?

### Категории

Coding, Algorithms, Backend, Optimization, Data structures, Live coding

### Краткий ответ

**Разница между структурой и классом в C#**  
- структура — **значимый тип**, класс — **ссылочный тип**  
- структура хранится на **стеке**, класс в **куче**  
- структуры не поддерживают наследование (только интерфейсы)  
- структура создаёт копию при присвоении, класс — ссылку  
- структуры не имеют деструкторов, классы могут  
- структуры обычно легче, быстрее создаются и удаляются  
- структуры подходят для небольших объемов данных, классы — для сложных моделей и с наследованием  
- практическое отличие: структура — для простых, **эффективных** данных; класс — для полноценных **объектно-ориентированных** моделей

### Подробный ответ
#### Основной ответ
В **C#** структура (**struct**) и класс (**class**) — это два разных типа данных с существенными отличиями. Основное различие в том, что **структура — это value type**, а **класс — reference type**. Это означает, что структуры хранятся и передаются по значению (копированием), а классы — по ссылке.

#### Ключевые моменты
- **Хранение в памяти:** Структуры обычно располагаются в стеке, что дает более быстрый доступ и меньшую нагрузку на сборщик мусора. Классы размещаются в управляемой куче, что добавляет накладные расходы на управление памятью.
- **Передача и копирование:** При присвоении или передаче структуры создаётся полная копия, что увеличивает безопасность данных, но может повлиять на производительность при больших структурах. Классы передаются по ссылке, поэтому копируется указатель, а не данные.
- **Наследование:** Структуры не поддерживают наследование (кроме реализации интерфейсов), тогда как классы могут участвовать в полной иерархии наследования и полиморфизме.
- **Инициализация и конструкторы:** В структуре всегда есть неявный конструктор по умолчанию, вызываемый без параметров, инициализирующий поля значениями по умолчанию. Классы требуют явной инициализации, могут иметь дефолтные и перегруженные конструкторы.
- **Использование:** Структуры подходят для небольших, неизменяемых объектов, где важна производительность и минимальная нагрузка на GC (например, **Point**, **DateTime** в .NET). Классы — для сложных объектов с динамическим поведением и большим размером.

#### Практический контекст
Типичные практические случаи для структур — координаты и базовые типы-сущности, где копирование безопасно и эффективно. Классы широко применяются для бизнес-логики, сложных моделей с наследованием и большим количеством методов. В .NET начиная с C# 7 появилась поддержка **readonly struct** для иммутабельности и улучшения оптимизации.
