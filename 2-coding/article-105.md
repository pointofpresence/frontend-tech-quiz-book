# Как устроено хранение массивов и срезов в памяти? Что такое capacity?

### Категории

Coding, Capacity, Performance, Infrastructure, Backend, Data structures, Memory

### Краткий ответ

**Как устроено хранение массивов и срезов в памяти? Что такое capacity?**

- Массивы: непрерывный блок памяти, фиксированный размер, элементы подряд  
- Срезы: структура-обёртка с указателем на массив, длиной и capacity  
- Capacity — максимальный размер среза без выделения новой памяти  
- При расширении среза > capacity происходит аллокация нового, большего массива  
- Срезы используют часть или весь массив, поддерживая гибкость и экономию памяти  
- Capacity показывает текущий резерв памяти для среза, влияет на производительность записи  
- Practical: capacity помогает понимать и оптимизировать перераспределение памяти

### Подробный ответ
#### Основной ответ  
В Go **массивы** — это последовательные блоки памяти фиксированного размера, где каждый элемент расположен подряд. Тип массива содержит в себе размер (количество элементов) и тип данных. Массивы хранятся напрямую в стеке или в куче, в зависимости от контекста, и размер массива не меняется после компиляции.

**Срезы (slices)** — это абстракция поверх массивов, которая представляет собой структуру из трех полей: указатель на базовый массив, длина (len) и ёмкость (capacity).  
- **Длина (length)** — текущее число элементов среза, доступных для чтения или записи.  
- **Ёмкость (capacity)** — максимальное число элементов, на которое может быть расширен срез без выделения новой памяти (то есть размер базового массива, начиная с позиции начала среза).

Срез не хранит сами данные, он указывает на подмножество массива. При расширении среза, если длина превышает ёмкость, выделяется новый массив большего размера, и данные копируются. Это позволяет эффективно управлять динамическими данными с контролем аллокаций.

#### Ключевые моменты
- **Массивы — fixed size:** размер известен на момент компиляции, располагаются в памяти подряд, не меняются. Их можно передавать по значению, что копирует все элементы.  
- **Срезы — динамические view:** срез содержит указатель, длину и ёмкость. Это lightweight структура, которая позволяет гибко манипулировать массивом без перемещения данных.  
- **Capacity (ёмкость):** важный параметр для оптимизации. Он показывает, сколько элементов можно добавить, не вызывая выделения новой памяти и копирования, что снижает накладные расходы и повышает производительность.

#### Практический контекст  
В Go, начиная с версии 1.15+, внутренняя реализация срезов оптимизирована для быстрой аллокации и копирования. При работе с большими срезами часто используют функции `make([]T, length, capacity)` для явного задания ёмкости, чтобы контролировать механизмы роста буфера и уменьшить количество выделений в куче. Это особенно важно в серверах с высоким throughput, где задержки аллокаций критичны.
