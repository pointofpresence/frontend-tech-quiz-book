# Что такое MVCC (Multiversion Concurrency Control)? Где используется?

**Что такое MVCC (Multiversion Concurrency Control)? Где используется?**

- Метод управления **конкурентным доступом** к БД
- Создаёт **множество версий** данных для чтения/записи
- Чтения не блокируют записи и наоборот
- Использует **снимки** состояния данных для транзакций
- Уменьшает **конфликты блокировок**, повышает параллелизм
- Основной метод в **PostgreSQL, Oracle, MySQL (InnoDB)**
- Идеально для высоконагруженных систем с **интенсивным чтением**

# Подробный ответ
## Основной ответ
**MVCC (Multiversion Concurrency Control)** — это метод управления параллелизмом в системах баз данных, позволяющий обеспечить согласованность данных при одновременном доступе множества транзакций без блокировок на чтение. Он основан на хранении нескольких версий одной и той же записи, чтобы чтения могли происходить из стабильной снимки данных, а записи — параллельно создавать новые версии.

## Ключевые моменты
- При **MVCC** каждая транзакция видит неизменяемый снимок данных на момент своего старта, что снижает конкуренцию читающих и пишущих транзакций и уменьшает блокировки.
- В качестве меток версий часто используются **timestamps** или уникальные идентификаторы транзакций (Transaction IDs), чтобы определять, какая версия записи подходит для текущей транзакции.
- Основные реализации MVCC есть в таких СУБД, как **PostgreSQL (начиная с версии 8.1)**, **Oracle**, **MySQL InnoDB** и **SQL Server snapshot isolation**.
- **Trade-off**: MVCC повышает производительность за счёт снижения блокировок, но требует дополнительного хранения и периодической очистки старых версий (garbage collection), что усложняет администрирование.

## Практический контекст
MVCC широко используется в **OLTP-системах с высокой конкуренцией** — там, где важно быстро выполнять больше транзакций без взаимных блокировок. В PostgreSQL 14+ это позволяет достичь стабильной пропускной способности при нагрузках в десятки тысяч запросов в секунду, минимизируя contention и сохраняя целостность данных.
