# Как в функциональных компонентах подписаться на события?

Категория: Программирование

### Краткий ответ

**Как в функциональных компонентах подписываться на события?**

- React functional components используют **useEffect** для подписки/отписки  
- Внутри useEffect добавляем слушатель события, например `window.addEventListener`  
- В функции очистки (return из useEffect) удаляем слушатель через `removeEventListener`  
- Такой подход предотвращает **утечки памяти** и избыточные подписки  
- useEffect с пустым массивом зависимостей `[]` — подписка один раз при монтировании  
- Для подписок на пропсы или состояние добавляем зависимости в массив useEffect  
- Пример:  
```jsx
useEffect(() => {
  const handleResize = () => console.log(window.innerWidth);
  window.addEventListener('resize', handleResize);
  return () => window.removeEventListener('resize', handleResize);
}, []);
```

Так обеспечивается **надежное управление жизненным циклом подписок** в функциональных компонентах React.

### Подробный ответ
#### Основной ответ  
В **функциональных компонентах** React для подписки на события чаще всего используется хук **useEffect**, который позволяет выполнить побочные эффекты, включая регистрацию и очистку обработчиков событий. При монтировании компонента внутри useEffect можно добавить слушателя события через `addEventListener`, а при размонтировании — убрать его через `removeEventListener`, чтобы избежать утечек памяти.  

Например, чтобы подписаться на событие `resize` окна:  
```jsx
useEffect(() => {
  const handleResize = () => { /* логика обработки */ };
  window.addEventListener('resize', handleResize);
  return () => {
    window.removeEventListener('resize', handleResize);
  };
}, []);
```
Пустой массив зависимостей (`[]`) означает, что подписка происходит один раз при монтировании и удаляется при размонтировании.  

#### Ключевые моменты  
- **useEffect** — стандартный способ для побочных эффектов, включая подписки на события и их очистку  
- Обязательно удалять подписчики в функции очистки, чтобы избежать утечек и дублирования при повторных рендерах  
- Если подписка зависит от пропсов или состояния, зависимости useEffect должны быть корректно указаны для актуализации handler’а  

#### Практический контекст  
В реальных проектах так подписываются на глобальные события браузера (scroll, resize, keyboard), события WebSocket или кастомные события глобального EventEmitter. В React 18+ это помогает избежать проблем с двойным вызовом эффектов в Strict Mode благодаря правильной очистке.
