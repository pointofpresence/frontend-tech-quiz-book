# Какие алгоритмы обхода дерева ты знаешь?

**Какие алгоритмы обхода дерева ты знаешь?**

- обходы дерева — классические алгоритмы структуры данных  
- **DFS (глубина):** preorder, inorder, postorder  
- preorder: узел → левый → правый  
- inorder: левый → узел → правый (важен для BST)  
- postorder: левый → правый → узел  
- **BFS (ширина):** уровень за уровнем (очередь)  
- применение: поиск, сортировка, сериализация, балансировка деревьев

# Подробный ответ
## Основной ответ
Для обхода дерева существует несколько классических алгоритмов, которые позволяют просматривать все узлы в определённом порядке. Основные типы обхода деревьев — это **обход в глубину** (DFS) и **обход в ширину** (BFS). В рамках обхода в глубину выделяют три основных способа: **прямой (pre-order)**, **центральный (in-order)** и **обратный (post-order)** обходы. Обход в ширину обычно реализуется через **поуровневый (level-order)** обход с помощью очереди.

## Ключевые моменты
- **Pre-order обход (прямой)**: посещаем сначала корень, затем рекурсивно — левое и правое поддерево. Часто используется для копирования дерева или сохранения его структуры.
- **In-order обход (центральный)**: сначала левое поддерево, затем корень, затем правое. Применяется в бинарных поисковых деревьях (BST), так как такой обход выдаёт отсортированный список.
- **Post-order обход (обратный)**: сначала левое и правое поддерево, затем корень. Полезен для удаления дерева или вычисления значения выражений в деревьях разбора.
- **Level-order обход** основан на очереди, посещает узлы по уровням сверху вниз, слева направо. Это BFS-обход, часто используется для поиска в ширину, определения глубины дерева и т.п.
- Рекурсивные решения наиболее наглядны, но для обхода в глубину можно применять и **стек** для обхода без рекурсии.

## Практический контекст
В реальных проектах, например, в React Virtual DOM дерево часто обходит pre-order для рендеринга. Для анализа синтаксических деревьев AST используется post-order — сначала вычисляются значения дочерних узлов. Level-order применяется в системах маршрутизации и графовых задачах с поиском наименьших расстояний. Важно уметь реализовывать и рекурсивный, и итеративный обходы с учетом требований по эффективности и стеку вызовов.
