# Что такое JWT и из каких частей он состоит?

Категория: Программирование

### Краткий ответ

**Что такое JWT и из каких частей он состоит?**  
- **JWT (JSON Web Token)** — стандартный формат токена для передачи информации в компактном, безопасном и самодостаточном виде  
- Состоит из трёх частей, разделённых точками:  
  - **Header (заголовок)** — содержит тип токена (JWT) и алгоритм подписи (например, HS256)  
  - **Payload (полезная нагрузка)** — содержит набор утверждений (claims), например, id пользователя, права, время истечения (exp)  
  - **Signature (подпись)** — криптографическая подпись Header и Payload с секретным ключом для проверки подлинности и целостности  
- Передаётся в виде base64url-кодированных строк: header.payload.signature  
- Используется для **аутентификации**, обмена данными между клиентом и сервером с гарантией безопасности и неизменности  
- Позволяет быть **безсессионным** (stateless), устраняя нагрузку на сервер по хранению состояния сессий  
- Популярно в REST API и микросервисах для быстрой проверки прав доступа без запроса к БД  
- Основное преимущество — **самодостаточность и проверяемость без обращения к серверу** при наличии секрета или публичного ключа для валидации подписи

### Подробный ответ
#### Основной ответ
**JWT (JSON Web Token)** — это компактный, URL-безопасный токен, который используется для передачи информации между сторонами в виде JSON-объекта. Он широко применяется для аутентификации и авторизации в распределённых системах, позволяя серверу удостовериться в подлинности клиента без необходимости хранить сессию на сервере.

JWT состоит из трёх частей, разделённых точками:

1. **Header (заголовок)** — содержит метаданные о токене, обычно тип токена (`typ: JWT`) и алгоритм подписи (`alg: HS256`, `RS256` и др.).
2. **Payload (полезная нагрузка)** — набор заявлений (claims) в формате JSON, таких как идентификатор пользователя (`sub`), срок жизни (`exp`), время выпуска (`iat`), а также любые кастомные данные.
3. **Signature (подпись)** — криптографическая подпись, сформированная на основе base64url-закодированного header и payload с использованием секретного ключа или приватного ключа для обеспечения целостности и аутентичности токена.

#### Ключевые моменты
- JWT являются **stateless** — серверу не нужно хранить состояние сессии, что улучшает масштабируемость.
- Стандарт определён в RFC 7519 и поддерживается большинством языков и библиотек, например, в Node.js — jsonwebtoken, в Java — jjwt.
- Безопасность зависит от защиты секретного ключа и правильного настройки алгоритма подписи; использование непроверенных или слабых алгоритмов — частая уязвимость.

#### Практический контекст
JWT широко используется в авторизации REST API: после успешного логина сервер выдаёт JWT, который клиент кладёт в заголовок Authorization (Bearer token) при последующих запросах. Такой подход снижает нагрузку на сервер и упрощает горизонтальное масштабирование.
