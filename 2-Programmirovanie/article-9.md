# Как хранить weak-ссылки на объекты (например, UIViewControllers) в массиве?

Категория: Программирование

### Краткий ответ

**Как хранить weak-ссылки в массиве (например, UIViewControllers)**  
- context: **iOS/Swift, память и ARC**  
- weak-ссылки **не хранятся прямо** в стандартных коллекциях, потому что они требуют типа-обертки  
- используют **обертку** с weak-свойством: класс/структура с `weak var value: ObjectType?`  
- создать массив таких оберток `[Weak<ObjectType>]` (например `[Weak<UIViewController>]`)  
- при доступе к элементам нужно **извлекать `.value`** и фильтровать nil, чтобы избежать висячих ссылок  
- обеспечивает **автоматическое освобождение** объекта из памяти без retain cycle  
- практическое применение — например, список делегатов или наблюдателей, когда важна слабая ссылка  

Итог: применяйте **обертки с weak свойством** для хранения слабых ссылок в массивах, так ARC корректно управляет жизненным циклом объектов без удержания.

### Подробный ответ
#### Основной ответ
В **Swift** для хранения слабых ссылок (weak references) на объекты, например на **UIViewController**, в массиве, нельзя просто использовать массив типа `[UIViewController?]` с модификатором `weak`, так как `weak` нельзя применять к элементам массива напрямую. Правильный способ — обернуть слабую ссылку в вспомогательную обертку-структуру или класс и хранить массив таких оберток.

#### Ключевые моменты
- **Обертка с weak-ссылкой**: создается класс `WeakWrapper` с единственным свойством `weak var value: UIViewController?`. В массиве хранится `[WeakWrapper]`.
- **Преобразование и очистка**: при доступе к элементам массива нужно фильтровать nil-ссылки, так как объекты могут быть удалены из памяти, и weak-ссылка станет nil.
- **Причина**: Swift не поддерживает `weak` для коллекций напрямую, потому что `weak` ссылается на экземпляры класса, а структуры (например, Array) копируют данные — это бы нарушило семантику weak.

#### Практический контекст
Часто применяется, например, в паттернах **delegate multicasting** или менеджерах навигации, где нужно отслеживать несколько контроллеров, не удерживая их навсегда и не создавая циклы ссылок. Так можно избежать утечек памяти и одновременно иметь быстрый доступ к живым объектам.

##### Пример кода (концептуально)

```swift
class WeakWrapper {
    weak var value: UIViewController?
    init(value: UIViewController) {
        self.value = value
    }
}

var weakControllers = [WeakWrapper]()
// Добавляем
weakControllers.append(WeakWrapper(value: someViewController))
// Фильтруем живые
let liveControllers = weakControllers.compactMap { $0.value }
```

Этот подход адаптируется для любых ссылочных объектов и является стандартной практикой в Swift 5+ и iOS разработке.
