# Что такое Data Race? Приведи пример

Категория: Программирование

### Краткий ответ

**Что такое Data Race?**

- проблема конкурентного **доступа к памяти**  
- несколько потоков **читают и пишут** общий ресурс без синхронизации  
- непредсказуемое поведение и **ошибки** (например, неправильные данные)  
- возникает из-за отсутствия **блокировок** или атомарных операций  
- пример: два потока одновременно увеличивают один счетчик без мьютекса  
- приводит к **нестабильности** и сложностям в отладке  
- решается через **мьютексы**, атомарные операции, или синхронизацию  

---

**Пример:**
```go
var counter int

func increment() {
    counter++  // неатомарно
}

// два потока вызывают increment() без блокировок
```

Здесь возможен Data Race, так как ++ неатомарна: чтение, увеличение, запись могут пересекаться.

### Подробный ответ
#### Основной ответ  
**Data race** — это ситуация в многопоточных программах, когда два или более потока одновременно обращаются к одной и той же памяти без надлежащей синхронизации, и при этом хотя бы один из потоков выполняет операцию записи. В результате возникает неопределённое поведение — программа может вести себя нестабильно или выдавать неверные данные. Data race часто приводит к трудноуловимым багам, так как проявляется лишь при определённых условиях гонки потоков.

#### Ключевые моменты  
- **Отсутствие синхронизации**: для предотвращения data race требуется использовать механизмы блокировок (mutex, semaphore) или атомарные операции.  
- **Непредсказуемость**: ошибки могут не проявляться всегда, что усложняет отладку. Даже если результат кажется правильным, data race остаётся серьёзной проблемой.  
- **Различие с deadlock**: data race — это конфликт доступа, а deadlock — это взаимная блокировка потоков, приводящая к зависанию.

#### Пример  
Представим два потока, которые одновременно увеличивают общий счётчик без синхронизации:

```c
int counter = 0;

void increment() {
    counter = counter + 1;
}
```

Если два потока вызовут `increment()` одновременно, то из-за операций чтения и записи значения `counter` итог может быть не 2, а 1 — один из инкрементов «потеряется». Чтобы исправить, нужно обернуть эту операцию в мьютекс или использовать атомарные типы (например, `std::atomic<int>` в C++11 и выше).

#### Практический контекст  
В современных многопоточных приложениях (Java concurrency, C++ с threading) контроль data race — ключевой аспект. Инструменты вроде ThreadSanitizer, Helgrind помогают обнаруживать data race во время тестирования. В Java используется `volatile`, `synchronized`, в C++ — std::mutex и атомарные операции для безопасного доступа к данным.
