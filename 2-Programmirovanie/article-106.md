# Как проверить, что переменная хранит массив?

Категория: Программирование

### Краткий ответ

**Как проверить, что переменная хранит массив?**

- контекст: проверка типов в JavaScript  
- основной способ: `Array.isArray(variable)` — возвращает true, если аргумент — массив  
- альтернатива: `variable instanceof Array` — работает, если нет разных фреймворков/окружений  
- можно проверить через `Object.prototype.toString.call(variable) === '[object Array]'` — универсальный способ  
- для объектов, похожих на массив, эти методы отличают настоящий массив  
- использование важно для корректной обработки данных и избежания ошибок  
- `Array.isArray` — наиболее надёжный и современный метод, рекомендован к применению  

Этот набор методов демонстрирует глубокое понимание различий между типами и типичными ловушками в JS.

### Подробный ответ
#### Основной ответ
В JavaScript для проверки, что переменная является массивом, рекомендуется использовать встроенный метод **Array.isArray()**. Он точно и надежно определяет, что объект — именно массив, а не просто похожая структура, например, объект с числовыми ключами.

#### Ключевые моменты
- **Array.isArray(value)** — возвращает `true`, если `value` действительно массив, и `false` в остальных случаях, включая объекты, null, undefined и т.д.
- Альтернативные способы, например, `instanceof Array`, работают корректно, но могут ошибаться при работе с массивами из разных execution contexts (например, в iframe), где прототипы не совпадают.
- Использовать `typeof` не имеет смысла, так как для массивов тип будет `"object"`, что не уникально.

#### Практический контекст
В React 18+ или Node.js 16+ коде почти всегда применяют **Array.isArray** для проверки входных данных, особенно при обработке props или API ответов. Это снижает баги, связанные с неправильным типом данных, и улучшает стабильность приложения.
