# Что такое record в C#? Какие преимущества они дают?

Категория: Программирование

### Краткий ответ

**Что такое record в C#? Преимущества**

- специфический тип данных C# с 9 версии  
- предназначены для **иммутабельных** моделей данных  
- поддерживают **сравнение по значению** (value equality), а не по ссылке  
- автоматически генерируют **конструкторы, свойства, методы Equals/GetHashCode и ToString**  
- легко создают **копии с изменениями** через встроенный метод `with`  
- улучшают чистоту и читаемость кода без лишнего шаблона  
- полезны для **DTO, immutable data, pattern matching и функционального стиля**

Итог: record — удобный способ работать с неизменяемыми объектами и делать безопасное сравнение по значениям при минимальном коде.

### Подробный ответ
#### Основной ответ
**Record** в C# — это специальный тип данных, введённый начиная с C# 9.0, который предназначен для представления **неизменяемых (immutable) объектов с семантикой сравнения по значению**. В отличие от обычных классов, рекорды имеют встроенную поддержку автоматического сравнения, копирования с изменениями и удобной дефолтной реализации для ToString().

#### Ключевые моменты
- **Неизменяемость:** рекорды по умолчанию создаются с readonly свойствами (init-only), что гарантирует неизменяемость состояния после инициализации. Это упрощает работу с безопасностью и многопоточностью.
- **Семантика сравнения по значению:** операторы Equals и GetHashCode в рекордах реализованы так, что сравнивают содержимое (значения всех свойств), а не ссылки, как в обычных классах. Это удобно для моделей данных и DTO.
- **Синтаксис сокращений:** с помощью параметризованного объявления `record Person(string Name, int Age);` можно быстро создать полноценный immutable тип с минимальным кодом.
- **Метод Clone с изменениями:** нельзя напрямую изменить поля, но можно создать копию объекта с изменёнными конкретными свойствами через метод `with`, например:  
  `var olderPerson = person with { Age = 30 };`

#### Практический контекст
В реальных проектах record активно применяются для описания **моделей данных в domain-driven design, DTO в API, а также для работы с неизменяемыми структурами при функциональном стиле программирования**. Например, в ASP.NET Core они упрощают сериализацию и позволяют избежать ошибок при передаче данных, сохраняя при этом удобство и безопасность.

Таким образом, рекорды в C# — мощный механизм для создания "value objects", уменьшающий шаблонный код и повышающий безопасность и читаемость.
