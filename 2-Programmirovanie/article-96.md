# Что такое utility types в TypeScript?

**Что такое utility types в TypeScript?**  
- встроенные **шаблонные типы**  
- позволяют **изменять и комбинировать** типы  
- улучшают повторное использование кода и типовую безопасность  
- примеры: **Partial** (делает свойства необязательными), **Readonly** (делает свойства только для чтения), **Pick** (выбирает подмножество свойств), **Record** (создает тип объекта с заданными ключами и значениями)  
- сокращают дублирование и повышают гибкость типизации  
- важны для масштабируемых крупных проектов и библиотек  
- применяются для создания более выразительного и строгого API типов

# Подробный ответ
## Основной ответ
**Utility types** в TypeScript — это встроенные обобщённые (generic) типы, которые позволяют трансформировать, изменять или комбинировать существующие типы для повышения гибкости и повторного использования кода. Они упрощают манипуляции с типами без необходимости писать собственные сложные конструкции. Utility types значительно улучшили разработку с TypeScript начиная с версии 2.1 и далее.

## Ключевые моменты
- **Основные utility types** включают `Partial<T>` (делает все свойства типа опциональными), `Readonly<T>` (делает все свойства неизменяемыми), `Pick<T, K>` (выбирает подмножество свойств), `Omit<T, K>` (исключает указанные свойства) и `Record<K, T>` (создаёт объект с фиксированными ключами и типами значений).
- Они используют преимущества **дженериков и условных типов**, что позволяет создавать продвинутые типовые конструкции без дублирования кода.
- Utility types помогают обеспечить **типобезопасность** и делают код более выразительным и поддерживаемым, особенно при работе с API, формами или сложными объектными структурами.

## Практический контекст
В реальных проектах часто применяют `Partial` для обновления сущностей (патчинг объектов), `Pick` для проецирования отдельной части интерфейса, а `Record` — для создания типизированных мэппингов, например, ключ-значение. Это уменьшает количество кода и снижает вероятность ошибок, связанных с несоответствием типов. Например, в React с TypeScript `Readonly` полезен при работе с пропсами, которые не должны изменяться внутри компонента.
