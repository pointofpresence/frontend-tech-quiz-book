# Если у нас 100 редюсеров, проверяет ли их все rootReducer или только до первого совпадения по типу?

Категория: Процессы

### Краткий ответ

**Обработка action в rootReducer с множеством редюсеров**

- Redux rootReducer — комбинация редюсеров через combineReducers  
- Каждый редюсер получает **все actions** независимо от типа  
- Все редюсеры обрабатываются **последовательно** при dispatch  
- Нет остановки после первого совпадения типа action  
- Каждый редюсер решает, изменять ли часть state на основе action.type  
- Такой подход обеспечивает **изолированность** и **прозрачность** изменений  
- Выгодно для сложных приложений с множеством независимых state-веток  

Итог: rootReducer всегда вызывает все редюсеры, они фильтруют action самостоятельно по типу — нет ранней остановки.

### Подробный ответ
#### Основной ответ
В Redux и подобных архитектурах с использованием **rootReducer**, когда приходит действие (*action*), все редьюсеры, которые объединены в rootReducer, получают это действие для обработки. То есть **все 100 редьюсеров будут вызваны**, а не только до первого совпадения по типу действия. Каждый редьюсер анализирует action.type и решает, обновлять ли он своё состояние или возвращать его без изменений.

#### Ключевые моменты
- **Комбинирование редьюсеров через combineReducers**: RootReducer — это обычно результат объединения нескольких редьюсеров с помощью функции `combineReducers`, где каждый редьюсер отвечает за часть состояния.
- **Одновременная обработка**: reducer-функции вызываются последовательно, но для каждого action, все редьюсеры получают возможность обработать это действие независимо друг от друга.
- **Фильтрация внутри редьюсеров**: обычно внутри каждого редьюсера пишется свитч или условие по `action.type`, и если тип не совпадает — состояние просто возвращается как есть без изменений.
- Такой подход обеспечивает **чистоту** и **предсказуемость**: никакой редьюсер не блокирует обработку других, и состояние обновляется по частям.

#### Практический контекст
Если у вас 100 редьюсеров — важно, чтобы они были «легковесными» и быстро определяли, нужно ли что-то делать при данном action. За счёт того, что Redux придерживается паттерна «все редьюсеры получают все действия», достигается предсказуемость состояния и легче отлаживать изменения. В крупных приложениях можно дополнительно использовать селекторы, мемоизацию и технологию code splitting для оптимизации.
