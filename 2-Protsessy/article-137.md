# Какие виды объявления функций всплывают (hoisting)?

**Какие виды объявления функций всплывают (hoisting)?**

- **Hoisting** — поднятие объявлений в JS
- Всплывают **Function Declarations** полностью
- Всплывают имена функций, тело доступно сразу
- **Function Expressions** (через var) — всплывает только имя переменной, значение undefined
- **Arrow functions** как Function Expressions не всплывают полностью
- let, const Function Expressions вообще не всплывают
- Используются для вызова функций до объявления в коде

# Подробный ответ
## Основной ответ
В JavaScript всплывают (hoisting) **объявления функций**, но при этом поведение различается в зависимости от способа объявления функции. Всплытие — это процесс, при котором объявления переменных и функций поднимаются в начало своей области видимости во время компиляции, позволяя обращаться к ним до того, как в коде текстуально проинициализируются.

Основные виды объявлений функций и их всплытие:

1. **Function Declaration** — классический способ:  
```js
function foo() { ... }
```  
Такие функции всплывают полностью, вместе с телом. Это значит, что функция доступна для вызова в любом месте своей области видимости, включая строки до объявления.

2. **Function Expression (с использованием var)** —  
```js
var foo = function() { ... };
```  
Всплывает только объявление переменной `foo` как `undefined`. Саму функцию присваивают уже на этапе выполнения кода. До этого `foo` существует, но как `undefined`, попытка вызвать ее — TypeError.

3. **Function Expression с let/const** —  
```js
let foo = function() { ... };
const foo = () => { ... };
```  
Переменные `foo` не доступны до инициализации (temporal dead zone), всплытия как такового нет.

## Ключевые моменты
- **Function Declaration** всплывает целиком — можно вызывать функцию до её объявления в тексте кода.  
- **Function Expression с var** всплывает как переменная с `undefined`, нельзя вызывать до инициализации.  
- **Function Expression с let/const** не всплывает и вызов до объявления приводит к ReferenceError.  
- В ES6 Arrow Functions — это синтаксический сахар для Function Expressions, поведение при hoisting такое же, как у let/const.  

## Практический контекст
Это важно учитывать при организации кода — чтобы не сталкиваться с ошибками вызова функций до их определения. В крупных проектах и в React 18+ рекомендуют использовать `const` с arrow functions для улучшения явности и предотвращения ошибок, связанных с hoisting.
