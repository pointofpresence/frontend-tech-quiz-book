# Что такое HTTP и HTTPS?

Категория: Процессы

### Краткий ответ

**Что такое HTTP и HTTPS?**

- HTTP — протокол **передачи** данных в вебе  
- Работает по схеме **запрос-ответ** между клиентом и сервером  
- Передача в HTTP — **открытый текст**, уязвим для прослушки  
- HTTPS — HTTP поверх **TLS/SSL**  
- Обеспечивает **шифрование** и **аутентификацию** сервера  
- Защищает от MITM-атак и подделки данных  
- HTTPS — стандарт для безопасных приложений, e-commerce, логинов

### Подробный ответ
#### Основной ответ
**HTTP (HyperText Transfer Protocol)** — это протокол прикладного уровня для передачи гипертекстовых данных в сети, благодаря которому работает веб. **HTTPS (HTTP Secure)** — это расширение HTTP с добавлением шифрования через TLS/SSL, обеспечивающее защищённый обмен данными между клиентом и сервером.

#### Ключевые моменты
- **HTTP** работает поверх TCP и передаёт данные в открытом виде, что делает его уязвимым к перехвату и атакам типа "man-in-the-middle".
- **HTTPS** использует TLS (Transport Layer Security) для шифрования каналов связи, что гарантирует конфиденциальность, целостность и аутентификацию сайта.
- В HTTPS применяется асимметричная криптография для установления безопасного соединения и последующего обмена симметричными ключами, что балансирует безопасность и производительность.
- Переключение с HTTP на HTTPS часто сопровождается использованием стандартного порта 443 вместо порта 80 и помогает улучшить доверие пользователей и SEO.

#### Практический контекст
В современных веб-приложениях HTTPS — обязательный стандарт для защиты пользовательских данных (паролей, платежей). Браузеры, начиная с Chrome 68+, помечают HTTP-сайты как небезопасные, поэтому все крупные проекты и сервисы используют HTTPS с сертификатами от проверенных центров выдачи (CA), например, Let’s Encrypt.
