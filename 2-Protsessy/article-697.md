# Для чего используются маркеры тестов?

Категория: Процессы

### Краткий ответ

**Для чего используются маркеры тестов?**

- организационная метка для тестов  
- группируют тесты по категориям (например, slow, smoke)  
- позволяют избирательно запускать тесты по признаку  
- упрощают управление большими наборами тестов  
- помогают исключать или приоритизировать тесты в CI/CD  
- интегрируются с тест-раннерами (pytest, JUnit)  
- повышают гибкость и скорость тестирования при регрессии

### Подробный ответ
#### Основной ответ
**Маркеры тестов** (test markers или test tags) используются для категоризации и управления тестовыми сценариями в автоматизированном тестировании. Они позволяют группировать тесты по определённым характеристикам — например, по типу (юнит, интеграционный, end-to-end), приоритету, функциональному модулю или условиям запуска (например, smoke, regression). Это значительно облегчает выборку и запуск нужного набора тестов в зависимости от контекста.

#### Ключевые моменты
- **Гибкость запуска**: с помощью маркеров можно запускать только определённые группы тестов, например, быстро выполняемые или критичные, что ускоряет цикл тестирования и экономит ресурсы.
- **Организация и документация**: маркеры помогают структурировать большой набор тестов, делая тестовый набор более понятным и поддерживаемым.
- **Интеграция с CI/CD**: в Jenkins, GitLab CI и других системах маркеры позволяют автоматически подставлять нужные фильтры для тестов, оптимизируя процесс развертывания и проверки качества.

#### Практический контекст
Типично в **JUnit 5** используются аннотации `@Tag`, в **pytest** — маркеры `@pytest.mark`. Например, в проекте на Python можно запускать только регрессионные тесты командой `pytest -m regression`, что позволяет быстро реагировать на критичные баги и экономить время при релизах.
