# Что такое индексы и как они работают?

Категория: Процессы

### Краткий ответ

**Что такое индексы и как они работают?**

- индексы — структура данных в БД  
- ускоряют поиск **и выборку** данных  
- обычно реализованы через **B-деревья** или **хеш-таблицы**  
- поддерживают быстрый доступ по столбцам (ключам)  
- уменьшают полные сканирования таблиц (full table scan)  
- пишутся с дополнительными затратами на обновление  
- критичны для оптимизации запросов и масштабируемости системы

### Подробный ответ
#### Основной ответ
**Индексы** — это структуры данных, которые используются в реляционных базах данных для ускорения поиска и выборки данных. По сути, индекс — это отдельная оптимизированная таблица, которая содержит ключевые значения из основного столбца и ссылки на строки в основной таблице. Благодаря индексам при выполнении запросов СУБД не приходится делать полное сканирование таблицы, что значительно снижает время отклика.

#### Ключевые моменты
- **Типы индексов**: наиболее распространённые — B-деревья (B-tree) для упорядоченного поиска по диапазону и хеш-индексы для точных совпадений (например, в PostgreSQL используются B-tree по умолчанию).
- **Механика работы**: при запросе с условием на индексируемый столбец СУБД быстро находит соответствующие значения в индексе и сразу переходит к нужным строкам таблицы, минуя остальные.
- **Trade-off**: индексы ускоряют чтение, но замедляют операции вставки, обновления и удаления, так как при изменениях надо обновлять и индексные структуры. Также индексы занимают дополнительное дисковое пространство.

#### Практический контекст
В реальных проектах индексы критически важны для быстрого реагирования на запросы с фильтрацией или сортировкой, особенно на больших таблицах с миллионами записей. Например, в PostgreSQL 14+ используем B-tree индексы для Primary Key, а для полнотекстового поиска — GIN или GiST индексы. Управление индексами и их грамотный выбор — важная часть оптимизации производительности сложных систем.
