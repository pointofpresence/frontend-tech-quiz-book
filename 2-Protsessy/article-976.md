# В чем разница между структурами и классами?

**Разница между структурами и классами**

- структуры — тип значения (value type), классы — тип ссылки (reference type)  
- структуры хранят данные непосредственно, классы — в куче, переменные ссылаются на объекты  
- структуры не поддерживают наследование, классы — поддерживают  
- структуры передаются по копии, классы — по ссылке  
- структуры не имеют деструкторов, классы могут иметь  
- структуры обычно легче и быстрее, подходят для небольших данных  
- классы лучше для сложных объектов с поведением и состоянием  

Основное: **структуры — легковесные значимые типы, классы — полноценные объекты с наследованием и ссылочной семантикой**.

# Подробный ответ
## Основной ответ
**Структуры (struct)** и **классы (class)** — это оба пользовательские типы данных в языках вроде C++ и C#, предназначенные для хранения набора связанных данных и поведения. Основное различие заключается в том, что **структуры являются типами значения**, а **классы — ссылочными типами**. Это влияет на поведение при присваивании, передаче параметров и управлении памятью.

## Ключевые моменты
- **Тип значения vs ссылка**: Структура хранится непосредственно в стеке (или inline), при копировании создаётся полная копия данных. Класс — в куче, переменная содержит ссылку на объект, что значит копируются ссылки, а не сами данные.
- **Наследование**: Классы поддерживают полноценное наследование и полиморфизм (виртуальные методы, интерфейсы). Структуры обычно не поддерживают наследование (в C# нет возможности наследоваться от другой структуры).
- **Управление памятью и производительность**: Структуры эффективны для небольших, часто создаваемых объектов, т.к. нет накладных расходов на сборку мусора. Классы удобнее для сложных объектов с долгим жизненным циклом и большим количеством методов.
- **Инициализация**: В C# структуры не могут иметь безпараметрические конструкторы (до версии C# 10), но классы могут. Также у структуры всегда есть неявный конструктор по умолчанию.

## Практический контекст
В проектах структуры часто применяются для lightweight DTO, геометрических примитивов и других маленьких значений с коротким временем жизни. Классы — для сложных моделей с богатой логикой, требующих наследования и управления состоянием. Например, в Unity **Vector3** — структура для координат из-за частого создания и копирования, а игровые объекты — классы, так как имеют сложную иерархию и управление памятью через движок.
