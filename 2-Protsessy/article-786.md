# Что знаешь про ORM?

Категория: Процессы

### Краткий ответ

**Что такое ORM?**

- Абстракция работы с БД через объекты  
- Автоматическое преобразование объектов в SQL-запросы  
- Позволяет работать с данными на уровне языка программирования  
- Снижает количество шаблонного кода и ошибок SQL  
- Поддерживает разные СУБД через общий интерфейс  
- Может влиять на производительность из-за генерации SQL  
- Используется для ускорения разработки и упрощения поддержки кода

### Подробный ответ
#### Основной ответ
**ORM (Object-Relational Mapping)** — это технология, которая позволяет разработчикам работать с базой данных через объекты и классы программного языка, вместо прямого написания SQL-запросов. Основная задача ORM — абстрагировать реляционные таблицы и связи в базе данных в виде привычных объектных моделей, что упрощает разработку и повышает читаемость кода.

#### Ключевые моменты
- **Автоматизация CRUD операций**: ORM генерирует SQL для создания, чтения, обновления и удаления записей, избавляя от рутинного написания SQL вручную.
- **Связи объектов и таблиц**: поддержка различных типов связей — один к одному, один ко многим, многие ко многим через декларацию в моделях.
- **Отложенная загрузка (lazy loading) и жадная загрузка (eager loading)**: позволяют эффективно управлять выборкой связанных сущностей, избегая избыточных запросов.
- **Trade-offs**: ORM ускоряет разработку и делает код чистым, но может вызывать проблемы с производительностью на сложных запросах, где избыточный сгенерированный SQL медленнее ручного.
- Современные версии, например, **Entity Framework Core 7** или **Hibernate 6+, TypeORM 0.3+**, дополняют традиционные возможности удобными функциями для миграций, кэширования и интеграции с LINQ или Criteria API.

#### Практический контекст
В реальных проектах ORM часто используются для быстрого старта и поддержки кода, особенно при работе со CRUD-приложениями. Однако в высоконагруженных системах критичных к производительности части запросов пишутся вручную, обходя ORM, чтобы достичь максимальной оптимизации. Например, в React + .NET Core стек часто применяют Entity Framework для низкоуровневой работы с данными и миграциями, а для сложных аналитических запросов пишут raw SQL или используют Dapper как микрофреймворк.
