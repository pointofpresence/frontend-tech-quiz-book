# Какие есть типы соединений (JOIN) в PostgreSQL?

Категория: Процессы

### Краткий ответ

**Типы соединений (JOIN) в PostgreSQL**

- базовый SQL: объединение таблиц по условию
- **INNER JOIN**: только совпадающие строки обеих таблиц
- **LEFT JOIN (LEFT OUTER JOIN)**: все строки слева + совпадающие справа
- **RIGHT JOIN (RIGHT OUTER JOIN)**: все строки справа + совпадающие слева
- **FULL JOIN (FULL OUTER JOIN)**: все строки обеих таблиц, NULL если нет совпадения
- **CROSS JOIN**: декартово произведение без условия
- **SELF JOIN**: соединение таблицы самой с собой с псевдонимами
- практическое применение: выборка связанных данных с разной степенью включения  

Основной смысл — выбирать, какие строки из таблиц включать при объединении в зависимости от бизнес-логики.

### Подробный ответ
#### Основной ответ
В PostgreSQL доступны основные типы **JOIN** для объединения данных из двух и более таблиц по заданным условиям. Среди них: **INNER JOIN**, **LEFT (OUTER) JOIN**, **RIGHT (OUTER) JOIN**, **FULL (OUTER) JOIN**, а также **CROSS JOIN** и **LATERAL JOIN**. Каждый тип обслуживает разные сценарии выборки данных в зависимости от требований к результату — включать ли непересекающиеся строки или только совпадающие.

#### Ключевые моменты
- **INNER JOIN** — возвращает строки, где есть совпадение по условию в обеих таблицах, исключая несопоставленные записи. Это наиболее часто используемый тип join.
- **LEFT JOIN (LEFT OUTER JOIN)** — возвращает все строки из левой таблицы и совпадающие из правой. Если совпадений нет, поля правой таблицы будут заполнены NULL.
- **RIGHT JOIN (RIGHT OUTER JOIN)** — зеркальный к LEFT JOIN: все строки из правой таблицы с совпадениями из левой или NULL там, где совпадений нет.
- **FULL JOIN (FULL OUTER JOIN)** — объединяет LEFT и RIGHT JOIN, возвращая все строки из обеих таблиц, заполняя NULL там, где нет совпадений.
- **CROSS JOIN** — декартово произведение, возвращает все возможные пары строк из обеих таблиц без условий.
- **LATERAL JOIN** — позволяет использовать данные из левой таблицы в подзапросе правой (например, для вызова функции на каждой строке), появилось в PostgreSQL 9.3+.

#### Практический контекст
В реальных проектах обычно используют **INNER JOIN** для выборки связанных данных, **LEFT JOIN** — чтобы получить все записи с добавлением данных, если они есть, например, для отчётов или dashboard’ов. **FULL JOIN** редок, чаще применим для сравнения двух наборов данных с выявлением несоответствий. **LATERAL JOIN** полезен при работе с массивами, JSON или функциями, возвращающими наборы строк, повышая выразительность запросов. В PostgreSQL 14+ оптимизации join значительно улучшили производительность сложных схем с множественными соединениями.
