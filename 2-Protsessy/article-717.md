# Как запустить поток (Thread)? Почему вызов run() напрямую не создаёт новый поток?

Категория: Процессы

### Краткий ответ

**Как запустить поток (Thread)? Почему вызов run() напрямую не создаёт новый поток?**

- Запуск потока: вызываем метод **start()** у объекта Thread  
- **start()** создаёт и запускает новый системный поток ОС  
- Вызов **run()** — это просто обычный метод, выполняется в текущем потоке  
- **run() не создаёт новый поток**, а выполняет код последовательно  
- **start()** инициализирует внутренние ресурсы, планировщик ОС начинает выполнение параллельно  
- Правильный запуск нужен для реализации **конкурентности** и распараллеливания  
- Вызов run напрямую — распространённая ошибка, приводит к **синхронному выполнению** вместо асинхронного  

**Итог:** Только через **start()** создаётся реальный параллелизм, вызов **run()** — обычное последовательное выполнение кода.

### Подробный ответ
#### Основной ответ  
Для запуска нового потока в Java нужно создать объект класса, реализующего интерфейс **Runnable** или наследующего **Thread**, и затем вызвать метод **start()**. Метод `start()` инициализирует новый поток операционной системы и вызывает внутри себя метод `run()`. Важно понимать, что вызов `run()` напрямую не создаёт новый поток, а просто выполняет метод `run()` в текущем потоке как обычную функцию.

#### Ключевые моменты  
- **Метод start()** — создает новую нить OS, и уже в ней вызывается `run()`. Без вызова `start()` поток не стартует параллельно, мы просто выполняем код последовательно.  
- **Вызов run() напрямую** — это просто обычный вызов метода, не создающий распараллеливание и не создающий новый стек выполнения. Всё выполняется в контексте текущего потока.  
- Это важное отличие нужно учитывать, чтобы обеспечить реальную многопоточность и избежать ошибок, когда программист ожидает параллельное выполнение, но на самом деле получает последовательное.

#### Практический контекст  
В современных Java-приложениях (например, на Java 11+) распараллеливание стартуют именно через `start()`, а `run()` используется для определения логики задачи. Часто для удобства используют **ExecutorService**, который управляет потоками и запускает задачи, но под капотом именно стартуем потоки или переиспользуем их. Понимание, как именно `start()` и `run()` работают, критично для правильной работы и отладки многопоточных приложений.
