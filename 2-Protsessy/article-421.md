# В чем разница между inline, block и inline-block элементами в CSS?

Категория: Процессы

### Краткий ответ

**Разница между inline, block и inline-block элементами в CSS**

- Контекст отображения: **inline** — в строке, **block** — на новой строке, занимает всю ширину  
- Размеры: у **inline** нельзя задавать ширину/высоту, у **block** и **inline-block** можно  
- Расположение: **inline-block** сочетает особенности inline (в строке) и block (размеры, отступы)  
- Отступы: у **inline** вертикальные margin/padding влияют слабо, у остальных полноценно  
- Использование:  
  - **inline** — тексты, ссылки  
  - **block** — параграфы, div, секции  
  - **inline-block** — кнопки, элементы меню, когда нужно inline с контролем размеров  
- Влияние на поток:  
  - **block** создает новый блок, переносит содержимое на новую строку  
  - **inline** и **inline-block** не разрывают строку, располагаются по горизонтали  
- Основной смысл: **inline-block** подходит для гибкой компоновки без разрыва строки, в отличие от block и с возможностью задания размеров в отличие от inline  

Это ключевые технические отличия и сферы применения для правильного выбора типа отображения элемента.

### Подробный ответ
#### Основной ответ
В CSS **inline**, **block** и **inline-block** — это три основные модели отображения элементов, которые определяют, как они ведут себя в потоке документа, занимают пространство и взаимодействуют с соседними элементами.  
**Inline** элементы располагаются в строке с другими элементами, не создавая разрыва строки.  
**Block** элементы занимают всю доступную ширину контейнера, начиная с новой строки.  
**Inline-block** — гибрид: располагаются как inline, но имеют свойства block, включая возможность задавать ширину и высоту.

#### Ключевые моменты
- **Inline** (например, `<span>`, `<a>`): не создают перенос строки, нельзя задать ширину и высоту, учитывают только ширину содержимого. Отступы по горизонтали работают, вертикальные часто игнорируются.
- **Block** (например, `<div>`, `<p>`): всегда начинаются с новой строки, занимаются полной шириной родителя по умолчанию, можно задать ширину/высоту, margin и padding работают полноценно.
- **Inline-block**: ведут себя как inline (располагаются в строке), но позволяют задавать размеры, margin и padding сработают корректно, удобно для создания горизонтальных списков или кнопок с фиксированными размерами.

#### Практический контекст
В React 18 или современных фреймворках типично использовать **inline-block** для элементов, требующих выравнивания по строке с контролируемыми размерами, например, иконок рядом с текстом. **Block** лучше подходит для структурных элементов, формирующих каркас страницы. Понимание этих поведений критично для верстки адаптивных интерфейсов и устранения багов с layout.
