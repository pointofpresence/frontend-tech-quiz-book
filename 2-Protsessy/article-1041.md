# Что такое первичный (PRIMARY KEY) и внешний (FOREIGN KEY) ключи?

**Что такое первичный (PRIMARY KEY) и внешний (FOREIGN KEY) ключи?**

- Реляционные базы данных, ключи  
- **PRIMARY KEY** — уникальный идентификатор строки  
- Обеспечивает **уникальность** и целостность данных  
- Включает один или несколько столбцов, не допускает NULL  
- **FOREIGN KEY** — ссылка на PRIMARY KEY другой таблицы  
- Обеспечивает **связь** и **ссылочную целостность** между таблицами  
- Предотвращает несогласованность данных при удалении/обновлении  

Итог: PRIMARY KEY — уникальность записи; FOREIGN KEY — связь таблиц и целостность данных.

# Подробный ответ
## Основной ответ
**Первичный ключ (PRIMARY KEY)** — это уникальный идентификатор каждой записи в таблице базы данных, который гарантирует отсутствие дубликатов и отсутствия NULL-значений. **Внешний ключ (FOREIGN KEY)** — это поле или набор полей, которые ссылаются на первичный ключ другой таблицы, обеспечивая целостность связей между таблицами и поддерживая референциальную целостность данных.

## Ключевые моменты
- **Первичный ключ** однозначно идентифицирует запись внутри таблицы, и обычно индексируется для быстрого поиска, например, в PostgreSQL 14+ автоматически создаётся уникальный индекс.
- **Внешний ключ** связывает данные из одной таблицы с данными в другой, предотвращая вставку несоответствующих значений и позволяя реализовывать каскадные операции (CASCADE DELETE/UPDATE).
- Эти ключи вместе обеспечивают структуру реляционной базы данных, минимизируют избыточность данных и поддерживают логические связи, что критично в сложных проектах с множественными связанными сущностями.

## Практический контекст
В реальных проектах первичные ключи применяются для идентификации пользователей, заказов или других сущностей, а внешние ключи — для связи, например, заказов с пользователями. Стандартный подход — использовать автоинкрементный столбец или UUID для PRIMARY KEY, а FOREIGN KEY — для жёсткого контроля целостности связей между таблицами.
