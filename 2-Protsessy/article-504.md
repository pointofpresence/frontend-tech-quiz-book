# В чём разница между <? extends T> и <? super T>?

Категория: Процессы

### Краткий ответ

**Разница между <? extends T> и <? super T>**  
- область дженериков в Java  
- <? extends T> — ограничение сверху, принимает T и его подтипы  
- позволяет безопасно **читать** значения, но нельзя их добавлять  
- <? super T> — ограничение снизу, принимает T и его суперклассы  
- позволяет безопасно **добавлять** объекты типа T, но для чтения тип Object  
- применение: **PECS** (Producer Extends, Consumer Super) — производитель использует extends, потребитель — super  
- помогает управлять ковариантностью и контравариантностью коллекций в Java Generics

### Подробный ответ
#### Основной ответ
В Java **`<? extends T>`** и **`<? super T>`** — это два варианта wildcard-обобщений, определяющие, какие типы можно использовать в generic-контейнерах с ограничениями по наследованию. `<? extends T>` ограничивает тип сверху, разрешая использовать T и любые его подклассы, тогда как `<? super T>` ограничивает тип снизу, позволяя использовать T и любые его суперклассы.

#### Ключевые моменты
- **`<? extends T>` ("продолжается с T")** — говорит компилятору, что мы работаем с неизвестным типом, который либо T, либо дочерний класс T. Это полезно когда нам нужен **только чтение** из контейнера: мы можем быть уверены, что элементы безопасно будут типа T или ниже. Но добавлять в такой контейнер элементы нельзя (кроме `null`), так как точный тип неизвестен.
- **`<? super T>` ("родительский класс для T")** — обозначает неизвестный тип, который либо T, либо любой из его предков. Это полезно для **записи**: мы можем смело добавлять объекты типа T, потому что нижняя граница гарантирует совместимость. При чтении результата — объект будет иметь тип `Object` (наименьшее общее), так как точный тип неизвестен.
- Классическая формулировка: «Producer extends, Consumer super» (PECS, Joshua Bloch). Если источник данных (происхождение объектов) — используем `extends`, если приемник объектов — используем `super`.

#### Практический контекст
В API Java Collections чаще `<? extends T>` применяется для методов, которые возвращают данные (например, чтение из списка для безопасного использования подклассов), а `<? super T>` — для добавления элементов (например, добавление различных реализаций T в коллекцию). Например, метод `Collections.copy()` использует `<? super T>` для destination, и `<? extends T>` для source, чтобы гарантировать корректность операций копирования.
