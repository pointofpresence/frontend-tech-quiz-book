# Что такое фрагментация индекса и как она влияет на производительность?

Категория: Процессы

### Краткий ответ

**Что такое фрагментация индекса и как она влияет на производительность?**  
- Индекс: структура для быстрого поиска в БД  
- Фрагментация: разброс данных индекса на диске  
- Виды: внутренняя (пустоты в страницах), внешняя (неупорядоченный порядок страниц)  
- Приводит к: увеличению количества операций чтения, снижению ефективности кеша  
- Замедляет выполнение запросов и операции вставки/обновления  
- Противодействие: реорганизация или перестройка индексa  
- Важна для поддержания высокой скорости доступа к данным и ресурсов БД

### Подробный ответ
#### Основной ответ
**Фрагментация индекса** — это состояние индекса в базе данных, при котором логическая последовательность ключей индекса расходится с физическим расположением данных на диске. Проще говоря, данные или страницы индекса разбросаны по диску, что приводит к неэффективному чтению. Это ухудшает производительность запросов, особенно при сканировании индекса, поскольку увеличивается количество операций ввода-вывода и снижается кэширование.

#### Ключевые моменты
- **Два типа фрагментации**: внутренняя (незаполненность страниц индекса) и внешняя (физический разрыв порядка страниц). Внутренняя снижает плотность данных на страницах, внешняя увеличивает количество случайных дисковых обращений.
- При фрагментации растёт **latency** на выборки, особенно для больших таблиц, где последовательный доступ критичен (например, даже на SSD случайный доступ дороже последовательного).
- Производительность ухудшается не только при чтении, но и при обновлении индекса — операции вставки/удаления приводят к частым перестроениям, снижающим пропускную способность.

#### Практический контекст
В PostgreSQL 14+ или Microsoft SQL Server фрагментацию обычно устраняют через **REINDEX** или **ALTER INDEX REBUILD**. В MySQL с InnoDB оптимальным может быть **OPTIMIZE TABLE**. Для крупных систем с высокими SLA важно регулярно мониторить фрагментацию — например, использовать **pg_stat_user_indexes** или DMV в SQL Server — и применять перестроение индекса в окне пониженной нагрузки для сохранения скорости запросов и снижения IO.
