# Для чего используется memo в React?

**Для чего используется memo в React?**

- React.memo — HOC для **оптимизации рендеринга**
- кэширует **результат** рендера компонента
- сравнивает пропсы **по умолчанию поверхностно**
- предотвращает **лишние рендеры** при одинаковых пропсах
- улучшает производительность для **функциональных компонентов**
- полезно при **часто обновляющемся UI** с неизменными пропсами
- снижает нагрузку на **виртуальный DOM** и рендеринг браузера

# Подробный ответ
## Основной ответ  
**`memo` в React** — это HOC (Higher Order Component), который используется для оптимизации производительности функциональных компонентов. Он предотвращает их повторный рендеринг, если props не изменились, что уменьшает ненужные перерисовки и снижает нагрузку на UI.

## Ключевые моменты  
- **Поведение**: `React.memo` сравнивает старые и новые props поверхностно (shallow comparison), и если они идентичны, React пропускает ререндер компонента. Это существенно повышает производительность при частых обновлениях родителя.  
- **User-provided сравнение**: можно указать свою функцию сравнения props, если нужны более глубокие проверки или специфические условия, что актуально для сложных объектов.  
- **Ограничения**: `memo` не защищает от повторных рендеров из-за внутренних состояний или контекста, а только по props. Также избыточное использование вместе с очень маленькими компонентами может привести к обратному эффекту из-за накладных затрат на сравнение.

## Практический контекст  
В React 18+ `memo` широко применяется для оптимизации списка элементов, компонентов с тяжелыми вычислениями и UI-компонентов, которые получают неизменяемые props. В больших приложениях с frequent re-renders использование `memo` помогает достичь плавных анимаций и снизить задержки до ~50 мс, улучшая user experience без глобального мемоизирования state.
