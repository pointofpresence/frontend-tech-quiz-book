# Как работает сборщик мусора в Go и что такое stop-the-world пауза?

**Как работает сборщик мусора в Go и что такое stop-the-world пауза?**

- Go использует **параллельный, конкоррентный GC**  
- Основной алгоритм — **три фазы**: маркировка (tracing), приостановка мира, очистка  
- Во время **маркировки** GC идентифицирует достижимые объекты  
- **Stop-the-world (STW) пауза** — момент, когда все goroutine приостанавливаются  
- STW пауза нужна для сохранения целостности данных и корректного обхода корней  
- Современный GC в Go уменьшает длительность STW, делая её почти незаметной (милисекунды)  
- Задача GC — **освобождение неиспользуемой памяти** без существенного влияния на производительность  

---

### Развёрнутый ответ для интервью

Сборщик мусора в Go реализован как **конкуррентный три-фазный GC**, который стремится минимизировать паузы программ. Основные фазы:

1. **Маркировка (Marking):** GC параллельно с работой приложения "маркирует" все достижимые объекты, начиная с корней (глобальные переменные, стеки goroutine). Для точности сборщик приостанавливает все goroutine – это и есть **stop-the-world пауза**. Во время STW происходит быстрое сканирование корней для фиксации актуальных ссылок. Паузу стараются минимизировать, обычно до нескольких миллисекунд.

2. **Очистка (Sweeping):** После маркировки GC освобождает память от немаркированных (недостижимых) объектов. Эта фаза тоже происходит параллельно с приложением.

3. **Конкуррентность:** Основная идея GC Go — работать максимально параллельно с выполнением программы, ограничивая STW паузы только критическими этапами.

**Stop-the-world пауза** — это кратковременная остановка всех goroutine для обеспечения консистентных данных GC, без которой результат маркера может быть некорректным. В новых версиях Go паузы значительно сокращены за счёт эвристик и оптимизаций.

Таким образом, Go GC сочетает эффектную **автоматическую очистку памяти** с низкими накладными потерями, что критично для серверных и высоконагруженных приложений.

# Подробный ответ
## Основной ответ
В Go используется **конкурентный, трёхфазный сборщик мусора с пометкой и очисткой (mark-and-sweep)**, который проектировался для минимизации задержек в многопоточных программах. Это значит, что большая часть работы сборщика выполняется параллельно с программой, но в определённые моменты все горутины приостанавливаются — это и есть **stop-the-world (STW) пауза**.

## Ключевые моменты
- **Трёхфазный алгоритм**: сначала происходит **сбор корней** (root scanning), затем **пометка достижимых объектов (marking)** с возможностью параллельной работы, и в конце — **удаление неиспользуемых объектов (sweeping)**.
- **Stop-the-world пауза** — это фазы, когда все горутины паузятся для снятия консистентного "снимка" памяти, например, во время initial root scan. В Go 1.15+ эти паузы значительно оптимизированы и обычно занимают менее 1мс на миллионы объектов.
- **Цель STW пауз** — обеспечить корректность и непротиворечивость данных для сборщика, снижая задержки для конечного пользователя. Go старается минимизировать время и частоту таких пауз, делая сборку максимально конкурентной.

## Практический контекст
В реальных Go-приложениях (beyond Go 1.11) сборщик мусора запускается автоматически, следит за нагрузкой памяти и паузами (через runtime metrics), обычно удерживая **GC pause в районе сотен микросекунд**. Инженеры могут настраивать параметр GOGC для контроля частоты срабатываний, балансируя использование памяти и латентность. Например, в latency-critical сервисах используют значения GOGC около 100 или меньше, снижая паузы, но увеличивая потребление памяти.
