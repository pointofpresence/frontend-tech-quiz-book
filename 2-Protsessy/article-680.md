# Соберёт ли сборщик мусора два объекта с циклической ссылкой при потере внешних ссылок?

Категория: Процессы

### Краткий ответ

**Соберёт ли сборщик мусора два объекта с циклической ссылкой при потере внешних ссылок?**

- касается работы **сборщика мусора** (Garbage Collector, GC)
- современные GC обычно реализуют **алгоритмы трассировки (mark-and-sweep)**
- такие алгоритмы **отмечают объекты**, достижимые из корней (root references)
- объекты с циклическими ссылками, но **без внешних ссылок**, не достижимы
- значит, GC **соберёт** эти циклично связанные объекты
- важно: в системах с подсчётом ссылок (reference counting) без трассировки циклы могут **не собираться**
- практическое значение — современные среды (Java, .NET, V8) эффективно очищают циклы без утечек

---

**Итог:** сборщик мусора с трассировкой устранит цикл, если объекты не доступны извне.

### Подробный ответ
#### Основной ответ
Да, большинство современных сборщиков мусора (GC) способны корректно обнаруживать и собирать объекты, которые образуют циклические ссылки, если на них нет внешних ссылок из корневых объектов (roots). Это связано с тем, что сборщики используют алгоритмы, выходящие за рамки простого подсчёта ссылок — например, трассировку по графу достижимости.

#### Ключевые моменты
- **Подсчёт ссылок не работает с циклами**: традиционный reference counting не может собрать циклически связанные объекты, так как каждый объект внутри цикла продолжает ссылаться друг на друга, и их счётчики не обнуляются.
- **Трассировочные алгоритмы**: такие как Mark-and-Sweep или Generational GC (например, в JVM, .NET, Python с gc-модулем) начинают от корневых объектов и «маркируют» все достижимые объекты. Объекты из цикла, не достижимые с корней, считаются мусором и очищаются.
- **Реализации**: современный сборщик в Java (HotSpot, начиная с Java 6 и далее) или в Python через модуль gc применяют подобные механизмы для циклов. В отличие от этого, в старых версиях CPython без gc-модуля циклы не очищались.

#### Практический контекст
В реальных проектах циклы широко встречаются (например, контейнеры с взаимными ссылками или callback-и с замыканиями). Надёжный сборщик особенно важен для предотвращения leaks. Например, в JavaScript V8 (по состоянию на версиях с 2015 г.) используется инкрементальный Mark-and-Sweep, эффективно собирающий циклы при отсутствии внешних ссылок. Это позволяет разработчикам не заботиться о ручном разрыве таких связей, если ссылки действительно потеряны.
