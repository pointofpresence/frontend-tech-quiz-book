# Как работал с формами в React? (React Hook Form, Formik)

Категория: Процессы

### Краткий ответ

**Как работал с формами в React? (React Hook Form, Formik)**  
- управление состоянием формы через **hook или HOC**  
- React Hook Form: минимальное перерендеривание, использует **рефы**  
- Formik: более декларативный, легко валидировать через **Yup**  
- обе библиотеки обеспечивают **валидацию**, управление ошибками  
- React Hook Form – более производительный при больших формах  
- Formik – подходит для сложной логики, удобен с кастомными UI  
- применял для **автоматизации валидации**, упрощения UX формы

### Подробный ответ
#### Основной ответ
В React для работы с формами широко используют библиотеки **React Hook Form** и **Formik**, которые упрощают управление состоянием, валидацией и обработкой сабмита. React Hook Form строится на хуках, обеспечивает высокую производительность за счёт минимального повторного рендеринга, а Formik предлагает декларативный подход с богатым API и встроенной поддержкой Yup для схем валидации.

#### Ключевые моменты
- **React Hook Form (v7+)** использует uncontrolled компоненты и рефы, что снижает нагрузку на рендеринг, подходит для сложных форм с большим количеством полей. Управление осуществляется через `register`, а валидацию можно настроить как синхронную, так и асинхронную.
- **Formik** использует controlled компоненты, что даёт полный контроль над значениями через state и props, хорошо подходит для простых и средних по сложности форм. Часто комбинируется с библиотеками валидации (например, Yup), что облегчает поддержку и масштабируемость.
- Важно учитывать trade-offs: React Hook Form быстрее и легче в освоении для новых проектов, с меньшим boilerplate; Formik же более понятен разработчикам, привыкшим к классическому контролируемому подходу React, но может приводить к большему количеству ререндеров.

#### Практический контекст
В реальных проектах я использовал React Hook Form для больших дашбордов с динамическими формами, где важна оптимизация рендеринга и минимальное время отклика (latency ~50ms). Formik предпочитал для быстрого прототипирования и форм с простой логикой, когда нужна тесная интеграция с Redux или другими state management инструментами. Для валидации обычно применял Yup, в обоих случаях, что обеспечивает декларативность и повторное использование схем.
