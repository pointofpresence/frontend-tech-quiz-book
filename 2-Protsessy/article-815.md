# Как работают оконные функции (window functions) в SQL?

Категория: Процессы

### Краткий ответ

**Как работают оконные функции (window functions) в SQL?**

- SQL-конструкт для **аналитических вычислений** по набору строк  
- Работают с **окном строк** вокруг текущей записи, не сводя результат к одной строке  
- Окно задаётся через **OVER()** с **PARTITION BY** (разбиение) и **ORDER BY** (порядок)  
- В отличие от агрегатных функций, возвращают значение для **каждой строки**  
- Примеры: **ROW_NUMBER()**, **RANK()**, **SUM() OVER()**, **LAG()**, **LEAD()**  
- Позволяют вычислять скользящие суммы, ранжирование, разницы между строками без подзапросов  
- Используются для сложной аналитики, отчетности, обработки данных в рамках групп и порядка  

Итог: оконные функции расширяют возможности SQL, позволяя делать продвинутую аналитику, сохраняя исходный набор данных.

### Подробный ответ
#### Основной ответ
**Оконные функции в SQL** — это специальные функции, которые позволяют выполнять вычисления по набору строк, относящихся к текущей строке, без группировки результатов. В отличие от агрегатных функций с `GROUP BY`, оконные функции возвращают значение для каждой строки исходного набора данных, расширяя аналитические возможности.

#### Ключевые моменты
- Оконные функции используют **клаузы `OVER()`**, в которой задаётся **окно** (часть выборки) с помощью `PARTITION BY` (разбиение данных на группы) и `ORDER BY` (порядок внутри группы).
- Примеры популярных оконных функций: `ROW_NUMBER()`, `RANK()`, `LAG()`, `LEAD()`, `SUM() OVER()`.
- Важно, что исходное количество строк не уменьшается, в отличие от `GROUP BY`: каждая строка обрабатывается с учётом её окна, сохраняя контекст.

#### Практический контекст
Оконные функции широко применяются для **ранжирования**, подсчёта скользящих сумм, вычисления разниц между строками, анализа временных рядов. В PostgreSQL 14+ и других современных СУБД они оптимизированы для больших наборов (~миллионы строк с приемлемой производительностью). Например, на проекте для финансовой аналитики с `SUM() OVER (PARTITION BY client ORDER BY date ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW)` считают накопленные итоги по операциям клиента.
