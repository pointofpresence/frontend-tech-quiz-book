# Как проверить, что переменная хранит функцию, не вызывая ее?

Категория: Процессы

### Краткий ответ

**Как проверить, что переменная хранит функцию, не вызывая ее?**

- JS, типизация
- typeof возвращает "function"
- безопасно для любых значений
- быстро и эффективно
- поддерживается во всех браузерах и средах
- применимо для проверки функций, методов и лямбд
- позволяет избежать ошибок вызова не-функций

Основное: использовать оператор `typeof var === "function"` для проверки, что переменная содержит функцию без её вызова.

### Подробный ответ
#### Основной ответ
Чтобы проверить, что переменная содержит функцию, **не вызывая её**, в JavaScript обычно используют оператор `typeof`. При применении к функции, `typeof` возвращает строку `"function"`. Это безопасный и распространённый способ отличить функции от других типов данных.

Пример:  
```js
if (typeof myVar === 'function') {
  // myVar — это функция
}
```

#### Ключевые моменты
- **typeof работает без вызова функции** — это важно для проверки без побочных эффектов.
- Альтернатива — использовать `instanceof Function`, но он может не работать корректно при работе с функциями из разных контекстов окружения (например, iframe).
- Нужно учитывать, что в некоторых языках или окружениях может быть иная логика, но в стандартном JavaScript `typeof` — универсальный метод.

#### Практический контекст
В React 18+ или Node.js 16+ этот способ широко используется, например, при проверке перед вызовом callback параметров или middleware, где важно не вызывать лишний раз функцию и избегать ошибок времени выполнения. Такой паттерн помогает писать более устойчивый и безопасный код.
