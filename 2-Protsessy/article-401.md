# Как устроен runtime в Go?

Категория: Процессы

### Краткий ответ

**Как устроен runtime в Go?**

- **Go runtime** — это встроенная среда выполнения, реализующая поддержку **конкурентности**, **сборки мусора**, **управления памятью** и **планирования**.
- Основные компоненты: **goroutine scheduler**, **garbage collector (GC)**, **память и стек**.
- **Scheduler** управляет тысячами легковесных goroutine на ограниченном числе ОС-потоков (M:N модель).
- **Garbage collector** — **маркировка-сборка** с низкой задержкой, оптимизирован для многопоточности.
- **Стек goroutine** динамически растёт/сжимается для экономии памяти.
- В runtime реализованы системные вызовы, таймеры, синхронизация (mutex, каналы).
- Обеспечивает взаимодействие Go кода с ОС и C (через cgo).
- Ключевая цель — высокая производительность, масштабируемость и минимальная задержка.

Так устроен runtime, который скрывает сложности низкоуровневого управления, позволяя писать эффективный параллельный код.

### Подробный ответ
#### Основной ответ  
**Runtime в Go** — это совокупность низкоуровневых компонентов, которые управляют выполнением Go-программ вне зависимости от ОС и аппаратной платформы. Он отвечает за управление памятью (включая сборку мусора), планирование горутин, работу с системными вызовами и обработку ошибок. Runtime реализует модель конкурентности языка и предоставляет базовые примитивы, необходимые для запуска Go-кода.

#### Ключевые моменты  
- **Менеджер горутин и планировщик**: В Go runtime реализован собственный M:N планировщик, который сопоставляет множество легковесных горутин (G) с системными потоками (M). Это критично для масштабирования конкурентных программ, позволяя эффективно использовать системные ядра. В Go 1.14+ улучшена прерываемость горутин (preemption), что повышает отзывчивость планировщика.  
- **Сборка мусора (GC)**: Go использует **конкурентный, триггерный, трёхфазный GC** с минимальной паузой (число миллисекунд), который работает одновременно с выполнением приложения, снижая задержки. Сборка мусора умеет обрабатывать большие кучи с низким impact на latency (~10-20 мс пауз в типичных нагрузках).  
- **Управление памятью**: Runtime включает в себя выделение памяти, а также слои для работы с кучей и стэком горутин. Стэки горутин динамические — они растут и сжимаются по мере необходимости, что отличается от фиксированных стэков в традиционных потоках и экономит память.  

#### Практический контекст  
В реальных проектах runtime Go позволяет писать масштабируемые распределённые системы с тысячами конкурентных задач, без ручного использования потоков ОС. Улучшения в runtime делают возможным использование Go в latency-sensitive сервисах, таких как сетевые прокси, микросервисы и высоконагруженные backend-компоненты. Runtime тесно интегрирован с компилятором и обеспечивает платформенную независимость — одинаковое поведение Go-кода на Linux, Windows и macOS.
