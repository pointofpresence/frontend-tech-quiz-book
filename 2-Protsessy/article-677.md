# Чем отличается .NET Framework от .NET Core (кроссплатформенность, поддержка Linux)?

**Отличия .NET Framework и .NET Core (кроссплатформенность, поддержка Linux)**  
- .NET Framework: только для Windows, устоявшаяся платформа  
- .NET Core: **кроссплатформенный** (.Windows, Linux, macOS)  
- .NET Core поддерживает контейнеры, микросервисы, облако  
- .NET Framework ограничен поддержкой и обновлениями  
- .NET Core ориентирован на производительность и модульность  
- .NET Core развивается быстрее, рекомендуем для новых проектов  
- .NET Framework лучше для старых корпоративных приложений с Windows-зависимостями

# Подробный ответ
## Основной ответ
**.NET Framework** — это полнофункциональная, Windows-ориентированная платформа разработки, появившаяся в начале 2000-х, работающая только на Windows и имеющая тесную интеграцию с технологиями Windows (например, COM, WPF, Windows Forms). **.NET Core** — это современная кроссплатформенная платформа с открытым исходным кодом, разработанная Microsoft для поддержки Windows, Linux и macOS, с акцентом на производительность, модульность и микросервисную архитектуру. 

## Ключевые моменты
- **Кроссплатформенность**: .NET Framework строго для Windows, в то время как .NET Core (и с версии 5+ — объединённый .NET) обеспечивает полноценную поддержку Linux и macOS, что важно для серверных и контейнеризированных приложений.
- **Архитектура и модульность**: .NET Framework имеет монолитный дизайн, тогда как .NET Core предоставляет модульные пакеты через NuGet, что снижает размер приложений и позволяет использовать только нужные компоненты.
- **Поддержка и развитие**: Microsoft прекратила активное развитие .NET Framework, сосредоточившись на .NET Core и новых версиях (.NET 5/6/7), которые быстро обновляются и получают новые возможности, включая улучшенную поддержку Linux, Docker, облачные сценарии.

## Практический контекст
В реальных проектах, если нужна поддержка Linux-серверов, микросервисы с высокой производительностью или контейнеризация в Docker, выбирают .NET Core (или современный .NET). Для наследуемых Windows-приложений с глубокими интеграциями часто продолжают использовать .NET Framework, но новые проекты ориентируются на кроссплатформенную архитектуру .NET Core.
