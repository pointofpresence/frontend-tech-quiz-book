# Как параллельно запросить данные из 4 независимых систем? (например, через CompletableFuture)

**Как параллельно запросить данные из 4 независимых систем? (через CompletableFuture)**  
- Java concurrency / асинхронность  
- CompletableFuture — обертка для асинхронных вычислений  
- Использовать **CompletableFuture.supplyAsync()** для запуска каждого запроса параллельно в пуле  
- Собрать 4 CompletableFuture, по одному на каждый запрос  
- Объединить с помощью **CompletableFuture.allOf()** для ожидания завершения всех  
- После allOf() получить результаты через .join() или .get() у каждого будущего  
- Параллелизм уменьшает общее время ожидания, улучшая throughput  
- Применять общий пул потоков (ForkJoinPool.commonPool) или кастомный Executor для контроля  
- Обрабатывать исключения с помощью .exceptionally() или handle(), чтобы не потерять ошибки  
- Практическое применение: оптимизация микросервисных запросов или сторонних API запросов  

---

**Примерный код (схематично):**  
```java  
CompletableFuture<Data1> cf1 = CompletableFuture.supplyAsync(() -> fetchFromSystem1());  
CompletableFuture<Data2> cf2 = CompletableFuture.supplyAsync(() -> fetchFromSystem2());  
CompletableFuture<Data3> cf3 = CompletableFuture.supplyAsync(() -> fetchFromSystem3());  
CompletableFuture<Data4> cf4 = CompletableFuture.supplyAsync(() -> fetchFromSystem4());

CompletableFuture<Void> all = CompletableFuture.allOf(cf1, cf2, cf3, cf4);

all.thenRun(() -> {
    Data1 d1 = cf1.join();
    Data2 d2 = cf2.join();
    Data3 d3 = cf3.join();
    Data4 d4 = cf4.join();
    // обработка результатов
}).exceptionally(ex -> {
    // обработка ошибок
    return null;
});
```  

---

**Кратко:** запуск параллельных запросов через supplyAsync(), ожидание allOf(), сбор результатов через join().

# Подробный ответ
## Основной ответ  
Для параллельного запроса данных из 4 независимых систем в Java оптимальным решением будет использование **CompletableFuture** из пакета `java.util.concurrent`. Это позволяет запускать асинхронные задачи, не блокируя основной поток, и удобно агрегировать результаты, когда все вызовы завершатся.

Алгоритм в общих чертах такой: для каждой системы создаётся своя `CompletableFuture`, который выполняет запрос асинхронно (например, через `supplyAsync`). Далее с помощью метода `allOf` объединяем все 4 future в один, который завершится только когда выполнены все запросы. После этого можно извлечь результаты и продолжить обработку.

## Ключевые моменты  
- **Асинхронность**: `CompletableFuture.supplyAsync(() -> fetchData())` запускает задачу в пуле потоков ForkJoinPool.commonPool() или собственном Executor’е, не блокируя главный поток.  
- **Объединение результатов**: `CompletableFuture.allOf(future1, future2, future3, future4)` возвращает CompletableFuture<Void>, который завершится когда все четыре задачи будут выполнены. Дальше через `futureX.join()` можно получить конкретные данные.  
- **Ошибкоустойчивость**: можно использовать методы `handle`, `exceptionally` или `whenComplete` для обработки возможных ошибок в каждом отдельном запросе, чтобы не прервать выполнение всего процесса.

## Практический контекст  
В реальных проектах я использовал такой подход для обращения к нескольким REST-сервисам (через WebClient или HttpClient), чтобы получить данные для агрегации на уровне сервера. При этом запускается 4 запроса почти одновременно с latency ~100ms на каждый вместо последовательных ~400ms. В React 18, например, похожий подход применяется с Suspense для параллельного рендеринга данных.  

Этот шаблон хорошо масштабируем и позволяет сохранять отзывчивость приложения, особенно в микросервисной архитектуре, где независимые внешние вызовы нужно выполнять параллельно для низкой общей задержки.
