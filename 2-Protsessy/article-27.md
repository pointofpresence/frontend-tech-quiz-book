# Что такое rest оператор?

Категория: Процессы

### Краткий ответ

**Что такое rest оператор?**  
- синтаксис ES6 для сбора аргументов  
- обозначается как `...` перед именем параметра  
- собирает остаток аргументов в **массив**  
- используется в функциях для переменного числа параметров  
- отличается от spread, который распаковывает массивы  
- упрощает работу с динамическими аргументами  
- широко применяется в React и современном JS

### Подробный ответ
#### Основной ответ
**Rest оператор** в JavaScript — это синтаксический сахар, позволяющий собрать нефиксированное количество аргументов функции или элементов массива в один массив. Он обозначается тремя точками (`...`) и помогает удобно работать с переменным числом параметров или выделять часть структуры данных.

#### Ключевые моменты
- **Использование в функциях**: позволяет собрать все переданные аргументы в один массив, например `function(...args)`, что заменяет необходимость использовать `arguments` и делает код чище и более понятным.
- **Деструктуризация объектов и массивов**: с помощью rest можно собрать остаток элементов или свойств в отдельный объект или массив: 
  ```js
  const {a, b, ...rest} = obj;
  ```
  — переменная `rest` получает все остальные свойства, кроме `a` и `b`.
- **Отличие от spread**: rest оператор похож на spread, оба используют `...`, но rest — на этапе объявления (сбора в один массив/объект), а spread — при распаковке элементов в вызове функции или массиве.

#### Практический контекст
В реальных проектах в React 18+ rest оператор часто используется при передаче пропсов (`...props`) или при обработке входных параметров функций с переменным числом аргументов, а в Node.js — для удобной работы с параметрами middleware или API методов. Это упрощает код и улучшает читаемость, особенно при работе с динамическими данными.
