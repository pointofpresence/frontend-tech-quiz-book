# Какими способами можно запустить корутину в Kotlin?

Категория: Процессы

### Краткий ответ

**Какими способами можно запустить корутину в Kotlin?**

- корутина — легковесный поток, управляемый **CoroutineScope**  
- запуск через **launch** — возвращает Job, используется для задач без результата  
- запуск через **async** — возвращает Deferred, используется для задач с результатом  
- создание coroutine с помощью **runBlocking** — блокирует текущий поток (главный при тестах)  
- использование **withContext** для смены контекста или потока  
- разные **CoroutineDispatcher** (Dispatchers.Default, IO, Main) для управления потоком  
- применение **GlobalScope** для глобального ливеспана (редко рекомендован)  

Запуск зависит от контекста и цели: fire-and-forget (launch), вычислить результат (async), блокирующий вызов (runBlocking), переключение потоков (withContext).

### Подробный ответ
#### Основной ответ
В Kotlin **корутину** можно запустить несколькими способами, в зависимости от контекста и цели. Основные подходы — использование функций **launch**, **async** и **runBlocking** из библиотеки kotlinx.coroutines. Эти механизмы дают гибкий контроль над жизненным циклом корутин, степенью параллелизма и возвращаемыми результатами.

#### Ключевые моменты
- **launch** — запускает корутину без ожидания результата, возвращает объект Job для контроля отмены или ожидания завершения. Часто используется для фоновых задач в UI или backend.
- **async** — запускает корутину с возвращением Deferred, которая представляет будущий результат; позволяет выполнять параллельные вычисления и далее вызывать await() для получения результата.
- **runBlocking** — блокирующая функция, которая запускает корутину в текущем потоке и ждет её завершения; полезна в main-модуле или тестах, где нельзя использовать suspend.
- Запуск корутины требует **CoroutineScope** — это контекст, содержащий Job и Dispatcher. Например: `GlobalScope.launch { ... }` (редко рекомендуют из-за глобальной природы) или лучше использовать `viewModelScope` (в Android) или собственный scope.
- Для выбора потока исполняемого кода важно указать **Dispatcher**: `Dispatchers.IO` для операций ввода-вывода, `Dispatchers.Default` для CPU-интенсивных задач, `Dispatchers.Main` — для UI.

#### Практический контекст
В React Native, Android и backend Kotlin сервисах обычно применяют `viewModelScope.launch` или кастомный CoroutineScope, чтобы запускать корутины безопасно и с управлением жизненным циклом. В тестах используют `runBlocking`, чтобы синхронно дождаться результата корутины. В многопоточных вычислениях часто применяют `async/await` для параллелизма с комбинированием результатов.
