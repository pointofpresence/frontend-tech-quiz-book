# Какие есть базовые значения display?

Категория: Процессы

### Краткий ответ

**Базовые значения display**

- CSS-свойство для управления отображением элементов  
- **block**: занимает всю ширину, перенос строки  
- **inline**: располагается в строке, ширина по содержимому  
- **inline-block**: как inline, но можно задавать размеры  
- **none**: элемент не отображается и не занимает место  
- **flex**: включает флекс-контейнер, управляет layout детей  
- **grid**: включает grid-контейнер, управление в двумерной сетке  
- базовые значения важны для построения layout и управления видимостью элементов

### Подробный ответ
#### Основной ответ
Свойство **display** в CSS определяет, как элемент будет визуально представлен на странице — его поведение в потоке документа и как он взаимодействует с соседними элементами. Базовые значения display задают основные типы рендеринга и влияют на блочную или строчную модель отображения.

#### Ключевые моменты
- **block** — элемент отображается как блочный, занимает всю доступную ширину контейнера, начинается с новой строки (например, `<div>`, `<p>` по умолчанию).
- **inline** — элемент отображается как строчный, занимает только нужное пространство, не переносит строку (например, `<span>`, `<a>` по умолчанию).
- **inline-block** — гибридный режим, элемент ведёт себя как строчный по расположению, но можно задавать ширину и высоту, что невозможно для inline.
- **none** — элемент полностью скрывается, не занимает место в потоке документа, полезно для динамического скрытия.
- **flex** — превращает элемент в flex-контейнер, активируя флексбокс-модель для удобного выравнивания вложенных элементов.
- **grid** — задаёт контейнер с grid-layout для сложного позиционирования по двум осям.

#### Практический контекст
Часто используют блок и инлайн при базовой вёрстке. Inline-block удобен, когда нужно разместить элементы в строке с контролируемым размером. Flex и grid — современные мощные инструменты для адаптивных интерфейсов. None — стандарт для условного скрытия без удаления элемента. Понимание этих значений важно для гибкой и корректной работы с layout и дизайном.
