# Чем map отличается от flatMap?

Категория: Процессы

### Краткий ответ

**Чем map отличается от flatMap?**

- Функции для работы с коллекциями или потоками  
- **map** преобразовывает каждый элемент отдельно  
- Возвращает коллекцию с элементами-преобразованиями  
- **flatMap** сначала применяет функцию, возвращающую список/коллекцию  
- Затем "расплющивает" вложенные списки в один уровень  
- Используется для работы с вложенными или множественными результатами  
- Практическая польза: flatMap удобен для обхода списков списков или потоков потоков без вложенности

### Подробный ответ
#### Основной ответ
**map** и **flatMap** — это функции высшего порядка, часто используемые для трансформации коллекций или потоков данных, но между ними есть ключевое различие. **map** применяет функцию к каждому элементу и возвращает результат с той же вложенностью, в то время как **flatMap** сначала применяет функцию, которая возвращает коллекцию или структуру внутри каждого элемента, а затем "расплющивает" (flatten) результат в плоскую коллекцию.

#### Ключевые моменты
- **map** трансформирует каждый элемент по отдельности, сохраняя структуру: например, `List<T>` в `List<R>`.
- **flatMap** сначала мапит элемент в коллекцию, например `T -> List<R>`, а затем объединяет все вложенные списки в один общий `List<R>`.
- Используется там, где нужно избежать вложенности коллекций, например, если функция возвращает множество результатов для каждого элемента, flatMap помогает получить единый плоский список.
- В Java 8+ `Stream.flatMap` часто применяется для работы с потоком списков, объединяя их в один поток; в функциональных языках (Scala, Kotlin) поведение аналогично.
  
#### Практический контекст
Типичный пример в Java: Suppose есть `List<String>` с предложениями, и вы хотите получить список всех слов — map вернёт список списков слов, а flatMap "расплющит" их в один список строк. Так flatMap удобен для операций с вложенными контролируемыми коллекциями, упрощая код и избегая лишней вложенности.
