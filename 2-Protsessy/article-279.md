# Какие встроенные структуры данных в Python ты знаешь и чаще всего используешь?

**Встроенные структуры данных в Python**

- базовые: **list**, **tuple**, **dict**, **set**
- **list**: изменяемый массив с индексированием, операции добавления/удаления
- **tuple**: неизменяемый, быстрее list, используется для ключей dict
- **dict**: **хеш-таблица**, хранение пар ключ-значение, быстрый поиск O(1)
- **set**: множество уникальных элементов, операции пересечений, объединений
- часто применяю **list** для коллекций, **dict** для отображений, **set** для уникальных данных
- использую **collections** (deque, defaultdict) для оптимизации и удобства
- структуры нужны для эффективного хранения, быстрого доступа и манипуляций с данными в проектах

# Подробный ответ
## Основной ответ
В Python есть несколько **встроенных структур данных**, которые я использую регулярно — эти структуры позволяют эффективно хранить и обрабатывать данные в большинство задач. Основные — это **list, tuple, dict, set и frozenset**. Каждая из них подходит под разные сценарии в зависимости от требований к мутабельности, производительности и типу операций.

## Ключевые моменты
- **list** — изменяемый упорядоченный контейнер, отлично подойдет для динамических коллекций, когда важен порядок и частые операции вставки/удаления элементов, особенно в конце списка (amortized O(1)). Использую в обработке последовательностей и очередях небольшого объема.
- **tuple** — неизменяемый упорядоченный контейнер, полезен для ключей в словарях и как упаковка нескольких элементов для передачи между функциями; tuple быстрее list по сравнению при переборе за счёт immutability.
- **dict** (с версии Python 3.7) — упорядоченное отображение ключей в значения, реализованное на основе хеш-таблиц; ключевой инструмент для быстрого доступа по ключу (O(1) в среднем). Часто использую для хранения конфигураций, агрегирования данных и кэширования.
- **set / frozenset** — контейнеры уникальных элементов, применяю для фильтрации дубликатов, операций множеств (пересечение, объединение) и проверки вхождения с высокой производительностью. frozenset — неизменяемая версия set, используется как ключ в dict.
- Из более специфичных встроенных типов можно отметить **collections.namedtuple, deque из collections** (для эффективной двунаправленной очереди) и **array.array** для компактного хранения примитивных типов.

## Практический контекст
В реальных проектах, например, я часто комбинирую dict с list для реализации быстрых поисков и обходов данных, а deque использую в обработке потоков или очередей, когда нужна высокая производительность вставки/удаления с обеих сторон. Всё это — базовые кирпичики, на которых строятся более сложные структуры и алгоритмы.
