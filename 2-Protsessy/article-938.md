# Как централизованно обрабатывать исключения в REST-контроллерах?

Категория: Процессы

### Краткий ответ

**Как централизованно обрабатывать исключения в REST-контроллерах?**

- Область: обработка ошибок, backend, REST API  
- Используют глобальные обработчики исключений (например, @ControllerAdvice в Spring)  
- Логика обработки отделена от контроллеров, поддержка DRY  
- Аннотируют методы @ExceptionHandler для конкретных исключений  
- Возвращают структурированный ответ с HTTP статусом и сообщением ошибки  
- Позволяет единообразно логировать и фильтровать ошибки  
- Улучшает клиентский опыт за счёт понятных ошибок и кода ответа  

Основной принцип: централизованная обработка через глобальные обработчики исключений с генерацией стандартизированных ответов REST API.

### Подробный ответ
#### Основной ответ
Централизованная обработка исключений в REST-контроллерах позволяет унифицировать реакции на ошибки, сделать код контроллеров чище и обеспечить согласованное форматирование ответов об ошибках. В Java/Spring Boot для такой задачи обычно используется аннотация **@ControllerAdvice** или **@RestControllerAdvice** в связке с методами, помеченными **@ExceptionHandler**. Это позволяет перехватывать исключения, выброшенные из любого контроллера, и возвращать кастомизированный HTTP-ответ с нужным статусом и телом.

#### Ключевые моменты
- **@RestControllerAdvice** работает глобально и применяется ко всем REST-контроллерам, что устраняет дублирование try-catch и ручной обработки ошибок в каждом методе.
- Методы с **@ExceptionHandler** могут принимать конкретные исключения или их иерархии, чтобы дифференцировать обработку: например, для **EntityNotFoundException** возвращать 404, а для **MethodArgumentNotValidException** — 400.
- Результатом обработки обычно является объект с единой структурой ошибки (код, сообщение, детали), что упрощает клиентам парсинг и логику по ошибкам. Для сериализации часто используют Jackson.

#### Практический контекст
В реальных проектах я часто создаю класс `GlobalExceptionHandler`, где обрабатываю типичные ошибки: валидация, бизнес-исключения, ошибки БД. Добавляю логирование и метрики через Prometheus. Такой подход повышает качество API и упрощает поддержку — ошибки становятся прозрачными, а клиенты всегда получают адекватный и понятный ответ без лишних деталей стека.
