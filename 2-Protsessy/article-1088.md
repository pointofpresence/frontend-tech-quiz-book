# Какие объекты размещаются в куче?

Категория: Процессы

### Краткий ответ

**Какие объекты размещаются в куче?**

- память для **динамически выделяемых** объектов  
- объекты с **неопределённым временем жизни**  
- объекты, размер которых **неизвестен на этапе компиляции**  
- в JVM: **объекты классов, строки (string pool исключение)**  
- в C++: объекты, выделенные через **new/malloc**  
- управляется сборщиком мусора или вручную  
- используется для **сохраняемых между вызовами функций данных**  

Итог: куча — это область памяти для динамических, долговременных объектов с гибким сроком жизни.

### Подробный ответ
#### Основной ответ
В **куче (heap)** размещаются объекты, которые требуют динамического выделения памяти во время выполнения программы. Это обычно объекты сложных типов, такие как экземпляры классов, структуры данных, массивы, создаваемые во время работы приложения. В отличие от стека, где хранятся примитивные локальные переменные и адреса вызовов функций, куча предоставляет гибкое пространство для объектов с неопределённым временем жизни.

#### Ключевые моменты
- **Объекты с динамическим временем жизни**: например, объекты в Java или C# создаются через `new` и живут столько, сколько на них существуют ссылки; сборщик мусора отвечает за их очистку из кучи.
- **Размер и структура**: объекты, размер которых нельзя определить на этапе компиляции (например, динамические массивы) — тоже размещаются в куче.
- **Управление памятью**: в куче используется сложное управление памятью, включая аллокацию и сборку мусора (GC), что может влиять на производительность, в отличие от стека с его LIFO-структурой и быстрой аллокацией.

#### Практический контекст
В Java или C# все объекты классов находятся в куче, а на стеке размещаются только ссылки на них. В C++ объекты можно создавать как на стеке, так и в куче вручную (через `new`). При работе с большим количеством данных или с объектами, которые должны жить долго вне области видимости функции, куча — предпочтительное место для хранения.
