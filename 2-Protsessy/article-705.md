# Какой сборщик по умолчанию в Java 11? (G1)

**Какой сборщик по умолчанию в Java 11? (G1)**

- Java 11 по умолчанию использует **Garbage-First (G1) GC**
- G1 — **региональный и параллельный** сборщик мусора
- Делит heap на мелкие **региональные зоны** (обычно 1–32 МБ)
- Собирает регионы с максимумом мусора в первую очередь — **уменьшает паузы**
- Поддерживает **многоядерность** и **конкурентную работу**
- Оптимален для **больших heap** и низких latencies
- Предназначен заменить **CMS** и улучшить производительность сборки мусора по умолчанию

G1 применят для балансировки **пропускной способности** и **отзывчивости** приложений в современных JVM.

# Подробный ответ
## Основной ответ
В **Java 11** сборщиком мусора по умолчанию является **G1 (Garbage-First)**. Этот сборщик предназначен для сочетания низкой задержки и хорошей пропускной способности, что делает его оптимальным выбором для большинства современных приложений.

## Ключевые моменты
- **G1** разбивает heap на множество регионов и пытается собирать сначала те регионы, которые содержат наибольшее количество мусора, минимизируя паузы.
- В **Java 9** был закреплён как экспериментальный, а с **Java 10+** стал сборщиком по умолчанию, заменив предыдущий Parallel GC.
- **G1** балансирует между сборкой в фоне (concurrent) и короткими паузами (stop-the-world), что позволяет легко достичь низкой латентности (~10-100 мс) при размере heap до нескольких сотен гигабайт.
- Для огромных heap (от нескольких сотен гигабайт) в новых версиях Java рекомендуют ZGC или Shenandoah, но в Java 11 G1 остаётся основным выбором.

## Практический контекст
G1 — стандартный сборщик в корпоративных приложениях, микросервисах и серверных системах, где важен баланс между пропускной способностью и минимальными задержками. Его конфигурация позволяет гибко настраивать целевые паузы по задержкам через параметры `-XX:MaxGCPauseMillis`, а мониторинг можно проводить через встроенные инструментальные средства JVM и Prometheus exporters.
