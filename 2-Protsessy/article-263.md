# Какие функции использовал в Charles Proxy?

**Функции Charles Proxy**

- **Перехват HTTP/HTTPS трафика** – анализ запросов и ответов
- **SSL Proxying** – расшифровка HTTPS для дебага защищённых запросов
- **Throttle** – имитация медленного соединения, тест производительности
- **Breakpoints** – остановка и изменение запросов/ответов в реальном времени
- **Map Local** – подмена ответов локальными файлами для тестирования
- **Rewrite** – динамическая модификация заголовков, тел запросов/ответов
- **Recording/Export** – сохранение сессий для последующего анализа и отчётов

# Подробный ответ
## Основной ответ
В Charles Proxy я использовал широкий набор функций для комплексного анализа и отладки сетевого трафика, что позволило эффективно выявлять и исправлять проблемы с API и взаимодействием клиента с сервером. Среди ключевых возможностей — перехват HTTPS-трафика, модификация запросов и ответов, а также сессий, что значительно облегчало тестирование различных сценариев.

## Ключевые моменты
- **SSL Proxying (перехват HTTPS)**: включал перехват зашифрованного трафика с помощью установки сертификата Charles, что позволяло видеть полный текст запросов и ответов, включая payload JSON или XML.
- **Rewrite и Breakpoint Tools**: применял правила перезаписи запросов и ответов (Rewrite) для имитации различных состояний сервера, а также ставил Breakpoints для остановки и правки данных в реальном времени перед отправкой.
- **Throttle и Bandwidth Throttling**: для симуляции медленного или нестабильного соединения использовал ограничение скорости сети, что важно для тестирования UX в реальных условиях с низкой пропускной способностью.
- **Export и Save Sessions**: сохранял и делился сессиями (архивами трафика) для совместного анализа или регрессионного тестирования, что улучшало коммуникацию внутри команды.
- **Map Local и Map Remote**: переадресация запросов на локальные файлы или другие URL для отладки фронтенда без изменения серверной части.

## Практический контекст
В реальных проектах эти функции Charles Proxy позволяют быстро диагностировать проблемы с API, отлавливать ошибки аутентификации, проверять корректность данных, оптимизировать цепочку запросов клиента и автоматизировать тестирование сценариев без прямого изменения серверного кода. Типично использовал Charles в связке с мобильными приложениями и SPA для детального сетевого аудита.
