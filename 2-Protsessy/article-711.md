# Разница между поверхностным (copy.copy()) и глубоким (copy.deepcopy()) копированием объектов.

Категория: Процессы

### Краткий ответ

**Разница между поверхностным и глубоким копированием**

- copy.copy() создаёт новый объект верхнего уровня, копируя ссылки на вложенные объекты  
- copy.deepcopy() рекурсивно создаёт полностью независимые копии всех вложенных объектов  
- поверхностная копия не дублирует вложенные изменяемые объекты → изменения отражаются в обеих копиях  
- глубокая копия полностью изолирует объекты → изменения в одной копии не влияют на другую  
- deepcopy медленнее и требует больше памяти, но безопаснее при работе с сложными вложенными структурами  
- поверхностная копия подходит для простых или неизменяемых вложенных данных  
- глубокая копия нужна при необходимости полного клонирования состояний без общих ссылок на вложенные объекты

### Подробный ответ
#### Основной ответ
**Поверхностное копирование (copy.copy())** создает новый объект, но собственные поля объекта копируются как ссылки на те же вложенные объекты. **Глубокое копирование (copy.deepcopy())** рекурсивно копирует все вложенные объекты, создавая полностью независимую структуру.

#### Ключевые моменты
- При **поверхностном копировании** изменяя вложенный изменяемый объект в копии, вы затрагиваете и оригинал, так как ссылки на вложенные объекты одинаковые.
- При **глубоком копировании** все объекты, включая вложенные, полностью дублируются, изменения в копии не влияют на оригинал.
- **deepcopy()** медленнее и требует больше памяти, особенно для сложных и вложенных структур, потому что рекурсивно обходится вся вложенность и создаётся копия каждого элемента.
- Некоторые объекты не копируются корректно через deepcopy (например, открытые файлы или сокеты), поэтому иногда нужно реализовывать __deepcopy__ метод для кастомных классов.

#### Практический контекст
В проектах с вложенными структурами данных (например, сложные словари, списки объектов) важен выбор копирования: shallow copy подходит для простых случаев, где не нужно менять вложенные объекты, а deepcopy используется для создания полностью независимых копий, например, при реализации undo/redo или клонирования сложных состояний. В Python 3.8+ deepcopy улучшили по производительности, но всё равно лучше применять с осторожностью.
