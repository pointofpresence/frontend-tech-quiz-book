# Какие виды тестирования выполнял с помощью Postman?

**Виды тестирования в Postman**

- функциональное тестирование: проверка API-эндпоинтов на корректность ответов и логики  
- тестирование на ошибки: обработка некорректных запросов, проверка кода ответа и сообщений об ошибках  
- тестирование производительности (ручное): замеры времени отклика API  
- тестирование безопасности: проверка авторизации, аутентификации, ролей и доступа  
- интеграционное тестирование: проверка связей между несколькими API и сервисами через последовательные запросы  
- параметризация и тестирование с разными данными: проверка вариативности поведения API  
- автоматизация тестов с использованием коллекций и Newman для CI/CD интеграции  

Postman позволяет комплексно покрыть API тесты для гарантии надежности и корректности бекенд-сервисов.

# Подробный ответ
## Основной ответ
С помощью **Postman** я выполнял несколько ключевых видов тестирования API, включая функциональное, интеграционное и нагрузочное тестирование в базовом виде. Постман — это удобный инструмент для отправки HTTP-запросов вручную и автоматизации тестов с использованием JavaScript в разделе "Tests", что позволяет проверять корректность ответов и логику API.

## Ключевые моменты
- **Функциональное тестирование**: проверка работы отдельных RESTful endpoint-ов, валидность ответов (статус-код, структура JSON, значения) с использованием assert-ов внутри Postman.
- **Интеграционное тестирование**: последовательное выполнение запросов с использованием pre-request скриптов, передача данных между запросами через переменные окружения, чтобы проверить взаимосвязь нескольких сервисов и сценариев.
- **Регрессионное тестирование**: создание наборов (collections) запросов с тестами для автоматического повторного прогона изменений API, что позволяет быстро обнаруживать баги после обновлений.
- **Нагрузочное тестирование (ограниченно)**: хотя Postman не специализируется на нагрузочном тестировании, я использовал коллекции с большим количеством параллельных запросов через Newman CLI, иногда в связке с JMeter для более серьезных нагрузок.

## Практический контекст
В реальных проектах Postman помогает быстро выставлять ожидаемый контракт API между фронтендом и бекендом, выявлять ошибки бизнес логики, а также создавать документацию API с примерами запросов, что повышает качество и скорость разработки. Использовал Postman версии 10+, интегрировал с CI/CD (например, Jenkins), чтобы прогонять тесты при каждом деплое сервера.
