# В чём основное различие между @RestController и @Controller?

Категория: Процессы

### Краткий ответ

**Различие @RestController и @Controller**

- Оба — аннотации Spring для веб-контроллеров  
- **@Controller** — для MVC, возвращает **view** (HTML, JSP)  
- **@RestController** = @Controller + @ResponseBody  
- @RestController автоматически сериализует ответы в **JSON/XML**  
- @Controller требует ручного добавления @ResponseBody для REST API  
- Используется для разных целей:  
  @Controler — веб-страницы  
  @RestController — REST сервисы (API)  
- Способ упрощает написание API без лишних аннотаций  

---  
Таким образом, **@RestController — специализированный контроллер для REST**, а **@Controller — для классического MVC с отображениями**.

### Подробный ответ
#### Основной ответ
**@RestController** и **@Controller** — это аннотации из Spring Framework, которые определяют компоненты, обрабатывающие HTTP-запросы, но с разной ролью. **@RestController** — это специализированная аннотация, которая сочетает в себе **@Controller** и **@ResponseBody**, автоматически сериализуя возвращаемые объекты в JSON или XML. В то время как **@Controller** чаще используется для построения MVC-приложений, где методы возвращают названия view (например, JSP, Thymeleaf), и данные передаются через модель.

#### Ключевые моменты
- **Автоматическая сериализация**: @RestController автоматически конвертирует объекты в JSON/XML в ответе, что оптимально для REST API. @Controller требует явного добавления @ResponseBody к методам для схожего поведения.
- **Использование контекста**: @Controller более гибок для традиционных веб-приложений с View layer, а @RestController упрощает создание API без необходимости разрабатывать дополнительные view-шаблоны.
- **Конвенции и удобство**: В Spring начиная с версии 4.0 удобно использовать @RestController для микросервисов и API — сокращается количество аннотаций и упрощается код.

#### Практический контекст
В реальных проектах, если пишут backend для SPA (например, React или Angular) или мобильного клиента — обычно выбирают **@RestController**, так как нужен JSON-ответ. Если веб-приложение со страницами и серверным рендерингом — применяют **@Controller** с шаблонизаторами типа Thymeleaf.
