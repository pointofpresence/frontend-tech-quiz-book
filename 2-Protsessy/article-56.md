# Как связаны промисы с Event Loop?

Категория: Процессы

### Краткий ответ

**Как связаны промисы с Event Loop?**

- область: асинхронность в JavaScript  
- промис — объект **асинхронного** результата  
- Event Loop управляет очередью **микротасков** и **макротасков**  
- `then/catch/finally` — **микротаски** в Event Loop  
- промисы резолвятся в микротасках **после текущего стека**  
- микротаски выполняются **до следующего макротаска**  
- связь нужна для гарантии **последовательного** асинхронного выполнения и обновления UI без блокировки  
- применение: упрощение управления асинхронным кодом с более точным контролем порядка выполнения  

Детально: промисы при резолве ставят свои колбэки в очередь микротасков Event Loop, что позволяет им выполниться сразу после завершения основного кода, но до событий ввода-вывода и таймеров (макротасков). Такая архитектура обеспечивает предсказуемую, эффективную и неблокирующую модель асинхронности.

### Подробный ответ
#### Основной ответ
**Промисы** и **Event Loop** тесно связаны в модели асинхронного выполнения JavaScript. Промисы представляют собой механизм для работы с асинхронными операциями, позволяя «обещать» результат в будущем. Event Loop отвечает за управление очередями задач и выполнение этих асинхронных операций, обеспечивая неблокирующую обработку.

Когда промис разрешается или отклоняется, соответствующие then/catch/callback-функции не вызываются немедленно. Вместо этого их коллбэки помещаются в так называемый **микротаск (microtask) queue**, которая обрабатывается после текущей синхронной работы и до обработки следующей макротаски (например, setTimeout). Таким образом, Event Loop сначала выполнит весь синхронный код, затем обработает микротаски, включая промисы, и только потом перейдет к макротаскам.

#### Ключевые моменты
- **Промисы ставят свои then/catch коллбэки в очередь микротасков**, что гарантирует их исполнение сразу после текущего стека вызовов, но до таймаутов и прочих макротасков.
- **Event Loop обеспечивает порядок выполнения**, гарантируя, что микротаски (промисы) обрабатываются строго после синхронного кода, но до UI обновлений и макротасков.
- В браузерах и Node.js есть отличия в приоритетах и очередях задач, но базовый принцип: промисы — это механизм, работающий через микротаски Event Loop.

#### Практический контекст
В реальных проектах это знание помогает правильно строить цепочки промисов и контролировать асинхронный поток, избегая неожиданных задержек или гонок. Например, в React 18 микротаски применяются для оптимизации обновления состояния и синхронизации рендера. Event Loop и микротаски — ключевые для понимания производительности и отзывчивости в браузерах и серверном JavaScript.
