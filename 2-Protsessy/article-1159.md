# Что такое List<T> и как реализованы его Count и Capacity?

Категория: Процессы

### Краткий ответ

**Что такое List<T> и как реализованы его Count и Capacity?**  
- List<T> — универсальный динамический массив в .NET  
- Count — текущее число элементов (подсчитывается, хранится явно)  
- Capacity — размер внутреннего массива, выделенного под элементы  
- При добавлении элементов Count увеличивается, Capacity — при переполнении массив расширяется (обычно в 2 раза)  
- Count ≤ Capacity, Capacity выделяется заранее для оптимизации операций  
- Count — O(1) чтение, Capacity — отражает максимальный текущий размер без перераспределения  
- Позволяет эффективное хранение и динамическое расширение данных без постоянного копирования всего массива

### Подробный ответ
#### Основной ответ
**List<T>** — это обобщённая коллекция из пространства имён System.Collections.Generic в .NET, представляющая динамический массив, который может автоматически расширяться при добавлении элементов. Его ключевой задачей является хранение элементов в последовательном блоке памяти с возможностью быстрой индексации и эффективного добавления в конец.

**Count** — свойство, которое возвращает текущее число элементов, фактически хранящихся в списке.

**Capacity** — свойство, определяющее размер внутреннего массива, выделенного под хранение элементов. Он не обязательно равен Count, так как List<T> заранее выделяет место с некоторым запасом для снижения числа перераспределений памяти.

#### Ключевые моменты
- **Реализация Count:** это открытое свойство, возвращающее внутреннее поле, хранящее точный счётчик вставленных элементов. Обращение к Count — операция O(1).
- **Реализация Capacity:** оно отражает длину внутреннего массива (T[]), который служит буфером. При добавлении элементов, когда Count достигает Capacity, вместимость увеличивается обычно в 2 раза, что снижает частоту дорогостоящих аллокаций.
- **Расширение массива:** при необходимости List создает новый массив большего размера, копирует туда данные и заменяет ссылку. Это «умное» поведение позволяет поддерживать амортизированную сложность вставки O(1).

#### Практический контекст
В реальных проектах часто важно управлять Capacity, чтобы избежать частых аллокаций, особенно при загрузке большого объема данных (например, при десериализации) — для этого можно задать начальную вместимость в конструкторе List<T> или вызвать метод TrimExcess для уменьшения Capacity до размера Count, освобождая память. Такой подход улучшает производительность и снижает фрагментацию памяти.
