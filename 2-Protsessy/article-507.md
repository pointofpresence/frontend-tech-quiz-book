# Что делает ключевое слово synchronized?

Категория: Процессы

### Краткий ответ

**Что делает ключевое слово synchronized?**

- механизм **синхронизации** потоков в Java  
- обеспечивает **взаимное исключение** (mutex) при доступе к коду/ресурсу  
- блокирует объект-монитор для одного потока  
- предотвращает **гонки данных** и состояние гонки  
- гарантирует **атомарность** и последовательность операций  
- влияет на производительность из-за блокировок  
- часто используется для защиты **общих ресурсов** в многопоточности

### Подробный ответ
#### Основной ответ
Ключевое слово **synchronized** в языке Java используется для обеспечения **взаимного исключения (mutex)** при доступе к разделяемым ресурсам в многопоточном окружении. Оно гарантирует, что в один момент времени только один поток может выполнить критический участок кода, помеченный этим ключевым словом, предотвращая состояния гонки и обеспечивая корректность данных.

#### Ключевые моменты
- **Монитор блокировки**: при входе в synchronized-блок или synchronized-метод поток захватывает связанный с объектом или классом монитор (lock). Другие потоки будут ждать освобождения монитора для входа.
- **Видимость изменений**: synchronized обеспечивает **memory visibility** — изменения переменных внутри синхронизированного блока становятся видимыми другим потокам после выхода из блока, благодаря синхронизации кэшей CPU через модель памяти Java (JMM).
- **Два варианта использования**: можно синхронизировать метод (на уровне объекта или класса для static методов) или блок кода внутри метода, что дает гибкость в управлении областью критической секции.

#### Практический контекст
В реальных проектах synchronized часто применяют для защиты доступа к коллекциям, изменяющим состояние объекта, либо для реализации thread-safe singleton. Однако в Java 8+ часто предпочтительнее использовать более современные инструменты из пакета **java.util.concurrent** (например, ReentrantLock), которые предоставляют расширенные возможности управления блокировками и лучшую производительность при высокой конкуренции потоков.
