# Как реализовать функцию сложения в формате add(1)(2)? Как добавить в прототип числа?

Категория: Процессы

### Краткий ответ

**Реализация add(1)(2) и добавление в прототип Number**

- **Каррирование функции** — функция возвращает другую функцию с замкнутым параметром
- `add(a) => (b) => a + b` — простая замкнутая реализация
- Для добавления к числам через прототип `Number.prototype.add = function(b) { return this + b }`
- Учесть, что `this` в методе прототипа — объект-обёртка, преобразовать в примитив через `+this`
- Можно реализовать цепочку вызовов через переопределение `valueOf` или `toString` для отложенного суммирования
- `add(1)(2)` — функция с возвратом функции, а не метод прототипа; метод прототипа — для вызова через `num.add(3)`
- Практическое применение: удобство цепочек, кастомизация поведения примитивов

Таким образом, для `add(1)(2)` нужна функция возвращающая функцию с накоплением аргументов, добавление метода в прототип `Number` расширяет возможности всех чисел через методы экземпляров.

### Подробный ответ
#### Основной ответ
Чтобы реализовать функцию сложения в виде цепочки вызовов `add(1)(2)` и так далее, можно использовать **функциональный замыкание** с переопределением метода `toString` или `valueOf` у возвращаемой функции для ленивого вычисления результата. Такой подход позволяет в момент преобразования функции к примитиву возвращать итоговое значение сложения всех переданных аргументов. 

#### Ключевые моменты
- Функция `add` возвращает другую функцию, которая принимает следующий операнд и аккумулирует сумму в замыкании.
- Переопределение метода `valueOf` у функции позволяет возвращать текущее значение суммы, например, при принудительном приведении к числу (`+add(1)(2)`).
- Аналогично, можно переопределить `toString` для вывода результата, что полезно при логировании.
- Добавление метода к прототипу `Number.prototype` — непривычная практика, но технически можно расширить через `Number.prototype.add = function (x) { return this.valueOf() + x; }`, чтобы вызывать `(5).add(3)`.

#### Практический пример реализации функции add:
```javascript
function add(x) {
  let sum = x;
  function inner(y) {
    sum += y;
    return inner;
  }
  inner.valueOf = function () {
    return sum;
  };
  return inner;
}

// Использование:
console.log(+add(1)(2)(3)); // 6
```

#### Добавление метода к прототипу Number (редко практикуется):
```javascript
Number.prototype.add = function (x) {
  return this.valueOf() + x;
};

// Использование:
console.log((5).add(10)); // 15
```

#### Практический контекст
Такой подход часто применяют для цепочных API в функциональном стиле, DSL и для реализации ленивых вычислений. Переопределение `valueOf` и `toString` — распространённый паттерн в JavaScript для объектов-функций, позволяющий интегрировать их с примитивными операциями. В реальных задачах по расширению стандартных примитивов стоит быть осторожным, чтобы не нарушить консистентность кода и совместимость.
