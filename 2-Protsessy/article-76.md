# this — как определяется и чем отличается в strict mode

**this — определение и отличие в strict mode**  
- this — контекст вызова функции  
- В non-strict mode: this указывает на глобальный объект (window/node global) при обычном вызове  
- В strict mode: this остаётся undefined при обычном вызове, предотвращая неявный глобальный контекст  
- В методах this ссылается на объект вызова в обоих режимах  
- Строгий режим жестче контролирует контекст, снижая ошибки из-за неявного привязывания this  
- Практическое значение: помогает писать более безопасный и предсказуемый код, избегая случайных глобальных переменных  
- Ключевая разница — поведение при вызове функции вне объекта: undefined (strict) vs глобальный объект (non-strict)

# Подробный ответ
## Основной ответ
**`this`** в JavaScript — это контекст вызова функции, который указывает, к какому объекту относится выполнение текущей функции. В обычном (non-strict) режиме при вызове обычной функции без объекта (`foo()`) `this` ссылается на глобальный объект (`window` в браузере или `global` в Node.js). В режиме **strict mode** (`"use strict"`) значение `this` при таком вызове будет `undefined`, что помогает избежать случайного доступа к глобальному объекту и повышает безопасность кода.

## Ключевые моменты
- В **non-strict mode**:  
  - В глобальном контексте (`this` вне функций) — глобальный объект.  
  - В обычных вызовах функций — глобальный объект, если не вызвана через объект.  
- В **strict mode**:  
  - Вызов обычной функции с `this` без контекста привязывает `this` к `undefined`.  
  - Устраняет неявное привязывание к глобальному объекту, снижая баги и улучшая предсказуемость.  
- В методах объектов и конструкторах `this` ведёт себя одинаково в обоих режимах, указывая на объект-владельца или новый экземпляр.  

## Практический контекст
В React 18 и TypeScript строгий режим — стандарт, что помогает избежать ошибок из-за неявного `this`. В серверном Node.js strict mode включён по умолчанию в модулях ES6, что усиливает безопасную разработку без утечек глобального состояния. Часто в современных кодах рекомендуется использовать стрелочные функции, которые не имеют собственный `this`, беря контекст из лексического окружения, что совместимо с строгим режимом и облегчает работу с методами.
