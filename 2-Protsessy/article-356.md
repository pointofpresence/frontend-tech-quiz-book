# Какие ещё области памяти ты знаешь, кроме стека и кучи?

Категория: Процессы

### Краткий ответ

**Области памяти кроме стека и кучи**

- **Data (Data Segment)** — для статических и глобальных переменных, выделяется при загрузке программы
- **BSS (Block Started by Symbol)** — подмножество Data для неинициализированных глобальных переменных, зануляется автоматически
- **Heap (Куча)** — динамическая память, управляется во время выполнения (malloc/new)
- **Stack (Стек)** — для локальных переменных и вызовов функций, LIFO структура
- **Text (Code Segment)** — область, где хранится исполняемый машинный код программы, обычно только для чтения
- **Memory-mapped I/O** — выделяется для взаимодействия с аппаратным обеспечением и встроенными устройствами
- **TLS (Thread Local Storage)** — область памяти для переменных, уникальных для каждого потока
- **Registers** — высокоскоростные ячейки памяти внутри CPU для быстрого доступа

Эти области разделяют управление памятью и оптимизируют использование ресурсов для разных целей выполнения программ.

### Подробный ответ
#### Основной ответ
Помимо **стека** (stack) и **кучи** (heap), в памяти программы выделяют несколько других важных областей: **data segment (сегмент данных)**, **bss segment**, **text segment (сегмент кода)** и **memory-mapped area**. Каждая из них играет свою роль в управлении памятью процесса и отличается по типу хранимых данных и времени жизни.

#### Ключевые моменты
- **Text segment (сегмент кода)**: содержит исполняемый машинный код программы. Эта область обычно доступна только для чтения, чтобы защитить код от изменений и повысить безопасность. Например, в ELF-файлах Linux это `.text`.
- **Data segment** делится на:
  - **Initialized data segment** — для глобальных и статических переменных, инициализированных значениями при запуске.
  - **BSS segment** — для глобальных и статических переменных без явной инициализации (заполняется нулями).
- **Memory-mapped area** — это область, куда OS может маппировать файлы или устройства, а также динамические библиотеки (shared objects). В ней хранятся динамически загруженные библиотеки (например, libc.so) и файлы, использующие mmap.
- **Stack и heap** отличаются по времени жизни объектов: стек — локальные переменные с четкой структурой вызовов, куча — динамические объекты с управлением вручную или сборщиком мусора.
- Кроме того, существуют аппаратные кеши и регистры процессора, которые тоже играют роль в управлении памятью, но на уровне процессора, а не процесса.

#### Практический контекст
В системном программировании (например, на C/C++) понимание сегментов памяти важно для оптимального размещения данных, отладки и предотвращения ошибок, таких как переполнение стека или утечки кучи. Также разделение памяти позволяет OS эффективно управлять процессами и защитой памяти. Встраиваемые системы могут использовать сегменты для размещения кода в энергонезависимой памяти, а данных — во флеше или RAM.
