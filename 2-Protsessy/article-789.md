# Для чего нужны сайд-эффекты (LaunchedEffect, DisposableEffect)? Приведите примеры.

**Для чего нужны сайд-эффекты (LaunchedEffect, DisposableEffect)? Примеры**

LaunchedEffect, DisposableEffect — ключевые API управления сайд-эффектами в Jetpack Compose.

- **LaunchedEffect:** запускает корутину с момента создания ключа, автоматически отменяется при изменении ключа или уничтожении композиции  
- используется для **запуска асинхронных операций** (например, загрузка данных, навигация)  
- пример: загрузка данных при открытии экрана (fetchData внутри LaunchedEffect(key))

- **DisposableEffect:** запускается вместе с жизненным циклом композиции, позволяет выполнять **очистку освобождение ресурсов**  
- принимает callback на очистку, вызывается при выходе из композиции или изменении ключа  
- пример: регистрация/отписка слушателей, освобождение ресурсов (например, отмена подписки)

- оба корректно связывают **жизненный цикл эффектов с композицией**, предотвращая утечки и избыточные вызовы  

Итог: сайд-эффекты нужны для **безопасного взаимодействия с внешним миром** (async процессы, ресурсы) в декларативном UI на Compose.

# Подробный ответ
## Основной ответ  
В **Jetpack Compose** сайд-эффекты, такие как **LaunchedEffect** и **DisposableEffect**, используются для управления жизненным циклом действий, которые выходят за рамки простого рендеринга UI и требуют выполнения кода с побочными эффектами — например, запуск корутин, подписка на события или очистка ресурсов. Они помогают интегрировать Compose с асинхронными и внешними операциями, правильно управляя состоянием с учётом recomposition.

**LaunchedEffect** запускает указанную корутину при первом вызове и автоматически отменяет её при уничтожении composable или изменении ключа, предотвращая утечки.

**DisposableEffect** позволяет выполнять действия при создании composable, а также чистить ресурсы в момент его удаления — например, отписываться от слушателей, останавливать таймеры.

## Ключевые моменты  
- **LaunchedEffect** — удобен для асинхронных операций, которые должны запуститься в момент монтирования UI, например, загрузки данных, инициирования анимаций. Использует ключи для повторного запуска, если параметры изменились.  
- **DisposableEffect** — полезен когда нужно выделить и освободить ресурсы при появлении и исчезновении composable, например, register/unregister BroadcastReceiver или listeners.  
- Оба работают в контексте **композиции** и автоматически учитывают ситуацию recomposition, что исключает дублирование или утечки вызовов.

## Практический контекст  
Например, в приложении с авторизацией в LoginScreen:  
- **LaunchedEffect(Unit)** запускает проверку сессии или запрос к API при появлении экрана.  
- **DisposableEffect** может отписывать от push-уведомлений или останавливать таймер загрузки, после ухода со страницы.  
  
Это элегантный способ управлять побочными эффектами в декларативном UI, обеспечивая чистоту и удобство сопровождения кода.
