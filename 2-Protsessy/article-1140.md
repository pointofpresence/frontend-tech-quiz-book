# Что такое приватная переменная в Go? Как она объявляется и какова её область видимости?

**Что такое приватная переменная в Go?**

- область видимости: **пакетная** (package-private)
- имя начинается с **строчной буквы**
- доступна **только внутри текущего пакета**
- объявляется как обычная переменная с идентификатором, например: `var myVar int`
- используется для **инкапсуляции** данных и ограничения внешнего доступа
- напротив, имя с заглавной буквы делает переменную **экспортируемой** (публичной)
- важна для контроля архитектуры и API библиотеки

Пример:  
```go
package example

var privateVar int // приватная, доступна только в пакете example
var PublicVar int  // публичная, доступна из других пакетов
```

# Подробный ответ
## Основной ответ
В **Go** приватная переменная — это переменная, видимая только внутри пакета, где она объявлена. В Go **инкапсуляция реализуется через видимость по имени**: если имя переменной (или функции, типа, константы) начинается с **заглавной буквы**, она экспортируется и доступна за пределами пакета. Если имя начинается с **строчной буквы**, то эта переменная считается приватной и недоступной снаружи.

## Ключевые моменты
- Приватная переменная объявляется просто с именем, начинающимся с **маленькой буквы**, например `count int` или `userName string`.
- Область видимости такой переменной — **весь пакет**, в котором она объявлена (не только функция, а пакет в целом).
- Это отличается от многих других языков, где «приватность» означает ограничение по классу или объекту — в Go упаковка и экспорт управляются на уровне пакета.

## Практический контекст
Типично в реальном проекте приватные переменные используются для служебных данных, функций и структур, которые не должны быть доступны вне пакета, что помогает контролировать API и предотвращает случайное изменение внутренних состояний модуля. Например, в пакете `database` переменная `dbPool` с маленькой буквы будет приватной, а `Connect()` с заглавной — экспортируемой функцией.
