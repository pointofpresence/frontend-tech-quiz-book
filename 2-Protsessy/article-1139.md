# Что такое Small Object Heap и Large Object Heap и зачем они нужны?

Категория: Процессы

### Краткий ответ

**Small Object Heap и Large Object Heap**

- компоненты **управляемой памяти** .NET CLR  
- **Small Object Heap (SOH)** — для объектов размером < 85 КБ  
- **Large Object Heap (LOH)** — для объектов ≥ 85 КБ  
- SOH быстро выделяет и освобождает память через **генерации** и **компактирование**  
- LOH хранит крупные объекты, **редко компактен**, освобождается реже  
- разделение оптимизирует **производительность GC**: SOH для мелких объектов, LOH для больших  
- снижение фрагментации и повышение эффективности за счет разных стратегий управления памятью  

Итог: SOH и LOH нужны для **эффективного управления разной по размеру памятью** в .NET, минимизируя GC издержки и улучшая производительность приложений.

### Подробный ответ
#### Основной ответ
**Small Object Heap (SOH)** и **Large Object Heap (LOH)** — это две части управляемой памяти в среде выполнения .NET CLR, предназначенные для хранения объектов разного размера. SOH используется для мелких объектов (обычно меньше 85 000 байт), LOH — для больших объектов (от 85 Кб и выше). Разделение обеспечивает оптимизацию работы сборщика мусора и эффективное управление памятью.

#### Ключевые моменты
- **Small Object Heap** выделяет и управляет памятью для множества маленьких, часто создаваемых и удаляемых объектов. Он организован в поколения (Gen 0, Gen 1, Gen 2), что позволяет эффективно собирать мусор с минимальными задержками.
- **Large Object Heap** предназначен для объектов большого размера, таких как большие массивы или строки, которые создаются реже и живут дольше. LOH не подвергается частым поколениям GC, сборка происходит обычно только при Gen 2 GC, что снижает накладные расходы на копирование больших блоков памяти.
- Trade-off: LOH может приводить к фрагментации памяти, поскольку большие объекты сложно компактировать, потому что они не перемещаются в оперативной памяти во время большинства сборок. В .NET 4.5+ появилась поддержка компактирования LOH (LOH compacting), но она не включается по умолчанию.

#### Практический контекст
Понимание SOH и LOH важно при оптимизации приложений под .NET, особенно там, где создаются или одновременно хранятся большие объемы данных. Например, часто используют буферы или массивы больших размеров — они окажутся в LOH, и неконтролируемое выделение таких объектов может привести к падению производительности из-за частого Gen 2 GC и фрагментации памяти. В этом случае рекомендуют повторное использование таких объектов или пул объектов.
