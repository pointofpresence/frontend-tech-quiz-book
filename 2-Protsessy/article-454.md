# Какие есть состояния у промисов?

Категория: Процессы

### Краткий ответ

**Какие есть состояния у промисов?**  
- асинхронный JavaScript  
- 3 базовых состояния:  
  - **pending** — ожидание выполнения  
  - **fulfilled** — успешно выполнен, есть результат  
  - **rejected** — отклонен, ошибка или отказ  
- переход происходит только один раз, состояние не меняется обратно  
- **цепочка обработчиков** .then/.catch срабатывает при переходе из pending  
- обеспечивает управление асинхронностью и обработку ошибок  
  
Это фундамент понимания промисов и управления асинхронным кодом в JS.

### Подробный ответ
#### Основной ответ
У **промисов (Promises)** в JavaScript существует три базовых состояния, которые отражают жизненный цикл асинхронной операции. Эти состояния описывают, как промис может изменяться с момента создания до завершения.

#### Ключевые моменты
- **Pending (Ожидание)** — начальное состояние, когда промис был создан, но асинхронная операция еще не завершена, и результат неизвестен.
- **Fulfilled (Выполнено)** — состояние, когда асинхронная операция успешно завершилась, и промис «разрешён» с определённым значением (value).
- **Rejected (Отклонено)** — состояние, когда операция завершилась с ошибкой, и промис «отклонён» с причиной (error).

Важно, что переходы между состояниями происходят один раз и без возврата: промис после перехода из pending в fulfilled или rejected уже не меняет состояния. Это гарантирует предсказуемое поведение.

#### Практический контекст
Понимание этих состояний важно для корректной обработки асинхронных результатов, например, с помощью методов `.then()` для fulfilled и `.catch()` для rejected. В React 18+ или Node.js 18+ это помогает грамотно управлять последовательностью задач и ошибками в приложениях с промисами.
