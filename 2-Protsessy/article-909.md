# Как работает @Transactional и что происходит, если метод без аннотации вызывает метод с аннотацией?

Категория: Процессы

### Краткий ответ

**Как работает @Transactional и вызов из метода без аннотации**

- @Transactional — аспект из Spring AOP для **управления транзакциями**
- Оборачивает метод в **транзакционный контекст**, откатывает/коммитит изменения автоматически
- Работает через **прокси**: только внешние вызовы запускают транзакцию
- Вызов метода с @Transactional из метода без неё **в одном классе не создает транзакцию** (прокси «перескакивается»)
- Для корректной транзакционности вызов должен быть **внешним** или через Spring bean
- Важно для гарантии **атомарности** и изоляции операций с БД

Итог: @Transactional активируется через прокси при внешних вызовах, внутренние — игнорируются, что влияет на поведение транзакций.

### Подробный ответ
#### Основной ответ
Аннотация **@Transactional** в Spring управляет транзакциями на уровне метода или класса, гарантируя целостность данных и согласованность операций. При вызове метода с @Transactional создаётся или подключается транзакционный контекст, в рамках которого все операции с базой данных выполняются как единое целое — либо все успешно, либо откатываются при ошибке. Если метод без @Transactional вызывает метод с @Transactional, то поведение зависит от того, как происходит вызов: если прямой вызов внутри одного класса (через this), то прокси Spring не сработает и аннотация игнорируется; если вызов идёт через прокси (например, наружу через другой бизмодуль), то транзакция начнётся как ожидалось.

#### Ключевые моменты
- **Прокси и транзакции:** В Spring AOP транзакции работают через прокси (JDK dynamic proxy или CGLIB). Вызов метода с @Transactional должен проходить через этот прокси, чтобы триггерить транзакцию.
- **Внутренние вызовы (self-invocation)** не проходят через прокси, следовательно, аннотация не сработает, и транзакция не начнётся при внутреннем вызове метода с @Transactional.
- **Propagation** — поведение транзакций задаётся параметром propagation. Например, propagation = REQUIRED (по умолчанию) подключается к существующей транзакции или создаёт новую.

#### Практический контекст
Часто встречается ситуация, когда в одном классе метод без @Transactional вызывает метод с аннотацией, и транзакции нет — это классическая ошибка новичков. Чтобы транзакция сработала, нужно либо делать вызов снаружи через бин Spring, либо рефакторить код, выделяя @Transactional-методы в отдельные сервисы. В крупных продуктах важно понимать механику прокси и propagation для правильного управления атомарностью бизнес-операций и предотвращения проблем с параллелизмом и откатами.
