# Какие промежуточные и терминальные операции в Stream API ты использовал?

Категория: Процессы

### Краткий ответ

**Промежуточные и терминальные операции в Stream API**

- Stream API — **конвейер обработки** данных в Java
- **Промежуточные операции**: ленивые, возвращают Stream, позволяют строить цепочки  
  - map (преобразование элементов)  
  - filter (фильтрация по условию)  
  - sorted (сортировка)  
  - distinct (удаление дубликатов)  
  - flatMap (разворачивание вложенных коллекций)  
- **Терминальные операции**: конечные, инициируют вычисление, возвращают результат или side-effect  
  - collect (сбор в коллекции, Map)  
  - forEach (итерация с побочными эффектами)  
  - reduce (агрегация в одно значение)  
  - count (подсчет элементов)  
  - anyMatch/allMatch (проверка условий на элементы)  
- Применял для **эффективной фильтрации, агрегации, преобразования** данных  
- Терминальные операции — точка запуска обработки, промежуточные — строят pipeline  
- Позволяют **безопасно и прозрачно** работать с большими наборами данных  

Использую чаще всего цепочки из map-filter-collect для преобразования и агрегации.

### Подробный ответ
#### Основной ответ
В **Java Stream API** операции делятся на две категории: **промежуточные** и **терминальные**. Промежуточные операции возвращают новый стрим и позволяют строить цепочку, а терминальные завершают обработку и могут возвращать результат или побочный эффект.

#### Ключевые моменты

- **Промежуточные операции**: Я часто использовал `filter` для отбора элементов по условию, `map` для преобразования данных (например, из объекта в поле), `flatMap` для распаковки вложенных коллекций, `sorted` для сортировки, а также `distinct` для удаления дубликатов. Эти операции ленивы и не выполняются, пока не вызовется терминальная операция.

- **Терминальные операции**: Самые частые — `collect` для агрегирования результата в коллекцию (`Collectors.toList()`, `Collectors.groupingBy()`), `forEach` для побочных эффектов, `reduce` для агрегирования с функцией свёртки, `anyMatch`, `allMatch` и `noneMatch` для проверки условий. Также использовал `count` для подсчёта элементов.

- Важно понимать, что терминальные операции запускают вычисления, а промежуточные создают pipeline без выполнения. Этот ленивый подход позволяет оптимизировать производительность.

#### Практический контекст
При работе с крупными коллекциями, например, списком пользователей, я строил pipeline из промежуточных операций для фильтрации, сортировки и трансформации данных, а затем собирал результат в нужную структуру через `collect`. Такой подход улучшал читаемость кода и снижал количество временных коллекций.
