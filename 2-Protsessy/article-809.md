# Для чего нужны const val и val? Почему const val нельзя использовать с var?

**Для чего нужны const val и val? Почему const val нельзя использовать с var?**

- **val** — ссылка на **неизменяемое** значение (read-only), инициализируется один раз, но может быть вычислена во время выполнения (run-time).
- **const val** — **константа на этапе компиляции** (compile-time), значение известно и фиксировано до запуска, должна быть примитивом или строкой.
- **var** — изменяемая переменная, допускает переназначение.
- **const val нельзя с var**, т.к. const требует неизменности и значения на этапе компиляции, а var предполагает изменяемость.
- const val используется для глобальных констант, val — для объявлений, где значение не изменяется после инициализации, но может быть runtime.
- const val экономит память и ускоряет выполнение за счет инлайнинга в байткод.
- var применяют, когда требуется состояние, изменяемое в ходе работы программы.

Итог: const val — **компиляторные константы**, val — **runtime неизменяемые ссылки**, var — **изменяемые переменные**.

# Подробный ответ
## Основной ответ
В **Kotlin** `val` и `var` — это ключевые слова для объявления переменных. `val` объявляет **неизменяемую** переменную (immutable), т.е. значение можно присвоить лишь один раз. `var` — **изменяемая** переменная, которой можно присвоить новое значение сколько угодно раз. 

`const val` — это особая форма `val`, которая указывает на **константу времени компиляции**. Она должна быть примитивного типа или строки, и её значение известно на этапе компиляции, а не во время выполнения.

Поэтому `const val` нельзя использовать с `var`, так как:
- `const` подразумевает **константу** — неизменяемое значение, являющееся compile-time constant.
- `var` — переменная, которая допускает изменение значения во время работы программы.

Таким образом, `const val` и `var` логически противоречат друг другу.

## Ключевые моменты
- **`val`**: ссылка на объект не меняется, но сам объект может быть мутируемым (например, изменяемый список).
- **`const val`**: значение фиксировано и вшито в байткод, работает только с top-level или `object` полями, доступно как компиляторная константа (Java static final).
- **Причина несоответствия `const val` и `var`** — `const` требует, чтобы значение было известно и не менялось скомпилированным кодом, а `var` — предполагает возможность изменения значения.

## Практический контекст
`const val` часто используют для объявления строковых констант, например, ключей Intent, URL или параметров, чтобы избежать магических строк в коде и обеспечить производительность. `val` применяется для переменных, значение которых нужно зафиксировать после инициализации, а `var` — когда нужно менять состояние объекта или примитивного типа во время исполнения программы.
