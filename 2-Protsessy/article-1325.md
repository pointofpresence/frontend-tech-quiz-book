# Как Python решает коллизии при хешировании?

Категория: Процессы

### Краткий ответ

**Как Python решает коллизии при хешировании?**

- контекст: реализация **словаря (dict)** в CPython
- используется метод **открытой адресации с двойным хешированием**
- при коллизии выбирается новая позиция через вычисление **вторичного хеша**
- шаг поиска непредсказуем, снижает кластеры коллизий
- хранит ключи и значения в одной структуре для оптимальной памяти
- допускает постепенное расширение таблицы для поддержания производительности (~0.66 load factor)
- результат: быстрый доступ и эффективное разрешение коллизий в большинстве случаев

### Подробный ответ
#### Основной ответ
В Python коллизии в хеш-таблицах (например, в словарях dict) решаются с помощью метода **открытой адресации с двойным хешированием** и **псевдослучайным пробингом**. При возникновении коллизии Python последовательно ищет следующую свободную ячейку, используя заранее определённую стратегию перебора индексов, что позволяет эффективно распределять элементы и минимизировать цепочки коллизий.

#### Ключевые моменты
- **Открытая адресация и perturbation**: В реализации CPython (начиная с версии 3.3+) используется perturbation — значение, комбинируемое с хешем для вычисления шага следующей проверки, что снижает вероятность кластеризации.
- **Использование специального алгоритма**: После коллизии Python меняет индекс с помощью формулы `i = (5 * i + perturb + 1) % size` с постепенным сдвигом `perturb >>= 5`, что даёт псевдослучайное чередование ячеек.
- **Оптимизация под производительность и память**: В отличие от цепочек (chaining), этот метод позволяет использовать компактный массив без необходимости дополнительной памяти для узлов или списков.

#### Практический контекст
Такой подход обеспечивает быстрый средний доступ к элементам — время поиска близко к O(1), что критично для производительности Python-словарей. В реальных проектах это помогает при работе с большими объёмами данных, где частые коллизии неизбежны, но благодаря smart probing время доступа остаётся низким.
