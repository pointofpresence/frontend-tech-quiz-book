# Какие примитивы синхронизации есть в asyncio?

Категория: Процессы

### Краткий ответ

**Примитивы синхронизации в asyncio**

- **Lock**: предотвращает одновременный доступ к ресурсу  
- **Event**: позволяет ожидать и сигнализировать событие  
- **Condition**: сочетает Lock с ожидающими потоками, для продвинутой синхронизации  
- **Semaphore**: ограничивает число одновременных задач  
- **BoundedSemaphore**: вариант Semaphore с контролем переполнения  
- **Queue**: потокобезопасная очередь для передачи данных между тасками  

Эти примитивы обеспечивают координацию копрограмм (тасков) и предотвращают гонки в асинхронном коде.

### Подробный ответ
#### Основной ответ
В **asyncio** существуют несколько основных **примитивов синхронизации**, которые помогают управлять конкурентным доступом к ресурсам и координировать асинхронные задачи. Основные из них: **Lock**, **Event**, **Condition**, **Semaphore**, а также **Barrier** (начиная с Python 3.11). Они служат для контроля порядка выполнения и предотвращения гонок данных в рамках событийного цикла.

#### Ключевые моменты
- **asyncio.Lock** — асинхронный аналог мьютекса, используется для обеспечения эксклюзивного доступа к разделяемым ресурсам. Задачи ждут блокировки с помощью `await lock.acquire()` и освобождают ее вызовом `lock.release()`.
- **asyncio.Event** — используется для уведомления одной или нескольких задач о наступлении события. Метод `wait()` блокирует до тех пор, пока не будет вызван `set()`. Отлично подходит для сигнализации.
- **asyncio.Condition** — расширяет возможности Lock, позволяя задачам ждать определенного условия с использованием `wait()` и уведомлять других через `notify()`, аналогично threading.Condition, но с async/await.
- **asyncio.Semaphore** — ограничивает количество одновременных задач, которые могут войти в критическую секцию (например, пул соединений). По умолчанию доступен счетчик, уменьшающийся при входе и увеличивающийся при выходе.
- **asyncio.Barrier** (с Python 3.11) — синхронизирует набор задач, заставляя их ждать друг друга до момента, когда все достигнут определенной точки выполнения.

#### Практический контекст
Эти примитивы активно применяются в высоконагруженных **network I/O** приложениях на asyncio (серверы, парсеры, микросервисы) для предотвращения одновременного доступа к общей памяти, координации запуска задач и ограничения параллелизма (например, одновременное количество запросов к API). Важно выбирать правильный примитив под конкретную задачу, чтобы избежать блокировок и "зависаний" в event loop, так как asyncio — однопоточный по сути и работает за счет кооперативной многозадачности.
