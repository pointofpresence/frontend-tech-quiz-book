# Зачем используется запрос с методом OPTIONS?

Категория: Процессы

### Краткий ответ

**Зачем используется запрос с методом OPTIONS?**  
- HTTP метод для **узнавания поддерживаемых методов** на сервере  
- отправляется для проверки **CORS-политики** браузером (preflight request)  
- возвращает в заголовках список **доступных HTTP методов** и разрешённых заголовков  
- позволяет клиенту понять, можно ли отправлять основной запрос с особыми методами/заголовками  
- не должен изменять состояние на сервере (безопасный, идемпотентный)  
- часто используется в **междоменных запросах** для обеспечения безопасности  
- помогает избежать ошибок при **кросс-доменном доступе**

### Подробный ответ
#### Основной ответ  
HTTP-запрос с методом **OPTIONS** используется для определения поддержки сервером определённых HTTP-методов и доступных опций для конкретного ресурса. По сути, он служит своего рода "зоной предварительного запроса", позволяя клиенту узнать, какие методы (GET, POST, PUT и т.д.) или заголовки разрешены перед выполнением основного запроса. Особенно важен в контексте **CORS** (Cross-Origin Resource Sharing), где браузеры отправляют preflight-запросы OPTIONS, чтобы проверить политику безопасности сервера.

#### Ключевые моменты  
- **CORS preflight:** Когда фронтенд пытается выполнить запрос с нестандартным методом или заголовками, браузер шлёт OPTIONS-запрос, чтобы убедиться, что сервер разрешает этот запрос и не нарушит политику безопасности.  
- **Идентификация поддерживаемых методов:** OPTIONS помогает клиенту понять, какие методы допустимы для ресурса, тем самым предотвращая ошибки и излишние запросы.  
- **Отделение "узнавания" от действий:** OPTIONS-запрос не должен изменять состояние сервера — он идемпотентен и может служить диагностическим инструментом.  

#### Практический контекст  
В современных SPA на React 18+ с backend на Node.js или Django OPTIONS-запрос используется автоматически браузером при вызове fetch с нестандартными заголовками Authorization или Content-Type, отличающимся от `application/x-www-form-urlencoded`. На сервере часто настраивают middleware, чтобы отвечать на OPTIONS 204 статусом и нужными заголовками CORS, что помогает избежать задержек или ошибок при взаимодействии front и backend.
