# Чем отличается TCP от UDP протоколы? Для чего каждый используется?

Категория: Процессы

### Краткий ответ

**Чем отличается TCP от UDP протоколы? Для чего каждый используется?**

- транспортные протоколы уровня TCP/IP  
- TCP: **установленное соединение (connection-oriented)**  
- UDP: **без установления соединения (connectionless)**  
- TCP: гарантирует **доставку, порядок, контроль ошибок**  
- UDP: минимальная задержка, **без гарантии доставки**  
- TCP: используется для **HTTP, FTP, SMTP** (надежные приложения)  
- UDP: подходит для **стриминга, игр, DNS** (скорость важнее надежности)

### Подробный ответ
#### Основной ответ
**TCP (Transmission Control Protocol)** и **UDP (User Datagram Protocol)** — это два основных транспортных протокола в стеке TCP/IP, которые решают разные задачи передачи данных. TCP ориентирован на надежность и установление соединения, гарантируя доставку и упорядоченность пакетов. В отличие от него, UDP — это протокол без установления соединения, который не гарантирует доставку и порядок, зато работает быстрее и с меньшими накладными расходами.

#### Ключевые моменты  
- **TCP** обеспечивает надежную передачу через механизмы установления соединения (3-way handshake), контроль ошибок, подтверждения (ACK), повторную отправку потерянных пакетов и упорядочивание данных. Благодаря этому он идеально подходит для приложений, где критична целостность данных, например, веб-серверы (HTTP/HTTPS), электронная почта (SMTP), файловый обмен (FTP).
- **UDP** не реализует механизмы подтверждения и восстановления, поэтому передача может быть быстрой, но ненадёжной. UDP часто используется для real-time приложений, где важна минимальная задержка и можно допустить потерю пакетов — это видеоконференции, голосовые звонки (VoIP), онлайн-игры, DNS-запросы.
- TCP добавляет overhead из-за контроля соединения и управления потоком, что увеличивает задержки, в то время как UDP минимален по накладным расходам, что критично в системах с требованиями к низкой латентности.

#### Практический контекст  
В проектах с высокими требованиями к качеству данных и надежности (например, банковские системы, файлообменники) обычно используют **TCP**, а для мультимедийных real-time задач, где приоритет — скорость и непрерывность потока, — **UDP** с дополнительными средствами компенсирования потерь (например, FEC). В React приложениях для стриминга видео или аудио часто комбинируют UDP для передачи медиа и TCP — для управления сессией.
