# Расскажи про event loop в React.

Категория: Процессы

### Краткий ответ

**Event Loop в React**

- React работает поверх JavaScript, использующего **однопоточный event loop**  
- Event loop — механизм браузера/среды для управления асинхронными задачами  
- React связывает события UI с обновлением состояния через **свой синтетический event system**  
- После срабатывания события React ставит обновления состояния в очередь, обрабатывая их **пакетно (batching)**  
- Обновления рендерятся не мгновенно, а в следующем цикле event loop для оптимизации производительности  
- React Fiber (в React 16+) разбивает работу рендера на **маленькие фрагменты**, позволяя event loop прерывать долгие операции  
- Это обеспечивает плавность интерфейса и возможность приоритетного выполнения задач (анимации, пользовательские события)  
- В итоге event loop координирует асинхронность React: обработка событий, обновление состояния и рендер — эффективно и без блокировок  

Таким образом, **event loop обеспечивает асинхронную, приоритетную и оптимизированную обработку UI в React**.

### Подробный ответ
#### Основной ответ
**Event loop** в контексте **React** — это механизм браузера (или среды выполнения, например Node.js), который обрабатывает асинхронные задачи и синхронизацию событий. React не реализует свой event loop, а использует event loop JavaScript-движка для планирования рендеринга и обработки событий. В React 18+ появился концепт **concurrent rendering**, что тесно связано с event loop: React разбивает работу на мелкие задачи, которые ставятся в очередь, позволяя браузеру обрабатывать другие события между рендерами, обеспечивая отзывчивость UI.

#### Ключевые моменты
- **Асинхронность и очередь задач**: Браузерный event loop состоит из очереди макротасков (click, load), микротасков (Promise, MutationObserver) и callback-очередей, React планирует обновления стейта и ререндеринг через эти механизмы.
- **Concurrent Mode (React 18+)**: React может прерывать рендеринг, если пользовательское взаимодействие требует внимания — этот процесс основывается на приоритетах в event loop и API, таких как `requestIdleCallback` или `MessageChannel`.
- **Fiber architecture**: React использует Fiber — виртуальную структуру для управления задачами рендера. Fiber разбивает тяжелые операции на небольшие части, которые ставятся в очередь event loop с разной приоритетностью.

#### Практический контекст
В React 18+ concurrent features позволяют создавать UI, который не «зависает» на больших обновлениях состояния. Например, при вводе текста в поле React может приостановить тяжелую работу и отдать управление обратно браузеру, чтобы обеспечить плавный отклик. В реальных проектах это улучшает UX на устройствах с низкой производительностью и при сложных рендерах.
