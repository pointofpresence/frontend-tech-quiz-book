# Какая разница между String, StringBuffer и StringBuilder?

Категория: Процессы

### Краткий ответ

**Разница между String, StringBuffer и StringBuilder**  
- **String** — **неизменяемая** строка (immutable), при изменении создаётся новый объект  
- **StringBuffer** — **изменяемая** строка с **потоко-безопасной** синхронизацией (synchronized)  
- **StringBuilder** — как StringBuffer, но **без синхронизации**, быстрее в однопоточных задачах  
- StringBuffer и StringBuilder эффективны для **многочисленных изменений** строки (конкатенаций)  
- Использовать String для **фиксированных значений**, StringBuilder — для **быстрых изменений в однопотоке**, StringBuffer — в **многопоточной** среде  
- Основное различие — **потоко-безопасность** и производительность  
- Применение зависит от **контекста параллелизма и требований к производительности**

### Подробный ответ
#### Основной ответ
**String**, **StringBuffer** и **StringBuilder** — это классы для работы со строками в Java, но у них разные свойства по изменяемости и потокобезопасности. **String** — неизменяемый (immutable) объект, в то время как **StringBuffer** и **StringBuilder** представляют изменяемые строки (mutable), позволяя эффективно изменять содержимое без создания новых объектов. Главное отличие между **StringBuffer** и **StringBuilder** — потокобезопасность: **StringBuffer** синхронизирован (thread-safe), а **StringBuilder** — нет, поэтому он быстрее, но небезопасен для многопоточных сред.

#### Ключевые моменты
- **String** — immutable, каждый раз при изменении создается новый объект, что может быть накладно при множественных операциях конкатенации.
- **StringBuffer** — mutable, синхронизированный, подходит для многопоточных приложений, где важна безопасность данных при параллельном доступе, но из-за синхронизации медленнее.
- **StringBuilder** — mutable, не синхронизированный, оптимален для однопоточных приложений или внутри локальных методов, где синхронизация не нужна, быстрее StringBuffer на 20-30% за счёт отсутствия overhead на синхронизацию.

#### Практический контекст
В современных приложениях, особенно начиная с Java 5 и выше, **StringBuilder** — стандартный выбор для построения строк в однопоточной среде (например, внутри метода). Если требуется использовать внутри многопоточного контекста — выбирают **StringBuffer** или более комплексные средства синхронизации. Для константных строк или небольших операций лучше использовать **String** для безопасности и читабельности кода. Например, при построении SQL запросов или логических сообщений в коде часто используют именно **StringBuilder** из-за его эффективности.
