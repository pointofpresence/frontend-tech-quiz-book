# Как работает аннотация @Transactional в Spring?

**Как работает аннотация @Transactional в Spring?**

- аспектно-ориентированное программирование (AOP), управление транзакциями  
- помечает метод/класс для **автоматического** начала, фиксации и отката транзакции  
- в момент вызова создаётся/присоединяется текущая транзакция из TransactionManager  
- при успешном завершении — **commit**, при исключении — **rollback**  
- поддерживает настройку **границ**, уровней изоляции, propagation, timeout  
- в основе лежит прокси (JDK или CGLIB), решает **повторное использование** транзакций  
- применяется для упрощения и стандартизации управления **консистентностью** данных в БД

# Подробный ответ
## Основной ответ
Аннотация **@Transactional** в Spring используется для декларативного управления транзакциями на уровне методов или классов. Она позволяет разработчику указать, что вызов метода должен выполняться в рамках транзакции, которую Spring автоматически откроет, зафиксирует (commit) или откатит (rollback) в зависимости от результата выполнения.

Spring реализует это через механизм **AOP (Aspect-Oriented Programming)**: при вызове метода, помеченного @Transactional, создаётся прокси-объект, который перехватывает вызов и подготавливает транзакционный контекст. Если выполнение прошло успешно, транзакция коммитится, а при возникновении исключений (по умолчанию — непроверяемых) происходит откат.

## Ключевые моменты
- **Propagation и Isolation** — можно управлять поведением вложенных транзакций и уровнем изоляции, чтобы контролировать конкурентный доступ к данным.
- **Rollback Rules** — по умолчанию Spring откатывает транзакции при RuntimeException и Error, но можно менять это поведение через параметры аннотации.
- **PlatformTransactionManager** — абстрактный интерфейс, через который Spring взаимодействует с конкретной системой управления транзакциями (например, JPA, JDBC, JTA).

## Практический контекст
В реальных проектах в Spring Boot 2.x и выше @Transactional часто ставится на сервисные методы для обеспечения атомарности бизнес-операций с базой данных. Вместе с JPA и Hibernate это позволяет избегать ручного управления транзакциями, упрощая код и повышая надёжность. Например, при создании заказов и списании товаров из склада вся логика может выполняться внутри одной транзакции с гарантией консистентности данных.
