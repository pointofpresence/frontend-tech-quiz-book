# Чем отличаются синхронные и асинхронные задачи в GCD?

Категория: Процессы

### Краткий ответ

**Чем отличаются синхронные и асинхронные задачи в GCD?**

- GCD (Grand Central Dispatch) — система управления задачами в iOS/macOS  
- **Синхронные задачи (sync):** вызывающий поток блокируется до завершения задачи  
- Выполнение происходит **последовательно**, ждать результата обязательно  
- **Асинхронные задачи (async):** задача ставится в очередь, вызывающий поток не блокируется  
- Задачи выполняются **параллельно/отложенно**, управление возвращается сразу  
- Выбор sync или async влияет на **плавность интерфейса:** async не блокирует UI  
- Используют sync для операций, требующих результата немедленно, async — для фоновых задач  
- Важно избегать блокировок и дедлоков при sync, особенно на главном потоке  
- Практика: обновление UI — только async на главном потоке, тяжелые — async на фоновом  

Так кратко суть отличий — блокировка vs неблокировка и момент возврата управления.

### Подробный ответ
#### Основной ответ
В **Grand Central Dispatch (GCD)** синхронные и асинхронные задачи отличаются тем, как происходит управление потоком исполнения. **Синхронная задача** (dispatch_sync) блокирует текущий поток до тех пор, пока задача не выполнится, тогда как **асинхронная задача** (dispatch_async) ставит задачу в очередь и немедленно возвращает управление, позволяя текущему потоку продолжить работу.

#### Ключевые моменты
- **Блокировка потока**: `dispatch_sync` выполняет задачу на указанной очереди и ждет завершения, что может привести к дедлокам, если вызвана из той же очереди. `dispatch_async` просто добавляет задачу и не блокирует вызывающий поток.
- **Параллелизм**: `dispatch_async` позволяет увеличить параллельность, особенно при работе с глобальными concurrent очередями или custom очередями, тогда как `dispatch_sync` гарантирует последовательное выполнение и синхронизацию.
- **Использование**: `dispatch_sync` применяют, когда результат задачи нужен сразу, а `dispatch_async` — для фоновой, неблокирующей работы (например, обновление UI из background).

#### Практический контекст
В iOS с GCD часто используют `dispatch_async` для выполнения сетевых запросов и тяжелых вычислений в background, чтобы UI оставался отзывчивым. `dispatch_sync` применяют для синхронизации доступа к данным внутри serial очередей, но осторожно, чтобы избегать взаимной блокировки. Например, в ReactOS13+ часто рекомендуют использовать `dispatch_async` с main queue для обновления интерфейса без блокировки.
