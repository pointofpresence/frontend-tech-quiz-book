# Стандартные способы управления потоками (ExecutorService, Future)

Категория: Процессы

### Краткий ответ

**Стандартные способы управления потоками (ExecutorService, Future)**  
- Java concurrency API для управления потоками  
- **ExecutorService** — интерфейс для пула потоков, управление жизненным циклом  
- **submit()** запускает задачи, возвращает **Future**  
- **Future** — результат асинхронной операции, позволяет получить результат с блокировкой  
- методы Future: **get()** (ожидание результата), **cancel()** (отмена задачи), **isDone()** (проверка завершения)  
- ExecutorService повышает **масштабируемость** и упрощает управление потоками  
- применяется для параллельного **выполнения задач** и контроля времени ожидания результата

### Подробный ответ
#### Основной ответ
Для управления потоками в Java стандартным способом является использование интерфейса **ExecutorService**, который представляет собой высокоуровневый API для асинхронного выполнения задач. Вместо прямого создания и управления потоками через `Thread` мы создаём задачи (Runnable или Callable) и передаём их в ExecutorService, который сам управляет пулом потоков. Результаты асинхронных вычислений часто получают через объект **Future**, который позволяет проверить состояние задачи, отменить её или получить результат после завершения.

#### Ключевые моменты
- **ExecutorService**: инкапсулирует пул потоков, стандартизирует запуск и управление задачами; есть разные реализация (FixedThreadPool, CachedThreadPool, ScheduledThreadPool).
- **Future**: представляет результат будущей асинхронной операции; метод `get()` блокирует вызывающего, ожидая завершения задачи; есть методы для отмены.
- С появлением **CompletableFuture** (Java 8+) появилась возможность более гибко обрабатывать асинхронные цепочки и события, что расширяет возможности над Future.
- ExecutorService лучше, чем ручное создание потоков, так как позволяет избежать проблем с управлением количеством потоков и ресурсами, улучшая производительность и устойчивость.

#### Практический контекст
В реальных приложениях ExecutorService применяется для параллельной обработки задач, например, при запросах к внешним сервисам, обработке больших объёмов данных или выполнении независимых вычислений. Использование Future даёт возможность синхронизации результатов без блокировки основного потока, а CompletableFuture позволяет строить сложные асинхронные конвейеры с удобным API и поддержкой обработки ошибок.
