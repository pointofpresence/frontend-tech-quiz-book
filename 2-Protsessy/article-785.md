# В чём разница между remember и rememberSaveable? Где хранится значение remember?

**Разница между remember и rememberSaveable, где хранится значение remember**

- **контекст**: управление состоянием в Jetpack Compose  
- **remember**: сохраняет значение в памяти **компоновщика**  
- **значение remember теряется** при пересоздании компонента (например, после поворота экрана)  
- **rememberSaveable**: сохраняет значение в **SavedStateHandle/Bundle**  
- **значение rememberSaveable выживает** при процессных и конфигурационных изменениях  
- хранение remember — в **памяти Runtime** компоновщика, без персистентности  
- rememberSaveable применяют для сохранения состояния UI при пересоздании Activity/Fragment  
- Практическое применение: remember — для временного состояния, rememberSaveable — для выживания между пересозданиями UI

# Подробный ответ
## Основной ответ
**`remember`** и **`rememberSaveable`** — это API в Jetpack Compose для сохранения состояния между рекомпозициями, но с важным отличием в том, как они справляются с сохранением состояния при изменении конфигурации, например, повороте экрана. Значение, сохранённое с помощью `remember`, хранится в **памяти текущей сессии композа** и теряется при пересоздании Activity или процесса. В отличие от него, `rememberSaveable` автоматически сохраняет и восстанавливает значение через `SavedStateHandle` (или Bundle), позволяя переживать пересоздание компонента, например, при повороте экрана.

## Ключевые моменты
- **`remember`** хранит состояние только в памяти текущей сессии UI-компонента, работает на время жизни композиции, но без сохранения при конфигурационных изменениях.
- **`rememberSaveable`** использует механизмы сохранения состояния Android (`SavedStateRegistry`), автоматически сериализует и восстанавливает данные (например, примитивы, Parcelable и другие), защищая от потери данных при пересоздании Activity.
- Значения, сохранённые в **`remember`**, помещаются в память процессора и доступны только пока жив Compose UI. Это быстро, но нестабильно для длительных случаев.
- Можно расширить поведение `rememberSaveable`, передав кастомные `Saver`s для поддержки сложных типов.

## Практический контекст
`remember` типично используется для временных состояний внутри однозначно живущей UI сессии, например, флагов видимости. `rememberSaveable` идеален для форм, пользовательских вводов и данных, которые должны сохраниться при поворотах экрана или возобновлении работы приложения, снижая необходимость ручной работы с `ViewModel` или `onSaveInstanceState`. В React Native похожая задача решается через state + `AsyncStorage` или context с персистентным слоем.
