# Что такое инвариант класса (class invariant)? Приведите пример.

**Что такое инвариант класса (class invariant)?**

- концепция в ООП, свойства объекта  
- условие, всегда истинное в течение жизни объекта  
- гарантирует корректность состояния объекта  
- проверяется после конструктора и публичных методов  
- предотвращает нарушение логики бизнес-правил  
- пример: баланс аккаунта ≥ 0 всегда  
- нужен для надежного, предсказуемого поведения класса

# Подробный ответ
## Основной ответ
**Инвариант класса** — это условие или набор условий, которые должны всегда оставаться истинными для объекта класса в течение всего его жизненного цикла после создания. Инварианты гарантируют корректное состояние объекта между вызовами методов и обеспечивают предсказуемость поведения. Они устанавливаются в конструкторе и поддерживаются всеми методами, которые изменяют состояние объекта.

## Ключевые моменты
- **Цель инвариантов** — защита целостности данных класса, предотвращение попадания объекта в неконсистентное состояние.
- **Поддержка инвариантов**: любой изменяющий метод (setter, mutator) должен сохранять инварианты, иначе класс становится ненадежным.
- Инварианты отличаются от предусловий и постусловий: предусловия проверяются перед выполнением метода, постусловия — после, а инварианты — всегда между методами у объекта.

## Пример
Рассмотрим класс **Rectangle** с инвариантом, что ширина и высота всегда положительны:
```python
class Rectangle:
    def __init__(self, width, height):
        assert width > 0 and height > 0, "Ширина и высота должны быть положительными"
        self.width = width
        self.height = height

    def set_width(self, width):
        assert width > 0, "Ширина должна быть положительной"
        self.width = width

    def set_height(self, height):
        assert height > 0, "Высота должна быть положительной"
        self.height = height
```
Здесь инвариант — `width > 0 and height > 0`, и он проверяется в конструкторе и методах изменения.

## Практический контекст
В промышленной разработке инварианты часто оформляют через **assert**, контракты (Design by Contract) или валидационные методы при работе с бизнес-логикой, что значительно упрощает отладку и повышает надежность кода. Например, в банковских приложениях инвариант может гарантировать, что баланс счета не падает ниже нуля.
