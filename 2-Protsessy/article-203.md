# Какие еще есть HTTP-методы, кроме GET, PUT, POST, PATCH?

Категория: Процессы

### Краткий ответ

**HTTP-методы кроме GET, PUT, POST, PATCH**

- DELETE — удаление ресурса
- HEAD — заголовки без тела ответа
- OPTIONS — описание поддерживаемых методов
- TRACE — диагностический цикл запроса
- CONNECT — установка туннеля (прокси, HTTPS)
- PATCH — частичное обновление ресурса (упомянут, для полноты)
- Метод POST часто используется для создания, PUT — для замены (для контекста)
- Используются для управления состоянием и ресурсами через REST API

Всегда поясняй назначение метода и область применения для чёткости.

### Подробный ответ
#### Основной ответ
Помимо часто используемых HTTP-методов **GET**, **PUT**, **POST** и **PATCH**, существует ряд других методов, которые играют важную роль в семантике REST API и управлении ресурсами. Они позволяют выполнять различные операции помимо чтения и изменения данных.

#### Ключевые методы кроме перечисленных:
- **DELETE** — удаление ресурса на сервере. Используется для полного удаления указанного URI.
- **HEAD** — возвращает только заголовки ответа от сервера без тела. Позволяет проверить, например, изменился ли ресурс (по ETag или Last-Modified), без передачи самого содержимого.
- **OPTIONS** — используется для получения информации о поддерживаемых сервером HTTP-методах или других возможностях для конкретного ресурса. Часто применяется в CORS предзапросах.
- **TRACE** — возвращает запрос обратно клиенту, используется для диагностики и отладки.
- **CONNECT** — устанавливает туннель к серверу, например, используется при организации SSL через прокси (HTTP-прокси туннелирование).
- **LINK и UNLINK** — менее популярные методы, определённые в некоторых расширениях HTTP, для управления связями между ресурсами.

#### Ключевые моменты
- Методы **DELETE**, **HEAD**, и **OPTIONS** являются частью стандартного набора HTTP/1.1 (RFC 7231) и широко поддерживаются.
- **TRACE** и **CONNECT** часто ограничены или отключены по соображениям безопасности, так как могут представлять уязвимости.
- Дополнительные нестандартные или экспериментальные методы встречаются редко, но в некоторых API и протоколах могут использоваться по специфике.

#### Практический контекст
Например, в REST API метод **OPTIONS** часто применяется для реализации CORS, позволяя браузеру уточнить, доступны ли кросс-доменные запросы. **HEAD** используется, чтобы проверить, не нужно ли обновлять кэш, без загрузки тяжелого контента. А **DELETE** применяется для удаления записей в базе через API. Понимание всех методов дает возможность корректно проектировать API с точным соблюдением HTTP-стандартов и особенностей.
