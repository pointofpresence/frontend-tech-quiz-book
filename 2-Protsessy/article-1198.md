# Какие операторы используются для обработки ошибок во Flow?

Категория: Процессы

### Краткий ответ

**Операторы обработки ошибок во Flow**

- Flow — статический **статический типизатор** для JavaScript, не предоставляет традиционные операторы обработки исключений
- Ошибки в Flow выявляются на этапе **статического анализа**, не во время выполнения
- Вместо операторов типа try/catch Flow работает через **аннотации типов**, чтобы предотвратить ошибки типов
- Для работы с возможными ошибками Flow использует конструкции типов, например: **?T** (nullable), **union types** (например, `T | Error`)
- Flow не имеет встроенных операторов обработки исключений, ошибки runtime обрабатываются стандартными механизмами JS (try/catch)
- Основная задача Flow — выявлять потенциальные ошибки заранее, еще до запуска кода
- Применение Flow повышает безопасность типов и снижает количество ошибок, но контроль ошибок времени выполнения делегируется JavaScript

**Вывод:** в Flow для обработки ошибок используются типовые аннотации и проверки, а не отдельные операторы; runtime-ошибки обрабатываются стандартным JS try/catch.

### Подробный ответ
#### Основной ответ
В **Flow** (библиотеке для работы с реактивными потоками в Kotlin или аналогичных реактивных фреймворках) для обработки ошибок используются специальные операторы, позволяющие реагировать на ошибочные события, перехватывать их, трансформировать или восстанавливать поток данных. Основные операторы работы с ошибками — это **catch**, **onCompletion**, **retry**, а также возможность использовать стандартные конструкции try/catch внутри потоков.

#### Ключевые моменты
- **catch** — оператор, который перехватывает исключения, возникшие в upstream Flow, и позволяет обработать их или заменить ошибочный поток на другой, не прерывая весь поток данных. Например, `catch { e -> emit(defaultValue) }`.
- **onCompletion** — вызывается при завершении потока, включая ошибочное завершение, и может использоваться для логирования, очистки ресурсов или условной обработки ошибки, опираясь на переданный параметр `cause` (исключение, если есть).
- **retry** — оператор, который позволяет повторять выполнение upstream Flow при возникновении ошибки, с возможностью задать число попыток или задержки между ними.
- Внутри блоков **flow { }** допускается использование **try/catch** для точечной обработки ошибок при эмиссии, что даёт гибкий контроль над ошибками.

#### Практический контекст
В продакшен-коде часто комбинация этих операторов помогает реализовать устойчивое и отказоустойчивое поведение потоков: например, при сетевых запросах `retry` используем для повторных попыток, `catch` — для замены ошибки на fallback данные, `onCompletion` — для завершающих действий и логирования. Нативные средства Flow обеспечивают эффективную и декларативную обработку исключений без блокировки потоков.
