# Как запустить тест с разными параметрами в pytest?

Категория: Процессы

### Краткий ответ

**Как запустить тест с разными параметрами в pytest?**

- **Параметризация** тестов через декоратор `@pytest.mark.parametrize`
- Подписываешь функцию теста на набор параметров (имена переменных + списки значений)
- pytest запускает тест **повторно для каждого набора** параметров
- Позволяет покрыть разные сценарии одним тестом без дублирования кода
- Позволяет удобно проверять **комбинации входных данных**
- Используется для **unit** и **интеграционных** тестов с разными входными значениями
- Практика помогает находить **граничные** и **единичные** случаи быстро

Пример:
```python
@pytest.mark.parametrize("input,expected", [(1,2), (3,4)])
def test_inc(input, expected):
    assert input + 1 == expected
```
Этот код запускает тест дважды с разными парами параметров.

### Подробный ответ
#### Основной ответ
В **pytest** для запуска тестов с разными параметрами используется механизм **параметризации** — специальная декорация `@pytest.mark.parametrize`. Она позволяет запускать одну и ту же тестовую функцию несколько раз с различными наборами входных данных, что значительно упрощает тестирование различных сценариев без дублирования кода.

#### Ключевые моменты
- Декоратор принимает два основных аргумента: имена параметров (строкой или списком) и список значений (кортежей или списков), например:  
  ```python
  @pytest.mark.parametrize("input,expected", [(1,2), (3,4)])
  ```
- Можно параметризовать несколько аргументов одновременно, что ведёт к комбинации всех вариантов. Это удобно для кросс-комбинаций разных входных данных.
- В **pytest 6+** появилась поддержка **indirect parametrization** — значения параметров передаются в фикстуры для более гибкой подготовки окружения.
- Для запуска конкретных параметризированных тестов при необходимости можно использовать опцию CLI `-k` с фильтром по имени или части имени теста, включая параметры.

#### Практический контекст
В реальных проектах параметризация помогает покрыть множество случаев, например, проверять работу функций с разными типами данных, граничными значениями или комбинациями конфигурационных флагов без написания множества тестов вручную. Это улучшает покрытие и снижает технический долг.

---
Пример базовой параметризации:
```python
@pytest.mark.parametrize("x, y, expected", [(1,2,3), (4,5,9), (10,20,30)])
def test_sum(x, y, expected):
    assert x + y == expected
```
Этот тест запустится трижды с разными входными значениями.
