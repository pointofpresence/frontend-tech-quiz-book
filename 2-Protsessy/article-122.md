# Что такое команда rsync и как ее использовать для синхронизации?

Категория: Процессы

### Краткий ответ

**Что такое команда rsync и как ее использовать для синхронизации?**  
- утилита для эффективной **синхронизации файлов**
- работает по алгоритму **дифференциального копирования** (копирует только изменённые части)  
- поддерживает локальное и удалённое копирование по SSH  
- основные ключи:  
  - `-a` — архивный режим (рекурсивно, сохраняет права, симлинки)  
  - `-v` — подробный вывод  
  - `-z` — сжатие при передаче по сети  
- пример использования для локальной синхронизации: `rsync -av /src/ /dst/`  
- пример для удалённого: `rsync -avz /src/ user@host:/dst/`  
- широко используется для **резервного копирования** и деплоя

### Подробный ответ
#### Основной ответ
**rsync** — это утилита командной строки для эффективной синхронизации и передачи файлов между локальными папками или удалёнными серверами. Она позволяет копировать только изменённые части файлов (дифференциальное копирование), что значительно снижает сетевой трафик и время передачи.

#### Ключевые моменты
- **Индикатор изменений**: rsync использует алгоритм с контрольными суммами (checksum-based delta transfer), передавая только изменённые блоки файла, а не весь файл целиком.
- **Поддержка протоколов**: синхронизация возможна локально или через SSH, что обеспечивает безопасность передачи.
- **Опции**: часто применяются `-a` (archive mode — сохраняет права, симлинки, временные метки), `-v` (verbose — подробный вывод), `--delete` (удаление файлов, отсутствующих в источнике) и `-z` (сжатие данных в процессе передачи).

#### Практический контекст
Например, чтобы синхронизировать локальную папку `/data/` с удалённой `/backup/data/` на сервере `example.com`, безопасно и эффективно, используют:
```
rsync -avz --delete /data/ user@example.com:/backup/data/
```
Этот пример поддерживает целостность данных, экономит трафик и поддерживает копию актуальной за счёт удаления удалённых файлов. В крупных инфраструктурах rsync широко применяется для резервного копирования и миграций данных, а также как дельта-репликация в CI/CD пайплайнах.

Таким образом, rsync — надёжный и оптимальный инструмент для синхронизации данных в системном администрировании и разработке.
