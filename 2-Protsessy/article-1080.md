# Если класс финализирован (final), но реализует метод протокола, какой тип диспетчеризации будет использоваться?

Категория: Процессы

### Краткий ответ

**Диспетчеризация методов в final классе**

- final класс нельзя наследовать, его метод нельзя переопределить
- метод протокола реализуется статически, без виртуального вызова
- компилятор применяет статический или прямой вызов (devirtualization)
- нет runtime lookup, вызов быстрее (обычно inline)
- если бы класс не final — динамическая диспетчеризация (вызов по vtable)
- итог: в final классе метод протокола вызывается статически, без полиморфизма
- практическая выгода — повышение производительности вызова и упрощение анализа кода

### Подробный ответ
#### Основной ответ
Если класс в языке программирования, поддерживающем протоколы и финализацию (например, Swift), объявлен как **final** и реализует метод протокола, то для вызова этого метода будет использоваться **статическая (или прямое) диспетчеризация**. Это значит, что компилятор может напрямую привязать вызов к конкретной реализации метода класса без дополнительного поиска во время выполнения.

#### Ключевые моменты
- В случае **final класса** известно, что никто не сможет наследоваться и переопределить методы, потому компилятор может оптимизировать вызовы.
- Для методов протокола обычно применяется **динамическая диспетчеризация** (через таблицы виртуальных функций или witness tables), чтобы обеспечить полиморфизм.
- Однако, когда класс финализирован, динамическая диспетчеризация для методов протокола заменяется на статическую, так как тип полностью известен на этапе компиляции — это улучшает производительность за счёт отсутствия indirection.

#### Практический контекст
В Swift, например, если протокол имеет метод, а класс объявлен как final и реализует этот протокол, вызовы метода через переменную типа протокола компилируются в статический вызов. Это уменьшает runtime overhead, что особенно важно для performance-critical участков кода. Такой подход обычно используется в Swift 5+ с улучшениями в оптимизации ABI.
