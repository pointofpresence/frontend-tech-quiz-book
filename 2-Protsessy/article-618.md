# В чём разница между фабричным методом и билдером?

**Разница между фабричным методом и билдером**  
- Паттерны **создания объектов**, решают разные задачи  
- **Фабричный метод**: создаёт объекты через **делегирование подклассам**  
- Использует **полиморфизм** для выбора конкретного класса  
- **Билдер**: шаг за шагом строит **сложный объект**  
- Отделяет процесс конструирования от представления  
- Подходит для объектов с **множеством параметров** и вариаций  
- Используется, когда нужно контролировать **порядок инициализации**  
- Фабричный метод отвечает за **выбор типа**, билдер — за **пошаговое построение**  
- Применение: фабрика — для **однородных продуктов**, билдер — для **комплексных сборок**

# Подробный ответ
## Основной ответ
**Фабричный метод** и **Builder** — это порождающие паттерны проектирования, предназначенные для создания объектов, но решающие разные задачи. Фабричный метод предоставляет интерфейс для создания объектов, позволяя подклассам изменить тип создаваемого объекта, а Builder предназначен для поэтапной и гибкой сборки сложных объектов с множеством опций.

## Ключевые моменты
- **Фабричный метод**: сосредоточен на выборе конкретного класса продукта из семейства, инкапсулирует логику выбора и создания объекта; часто используется, когда система должна работать с разными типами продуктов через общий интерфейс. Например, в Java 8 `StreamSupport.stream()` возвращает разные реализации в зависимости от параметров.
- **Builder**: позволяет создавать сложные объекты пошагово, отделяя процесс конструирования от представления; удобен при необходимости создавать объекты с множеством опциональных параметров или конфигураций, например, в `StringBuilder` или сложных конфигурациях сетевых запросов.
- **Trade-off**: фабричный метод проще и чаще используется для выбора одного из готовых объектов, тогда как Builder даёт гибкость и контролируемость при настройке сложных объектов, но требует больше кода и шаблонов.

## Практический контекст
Типично фабричный метод применяют для создания объектов в системах с плагинами или разными реализациями интерфейсов, например, драйверов. Builder используют для конструирования запросов к API, создания сложных DTO или конфигов, где параметры могут иметь много вариантов и комбинаций без комбинаторного взрыва конструкторов.
