# Как работает параметризация тестов в pytest?

Категория: Процессы

### Краткий ответ

**Как работает параметризация тестов в pytest?**

- функционал **pytest** для многократного запуска теста
- декоратор **@pytest.mark.parametrize**
- принимает имена параметров и список значений
- pytest подставляет значения поочерёдно в тест
- повышает покрытие без дублирования кода
- упрощает тестирование с разными входными данными
- широко используется для проверки разных сценариев и граничных случаев

### Подробный ответ
#### Основной ответ
**Параметризация тестов в pytest** — это механизм, позволяющий запускать один и тот же тестовый метод с разными наборами входных данных. Это значительно упрощает тестирование функций с множеством вариантов, снижает дублирование кода и повышает покрытие. В pytest параметризация реализуется с помощью декоратора `@pytest.mark.parametrize`, который позволяет задать один или несколько аргументов теста и список значений для них.

#### Ключевые моменты
- `@pytest.mark.parametrize` принимает первым аргументом строку с названиями параметров (через запятую), вторым — список кортежей или значений, каждый из которых соответствует подстановке для одного запуска.
- Поддерживается параметризация нескольких аргументов одновременно, что удобно для комплексных тестов с вариативными входными данными.
- При запуске тесты с разными параметрами считаются отдельными тестами с разными id, что хорошо видно в отчетах (например, `test_func[param1-param2]`).
- Можно комбинировать с фикстурами для более гибкой и масштабируемой настройки окружения и данных.
- В pytest 7+ поддерживается более гибкий синтаксис с `ids` для понятного именования параметризованных случаев, а также возможность параметризации на уровне классов и модулей.

#### Практический контекст
Типичный пример: тест функции сложения с разными парами чисел. Вместо множества похожих тестов пишем один, параметризованный. В крупных проектах параметризация помогает покрыть множество вариантов с входными данными или конфигурациями без дублирования и облегчает поддержку тестов. Часто используется совместно с `pytest-xdist`, позволяя параллельно запускать параметры для ускорения тестирования.
