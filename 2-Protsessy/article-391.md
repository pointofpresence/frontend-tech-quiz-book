# Какие функции высшего порядка ты знаешь? Сравни map, compactMap, flatMap.

Категория: Процессы

### Краткий ответ

**Функции высшего порядка и сравнение map, compactMap, flatMap**

- **Функции высшего порядка** — принимают функции как аргументы или возвращают функции  
- **map** — преобразует каждый элемент коллекции, возвращает коллекцию того же размера  
- **compactMap** — аналог map, но пропускает nil-значения (опционалы), возвращает коллекцию без nil  
- **flatMap** — сначала преобразует элементы (как map), затем **разворачивает вложенные коллекции** в плоскую структуру  
- В Swift flatMap и compactMap отличаются: flatMap раскрывает вложенные массивы, compactMap — удаляет nil в результатах  
- Используются для **чистого, декларативного** обхода и преобразования коллекций с фильтрацией и "развёртыванием"  
- Пример: `[1, 2, nil, 4].compactMap { $0 }` уберёт nil, `[[1,2],[3,4]].flatMap { $0 }` превратит в `[1,2,3,4]`  
- Выбор зависит от цели: simple преобразование (map), фильтрация опционалов (compactMap), работа с вложенными коллекциями (flatMap)

### Подробный ответ
#### Основной ответ  
**Функции высшего порядка** — это функции, которые принимают другие функции в качестве аргументов или возвращают функции. В практике программирования на Swift (и многих других языках) часто используются такие функции, как **map**, **compactMap**, **flatMap** для работы с коллекциями и опциональными значениями.  

- **map** применяет переданную функцию ко всем элементам коллекции и возвращает массив с результатами, сохраняя количество элементов. Например, преобразование `[1, 2, 3].map { $0 * 2 }` даст `[2, 4, 6]`.  
- **compactMap** похож на map, но исключает из результата все элементы, для которых функция вернула nil. Работает как фильтр и трансформация одновременно. Например, преобразование `["1", "a", "3"].compactMap { Int($0) }` даст `[1, 3]`.  
- **flatMap** объединяет результат применения функции к элементам коллекции в один плоский массив, «разворачивая» вложенные коллекции. Более того, в Swift flatMap в работе с Optional может «распаковывать» вложенные опционалы.  
   
#### Ключевые моменты  
- **map** – преобразует элементы без изменения структуры, подходит для неглубоких преобразований.  
- **compactMap** – фильтрует nil, полезно для преобразования опциональных значений в непустой массив.  
- **flatMap** – упрощает вложенные коллекции, объединяет массивы в один уровень, также обрабатывает вложенные Optional (deprecated для Optional в Swift 5.1+, рекомендуется использовать compactMap).  
- В Swift 5.1+ рекомендуют применять **compactMap** вместо flatMap для преобразования Optional → Non-Optional, так как flatMap для Optional устарел.  
   
#### Практический контекст  
При работе с JSON парсингом compactMap помогает избавиться от невалидных значений, map используется для элементарных преобразований, а flatMap — для объединения результатов из вложенных массивов (например, собираем все теги из разных объектов в один массив). В React с JavaScript эти функции имеют аналогичный смысл, что универсально для функционального программирования.
