# Что такое CORS?

**Что такое CORS?**  
- механизм безопасности веб-браузеров  
- **Cross-Origin Resource Sharing** — обмен ресурсами между разными доменами  
- регулирует доступ к ресурсам при запросах с другого источника (домен, протокол, порт)  
- работает через заголовки HTTP (например, `Access-Control-Allow-Origin`)  
- предотвращает **межсайтовые атаки**, но позволяет легитимный обмен  
- сервер явно указывает, какие источники разрешены  
- важен для современных фронтенд-приложений с API и микросервисами

# Подробный ответ
## Основной ответ  
**CORS (Cross-Origin Resource Sharing)** — это механизм безопасности браузера, который регулирует доступ веб-страниц к ресурсам, расположенным на других доменах (origins). Он предназначен для предотвращения несанкционированных запросов и обеспечивает серверу контроль над тем, кто и как может обращаться к его API или ресурсам из разных источников.

## Ключевые моменты  
- **Origin** — это сочетание протокола, имени хоста и порта (например, https://example.com:443). Браузер по умолчанию блокирует запросы между разными origin, чтобы предотвратить атаки типа CSRF.  
- С помощью **CORS сервер отвечает заголовками** (например, `Access-Control-Allow-Origin`), которые разрешают или запрещают доступ клиенту с определённого origin.  
- Есть разные типы запросов: **простые** (GET, POST с ограниченным набором заголовков) и **предварительные (preflight)**, когда браузер отправляет OPTIONS-запрос, чтобы проверить, разрешён ли настоящий запрос с нестандартными заголовками или методами (PUT, DELETE и др).  
- В настройках CORS можно указывать допустимые методы, заголовки, время кэширования результата preflight и допустимость отправки куков (`Access-Control-Allow-Credentials`).

## Практический контекст  
CORS широко используется при разработке SPA (React, Vue, Angular), когда frontend и backend расположены на разных доменах или портах. Например, React-приложение на localhost:3000 делает AJAX-запросы к API на api.example.com. Без правильной настройки CORS браузер просто заблокирует такие запросы, что критично для современных распределённых приложений и микросервисной архитектуры.
