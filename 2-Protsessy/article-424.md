# Какой порядок выполнения выражения: A || B && C?

**Порядок выполнения выражения: A || B && C**

- логические операторы из области **приоритетов операций**
- оператор **&&** имеет приоритет выше, чем **||**
- сначала вычисляется выражение с **B && C**
- затем результат используется в операции с **A || ...**
- обе операции **короткозамкнуты**  
- практический пример: `true || false && false` → сначала `false && false = false`, затем `true || false = true`
- такой порядок гарантирует корректную логическую оценку без лишних вычислений

# Подробный ответ
## Основной ответ
В выражении `A || B && C` порядок выполнения определяется приоритетами логических операторов в большинстве языков программирования. Оператор **логического AND (`&&`) имеет более высокий приоритет**, чем логический OR (`||`) — значит, сначала будет вычислено `B && C`, а затем результат этой операции будет использован в выражении с `A || ...`.

Иными словами, выражение парсится как `A || (B && C)`.

## Ключевые моменты
- **Приоритеты операторов:** `&&` выполняется перед `||` в языке C, C++, Java, JavaScript, Python и многих других.
- **Ассоциативность:** Оба оператора — левосторонние, но из-за разных приоритетов сначала вычисляется `B && C`, затем результат объединяется с `A` через `||`.
- **Короткое замыкание (short-circuit):** при вычислении `A || (B && C)`, если `A` истинно, то правая часть не вычисляется, и если `B` ложно, то `C` тоже не вычисляется (в части `B && C`).

## Практический контекст
При написании условных операторов в таких языках важно явно понимать эти правила, чтобы избежать неожиданных результатов. Если есть сомнения, код часто делают понятнее с помощью скобок: `A || (B && C)`. Это полезно для читаемости и предотвращает ошибки, особенно в сложных выражениях логики.
