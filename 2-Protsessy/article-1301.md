# Как устроены переменные в Python

Категория: Процессы

### Краткий ответ

**Как устроены переменные в Python**

- переменная — это имя, ссылающееся на объект в памяти  
- переменные не хранят данные, а указывают на объекты  
- все объекты в Python имеют **идентификатор (id)**, тип и значение  
- переменные динамически типизированы (тип объекта, а не переменной)  
- присваивание создаёт ссылку на объект, не копирует данные  
- объекты изменяемые и неизменяемые различаются (mutability)  
- сборщик мусора управляет памятью через подсчёт ссылок и GC  
- переменные важны для оптимизации и управления памятью

### Подробный ответ
#### Основной ответ
В Python **переменные** — это именованные ссылки на объекты в памяти, а не области памяти, как в языках низкого уровня. Когда создаёшь переменную, создаётся объект нужного типа (число, строка и т.д.), а переменная хранит ссылку на этот объект. Это значит, что переменная сама по себе не содержит данные, а указывает на место, где они лежат.

#### Ключевые моменты
- **Объектно-ориентированное управление памятью**: каждое значение в Python — объект с метаданными (тип, счетчик ссылок, данные). Переменная — имя, указывающее на объект.
- **Изменяемость и неизменяемость**: для неизменяемых объектов (int, str, tuple) при «изменении» создаётся новый объект, а переменная начинает ссылаться на него. Для изменяемых объектов (list, dict) изменения происходят по ссылке.
- **Сборщик мусора**: Python использует подсчёт ссылок и циклический сборщик мусора, чтобы автоматически освобождать память, когда на объект больше нет ссылок.
- **Локальные и глобальные имена** Хранатся в отдельных namespaces — словарях, например, `locals()` и `globals()`, что влияет на время доступа и область видимости переменной.

#### Практический контекст
Такое устройство объясняет поведение, когда две переменные могут указывать на один и тот же изменяемый объект (shared mutable state). Например, передача списков в функции — это передача ссылок, а не копий. В высоконагруженных приложениях важно учитывать стоимость создания новых объектов и управление ссылками для оптимизации памяти.
