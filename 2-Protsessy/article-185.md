# Как в рантайме определяется тип объекта, если переменная объявлена как базовый класс, а инициализирована наследником? (устройство виртуальных таблиц)

**Как в рантайме определяется тип объекта через виртуальные таблицы?**  
- Область: ПЗУ, объектно-ориентированное программирование, полиморфизм  
- Основное: тип определяется через **виртуальную таблицу (vtable)**  
- При создании объекта наследника в память записывается указатель на vtable с его методами  
- По базовой ссылке вызываются методы через vtable, выбирается реализация наследника  
- vtable содержит адреса переопределённых методов, обеспечивая **динамическую диспетчеризацию**  
- Это обеспечивает рантайм-полиморфизм без ветвлений по типу  
- Практическое: используется в C++, C# для вызова виртуальных функций через базовый тип  
- Итог: **runtime-тип объекта — через vtable**, а не статический тип переменной, что гарантирует корректное поведение методов наследников

# Подробный ответ
## Основной ответ
Время определения реального типа объекта, если переменная объявлена как указатель или ссылка на базовый класс, но инициализирована экземпляром наследника, обеспечивается благодаря **механизму виртуальных таблиц (vtable)**. При использовании виртуальных функций компилятор создаёт для каждого класса виртуальную таблицу — массив указателей на реализации виртуальных методов. Объект хранит указатель на эту таблицу (vptr). При вызове виртуальной функции через базовый класс происходит обращение к таблице, откуда берётся текущая реализация метода, соответствующая реальному типу объекта.

## Ключевые моменты
- **vtable/vptr** — это ключевая структура для поддержки полиморфизма в C++ и похожих языках. Каждому объекту с виртуальными методами сопоставлен скрытый указатель (vptr) на таблицу методов.
- Благодаря этому при вызове виртуального метода через базовую ссылку реально вызывается переопределённый метод наследника — определение типа происходит динамически, в рантайме.
- Такой механизм обеспечивает динамический диспетчер вызовов (dynamic dispatch) с небольшой накладной по производительности (~несколько инструкций косвенного перехода).
- Важно помнить, что для работоспособности этого механизма методы должны быть объявлены с ключевым словом **virtual**; без него вызывается метод базового класса (статический полиморфизм отсутствует).

## Практический контекст
В реальных приложениях, например, при работе с иерархиями GUI-компонентов, когда базовый класс объявляет интерфейс, а наследники переопределяют логику отрисовки и обработки, именно vtable позволяет вызывать корректные методы без знания точного типа объекта в момент компиляции. Это базовая часть объектно-ориентированного дизайна и полиморфизма в C++11+ и других языках с подобной моделью (например, виртуальные методы в Java используют аналогичные механизмы под капотом).
