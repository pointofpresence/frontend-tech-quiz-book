# Объясните концепцию прототипного наследования в JavaScript?

Категория: Процессы

### Краткий ответ

**Объясните концепцию прототипного наследования в JavaScript**

- JavaScript использует **прототипную модель наследования**, а не классическую
- Каждый объект имеет скрытую ссылку на **прототип** (другой объект)
- При доступе к свойству поиск идёт по цепочке прототипов (**prototype chain**)
- Наследуемые свойства и методы хранятся в прототипе, а не копируются
- Изменения прототипа отражаются во всех объектах-наследниках
- Прототип задаётся через **`Object.create`** или `constructor.prototype`
- Используется для **повторного использования кода** и динамического расширения объектов

### Подробный ответ
#### Основной ответ
**Прототипное наследование** в JavaScript — это механизм, с помощью которого объекты наследуют свойства и методы от других объектов через ссылку на прототип. В отличие от классического наследования, здесь нет классов как таковых (до ES6), а объекты связаны цепочкой прототипов (prototype chain). При обращении к свойству объекта, если оно отсутствует, движок ищет его в прототипе, затем в прототипе прототипа и так далее, пока не достигнет `null`.

#### Ключевые моменты
- Каждый объект в JavaScript имеет внутреннее свойство `[[Prototype]]` (доступно через `__proto__` или `Object.getPrototypeOf`), которое указывает на другой объект-прототип.
- Создание нового объекта через `Object.create(proto)` устанавливает прототип явно, позволяя задать наследование без использования функций-конструкторов.
- В ES6 введён синтаксис классов, который под капотом использует прототипное наследование, предоставляя синтаксический сахар для понятной объектно-ориентированной модели.
- Прототипная цепочка позволяет реализовать полиморфизм и наследование поведения без дублирования кода, при этом изменения в прототипе сразу отражаются у всех объектов-наследников.

#### Практический контекст
Типично используют прототипное наследование для расширения функциональности через общий базовый объект (например, `Array.prototype` или `Function.prototype`), а также при создании пользовательских типов с помощью конструкторов (`function User(){}`) или ES6 классов, где методы объявлены в прототипе, что позволяет экономить память и удобно управлять поведением.
