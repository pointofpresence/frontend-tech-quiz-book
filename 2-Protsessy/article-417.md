# Как работает memo при поверхностном сравнении пропсов?

Категория: Процессы

### Краткий ответ

**Как работает React.memo при поверхностном сравнении пропсов?**  
- React.memo — HOC для оптимизации функциональных компонентов  
- Выполняет **поверхностное сравнение** (shallow compare) новых и предыдущих пропсов  
- Сравниваются только **ссылки** на объекты и примитивные значения  
- При совпадении ссылок и значений компонент **не перерендеривается**  
- Не выполняется глубокое сравнение вложенных структур, из-за чего могут быть пропущены изменения внутри объектов/массивов  
- Можно передать кастомную функцию сравнения для тонкой настройки логики обновления  
- Используется для избегания лишних ререндеров и повышения производительности UI  

Итог: React.memo оптимизирует рендеринг с помощью поверхностного сравнения пропсов, ложась на ссылочную идентичность для принятия решения о перерисовке.

### Подробный ответ
#### Основной ответ  
В React **memo** — это HOC (Higher-Order Component), который предотвращает перерисовку компонента, если его пропсы не изменились. При этом React выполняет **поверхностное сравнение** (shallow comparison) пропсов: сравниваются только ссылки на объекты и примитивные значения на верхнем уровне. Если ссылки не изменились и значения равны, компонент не перерисовывается, что экономит ресурсы.  

#### Ключевые моменты  
- **Поверхностное сравнение**: React сравнивает новые и старые пропсы по ключам, используя оператор `===`. Для примитивов это сравнение по значению, для объектов и массивов — по ссылке.  
- **Ограничения**: если пропс — сложный объект (например, массив или объект), и он создается заново при каждом рендере, `React.memo` не сработает, даже если содержимое одинаковое. Для этого нужно использовать `useMemo` или стабилизировать пропсы.  
- **Опциональная кастомизация**: можно передать вторым аргументом функцию сравнения, которая реализует глубокое или другое специфическое сравнение пропсов.

#### Практический контекст  
Типично используют `React.memo` в компонентах-функциях в React 16.6+ для оптимизации производительности, особенно для часто обновляемых списков или сложных UI. В React 18 подход остался тот же, но благодаря улучшениям в рендеринге и concurrency-модели эффект может быть ещё более значимым.
