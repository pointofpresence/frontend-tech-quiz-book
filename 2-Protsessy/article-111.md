# Что такое PostgreSQL replication?

**Что такое PostgreSQL replication?**

- Технология **репликации данных** в PostgreSQL  
- Копирование изменений с **мастер** сервера на **реплики**  
- Основные виды: **streaming replication** (потоковая) и **logical replication** (логическая)  
- Streaming: реплика в реальном времени через WAL (Write-Ahead Log)  
- Logical: выборочная синхронизация таблиц/данных, поддержка разных форматов  
- Обеспечивает **балансировку нагрузки** и **резервирование**  
- Используется для повышения **доступности** и отказоустойчивости систем

# Подробный ответ
## Основной ответ
**PostgreSQL replication** — это процесс копирования и синхронизации данных из главной базы данных (primary) на одну или несколько вторичных (standby) для обеспечения отказоустойчивости, масштабируемости и балансировки нагрузки. Основная цель — иметь актуальную резервную копию, способную при необходимости принять на себя работу при сбое основного сервера.

## Ключевые моменты
- **Streaming replication** (начиная с PostgreSQL 9.0) — потоковая передача WAL-журналов (Write-Ahead Log) для минимальной задержки между основным сервером и репликами, используется для горячего standby.
- **Logical replication** (с версии 10) — репликация на уровне логических изменений, позволяет выборочно реплицировать таблицы и более гибко настраивать сценарии, например, репликация между разными версиями или базами с разной структурой.
- **Типы реплик**: синхронная (application ждет подтверждения записи на реплику) и асинхронная (запись на реплику происходит с задержкой) — выбор влияет на производительность и уровень надежности.
- Репликация в PostgreSQL реализована на уровне WAL, что обеспечивает высокую производительность и совместимость.

## Практический контекст
В реальных проектах часто используют **streaming replication** для организации **HA-кластеров** с автоматическим failover через инструменты типа Patroni или repmgr. **Logical replication** применяют при миграциях, выборочной репликации или интеграции данных между разными системами. Мониторинг репликации — важный элемент, для него используют Prometheus с экспортерами PostgreSQL, чтобы отслеживать задержки (lag) и состояние реплик.
