# В чём разница между UNION и JOIN?

Категория: Процессы

### Краткий ответ

**Разница между UNION и JOIN**

- SQL-операторы для работы с таблицами
- **UNION**: объединяет строки из нескольких таблиц в один набор, добавляя строки друг под другом
- UNION требует одинаковое количество и типы столбцов
- Удаляет дубликаты или с UNION ALL оставляет все
- **JOIN**: объединяет таблицы по связанным столбцам, расширяя строки колонками из другой таблицы
- JOIN типы: INNER, LEFT, RIGHT, FULL — управляют, какие строки выводить
- UNION применяется для объединения данных с одинаковой структурой, JOIN — для связывания связанных записей из разных таблиц

### Подробный ответ
#### Основной ответ
**UNION** и **JOIN** — это два разных способа объединения данных из нескольких таблиц в SQL, при этом они решают разные задачи. **UNION** объединяет строки из двух (или более) запросов, возвращая результат как один набор строк, а **JOIN** объединяет столбцы из связанных таблиц, расширяя каждую строку информацией из другой таблицы.

#### Ключевые моменты
- **UNION** действует по вертикали: он складывает строки из двух выборок, при этом количество столбцов и их типы должны совпадать. По умолчанию убирает дубликаты (UNION ALL сохраняет все).
- **JOIN** действует по горизонтали: добавляет столбцы из одной таблицы к строкам другой, используя условия связывания (например, по ключу). Есть несколько видов JOIN — INNER, LEFT, RIGHT, FULL, которые определяют, какие строки попадут в результат.
- **Назначение и использование** принципиально отличаются: UNION полезен, когда нужно объединить однотипные данные из разных источников (например, списки клиентов из разных регионов), а JOIN — когда требуется "склеить" информацию из связанных таблиц (например, детали заказа с информацией о клиенте).

#### Практический контекст
В реальных проектах, например, в PostgreSQL 14+, часто применяют JOIN для объединения таблиц пользователей и заказов, чтобы получить полную картину, и используют UNION для создания отчетов, объединяющих данные за разные периоды или из разных категорий, где структура данных совпадает. Trade-off: JOIN может быть медленнее из-за необходимости связывать данные, но позволяет получить более детальную картину, в то время как UNION быстрее, но лишь расширяет результирующий набор строк.
