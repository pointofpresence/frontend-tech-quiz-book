# Какие аннотации используются для создания бинов и компонентов в Spring?

**Аннотации для бинов и компонентов в Spring**

- Spring — **контейнер управления зависимостями**
- @Component — базовая аннотация для **регистрации компонента** в контексте Spring
- @Service — специальный компонент для **бизнес-логики**
- @Repository — компонент для **слоя доступа к данным**, поддерживает обработку исключений БД
- @Controller — для **веб-контроллеров** в MVC
- @Bean — объявляет метод для создания **ручного бина** внутри @Configuration класса
- @Configuration — класс с методами @Bean, конфигурирует контекст Spring
- Все аннотации создают бины как **singleton** по умолчанию
- Позволяют Spring осуществлять **автоматический скан и инжекцию** зависимостей
- Практическое значение: упрощают разработку и повышают модульность приложения

# Подробный ответ
## Основной ответ
В **Spring Framework** для создания бинов и компонентов применяются специальные аннотации, которые позволяют контейнеру автоматически сканировать и управлять их жизненным циклом. Основными являются **@Component**, **@Service**, **@Repository**, **@Controller** и **@Bean**. Эти аннотации обозначают классы или методы как кандидатов для автоматического создания бина в контексте Spring.

## Ключевые моменты
- **@Component** — базовая универсальная аннотация, отмечающая класс как компонент Spring; все остальные аннотации — специализированные варианты @Component с семантическим смыслом.
- **@Service**, **@Repository**, **@Controller** — специализированные stereotype-аннотации для сервисов, компонентов доступа к данным и контроллеров MVC соответственно. Они дополнительно дают смысловую нагрузку и могут влиять на конкретное поведение (например, @Repository обрабатывает исключения базы).
- **@Bean** — используется в конфигурационных классах (@Configuration), чтобы объявить метод, возвращающий бин, который будет создан и зарегистрирован в контексте. Это аннотация для программного определения бинов в отличие от сканирования компонентов.
- Для автоматического создания бинов требуется включить **component scanning** через аннотацию **@ComponentScan** или через XML-конфигурацию.

## Практический контекст
В современных проектах на Spring Boot в версиях 2.x и 3.x чаще всего применяются stereotype-аннотации для классов бизнес-логики, репозиториев и контроллеров, а также @Bean внутри конфигурационных классов для более тонкого управления созданием бинов — например, для сторонних библиотек или кастомных настроек. Это упрощает конфигурацию, делает код чище и улучшает модульность.
