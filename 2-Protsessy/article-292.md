# Streams. Какие типы стримов бывают

Категория: Процессы

### Краткий ответ

**Streams: основные типы**

- Поток — последовательность данных в Node.js или других средах  
- **Readable** — источник данных, можно читать (например, файл, сеть)  
- **Writable** — приемник данных, можно записывать (файл, HTTP ответ)  
- **Duplex** — одновременно чтение и запись (например, TCP-сокет)  
- **Transform** — duplex с преобразованием данных (шифрование, сжатие)  
- Работа с потоками по частям **ускоряет** обработку больших данных, снижая нагрузку на память

### Подробный ответ
#### Основной ответ
**Streams (потоки)** — это абстракция, позволяющая работать с потоками данных как с последовательностями, которые можно читать, записывать или преобразовывать по мере поступления. В программировании и в системах ввода/вывода обычно выделяют несколько типов стримов, которые различаются по направлению и функциональности.

#### Ключевые типы стримов
- **Readable Stream (читаемый поток)** — источник данных, который можно читать последовательно. Например, чтение файла, сетевого сокета или стандартного ввода. В Node.js такой поток умеет выдавать данные частями (chunk).
- **Writable Stream (записываемый поток)** — приемник данных, поддерживающий запись целиком или по частям, например запись в файл, отправка по сети или стандартный вывод.
- **Duplex Stream (двунаправленный поток)** — объединяет возможности чтения и записи. Используется, например, для сетевых соединений (TCP-сокеты), где можно одновременно принимать и отправлять данные.
- **Transform Stream (преобразующий поток)** — особый вид duplex, который преобразует данные, проходящие через него. Например, сжатие, шифрование, фильтрация или парсинг данных на лету.

#### Дополнительные нюансы
- В экосистемах, как Node.js 14+, эти типы реализованы через стандартный Stream API с различными методами (`pipe()`, `on('data')` и пр.).
- Потоки могут работать в режимах **flowing** (автоматическая эмиссия данных) и **paused** (контролируемое чтение).
- Основное преимущество — экономия памяти и возможность обработки данных больших размеров или в реальном времени без загрузки всего объема в память.

#### Практический контекст
В реальных проектах, например при работе с большими файлами или сетевым взаимодействием, используют **Readable streams** для приема данных, **Writable streams** — для сохранения/отправки, **Transform streams** — для обработки данных по пути (например, gzip сжатие), а **Duplex streams** — для bidirectional communication в сетевых протоколах. Это позволяет организовывать эффективные и масштабируемые системы с низкой задержкой и экономным использованием ресурсов.
