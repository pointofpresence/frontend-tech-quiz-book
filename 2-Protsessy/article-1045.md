# Как работает выделение памяти при добавлении элементов в Array?

**Как работает выделение памяти при добавлении элементов в Array?**

- динамический массив с **динамическим выделением памяти**  
- при добавлении проверяется свободное место  
- если места нет — выделяется **больше памяти** (~в 2 раза)  
- существующие элементы копируются в новый блок  
- амортизированная сложность вставки — **O(1)**  
- оптимизирует операции за счёт минимизации частых аллокаций  
- широко применяется в стандартных реализациях ArrayList, Vector и т.п.

# Подробный ответ
## Основной ответ  
При добавлении элементов в массив (**Array**) в большинстве языков программирования происходит динамическое выделение памяти с возможной реаллокацией. Изначально создаётся массив с определённой ёмкостью. Когда добавляем новый элемент и текущей ёмкости не хватает, в фоновом режиме выделяется новый участок памяти, обычно большего размера (часто *вдвое больше*), и старые данные копируются туда. После этого старый участок освобождается, а добавление элемента продолжается в новом месте.

## Ключевые моменты  
- **Амортизированная сложность**: Реаллокация происходит не при каждом добавлении, а периодически, поэтому средняя сложность добавления — O(1), хотя отдельная операция может быть O(n) из-за копирования.  
- **Увеличение ёмкости**: Обычно размер массива увеличивают по экспоненте (например, умножение на 2 или 1.5), чтобы уменьшить число копирований и повысить производительность.  
- **Особенности языков**: Например, в JavaScript массивы — объекты с динамической структурой, но движок оптимизирует "плотные" массивы с авторасширением. В Java массивы фиксированной длины, для динамических используют ArrayList, который именно так и реализован.

## Практический контекст  
В Java `ArrayList` использует внутренний массив с начальной ёмкостью (обычно 10). При превышении размера создаётся новый массив в 1.5-2 раза больше, все элементы копируются, что гарантирует эффективное добавление элементов и уменьшает количество перераспределений. Знание этого помогает оптимизировать работу с коллекциями и снижать накладные расходы на выделение памяти в высоконагруженных приложениях.
