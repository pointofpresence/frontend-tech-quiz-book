# Как работает замыкание (closure) в C#, когда в лямбду передаётся внешняя переменная (например, counter)?

**Как работает замыкание (closure) в C# с внешней переменной**

- замыкание — сохранение **контекста окружения** лямбды  
- внешняя переменная (например, counter) захватывается **по ссылке**, не копируется  
- все ссылки на эту переменную внутри лямбды **ссылаются на один экземпляр**  
- переменная превращается в поле **сгенерированного класса**-компилятора  
- изменения переменной после создания лямбды **видны во всех вызовах**  
- гарантирует **доступ к окружению** даже после выхода из метода  
- широко используется для **создания состояния** в функциях и обработчиках событий

# Подробный ответ
## Основной ответ
В **C# замыкание (closure)** — это механизм, который позволяет лямбда-выражениям или анонимным методам захватывать и использовать переменные из внешнего контекста, в котором были созданы. Когда лямбда обращается к переменной внешнего метода (например, `counter`), компилятор преобразует эту переменную в поле компилируемого вспомогательного объекта, который называется **display class**. Таким образом, лямбда хранит ссылку на этот объект, а не просто копирует значение переменной.

Это значит, что если переменная `counter` изменяется между вызовами лямбды, она будет видна в текущем состоянии, поскольку все лямбды ссылаются на один и тот же объект-хранитель. Замыкания в C# работают за счет создания скрытых классов с полями для таких переменных и связыванием их с лямбдами как методами этого класса.

## Ключевые моменты
- **Capture by reference**: внешняя переменная не копируется, а захватывается по ссылке, поэтому изменения видны всем замыканиям, ссылающимся на неё.
- **Display class**: компилятор генерирует скрытый класс, где переменная становится полем; лямбда превращается в метод этого класса.
- **Память и время жизни**: если лямбда сохраняется (например, возвращается из метода), объект display class живёт дольше локального метода, что может влиять на сборку мусора.

## Практический контекст
Типичный пример — счётчики, делегаты, обработчики событий, где нужно «захватить» локальное состояние. В React 18+ ленивое вычисление состояния напоминает замыкания, но в C# эта конструкция обеспечивает удобную и безопасную работу с внешним состоянием без передачи его явно в параметры. Понимание этой механики важно для предотвращения проблем с изменяемым состоянием или многопоточностью.
