# Можно ли в DI-контейнере зарегистрировать несколько реализаций одного интерфейса и как их затем получать?

Категория: Процессы

### Краткий ответ

**Можно ли в DI-контейнере зарегистрировать несколько реализаций одного интерфейса и как их затем получать?**

- DI-контейнеры поддерживают **множественную регистрацию интерфейса**  
- Каждая реализация регистрируется с **разными идентификаторами** (ключами, именами, тегами)  
- Получение через **коллекцию всех реализаций** (например, список, IEnumerable)  
- Или по конкретному **именованному ключу** (Named, Keyed)  
- Позволяет выбирать реализацию по контексту или конфигурации  
- Используется для паттерна **"стратегия"** или **разных вариантов поведения**  
- Применяется для **гибкости архитектуры** и масштабируемости  

---
Этот подход часто поддерживается популярными DI-фреймворками (Autofac, Spring, .NET Core DI и др.) и позволяет динамично управлять зависимостями и подставлять нужную реализацию в зависимости от ситуации.

### Подробный ответ
#### Основной ответ
Да, в большинстве современных **DI-контейнеров** можно зарегистрировать несколько реализаций одного интерфейса. Для получения этих реализаций обычно используется либо разрешение всех зарегистрированных сервисов данного интерфейса, либо именованная регистрация с последующим запросом конкретной реализации.

#### Ключевые моменты
- **Множественная регистрация**: В таких DI-средах, как Microsoft.Extensions.DependencyInjection (.NET Core 3.1+/6), Spring (Java), Autofac или NestJS, можно последовательно зарегистрировать несколько сервисов под одним интерфейсом. Контейнер хранит их в коллекции.
- **Разрешение коллекции**: Для получения всех реализаций интерфейса используется механизм разрешения `IEnumerable<T>` или аналог (например, `List<T>` в Java/Spring). Тогда вызывающий получает список всех экземпляров и может самостоятельно выбрать нужный.
- **Нейминг или квалификаторы**: Если нужно точечно использовать конкретную реализацию, то часто применяется регистрация с ключом, тэгом или именем (Named/Keyed registration). Это позволяет запросить реализацию по имени через контейнер.

#### Практический контекст
В .NET Core 6 при регистрации в `IServiceCollection`:

```csharp
services.AddTransient<IService, ServiceA>();
services.AddTransient<IService, ServiceB>();
```

Для получения всех:

```csharp
public MyClass(IEnumerable<IService> services) { ... }
```

В Spring можно использовать аннотации `@Qualifier` и `@Autowired` для выборочной инъекции, а в Autofac есть поддержка ключей `Named`/`Keyed`.

Таким образом, мультиреализация интерфейса — стандартный паттерн для расширяемости и плагинов, позволяющий гибко выбирать нужный сервис в рантайме.
