# Что хранится в стеке потока исполнения?

**Что хранится в стеке потока исполнения?**  
- структура памяти каждого **потока**  
- хранит **локальные переменные**, параметры функций  
- содержит адреса возврата вызовов (return addresses)  
- управляет порядком выполнения **вложенных функций**  
- обеспечивает контекст выполнения текущей функции  
- отвечает за **управление вызовами и возвратами**  
- размер стека фиксирован или настраивается, может привести к **переполнению стека**  
- используется JVM, ОС, языками с поддержкой многопоточности  
- критичен для быстрого доступа и управления **контекстом вызова**

# Подробный ответ
## Основной ответ
В **стеке потока исполнения** (call stack) хранится информация, которая необходима для правильного выполнения последовательности вызовов функций в рамках одного потока. Этот стек содержит **кадры вызовов** (stack frames или activation records), каждый из которых представляет контекст вызова конкретной функции.

## Ключевые моменты
- **Локальные переменные и параметры функции**: для каждой вызванной функции в её stack frame хранятся параметры, локальные переменные и иногда временные данные.
- **Адрес возврата**: после завершения функции управление должно вернуться в точку вызова, поэтому стек хранит адрес инструкции, с которой нужно продолжить выполнение.
- **Контекст выполнения**: регистры процессора, состояние флагов и другие элементы, необходимые для корректного возобновления работы после возврата.
- Стек работает по принципу **LIFO (last in, first out)** — последний вошедший вызов первым завершается.
- В отличие от куча (heap), стек ограничен памятью и работает гораздо эффективнее, но хранит только временные данные для текущих вызовов.

## Практический контекст
Когда в приложении возникает цепочка вызовов функций, например, в Java или C++, стек помогает системе отслеживать, где именно находится выполнение, какие переменные нужны и что делать после завершения текущей функции. Переполнение стека (stack overflow) возникает, если глубина рекурсии или вложенность вызовов слишком велика и превышает выделенный размер стека, что типично для ошибок в коде.
