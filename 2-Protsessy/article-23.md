# Как определяется this внутри функции-конструктора?

Категория: Процессы

### Краткий ответ

**Как определяется this внутри функции-конструктора?**

- функция-конструктор вызывается с ключевым словом **new**  
- this ссылается на **новый создаваемый объект**  
- внутри конструктора this **напрямую связан** с создаваемым экземпляром  
- если конструктор явно возвращает объект, this **переназначается**  
- служит для **инициализации свойств** нового объекта  
- this не зависит от контекста вызова, а **жестко привязан к new**  
- важно для создания **экземпляров классов и объектов** через функцию  

---

**Детально:**  
При вызове функции с `new` происходит следующее: создаётся пустой объект, this внутри функции устанавливается в этот объект, вызывается функция-конструктор и инициализирует свойства через this, после чего возвращается этот объект, если явный возврат не переопределяет его. Если вызвать конструктор без `new`, this либо будет глобальным (в старом режиме), либо `undefined` (в strict), что приводит к ошибкам или нежелательному поведению.

### Подробный ответ  
#### Основной ответ  
В контексте **функции-конструктора** в JavaScript ключевое слово **`this`** ссылается на вновь создаваемый объект. Когда функция вызывается с помощью оператора `new`, происходит следующий процесс: создаётся пустой объект, затем `this` внутри функции-конструктора указывает на этот объект, и, наконец, функция возвращает его (если явно не вернуть другой объект). Таким образом, `this` внутри конструктора — это объект, который инициализируется и который в итоге становится результатом вызова.

#### Ключевые моменты  
- При вызове через `new` создаётся новый объект с прототипом, равным `FunctionName.prototype`, и `this` указывает на этот объект.  
- Если функция-конструктор возвращает примитив, он игнорируется, а объект `this` возвращается по умолчанию. Если возвращается объект — возвращается он.  
- Без использования `new` значение `this` внутри функции зависит от контекста вызова (глобальный объект, `undefined` в строгом режиме либо явно заданное).

#### Практический контекст  
Типичный пример — в React классовых компонентах или при создании custom объектов: используется `this` для установки свойств экземпляра. В ES6+ часто заменяют конструкторы классами, где поведение `this` аналогично, но синтаксис удобнее и безопаснее. Важно точно помнить, что пропуск `new` при вызове конструктора — частая ошибка, которая меняет смысл `this` и может привести к багам.
