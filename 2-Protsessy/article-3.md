# Какие основные виды документации обычно создаются в процессе работы над проектом?

**Основные виды документации в проекте**

- **Техническая документация**: архитектура, API, схемы, структура кода  
- **Требования (техническое задание)**: описывает функционал, бизнес-логику  
- **Пользовательская документация**: инструкции, гайды по использованию продукта  
- **Тестовая документация**: тест-кейсы, планы и отчёты по тестированию  
- **Документация по развертыванию**: инструкции по установке, настройке и CI/CD  
- **Проектная документация**: roadmap, планирование, задачи, статус  
- **Документация по поддержке**: баги, известные проблемы, решения и миграции  

Это даёт полный охват процессов разработки, внедрения, поддержки и использования продукта.

# Подробный ответ
## Основной ответ
В процессе разработки проекта создаётся несколько ключевых видов документации, каждый из которых решает свои задачи и помогает разным участникам команды. Основные типы — это **техническая документация**, **пользовательская документация** и **процессная документация**.

## Ключевые моменты
- **Техническая документация** включает в себя архитектурные схемы, спецификации API, описание кода и базу знаний по инфраструктуре. Она помогает разработчикам понимать устройство системы и быстро внедряться в проект. Часто создаётся в формате Markdown с использованием систем вроде Confluence или GitLab Wiki, иногда с автогенерацией из кода (например, Swagger/OpenAPI для API).
- **Пользовательская документация** — это инструкции по использованию продукта, руководства пользователя, FAQ и обучающие материалы. Они рассчитаны на конечных пользователей или клиентов и должны быть максимально понятны и структурированы.
- **Процессная документация** описывает методы работы команды, стандарты кодирования, процессы CI/CD, планы тестирования и релизов. Такая документация обеспечивает стабильность и качество разработки, помогает новичкам быстро адаптироваться.

## Практический контекст
В крупных проектах стандартизируют документацию, используют шаблоны и инструменты для генерации, например, Swagger для API, JSDoc/Doxygen для кода и Storybook для UI компонентов. Это сокращает коммуникационные издержки и минимизирует ошибки при масштабировании команды или продукта.
