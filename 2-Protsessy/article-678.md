# Что такое frozenset и чем отличается от set?

Категория: Процессы

### Краткий ответ

**Что такое frozenset и чем отличается от set?**

- frozenset — **неизменяемое множество** (immutable set)
- set — **изменяемое множество** (mutable set)
- frozenset нельзя изменять: нет методов add, remove и т.п.
- frozenset может использоваться как **ключ в dict** и элемент других set
- set обеспечивает динамическое добавление и удаление элементов
- оба имеют **уникальные элементы** и обеспечивают **поиск O(1)**
- frozenset полезен для **хешируемых** коллекций с множествами

**Итог:** frozenset — **immutable, hashable set** для использования в ключах и константных множествах; set — динамичное множество для изменения.

### Подробный ответ
#### Основной ответ
**frozenset** — это неизменяемый (immutable) вариант множества (set) в Python. Как и set, frozenset хранит уникальные элементы без определённого порядка, но ключевое отличие в том, что после создания frozenset нельзя изменить — добавить или удалить элементы. В то время как set — изменяемый тип коллекций.

#### Ключевые моменты
- **Изменяемость**: set можно модифицировать методами add(), remove(), clear(), а frozenset — нет. Это обеспечивает безопасность и предсказуемость при использовании frozenset.
- **Хешируемость**: frozenset является хешируемым объектом, значит его можно использовать в качестве ключа словаря или элемента другого множества, чего нельзя сделать с обычным set.
- **Применение**: frozenset подходит для фиксированных наборов данных, которые нужно использовать как константы, или для организации вложенных структур данных (например, set из frozenset).

#### Практический контекст  
В проектах, где требуется гарантия неизменности множества (например, для кэширования уникальных наборов параметров или в качестве ключей в кэшах/словарях), frozenset обеспечивает защиту от случайных изменений. Стандартный set часто используется там, где коллекция динамически модифицируется в процессе работы программы.
