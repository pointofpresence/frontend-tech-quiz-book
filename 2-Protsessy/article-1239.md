# Что такое data class в Kotlin?

Категория: Процессы

### Краткий ответ

**Что такое data class в Kotlin?**  
- класс для хранения **данных**  
- автоматически генерирует: **equals, hashCode, toString, copy, componentN**  
- все свойства должны быть в **primary constructor**  
- удобно для **immutable** моделей и DTO  
- упрощает сравнение и клонирование объектов  
- снижает **бо́льшую часть шаблонного кода**  
- используется для бизнес-моделей и API-ответов

### Подробный ответ
#### Основной ответ  
**Data class** в Kotlin — это специализированный класс, предназначенный для хранения данных. Он генерирует автоматически базовый набор полезных методов: `equals()`, `hashCode()`, `toString()`, а также методы `copy()` и `componentN()` для удобной работы с данными. Это значительно уменьшает шаблонный код и улучшает читаемость и поддержку.

#### Ключевые моменты  
- **Автоматически сгенерированные методы**: конструктор с параметрами, `equals()`, `hashCode()`, `toString()`, `copy()` и разбивка на компоненты (деструктуризация). Это позволяет безопасно сравнивать объекты и делать их неизменяемыми по умолчанию.  
- **Принцип работы с состоянием**: В data class рекомендуется хранить только свойства, которые участвуют в логике, комплектуются в первичном конструкторе, что облегчает сравнение и копирование объектов.  
- **Ограничения**: нельзя наследоваться от других data class, и у data class должен быть хотя бы один параметр в конструкторе. Кроме того, data class не должна содержать mutable состояние без должного контроля.

#### Практический контекст  
В реальных проектах data class часто используют как модели данных, DTO (Data Transfer Objects), элементы для сериализации и десериализации JSON (особенно совместно с kotlinx.serialization или Gson), в реактивном программировании для состояния UI (например, в ViewModel) и для чистых, лаконичных доменных моделей. В Kotlin 1.5+ data class продолжает оставаться ключевым инструментом для безопасного и простого управления данными с минимальной бойлерплейтной нагрузкой.
