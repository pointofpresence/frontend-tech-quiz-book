# Что будет, если сделать запрос со * (звёздочкой) в Redis?

Категория: Процессы

### Краткий ответ

**Что будет, если сделать запрос со * (звёздочкой) в Redis?**

- Redis — **in-memory** key-value хранилище
- * в команде, например `KEYS *` — шаблон для поиска всех ключей
- `KEYS *` возвращает **все ключи** БД
- команда **блокирует сервер** пока выполняется
- при большом количестве ключей — **снижение производительности**
- рекомендовано использовать **SCAN** для итеративного поиска
- `KEYS *` полезна для отладки, но **не для продакшена**

### Подробный ответ
#### Основной ответ
Если в Redis выполнить запрос с ключом `*` (звёздочка), например команду `GET *` или использовать `KEYS *`, то результат будет зависеть от типа команды. В Redis `*` выступает как wildcard-паттерн, который означает "все ключи", но он не подставляется автоматически для всех команд. Например, `KEYS *` вернёт список всех ключей в базе, а `GET *` — выдаст ошибку, потому что `GET` принимает один ключ, а не шаблон.

#### Ключевые моменты
- **Команда `KEYS *`**: возвращает все ключи, совпадающие с шаблоном `*` — то есть все ключи в текущей базе данных. Это полезно, но при большом количестве ключей может привести к значительной нагрузке и блокировке.
- **Команды с шаблонами** применимы только к определённым командам — например, `SCAN`, `KEYS`, `DEL` (с несколькими ключами, но без шаблона), а не к `GET` или `SET`.
- **Промежуток масштабируемости**: использование `KEYS *` на продакшн-сервере не рекомендуется, т.к. она блокирует Redis и может замедлить работу (замеряют до сотен тысяч и больше ключей).
- Если попытаться сделать что-то вроде `GET *`, Redis вернет ошибку синтаксиса, потому что `GET` ожидает точное имя ключа.

#### Практический контекст
В реальных проектах для перебора ключей по шаблону используют команду `SCAN` с паттерном `*`, она итеративна и не блокирует сервер. `KEYS *` подходит для отладки и небольших баз. Если нужно удалить множество ключей по шаблону, используют скрипты с `SCAN` и `DEL`, чтобы избежать проблем с производительностью.
