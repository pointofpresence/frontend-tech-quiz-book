# Что такое стек (stack)? Что такое очередь (queue)?

Категория: Процессы

### Краткий ответ

**Что такое стек (stack)? Что такое очередь (queue)?**

**Стек (stack)**  
- структура данных с доступом **LIFO** (Last In, First Out)  
- операции: push (добавить), pop (удалить последний) — обе **O(1)**  
- используется для **отката состояний**, рекурсии, вызовов функций  
- реализуется массивом или связным списком  

**Очередь (queue)**  
- структура данных с доступом **FIFO** (First In, First Out)  
- операции: enqueue (добавить в конец), dequeue (удалить с начала) — обе **O(1)** при правильной реализации  
- применяется для планирования задач, передачи сообщений  
- варьируется в вариантах: простая, кольцевая, приоритетная очередь  

**Итог**:  
Стек — управление порядком обработки с **последним добавленным** элементом, очередь — с **первым добавленным**. Оба — базовые структуры для эффективного управления данными и алгоритмами.

### Подробный ответ
#### Основной ответ
**Стек (stack)** — это структура данных, работающая по принципу **LIFO** (Last In, First Out), где последний добавленный элемент извлекается первым. Она напоминает стопку книг: чтобы взять книгу из середины, нужно сначала убрать верхние. **Очередь (queue)** — это структура данных с принципом **FIFO** (First In, First Out), где элемент, добавленный первым, извлекается первым, как в очереди в магазине.

#### Ключевые моменты
- **Стек** поддерживает операции: `push` (добавить элемент сверху), `pop` (удалить верхний элемент), `peek/top` (просмотреть верхний элемент без удаления). Используется, например, для **обратного обхода**, парсинга выражений, реализации вызова функций (стек вызовов).
- **Очередь** использует операции: `enqueue` (добавить элемент в конец), `dequeue` (удалить элемент из начала), `peek/front` (посмотреть первый элемент). Часто применяется в алгоритмах обхода графов, планировании задач, буферизации.
- Оба могут быть реализованы через массивы или связные списки, при этом в реальных проектах для очередей часто используют **кольцевой буфер** для повышения эффективности.

#### Практический контекст
В **JavaScript** для стека удобно использовать **массив с push/pop**, а для очереди - массив с shift (хотя shift дорогой), лучше реализации через LinkedList. В системах с реальными нагрузками применяют специализированные классы и структуры, обеспечивающие стабильное время операций и минимальную аллокацию памяти.
