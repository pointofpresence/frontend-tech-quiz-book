# Как работает asyncio.gather?

Категория: Процессы

### Краткий ответ

**Как работает asyncio.gather?**  
- инструмент библиотеки **asyncio** для Python  
- принимает несколько **корутин** или awaitable объектов  
- запускает их **конкурентно**, не блокируя основной поток  
- возвращает **одну корутину**, которая ждёт завершения всех входящих  
- возвращаемое значение — кортеж с результатами выполнения входных корутин в порядке передачи  
- удобен для одновременного выполнения нескольких асинхронных задач и сбора их результатов  
- обработка исключений: по умолчанию исключение в одной корутине отменяет остальные, можно изменить через параметр `return_exceptions`  
- широко используется для повышения производительности IO-bound задач через параллелизм в asyncio-цикле

### Подробный ответ
#### Основной ответ
**asyncio.gather** — это функция из стандартного модуля Python **asyncio**, которая применяется для одновременного запуска нескольких сопрограмм (coroutines) и сбора их результатов в одну структуру. Она принимает на вход один или несколько awaitable-объектов (корутин, задач или других awaitable) и возвращает новую корутину, которая завершится, когда завершатся все переданные задачи. Результат — кортеж значений, возвращённых всеми корутинами, в порядке их передачи.

#### Ключевые моменты
- **Параллельное выполнение**: asyncio.gather запускает все задачи вместе, не дожидаясь поочерёдного завершения, что эффективно использует однопоточное событийное управление Python 3.7+.
- **Обработка исключений**: по умолчанию, если одна из задач возбуждает исключение, оно будет проброшено в gather, остальные задачи продолжат работу (если gather не вызван с `return_exceptions=True`).
- **return_exceptions=True**: в этом режиме исключения не прерывают gather, а возвращаются как объекты в результирующем кортеже — удобно для обработки ошибок в группе задач.
- **Тип результата**: asyncio.gather всегда возвращает один awaitable, который нужно дождаться, например, через `await`.

#### Практический контекст
В реальных проектах **asyncio.gather** удобно применять для параллельного выполнения независимых сетевых запросов или операций ввода-вывода, где важна скорость отклика и максимальная загрузка событийного цикла. Например, в **FastAPI 0.75+** часто используют gather для одновременного запроса к нескольким микросервисам, значительно снижая latency (~50ms типично).

Таким образом, asyncio.gather — стандартный инструмент для организации конкурентных async-вызовов в Python, который обеспечивает оптимальное выполнение и удобное управление результатами.
