# Как работает ARC (Automatic Reference Counting) в Swift?

Категория: Процессы

### Краткий ответ

**Как работает ARC (Automatic Reference Counting) в Swift?**

- Менеджер памяти Swift для объектов классов  
- Отслеживает количество активных ссылок на объект (**счетчик ссылок**)  
- При увеличении ссылок — счетчик +1, при устранении — -1  
- Когда счетчик достигает 0 — объект автоматически удаляется из памяти  
- Управляет через **strong**, **weak** и **unowned** ссылки:  
   - **strong** удерживает объект  
   - **weak** и **unowned** — для избежания циклических ссылок  
- Позволяет эффективно управлять памятью без ручного освобождения  
- Применяется только к типам классов, не работает с структурами или enum  
- Ключевой механизм предотвращения утечек — **разрешение циклических зависимостей** через weak/unowned ссылки  
- Используется во всех современных проектах Swift для безопасного и автоматического освобождения памяти

### Подробный ответ
#### Основной ответ
**ARC (Automatic Reference Counting)** в Swift — это механизм управления памятью, который автоматически отслеживает и управляет временем жизни объектов, подсчитывая количество сильных ссылок (strong references) на них. Когда количество таких ссылок становится нулём, объект автоматически уничтожается и память освобождается. Это позволяет избежать утечек памяти без необходимости вручную вызывать освобождение, как в традиционных языках с ручным управлением памятью.

#### Ключевые моменты
- **Подсчёт сильных ссылок:** Каждый объект хранит счетчик, увеличивающийся при создании новой сильной ссылки и уменьшающийся при её удалении. Деструктор объекта вызывается, когда счетчик достигает нуля.
- **Слабые (weak) и невладельческие (unowned) ссылки:** Чтобы избежать retain cycles (цикл сильных ссылок), Swift поддерживает weak и unowned ссылки, которые не увеличивают счетчик и позволяют разорвать циклы, например, в случаях делегатов и замыканий.
- **Работает на уровне компилятора:** ARC интегрирован в компилятор Swift с версии 1.0, что гарантирует минимальные накладные расходы и высокую производительность по сравнению с традиционным GC.
- **Отличие от Garbage Collector:** ARC — детерминированный механизм, освобождает память сразу после исчезновения сильных ссылок, без непредсказуемых пауз, характерных для сборщиков мусора.

#### Практический контекст
В реальных проектах, например, при создании iOS-приложений на Swift 5+, ARC выступает базовым принципом управления памятью. При работе с многопоточностью стоит аккуратно обращаться с памятью и избегать retain cycles в замыканиях, используя capture lists `[weak self]` или `[unowned self]`. Это критично для поддержания производительности и предотвращения утечек памяти, особенно в сложных UI- и сетевых компонентах.
