# Чем Mock отличается от Spy в Mockito?

Категория: Процессы

### Краткий ответ

**Чем Mock отличается от Spy в Mockito?**

- Mockito — фреймворк для **тестирования** с имитациями
- **Mock** — полностью **поддельный** объект, все методы можно настраивать
- **Spy** — частично **реальный** объект с возможностью переопределения отдельных методов
- Mock не вызывает реальных методов, Spy вызывает реальные по умолчанию
- Spy позволяет тестировать поведение реального объекта с контролем вызовов
- Mock удобен для изоляции зависимости, Spy — для частичного контроля реального объекта
- Практическое применение: Mock когда нужен полный контроль, Spy для смешанных тестов с реальными данными

### Подробный ответ
#### Основной ответ
В **Mockito** существуют два основных типа тестовых двойников — **mock** и **spy**, которые служат для разных целей в юнит-тестировании. **Mock** — это полностью искусственный объект, у которого все методы по умолчанию «пустые» и их поведение нужно явно задавать. В то время как **spy** — это «шпион» на реальном объекте, который вызывает реальные методы, если их не переопределить.

#### Ключевые моменты
- **Mock:** предоставляет полностью фальшивую реализацию объекта. Все вызовы методов возвращают значения по умолчанию (null, 0, false), пока тестировщик явно не задаст поведение с помощью when(...).thenReturn(...). Используется там, где нужен полный контроль над поведением зависимостей.
- **Spy:** оборачивает реальный объект, позволяя по умолчанию выполнять настоящие методы. При этом можно перехватывать вызовы и подменять поведение отдельных методов. Это удобно, если нужно проверить взаимодействия, не теряя при этом поведения оригинального объекта.
- **Trade-off:** Mock проще изолирует тестируемый код, но иногда мало реалистичен; Spy гибче, но может привести к хрупким тестам, если реальные методы имеют побочные эффекты или внешние зависимости.

#### Практический контекст
В реальных проектах, например, в Spring Boot с Mockito 4+ mock часто используют для репозиториев или сервисов, чтобы полностью изолировать тест, а spy — для частичной подмены поведения в сложных объектах с большими side-effects. При этом всегда важно помнить про чистоту юнит-теста и избегать слишком глубоких шпионов на сложных объектах.
