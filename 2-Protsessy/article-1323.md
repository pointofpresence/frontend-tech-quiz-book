# На чём основана асинхронность в Python? Как она работает под капотом?

**На чём основана асинхронность в Python? Как она работает под капотом?**

- основана на конструкции **async/await**, появилась в Python 3.5+
- основана на **корутинах** — специальных объектах функций, умеющих приостанавливать выполнение
- использует **событийный цикл** (Event Loop) из модуля **asyncio** для планирования задач
- корутины регистрируются в цикле, который поочерёдно переключается между ними без создания новых потоков
- переключение происходит на уровне байткода Python, без системных переключений контекста (легковеснее потоков)
- ввод/вывод (I/O) — ключевая точка асинхронности: цикл ждёт событий (например, готовность сокета), не блокируя поток
- позволяет писать **однопоточный конкурентный** код с высокой производительностью при работе с I/O
- популярно для сетевых приложений, веб-сервисов и задач с большим числом ожиданий

---

Детали реализации:

- **Корутину** можно представить как генератор с возможностью получать и отдавать управление через await/yield
- **Event Loop** использует системные неблокирующие вызовы (например, epoll, kqueue) для отслеживания готовности операций
- Когда операция вне CPU готова — цикл возобновляет выполнение соответствующей корутины
- Отсутствие блокирующего sleep или ожидания позволяет запускать тысячи конкурентных операций в одном потоке

---

Этот механизм выгодно отличается от потоков и процессов по эффективности при высоком числе I/O-заданий, минимизируя накладные расходы на переключения и синхронизацию.

# Подробный ответ
## Основной ответ
Асинхронность в Python основана на модели **event loop** (цикле событий), которая управляет выполнением задач без блокировки основного потока. Главным механизмом является ключевое слово **async/await**, введённое в Python 3.5+, позволяющее писать неблокирующий код с помощью **корутин** — специальных функций, которые могут приостанавливать своё выполнение и отдавать управление обратно event loop'у. Под капотом Python использует объект event loop (в стандартной библиотеке — `asyncio`), который отслеживает состояние всех корутин и коллбэков, исполняет их, когда ресурсы (например, операции ввода-вывода) готовы, и переключается между задачами без создания новых потоков или процессов.

## Ключевые моменты
- **Event loop** централизует выполнение, обрабатывая готовые к продолжению задачи, что позволяет эффективно использовать одно поток исполнения без блокирующих вызовов (например, сетевых или файловых операций).
- В основе асинхронности лежат **корутины** — объекты `coroutine` и `Future`, позволяющие планировать отложенное или неблокирующее выполнение и взаимодействовать с результатами асинхронных операций.
- Асинхронный ввод-вывод обычно реализован поверх неблокирующих системных вызовов (например, epoll на Linux, IOCP на Windows), которые позволяют event loop уведомлять приложение о готовности ресурсов без активного ожидания.
- Важно понимать, что асинхронность в стандартном `asyncio` не создаёт автоматическую параллельность на уровне CPU, для этого нужны дополнительные механизмы (многопоточность, multiprocessing или внешние библиотеки).

## Практический контекст
В реальных проектах `asyncio` используется для построения высоконагруженных серверов, драйверов для сетевых протоколов, микросервисов с низкой задержкой и масштабируемых приложений с большим количеством параллельных операций ввода-вывода. Например, в Python 3.8+ `asyncio` стал более производительным благодаря улучшениям в event loop и оптимизациям в runtime, что делает его конкурентоспособным решением для современных сервисов.
