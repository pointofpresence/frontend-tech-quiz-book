# Как устроен жизненный цикл Fragment и чем он отличается от Activity?

Категория: Процессы

### Краткий ответ

**Жизненный цикл Fragment vs Activity**

- **Область:** Android UI-компоненты  
- **Fragment** — модуль UI, встроенный в Activity  
- **Фазы Fragment:** onAttach → onCreate → onCreateView → onViewCreated → onStart → onResume → взаимодействие → onPause → onStop → onDestroyView → onDestroy → onDetach  
- **Отличие:** Fragment жизненный цикл тесно связан с Activity, может существовать как часть UI или быть добавлен динамически  
- **Activity** — контейнер для UI и логики, жизненный цикл: onCreate → onStart → onResume → активность → onPause → onStop → onDestroy  
- Fragment вводит дополнительные методы для управления отображением и состоянием UI (onCreateView, onDestroyView), которых нет в Activity  
- Fragment может быть повторно использован в нескольких Activity, облегчает адаптивный дизайн и модульность  
- Жизненный цикл Fragment зависит от жизненного цикла Activity, но при этом имеет собственные состояния и обязанности  
- **Практическое значение:** Fragment используется для создания гибких интерфейсов с многокомпонентной структурой, Activity — для управления основным окном приложения и навигацией между экранами

### Подробный ответ
#### Основной ответ
**Жизненный цикл Fragment** тесно связан с жизненным циклом его хост-Activity, но при этом обладает собственной последовательностью состояний и методами. В Android Fragment представляет собой часть пользовательского интерфейса или поведения, которая может быть встроена в Activity для гибкости и переиспользования UI-компонентов.

Основные этапы жизненного цикла Fragment — это attach, create, createView, start, resume, pause, stop, destroyView, destroy, detach. Важно понимать, что Fragment может иметь живой объект (attach), но без видимого UI (destroyView). В отличие от Activity, Fragment более модулярен и допускает множественные операции с интерфейсом, включая динамическое добавление и удаление во время выполнения.

#### Ключевые моменты
- **Фазы жизненного цикла**: Fragment начинается с `onAttach()`, когда фрагмент связывается с Activity, далее `onCreate()`, `onCreateView()` для создания интерфейса, и затем стандартные методы `onStart()`, `onResume()`. При уничтожении сначала уничтожается UI в `onDestroyView()`, а затем сам фрагмент в `onDestroy()` и `onDetach()`.
- **Отличие от Activity**: В отличие от Activity, у Fragment нет собственного окна и он не может жить сам по себе — жизненный цикл фрагмента зависит от Activity. Activity управляет общим жизненным циклом, а Fragment — лишь частью UI и поведения.
- **Управление через FragmentManager**: Fragment добавляются, удаляются и заменяются динамически через FragmentManager, что обеспечивает гибкость. Activity же имеют более жёсткий жизненный цикл, определяющий весь экран.

#### Практический контекст
В React Native или современных Android-приложениях (например, с использованием Jetpack Navigation) часто используют Fragment для адаптивных интерфейсов на планшетах и телефонах, позволяя перерисовывать только часть экрана без пересоздания Activity. Это снижает нагрузку и улучшает UX, поддерживая сложные сценарии навигации и анимации.
