# Что такое упаковка ошибок (error wrapping)?

Категория: Процессы

### Краткий ответ

**Что такое упаковка ошибок (error wrapping)?**  
- обработка ошибок в программах  
- добавление контекста к исходной ошибке  
- сохраняет оригинальную ошибку внутри новой  
- помогает отследить полную цепочку причин  
- улучшает диагностику и отладку  
- примеры: Go (fmt.Errorf с %w), Java (Throwable cause)  
- важно для прозрачного логирования и трассировки ошибок

### Подробный ответ
#### Основной ответ
**Упаковка ошибок (error wrapping)** — это техника обработки ошибок, при которой исходная ошибка оборачивается в новую ошибку с дополнительным контекстом, не теряя при этом информации об оригинальной ошибке. Это позволяет добавить пояснения, где и почему произошла ошибка, сохраняя возможность последующего извлечения первопричины для дебага и логирования.

#### Ключевые моменты
- **Контекст и трассировка**: упаковка помогает понять цепочку вызовов, которые привели к ошибке, что особенно важно в сложных, распределённых системах.
- **Сохраняется оригинал**: часто применяется паттерн, где новая ошибка содержит ссылку на исходную, например, через `cause` или `innerError`. В Go 1.13+ используется встроенное `fmt.Errorf("context: %w", err)` для упаковки с возможностью распаковки.
- **Удобство обработки**: благодаря упаковке можно одним вызовом обработать чисто бизнес-логику и добавить человекочитаемый контекст, при этом инструмент (логгер или мониторинг) может при необходимости узнать природу начальной ошибки.

#### Практический контекст
В реальных приложениях, например, при вызове нескольких уровней функций, которые могут возвращать ошибки, упаковка помогает отследить, где именно при загрузке данных, валидации или сетевом запросе произошёл сбой. В Go используют встроенный механизм с `%w`, в Java применяют `Throwable.initCause()`, в Python — исключения с chaining через `raise from`. Это улучшает диагностику, повышает качество логов и упрощает поддержку кода.
