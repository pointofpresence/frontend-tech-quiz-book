# Что такое DispatchGroup и DispatchSemaphore?

**Что такое DispatchGroup и DispatchSemaphore?**  
- iOS/macOS **Concurrency primitives** в GCD  
- **DispatchGroup** — синхронизация групп асинхронных задач  
  - позволяет **отслеживать завершение** всех задач в группе  
  - используют для выполнения кода после завершения всех операций  
- **DispatchSemaphore** — механизм контроля доступа к ресурсам  
  - реализует **счётчик ограничений** (семафор)  
  - управляет максимальным числом одновременно выполняемых задач  
  - блокирует и разблокирует потоки, предотвращая гонки и перегрузку  
- Практическое применение:  
  - DispatchGroup для **ожидания асинхронных вызовов**  
  - DispatchSemaphore для **лимитирования параллелизма** и синхронизации  
- Отличие:  
  - Group — для **координации завершения**  
  - Semaphore — для **контроля доступа** / лимита ресурсов  

Эффективные инструменты для **управления асинхронностью и синхронизацией** в многопоточном программировании на Swift.

# Подробный ответ
## Основной ответ
**DispatchGroup** и **DispatchSemaphore** — это механизмы синхронизации из библиотеки **Grand Central Dispatch (GCD)** в iOS/macOS, предназначенные для управления параллелизмом и координации многопоточных задач.  
**DispatchGroup** используется для группирования нескольких асинхронных задач и получения уведомления, когда все они завершены.  
**DispatchSemaphore** — инструмент для контроля доступа к ограниченному ресурсу, позволяющий ограничить количество одновременно работающих потоков.

## Ключевые моменты
- **DispatchGroup** позволяет запускать несколько асинхронных операций и затем с помощью метода `notify` или `wait` определить, когда все они завершатся. Это полезно для координации и объединения результата нескольких параллельных запросов или задач.
- **DispatchSemaphore** создаётся с определённым значением (обычно начинающимся с количества доступных ресурсов). Когда поток захватывает семафор, значение уменьшается; при освобождении — увеличивается. Если значение равно нулю, последующие потоки блокируются, ожидая ресурс.
- **DispatchGroup** не блокирует поток по умолчанию, а скорее уведомляет о завершении, что подходит для асинхронных workflow, в то время как **DispatchSemaphore** часто используется для блокировки/разблокировки потоков и ограничения параллелизма.
  
## Практический контекст
В iOS-приложениях DispatchGroup часто применяется при параллельных сетевых запросах — например, загрузка данных с нескольких API, после чего UI обновляется по окончании всех загрузок. DispatchSemaphore пригоден, если нужно ограничить число одновременных операций, например, ограничить количество одновременных запросов к базе или файловой системе для предотвращения перегрузки.  
В React Native или SwiftUI с iOS 14+ лучше использовать `async/await` и `TaskGroup`, но DispatchGroup и Semaphore остаются важными низкоуровневыми инструментами.
