# Чем интерфейс отличается от абстрактного класса?

Категория: Процессы

### Краткий ответ

**Чем интерфейс отличается от абстрактного класса?**

- объектно-ориентированное программирование  
- **интерфейс** задаёт только **контракт (методы без реализации)**  
- **абстрактный класс** может иметь **реализацию методов и состояние**  
- класс может **наследовать несколько интерфейсов**, но обычно только **один абстрактный класс**  
- интерфейс обеспечивает **полную абстракцию**, абстрактный класс — **частичную**  
- интерфейс — для **гибкого полиморфизма и расширяемости**  
- абстрактный класс — для **шаблонного проектирования с общим кодом**

### Подробный ответ
#### Основной ответ
**Интерфейс** — это контракт, определяющий набор методов, которые класс должен реализовать, без предоставления какой-либо реализации. **Абстрактный класс** — это класс, который может содержать как абстрактные (без реализации), так и конкретные методы с реализацией, и служит базовым шаблоном для наследников.

#### Ключевые моменты
- **Множественное наследование**: В большинстве языков (например, Java, C#) класс может реализовывать несколько интерфейсов, но наследоваться только от одного абстрактного класса. Это дает интерфейсам гибкость для объединения разных контрактов.
- **Реализация по умолчанию**: Абстрактный класс может содержать логику и хранить состояние (поля), интерфейс традиционно – только объявления методов и константы. В современных версиях Java 8+ появились default методы в интерфейсах, но поля без состояния всё равно не разрешены.
- **Назначение**: Интерфейсы описывают “что” должен делать класс (функциональность), абстрактные классы — “как” эту функциональность можно частично реализовать или объединить через наследование.
  
#### Практический контекст
В реальных проектах интерфейсы широко используют для определения контрактов API или сервисов (например, в Spring или ASP.NET), что обеспечивает слабую связанность. Абстрактные классы применяют, когда есть общий базовый функционал, который удобно переиспользовать в нескольких связанных классах, например абстракция базового контроллера или репозитория.
