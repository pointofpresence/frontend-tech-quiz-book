# Как объединить последние N коммитов в один коммит?

Категория: Процессы

### Краткий ответ

**Как объединить последние N коммитов в один коммит?**

- Git, работа с историей  
- Используется команда **git rebase -i HEAD~N** для интерактивного ребейза  
- В открывшемся редакторе, первая строка — **pick**, остальные — меняются на **squash** или **fixup**  
- **squash** объединит коммиты, сохранив сообщения, **fixup** — сольет, удалив дополнительные сообщения  
- После сохранения, при необходимости, редактируется итоговое сообщение коммита  
- Затем применяется **git push --force** для обновления удалённого репозитория  
- Практическое применение — очистка истории перед публикацией, улучшение читаемости коммитов

### Подробный ответ
#### Основной ответ
Чтобы объединить последние N коммитов в один, используют команду **git rebase** в интерактивном режиме с опцией `-i`. Это позволяет "переписать" историю коммитов, слепив несколько коммитов в один. Конкретно, выполняют команду `git rebase -i HEAD~N`, где N — число последних коммитов, которые нужно объединить. Затем в открывшемся редакторе указывают `pick` для первого коммита и `squash` (или `s`) для остальных, что означает слияние их с предыдущим. После сохранения появляется окно для редактирования итогового сообщения коммита, где нужно согласовать сообщение.

#### Ключевые моменты
- **Git rebase -i** — стандартный и гибкий способ переписывания истории, используется во всех современных версиях Git (1.7+).
- **Squash** — сохраняет изменения всех коммитов, объединяя их в один, при этом можно отредактировать итоговое сообщение.
- Важно понимать, что **history rewriting** изменяет SHA коммитов, поэтому не рекомендуется делать на публичных ветках без согласования с командой.

#### Практический контекст
В реальной разработке это часто применяется перед merge в основную ветку, чтобы сделать историю чище и коммиты более логичными. Например, перед созданием Pull Request оптимально объединить мелкие фиксы в один коммит с осмысленным описанием. Так же удобно исправлять коммиты, когда нужно исправить typos или добавить упущенные изменения, не создавая лишних коммитов.

---

Пример команды:

```bash
git rebase -i HEAD~3
```

Далее в редакторе:

```
pick abc1234 Первый коммит
squash def5678 Второй коммит
squash ghi9012 Третий коммит
```

Итог будет один коммит с объединённым содержимым.
