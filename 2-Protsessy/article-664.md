# Для чего нужен Left Join?

Категория: Процессы

### Краткий ответ

**Для чего нужен Left Join?**

- SQL-оператор для связи таблиц  
- Возвращает все строки из **левой** таблицы  
- Добавляет совпадающие данные из правой таблицы  
- Если совпадений нет — в столбцах правой таблицы будут **NULL**  
- Используется для сохранения всей информации из левой таблицы  
- Применяется при анализе, когда важны все записи базовой таблицы  
- Помогает выявлять отсутствующие связанные данные (например, незавершённые заказы)

### Подробный ответ
#### Основной ответ
**Left Join** — это один из видов соединения таблиц в SQL, который возвращает все строки из левой таблицы и соответствующие строки из правой. Если для левой строки нет совпадений в правой таблице, то в результирующем наборе значения из правой таблицы будут заполнены `NULL`. Это особенно полезно для сохранения всей информации из основной таблицы, даже если связанная таблица не содержит соответствующих данных.

#### Ключевые моменты
- **Полное сохранение левой таблицы**: Left Join гарантирует, что мы не потеряем данные из основной таблицы, что важно для аналитики и отчетов, где нужны все объекты с возможной дополнительной информацией.
- **Обработка отсутствующих данных**: вместо исключения строк с несоответствием в правой таблице, мы получаем `NULL` — это помогает выявлять и обрабатывать неполные связи между сущностями.
- **Отличие от Inner Join**: Inner Join возвращает только те строки, где есть совпадение в обеих таблицах, а Left Join сохраняет все из левой, давая возможность работать с "полными" наборами данных и выявлять отсутствующие связи.

#### Практический контекст
Типичный кейс: есть таблица пользователей и таблица заказов. Чтобы получить список всех пользователей вместе с их заказами, даже если у пользователя пока нет заказов, используют Left Join. Это важно для построения отчетов о пользователях, где нельзя потерять данные о тех, кто еще не сделал покупку, но их нужно отобразить.
