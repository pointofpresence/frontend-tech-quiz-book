# Какие статические методы для работы с промисами есть?

**Статические методы для работы с промисами**

- Promise — встроенный **конструктор** в JS
- **Promise.all(iterable)** — ждёт **все** промисы, отклоняется при первой ошибке
- **Promise.race(iterable)** — ждёт **первый** завершённый (выполнен или отклонён) промис
- **Promise.allSettled(iterable)** — ждёт завершения всех, возвращает их статусы и значения
- **Promise.any(iterable)** — ждёт **первое** успешно выполненное, отклоняется если все ошибочны
- **Promise.resolve(value)** — создаёт **выполненный** промис из значения
- **Promise.reject(reason)** — создаёт **отклонённый** промис с причиной ошибки

Эти методы упрощают **параллельную** и **агрегационную** работу с асинхронным кодом, обеспечивают удобное управление состояниями множества промисов.

# Подробный ответ
## Основной ответ
В JavaScript для работы с **Promise** существуют несколько ключевых статических методов, которые позволяют контролировать выполнение нескольких промисов, создавать уже завершённые или отклонённые промисы и комбинировать их результаты. Основные из них — это `Promise.resolve()`, `Promise.reject()`, `Promise.all()`, `Promise.allSettled()`, `Promise.race()` и `Promise.any()`.

## Ключевые моменты
- **`Promise.resolve(value)`** создаёт успешно выполненный промис с заданным значением. Если передан уже промис, он вернёт его без изменений. Это удобно для гарантирвания, что возвращается промис.
- **`Promise.reject(reason)`** создаёт промис, который сразу отклоняется с указанной причиной — полезно для генерации ошибок в цепочках промисов.
- **`Promise.all(iterable)`** ожидает завершения всех промисов из массива или другого итерируемого объекта. Возвращает массив результатов в исходном порядке, но сразу отклоняется, если хотя бы один промис отклонён. Подходит для параллельной работы с зависимыми задачами.
- **`Promise.allSettled(iterable)`** ждёт завершения всех промисов, независимо от результата (fulfilled или rejected), возвращая для каждого объект с состоянием и значением/ошибкой. В React 18+ часто используется для безопасного сбора результатов без ошибки.
- **`Promise.race(iterable)`** возвращает первый выполненный или отклонённый промис из переданного набора. Полезно для таймаутов и ожидания самого быстрого результата.
- **`Promise.any(iterable)`** возвращает первый успешно выполненный промис, игнорируя отклонённые, и отклоняется только если все промисы отклонены (добавлен в ES2021). Удобен, когда нужен результат хотя бы одного успешного промиса.

## Практический контекст
На практике `Promise.all` часто применяют для параллельных API-запросов, где нужна успешная загрузка всех данных. `Promise.race` — для реализации таймаутов на сетевые операции. `Promise.allSettled` полезен при работе с массивом независимых задач, где важно получить статус каждой, а не прерывать цепочку на первой ошибке. Использование `Promise.any` помогает быстрее получить успешный ответ из группы источников, например, при call failover в распределённых системах.
