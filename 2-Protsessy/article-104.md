# Как в Go реализуется наследование?

**Как в Go реализуется наследование?**

- Go не имеет классического наследования (нет классов, extends)
- Используется **встраивание (embedding)** структур для повторного использования кода
- Встраивание добавляет поля и методы вложенной структуры в "родительскую"
- Позволяет реализовать **поведение типа "has-a" и "is-a"** через интерфейсы
- Интерфейсы отделяют поведение от конкретной реализации
- Комбинация embedding + интерфейсы заменяет классическое наследование
- Применение: расширение структур без дублирования логики, компоновка функционала

# Подробный ответ
## Основной ответ
В **Go** прямого механизма наследования, как в классических объектно-ориентированных языках, нет. Вместо этого используется концепция **встраивания** (embedding) структур и **интерфейсов** для достижения полиморфизма и повторного использования кода. Встраивание позволяет одной структуре включать другую структуру как анонимное поле, благодаря чему методы вложенной структуры становятся доступными для внешней без явного переопределения.

## Ключевые моменты
- **Встраивание структур** — базовая способность Go для компоновки «наследуемого» поведения без наследования классов. Это похоже на композицию, но с «прозрачным» доступом к методам.
- **Интерфейсы** — позволяют определить набор методов, которые должна реализовать структура, реализуя таким образом полиморфизм без жесткой иерархии классов.
- **Отсутствие сложной иерархии** — Go поощряет плоскую структуру и предпочтение композиции над наследованием, что повышает простоту и читаемость кода.

## Практический контекст
Типично в Go struct A встраивает struct B для «наследования» методов B. Например, в стандартной библиотеке io.Writer часто встраивается в структуры для добавления возможности записи. Такой подход хорошо подходит для написания легковесных и гибких компонентов с минимальной связностью, что актуально для высокопроизводительных приложений на Go.
