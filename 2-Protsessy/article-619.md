# В чём разница между push и pull моделями?

**Разница между push и pull моделями**  
- Архитектурные паттерны передачи данных  
- Push: источник **активно отправляет** данные подписчикам  
- Pull: получатель **инициирует запросы** данных у источника  
- Push снижает задержки, но **сложнее контролировать поток**  
- Pull упрощает контроль и **устраняет избыточные данные**  
- Push подходит для **универсальных обновлений**, Pull — для **детализированных запросов**  
- Используется в потоках данных, микросервисах, event-driven системах

# Подробный ответ
## Основной ответ
**Push-модель** и **pull-модель** — это два подхода к передаче данных или событий от источника к потребителю. В push-модели данные активно "толкаются" от провайдера к клиенту без явного запроса, а в pull-модели клиент самостоятельно запрашивает данные у сервера или источника.

## Ключевые моменты
- В **push-модели** (например, WebSocket, Server-Sent Events, push-уведомления) сервер сам инициирует отправку данных сразу, как они становятся доступны. Это снижает задержки (latency) для клиента, что важно для real-time систем, но требует у клиента готовности принимать данные в любое время.
- В **pull-модели** (например, REST API с периодическим опросом, HTTP polling) клиент регулярно или по событию запрашивает актуальную информацию у сервера. Это проще в реализации и масштабировании, но часто ведет к избыточным запросам и более высокой задержке обновлений.
- Выбор между push и pull зависит от требований к скорости обновления, нагрузке и ресурсам. Например, системам с high-frequency events и низкой latency предпочтительна push-модель, а для данных с низкой изменчивостью — pull-модель чаще достаточно.

## Практический контекст
В современных веб-приложениях push-модель применяют для real-time чатов, торговых платформ (WebSocket), а pull — для стандартных REST-запросов к API с кешированием. Часто используют гибрид: initial pull, затем push-уведомления о смене состояния.
