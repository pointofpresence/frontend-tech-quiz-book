# Как организовать параметризованные тесты? (использование @pytest.mark.parametrize)

Категория: Процессы

### Краткий ответ

**Как организовать параметризованные тесты с pytest**

- pytest — популярный фреймворк для тестирования Python
- `@pytest.mark.parametrize` — декоратор для **повторения теста с разными данными**
- Принимает **имена параметров** строкой и список значений (кортежей)
- Позволяет запускать один тест с множеством вариаций входных данных
- Упрощает тестирование с разными сценариями и данными
- Повышает покрытие и уменьшает код дублирования
- Полезен при проверке граничных случаев и разных типов входа

**Пример**:
```python
@pytest.mark.parametrize("input,expected", [(1,2), (3,4), (5,6)])
def test_increment(input, expected):
    assert input+1 == expected
```
  
Вывод: параметризация — ключ к эффективному и компактному тестированию с pytest.

### Подробный ответ
#### Основной ответ
Для организации **параметризованных тестов** в **pytest** используют декоратор **`@pytest.mark.parametrize`**, который позволяет запускать одну и ту же тестовую функцию с разными наборами входных данных. Это значительно повышает читаемость и поддержку тестов, сокращая дублирование кода.

Синтаксис прост: первым аргументом передаётся строка с именами параметров через запятую, вторым — список кортежей или других итерируемых структур с соответствующими значениями. Например, `@pytest.mark.parametrize("input,expected", [(1, 2), (3, 4)])` запустит тест дважды — с входными значениями 1 и 3 соответственно.

Параметризация полезна для проверки функции на разных входных данных, покрывая множество случаев в одном тесте.

#### Ключевые моменты
- Можно параметризовать несколько аргументов одновременно — улучшает комбинирование сценариев.
- При большом наборе данных легко интегрируется с генераторами, списками из CSV или JSON, что автоматизирует подготовку данных.
- Важно, что каждый запуск parametrized test воспринимается pytest как отдельный тест с разными идентификаторами в отчётах, что облегчает диагностику ошибок.

#### Практический контекст
В проекте на Python 3.9+ с pytest 7+ при тестировании REST API я часто параметризую разные варианты запросов и ожидаемых ответов, экономя сотни строк отдельных функций. Это ускоряет тестирование и упрощает поддержку, особенно при расширении набора кейсов при новых требованиях.
