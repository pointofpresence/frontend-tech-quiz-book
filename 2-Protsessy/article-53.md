# Что такое Code on Demand в REST?

**Что такое Code on Demand в REST?**

- одно из шести **ограничений REST**
- сервер передаёт клиенту **код для исполнения** (например, скрипты)
- позволяет расширять функционал клиента динамически
- уменьшает количество запросов к серверу
- усложняет кэширование и снижает безопасность
- используется редко, т.к. нарушает принцип простоты REST
- практическое применение — **JavaScript в веб-браузерах** для динамических интерфейсов

# Подробный ответ
## Основной ответ
**Code on Demand** — это один из пяти архитектурных стилей, определённых в REST (Representational State Transfer) в классической работе Роя Филдинга. Он позволяет серверу динамически передавать клиенту исполняемый код (обычно скрипты), который клиент может запускать для расширения своей функциональности. Это повышает гибкость и уменьшает жёсткую привязку клиента к серверу, поскольку логика обработки может изменяться без обновления клиентского приложения.

## Ключевые моменты
- **Опциональность**: Code on Demand считается необязательной REST-архитектурой; большинство API не используют её, но она даёт мощный инструмент для сложных клиентских сценариев.
- **Типы кода**: чаще всего это JavaScript-код в веб-приложениях, загружаемый вместе с ответом сервера и исполняемый клиентом для динамической работы с данными.
- **Преимущества и риски**: Code on Demand повышает расширяемость и динамичность, но увеличивает безопасность и производительность приложений, т.к. исполняемый код может быть потенциально опасным.

## Практический контекст
В реальных REST API Code on Demand используется редко — наиболее близкий пример — веб-приложения, где сервер отдаёт клиенту JavaScript для динамического рендера, либо REST сервисы, возвращающие с ответом скрипты для обработки данных на стороне клиента. Такой подход позволяет обновлять логику клиентов без их перезагрузки или обновления.
