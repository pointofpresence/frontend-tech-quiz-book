# Что такое абстрактные классы?

Категория: Процессы

### Краткий ответ

**Что такое абстрактные классы?**  
- концепт объектно-ориентированного программирования  
- класс с **абстрактными методами** без реализации  
- нельзя создать экземпляр напрямую  
- служит шаблоном для наследников  
- обеспечивает **обязательные** методы в подклассах  
- часто содержит реализованные полезные методы  
- помогает задать общий интерфейс и логику  
- улучшает архитектуру и поддерживаемость кода

### Подробный ответ
#### Основной ответ
**Абстрактный класс** — это класс в объектно-ориентированном программировании, который не может быть инстанцирован напрямую и служит для определения общего интерфейса и базового поведения для подклассов. Он содержит как реализации методов, так и абстрактные (не реализованные) методы, которые обязаны переопределять наследники.

#### Ключевые моменты
- **Нельзя создать объект напрямую** — абстрактный класс задаёт общий контракт, а конкретная реализация предоставляется в подклассах.
- **Абстрактные методы** — методы без тела, которые требуют обязательной реализации в наследниках, что обеспечивает полиморфизм и консистентность интерфейса.
- Абстрактные классы удобно использовать для организации сложной иерархии, где базовый класс задаёт шаблон поведения, а производные — конкретные детали.
- В языках как Java (начиная с версии 1.0) или C# абстрактные классы часто применяются для шаблонного проектирования (Template Method pattern).

#### Практический контекст
Например, в проекте с разными типами сотрудников абстрактный класс `Employee` может содержать общие методы и поля (как `calculateSalary()`) и абстрактный метод `work()`, который будет реализован по-разному для `Developer` и `Manager`. Это облегчает поддержку и расширяемость кода, а также гарантирует, что все подклассы используют один и тот же базовый контракт.
