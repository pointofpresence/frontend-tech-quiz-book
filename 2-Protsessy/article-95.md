# Почему Go runtime использует “work stealing” модель и как она работает?

**Почему Go runtime использует “work stealing” и как она работает?**  
- concurrency model Go (Goroutine scheduler)  
- уменьшение простоя CPU через балансировку нагрузки  
- каждый P (processor) имеет локальную очередь задач (goroutines)  
- при опустошении локальной очереди P “крадёт” задачи у других загруженных P  
- обеспечивает равномерное распределение goroutine по потокам OS  
- снижает конкуренцию за глобальные структуры, ускоряет планирование  
- повышает общую производительность и масштабируемость программ на Go

# Подробный ответ
## Основной ответ
**Go runtime использует модель "work stealing"** для эффективного распределения задач (горутин) между имеющимися потоками операционной системы (M — ОС потоки). Эта модель помогает максимально использовать доступные CPU, снижать простой потоков и улучшать масштабируемость многопоточных приложений. Идея в том, что если один поток (M) закончил обработку своей очереди горутин (task queue), он может "украсть" задачи у другого загруженного потока, балансируя нагрузку динамически.

## Ключевые моменты
- В Go у каждого потока ОС есть локальная очередь горутин (P — процессоры), что уменьшает конкуренцию за общие ресурсы и повышает кэш-попадания.
- Если локальная очередь пуста, поток пытается украсть задачи из очередей других потоков, что предотвращает ситуацию, когда один поток простаивает, а другой перегружен.
- Такая модель значительно улучшает производительность и масштабируемость, особенно на системах с большим числом ядер (в Go runtime версии 1.5+).
- Work stealing в Go работает вместе с гибкой планировкой горутин, позволяя runtime эффективно работать на уровне сотен тысяч легковесных горутин.

## Практический контекст
В реальных проектах благодаря work stealing Go отлично подходит для высоконагруженных серверов и распределённых систем, где количество горутин может значительно превышать число ОС потоков. Это приводит к хорошему распределению CPU ресурсов, низкой латентности и высокой throughput, например, в Kubernetes (на Go) или в системах обработки потоков данных.
