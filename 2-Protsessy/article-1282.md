# Что такое tree shaking в Webpack?

Категория: Процессы

### Краткий ответ

**Что такое tree shaking в Webpack?**

- оптимизация **уборки неиспользуемого кода**
- базируется на **анализе экспорта/импорта ES6 модулей**
- удаляет **мертвый код** (dead code elimination)
- повышает **эффективность бандла** и снижает размер
- работает только с **статическими импортами** (ESM), не с CommonJS
- требует **минимизации кода** (например, Terser) для финального удаления
- широко используется в **production-сборках** для оптимизации загрузки страниц

### Подробный ответ
#### Основной ответ
**Tree shaking** в **Webpack** — это оптимизация, которая позволяет удалить из финального бандла неиспользуемый (dead) код на этапе сборки. Основная идея — "стряхнуть" неиспользуемые экспорты из модулей, чтобы уменьшить размер итогового файла и улучшить производительность загрузки.

#### Ключевые моменты
- Tree shaking работает на базе анализа **ES6 модулей** (ES Modules), поскольку они содержат статические `import` и `export`, что позволяет определить, какие части кода реально используются.
- В Webpack (начиная с версии 2) эта технология интегрирована в процесс сборки, но для эффективного tree shaking требуется, чтобы код был написан с использованием ES6 `import/export`. CommonJS модули не поддерживают статический анализ на том же уровне.
- Важна правильная конфигурация: режим сборки должен быть установлен в `'production'`, где включен оптимизатор `TerserPlugin` для удаления мертвого кода на этапе минификации.
- Tree shaking не работает с динамическими импортами или неявными side effects, поэтому часто нужно правильно указывать в `package.json` поле `"sideEffects": false` либо указывать, какие файлы могут иметь побочные эффекты.

#### Практический контекст
В реальных проектах tree shaking помогает значительно уменьшать bundle size, особенно при использовании крупных библиотек типа **lodash-es** или **RxJS**, где можно импортировать только необходимые функции вместо всей библиотеки. В React 18+, например, tree shaking позволяет исключить неиспользуемые внутренние утилиты, что снижает нагрузку на клиент.
