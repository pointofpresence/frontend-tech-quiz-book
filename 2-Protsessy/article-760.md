# Чем __init__() отличается от __new__()?

Категория: Процессы

### Краткий ответ

**Чем __init__() отличается от __new__()?**

- __new__ — **создаёт экземпляр класса** (вызывается первым)
- __init__ — **инициализирует созданный объект**
- __new__ — статический метод, возвращает **новый экземпляр**
- __init__ — обычный метод, **настраивает атрибуты**
- __new__ используется в основном для **наследования immutable типов**
- без __new__ объект не создастся, без __init__ — будет пустой
- __init__ — чаще переопределяется для логики и параметров

Итог: __new__ отвечает за создание объекта, __init__ — за его настройку.

### Подробный ответ
#### Основной ответ
Метод **`__new__()`** отвечает за создание нового экземпляра класса, то есть за выделение памяти и возврат нового объекта. В то время как **`__init__()`** вызывается уже после создания объекта и выполняет его инициализацию, то есть наполняет атрибутами и задаёт начальное состояние. Проще говоря, `__new__()` создаёт объект, а `__init__()` — настраивает его.

#### Ключевые моменты
- **`__new__()` — статический метод**, принимает класс в качестве первого аргумента и должен вернуть экземпляр этого класса или его производного. Его редко переопределяют, кроме случаев работы с неизменяемыми типами (например, с `tuple` или `str`), или когда нужно контролировать сам процесс создания объекта.
- **`__init__()` — метод экземпляра**, вызывается сразу после `__new__()` для конкретного объекта, инициализирует внутреннее состояние, например, присваивает значения атрибутам.
- Если `__new__()` возвращает не объект текущего класса, `__init__()` вызван не будет, что важно учитывать при наследовании и паттернах проектирования.

#### Практический контекст
В типичных пользовательских классах на Python обычно достаточно реализовать только `__init__()`. Переопределение `__new__()` встречается в библиотеках и продвинутых сценариях, например, когда нужно реализовать паттерн Singleton или создавать объекты с изменённым поведением на уровне памяти. В Python 3.7+ чаще всего `__new__()` вызывает `super().__new__()` для корректного наследования и создания экземпляра.
