# Что такое хэш-таблица? Что под капотом у словаря (dict) и множества (set) в Python? (Хэш, коллизии, открытая адресация).

Категория: Процессы

### Краткий ответ

**Что такое хэш-таблица? Словарь и множество в Python (хэш, коллизии, открытая адресация)**

- структура данных **ключ → значение** (dict) или уникальные элементы (set)  
- основаны на **хэш-таблице** с вычислением хэша ключа через __hash__  
- **хэш-функция** преобразует ключ в индекс массива  
- коллизии решаются через **открытую адресацию** (линейное пробирование)  
- при коллизии ищется следующая свободная ячейка  
- Python dict использует **динамическое изменение размера** (рехэшинг) для производительности  
- высокая скорость операций: **O(1)** для вставки, поиска и удаления  
- практическое: универсальная и быстрая реализация для ключ-значение и уникальных элементов  

Этот механизм обеспечивает эффективность и гибкость dict и set в Python, сохраняя производительность даже при росте данных.

### Подробный ответ
#### Основной ответ
**Хэш-таблица** — это структура данных, которая обеспечивает быстрый доступ к элементам за амортизированное время O(1) благодаря использованию специальной хэш-функции для вычисления индекса в массиве. В Python объекты типа **dict** и **set** реализованы на основе хэш-таблиц.

Под капотом у **dict** и **set** используется массив с элементами-записями, где ключи (а для множества — сами элементы) хэшируются с помощью встроенной хэш-функции `__hash__()`. Результат хэширования преобразуется в индекс массива. Если возникают **коллизии** (два разных объекта дают один и тот же индекс), Python применяет технику **открытой адресации** с использованием квадратичного пробирования — при коллизии ищется следующий индекс по формуле, минимизируя цепочки и замедление.

#### Ключевые моменты
- **Хэш и коллизии:** Хэш-функция в Python рассчитана на равномерное распределение по корзинам. При коллизии используется квадратичное пробирование, а не цепочки (linked lists), что снижает издержки, но требует аккуратного управления размером массива.
- **Отношение загрузки (load factor):** Когда количество элементов достигает примерно 2/3 от размера массива, происходит **резайзинг** — массив увеличивается в 2 раза, и все элементы повторно хэшируются для поддержания латентности доступа ~O(1).
- **Dict vs Set:** По сути, dict хранит пары (ключ, значение), а set — только ключи. При этом реализация идентична, отличие лишь в структуре записи.

#### Практический контекст
В Python 3.6+ словари сохраняют порядок вставки элементов благодаря оптимизациям в хэш-таблице (compact dict). Эта реализация увеличивает эффективность памяти и оптимизирует перебор элементов без потери быстродействия доступа. Для множества используется аналогичный подход. Знание этих деталей важно, когда нужно оптимизировать производительность кэширования, реализации мемоизации или уникальные коллекции.
