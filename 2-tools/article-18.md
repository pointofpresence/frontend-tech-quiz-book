# Какие способы загрузки библиотек существуют?

### Категории

iOS, Loading, Mobile, Libraries, Tools

### Краткий ответ

**Какие способы загрузки библиотек существуют?**

- статическая загрузка: библиотеки подключаются на этапе компиляции, код встраивается в файл
- динамическая загрузка: библиотеки загружаются во время выполнения (runtime), например, DLL, SO
- ленивое (отложенное) подключение: загрузка при первом вызове функции из библиотеки
- загрузка через пакетные менеджеры: npm, pip, Maven — динамическое получение и установка
- загрузка из CDN: для веб-библиотек, подключение по URL без локальной установки
- внедрение через модульные системы: ES Modules, CommonJS, AMD — управление загрузкой и зависимостями
- загрузка с помощью reflection или runtime API (например, .NET Assembly.Load)

Основная цель — оптимизация скорости старта, контроль зависимостей и размер конечного приложения.

### Подробный ответ
#### Основной ответ
Существует несколько основных способов загрузки библиотек, которые зависят от среды и языка программирования. В общем случае можно выделить **статическую**, **динамическую (загрузку во время исполнения)** и **отложенную загрузку (lazy loading)**. Каждый способ применяется в зависимости от требований к производительности, размеру бинарника и гибкости обновления.

#### Ключевые моменты
- **Статическая загрузка**: библиотеки связываются с программой при компиляции и включаются в итоговый бинарник или как отдельные статические либы (.lib, .a). Такой подход гарантирует, что все необходимые зависимости доступны и не требуют загрузки во время выполнения, но увеличивает размер итогового файла и уменьшает гибкость обновлений.
- **Динамическая загрузка (shared libs, DLL, .so)**: библиотеки подключаются во время запуска или исполнения программы операционной системой. Позволяет экономить память и облегчает обновления библиотек без перекомпиляции основного приложения. В Windows — DLL, в Linux/macOS — .so/.dylib. Существуют также API для динамической загрузки (например, dlopen/dlsym или LoadLibrary/GetProcAddress), что позволяет загружать библиотеки и функции непосредственно в рантайме.
- **Отложенная загрузка (lazy loading)**: библиотеки или их части загружаются только по требованию, например, при первом вызове функции. Это снижает время старта приложения и экономит ресурсы, особенно в больших системах. Часто используется в веб-разработке (например, динамический импорт в ES6-модулях или код-сплиттинг в Webpack).
  
#### Практический контекст
В реальных проектах часто комбинируют эти подходы. Например, базовые библиотеки загружаются статически для скорости, а плагины или редко используемые модули — динамически или лениво. В React 18+ применяют lazy loading для улучшения UX, а в бэкендах микросервисов могут подгружать shared libs через Docker образы для снижения размера контейнеров.
