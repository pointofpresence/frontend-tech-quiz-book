# Что такое мутабельные и иммутабельные типы в Java?

### Категории

System design, Data types, Mutability, Memory management, Backend

### Краткий ответ

**Мутабельные и иммутабельные типы в Java**

- концепции управления **изменяемостью** объектов  
- **мутабельные** объекты можно **изменять** после создания  
- **иммутабельные** объекты нельзя менять после создания  
- мутабельные: примеры — `StringBuilder`, массивы, коллекции (например, `ArrayList`)  
- иммутабельные: примеры — `String`, класс `Integer`, `LocalDate`  
- иммутабельность обеспечивает **потокобезопасность** и предсказуемость  
- предпочтение иммутабельных объектов для **конкурентных программ** и **читаемости** кода

### Подробный ответ
#### Основной ответ
**Мутабельные типы** в Java — это объекты, состояние которых может изменяться после создания. **Иммутабельные типы** — объекты, чьё состояние фиксировано и не меняется после инициализации. То есть, для иммутабельного объекта любые операции возвращают новый объект или не меняют внутреннее состояние существующего.

#### Ключевые моменты
- **Примеры мутабельных типов**: `StringBuilder`, коллекции из `java.util` (например, `ArrayList`, `HashMap`). Их методы изменяют внутренние данные объекта.
- **Примеры иммутабельных типов**: `String`, классы-обертки примитивов (`Integer`, `Double`), `LocalDate` из `java.time`. Они гарантируют, что при повторном использовании объекта не произойдёт неожиданного изменения состояния, что важно для потокобезопасности.
- **Преимущества иммутабельных объектов**: проще использовать в конкурентных системах, кэшировать, избегать side-effect'ов. Однако в мутабельных — меньше накладных расходов на создание новых объектов, что может быть эффективнее по памяти и производительности при частых изменениях.

#### Практический контекст
В Java проекты часто сочетают оба типа: иммутабельные объекты используют для представления данных (DTO, value objects), а мутабельные — для построения сложных структур и промежуточных состояний. Например, в Java 8+ предпочтительно использовать `LocalDate` и `String`, чтобы избежать ошибок многопоточности, а для эффективной обработки строк — `StringBuilder`.
