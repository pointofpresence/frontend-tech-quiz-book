# Что происходит при перерендере компонента?

### Категории

System design, Performance, Component, Rendering, Frontend, Live coding

### Краткий ответ

**Что происходит при перерендере компонента?**  
- UI-фреймворк инициирует повторный вызов функции/метода компонента  
- происходит повторный расчёт JSX/шаблона на основе текущих пропсов и состояния  
- сравнение с предыдущим рендером (diffing) для оптимизации изменений  
- обновляется виртуальное DOM (React) или аналогичный промежуточный слой  
- минимальное изменение реального DOM через батчинг и реконсиляцию  
- при изменении состояния или пропсов гарантирует актуальность интерфейса  
- практическая задача — повысить отзывчивость и синхронизацию UI с данными

### Подробный ответ
#### Основной ответ
При **перерендере компонента** в React происходит повторный вызов функции компонента или метода render (в классовых компонентах), что приводит к формированию нового виртуального DOM-дерева. После этого React сравнивает новое виртуальное дерево с предыдущим (процесс называется reconciliation) и эффективно обновляет реальный DOM, изменяя только те части, которые действительно изменились.

#### Ключевые моменты
- **Триггеры перерендеринга**: обновление state или props компонента, вызов forceUpdate или контекстные изменения (React Context). В React 18+ оптимизации и батчи делают процесс более эффективным.
- **Reconciliation**: React сравнивает новое виртуальное DOM с предыдущим, используя алгоритм диффинга (diffing), чтобы минимизировать операции с реальным DOM, так как они медленные.
- **Оптимизации**: использование React.memo для функциональных и shouldComponentUpdate для классовых компонентов позволяет избежать ненужных перерендерингов, сохраняя производительность.
- Особенность: перерендер не всегда значит изменение реального DOM — если выход render совпадает с предыдущим, изменений не происходит.

#### Практический контекст
В реальных приложениях перерендеры могут приводить к лагам, если не использовать мемоизацию, оптимизировать передачу пропсов и компоненты, особенно в больших списках (например, через виртуализацию с react-window). React DevTools помогает отслеживать что именно перерендеривается и почему.
