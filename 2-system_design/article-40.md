# Какие хранилища подходят для больших данных (LocalStorage, IndexedDB)?

### Категории

System design, Browser storage, Performance, Storage, Big data, Frontend

### Краткий ответ

**Хранилища для больших данных (LocalStorage vs IndexedDB)**  
- обе технологии — веб-хранилища браузера для данных  
- **LocalStorage**: синхронное, ограничено ~5 МБ, ключ-значение, простое API  
- **IndexedDB**: асинхронное, объём практически неограничен, основано на индексе, сложный API  
- LocalStorage подходит для небольших строковых данных и настроек  
- IndexedDB оптимален для больших объёмов, структурированных и бинарных данных  
- IndexedDB поддерживает транзакции, поиск, курсоры и сложные запросы  
- для **больших данных подходит IndexedDB**, LocalStorage слишком ограничен по размеру и производительности  

Итог: **IndexedDB — лучший выбор для хранения больших объёмов данных в браузере** благодаря производительности, масштабируемости и функциональности.

### Подробный ответ
#### Основной ответ
Для хранения **больших данных** в браузерном окружении из перечисленных подходящих вариантов — **IndexedDB** значительно предпочтительнее, чем LocalStorage. LocalStorage ограничен примерно 5-10 МБ и предназначен для простого хранения строковых данных, а IndexedDB — это полноценная база данных, которая позволяет хранить большие объемы разнообразных структурированных данных.

#### Ключевые моменты
- **LocalStorage**: синхронное API, ограниченный размер (5-10 МБ в зависимости от браузера), хранит данные только в формате строки без индексации, что делает его неэффективным для больших и сложных структур.
- **IndexedDB**: асинхронное API с возможностью хранения десятков мегабайт и более, поддерживает бинарные данные (Blob, ArrayBuffer), поисковые индексы и транзакции, что подходит для масштабных локальных хранилищ.
- IndexedDB проектировался для офлайн-приложений и позволяет эффективно хранить и быстро обращаться к большим объемам данных с минимальным влиянием на UI благодаря асинхронности.

#### Практический контекст
IndexedDB обычно используется в PWA и веб-приложениях, где нужно сохранить **большие изображения, видео, или сложные структуры данных** клиента при ограниченном доступе к серверу. LocalStorage — хорош для настроек или небольших кэшированных значений, но не для больших данных из-за ограничений размера и синхронного блокирования потока JavaScript.
