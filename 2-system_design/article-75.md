# Что такое CTE (Common Table Expressions) в PostgreSQL?

### Категории

System design, Query optimization, Backend, Database, Real cases, Common table expression, CTE, SQL

### Краткий ответ

**Что такое CTE (Common Table Expressions) в PostgreSQL?**  
- SQL-конструкция для **временных именованных результатов**  
- определяется через ключевое слово **WITH**  
- действует в пределах одного запроса  
- улучшает читабельность и структуру сложных запросов  
- поддерживает рекурсивные запросы (**WITH RECURSIVE**)  
- позволяет избежать дублирования подзапросов  
- часто ускоряет разработку и отладку запросов в аналитике и ETL

### Подробный ответ
#### Основной ответ
**CTE (Common Table Expressions)** в **PostgreSQL** — это временные именованные результаты запроса, которые можно использовать внутри основного SQL-запроса. Они определяются с помощью ключевого слова `WITH` и позволяют структурировать сложные запросы, улучшая читаемость и повторное использование промежуточных результатов.

#### Ключевые моменты
- **Рекурсивные и не рекурсивные CTE**: рекурсивные CTE (с использованием `WITH RECURSIVE`) позволяют строить иерархические запросы, например, обход древовидных структур.
- **Оптимизация и читаемость**: CTE делают сложные запросы модульными, упрощают поддержку и дебаг, но в PostgreSQL до версии 12 они по умолчанию материализуются — то есть вычисляются один раз и хранятся в памяти, что иногда может влиять на производительность.
- **Использование в современных версиях**: начиная с PostgreSQL 12+, оптимизатор может "inline"-ить CTE, превращая их в подзапросы, если это выгоднее, улучшая execution план.

#### Практический контекст
CTE широко используют для сложных агрегаций, декомпозиции запросов в data warehousing, построения отчетов, иерархий (например, структуры организационной схемы) и оптимизации запросов с множественными повторными вычислениями одних и тех же подзапросов. Типично применяются с аналитическими функциями и рекурсивными алгоритмами в PostgreSQL 14+ проектах.
