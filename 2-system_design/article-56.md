# В чём разница между IP и MAC адресацией?

### Категории

Profiling, Addresses, IP, System design, Concurrency, Memory management, Backend, Messaging, Mac, Networking

### Краткий ответ

**В чём разница между IP и MAC адресацией?**

- IP — логический адрес в сети Интернет (сетевой уровень OSI)  
- MAC — физический адрес сетевого интерфейса (канальный уровень OSI)  
- IP адрес может меняться при переходе между сетями (динамический или статический)  
- MAC адрес фиксирован и уникален для устройства производителем  
- IP помогает маршрутизировать пакеты через разные сети (глобальная адресация)  
- MAC используется для передачи данных внутри одной локальной сети (LAN)  
- IP поддерживает иерархию и деление сети (подсети), MAC — плоская структура  
- IP-адреса ассоциируются с устройствами временно, MAC — постоянно  
- IP — универсальный в интернете, MAC — важен для локальных коммуникаций (ARP)  
- Вместе обеспечивают корректную адресацию и доставку пакетов в сети  
---
**Итого:** IP — логический, маршрутизируемый адрес; MAC — физический, уникальный идентификатор устройства в локальной сети.

### Подробный ответ
#### Основной ответ
**IP-адресация** и **MAC-адресация** решают разные задачи в сетях. **MAC-адрес** — это уникальный физический идентификатор сетевого интерфейса, используемый для локальной доставки пакетов в одном сегменте LAN. **IP-адрес** — это логический адрес, который используется для маршрутизации пакетов между разными сетями в интернете или корпоративной сети.

#### Ключевые моменты
- **MAC-адрес** — 48-битный уникальный идентификатор, закрепленный за сетевым адаптером производителем по стандарту IEEE 802, используется на канальном уровне (Data Link Layer, уровень 2 модели OSI). Он статичен и не меняется при перемещении устройства.
- **IP-адрес** — 32-битный (IPv4) или 128-битный (IPv6) логический адрес, назначаемый устройству для идентификации в глобальной или локальной сети, работает на сетевом уровне (Network Layer, уровень 3). Может меняться, например, при переходе между сетями или использовании DHCP.
- В процессе передачи данных: на локальном сегменте ориентируются на MAC, а для межсетевой доставки — с помощью IP. Протокол ARP переводит IP-адрес в MAC для локальной передачи.

#### Практический контекст
В инфраструктуре часто используют **DHCP** для динамического назначения IP, а MAC помогает контролировать доступ и фильтрацию на уровне коммутаторов. Без MAC невозможна физическая доставка кадров, а без IP — маршрутизация и связь в интернете невозможны. Например, в Cisco сетях MAC работает с VLAN и коммутаторами, а IP — с маршрутизаторами для межсетевого взаимодействия.
