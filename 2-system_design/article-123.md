# Какие паттерны программирования знаешь?

### Категории

Programming patterns, System design, Backend, Software design

### Краткий ответ

**Какие паттерны программирования знаешь?**

- **Порождающие:** создание объектов (Singleton, Factory, Builder)  
- **Структурные:** связи и компоновка (Adapter, Decorator, Composite)  
- **Поведенческие:** взаимодействие и алгоритмы (Observer, Strategy, Command)  
- SOLID помогает понимать дизайн  
- Применяются для **повторного использования** и **поддержки** кода  
- Знаю разные уровни абстракции: от простых до архитектурных (MVC, MVVM)  
- Использую паттерны в реальных проектах для повышения качества и масштабируемости

### Подробный ответ
#### Основной ответ
Паттерны программирования — это проверенные решения часто возникающих задач проектирования и разработки, позволяющие создавать более читаемый, поддерживаемый и расширяемый код. Я знаком с основными категориями паттернов: **порождающие**, **структурные** и **поведенческие**. Использую их в зависимости от задачи и архитектурного контекста.

#### Ключевые моменты
- **Порождающие паттерны**: например, *Singleton* (гарантирует единственный экземпляр класса), *Factory Method* (создаёт объекты без прямого вызова конструктора), *Builder* (пошаговое создание сложного объекта). В React 18 часто встречаю `Factory` для абстрагирования создания компонентов.
- **Структурные паттерны**: *Adapter* (приспосабливает интерфейс), *Decorator* (добавляет функциональность объекту динамически), *Facade* (упрощает взаимодействие с подсистемой). Например, Facade помогает скрыть сложность интеграции с внешним API.
- **Поведенческие паттерны**: *Observer* (подписка на события, как в Redux или EventEmitter), *Strategy* (выбор алгоритма в runtime), *Command* (инкапсуляция запроса в объект). Обычно Observer используем для реализации событийных систем с высокой отзывчивостью и масштабируемостью.

#### Практический контекст
В продакшене регулярно применяю *Factory* для создания сервисов с разными параметрами, *Observer* для event-driven архитектуры и *Decorator* для расширения функциональности middleware в Node.js. Знания паттернов помогают писать код, который легче тестировать и развивать в долгосрочной перспективе.
