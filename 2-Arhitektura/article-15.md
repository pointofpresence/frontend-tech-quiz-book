# Опишите шаги тестирования в Golang?

Категория: Архитектура

### Краткий ответ

**Опишите шаги тестирования в Golang**

- тестирование встроено в пакет **testing**
- создаём файл с суффиксом **_test.go**
- пишем функции **TestXxx(t *testing.T)**
- запускаем тесты командой **go test**
- используем **t.Error, t.Fatal** для ошибок
- для сложных проверок применяем **assert библиотеки**
- обеспечиваем покрытие через **go test -cover**

### Подробный ответ
#### Основной ответ
Шаги тестирования в **Golang** включают написание юнит-тестов с использованием стандартного пакета **testing**, запуск тестов и анализ их результатов. В Go тесты организованы по принципу отдельных функций, начинающихся с `Test`, размещённых в файлах с суффиксом `_test.go`. Запуск тестирования осуществляется через команду `go test`, которая автоматически находит и выполняет эти функции.

#### Ключевые моменты
- **Написание тестов**: Каждая функция теста принимает параметр типа `*testing.T`, который используется для проверки условий и логирования ошибок. Важно покрывать разные случаи — позитивные, негативные и пограничные варианты.
- **Использование бенчмарков и примеров**: Для измерения производительности используются функции с префиксом `Benchmark`, принимающие `*testing.B`. Для документации и демонстрации поведения кода есть функции с префиксом `Example`.
- **Модульное тестирование и изоляция**: При необходимости можно использовать мок объекты, для чего применяют внешние библиотеки вроде `gomock` или `testify/mock`. Стандартный пакет сам по себе минимален, но интегрируется с CI/CD.
- **Запуск и фильтрация тестов**: `go test -v ./...` запускает все тесты с подробным выводом, а флаг `-run` позволяет запускать выборочные тесты по регулярному выражению. Для гонок используется флаг `-race`.
- **Покрытие кода**: Команда `go test -cover` анализирует покрытие тестами исходного кода и помогает визуализировать непроверенные участки.

#### Практический контекст
В продакшн-проектах обычно реализуют интеграционные тесты наряду с юнитами, используют Docker для поднятия необходимых сервисов, а также подключают инструменты мониторинга и отчётности о покрытии (например, Codecov). В проектах с высокими требованиями к качеству тесты запускаются при каждом коммите через CI (GitHub Actions, GitLab CI).
