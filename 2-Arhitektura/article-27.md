# Объясни разницу между Kubernetes DaemonSet и Deployment. Для чего каждый используется?

**Разница Kubernetes DaemonSet и Deployment, применение**

- **Deployment** — управление **набором реплик** подов, обеспечивает **масштабируемость** и **обновления**  
- **DaemonSet** — запускает **копию пода на каждом (или выбранных) узле** кластера  
- Deployment **подходит для приложений с несколькими экземплярами**, которые можно масштабировать  
- DaemonSet нужен для **агентов, мониторинга, логирования или системных сервисов**, требующих присутствия на каждом/выбранных узлах  
- Deployment управляет **обновлением, откатом** версии приложения через контроллер  
- DaemonSet обеспечивает **однородность приложений на всех узлах** без лишних копий  
- Используйте Deployment для бизнес-логики, DaemonSet для функций инфраструктуры  

**Итог:** Deployment — масштабируемость и обновления приложений, DaemonSet — запуск сервисов на каждом узле кластера.

# Подробный ответ
## Основной ответ
**Kubernetes DaemonSet** и **Deployment** — это контроллеры управления состоянием для запуска подов, но с разным назначением и поведением. **DaemonSet** гарантирует запуск одного экземпляра пода на каждом узле (node) кластера, что полезно для задач, требующих локального присутствия на каждом сервере. В то время как **Deployment** создаёт и управляет указанный масштабируемый набор реплик подов, распределённых по кластерам для обработки нагрузки или обновления приложений.

## Ключевые моменты
- **DaemonSet** запускает по одному поду на каждый узел: идеально для систем мониторинга (например, Prometheus Node Exporter), сетевых агентов (например, Calico, Fluentd) или сервисов, требующих доступа к локальному хранилищу или хост-ресурсам.
- **Deployment** предназначен для запуска приложений с масштабируемостью и обновлением релизов (rolling update). Управляет количеством реплик, балансирует трафик, обеспечивает декларативные апдейты.
- **Lifecycle и обновления**: Deployment поддерживает стратегии развёртывания (rolling, recreate), а DaemonSet чаще обновляется при добавлении новых узлов, автоматически запускляя поды на них.

## Практический контекст
В реальной практике DaemonSet часто используется для запуска агентов сбора логов (Fluentd), мониторинга, безопасности, когда нужна локальная видимость на каждом ноде. Deployment же — базовый контроллер для запуска бизнес-логики приложений, API-серверов и микросервисов с гибким масштабированием и быстрыми обновлениями.

Таким образом, выбор зависит от задачи: если нужно что-то "на каждом узле" — DaemonSet; если нужна управляемая группа одинаковых приложений — Deployment.
