# Как в Go реализовать middleware?

Категория: Архитектура

### Краткий ответ

**Как в Go реализовать middleware?**

- HTTP middleware — это функция, оборачивающая http.Handler
- принимает и возвращает http.Handler, добавляя логику до/после вызова основного обработчика
- чаще используется для логирования, авторизации, обработки ошибок, CORS
- обычно реализуется как функция верхнего уровня: func(next http.Handler) http.Handler
- возвращаемый обработчик вызывает next.ServeHTTP(w, r) внутри своей логики
- middleware накладываются цепочкой, обеспечивая последовательную обработку запроса/ответа
- популярные фреймворки (Gorilla, Echo) имеют встроенную поддержку middleware

Это позволяет разделить функциональность, переиспользовать код и структурировать обработку HTTP-запросов гибко и прозрачно.

### Подробный ответ
#### Основной ответ
В Go **middleware** обычно реализуется как функция-обёртка над HTTP-хендлером, которая принимает `http.Handler` и возвращает новый `http.Handler`. Это позволяет добавлять дополнительную логику (логирование, аутентификация, обработку ошибок и т.д.) вокруг основного обработчика запросов без изменения его кода. Такой подход широко используется с пакетом **net/http** и фреймворками вроде **gin** или **chi**.

#### Ключевые моменты
- **Функции высшего порядка**: middleware — это функции вида `func(http.Handler) http.Handler`, что позволяет цепочечно "оборачивать" хендлеры.
- **Контекст выполнения**: внутри middleware можно изменять запрос, дописывать заголовки, проверять аутентификацию, логировать или изменять ответ.
- **Независимость и переиспользуемость**: middleware делит логику на мелкие модульные части, упрощающие тестирование и повторное использование.
- В Go стандартно используют интерфейс `http.Handler` и метод `ServeHTTP(ResponseWriter, *Request)`, что даёт универсальность.

#### Пример реализации
```go
func loggingMiddleware(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        log.Printf("Request URI: %s", r.RequestURI)
        next.ServeHTTP(w, r)
    })
}
```
Используется так:  
```go
http.Handle("/path", loggingMiddleware(myHandler))
```

#### Практический контекст
В реальных проектах, например на Go 1.19+, middleware часто строят цепочкой с помощью утилит вроде `alice` или встроенных конструкций фреймворков. Это позволяет гибко формировать pipeline обработки запросов с гарантией, что каждый middleware корректно вызовет следующий и обработает ответ.
