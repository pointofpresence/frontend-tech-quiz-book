# Как объединить обновление состояния для нескольких setState?

Категория: Архитектура

### Краткий ответ

**Как объединить обновление состояния для нескольких setState?**  
- Редактирование состояния в React, обновление UI  
- setState асинхронный, обновления могут **сливаться**  
- Использовать **функциональный setState** для доступа к текущему состоянию  
- Вызовы объединить в один объект с нужными изменениями  
- Например: `setState(prev => ({...prev, обновления}))`  
- Это предотвращает потерю данных при последовательных вызовах  
- Оптимизирует ререндер и обеспечивает консистентное состояние

### Подробный ответ
#### Основной ответ
В React, когда нужно обновить состояние с помощью нескольких вызовов `setState`, лучше объединять обновления, чтобы избежать непредсказуемых состояний и лишних перерисовок. Если использовать классические классовые компоненты, корректно объединять изменения нужно через функциональный вариант `setState`, который принимает функцию с текущим состоянием, позволяя гарантировать, что обновления основаны на актуальном значении. В функциональных компонентах с хуком `useState` стоит использовать одну функцию обновления или использовать `useReducer` для управления сложным состоянием.

#### Ключевые моменты
- В классах: **`setState` асинхронен и батчится**, поэтому несколько вызовов подряд могут быть слиты в одно обновление, но с риском потерять видимые промежуточные изменения без использования функционального варианта:  
```js
this.setState((prevState) => ({count: prevState.count + 1}));
```
- В функциональных компонентах `useState` значения обновляются также асинхронно — при нескольких обновлениях состояния рекомендуется объединить их в одно, либо использовать функции обновления:  
```js
setCount(prev => prev + 1);
setFlag(prev => !prev);
```
- Если состояние комплексное, лучше применить **`useReducer`**, который позволяет атомарно обновлять несколько частей состояния через один action с предсказуемой логикой.

#### Практический контекст
Например, при увеличении счетчика и переключении флага одновременно, не стоит писать  
```js
setCount(count + 1);
setFlag(!flag);
```
потому что `count` и `flag` могут быть неактуальными. Лучше использовать феномен функциональных обновлений или объединять состояние в объект и обновлять одним вызовом. Это критично для избегания багов, связанных с race conditions и обеспечением консистентности UI в React 18+.
