# Какие виды интеграции между системами вы знаете? (Синхронная, асинхронная, файловый обмен и т.д.)

Категория: Архитектура

### Краткий ответ

**Виды интеграции между системами**

- синхронная: запрос-ответ, блокирующая коммуникация  
- асинхронная: обмен сообщениями, очереди, события  
- файловый обмен: периодическая передача файлов (CSV, XML)  
- API-интеграция: REST, SOAP для прямого взаимодействия  
- ESB/шина интеграции: маршрутизация, трансформация данных  
- ETL-процессы: извлечение, трансформация, загрузка данных  
- брокеры сообщений: Kafka, RabbitMQ для масштабируемости  
- практическое применение: выбор зависит от требований по задержкам, надежности и масштабируемости

### Подробный ответ
#### Основной ответ
Интеграция между системами — ключевой аспект построения распределённых решений, позволяющий обеспечить взаимодействие и обмен данными. Существует несколько основных видов интеграции: **синхронная**, **асинхронная**, через **файловый обмен**, а также через API и шины данных. Каждая из них имеет свои особенности и применимость в зависимости от требований по задержкам, надёжности и сложности.

#### Ключевые моменты
- **Синхронная интеграция** — запрос-ответ в реальном времени (например, REST API или SOAP). Подходит, когда требуется мгновенный ответ, но есть высокая связность и зависимость от доступности обеих систем. Минус — возможные задержки и риски таймаута.
- **Асинхронная интеграция** — обмен сообщениями через брокеры (Kafka, RabbitMQ) или очереди. Позволяет разгрузить системы, обеспечить буферизацию и устойчивость к сбоям, важна для event-driven архитектур и микросервисов.
- **Файловый обмен** — системы генерируют и обмениваются файлами (CSV, XML) по FTP/SFTP. Этот метод часто используют в legacy-интеграциях и там, где нет возможности поддерживать API. Плюс в простоте, минус — высокая задержка и сложность обработки ошибок.
- **Шины данных и ESB** (Enterprise Service Bus) — централизованное посредническое ПО, которое стандартизирует и маршрутизирует сообщения между системами, поддерживает трансформации данных, протоколы и мониторинг.

#### Практический контекст
В крупных компаниях часто комбинируют эти подходы: например, синхронные REST API для быстрых клиентских запросов, асинхронные события для обработки заказов, а архивирование и интеграцию с внешними партнёрами делают через файловый обмен. Выбор зависит от SLA, объёмов данных, инфраструктурных ограничений и архитектурных решений.
