# В чём разница между Future и CompletableFuture?

**Future vs CompletableFuture**

- Future — базовый интерфейс асинхронного результата в Java Concurrency.
- Позволяет получить результат операции и отменить её.
- Не поддерживает удобное **асинхронное** объединение или цепочки задач.
- CompletableFuture расширяет Future, добавляя:
  - метод для **компоновки асинхронных действий** и их комбинирования.
  - поддержку **non-blocking** операций и событийных колбеков.
  - API для управления завершением, обработки исключений и составных последовательностей.
- CompletableFuture подходит для сложной логики **асинхронного программирования** и реактивных потоков.
- Future ограничен базовым контролем, CompletableFuture — полноценный инструмент Promise-подобного строя.
- Используются в современных приложениях для улучшения читаемости и производительности асинхронных вызовов.

# Подробный ответ
## Основной ответ
**Future** и **CompletableFuture** — это интерфейсы в Java, предназначенные для работы с результатами асинхронных вычислений, но они обладают разной функциональностью и возможностями управления.

**Future** (введён в Java 5) позволяет лишь проверять статус выполнения, отменять задачу и получать результат (методом `get()`), который блокирует вызывающий поток до завершения задачи. Однако он не поддерживает композицию асинхронных операций и не предоставляет механизмов для реактивного программирования.

**CompletableFuture** (добавлен в Java 8) — расширение Future, которое реализует интерфейс **CompletionStage**. Это полноценный асинхронный механизм, позволяющий не только получать результат, но и программно создавать цепочки из нескольких асинхронных вызовов, обрабатывать ошибки, комбинировать несколько `CompletableFuture`, а также запускать callback-функции по завершении задач без блокировок.

## Ключевые моменты
- **Функциональность:** Future — пассивный, требуется blocking для результата; CompletableFuture — активный, поддерживает цепочки, трансформации, обработку ошибок.
- **API:** CompletableFuture имеет богатый набор методов (`thenApply`, `thenCompose`, `exceptionally`, `allOf`), что упрощает построение сложных асинхронных workflow.
- **Многопоточность:** CompletableFuture может исполнять задачи в рамках thread pool и позволяет явно управлять executor’ом, Future такой гибкости не даёт.

## Практический контекст
В современных Java-проектах, особенно с React-style или event-driven архитектурами, предпочтительно использовать **CompletableFuture** для асинхронной обработки, так как он снижает блокировки и упрощает поддержку сложной логики. Future же остаётся в коде для простой синхронной проверки, но в новых разработках постепенно вытесняется.
