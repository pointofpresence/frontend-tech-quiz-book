# Отличие useMemo от useCallback.

### Категории

React hooks, Callback, Frontend, Memoization, Hooks, Optimization, React, Performance

### Краткий ответ

**Отличие useMemo от useCallback**  
React хук оптимизации  

- **useMemo**: мемоизация **значения** (результата функции)  
- **useCallback**: мемоизация **функции**  
- useMemo возвращает **кэшированный результат вызова** функции  
- useCallback возвращает **тот же экземпляр функции** между рендерами  
- useMemo полезен для **дорогих вычислений**  
- useCallback полезен для **предотвращения лишних перерендеров** дочерних компонентов, зависящих от функций  
- Оба принимают массив зависимостей и пересчитывают значение/функцию при изменении  

**Итог:** useMemo оптимизирует **значения**, useCallback — **функции** для стабильности ссылок в React.

### Подробный ответ
#### Основной ответ
**useMemo** и **useCallback** — это React хуки, служащие для оптимизации производительности через мемоизацию, но они предназначены для разных целей. **useMemo** мемоизирует результат вычисления (значение), тогда как **useCallback** мемоизирует саму функцию.

#### Ключевые моменты
- **useMemo** принимает функцию и зависимости, возвращая запомненное значение, которое пересчитывается только при изменении зависимостей. Используется для тяжелых вычислений или сложных объектов, чтобы избежать лишних перерасчетов и ререндеров.
- **useCallback** принимает функцию и зависимости, возвращая мемоизированный callback — то есть стабильный по ссылке handler, что важно при передаче функций в дочерние компоненты, зависящие от сравнения пропсов или в useEffect.
- В React 18+ эти хуки помогают уменьшить количество ненужных ререндеров, но нужно помнить, что избыточное использование может привести к усложнению кода и не всегда даёт положительный эффект.

#### Практический контекст
В проектах на React 18 часто применяют useCallback для стабилизации callback-функций, например для событий onClick в компонентах, а useMemo — для мемоизации дорогостоящих вычислений, например фильтрации больших списков или вычисления производных данных в селекторах. Оба хука сокращают количество обновлений дочерних компонентов благодаря оптимизации сравнения ссылок.
