# Чем отличаются getStaticProps, getServerSideProps и getStaticPaths?

### Категории

Static generation, SSG, Frontend, Web, SSR, React, Server side rendering

### Краткий ответ

**Чем отличаются getStaticProps, getServerSideProps и getStaticPaths?**

- Next.js методы для **получения данных** при рендеринге страниц
- **getStaticProps**:
  - выполняется во время сборки (build time)
  - для статического предрендеринга страниц
  - данные не меняются динамически
- **getServerSideProps**:
  - выполняется при каждом запросе (runtime)
  - для серверного рендеринга с динамическими данными
  - подходит для персонализации и часто обновляемых страниц
- **getStaticPaths**:
  - используется вместе с getStaticProps для динамических маршрутов
  - указывает, какие пути (routes) нужно сгенерировать на сборке
  - обеспечивает статическую генерацию страниц с динамическими параметрами
- Разница по времени выполнения: **build-time** (getStaticProps, getStaticPaths) vs **request-time** (getServerSideProps)
- Использование зависит от **динамичности данных** и **потребности в SEO**

Таким образом, выбор между ними зависит от частоты обновления данных и особенностей маршрутизации.

### Подробный ответ
#### Основной ответ
**getStaticProps**, **getServerSideProps** и **getStaticPaths** — это три специальных метода в Next.js, используемые для генерации данных и страниц с разной стратегией рендеринга. Они решают задачи, связанные с подготовкой данных на этапе сборки или на сервере во время запроса.

- **getStaticProps** — выполняется на этапе сборки (build time) и позволяет подготовить статические пропсы для страницы. Это подходит для контента, который редко меняется, что обеспечивает очень быструю отдачу страницы через CDN.
- **getServerSideProps** — вызывается при каждом запросе на сервере (runtime), генерируя страницу динамически. Используется, когда данные должны быть всегда актуальными.
- **getStaticPaths** — применяется вместе с getStaticProps для статической генерации динамических маршрутов с параметрами (например, [id].js). Он указывает, какие пути нужно сгенерировать заранее.

#### Ключевые моменты
- **getStaticProps** подходит для страниц с неизменяемым или редко обновляющимся контентом, оптимизирует SEO и скорость (SSG).
- **getServerSideProps** идеален для персонализированных данных, частого обновления или контента, зависящего от запроса (SSR).
- **getStaticPaths** используется для определения списка динамических URL при статической генерации, обязательный при использовании динамических маршрутов.
- В Next.js 12+ есть возможность использовать **Incremental Static Regeneration** (ISR) через параметр `revalidate` в getStaticProps для частичного обновления страниц после билда.
  
#### Практический контекст
В реальных проектах с CMS-контентом обычно используют getStaticProps + getStaticPaths для создания статических страниц продуктов, блогов, а getServerSideProps — для страниц с пользовательскими сессиями, личной информацией или часто меняющейся статистикой. Это позволяет добиться баланса между производительностью и актуальностью данных.
