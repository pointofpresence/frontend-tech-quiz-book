# Правила вызова хуков, и для чего они нужны?

### Категории

Lifecycle, React hooks, Frontend, Hooks, Optimization, React, State management, Browser

### Краткий ответ

**Правила вызова хуков и их назначение**

- React hooks — **функции управления состоянием и жизненным циклом** компонентов
- Правила вызова: 
  - вызывать **только на верхнем уровне функции компонента или кастомного хука**
  - вызывать **только из React-функций** (функциональных компонентов или хуков)
- Запрещено вызывать хуки внутри **условий, циклов, вложенных функций**
- Соблюдение правил обеспечивает **предсказуемый порядок вызова**
- Позволяет React правильно связывать состояние и эффекты с компонентами
- Хуки нужны для **удобного повторного использования логики** и упрощения кода
- Вместо классов — **более декларативный и чистый API**

Итог: правила обеспечивают **корректное и стабильное поведение хуков** в React, что позволяет эффективно управлять состоянием и побочными эффектами.

### Подробный ответ
#### Основной ответ  
**Правила вызова хуков** в React — это набор ограничений, которые гарантируют корректную работу хуков, таких как `useState`, `useEffect` и других. Основное правило — хуки должны вызываться всегда на одном и том же уровне вложенности и только из функциональных компонентов или пользовательских хуков. Это необходимо, чтобы React мог правильно сопоставлять вызовы хуков с внутренним состоянием при каждой отрисовке компонента. Если нарушить эти правила, например вызвать хук внутри условия или цикла — порядок вызовов изменится, и React потеряет синхронизацию, что приведет к ошибкам.

#### Ключевые моменты  
- **Только в функциональных компонентах или кастомных хуках.** Нельзя вызывать хуки из обычных функций, классов или вне тела компонента.  
- **На верхнем уровне.** Нельзя вызывать хуки внутри условий, циклов, вложенных функций — вызовы должны идти одном порядке при каждой отрисовке (например, нельзя делать `if (isReady) useEffect(...)`, надо делать `useEffect` всегда, а внутри него — условные действия).  
- **Обеспечение согласованного порядка вызовов.** React использует индекс вызова хуков, чтобы привязать их к состоянию и эффектам, и изменение этого порядка ломает логику.

#### Практический контекст  
Правила нужны, чтобы React мог выполнять оптимизации, эффективно обновлять компоненты и поддерживать корректное состояние и жизненный цикл в функциональных компонентах (React 16.8+). В реальных проектах нарушение этих правил приводит к багам, часто трудноуловимым, поэтому линтеры типа **eslint-plugin-react-hooks** помогают автоматически контролировать корректность использования хуков по стандартам React.
