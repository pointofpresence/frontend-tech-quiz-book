# Что такое PureComponent и зачем он нужен?

### Категории

UI, Component, Optimization, React, Performance

### Краткий ответ

**Что такое PureComponent и зачем он нужен?**  
- React-компонент с **оптимизацией производительности**  
- реализует **shouldComponentUpdate** с поверхностным сравнением props и state  
- предотвращает лишние **перерендеры** при неизменных данных  
- работает как **поверхностный мемоизатор**  
- удобен для классовых компонентов  
- снижает нагрузку на DOM и улучшает отзывчивость  
- полезен при **частом обновлении** и больших объемах данных

### Подробный ответ
#### Основной ответ
**PureComponent** в React — это базовый класс компонента, который реализует оптимизированный метод `shouldComponentUpdate` с поверхностным сравниванием пропсов и состояния. Это значит, что компонент перерисовывается только в случае, если новые значения пропсов или состояния отличаются от предыдущих по ссылке, что позволяет избежать ненужных рендеров и повысить производительность.

#### Ключевые моменты
- В React 16 и выше **PureComponent** — это удобный способ повысить производительность без ручной оптимизации, поскольку избегает лишних вызовов `render` при неизменных данных.
- Важно помнить, что поверхностное сравнение работает только для **примитивных типов** и **ссылок** на объекты, поэтому если пропсы — это вложенные объекты или массивы, которые мутируют, PureComponent может не сработать корректно.
- Если нужны более тонкие оптимизации, иногда используют **React.memo** (для функциональных компонентов) или собственную реализацию `shouldComponentUpdate` для полного контроля логики сравнения.

#### Практический контекст
PureComponent полезен в больших приложениях с множеством повторяющихся компонентов — например, списках или таблицах, где частые обновления могут вызвать проблемы с производительностью. В React 18 с акцентом на concurrent режим и оптимизации, PureComponent по-прежнему актуален для снижения лишних рендеров при неизменных данных.
