# Какие есть жизненные циклы в React?

### Категории

UI, Lifecycle, React lifecycle, Frontend, Algorithmic complexity, Component, Debugging, React, Event loop, TypeScript

### Краткий ответ

**Жизненные циклы в React**

- React компоненты управляют от рендера до удаления — жизненный цикл
- Классовые компоненты: 3 основных этапа — Mounting, Updating, Unmounting
- Mounting: constructor → render → componentDidMount (инициализация, загрузка данных)
- Updating: вызывается при изменении пропсов или состояния: shouldComponentUpdate → render → componentDidUpdate
- Unmounting: componentWillUnmount — очистка подписок, таймеров, освобождение ресурсов
- Функциональные компоненты используют хуки: useEffect, useLayoutEffect для аналогов жизненного цикла
- Современный React акцентирует хуки для управления побочными эффектами и состоянием
- Жизненный цикл важен для контроля рендера и оптимизации производительности

Это понимание показывает опыт с обоими подходами и глубокое знание React жизненных этапов.

### Подробный ответ
#### Основной ответ
В **React** жизненный цикл компонента — это набор методов и этапов, через которые проходит компонент от создания до удаления из DOM. В современных React (начиная с версии 16.3 и выше) жизненный цикл значительно изменился с появлением **функциональных компонентов с хуками** и обновлённого API классовых компонентов. Классовые компоненты имеют 3 основных фазы: **монтаж (mounting)**, **обновление (updating)** и **размонтирование (unmounting)**.

#### Ключевые моменты
- **Монтаж**: происходит инициализация компонента и вставка в DOM. В классовых компонентах основные методы — `constructor()`, `static getDerivedStateFromProps()`, `render()` и `componentDidMount()`. В функциональных компонентах аналог — хук `useEffect(() => {}, [])`.
- **Обновление**: происходит при изменении пропсов или состояния. Методы классов: `static getDerivedStateFromProps()`, `shouldComponentUpdate()`, `render()`, `getSnapshotBeforeUpdate()`, `componentDidUpdate()`. В функциональных компонентах — хук `useEffect()`, который может срабатывать при изменении зависимостей.
- **Размонтирование**: удаление компонента из DOM. В классах — `componentWillUnmount()`, в функциональных — `useEffect` с функцией очистки (return cleanup).

#### Практический контекст
Понимание жизненного цикла важно для организации побочных эффектов (запросы к API, подписки, таймеры) и оптимизации производительности (предотвращение ненужных рендеров через `shouldComponentUpdate` или мемоизацию хуков). В React 18+ для функциональных компонентов появились дополнительные особенности, например асинхронный рендеринг (concurrent mode), который влияет на поведение эффектов.

Таким образом, жизненный цикл — это фундаментальное понятие для контроля логики компонентов, особенно при переходе с классов на функциональные компоненты с хуками.
