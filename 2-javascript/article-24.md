# Есть два параллельных запроса: один возвращает A, другой B. Как получить их результаты и объединить A и B, если нельзя использовать Promise.all?

### Категории

JavaScript, Async, API, Concurrency, Promise

### Краткий ответ

**Объединение результатов параллельных запросов без Promise.all**

- Асинхронность в JS — с помощью **Promise** и колбеков  
- Выполнить оба запроса **параллельно** — запустить без await  
- Каждый запрос возвращает Promise, подписаться на **.then()**  
- Внутри каждого then сохранять результат (например, во внешние переменные)  
- После завершения второго запроса — вручную объединить A и B  
- Управлять состоянием с помощью счётчика или флага завершения  
- Можно использовать **async/await с последовательным контролем**, но параллельно — через два промиса с отдельными обработчиками  
- Альтернативно — построить кастомный механизм синхронизации (например, через события)

**Итог:** Без Promise.all контролируем завершение каждого Promise вручную, собираем результаты и потом объединяем.

### Подробный ответ
#### Основной ответ
Если нельзя использовать **Promise.all**, то для параллельного выполнения двух асинхронных запросов и объединения их результатов можно использовать отдельные промисы с независимым вызовом и последующей обработкой в промисах или async/await. Главное — запустить оба запроса параллельно, а потом дождаться завершения каждого индивидуально. Например, запускать оба промиса без ожидания, а затем ждать каждого с помощью `await` или `then`, сохраняя результат во внешних переменных, где после будет объединение.

#### Ключевые моменты
- Можно запустить оба запроса параллельно, вызвав их до `await` (например, `const promiseA = fetchA(); const promiseB = fetchB();`), чтобы оба запроса выполнялись одновременно.
- После запуска ждать каждый по отдельности: `const a = await promiseA; const b = await promiseB;`. Это позволит сохранить параллельность без использования `Promise.all`.
- Такой подход полезен, если нужно более сложное управление запросами, например, обработка ошибок каждого запроса по-разному без прерывания всего потока.
- Важно помнить, что `await` по отдельности не делает запросы последовательно, если промисы уже запущены.

#### Практический контекст
В реальных приложениях, например, с **fetch API** в React 18+, часто мы запускаем несколько запросов параллельно, чтобы минимизировать задержку (~50-100ms). Такой подход обеспечивает низкую latency по сравнению с последовательным вызовом `await fetchA(); await fetchB();`. Отказ от `Promise.all` может быть вызван требованием более гранулированной обработки ошибок или логики объединения данных после каждого успешного запроса.
