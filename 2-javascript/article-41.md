# Чем отличается strict mode от нестрогого режима

### Категории

Strict mode, JavaScript, Memory optimization, Live coding, Optimization, Coding challenge

### Краткий ответ

**Отличия strict mode от нестрогого режима в JavaScript**

- **Строгий режим (strict mode)** — специальный режим выполнения JS с повышенными требованиями к безопасности и качеству кода
- Запрещает неявные ошибки: например, присваивание переменной без объявления вызывает ошибку (в обычном режиме — создаёт глобальную переменную)
- Блокирует некоторые «опасные» действия, например, удаление необъявленных переменных или дублирование параметров функции
- Отключает автоматическое преобразование this в глобальный объект, вместо этого this становится undefined в функциях (исключает ошибки с контекстом)
- Запрещает использование устаревших или потенциально проблемных функций (например, с eval или arguments)
- Улучшает оптимизацию движка JS за счёт более строгих правил разбора и исполнения
- Использование strict mode повышает **безопасность кода**, облегчает поиск ошибок и улучшает поддержку стандарта ES5 и выше  
- В нестрогом режиме допускается больше вольностей, что может приводить к скрытым багам и менее предсказуемому поведению

Итог: **strict mode** — это механизм повышения надёжности и предсказуемости кода за счёт ограничения слабых мест обычного JavaScript.

### Подробный ответ
#### Основной ответ
**Strict mode** в JavaScript — это специальный режим, который вводит более строгие правила синтаксиса и поведения языка по сравнению с нестрогим (обычным) режимом. Его цель — помочь разработчикам писать более безопасный и предсказуемый код, избегая распространённых ошибок, а также улучшая оптимизацию движков JS.

#### Ключевые моменты
- **Запрет ошибок и неочевидных конструкций**: в strict mode запрещены некоторые небезопасные действия, например, использование не объявленных переменных (будет выброшена ошибка), дублирование имен параметров функции, присваивание значений неизменяемым свойствам.
- **Поведенческие изменения**: контекст `this` в функциях без вызова через объект становится `undefined` вместо глобального объекта, предотвращается неявное создание глобальных переменных.
- **Оптимизация**: strict mode позволяет JS-движкам выполнять более агрессивные оптимизации за счёт исключения некоторых сценариев с «неожиданным» поведением.
- **Совместимость**: он не позволяет использовать зарезервированные слова для будущих версий языка (например, `implements`, `interface`), что снижает риск конфликтов.

#### Практический контекст
В современных проектах strict mode включается по умолчанию в ES6 модулях (import/export). В старом коде его добавляют директивой `'use strict';` в начале скрипта или функции. Это помогает выявлять баги на ранних этапах, улучшает поддержку и уменьшает вероятность багов, связанных с глобальным состоянием и ошибками объявления переменных.

Таким образом, strict mode — это инструмент повышения надёжности и качества JavaScript-кода, особенно полезный в крупных приложениях с командной разработкой.
