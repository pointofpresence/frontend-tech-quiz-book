# Что вернёт filter, если ни один элемент не подходит под условие?

### Категории

Array, JavaScript, Filter, Functional programming, Live coding, Algorithm

### Краткий ответ

**Что вернёт filter, если ни один элемент не подходит под условие?**

- filter — метод массива для **фильтрации** по условию  
- возвращает **новый массив**  
- если элементы не подходят — возвращается **пустой массив**  
- исходный массив **не изменяется**  
- полезно для вывода результатов подходящих фильтру  
- всегда можно проверить длину результата для определения наличия подходящих элементов

### Подробный ответ
#### Основной ответ
Метод **filter** в JavaScript возвращает **новый массив**, содержащий все элементы исходного массива, которые удовлетворяют заданному условию (предикату). Если ни один элемент не подходит под это условие, то **возвращается пустой массив** — `[]`, а не `null` или `undefined`.

#### Ключевые моменты
- **Новые объекты**: `filter` не изменяет исходный массив, всегда создаёт новый, даже если подходящих элементов нет.
- **Пустой массив** — это типичный способ обозначить отсутствие совпадений, что удобно для дальнейшей цепочки вызовов и не вызывает ошибок.
- **Работает с любым исходным массивом** — например, в `[].filter(...)` тоже вернётся пустой массив, что логично с точки зрения согласованности API.

#### Практический контекст
В реальных проектах это поведение удобно для безопасной фильтрации данных — например, искать активных пользователей с условием `user.isActive`, если таких нет — результат просто пустой массив, и можно без дополнительной проверки инициализировать UI или провести последующую обработку без исключений.
