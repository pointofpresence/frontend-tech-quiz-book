# Почему стрелочные функции не работают с this в некоторых контекстах?

### Категории

Functions, JavaScript, Frontend, Scope, Live coding, Event loop, Arrow functions, React, This, TypeScript, Context, Code review

### Краткий ответ

**Почему стрелочные функции не работают с this в некоторых контекстах?**

- Стрелочные функции не имеют собственного контекста `this`  
- `this` берётся из внешней (лексической) области видимости  
- Не подходят для методов объектов, где нужен динамический `this`  
- Не могут использоваться как конструкторы (`new` вызов)  
- Проблемы при вызове с разным контекстом через `call/apply`  
- Удобны для лямбд и вложенных функций без изменения `this`  
- Важно выбирать по задаче: если нужен привязанный `this` — стрелочная, иначе — обычная функция

Итог: стрелочные функции фиксируют `this` при объявлении, что ограничивает их гибкость в динамичном контексте.

### Подробный ответ
#### Основной ответ
**Стрелочные функции** в JavaScript не имеют собственного контекста `this`. Вместо этого они лексически наследуют `this` из окружающей функции во время определения. Это значит, что значение `this` внутри стрелочной функции зафиксировано и не изменится в зависимости от того, как функция вызывается. В некоторых контекстах это вызывает проблемы — например, когда требуется динамический `this`, связанный с объектом-владельцем функции.

#### Ключевые моменты
- В обычных функциях значение `this` определяется во время вызова и зависит от контекста вызова (объект, оператор `new`, `call/apply`).
- Стрелочные функции берут `this` из внешней области видимости в момент определения, что удобно для сохранения контекста внутри callback-ов, но мешает, если нужен динамический объект.
- Из-за отсутствия собственного `this` стрелочные функции нельзя использовать в качестве методов, если внутри метода предполагается манипуляция с текущим объектом (`this` будет указывать не на объект, а на внешний контекст).

#### Практический контекст
Типичная ошибка — объявить метод объекта стрелочной функцией и ожидать, что `this` внутри укажет на объект. Например, в React до Hooks часто использовали стрелочные функции для методов, чтобы избежать привязки `this`. Но в классах необходимо либо использовать обычные функции с `.bind()`, либо классовые поля со стрелочными функциями, чтобы лексически захватить `this`. В Node.js или при обработке событий важно понимать, что стрелочные функции не подходят, если нужен собственный контекст объекта.
