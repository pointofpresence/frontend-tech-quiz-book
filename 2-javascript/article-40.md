# Для чего используется директива use strict в JavaScript и как она влияет на код?

### Категории

Strict mode, "use strict", JavaScript, Frontend, Memory optimization, Live coding, Optimization, Algorithms

### Краткий ответ

**Для чего используется директива use strict в JavaScript и как она влияет на код?**

- механизм активации **строгого режима** в JS  
- запрещает **неявные** ошибки (например, использование необъявленных переменных)  
- блокирует дублирование имен параметров функций  
- меняет поведение `this` в функциях (в strict — `undefined` вместо глобального объекта)  
- исключает **тихие ошибки** и облегчает отладку  
- улучшает производительность за счёт оптимизаций движка  
- повышает безопасность и качество кода, снижает баги  
- широко используется в современных проектах и модулях ES6+

### Подробный ответ
#### Основной ответ
**Директива `use strict`** в JavaScript активирует режим строгой типизации, который изменяет поведение движка в сторону более строгой обработки ошибок и ограничений. Она служит для повышения качества и безопасности кода, выявляя потенциальные ошибки еще на этапе выполнения, предотвращая распространённые "лазейки" и нестандартные поведения.

#### Ключевые моменты
- В режиме строгой типизации запрещены неявные глобальные переменные, что помогает избежать случайных ошибок из-за опечаток в именах переменных.
- Запрещено удалять переменные, функции и аргументы (`delete`), что повышает предсказуемость кода.
- Для некоторых операций и синтаксических конструкций вводятся ограничения, например, нельзя использовать дублирующиеся имена параметров функции, что снижает неоднозначность.
- `use strict` предотвращает использование устаревших конструкций, таких как `with`.
- В строгом режиме `this` в функциях, вызванных без контекста, не будет ссылаться на глобальный объект, что снижает риск неожиданных багов.
- Директива может использоваться как для всего скрипта, так и для отдельных функций (локально), что даёт гибкость при миграции старого кода.

#### Практический контекст
В современных проектах (например, на базе ES6+) режим строгой типизации включается по умолчанию во многих модулях. В legacy-коде `use strict` добавляют, чтобы постепенно ужесточить проверку, снизить количество subtle bugs и улучшить качество кода без полной переработки архитектуры. Также его рекомендуют использовать в Node.js и браузерных приложениях для повышения безопасности и контроля.
