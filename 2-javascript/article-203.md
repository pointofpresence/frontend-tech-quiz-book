# Что такое requestAnimationFrame?

### Категории

Animation, JavaScript, Async, Frontend, Containers, API, Optimization, Performance, Browser

### Краткий ответ

**Что такое requestAnimationFrame?**

- API браузера для **эффективной анимации**
- синхронизирует вызов с **перерисовкой экрана**
- обеспечивает **60 FPS** или частоту обновления дисплея
- заменяет **setTimeout/setInterval** для плавности
- минимизирует **затраты CPU и энергопотребление**
- коллбек вызывается **до следующего repaint**
- подходит для **гейминга**, UI анимаций, плавных переходов

### Подробный ответ
#### Основной ответ
**requestAnimationFrame** — это нативный API браузера для оптимизации анимаций и обновления графики в веб-приложениях. Он позволяет браузеру синхронизировать выполнение функции с перерисовкой экрана, обеспечивая плавный и эффективный рендеринг с частотой обновления, близкой к 60 кадрам в секунду.

#### Ключевые моменты
- **Синхронизация с экраном**: вместо вызова таймеров с фиксированным интервалом (setTimeout/setInterval), requestAnimationFrame вызывает callback перед следующим repaint браузера, что минимизирует пропуски кадров и обеспечивает более ровную анимацию.
- **Экономия ресурсов**: если вкладка не видна (неактивна или свернута), браузер приостанавливает вызовы requestAnimationFrame, снижая нагрузку на CPU и GPU и экономя энергию — в отличие от таймеров, которые продолжают работать.
- **Использование**: callback принимает timestamp, позволяя рассчитывать изменения в анимации с точностью времени кадра, что полезно для контроля скорости анимации.

#### Практический контекст
В React 18+ или при работе с canvas 2D/WebGL requestAnimationFrame — стандартный способ организовать smooth анимацию, например, для игр, визуализаций данных или эффектов интерфейса. Он значительно лучше подходит, чем setTimeout для UI-анимаций с высокой частотой кадров, обеспечивая **латентность около 16ms** при 60fps и плавность, которую ценят пользователи.
