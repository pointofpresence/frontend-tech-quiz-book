# Как выполняется экранирование специальных символов в JSON?

### Категории

Parsing, JavaScript, Data format, JSON, Security

### Краткий ответ

**Как выполняется экранирование специальных символов в JSON?**

- JSON — текстовый формат обмена данными, основан на JavaScript
- Специальные символы **экранируются обратным слэшем** (`\`)
- Экранируются символы: `"` (кавычка), `\`, `/` (опционально), `\b`, `\f`, `\n`, `\r`, `\t`
- Юникод символы вне ASCII кодируются как `\uXXXX`
- Экранирование сохраняет корректность парсинга JSON
- Обеспечивает правильную передачу строк с управляющими символами
- Используется стандартный синтаксис JSON для сериализации и десериализации

Так экранирование гарантирует, что специальные символы в строках не нарушат структуру JSON и будут правильно интерпретированы на стороне читающего.

### Подробный ответ
#### Основной ответ
В **JSON** экранирование специальных символов выполняется с помощью обратного слэша (`\`), чтобы сохранить корректный формат и избежать ошибок парсинга. Специальные символы, такие как кавычки, обратный слэш и управляющие символы (например, перенос строки или табуляция), должны быть экранированы для корректной сериализации и передачи данных.

#### Ключевые моменты
- В **JSON-строках** экранируются следующие символы:  
  - Кавычка (`"`), чтобы не прерывать строку  
  - Обратный слэш (`\`), чтобы избежать неоднозначности  
  - Символы управления: перенос строки (`\n`), возврат каретки (`\r`), табуляция (`\t`), backspace (`\b`), form feed (`\f`)  
- Unicode-символы вне допустимого диапазона или управляющие символы могут быть представлены в виде `\uXXXX` (шестнадцатеричный код), что обеспечивает универсальность и безопасность передачи данных  
- Экранирование происходит на этапе сериализации — в современных языках (например, `JSON.stringify` в JavaScript или аналогичные функции в Python, Java) встроена поддержка корректного экранирования, что минимизирует ошибки

#### Практический контекст
При работе с API и передачей JSON данные, содержащие спецсимволы (например, пользовательский ввод с кавычками или спецсимволами), обязательно экранируются автоматически. Это предотвращает ошибки на стороне клиента или сервера, обеспечивает правильный парсинг и защиту от инъекций через строки. Например, в React 18 или Node.js встроенные JSON парсеры всегда учитывают эти нюансы при сериализации и десериализации.
