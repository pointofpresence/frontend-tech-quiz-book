# Как работают неизменяемые и изменяемые объекты?

### Категории

Mutable, Backend, JavaScript, Immutability, Data structures

### Краткий ответ

**Как работают неизменяемые и изменяемые объекты?**

- Объекты с **изменяемым состоянием** (mutable) допускают изменение данных после создания  
- Изменяемые объекты хранят ссылки на изменяемые поля внутри  
- Изменение неизменяемого объекта невозможно после создания — любой «изменённый» объект создаёт копию  
- Неизменяемость повышает **безопасность**, упрощает **конкурентность** и уменьшает ошибки  
- Изменяемые объекты более эффективны по памяти и скорости при изменениях без копирования  
- Практически: строки в Java и Python неизменяемы, списки и словари изменяемы  
- Используют неизменяемость для легкой отладки, кэширования, функционального программирования  

Итог: **mutable** — изменяемые данные, **immutable** — данные с защитой от изменений; выбор зависит от требований к безопасности и производительности.

### Подробный ответ
#### Основной ответ
**Неизменяемые (immutable) объекты** — это объекты, состояние которых нельзя изменить после создания. При попытке изменить такой объект создаётся новая копия с изменениями, а исходный остаётся прежним. В противоположность этому, **изменяемые (mutable) объекты** поддерживают прямое изменение своего состояния после создания. Такое поведение влияет на управление памятью, безопасность данных и параллелизм.

#### Ключевые моменты
- **Безопасность и предсказуемость:** неизменяемые объекты позволяют избежать непреднамеренных побочных эффектов и race condition в многопоточных приложениях. Например, строки в Java и Python — неизменяемы.
- **Управление памятью:** при изменяемых объектах изменения происходят in-place, что экономит память, но требует аккуратности при совместном использовании. Неизменяемые объекты могут создавать нагрузку на сборщик мусора из-за копирования, но упрощают кеширование и оптимизации на уровне компилятора.
- **Производительность и семантика:** изменяемые объекты быстрее для частых изменений, неизменяемые — удобнее для функционального программирования и сохранения истории состояний (например, в Redux).

#### Практический контекст
В реальных проектах неизменяемые объекты широко используются в функциональных языках (Scala, Haskell), а также во frontend-фреймворках (React 18+), где изменяемость минимизируют для оптимизации ререндеров. Изменяемые объекты характерны для большинства ООП-языков (Java, C#), где удобство и производительность важнее консистентности. Понимание этих различий помогает выбирать правильную модель данных под конкретную задачу.
