# Как отменить fetch-запрос?

### Категории

Project management, JavaScript, Async, Closures, API, Networking, HTTP, Debugging, Performance, Cancellation, Promises, Error handling

### Краткий ответ

**Как отменить fetch-запрос?**

- Работа с **асинхронными** HTTP-запросами (fetch API)  
- Используется **AbortController** для отмены  
- Создаётся контроллер: `const controller = new AbortController()`  
- Передается сигнал: `fetch(url, { signal: controller.signal })`  
- Вызов `controller.abort()` прерывает запрос  
- При отмене происходит ошибка с именем **AbortError**  
- Позволяет управлять временем жизни запроса и экономить ресурсы  

Итог: отмена fetch — через **AbortController** и сигнал abort для управления запросом.

### Подробный ответ
#### Основной ответ
Отменить **fetch**-запрос в браузере можно с помощью **AbortController**, который позволяет прервать асинхронную операцию путём вызова метода `abort()`. В самом fetch API отсутствует встроенный метод отмены, поэтому именно AbortController стал стандартным решением начиная с современных версий браузеров (например, Chrome 66+, Firefox 57+).

#### Ключевые моменты
- Для использования необходимо создать экземпляр `AbortController` и передать его `signal` в опции запроса `fetch(url, { signal: controller.signal })`.
- Чтобы отменить запрос, вызывается метод `controller.abort()`, что приводит к отклонению промиса fetch с ошибкой `DOMException` типа `"AbortError"`.
- Этот механизм удобен для отмены запросов при размонтировании компонентов (например, в React 18+), для сокращения трафика и улучшения UX, особенно при вводе с авто-подсказками или динамическом поиске.

#### Практический контекст
В проектах на React часто делают отмену fetch-запросов в хуке `useEffect` через AbortController, чтобы избежать утечки памяти и попыток обновления состояния после размонтирования компонента. В Next.js и других фреймворках это помогает предотвращать конкуренцию запросов при быстрой смене маршрутов и экономить сетевой трафик.
