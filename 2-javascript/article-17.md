# Какие структуры данных существуют в JavaScript?

### Категории

JavaScript, Frontend, Data structures

### Краткий ответ

**Структуры данных в JavaScript**

- базовые примитивы: **string, number, boolean**
- коллекции: **Array** (упорядоченный список), **Object** (ключ-значение)
- новые структуры ES6:  
  - **Map** — упорядоченный ключ-значение, ключи любых типов  
  - **Set** — уникальные значения, упорядоченные  
- специализированные: **WeakMap, WeakSet** — для слабых ссылок, GC-friendly  
- важна мутабельность: массивы и объекты — изменяемые, примитивы — нет  
- для управляемой структуры: используют классы и иммутабельные библиотеки (Immutable.js)  
- применение: выбор структуры влияет на **поиск, вставку, удаление** (эффективность) в приложении

### Подробный ответ
#### Основной ответ
В **JavaScript** есть несколько основных структур данных, которые позволяют эффективно организовывать и хранить данные. Самыми распространёнными являются **примитивы** (string, number, boolean и т.д.) и **объекты**, но в контексте структур данных чаще говорят о более сложных типах: **массивы, объекты, Set, Map, WeakSet и WeakMap**. Каждая из них обеспечивает разные возможности для хранения и доступа к данным, адаптируясь под разные задачи.

#### Ключевые моменты
- **Массивы (Array)** — упорядоченный список элементов, индексируемый целыми числами, обладает методами для добавления, удаления и итерации. В современных движках реализованы оптимизации для быстрого доступа и сортировки.
- **Объекты (Object)** — хеш-таблица ключей-значений, где ключи — строки или символы. Используются для ассоциативного хранения. С выходом ES6 появились **Map и Set** для более универсальной работы с ключами любого типа.
- **Map** — структура данных для ключей любого типа с сохранением порядка вставки, отличается лучше предсказуемой производительностью по сравнению с обычным объектом.
- **Set** — коллекция уникальных значений, полезна для быстрого поиска и предотвращения дубликатов.
- **WeakMap и WeakSet** — похожи на Map и Set, но ключи (в WeakMap) или значения (в WeakSet) хранятся слабыми ссылками, что важно для управления памятью и предотвращения утечек.

#### Практический контекст
В React 18+ часто используют **Map** для хранения кэшированных данных с произвольными ключами, а **Set** — для управления уникальными событиями или идентификаторами. **WeakMap** полезен для приватных данных в классах и хранении контекста без риска утечки памяти. Каждый тип имеет свои trade-offs в производительности и применении, что важно учитывать при оптимизации больших приложений.
