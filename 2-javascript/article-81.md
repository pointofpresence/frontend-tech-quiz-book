# Что такое сборщик мусора?

### Категории

High load, DevOps, Backend, JavaScript, Transactions, Networking, Kubernetes, Concurrency, Distributed systems, Garbage collection

### Краткий ответ

**Что такое сборщик мусора?**  
- компонент среды исполнения (например, JVM, CLR, JS-движок)  
- автоматически освобождает неиспользуемую память  
- выявляет объекты без живых ссылок  
- предотвращает утечки памяти и ошибки с указателями  
- работает по алгоритмам: mark-and-sweep, generational, reference counting  
- снижает нагрузку на программиста, упрощает управление памятью  
- критичен для стабильности и производительности ПО

### Подробный ответ
#### Основной ответ
**Сборщик мусора (Garbage Collector, GC)** — это системный компонент в средах выполнения программ (например, JVM, CLR, V8 в Node.js), который автоматически управляет памятью, освобождая неиспользуемые объекты, чтобы предотвратить утечки памяти и повысить стабильность приложения. Это позволяет разработчику не заботиться о ручном управлении памятью, как в языках типа C/C++.

#### Ключевые моменты
- **Автоматическое управление памятью**: сборщик отслеживает объекты, на которые нет ссылок, и освобождает занимаемое ими место.
- **Основные алгоритмы**: mark-and-sweep (отметка и очистка), reference counting, generational GC (разделение объектов по возрасту для оптимизации).
- **Trade-offs**: GC снижает риск ошибок (например, use-after-free), но может вызвать паузы в работе приложения — в современных реализациях с минимальной задержкой, например, G1 в JVM или V8's incremental GC, паузы минимизируются.

#### Практический контекст
В Java 17 и .NET 6 сборщики мусора достаточно оптимизированы для приложений с высоким нагрузочным профилем, обеспечивая **99.9% uptime** и задержки в миллисекунды. В веб-приложениях на Node.js V8 обеспечивает инкрементальный GC, который минимизирует лаги в UI и серверной логике.
