# Что такое Event Loop и Event Emitter?

### Категории

Backend, JavaScript, Async, Runtime, Non-blocking, Event loop, Event-driven, Event emitter

### Краткий ответ

**Что такое Event Loop и Event Emitter?**

- Event Loop: механизм **асинхронного** управления циклами в JS  
- Обрабатывает **очередь событий**, выполняет колбэки после основного кода  
- Поддерживает неблокирующее выполнение и **однопоточный** JS  
- Event Emitter: объект для **генерации событий** и подписки на них  
- Позволяет реализовать **паттерн pub/sub** (издатель-подписчик)  
- Используется в Node.js для коммуникации между компонентами  
- В совокупности обеспечивают **асинхронность** и **событийно-ориентированное** программирование

### Подробный ответ
#### Основной ответ
**Event Loop** — это механизм в JavaScript и других однопоточных средах исполнения, который позволяет эффективно обрабатывать асинхронные операции, не блокируя основной поток. Он отслеживает очередь событий (events) и задач, обрабатывая их по мере готовности, что обеспечивает неблокирующее выполнение кода.  
**Event Emitter** — это паттерн (часто реализованный в Node.js через класс EventEmitter), позволяющий объектам генерировать и обрабатывать пользовательские события. Он предоставляет API для подписки (listeners) на события и их последующей эмиссии (emit), обеспечивая коммуникацию между разными частями приложения.

#### Ключевые моменты
- **Event Loop**: в Node.js 14+ состоит из нескольких фаз (timers, I/O callbacks, idle, poll, check, close callbacks), каждая отвечает за свою очередь задач. Это позволяет реализовать non-blocking I/O с высокой производительностью (латентность ~50мс или меньше при нормальной нагрузке).
- **Event Emitter**: реализует подписку на события через методы `on`, `once` и генерацию событий через `emit`. Часто используют для decoupling компонентов и построения обработчиков событий (например, обработка ошибок, получение данных).
- Взаимодействие: Event Emitter работает на основе callback-функций, которые ставятся в очередь событий, а Event Loop обеспечивает их последовательное выполнение без блокировки.

#### Практический контекст
В реальных проектах Event Loop — это фундамент для работы с асинхронным кодом на Node.js, например, ожидание ответа от базы данных без блокировки сервера. Event Emitter широко используется для построения modular event-driven архитектур, например, для обработки потоков данных или реагирования на системные события (например, завершение загрузки файла).
