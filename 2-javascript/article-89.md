# Что такое async?

### Категории

Backend, JavaScript, Async, String manipulation, Live coding

### Краткий ответ

**Что такое async?**

- концепция **асинхронного** программирования  
- позволяет выполнять операции без блокировки потока  
- возвращает **Promise** в JS (ожидаемый результат)  
- обычно используется с **await** для удобного синтаксиса  
- улучшает **производительность** при I/O и сетевых вызовах  
- упрощает обработку **отложенных** или длительных задач  
- широко применяется в веб-разработке и работа с API

### Подробный ответ
#### Основной ответ
**async** — это концепция асинхронного программирования, позволяющая выполнять операции без блокировки основного потока выполнения. В языках программирования, таких как JavaScript (начиная с ES2017), ключевое слово `async` используется для объявления функций, которые возвращают **промис** и позволяют применять оператор `await` внутри тела функции для удобного ожидания результатов асинхронных операций.

#### Ключевые моменты
- **Неблокирующее выполнение**: `async` функции позволяют выполнять длительные операции (например, сетевые запросы, чтение файлов) без заморозки пользовательского интерфейса или основного потока сервера.
- **Синтаксический сахар для промисов**: использование `async/await` значительно упрощает написание и чтение асинхронного кода по сравнению с классическими колбэками или цепочками `.then()`.
- **Обработка ошибок через try/catch**: асинхронный код становится более предсказуемым, так как ошибки обрабатываются через привычные блоки, в отличие от методов `.catch()`.

#### Практический контекст
В React 18 и Node.js 14+ `async/await` уже стандартный способ работы с API, базами данных и другими асинхронными источниками. Например, в Express.js асинхронные обработчики позволяют писать чистый и понятный код для запросов к БД, а в frontend-части получение данных через `fetch` и `await` повышает читаемость и упрощает отладку при latency ~50-200ms.
