# Как перехватить JS ошибки на странице?

### Категории

JavaScript, Async, Frontend, Closures, Problem solving, Debugging, Timing, Error handling

### Краткий ответ

**Как перехватить JS ошибки на странице?**

- **Глобальный обработчик ошибок** через `window.onerror` для синхронных ошибок
- `window.addEventListener('error', ...)` для детального перехвата ошибок загрузки ресурсов и JS
- `window.addEventListener('unhandledrejection', ...)` ловит ошибки из Promise без `catch`
- Использование `try...catch` для локального контроля ошибок в блоках кода и функциях
- Интеграция с системами логирования (Sentry, LogRocket) для сбора и анализа ошибок
- Асинхронные ошибки: Promise по умолчанию не выбрасывают, нужны обработчики `unhandledrejection`
- Для React/Angular — свои Error Boundaries или механизмы обработки ошибок компонентов

Итог: комбинируют глобальные ловушки, локальные блоки `try...catch` и мониторинг для полного контроля и аналитики сбоев.

### Подробный ответ
#### Основной ответ
Для перехвата JavaScript ошибок на веб-странице обычно используют глобальные обработчики ошибок, которые позволяют отловить как синхронные исключения, так и асинхронные ошибки (например, из Promise). Основные способы — это использование `window.onerror`, `window.addEventListener('error')` и `window.addEventListener('unhandledrejection')`.

#### Ключевые моменты
- **window.onerror** — классический способ для обработки необработанных исключений, возвращает информацию о сообщении ошибки, URL, строке и столбце. Подходит для синхронных ошибок, но не поймает Promise rejection.
- **window.addEventListener('error')** — полезен для отлова не только JS-ошибок, но и ошибок загрузки ресурсов (картинок, скриптов). Можно отфильтровать события по типу.
- **window.addEventListener('unhandledrejection')** — ловит необработанные Promise rejection, которые не перехватываются `window.onerror`.
  
Для комплексного мониторинга стоит использовать все три, что особенно актуально в современных SPA с обширным использованием асинхронного кода.

#### Практический контекст
В продакшен-приложениях часто применяют сервисы типа Sentry, которые используют эти механизмы для сбора и агрегирования ошибок с подробным стеком вызовов и метаданными. Также можно внедрить локальный логгер, отправляющий ошибки на сервер для последующего анализа или оповещений. В React 18, например, есть собственные Error Boundaries для перехвата ошибок в компонентах UI, что дополняет глобальные методы.
