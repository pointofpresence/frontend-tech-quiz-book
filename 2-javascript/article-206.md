# Какие есть фазы события?

### Категории

Coverage, JavaScript, Async, Frontend, String manipulation, Event phases, Events, Practical task, GroupBy, Design system, Event, Self development, Testing, Algorithm

### Краткий ответ

**Фазы события (Event Phases)**  
- модель DOM-событий в браузерах  
- три фазы: **capturing**, **target**, **bubbling**  
- capturing: событие идёт сверху вниз по иерархии DOM  
- target: событие достигло целевой элемент, обработка на нем  
- bubbling: событие всплывает снизу вверх к корню DOM  
- обработчики можно назначить для каждой фазы отдельно  
- контролирует порядок срабатывания слушателей  
- важно для правильной обработки событий и делегирования  

Фазы события — ключевая часть DOM Event Model для точного управления поведением интерфейса.

### Подробный ответ
#### Основной ответ
В контексте веб-разработки и моделирования событий в браузере события проходят через три основные фазы: **фаза захвата (capturing phase)**, **целевая фаза (target phase)** и **фаза всплытия (bubbling phase)**. Эта модель событий называется **Event Propagation** и применяется в DOM API.

#### Ключевые моменты
- **Фаза захвата (Capturing phase)** – событие движется сверху вниз от корневого элемента документа к целевому элементу, но обработчики на этом этапе вызываются редко, потому что по умолчанию события обрабатываются на фазе всплытия.
- **Целевая фаза (Target phase)** – событие достигло своего целевого элемента, и вызываются обработчики, напрямую привязанные к этому элементу.
- **Фаза всплытия (Bubbling phase)** – после обработки на целевом элементе событие поднимается обратно по DOM-дереву вверх к корню, вызывая обработчики на родительских элементах. Именно эта фаза чаще всего используется для делегирования событий.
- В современных браузерах по умолчанию обработчики регистрируются на фазе всплытия, но с помощью параметра `{ capture: true }` можно подписаться на фазу захвата.
- Важно понимать, что не все события поддерживают фазу захвата; например, события `focus` и `blur` не всплывают.

#### Практический контекст
При реализации сложных интерфейсов часто используют **event delegation** на фазе всплытия, чтобы минимизировать количество обработчиков. В React 18+ используется синтетическая система событий, имитирующая такие фазы для упрощения разработки. Управление фазами событий помогает оптимизировать производительность и контролировать поведение интерфейса, особенно в больших SPA.
