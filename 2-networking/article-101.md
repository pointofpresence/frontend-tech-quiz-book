# Чем отличаются GET и POST-запросы?

### Категории

Post, Async, Frontend, Get, API, Networking, Events, HTTP, Event loop, Live coding, Network, Request, Web, Security, Promises, Requests, Code review

### Краткий ответ

**Чем отличаются GET и POST-запросы?**  
- HTTP методы для передачи данных клиент-сервер  
- GET: параметры в URL, ограничение длины, **идемпотентный**  
- POST: данные в теле запроса, нет лимита по размеру  
- GET не меняет состояние сервера, POST — создаёт/модифицирует ресурсы  
- GET кэшируется, POST — обычно нет  
- POST безопаснее для передачи чувствительных данных  
- Используются соответственно для **чтения** (GET) и **изменения** (POST) данных

### Подробный ответ
#### Основной ответ
**GET** и **POST** — два основных метода HTTP-запросов, используемых для взаимодействия клиента с сервером. **GET** запрашивает данные с сервера и передаёт параметры через URL, в то время как **POST** отправляет данные в теле запроса, обычно для создания или обновления ресурсов.

#### Ключевые моменты
- **Идемпотентность и кэширование**: GET-запросы являются идемпотентными и кэшируются браузерами и прокси, что повышает производительность при повторных вызовах. POST-запросы не кэшируются и не считаются идемпотентными, так как изменяют состояние сервера (например, создание записи).
- **Передача данных**: В GET данные передаются в URL (в query string), что ограничивает объём (обычно около 2048 символов) и видимы в адресной строке. POST позволяет отправлять большие объёмы данных в теле запроса, включая двоичные данные (например, файлы).
- **Безопасность и применение**: GET подходит для запросов чтения (например, получение страницы, списка). POST чаще используется при отправке форм с конфиденциальной информацией, так как данные не попадают в URL и не сохраняются в истории браузера, хотя сам по себе HTTP не шифрует содержимое — для безопасности требуется HTTPS.

#### Практический контекст
В современной веб-разработке, к примеру в REST API (PostgreSQL 14+, Node.js), GET применяется для получения данных ресурсов, а POST — для создания новых. Например, форма авторизации должна отправлять данные через POST, чтобы избежать логирования пароля в URL, а при поиске лучше использовать GET, чтобы можно было сохранить URL с параметрами запроса.
