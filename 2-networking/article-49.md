# Что такое REST и на каких принципах он основан?

### Категории

Backend, Database, Message queue, API, Networking, Architecture

### Краткий ответ

**Что такое REST и на каких принципах он основан?**  
- архитектурный стиль для веб-сервисов  
- основан на протоколе HTTP  
- использует **ресурсы** с уникальными URI  
- операции через стандартные методы HTTP: GET, POST, PUT, DELETE  
- **stateless** — сервер не хранит состояние клиента  
- единый интерфейс: идентификация ресурсов, манипуляция ими через представления  
- поддержка кэширования для производительности  
- масштабируемость и **самоописуемость** сообщений  
- применим для создания легко расширяемых API, широко используется в микросервисах и мобильных приложениях

### Подробный ответ
#### Основной ответ
**REST (Representational State Transfer)** — это архитектурный стиль для разработки распределённых систем, в частности, веб-сервисов. Он определяет набор принципов и ограничений, которые позволяют создавать масштабируемые, эффективные и простые в использовании API, использующие HTTP протокол. REST ориентирован на ресурсы — каждую сущность системы представляют как ресурс с уникальным URI.

#### Ключевые моменты
- **Клиент-серверная архитектура** — разделение интерфейса и хранения данных, что обеспечивает независимое развитие и масштабирование обеих частей.
- **Бесcостояние (statelessness)** — каждый запрос от клиента к серверу должен содержать всю необходимую информацию, сервер не хранит состояние между запросами, что упрощает масштабирование и повышает отказоустойчивость.
- **Единый интерфейс** — стандартизация взаимодействия через ограниченный набор хорошо определённых методов HTTP (GET, POST, PUT, DELETE и др.), что облегчает понимание и интеграцию.
- **Кешируемость** — возможность кэшировать ответы на уровне клиента или прокси для оптимизации производительности и снижения нагрузки.
- **Многоуровневая система** — архитектура может включать множества уровней (прокси, балансировщики), которые полностью прозрачны для клиента.
- **Код по требованию (опционально)** — сервер может передавать исполняемый код (например, JavaScript), но это редко используется.

#### Практический контекст
REST широко применяется для создания веб-API в микросервисах, где важна масштабируемость и простота интеграции. Например, в **React 18** фронтенде запросы к REST API позволяют загружать данные для UI. Многие современные REST API используют HTTP статус-коды и заголовки для передачи статуса и метаданных, а JSON — как формат обмена данными, что соответствует принципу представления ресурсов.
