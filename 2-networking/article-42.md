# По какому протоколу работает REST?

### Категории

Templates, Async, Frontend, API, Networking, Protocol, Promises

### Краткий ответ

**По какому протоколу работает REST?**

- REST (Representational State Transfer) — архитектурный стиль веб-сервисов  
- Работает поверх протокола **HTTP/HTTPS**  
- Использует стандартные HTTP-методы: GET, POST, PUT, DELETE для операций  
- Зависит от методов и URI для взаимодействия с ресурсами  
- Основывается на принципах безсостояния (stateless) и кеширования  
- Форматы обмена: JSON, XML и др. через HTTP-заголовки  
- Используется для создания масштабируемых и простых в интеграции API  

Итог: REST — это архитектурный стиль, работающий **через HTTP/HTTPS протокол**.

### Подробный ответ
#### Основной ответ
**REST (Representational State Transfer)** — это архитектурный стиль для построения распределённых систем, который в основном использует протокол **HTTP** для передачи данных между клиентом и сервером. Сам по себе REST не накладывает требований к конкретному протоколу, но на практике и по стандарту он наиболее распространён именно поверх HTTP.

#### Ключевые моменты
- **HTTP методы** (GET, POST, PUT, DELETE и др.) отражают операции над ресурсами, что соответствует принципам REST.
- REST использует стандартные коды ответа HTTP (например, 200, 404, 500) для передачи статуса операции, что облегчает обработку ошибок.
- Хотя REST API могут теоретически работать и поверх других транспортных протоколов (например, SMTP или FTP), в индустрии принято использовать именно HTTP/HTTPS из-за его ubiquity, поддержки в браузерах и инфраструктуре.

#### Практический контекст
В реальных проектах RESTful веб-сервисы обычно реализуются с использованием HTTP/1.1 или HTTP/2, что позволяет эффективно использовать такие возможности как кеширование, аутентификацию, шифрование (через HTTPS). Например, многие публичные API (GitHub API, Twitter API) используют REST и протокол HTTP для взаимодействия клиентов и серверов с отказоустойчивостью и масштабируемостью.
