# Какой заголовок указывает на формат JSON?

### Категории

Data format, API, Networking, Testing, HTTP headers, Format

### Краткий ответ

**Какой заголовок указывает на формат JSON?**

- HTTP заголовок **Content-Type**
- значение для JSON: **application/json**
- подтверждает формат тела запроса/ответа
- важно для корректной обработки на клиенте и сервере
- стандартизирован в RFC 7159
- используется в REST API, AJAX
- гарантирует парсинг как JSON

Этот заголовок сигнализирует серверу или клиенту, что содержимое в формате JSON, что критично для правильной десериализации и обработки данных.

### Подробный ответ
#### Основной ответ
Заголовок HTTP, который указывает на формат данных JSON, это **`Content-Type: application/json`**. Он используется как в запросах, так и в ответах для обозначения того, что тело сообщения содержит данные в формате JSON.

#### Ключевые моменты
- **Content-Type** — стандартный HTTP-заголовок, определяющий тип содержимого в теле запроса или ответа.
- **`application/json`** — MIME-тип, официально зарегистрированный для JSON-документов согласно RFC 8259.
- В REST API именно этот заголовок позволяет клиентам и серверам правильно распознавать и парсить JSON-данные, обеспечивая корректную сериализацию и десериализацию.

#### Практический контекст
Часто при вызове API на стороне клиента (например, в fetch или axios) нужно явно указать `Content-Type: application/json` при отправке JSON-объекта, чтобы сервер обработал данные корректно. Аналогично, сервер в ответе выставляет этот заголовок для информирования клиента о формате возвращаемого содержимого.
