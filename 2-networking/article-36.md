# Что означает аббревиатура REST?

### Категории

Templates, Interview, Async, Frontend, API, Networking, Promises

### Краткий ответ

**Что означает аббревиатура REST?**  
- архитектурный стиль для взаимодействия в сети  
- Representational State Transfer (передача представлений состояния)  
- опирается на **HTTP методы** (GET, POST, PUT, DELETE)  
- использует **ресурсы** с уникальными URI  
- статeless (без сохранения состояния на сервере)  
- поддерживает кэширование для повышения производительности  
- применяется для разработки **веб-API** с простотой и масштабируемостью

### Подробный ответ
#### Основной ответ
**REST** — это аббревиатура от **Representational State Transfer**, что можно перевести как «передача состояния представления». Это архитектурный стиль для построения распределённых систем, прежде всего веб-сервисов, который определяет набор принципов и ограничений для создания масштабируемых и удобных в поддержке API.

#### Ключевые моменты
- **Ресурсы** — основа REST, каждый объект системы (пользователь, заказ, продукт) представлен в виде ресурса с уникальным URI.
- **Представления ресурсов** — обмен данными происходит в виде представлений (JSON, XML, HTML), что позволяет клиентам работать с нужным форматом.
- **Статус состояния** — сервер не хранит состояние между запросами (stateless), каждый запрос должен быть самодостаточным, что облегчает горизонтальное масштабирование.
- REST использует стандартные HTTP-методы: **GET** (получение), **POST** (создание), **PUT** (обновление), **DELETE** (удаление), что делает API интуитивно понятным и совместимым.

#### Практический контекст
REST широко применяется в веб-разработке — начиная с RESTful API для микросервисов и заканчивая публичными API крупных компаний. Например, API в **GitHub** или **Twitter** основаны на REST, что облегчает интеграцию и сопровождение. В React 18 и других современных фронтенд-фреймворках REST API часто используется для взаимодействия с бэкендом по протоколу HTTPS.
