# Что такое идемпотентность (idempotentness)?

### Категории

System analysis, Idempotency, Backend, Idempotentness, Database, Message brokers, REST, Idempotent, Messaging, API, Networking, Web, Queues, System design, Web applications, Algorithms, Architecture, Reliability

### Краткий ответ

**Что такое идемпотентность (idempotentness)?**  
- концепт из **математики** и **информатики**  
- операция, применённая много раз, даёт **тот же результат**  
- в API: повторные запросы не меняют состояние системы  
- примеры: HTTP методы — **GET, PUT, DELETE** идемпотентны  
- важна для **надёжности** и **отказоустойчивости** систем  
- упрощает повторы и обработку сбоев в распределённых системах  
- защищает от нежелательных побочных эффектов при повторении действий

### Подробный ответ
#### Основной ответ
**Идемпотентность** — это свойство операции, при котором многократное её повторное выполнение даёт тот же результат, что и однократное. В контексте программирования и систем это означает, что повторный вызов команды или запроса не изменит состояние системы дополнительно и не приведёт к ошибкам.

#### Ключевые моменты
- **В HTTP**: методы GET, PUT, DELETE считаются идемпотентными — повторить запрос несколько раз безопасно (например, несколько одинаковых DELETE не сломают ресурс).
- **В микросервисах и интеграции**: идемпотентность критична для обеспечения устойчивости к сетевым сбоям и повторным запросам, особенно при отсутствии гарантии доставки.
- **Отличие от мутабельных операций**: операции, изменяющие состояние с разным результатом при каждом вызове (например, POST для создания ресурса), не идемпотентны.

#### Практический контекст
Идемпотентные операции позволяют реализовать надёжные retry-механизмы без риска дублирования данных или побочных эффектов. В REST API это снижает количество ошибок при плохом соединении, а в распределённых системах — помогает гарантировать консистентность данных. Часто для реализации идемпотентности используют уникальные request ID и проверку уже обработанных запросов.
