# Можешь вкратце описать работу технологии аутентификации на базе JWT?

### Категории

Authentication, API, Networking, Design, Security, Architecture

### Краткий ответ

**Аутентификация на базе JWT**

- метод **безсессионной** аутентификации
- сервер создаёт и подписывает **токен** после логина
- токен содержит **закодированную информацию** пользователя (payload)
- клиент хранит токен локально (например, localStorage)
- при каждом запросе клиент отправляет токен в заголовке Authorization
- сервер верифицирует подпись и права без обращения к базе
- уменьшает нагрузку и упрощает масштабирование **API** и микросервисов

### Подробный ответ
#### Основной ответ
**JWT (JSON Web Token)** — это компактный и самодостаточный стандарт для передачи информации об аутентификации между клиентом и сервером в виде зашифрованного JSON-объекта. При аутентификации сервер создает JWT с полезными данными (claims), подписывает его секретным ключом или с помощью асимметричного шифрования, и высылает клиенту. Клиент при последующих запросах отправляет этот токен, сервер проверяет подпись и извлекает данные, подтверждая личность пользователя без необходимости хранить сессии.

#### Ключевые моменты
- **Структура**: JWT состоит из трех частей — header (алгоритм подписи), payload (данные пользователя и параметры), signature (подпись для проверки целостности).
- **Бессерверность**: при проверке токена сервер не должен обращаться к базе, что улучшает масштабируемость и производительность (важно в микросервисной архитектуре и API).
- **Безопасность**: нельзя хранить в payload чувствительные данные без шифрования, так как токен легко декодируется; критична надежная подпись и защита секретного ключа.
- **Срок действия**: обычно JWT содержит экспирацию (claim `exp`), чтобы ограничить возможность использования украденного токена.

#### Практический контекст
В современных REST API и single page приложениях (React/Vue) JWT применяется для аутентификации и авторизации пользователей, позволяя бездисковую проверку при каждом запросе. Но для защиты от угона токена часто вместе используют механизмы refresh tokens и HTTPS. В React 18 и Node.js с Express JWT часто интегрируют через middleware для автоматической валидации.
