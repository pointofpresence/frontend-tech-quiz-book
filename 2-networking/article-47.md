# Что такое протокол HTTPS?

### Категории

Templates, Async, Frontend, Encryption, API, Networking, Protocol, Security

### Краткий ответ

**Что такое протокол HTTPS?**  
- протокол передачи данных на базе **HTTP с шифрованием**  
- использует **TLS/SSL** для защиты канала  
- обеспечивает **конфиденциальность** и **целостность** данных  
- поддерживает **аутентификацию** сервера  
- защищает от атак **Man-in-the-Middle**  
- применяется для безопасного обмена информацией в интернете  
- ключевой стандарт для **защищённых веб-сайтов** и онлайн-транзакций

### Подробный ответ
#### Основной ответ
**HTTPS (HyperText Transfer Protocol Secure)** — это расширение протокола HTTP, которое обеспечивает защищённую передачу данных между клиентом (браузером) и сервером с помощью шифрования. Основная задача HTTPS — гарантировать конфиденциальность, целостность и аутентичность данных в сети.

#### Ключевые моменты
- **Шифрование TLS/SSL**: HTTPS использует протоколы TLS (Transport Layer Security) или его предшественник SSL для шифрования трафика, предотвращая перехват и подслушивание.
- **Аутентификация сервера**: с помощью цифровых сертификатов сертификаты гарантируют, что клиент соединяется именно с нужным сервером, а не с злоумышленником (атака типа "man-in-the-middle" предотвращается).
- **Целостность данных**: TLS обеспечивает проверку, что данные не были изменены в процессе передачи.
- HTTPS работает поверх порта 443, в отличие от стандартного HTTP, который использует порт 80.
  
#### Практический контекст
Сегодня HTTPS — стандарт для всех веб-сервисов, особенно для тех, где передаётся чувствительная информация (пароли, платежи). В React-приложениях или мобильных клиентах чаще всего используют HTTPS API-запросы для повышения безопасности и доверия пользователей. Сертификаты обычно выпускаются и автоматически обновляются через Let's Encrypt или платные центры сертификации.
