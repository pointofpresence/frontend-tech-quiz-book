# Чем отличаются HTTP GET и POST запросы?

### Категории

API, Networking, HTTP, Protocol, Request, Network

### Краткий ответ

**Чем отличаются HTTP GET и POST запросы?**

- HTTP методы для обмена данными между клиентом и сервером  
- **GET**: запрашивает данные, параметры в URL  
- **POST**: отправляет данные в теле запроса  
- GET — идемпотентен (безопасен для повторных запросов)  
- POST — не идемпотентен, может изменять состояние сервера  
- GET ограничен длиной URL, POST — практически не ограничен  
- GET кэшируется, POST обычно нет  
- POST лучше для отправки чувствительных/больших данных (форма, JSON)  
- используйте GET для чтения, POST для создания/изменения ресурсов

### Подробный ответ  
#### Основной ответ  
**HTTP GET** и **POST** — это два из наиболее часто используемых методов протокола HTTP для обмена данными между клиентом и сервером. Основное различие в их назначении и способе передачи данных: GET используется для получения ресурсов и передаёт параметры в URL, тогда как POST — для отправки данных на сервер (например, формы) и передает данные в теле запроса.  

#### Ключевые моменты  
- **GET**: идемпотентный и безопасный метод, параметры передаются в строке URL (в query string), что ограничивает объём данных (~2 Кб, зависит от браузера/сервера). Параметры видны в адресной строке, поэтому не подходят для передачи чувствительных данных. Часто кешируется и логируется.  
- **POST**: нестабилен (не идемпотентен), данные отправляются в теле запроса, что позволяет передавать большие объёмы и более сложные структуры (JSON, файлы). Используется для создания или обновления ресурсов. Обычно не кэшируется.  
- **Безопасность и кэширование**: GET-запросы проще кешируются прокси и браузерами, POST требует дополнительных настроек для кеширования. GET хорошо для чтения, POST для модификации.  

#### Практический контекст  
В реальных приложениях GET используют для получения данных без побочных эффектов (например, запрос списка товаров), POST — для операций с изменением состояния сервера (регистрация пользователя, отправка формы). В REST API метод GET соответствует операции чтения (READ), POST — созданию нового ресурса (CREATE).
