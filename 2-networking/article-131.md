# Какие коды HTTP-ответов относятся к группам 2xx, 3xx, 4xx, 5xx? Приведите примеры (200, 201, 304, 400, 401, 403, 404, 500).

### Категории

REST, API, Networking, HTTP, HTTP status codes, Testing, Response codes, Architecture, QA

### Краткий ответ

**Группы HTTP-кодов и примеры**

- 2xx: **успешные** ответы, операция выполнена  
  - 200 OK: запрос успешно обработан  
  - 201 Created: ресурс создан  

- 3xx: **перенаправления**, дополнит. действия нужны  
  - 304 Not Modified: ресурс не изменён, использовать кеш  

- 4xx: **ошибки клиента**, неверный запрос  
  - 400 Bad Request: синтаксическая ошибка в запросе  
  - 401 Unauthorized: требуется аутентификация  
  - 403 Forbidden: доступ запрещён  
  - 404 Not Found: ресурс не найден  

- 5xx: **ошибки сервера**, проблема на стороне сервера  
  - 500 Internal Server Error: внутренняя ошибка сервера  

**Объяснение:** Коды классифицируют результат HTTP-запроса по состоянию, помогают корректно обрабатывать ответы, диагностировать проблемы и управлять потоками данных между клиентом и сервером.

### Подробный ответ
#### Основной ответ
HTTP-коды ответов делятся на пять основных категорий в зависимости от первой цифры кода. В частности, **коды групп 2xx, 3xx, 4xx и 5xx** широко используются для индикации статуса запроса и его результата.

- **2xx (Успешные ответы)** означают, что запрос был успешно обработан сервером.
- **3xx (Перенаправления)** сигнализируют, что клиент должен предпринять дополнительные действия для завершения запроса, обычно это перенаправление на другой URL.
- **4xx (Ошибки клиента)** указывают на проблемы с запросом со стороны клиента, например, неверные параметры или отсутствие доступа.
- **5xx (Ошибки сервера)** обозначают, что сервер столкнулся с проблемой при обработке корректного запроса.

Примеры:

- **200 OK**: Запрос успешно выполнен, и результат возвращается клиенту.
- **201 Created**: Ресурс успешно создан, обычно в ответ на POST-запрос.
- **304 Not Modified**: Ресурс не изменился с момента последнего запроса, клиент может использовать кэшированную версию.
- **400 Bad Request**: Некорректный или повреждённый запрос, сервер не может его обработать.
- **401 Unauthorized**: Требуется аутентификация, клиент не прошёл проверку.
- **403 Forbidden**: Доступ запрещён, даже при аутентификации.
- **404 Not Found**: Запрашиваемый ресурс не найден.
- **500 Internal Server Error**: Внутренняя ошибка сервера, непредвиденная проблема при обработке.

#### Ключевые моменты
- 2xx — успешные операции, часто встречается 200 и 201 при REST API.
- 3xx — перенаправления, например, 304 оптимизирует работу с кэшем.
- 4xx — ошибки клиента, часто связаны с валидацией или доступом, ключевые 400, 401, 403, 404.
- 5xx — серверные ошибки, показывают необходимость исправления или диагностики с серверной стороны (500 — наиболее общий).

#### Практический контекст
Понимание и правильная обработка этих кодов требуется для построения надёжных API, отладки приложений и обеспечения комфортного UX. Например, 401 и 403 используются в системах авторизации, 304 помогает уменьшить нагрузку на сеть, а 500 требует мониторинга серверного стека (Prometheus, Sentry).
