# В чём разница между HTTP и HTTPS?

### Категории

Backend, Database, Protocols, API, Networking, Web, Testing, Network, Security

### Краткий ответ

**Разница между HTTP и HTTPS**

- HTTP — протокол передачи данных в интернете без шифрования  
- HTTPS — HTTP поверх **TLS/SSL**, обеспечивает **шифрование**  
- HTTPS использует сертификаты для **аутентификации сервера**  
- HTTP уязвим к перехвату данных и атакам типа "man-in-the-middle"  
- HTTPS гарантирует **целостность** и **конфиденциальность** данных  
- HTTPS требует больше ресурсов из-за криптографии (незначительно)  
- HTTPS обязательный стандарт для сайтов с авторизацией и платежами, повышает **доверие пользователей**

### Подробный ответ
#### Основной ответ
**HTTP** (HyperText Transfer Protocol) — это протокол для передачи данных в интернете, который работает поверх TCP и предназначен для обмена текстовыми и другими ресурсами между клиентом и сервером. **HTTPS** — это расширение HTTP с использованием протокола **TLS/SSL** для шифрования передаваемых данных, что обеспечивает безопасность связи.

#### Ключевые моменты
- **Шифрование**: HTTP передаёт данные в открытом виде, что делает их уязвимыми к перехвату и атаке типа man-in-the-middle. HTTPS защищает данные с помощью TLS, обеспечивая конфиденциальность и целостность информации.
- **Аутентификация сервера**: HTTPS требует наличия SSL-сертификата, который подтверждает подлинность сервера, снижая риск фишинга и подмены узла.
- **Порт и производительность**: Стандартный порт HTTP — 80, HTTPS — 443. Шифрование в HTTPS добавляет небольшой оверхед, но современные алгоритмы и аппаратное ускорение делают влияние на производительность минимальным.

#### Практический контекст
В современном веб-разработке HTTPS стал обязательным стандартом из-за требований безопасности и SEO — браузеры маркируют HTTP-сайты как небезопасные, а Google учитывает HTTPS в ранжировании. HTTPS широко используется для интернет-банкинга, онлайн-магазинов и любых ресурсов, работающих с персональными данными.
