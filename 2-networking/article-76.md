# JWT что такое? Из каких частей состоит токен?

### Категории

Authentication, Token, API, Networking, Security

### Краткий ответ

**JWT что такое? Из каких частей состоит токен?**

- JWT — **JSON Web Token**, стандарт передачи данных  
- Самостоятельный, компактный, безопасный **токен с подписью**  
- 3 части, разделённые точками:  
  - **Header**: алгоритм + тип токена (обычно HS256)  
  - **Payload**: утверждения (claims) — данные пользователя или контекста  
  - **Signature**: хэш от header + payload + секрет, для проверки подлинности  
- Используется для **аутентификации и авторизации** в веб-приложениях  
- Позволяет без серверной сессии **проверять права** пользователя  
- Токен хранится на клиенте, обычно в localStorage или cookie  
- Обеспечивает безопасность при передаче данных через HTTPS  

Итого: JWT = Header + Payload + Signature для безопасного обмена данными с проверкой подлинности.

### Подробный ответ
#### Основной ответ
**JWT (JSON Web Token)** — это компактный, URL-безопасный формат токена, который используется для передачи информации между участниками в виде JSON-объекта. Он широко применяется для аутентификации и авторизации в веб-приложениях, позволяя передавать данные, проверяемые по подписи без необходимости постоянного обращения к серверу.

#### Ключевые моменты
- JWT состоит из трёх частей, разделённых точками: **Header (заголовок), Payload (полезная нагрузка), Signature (подпись)**.
- **Header** содержит информацию о типе токена и алгоритме подписи (например, HS256 — HMAC SHA-256).
- **Payload** — полезная нагрузка, включает стандартные поля (claims) вроде `iss` (issuer), `exp` (expiration), а также кастомные данные, например, ID пользователя.
- **Signature** формируется на основе закодированных Header и Payload, плюс секретный ключ или публичный/приватный ключ (для асимметричных алгоритмов).
- Благодаря подписи сервер может проверить целостность и подлинность токена, не сохраняя состояние на сервере (stateless authentication).

#### Практический контекст
В проектах, например в **Node.js с Express** и библиотекой **jsonwebtoken (v8+)**, JWT обычно выдают при логине пользователя, затем клиент хранит токен в localStorage или cookie и отправляет в заголовке Authorization. Проверка подписи (и срока жизни) на каждом запросе обеспечивает безопасность без необходимости сессионного хранения.
