# Как написать React-компонент, чтобы при монтировании запускался setInterval, выводящий в консоль "Hello world" каждые 10 секунд? Что важно учесть при такой реализации с setInterval?

**React setInterval с выводом в консоль**

- функциональный компонент с хуком **useEffect**  
- внутри useEffect создаём **setInterval** с интервалом 10 секунд (10000 мс)  
- обязательно **чистим интервал** при размонтировании через return из useEffect (clearInterval)  
- useEffect с пустым массивом зависимостей [], чтобы интервал запускался только один раз при монтировании  
- без очистки интервала возникает **утечка памяти** и повторные вызовы после размонтирования  
- useRef можно использовать для хранения ID интервала, если нужно другой доступ  
- вывод в консоль внутри callback setInterval: console.log("Hello world")

---

**Код-пример:**

```jsx
import React, { useEffect } from 'react';

function HelloWorldInterval() {
  useEffect(() => {
    const intervalId = setInterval(() => {
      console.log("Hello world");
    }, 10000);

    return () => clearInterval(intervalId);
  }, []);

  return <div>Interval запущен</div>;
}
```

---

Такой подход гарантирует корректный запуск и остановку интервала без утечек и лишних вызовов.

# Подробный ответ
## Основной ответ
Для реализации React-компонента, который при монтировании запускает `setInterval` с выводом в консоль "Hello world" каждые 10 секунд, оптимальный подход — использовать хук **useEffect**. Внутри эффекта создаём интервал, а при размонтировании компонента обязательно очищаем его через `clearInterval`, чтобы избежать утечек памяти и нежелательных повторных вызовов.

```jsx
import React, { useEffect } from 'react';

function HelloWorldLogger() {
  useEffect(() => {
    const id = setInterval(() => {
      console.log('Hello world');
    }, 10000);

    return () => clearInterval(id); // Очистка при размонтировании
  }, []);

  return null; // Или любой другой UI, если нужен
}
```

## Ключевые моменты
- **useEffect с пустым массивом зависимостей** (`[]`) запускается один раз при монтировании, аналогично `componentDidMount`.
- **Очистка интервала в возвращаемой функции эффекта** предотвращает сохранение работающих таймеров после удаления компонента, что особенно важно при динамическом появлении/исчезновении компонента.
- Время интервала в 10 секунд задаётся в миллисекундах — то есть `10000`.
- При необходимости можно рассмотреть использование `useRef` для хранения идентификатора интервала, если будет нужно управлять им из другого хука или обработчика.

## Практический контекст
В реальных проектах такой паттерн применим для регулярного обновления данных, heartbeat-запросов или логирования. Но важно учитывать, что частое выполнение операций через `setInterval` может влиять на производительность и энергопотребление, поэтому интервалы подбираются с учётом бизнес-логики и нагрузки. В React 18+ с режимом Strict Mode эффект рендерится дважды в разработке для обнаружения сайд-эффектов — это стоит учитывать при тестировании.
