# Что такое мемоизация, и как её можно использовать для оптимизации производительности?

Категория: Технологии

### Краткий ответ

**Что такое мемоизация и как её использовать для оптимизации?**  
- техника **кеширования** результатов функции  
- хранит результаты для **входных аргументов**  
- при повторном вызове возвращает **предварительно вычисленное** значение  
- снижает количество **вычислений**, особенно в рекурсии  
- повышает производительность при дорогих **функциях pure**  
- подходит для оптимизации **чистых функций** без побочных эффектов  
- широко используется в **динамическом программировании** и UI-рендерингах  

Мемоизация — это способ уменьшить время выполнения за счёт экономии повторных вычислений путём сохранения и повторного использования результатов.

### Подробный ответ
#### Основной ответ  
**Мемоизация** — это техника оптимизации, которая заключается в кэшировании результатов функций с целью избежать повторных вычислений одних и тех же входных данных. При вызове функции с теми же аргументами результат получается не заново, а берётся из заранее сохранённого кеша, что значительно ускоряет работу, особенно при ресурсоёмких вычислениях.

#### Ключевые моменты  
- **Идемпотентность функций**: мемоизация работает эффективно для чистых функций, где вывод зависит только от входных параметров и не вызывает побочных эффектов.  
- **Хранение состояния**: обычно реализуется через словарь или объект, где ключ — совокупность аргументов, а значение — результат.  
- **Trade-off — память vs скорость**: мемоизация уменьшает время выполнения, но увеличивает использование памяти, поэтому важно контролировать размер кеша, например, с помощью LRU-стратегий.  

#### Практический контекст  
В React 18 часто применяют мемоизацию для оптимизации ререндеров компоненты через useMemo или useCallback, минимизируя ненужные пересчёты и вызовы функций. В алгоритмах динамического программирования (например, вычисление чисел Фибоначчи) мемоизация позволяет сократить экспоненциальную сложность до линейной. В backend сервисах — кэш запросов к внешним API или тяжёлых вычислений, чтобы повысить throughput и снизить latency (~50ms вместо секунд).
