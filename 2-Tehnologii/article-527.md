# Какие SQL-запросы чаще всего используются при тестировании и проверке данных в БД?

Категория: Технологии

### Краткий ответ

**Типичные SQL-запросы для тестирования и проверки данных**

- выборка с помощью SELECT для проверки корректности данных  
- фильтрация WHERE для проверки условий и границ  
- агрегатные функции (COUNT, SUM, AVG) для контроля статистики  
- JOIN для проверки связей между таблицами  
- EXPLAIN для оценки плана выполнения запросов  
- INSERT/UPDATE/DELETE с возвратом изменений для тестирования операций  
- транзакции с ROLLBACK для безопасного тестирования изменений  

Основная цель — обеспечить точность, целостность и корректную работу данных и бизнес-логики в БД.

### Подробный ответ
#### Основной ответ
При тестировании и проверке данных в базе данных чаще всего используют **SQL-запросы**, которые позволяют анализировать целостность, корректность и полноту данных. Это обычно **SELECT-запросы с условиями**, агрегациями и фильтрацией, а также запросы для проверки связей между таблицами. Особое внимание уделяют проверкам дублирующихся строк, отсутствующих и неконсистентных данных.

#### Ключевые моменты
- **SELECT с WHERE** — для выборки данных по конкретным условиям, например, по дате, статусу или идентификаторам, чтобы проверить корректность конкретных записей.
- **Агрегации (COUNT, SUM, AVG, MAX, MIN)** — используются для подсчёта записей, проверки суммарных значений, нахождения граничных дат и уровней, чтобы убедиться в полноте и логичности данных.
- **JOIN-запросы** — помогают проверить, корректно ли связаны таблицы, например, нет ли orphan-строк или неверных связей по внешним ключам.
- **Запросы на проверку дубликатов** — для определения повторяющихся записей по наборам полей с помощью GROUP BY HAVING COUNT > 1.
- **EXISTS/NOT EXISTS, IN/NOT IN** — для проверки наличия связанных записей или выявления отсутствующих записей, что важно в тестах на referential integrity.
  
#### Практический контекст
В реальных проектах, например, с PostgreSQL 14+, используют запросы для верификации данных после миграций или ETL-процессов: проверяют счётчики строк, контрольные суммы, отсутствие null-значений в обязательных колонках, а также сравнивают изменение данных до и после операций. Помимо сырого SQL часто применяют автоматизированные скрипты тестирования с использованием pytest или Airflow, где SQL выступает частью проверки данных.
