# Чем GraphQL принципиально отличается от REST?

**Чем GraphQL принципиально отличается от REST?**

- API запросы: GraphQL — **один endpoint**, REST — много эндпоинтов  
- Запросы: GraphQL запрашивает **только нужные поля**, REST возвращает **фиксированный набор данных**  
- Гибкость: GraphQL позволяет клиенту **динамично формировать запросы**, REST — фиксированные ресурсы  
- Типизация: GraphQL имеет **жёсткую схему (schema)** и типы, REST — без встроенной типизации  
- Эффективность: GraphQL снижает количество запросов — **агрегация данных**, REST может требовать множественные вызовы  
- Версионирование: GraphQL избегает версий через **эволюцию схемы**, REST часто нуждается в версионировании API  
- Ошибки: GraphQL возвращает ошибки вместе с данными, REST — чаще отдельно через HTTP-коды

Итого: GraphQL — цеоентричный, **гибкий и типизированный** API с оптимизацией передачи данных, REST — **ресурсно-ориентированный** с фиксированными путями.

# Подробный ответ
## Основной ответ
**GraphQL** и **REST** — это разные подходы к построению API. Главное отличие в том, что GraphQL позволяет клиенту точно запрашивать только те данные, которые ему нужны, в одной структуре запроса, в то время как REST оперирует фиксированными эндпоинтами с заранее определённым форматом ответов. GraphQL концентрируется на структуре данных и запросах, REST — на ресурсах и действиях над ними.

## Ключевые моменты
- **Гибкость запросов в GraphQL**: клиент формирует запрос с нужными полями, уменьшая избыточность данных и количество запросов. В REST часто приходится перегружать сеть множеством отдельных вызовов или получать лишние поля.
- **Единая точка входа в GraphQL против множества эндпоинтов в REST**: GraphQL использует один endpoint (обычно `/graphql`), где описывается схема и можно делать сложные запросы и мутации. REST формирует несколько endpointов для разных ресурсов, что усложняет управление большим числом маршрутов.
- **Типизированная схема GraphQL**: сервер предоставляет явное описание схемы с типами данных, что облегчает генерацию документации и статическую проверку запросов. REST не стандартизует схему, документирование обычно делается отдельно (Swagger/OpenAPI).
- **Функциональные отличия**: GraphQL объединяет запросы и мутации (создание, обновление, удаление) в одном API, REST же разделяет их на HTTP-методы (GET, POST, PUT, DELETE).

## Практический контекст
В реальных проектах GraphQL часто выбирают для фронтенд-приложений с динамическими UI (например, React 18+), где нужно оптимизировать передачу данных и снизить количество запросов. REST используется широко благодаря простоте и зрелости — подходит для публичных API и микросервисов с чётко очерченными ресурсами. Trade-off — GraphQL быстрее в разработке гибких клиентских приложений, но требует больше усилий на серверной стороне и потенциально сложнее в кэшировании, тогда как REST проще масштабировать и кэшировать HTTP-уровнево.
