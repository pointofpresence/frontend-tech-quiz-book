# Когда происходит боксинг?

Категория: Технологии

### Краткий ответ

**Когда происходит боксинг?**  
- контекст: JavaScript — типы данных  
- боксинг — **автоматическое обёртывание** примитива в объект  
- происходит при вызове методов у примитивов (строка, число, булево)  
- временно создаётся объект-обёртка (String, Number, Boolean)  
- после выполнения метода объект удаляется, остаётся примитив  
- позволяет использовать объекты как если бы это были примитивы с методами  
- улучшает удобство работы с примитивными значениями в JS и других языках (например, C# — похожий механизм)

### Подробный ответ
#### Основной ответ
**Боксинг (boxing)** — это процесс преобразования значения типа **значения (value type)** в объектную оболочку (в .NET — объект типа `System.Object`). Проще говоря, когда примитивное значение, например `int`, упаковывается в объект для совместимости с типами ссылок. Это происходит автоматически во время компиляции или выполнения, когда значение value type нужно использовать как object или interface.

#### Ключевые моменты
- **Когда боксинг происходит:** при присваивании value type переменной типа `object` или интерфейса, например `object obj = 123;` происходит упаковка `int` в объект.
- **Технически:** CLR выделяет память в управляемой куче и копирует туда значение, обращаясь к нему как к объекту.
- **Преимущества и накладные расходы:** позволяет value type использоваться в контексте объектов, но боксинг дорог по времени и памяти (allocation + сборка мусора).
- **Анбоксинг (unboxing):** обратный процесс — извлечение значения из объекта с проверкой типа.

#### Практический контекст
В реальных проектах боксинг стоит минимизировать, особенно в горячих участках кода, например в коллекциях без дженериков (`ArrayList`) или при использовании non-generic interface. В .NET начиная с версии 2.0, дженерики значительно уменьшают необходимость боксинга, позволяя работать с типами значений без упаковки.
