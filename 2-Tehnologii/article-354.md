# Корутины и event loop.

Категория: Технологии

### Краткий ответ

**Корутины и event loop**

- корутины — легковесные функции для **кооперативной многозадачности**
- позволяют **приостанавливать** и **возобновлять** выполнение
- работают на уровне **языка программирования**
- event loop — цикл обработки событий, основа **асинхронного ввода-вывода**
- управляет вызовом коллбэков **без блокировки**
- event loop запускает корутины, обеспечивая переключение между ними
- главное отличие: корутины — это **логика переключения**, а event loop — **механизм планирования**
- вместе позволяют писать асинхронный код без потоков
- применяются в Node.js, Python asyncio, Kotlin, Go (goroutines + runtime)

### Подробный ответ
#### Основной ответ
**Корутины** — это специальные функции, которые могут приостанавливать своё выполнение и возобновлять его позже, что позволяет писать асинхронный код, выглядящий как синхронный. **Event loop** — это цикл событий, который управляет выполнением асинхронного кода, ожидая завершения операций ввода-вывода или таймеров и распределяя управление между корутинами или другими задачами в неблокирующем режиме.

#### Ключевые моменты
- В языках вроде **Python 3.5+** с `asyncio` корутины создают видимость последовательного кода при асинхронных операциях; при вызове `await` выполнение приостанавливается, и управление возвращается в event loop.
- **Event loop** — это ядро асинхронной модели, которое отслеживает готовность событий (например, I/O, таймеры), чтобы эффективно переключать задачи без блокировок и не создавая потоки.
- В отличие от потоков и коллбеков, корутины + event loop позволяют писать масштабируемый, лёгкий, не блокирующий код с низкими накладными расходами на переключение контекста.

#### Практический контекст
В реальных проектах на Python event loop поддерживается `asyncio`/`uvloop` — высокопроизводительный loop с уменьшенной задержкой (~50ms для сетевых операций). Корутины применяют для обработки запросов в серверных приложениях (например, FastAPI), web scraping, real-time систем, где важно не блокировать главный поток и эффективно управлять тысячами клиентских соединений.
