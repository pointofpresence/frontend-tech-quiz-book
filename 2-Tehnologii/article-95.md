# Что такое idempotency key и как он применяется?

**Что такое idempotency key и как он применяется?**  
- Контроль повторных запросов в API  
- Уникальный идентификатор для операции  
- Обеспечивает идемпотентность: повторный запрос не приводит к повторному изменению  
- Позволяет серверу распознать дубли и вернуть тот же результат без повторного выполнения  
- Активно используют в платежных и заказных системах  
- Уменьшает ошибки и накладные расходы на клиенте и сервере  
- Ключевой элемент для надёжности и безопасности распределённых систем

# Подробный ответ
## Основной ответ
**Idempotency key** — это уникальный идентификатор, который клиент присылает серверу для однозначной идентификации конкретного запроса с целью гарантировать, что при повторной отправке одного и того же запроса он будет выполнен сервером **один раз**. Это позволяет избежать дублирования операций, например, при сетевых ошибках или повторных попытках.

## Ключевые моменты
- Обычно используется в **финансовых операциях** (например, платёжных системах) или любых критичных API, где повторный запрос может привести к созданию дублирующих сущностей или транзакций.
- Сервер хранит состояние результата для ключа — если запрос с таким ключом уже была обработан, сервер возвращает сохранённый результат, не выполняя повторно логику.
- Idempotency key позволяет обеспечить **идемпотентность** в системах, которые по своей природе не имеют её, например, POST-запросы в HTTP.

## Практический контекст
В API платежных шлюзов, таких как Stripe или PayPal, клиент передаёт idempotency key вместе с запросом оплаты. Если сеть обрывается и клиент повторяет запрос, сервер понимает, что это дубликация и не создаёт вторую оплату. Такой подход повышает надёжность систем и предотвращает финансовые ошибки. В реализации часто используют TTL для хранения ключей — чтобы не накапливать данные бесконечно.
