# Какие есть уровни приоритетов (QoS)?

**Уровни приоритетов QoS**  
- область: Управление сетью и трафиком  
- QoS (Quality of Service) — механизм **управления приоритетом трафика**  
- базовые уровни:  
  - **Высокий приоритет**: критичные задержки, например, VoIP, видео  
  - **Средний приоритет**: интерактивные приложения, веб-серфинг  
  - **Низкий приоритет**: фоновая загрузка, резервные копии  
- технически может использоваться классификация по DSCP, IP precedence  
- важна для **обеспечения гарантированной пропускной способности** и минимизации задержек  
- применяется в маршрутизаторах, коммутаторах, VPN, облачных сервисах  
- помогает избегать **перегрузки сети** и потери пакетов  
- ключ к оптимизации работы **реального времени** и мультимедийных сервисов

# Подробный ответ
## Основной ответ
**Уровни приоритетов QoS (Quality of Service)** — это механизмы управления качеством обслуживания в сетях, которые позволяют классифицировать и приоритезировать трафик для обеспечения гарантированной пропускной способности, задержек и надежности. Основная цель — выделить ресурсы сети под разные типы данных в зависимости от их требований к качеству и времени доставки.

## Ключевые моменты
- **Дифференцированное обслуживание (DiffServ)** — наиболее распространённая модель QoS, где трафик разделяется по классам с разным приоритетом. Каждый пакет маркируется с помощью DSCP (Differentiated Services Code Point), который определяет уровень обслуживания: от высокой приоритетности (для голосового трафика) до низкой (для фоновых загрузок).
- **Уровни приоритетов обычно включают**: 
  - **Высокий приоритет (например, EF — Expedited Forwarding)** для real-time трафика (VoIP, видео-конференции), требующего минимальных задержек и джиттера.
  - **Средний приоритет** для интерактивных приложений (веб, обмен сообщениями).
  - **Низкий приоритет (например, BE — Best Effort)** для фонового, не чувствительного к задержкам трафика, такого как загрузки или email.
- **Механизмы контроля**: очередь с разными приоритетами, полисы формирования трафика (shaping), ограничение скорости (rate limiting) и прерывания.
- **Стандарты**: кроме DiffServ, есть и другие, например, IntServ с reservation для каждого потока, но он менее масштабируемый и редко используется в крупных сетях.

## Практический контекст
В крупных корпоративных и операторских сетях QoS используется для разграничения голосовых вызовов с задержкой ~50 мс, приоритетного видео с гарантированной пропускной способностью и при этом поддержания сетевого трафика с меньшим приоритетом без перебоев. Например, в Cisco IOS для настройки QoS применяют классы и политики с разной очередностью, поддерживаемые с версии IOS 15+.
