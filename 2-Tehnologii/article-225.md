# В чём отличие array от List в C#?

**Отличие array от List в C#**

- array: **фиксированный размер**, статическая структура данных  
- List<T>: **динамический размер**, реализован на базе массива  
- array быстрее за счёт отсутствия перераспределения памяти  
- List<T> поддерживает удобные методы (Add, Remove, Insert)  
- array лучше для **предсказуемых** данных, List<T> — для изменяемых коллекций  
- List<T> реализует интерфейсы ICollection, IList и др. для гибкой работы  
- практическое применение: array — для **оптимизированного** доступа, List<T> — для **гибкого управления** элементами

# Подробный ответ
## Основной ответ
**Array** и **List<T>** в C# — это коллекции для хранения элементов, но они различаются по размеру, функционалу и применению. **Array** имеет фиксированный размер, заданный при создании, тогда как **List<T>** — динамический список, который автоматически изменяет размер по мере добавления или удаления элементов.

## Ключевые моменты
- **Размер и изменяемость**: Массив фиксирован по размеру и не может менять длину после создания (например, `int[] arr = new int[10];`), а `List<T>` автономно расширяется, что удобно для динамических данных.
- **Функциональность и производительность**: `List<T>` предоставляет множество встроенных методов — `Add`, `Remove`, `Insert`, `Contains` и т.д., что упрощает работу с данными. Массивы же минималистичны и обеспечивают более быстрый доступ к элементам, поскольку нет накладных расходов на управление размером.
- **Типизация и безопасность**: Оба являются типобезопасными, при условии использования generic в `List<T>`. Массивы поддерживают covariance (например, базовый класс массив можно присвоить массиву производного класса), что иногда может вести к runtime ошибкам.

## Практический контекст
В реальных проектах **массивы** часто используют для работы с фиксированным размером данных или когда важна максимальная производительность (например, операции с большими объемами чисел в Unity3D или .NET Core 6+). **List<T>** обычно применяют для коллекций с неизвестным заранее числом элементов и когда важна удобная работа с методами коллекций — к примеру, при загрузке динамического контента или результатах запросов из БД.
