# Какими способами на sequence-диаграмме можно отобразить альтернативный сценарий (if/else)?

Категория: Технологии

### Краткий ответ

**Отображение альтернативного сценария на sequence-диаграмме (if/else)**  
- UML-диаграммы взаимодействия  
- Использование фрейма «alt» (альтернативы)  
- Разделение фрейма на несколько окон (операторы if, else)  
- В каждом окне — свой поток сообщений  
- Условия в каждом разделе фрейма (guard expressions)  
- Позволяет показать ветвления логики на уровне взаимодействий  
- Облегчает визуализацию выбора сценариев исполнения в системе

### Подробный ответ
#### Основной ответ
На **sequence-диаграмме** альтернативный сценарий (эквивалент конструкции if/else) обычно отображается с помощью **функционала комбинированных фрагментов (combined fragments)**, который поддерживают UML 2.x стандарты. Основной способ — использовать **alt-механизм (альтернативы)**, где разные варианты поведения показываются в отдельных операциях, разделённых рамками с меткой **alt**.

#### Ключевые моменты
- **Alt combined fragment** — это прямоугольник, делённый на несколько фреймов (альтернатив), каждый из которых содержит свой сценарий, а сверху указывается условие (guard condition), например, `[условие A]`, `[иначе]`.
- В каждом альтернативном блоке отображается последовательность сообщений, которая выполняется, если соответствует условие.
- В некоторых случаях также применяют **opt** (опциональный фрагмент) для отображения одиночного условия без else-ветвления, либо **else** ветвь внутри alt для полной if/else конструкции.

#### Практический контекст
В инструментах моделирования (например, StarUML, Enterprise Architect, PlantUML) alt-фрагменты — стандартный и удобный способ показать ветвления без разрыва логики. Это помогает четко видеть все варианты поведения системы в одном месте, упрощая коммуникацию между разработчиками и аналитиками. Например, условие проверки авторизации пользователя может быть показано в alt с двумя ветвями — успешная и неуспешная авторизация с разной последовательностью сообщений.
