# Как в асинхронном коде вызвать синхронную функцию?

Категория: Технологии

### Краткий ответ

**Как в асинхронном коде вызвать синхронную функцию?**

- Асинхронный код запускает операции без блокировки потока  
- Синхронная функция выполняется мгновенно, блокируя поток  
- Для вызова синхронной функции внутри async можно просто вызвать её напрямую — она выполнится синхронно внутри async-контекста  
- Если результат синхронной функции нужен как промис, обернуть в Promise.resolve() или async/await  
- Используется, чтобы объединить синхронную логику с асинхронным потоком без сложностей  
- В Node.js: синхронные методы спокойно вызываются из async, но блокируют event loop на время выполнения  
- Важно помнить, что длительные синхронные функции ухудшают производительность асинхронного кода  

Итого:  
Для вызова синхронной функции из асинхронного кода обычно просто вызывают её напрямую, при необходимости оборачивая результат в Promise для интеграции с async/await.

### Подробный ответ
#### Основной ответ
Вызов синхронной функции из асинхронного кода — распространённая задача, требующая аккуратного подхода, чтобы не блокировать event loop. Основная идея — **вынести синхронную функцию в отдельный поток или процесс**, чтобы главный поток не блокировался. В Python, например, стандартный способ — использовать `loop.run_in_executor()`, который позволяет выполнить блокирующую функцию в отдельном потоке или процессе без тормозов асинхронного цикла.

#### Ключевые моменты
- **Использование Executor**: `asyncio.get_event_loop().run_in_executor()` позволяет запускать синхронный код строго параллельно, сохраняя отзывчивость async-программы. По умолчанию используется `ThreadPoolExecutor`, но можно указать `ProcessPoolExecutor` для задач, чувствительных к GIL.
- **Синтаксис с await**: асинхронный вызов оборачивается в `await loop.run_in_executor(None, sync_func, *args)`, что делает интеграцию плавной и не блокирует event loop.
- **Альтернативы**: если синхронная функция быстрая и почти не блокирует, можно вызвать напрямую, но на практике это чревато задержками; для тяжёлых вычислений лучше использовать multiprocessing.

#### Практический контекст
В проектах на Python 3.7+ использование `asyncio` и `run_in_executor` — стандартный паттерн для интеграции старых либ, библиотек с синхронным интерфейсом (например, работа с файлами, сетевыми вызовами в blocking mode). В React Native или Node.js применяются похожие подходы с worker threads или отдельными процессами. Это позволяет сохранять высокий throughput и минимизировать latency (~50ms или ниже при правильной настройке).
