# Отличие свойства animation от transition

**Отличие свойства animation от transition**

- Область: CSS-анимация и переходы  
- **transition** создает плавный переход между двумя значениями стилей при изменении свойства  
- **animation** задает сложные ключевые анимации с несколькими шагами (keyframes)  
- transition ограничен изменением при событии (hover, focus), animation — цикличное или многократное выполнение  
- animation позволяет управлять таймингом, повторением, направлением и задержкой с помощью keyframes  
- transition проще, оптимальна для базовых эффектов, animation — для комплексных и повторяющихся движений  
- Практическое применение: transition — hover-эффекты, animation — сложные интерактивные анимации UI/UX

# Подробный ответ
## Основной ответ
**Свойства `animation` и `transition` в CSS** применяются для создания анимаций, но принципиально различаются по гибкости и управлению. `transition` задаёт плавный переход между двумя состояниями CSS-свойств при изменении (например, при hover), а `animation` описывает сложные последовательности изменений с возможностью нескольких ключевых кадров, управления повторениями, задержками и направлениями анимации.

## Ключевые моменты
- **Transition** управляет промежутком между текущим и следующим состоянием стилей, без возможности циклических или сложных сценариев. Типично используется для простых эффектов (например, плавное изменение цвета, позиции) при взаимодействии пользователя.
- **Animation** использует `@keyframes`, позволяя задавать серию состояний анимации с точным контролем временных точек, повторений (`animation-iteration-count`), направления (`animation-direction`) и задержек. Эффекты более сложные и независимы от внешних событий.
- По производительности `transition` быстрее и легче для браузера, особенно для простых изменений, тогда как `animation` требует больше ресурсов, зато даёт полную свободу для сложной, цикличной анимации.

## Практический контекст
Например, в React 18 для наведения на кнопку обычно используют `transition` — плавное изменение цвета или тени, потому что триггером служит событие hover. Для более сложных вступительных эффектов элементов при загрузке страницы используют `animation` с несколькими ключевыми кадрами. В продакшн часто сочетают обе техники, применяя transition для взаимодействий и animation для фоновых или сложных анимационных эффектов.
