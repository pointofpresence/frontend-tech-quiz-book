# Что происходит при dirty-RectTransform?

**Что происходит при dirty RectTransform?**  
- UI-система Unity, часть Canvas UI  
- dirty — признак, что **RectTransform изменился** (позиция, размер, вращение)  
- триггерит пересчет **Layout и рендеринга**  
- фрейм обновления переоценивает **позиционирование, размеры элементов**  
- обновляется связанный **CanvasRenderer** для отрисовки корректной геометрии  
- влияет на **производительность** — много изменений → частые пересчеты  
- цель — поддерживать **актуальную визуализацию UI** с минимальными затратами ресурсов

# Подробный ответ
## Основной ответ
**Dirty RectTransform** — это состояние в Unity UI-системе, когда компонент RectTransform помечается как "грязный" (dirty), то есть к нему применились изменения, которые требуют перерасчёта его геометрии, позиции или размера. Это триггер для пересчёта layout и рендеринга, чтобы UI корректно отобразился с учётом обновлённых параметров.

## Ключевые моменты
- Когда RectTransform становится dirty, Unity помещает его в очередь на обновление; это может произойти из-за изменения параметров Anchors, Pivot, SizeDelta, LocalPosition и других.
- В процессе следующего цикла UI Layout rebuild система пересчитывает размеры и позиционирование с учётом новых значений, а также обновляет визуальное представление.
- Частое изменение RectTransform (особенно в сложных иерархиях UI) может привести к накладным расходам по производительности из-за многочисленных повторных пересчётов layout и батчинга.

## Практический контекст
В реальных проектах важно минимизировать количество dirty RectTransform вызовов, агрегируя изменения и избегая ненужного обновления UI в каждом кадре. В Unity 2021+ есть более оптимизированные механизмы layout rebuild, однако на больших UI-сценах грамотное управление dirty RectTransform критично для сохранения высокого FPS.
