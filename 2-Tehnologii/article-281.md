# Что такое Dockerfile?

Категория: Технологии

### Краткий ответ

**Что такое Dockerfile?**

- текстовый скрипт для Docker  
- содержит последовательность инструкций  
- описывает шаги сборки образа  
- задаёт базовый образ (FROM)  
- копирует файлы, устанавливает зависимости  
- определяет команды запуска контейнера (CMD/ENTRYPOINT)  
- нужен для автоматизации и воспроизводимости сборки образа

### Подробный ответ
#### Основной ответ
**Dockerfile** — это текстовый файл со скриптом, описывающим последовательность инструкций для автоматической сборки Docker-образа. Он позволяет декларативно описать, как собрать контейнер, начиная с базового образа, устанавливая зависимости, копируя файлы, настраивая окружение и указывая команду запуска.

#### Ключевые моменты
- **Инструкции Dockerfile**: основные команды — `FROM` (базовый образ), `RUN` (выполнение команд в контейнере при сборке), `COPY`/`ADD` (копирование файлов), `CMD` и `ENTRYPOINT` (указание запускаемого процесса).
- **Кэширование слоёв**: Docker кэширует каждый слой после выполнения инструкции, что ускоряет повторные сборки при минимальных изменениях.
- **Контролируемая, повторяемая сборка**: Dockerfile гарантирует, что любой разработчик или CI/CD пайплайн смогут собрать идентичный образ с минимальным человеческим фактором.

#### Практический контекст
В реальных проектах Dockerfile используется для стандартизации среды разработки и деплоя: например, в React-приложениях на основе Node.js или микросервисах на Python с использованием конкретной версии интерпретатора. Это упрощает интеграцию с CI/CD, где билд образа — часть пайплайна, и позволяет обеспечить однородность окружения в продакшене и локально.
