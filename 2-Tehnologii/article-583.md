# Какие существуют способы сохранения данных в iOS?

Категория: Технологии

### Краткий ответ

**Способы сохранения данных в iOS**

- локальное хранение в **UserDefaults**  
  — для мелких настроек, ключ-значение  
- **Keychain**  
  — безопасное хранение паролей, токенов  
- **Файлы** (Documents, Caches)  
  — пользовательские файлы, кэширование  
- **Core Data**  
  — ORM для сложных моделей и связей  
- **SQLite**  
  — реляционная СУБД с низкоуровневым доступом  
- **NSCoding/ Codable**  
  — сериализация объектов в файлы/бинарные данные  
- iCloud и другие облачные сервисы  
  — синхронизация и бэкап данных пользователя  

Используются в зависимости от объёма, структуры и требований к безопасности данных.

### Подробный ответ
#### Основной ответ
В iOS существует несколько основных способов сохранения данных, каждый из которых подходит под разные задачи в зависимости от объема, сложности и требований к безопасности. Основные методы включают **UserDefaults**, **Keychain**, **Core Data**, **Файловая система** и **SQLite**.

#### Ключевые моменты
- **UserDefaults** — удобен для хранения легковесных и простых данных, например, настроек или флагов. Не предназначен для больших объемов или сложных объектов.
- **Keychain** — специализированный безопасный хранилище для конфиденциальных данных, таких как пароли, токены аутентификации. Шифрование и доступ с использованием системных API.
- **Core Data** — высокоуровневая ORM для хранения сложных моделей данных с поддержкой связей, версионности и запросов. Чаще всего используется для больших объемов структурированных данных.
- **Файловая система** — применяется для сохранения произвольных файлов, документов, изображений, аудио. Доступ осуществляется через `FileManager`. Можно реализовать собственные форматы и логику.
- **SQLite** — реляционная легковесная СУБД, доступ к которой можно получить напрямую через SQL-запросы или через обертки, например, FMDB. Используется, когда нужна точечная работа с реляционными данными без дополнительного уровня Core Data.

#### Практический контекст
В реальных iOS проектах для настроек и флагов обычно применяют **UserDefaults**. Для секретов — **Keychain** (например, через Keychain Services или сторонние библиотеки). Для сложных моделей с отношениями — **Core Data** (в том числе NSPersistentContainer с iOS 10+). Для кэширования больших объектов или временных данных — файловую систему. При необходимости тонкого контроля над SQL чаще берут SQLite напрямую. Такой подход позволяет сочетать простоту, безопасность и масштабируемость.
