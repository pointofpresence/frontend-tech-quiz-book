# Что произойдет с дочерним компонентом, если поменяется key у родительского компонента?

Категория: Технологии

### Краткий ответ

**Что произойдет с дочерним компонентом, если поменяется key у родительского компонента?**

- React использует **key** для идентификации компонентов в DOM  
- При смене key родительский компонент **фундаментально пересоздаётся**  
- Все дочерние компоненты внутри родителя также **размонтируются (unmount)**  
- После размонтирования — дочерние компоненты **создаются заново (mount)**  
- Это сбрасывает их состояние и жизненный цикл  
- Поведение полезно для **полного сброса состояния** компонента  
- Часто применяется для **форсированного реинициалирования UI** при изменении данных  

Итог: смена key у родителя вызывает переработку всего subtree, включая дочерние компоненты с полным пересозданием и сбросом состояний.

### Подробный ответ
#### Основной ответ
Если у **родительского компонента** в React изменится **key**, React воспринимает это как сигнал, что весь компонент (включая всех его дочерних) должен быть **удалён и создан заново**. То есть React не будет пытаться обновлять текущий компонент, а полностью размонтирует старый и смонтирует новый, вместе с его дочерними элементами. Это приводит к **пересозданию** дочерних компонентов с нуля, что вызывает их повторный вызов жизненных циклов — например, `componentWillUnmount` у старого и `componentDidMount` у нового.

#### Ключевые моменты
- **Key** служит для идентификации компонента в дереве React, если он меняется, React считает, что это принципиально другой компонент.
- При смене key происходит полностью новый монтинг компонента и всех вложенных, а не обновление.
- Это полезно, когда нужно сбросить внутреннее состояние компонента и его дочерних, например, при смене "страницы" или уникального контекста.
- Однако чрезмерное использование смены key может негативно сказаться на производительности и привести к потере состояния.

#### Практический контекст
Типичный кейс — списки с динамическими элементами, где каждому элементу назначен уникальный key. Если key меняется у родителя (например, при переключении ключа у корневого компонента), это гарантирует полную перезагрузку содержимого и состояния дочерних компонентов, что иногда используют для принудительного сброса. В React 18 и новее такая перезагрузка может быть особенно заметна из-за новых механизмов concurrent rendering.
