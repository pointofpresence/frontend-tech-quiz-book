# Что такое группировка, фильтрация, JOIN-ы?

Категория: Технологии

### Краткий ответ

**Группировка, фильтрация, JOIN-ы**

- **Группировка** (GROUP BY) — объединение строк по значению столбцов для агрегирования (SUM, COUNT)
- Используется для **подсчёта**, **средних значений** и других агрегатов по категориям
- **Фильтрация** (WHERE, HAVING) — отбор строк по условию
- WHERE фильтрует до группировки, HAVING — после агрегирования
- **JOIN** — объединение таблиц по ключам
- JOIN виды: INNER (пересечение), LEFT/RIGHT (включение всех из одной таблицы), FULL (включение всех из обеих)
- JOIN-ы нужны для сложных запросов с данными из разных таблиц, обеспечивая связь и целостность данных

Итого: группировка агрегирует данные, фильтрация отбирает нужные строки, JOIN-ы объединяют таблицы по ключам для комплексного анализа.

### Подробный ответ
#### Основной ответ
**Группировка** (GROUP BY), **фильтрация** (WHERE/HAVING) и **JOIN-ы** — это фундаментальные операции в реляционных базах данных для организации, выборки и объединения данных.

**Группировка** используется для агрегирования строк из таблицы по одному или нескольким столбцам, позволяя вычислять суммирование, подсчёт, среднее и другие агрегатные функции по каждой группе.

**Фильтрация** — это процесс отбора только тех записей, которые удовлетворяют определённому условию. Для базовых выборок применяется `WHERE`, а для условия по сгруппированным данным — `HAVING`.

**JOIN-ы** позволяют объединять строки из двух или более таблиц на основании логического условия, часто по ключевым полям, например, `INNER JOIN`, `LEFT JOIN`, `RIGHT JOIN` и `FULL JOIN`.

#### Ключевые моменты
- **GROUP BY** обрабатывает данные для агрегатных функций, например, `SUM()` или `COUNT()`, группируя по ключам, и всегда используется после основной фильтрации `WHERE`.
- **WHERE** фильтрует строки до группировки, а **HAVING** — после группировки, для агрегатных данных.
- **JOIN-ы** бывают разных типов, и выбор зависит от задачи: `INNER JOIN` возвращает только совпадающие записи, `LEFT JOIN` — все строки из левой таблицы с сопоставлением справа, заполняя NULL, если нет совпадений.
- Использование **индексов** на внешних ключах критично для производительности JOIN-ов, особенно в больших БД.

#### Практический контекст
Типично в проектах сначала используют `WHERE` для ограничения исходного набора данных, затем `GROUP BY` для подсчёта метрик по категориям, и через `JOIN` объединяют с другими таблицами — например, связывают заказы с клиентами или товарами. В PostgreSQL 14+ и MySQL 8+ оптимизации JOIN-ов достигаются через статистику и индексные структуры, что важно для работы с OLAP-системами и web-приложениями с высокими нагрузками.
