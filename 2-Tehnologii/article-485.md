# В чем разница между интерфейсом (interface) и типом (type) в TypeScript?

Категория: Технологии

### Краткий ответ

**Разница между interface и type в TypeScript**

- Оба описывают **структуры типов** и работают на этапе компиляции
- **interface** предназначен для описания формата объектов и классов, поддерживает **расширение (extends, implements)**
- **type** более универсален: описывает **примитивы, объединения, кортежи, алиасы типов**
- interface можно объявлять **несколько раз** (декларативное слияние), type нельзя
- type позволяет использовать **операторы объединения (|), пересечения (&)**
- interface лучше подходит для **объектно-ориентированного** стиля и API
- type удобен для **композиции типов и сложных выражений**

Итог: **interface** — для описания объектных контрактов с расширяемостью, **type** — универсальный инструмент для более сложных и гибких типов.

### Подробный ответ
#### Основной ответ
В **TypeScript** ключевое различие между **interface** и **type** заключается в их назначении и возможностях: **interface** предназначен для описания контракта объектов, классов и структур данных, а **type** — это более универсальный инструмент для задания алиасов типов, включая примитивы, объединения, пересечения, кортежи и более сложные композиции.

#### Ключевые моменты
- **Расширение и декларативное слияние**: интерфейсы поддерживают декларативное слияние (declaration merging), то есть несколько описаний одного интерфейса компилируются в один, что полезно при расширении внешних типов. Типы такого поведения не имеют.
- **Возможности type**: с помощью type можно создавать объединённые типы (`union`), пересечения (`intersection`), использовать кортежи, условные типы и другие продвинутые шаблоны типизации, которые интерфейсы не поддерживают.
- **Сравнительная читаемость и семантика**: interface лучше подходит для описания структуры классов и публичных API, делая контракт более явно, а type — для более сложных или вычисляемых типов.

#### Практический контекст
В современных проектах часто используют **interface** для описания внешних API и объектов, где важна расширяемость (например, типы props в React-компонентах), а **type** — для создания комплексных типов, объединений, опциональных или условных логик. В React 18 и выше стала популярна комбинация обоих — interface для деклараций и type для комбинирования типов.
