# Как в query-параметре передать знак амперсанда (&)?

**Как в query-параметре передать знак амперсанда (&)?**

- веб: строка URL и параметры
- амперсанд & разделяет параметры в URL
- чтобы вставить & в значение, нужно **экранировать** ( encode )
- используют **URL-кодировку**: `&` → `%26`
- encodeURIComponent или аналог для кодирования
- без кодирования параметры разобьются неправильно
- важно для корректной передачи данных через GET-запросы

# Подробный ответ
## Основной ответ
Чтобы передать знак амперсанда (`&`) в query-параметре URL, его необходимо **произвести URL-энкодинг** — заменить символ на его процентное представление `%26`. Это связано с тем, что амперсанд используется как разделитель параметров в URL-запросе, и без экранирования он будет восприниматься как граница между ключами или значениями, что исказит смысл параметров.

## Ключевые моменты
- **URL-энкодинг**: специальные символы, включая `&`, заменяются на `%` + их код в шестнадцатиричном формате ASCII, здесь `%26` — код амперсанда.
- **Автоматизация**: в современных языках и библиотеках есть встроенные функции для безопасного кодирования query-параметров, например, `encodeURIComponent` в JavaScript или `urllib.parse.quote` в Python.
- **Безопасность**: неправильная передача `&` без кодирования может привести к ошибкам парсинга данных на сервере, а также к уязвимостям в веб-приложениях (например, нарушению логики обработки параметров).

## Практический контекст
В реальных проектах при формировании URL с динамическими параметрами всегда используют URL-энкодинг. Например, если параметр содержит список или спецсимволы, включая амперсанд, его значение перед отправкой на сервер кодируют через стандартные функции, чтобы гарантировать корректную передачу. Это важно, например, в REST API вызовах, где query-параметры влияют на фильтрацию данных.
