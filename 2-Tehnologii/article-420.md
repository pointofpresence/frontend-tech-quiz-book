# Pick, Omit, Partial в TS

**Pick, Omit, Partial в TypeScript**

- утилиты работы с типами объектов  
- **Partial<T>** делает все свойства необязательными  
- **Pick<T, K>** создает тип с выбранными ключами K из T  
- **Omit<T, K>** исключает ключи K из типа T  
- помогают формировать новые типы из существующих без дублирования  
- применяются для гибкой типизации параметров и пропсов  
- упрощают поддержку и расширение интерфейсов в больших кодовых базах

# Подробный ответ
## Основной ответ
В **TypeScript** утилиты **Pick**, **Omit** и **Partial** позволяют строить новые типы на основе существующих, повышая выразительность и переиспользуемость кода.  

- **Pick<T, K>** создает новый тип, выбирая из типа `T` только свойства с ключами из множества `K`. Это полезно, когда нужно ограничить интерфейс конкретным подмножеством полей.  
- **Omit<T, K>** — зеркальная утилита, исключающая из типа `T` свойства с ключами `K`, создавая тип с "оставшимися" полями. Отлично подходит для исключения чувствительных или неактуальных полей.  
- **Partial<T>** преобразует все свойства типа `T` в опциональные, что удобно для описания объектов частичной инициализации, например, при обновлении данных.

## Ключевые моменты
- **Pick и Omit** работают с ключами свойств — при этом ключи задаются как литеральные типы строк или union-тип строк (например, `'id' | 'name'`).  
- **Partial** рекурсивно не делает вложенные свойства опциональными — для глубокого Partial есть отдельные решения.  
- В TypeScript 3.5+ реализация Omit входит в стандартную библиотеку, и она построена через пересечение типов и `Pick`, что обеспечивает производительность и безопасность типов.  

## Практический контекст
Например, при работе с REST API можно использовать `Partial<User>` для описания DTO обновления, где принимаются неполные поля пользователя. Если нужно сформировать возвращаемый тип с ограниченным набором полей, используют `Pick<User, 'id' | 'name'>`. Чтобы убрать поля вроде `password` из интерфейса пользователя перед передачей клиенту – `Omit<User, 'password'>`. Такое типобезопасное управление интерфейсами снижает ошибки и повышает качество кода в реальных проектах.
