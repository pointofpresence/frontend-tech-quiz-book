# В чём разница между errors.Is, errors.As и errors.Join?

Категория: Технологии

### Краткий ответ

**Разница между errors.Is, errors.As и errors.Join**

- Go 1.13+ улучшения для работы с ошибками  
- **errors.Is**: проверяет, содержит ли ошибка целевую ошибку (сравнение через `Is` или цепочку оберток)  
- **errors.As**: пытается привести ошибку или вложенную ошибку к конкретному типу (для извлечения дополнительной информации)  
- **errors.Join**: объединяет несколько ошибок в одну, создавая цепочку или множество ошибок  
- `errors.Is` — для проверки эквивалентности ошибок  
- `errors.As` — для доступа к конкретным ошибочным типам  
- `errors.Join` — для агрегирования нескольких ошибок в одну для удобства передачи/возврата  
- Используются вместе для удобного и безопасного управления сложными ошибками в Go-коде

### Подробный ответ
#### Основной ответ
В Go, начиная с версии 1.13, добавлены новые функции для работы с ошибками: **errors.Is**, **errors.As** и **errors.Join** — каждая решает разные задачи, связанные с обработкой и проверкой ошибок.

- **errors.Is** проверяет, совпадает ли ошибка или её цепочка с целевой ошибкой (с помощью сравнения интерфейса `error` или `Is` метода).
- **errors.As** пытается привести ошибку в цепочке к конкретному типу и сохранить его в переданном указателе.
- **errors.Join** объединяет несколько ошибок в одну, возвращая объект, который содержит все эти ошибки и позволяет работать с ними как с единой ошибкой.

#### Ключевые моменты
- **errors.Is(err, target error) bool** — используется для проверки, что ошибка равна или оборачивается с помощью `fmt.Errorf("%w")`. Паттерн очень полезен, чтобы понять, произошла ли конкретная ошибка в цепочке, например, `sql.ErrNoRows`.
  
- **errors.As(err, target interface{}) bool** — позволяет определить, реализует ли какая-то ошибка в цепочке указанный тип (например, пользовательский тип ошибки). При успешном приведении помещает найденную ошибку в `target`. Это удобно, если надо работать с ошибкой конкретного типа, а не просто сравнивать по значению.
  
- **errors.Join(errs ...error) error** — конструирует ошибку, объединяющую несколько ошибок, при этом при использовании `errors.Is` и `errors.As` она действует как композитная: проверяет по всему набору. Особенно актуально, когда нужно вернуть результаты сразу нескольких ошибок, например, при одновременной работе с несколькими внешними сервисами.

#### Практический контекст
- В реальных проектах часто используют `errors.Is` для простой проверки ошибок из стандартной библиотеки или обёрнутых ошибок с помощью `%w`.
- `errors.As` подходит для логики, когда поведение зависит от точного типа ошибки — например, распознавание ошибок валидации или кастомных ошибок API.
- `errors.Join` распространено в ситуациях агрегации ошибок из параллельных операций или batch обработок (например, параллельный вызов нескольких запросов к разным сервисам). 

Этот набор значительно упрощает чистую и надежную работу с ошибками по сравнению с подходами на ранних версиях Go.
