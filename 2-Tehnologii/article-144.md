# Какой метод HTTP и какие параметры следует использовать для получения списка сущностей с возможностью фильтрации, пагинации и сортировки?

**HTTP метод и параметры для получения списка сущностей с фильтрацией, пагинацией и сортировкой**

- метод: **GET** — для получения данных без изменения ресурса
- параметры query string:
  - **filter** (или по полям) — для условий фильтрации (например, `?status=active`)
  - **page** и **limit (size)** — для пагинации, указывают номер страницы и размер выборки
  - **sort** — для сортировки, обычно в виде `?sort=field,-field` (по возрастанию/убыванию)
- URL должен оставаться **безопасным и идемпотентным**
- данные передаются в параметрах, а не в теле запроса
- важно корректно возвращать **общие метаданные** (total count, pages) в ответе для пагинации
- стандарты: часто используют OpenAPI, JSON API или GraphQL для унификации
- обеспечивает **масштабируемый**, легко кэшируемый и понятный API

Такой подход — стандарт REST для удобного и эффективного получения коллекций с контролем вывода.

# Подробный ответ
## Основной ответ
Для получения списка сущностей с возможностью **фильтрации, пагинации и сортировки** обычно используется HTTP-метод **GET**, так как этот метод предназначен для запроса данных без изменения состояния сервера. Все параметры запроса передаются через **query-параметры URL**, что соответствует RESTful практикам и улучшает кэшируемость.

## Ключевые моменты
- **Фильтрация** обычно осуществляется через query-параметры, например `?status=active&category=books`. Часто используют именованные параметры, отражающие поля сущности.
- **Пагинация** реализуется через параметры `page` и `limit` или `offset` и `limit` (например, `?page=2&limit=20`), чтобы ограничить количество результата и управлять страницами.
- **Сортировка** задается через параметр, например `sort`, где можно указать поле и направление `?sort=createdAt,-name` (минус перед полем — сортировка по убыванию).
- Важно придерживаться консистентного формата параметров для удобства клиентов API и легкой интеграции.

## Практический контекст
В современных API (например, в REST API, реализованном с использованием **Express** или **Spring**) в React-приложениях при запросах на сервер часто используют метод GET с query-параметрами для списка ресурсов. Это улучшает производительность благодаря кэшированию GET-запросов на уровне CDN или браузера и упрощает дебаг и логирование.
