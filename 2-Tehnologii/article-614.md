# Как обновить только один TextView в списке без полного ребинда всего элемента?

Категория: Технологии

### Краткий ответ

**Как обновить один TextView в списке без полного ребинда?**

- Область: Android, RecyclerView/ListView  
- Использовать вызов **notifyItemChanged(position, payload)**  
- Передать конкретный **payload** (например, ключ обновления TextView)  
- В адаптере переопределить **onBindViewHolder(holder, position, payloads)**  
- Проверить payload и обновить только **нужный TextView** без полной перерисовки элемента  
- Сокращает нагрузку на UI, повышает производительность  
- Применяется для частичного обновления данных в элементах списка  

Так обновляется только изменённое поле, без вызова полного bind-процесса и лишнего перерасчёта всего элемента.

### Подробный ответ
#### Основной ответ
Чтобы обновить только один **TextView** в элементе списка (например, в `RecyclerView`), без полного ребинда всего элемента, нужно выбрать подход, отличающийся от стандартного вызова `notifyItemChanged(position)`, который приводит к пересозданию всего view holder. Вместо этого удобно использовать **частичное обновление с payloads** в `RecyclerView.Adapter`, где можно передать параметры, указывающие, что изменился именно один конкретный subview.

#### Ключевые моменты
- В переопределенном методе `onBindViewHolder(holder, position, List<Object> payloads)` проверяют содержимое `payloads`: если список непустой, делают точечное обновление, например, только для `TextView.setText(...)`. Если пустой — делают полное биндинг.
- Вызов обновления происходит через `adapter.notifyItemChanged(position, payload)`, где `payload` — кастомный объект или строка, сигнализирующая о типе изменения (например, `"text_update"`). Это позволяет избежать полной перерисовки.
- Такой подход снижает нагрузку на рендеринг, уменьшает мерцание и повышает производительность при больших списках и frequent updates.

#### Практический контекст
В реальных Android-приложениях с **RecyclerView 1.2+** частичное обновление с payloads — стандартный подход для оптимизации UI. Например, при обновлении лайков или статуса сообщения в чате меняется лишь одно поле, не трогая остальные. Это также удобно сочетать с библиотеками типа `DiffUtil`, которые могут указывать payload для fine-grained обновлений.
