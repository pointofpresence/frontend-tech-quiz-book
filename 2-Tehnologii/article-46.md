# Объясните разницу между LEFT JOIN и RIGHT JOIN.

**Разница между LEFT JOIN и RIGHT JOIN**  
- SQL операции объединения таблиц  
- **LEFT JOIN**: сохраняет все строки из левой таблицы  
- Подключает совпадающие строки из правой таблицы  
- Если совпадений нет — в правых полях будут NULL  
- **RIGHT JOIN**: зеркальный оператор, сохраняет все строки правой таблицы  
- Подключает совпадающие строки из левой таблицы  
- Используется для разных требований к выборке данных  
- Практически, разница в том, какую таблицу считают "главной" для сохранения всех записей  

Итог: выбор зависит от того, данные какой таблицы нужно сохранить полностью при объединении.

# Подробный ответ
## Основной ответ
**LEFT JOIN** и **RIGHT JOIN** — это типы внешних соединений (outer joins) в SQL, которые возвращают строки из двух таблиц с сохранением всех записей из одной из них. **LEFT JOIN** возвращает все строки из левой таблицы и совпадающие строки из правой, а **RIGHT JOIN** — наоборот, все строки из правой таблицы и совпадающие из левой.

## Ключевые моменты
- В **LEFT JOIN** «левая» таблица — основная, ее записи всегда присутствуют в результате, даже если в правой таблице нет соответствий (тогда правые поля — NULL).
- В **RIGHT JOIN** аналогично, но основная таблица — правая; она гарантированно полностью в результатах.
- Практически **RIGHT JOIN** можно заменить на **LEFT JOIN**, поменяв местами таблицы. В больших и сложных запросах **LEFT JOIN** встречается гораздо чаще из-за большей читаемости и предсказуемости.

## Практический контекст
Например, если нужно получить список всех пользователей и их заказы, даже если у пользователя заказов нет, выбираем **LEFT JOIN** с таблицей пользователей слева. Если нужно показать все заказы и тех пользователей, кто их сделал, включая заказы без пользователя (в редких случаях), подойдет **RIGHT JOIN**. Обычно **LEFT JOIN** — стандартный выбор, а **RIGHT JOIN** применяют реже, это больше вопрос стиля и удобства чтения запроса.
