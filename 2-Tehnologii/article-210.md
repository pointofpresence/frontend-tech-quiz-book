# Есть ли generics в PHP?

Категория: Технологии

### Краткий ответ

**Есть ли generics в PHP?**

- PHP не поддерживает **настоящие generics** (как в Java, C#)
- Современные версии (8.x) имеют **дженерики через аннотации** в PHPDoc
- Статический анализ (PHPStan, Psalm) использует эти аннотации для проверки типов
- Генерики реализуются **символически**, без поддержки на уровне интерпретатора
- Используются для повышения **типобезопасности** и автодополнения IDE
- Полноценные generics ожидаются в будущих версиях PHP
- Практика: пишут шаблонный код с аннотациями для отлова ошибок на этапе разработки

### Подробный ответ
#### Основной ответ
В классическом понимании, **generics (дженерики)**, как в языках Java или C#, в PHP отсутствуют. На момент PHP 8.2 и даже в последних версиях языка, нет встроенной поддержки параметризированных типов, которые позволяли бы объявлять универсальные классы или функции с заранее неизвестным типом. Однако PHP реализует типизацию через **PHPDoc-аннотации** и **типы в сигнатурах методов**, что позволяет описывать типы параметров и возвращаемых значений, но без строгой проверки параметров типа в рантайме.

#### Ключевые моменты
- В PHP типа **generics** официально нет, но с помощью **PHPDoc** можно эмулировать generic-подход, указывая параметры типа (`@template`, `@param T`, `@return T`), что помогает IDE и статическому анализу (например, PhpStan, Psalm).
- **Типизация в сигнатурах** (например, `function foo(array $items): array`) усилилась в последних версиях (PHP 7+), но параметров типа это не заменяет.
- Существуют библиотеки и фреймворки (например, `doctrine/collections`), которые используют PHPDoc для имитации generics, но это чисто статический инструмент без проверки на уровне самого языка.
- PHP 8.2 добавил ограниченные возможности для **readonly properties** и усиленной типизации, но generics остаются за пределами языка.

#### Практический контекст
На практике generics в PHP эмулируются, чтобы повысить качество кода и улучшить автодополнение в IDE, а также статическую проверку, особенно в больших проектах. Например, в Symfony или Laravel вы встретите обширные аннотации, которые помогают понять, какие типы ожидаются. В то же время на runtime PHP это никак не влияет – ошибки типов generics не отлавливаются, что является trade-off между динамической природой PHP и строгой типизацией.

Таким образом, несмотря на отсутствие native generics, опытные PHP-разработчики применяют сочетание аннотаций и строгой типизации для достижения похожих целей.
