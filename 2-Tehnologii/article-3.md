# Как обычно реализуются пагинация и фильтрация данных в API?

Категория: Технологии

### Краткий ответ

**Пагинация и фильтрация в API**

- REST API и GraphQL — базовые подходы  
- Пагинация:  
  - параметры **limit и offset** (или page и per_page)  
  - cursor-based для производительности на больших данных  
- Фильтрация:  
  - query-параметры (фильтр по полям)  
  - комбинирование через операторы (eq, gt, lt, in)  
- Обработка на стороне БД для оптимальности  
- Возвращать метаинформацию: total, pages, current_page  
- Это уменьшает нагрузку, улучшает UX и масштабируемость  
- Используется во всех крупных API (Stripe, GitHub, Twitter)

### Подробный ответ
#### Основной ответ
Пагинация и фильтрация — классические подходы для управления большим объемом данных в API, позволяющие клиенту получать данные порционно и выбирать только нужные записи. Обычно пагинация реализуется с помощью параметров запроса, таких как `page` и `limit` или через механизмы cursor-based пагинации для повышения производительности. Фильтрация часто строится на базе query-параметров, задающих условия по одному или нескольким полям, например `status=active` или `created_after=2023-01-01`.

#### Ключевые моменты
- **Типы пагинации**:  
  - *Offset-based* (с указанием смещения и лимита) — прост и понятен, но при больших смещениях производительность падает (например, `?page=10&limit=20`).  
  - *Cursor-based* (использует уникальный идентификатор или временную метку как точку отсчёта) — более эффективен и масштабируем, особенно для изменяющихся данных, например `?cursor=abc123&limit=20`.
- **Фильтрация через query-параметры** обычно построена по принципу “ключ-значение”, где API позволяет комбинировать параметры с операторами (`eq`, `gt`, `in`, `like`). Часто применяются стандартные соглашения, например в REST или GraphQL.
- **Безопасность и производительность**: фильтрацию нужно валидировать и ограничивать, чтобы избежать SQL-инъекций и слишком тяжелых запросов. Индексация по полям, которые часто фильтруются или пагинируются, существенно влияет на скорость обработки.

#### Практический контекст
В REST API стандартный подход — использовать `GET /items?page=2&limit=50&status=active`. В GraphQL фильтрация и пагинация обычно реализованы через аргументы `filter` и `first/after` (cursor). Во многих крупных системах (Facebook, Twitter API v2+) курсорная пагинация — выбор по умолчанию для быстрого и консистентного скроллинга больших наборов данных. Для мониторинга можно использовать Prometheus, чтобы отслеживать влияние сложных фильтров на время ответа и оптимизировать индексы.
