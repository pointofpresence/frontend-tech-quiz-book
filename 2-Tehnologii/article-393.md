# Какие есть обменники в RabbitMQ

Категория: Технологии

### Краткий ответ

**Какие есть обменники в RabbitMQ**

- ключевой элемент **маршрутизации** сообщений  
- **direct**: маршрутизирует по **точному совпадению** routing key  
- **topic**: маршрутизация по шаблонам с **подстановочными знаками** (*, #)  
- **fanout**: рассылает сообщения на **все очереди** без фильтрации  
- **headers**: маршрутизация по **заголовкам сообщений** вместо routing key  
- гибкие схемы для разных сценариев доставки и фильтрации  
- широко используется в микросервисах и event-driven архитектурах

### Подробный ответ
#### Основной ответ
В **RabbitMQ** существуют несколько типов **exchange** (обменников), которые определяют, как сообщения маршрутизируются к очередям. Основные типы обменников — **direct**, **topic**, **fanout** и **headers**. Каждый из них решает разные задачи маршрутизации и подходит для различных сценариев взаимодействия между продюсерами и консьюмерами.

#### Ключевые моменты
- **Direct exchange**: сообщения попадают в очередь с точно совпадающим routing key. Используется для точечного, однозначного маршрута. Например, логирование по уровню "error", "info".
- **Topic exchange**: маршрутизация на основе шаблонов routing key с поддержкой wildcard (`*` для одного слова, `#` — для нескольких). Идеально для сложных маршрутов, например, `"logs.*.critical"`.
- **Fanout exchange**: отправляет сообщение во все привязанные к нему очереди, игнорируя routing key. Подходит для широковещательных рассылок, например, real-time уведомлений.
- **Headers exchange**: маршрутизация происходит по совпадению значений заголовков сообщения, а не routing key. Встречается реже, но полезен, когда надо маршрутизировать по нескольким критериям одновременно.

#### Практический контекст
В реальных проектах часто комбинируют **topic** и **direct** обменники для гибкой маршрутизации. **Fanout** полезен для broadcast-сообщений, а **headers** — при сложных бизнес-правилах маршрутизации, где ключ непригоден. RabbitMQ версии 3.8+ поддерживает эти типы стандартно, что позволяет строить как простые, так и очень гибкие системы обмена сообщениями с гарантированной доставкой и высокой отказоустойчивостью.
