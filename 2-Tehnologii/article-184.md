# Что происходит при вызове методов (GetType, ToString) на struct?

Категория: Технологии

### Краткий ответ

**Что происходит при вызове методов (GetType, ToString) на struct?**

- struct — **значимый тип** в .NET  
- методы GetType и ToString **унаследованы от System.ValueType → Object**  
- GetType **возвращает runtime-тип** через механизмы рефлексии  
- ToString по умолчанию возвращает имя типа (если не переопределён)  
- вызов методов вызывает **боксинг** (упаковку struct в объект)  
- упаковка делает struct объектом на куче для вызова виртуальных методов  
- важно учитывать бокcинг из-за влияния на **производительность** и память

### Подробный ответ
#### Основной ответ
При вызове методов **GetType()** и **ToString()** на **структуре (struct)** в C# происходит следующее: поскольку структуры наследуют от **System.ValueType**, который в свою очередь наследуется от **object**, эти методы вызываются из базового класса **object** или могут быть переопределены в структуре. Для **GetType()** происходит **боксинг** — структура оборачивается в объект, чтобы получить системный тип. Для **ToString()**, если он не переопределён в структуре, вызывается реализация из **ValueType.ToString()**, которая возвращает полное имя типа, но без отражения полей.

#### Ключевые моменты
- **Boxing при вызове GetType():** структура копируется в кучу как объект, это — затратная операция по времени и памяти.
- **ToString() зачастую не требует бокса:** если структура переопределяет ToString(), вызывается кастомная версия без упаковки; если нет — вызов идет в ValueType.ToString(), который создает строковое представление типа.
- **Разница с классами:** классы — ссылочные типы, вызовы методов сразу происходят без упаковки, что дешевле по производительности.

#### Практический контекст
В высокопроизводительном коде с большим числом вызовов GetType() на структурах стоит избегать частого бокса — лучше кэшировать тип или явно работать с typeof(StructName). В ToString() часто рекомендуют переопределять метод в структурах для читабельного вывода без лишних накладных расходов.
