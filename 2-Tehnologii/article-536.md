# Какими способами можно параллельно запустить две задачи в асинхронном коде?

Категория: Технологии

### Краткий ответ

**Параллельный запуск задач в асинхронном коде**

- асинхронность в JS и Python (Node.js, asyncio)
- Promise.all / asyncio.gather — запускают задачи параллельно, ждут завершения всех
- запуск задач без await — инициируют, запускаются одновременно
- use setTimeout/setImmediate для асинхронного отложенного запуска (спец. случаи)
- Web Workers / ThreadPool (Node.js Worker Threads) — параллелизм на уровне потоков (для CPU-работ)
- Fork/Spawn процессов для изоляции и масштабирования задач
- практическое: для IO-зависимых задач используется Promise.all/asyncio.gather, для CPU — воркеры

Таким образом, главное — инициировать задачи одновременно и управлять ожиданием их результатов для эффективного параллелизма.

### Подробный ответ
#### Основной ответ
В асинхронном коде запустить две задачи параллельно можно несколькими способами, ориентируясь на возможности языка и используемого фреймворка. В контексте JavaScript и Python — двух популярных языков для асинхронного программирования — ключевыми подходами являются использование промисов (Promises) и async/await с методами для параллельного запуска.

#### Ключевые моменты
- **JavaScript (ES6+):**  
  - Метод `Promise.all([task1, task2])` запускает обе задачи параллельно и дождётся их завершения. Если одна из задач завершится с ошибкой, `Promise.all` сразу отклоняется.  
  - `Promise.allSettled` позволяет дождаться завершения обеих задач независимо от успеха или ошибки каждой.  
  - Альтернативно, можно запускать задачи без await, например:  
    ```js
    const p1 = task1();
    const p2 = task2();
    await p1;
    await p2;
    ```  
    — где оба промиса запускаются почти одновременно.
  
- **Python (asyncio):**  
  - `asyncio.gather(task1(), task2())` одновременно запускает несколько корутин, агрегируя результаты. Если одна из корутин падает с исключением, поведение можно контролировать через параметр `return_exceptions=True`.  
  - `asyncio.create_task()` создаёт отдельные задачи, которые сразу планируются к выполнению, позволяя запустить несколько тасков параллельно. После создания можно отдельно ждать их через `await`.  
  - В Python при всех этих подходах используется event loop, параллелизм достигается за счёт переключения контекста, что эффективно для I/O-bound задач.
  
- **Общий контекст:**  
  - Параллельное выполнение не всегда значит многопоточность — часто это конкурентное, асинхронное выполнение внутри одного потока. Для CPU-bound задач нужен другой подход (например, multiprocessing в Python или worker threads в Node.js).  
  - Важно учитывать обработку ошибок и время ожидания, чтобы одна "тяжёлая" задача не блокировала другую.

#### Практический контекст
В реальных проектах, например в Node.js (React 18+, Express) или Python с FastAPI, часто используют `Promise.all` и `asyncio.gather` для параллельных запросов к БД или внешним API, что существенно снижает общую задержку и повышает throughput. При этом обработка ошибок и таймаутов делается по-разному в зависимости от бизнес-логики.
