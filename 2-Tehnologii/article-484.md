# Что означает indirect enum в Swift?

**Что означает indirect enum в Swift?**

- Swift, перечисления (enum), программирование
- indirect позволяет **рекурсивные** case
- case хранит ссылку, а не значение
- используется для описания **составных** структур (деревья, списки)
- избавляет от бесконечной вложенности в памяти
- позволяет более **безопасно** работать с рекурсией
- практическое применение: структуры данных с вложенностью (например, AST, бинарные деревья)

# Подробный ответ
## Основной ответ
**Indirect enum** в Swift — это перечисление, у которого один или несколько кейсов (вариантов) помечены ключевым словом `indirect` для указания того, что связанные с этим кейсом значения должны храниться косвенно (через ссылку), а не напрямую внутри enum. Такая конструкция позволяет создавать рекурсивные структуры данных, которые сами себя содержат (например, деревья или списки), без ошибок компиляции из-за бесконечного вложения.

## Ключевые моменты
- По умолчанию enum в Swift — это value type с фиксированным размером, и рекурсивные enum создавать напрямую невозможно, так как размер будет неопределённым.
- Использование `indirect` разрешает эту проблему, заставляя компилятор хранить связанное значение в heap-области и держать внутри enum только ссылку на него.
- Есть два варианта применения: пометить отдельные кейсы как `indirect case` или весь enum целиком как `indirect enum`.
- Это важно для построения структур данных типа деревьев (BST), связанных списков, синтаксических деревьев парсеров.

## Практический контекст
Типичный пример — рекурсивное перечисление бинарного дерева:

```swift
indirect enum BinaryTree {
  case empty
  case node(value: Int, left: BinaryTree, right: BinaryTree)
}
```

Здесь без `indirect` компилятор выдаст ошибку из-за бесконечного вложения `BinaryTree` в себя. В реальных проектах `indirect enum` часто применяется в парсерах, компиляторах, реализации коллекций и любых местах, где нужна рекурсивная структура с value semantic.
