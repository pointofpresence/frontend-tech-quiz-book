# Что такое SSR и как его реализовать с React?

Категория: Технологии

### Краткий ответ

**Что такое SSR и как его реализовать с React?**

- SSR (Server-Side Rendering) — рендеринг React-компонентов на сервере  
- Позволяет отправлять готовый HTML клиенту, улучшая **SEO** и время первого рендера (TTFB)  
- Реализуется с помощью метода **ReactDOMServer.renderToString()** или **renderToNodeStream()**  
- Сервер рендерит React-элементы в строку HTML и вставляет её в шаблон страницы  
- После загрузки клиент "гидратирует" этот HTML через **ReactDOM.hydrate()** для интерактивности  
- Часто используется вместе с Node.js сервером (Express, Koa) или фреймворками типа Next.js  
- Next.js упрощает SSR, добавляя маршрутизацию и оптимизацию "из коробки"  
- Основные сложности: синхронизация состояния между сервером и клиентом, управление данными и асинхронностью

### Подробный ответ
#### Основной ответ
**SSR (Server-Side Rendering)** — это процесс рендеринга React-компонентов на сервере с генерацией готового HTML, который отправляется клиенту. Это улучшает SEO, уменьшает время до первого отображения (First Paint) и повышает производительность на слабых устройствах. В React SSR позволяет загружать страницу быстрее, а затем "гидратировать" её на клиенте для интерактивности.

#### Ключевые моменты
- React предоставляет функцию **ReactDOMServer.renderToString()** (или в React 18 — **renderToPipeableStream**) для генерации HTML из компонентов на сервере.
- После отрисовки сервер отправляет вместе с HTML необходимые данные и начальное состояние, чтобы клиентский React мог "гидратировать" (hydrate) этот HTML, обеспечив полноценное SPA-поведение.
- Для реализации SSR обычно используют Node.js сервер и framework типа **Next.js**, который абстрагирует большинство сложностей SSR с React, либо можно настроить собственный сервер с Express и Babel.
- Важное отличие SSR от CSR — на сервере нет доступа к browser-specific API (window, document), поэтому нужно делать защиту кода.
- SSR увеличивает нагрузку на сервер, требует управления кэшированием и асинхронной загрузкой данных до рендера.

#### Практический контекст
В крупных React-проектах (например, маркетплейсы или новостные сайты) SSR часто используют вместе с Next.js 13+, где автоматически рендерится и кэшируется первая страница, а также есть интеграция с клиентскими компонентами через React Server Components. Ручной SSR помогает понять базовые технологии и гибко настраивать сервер, например, для API-интеграции и кастомной логики.
