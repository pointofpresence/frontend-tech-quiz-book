# Как добраться к переменной окружения из Node.js

**Как добраться к переменной окружения из Node.js**

- Node.js использует объект **process.env** для доступа к переменным окружения  
- Переменные доступны как строки по ключу, например **process.env.VARIABLE_NAME**  
- Если переменная не задана, возвращается **undefined**  
- Значения из окружения всегда строки (нужно приводить типы вручную)  
- Используется для настройки конфигурации приложения (порт, ключи API)  
- Часто применяют с пакетами типа **dotenv** для загрузки из .env файлов  
- Удобно для разделения окружений: dev, prod, test  

Итого: доступ через **process.env**, ключ-значение, строковый тип, для конфигурации и безопасности.

# Подробный ответ
## Основной ответ
В **Node.js** переменные окружения доступны через глобальный объект `process.env`. Это объект, где ключи — имена переменных, а значения — их строковое содержимое. Чтобы получить значение переменной окружения, достаточно обратиться, например, так: `process.env.MY_VARIABLE`.

## Ключевые моменты
- **process.env** — это стандартный способ работы с переменными окружения в Node.js начиная с самых ранних версий и до последней LTS (включая Node.js 18+).
- Все значения переменных окружения передаются как строки, поэтому при необходимости нужно явно приводить тип (например, для чисел вызовом `Number(process.env.PORT)`) или обрабатывать отсутствующие переменные.
- Для удобства разработки часто применяют пакет **dotenv**, который загружает переменные из файла `.env` в `process.env`, что позволяет хранить конфигурацию локально и безопасно разделять окружения (development, production).

## Практический контекст
В большинстве real-world приложений доступ к переменным окружения необходим для настройки базы данных, API ключей, портов сервера и других параметров конфигурации. Например, при запуске Express сервера стандартно берут порт из `process.env.PORT || 3000`, что позволяет гибко конфигурировать приложение без изменения кода.
