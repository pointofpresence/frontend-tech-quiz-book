# Что принимает функция Depends() в FastAPI как параметр? Какой это объект с точки зрения Python?

Категория: Технологии

### Краткий ответ

**Что принимает функция Depends() в FastAPI как параметр? Какой это объект с точки зрения Python?**

- FastAPI: **Depends** — механизм **внедрения зависимостей**  
- Принимает: **вызываемую функцию или класс**  
- Тип параметра: объект с `__call__` (callable) — **callable** в Python  
- Позволяет FastAPI **автоматически решать зависимости** функций с помощью сигнатур  
- Используется для декларативного описания зависимостей в обработчиках запросов  
- Может принимать обычные функции, async функции, классы с методом `__call__`  
- Обеспечивает **инъекцию аргументов** при вызове эндпоинтов через introspection  

Итого: **Depends() принимает callable (вызываемый объект), который FastAPI вызывает для получения значения зависимости.**

### Подробный ответ
#### Основной ответ
В FastAPI функция `Depends()` принимает в качестве параметра **вызов callable-объекта** — обычно это другая функция или класс, предназначенные для внедрения зависимостей (dependency injection). С точки зрения Python, этот параметр — любой объект, который можно вызвать, то есть реализует протокол `__call__` и ведёт себя как функция.

#### Ключевые моменты
- В аргументах `Depends()` обычно передают **функцию или класс**, которые FastAPI при обработке запроса вызывает для разрешения зависимости.
- Объект с точки зрения Python — **callable**, это может быть обычная функция, метод, а иногда и класс (который создаёт экземпляр с помощью `__call__`).
- Благодаря использованию типизации и аннотаций FastAPI умеет вызывать такую функцию автоматически, передавая ей подходящие параметры (например, параметры запроса, body, или другие зависимости).
- `Depends()` — это не статический объект, а динамический механизм, который подставляет в маршрут вызов этой callable, создавая цепочку зависимостей на уровне runtime.

#### Практический контекст
В реальных проектах через `Depends()` внедряют сервисы, базы данных, авторизацию, session, или любые повторно используемые объекты. Например, функция `get_current_user` может быть callable с аргументами и логикой, а `Depends(get_current_user)` обеспечит её вызов и передачу результата в endpoint. Это удобный и декларативный способ управлять зависимостями без явного вызова в каждом маршруте.
