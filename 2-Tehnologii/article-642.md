# В чём разница между Application Context и Activity Context?

Категория: Технологии

### Краткий ответ

**Разница между Application Context и Activity Context**

- **Application Context** – контекст всего приложения, живет все время работы процесса, **не зависит** от UI
- **Activity Context** – контекст конкретной Activity, привязан к её жизненному циклу, содержит UI-связанные ресурсы
- Application Context безопасен для операций, не завязанных на UI (например, база, сервисы)
- Activity Context используют для UI-инфляции, создания диалогов, запуска Activity
- Использование Activity Context вне Activity приводит к утечкам памяти из-за зависимости от UI
- Применять Application Context при долгоживущих объектах, Activity Context – при работе с UI и жизненным циклом экранов
- Ошибки в выборе контекста приводят к **утечкам памяти** и сбоям в UI

Итог: **Application Context – широкая, без UI; Activity Context – UI-ориентированная, кратковременная**.

### Подробный ответ
#### Основной ответ
**Application Context** и **Activity Context** — это два типа контекстов в Android, которые предоставляют доступ к ресурсам и системным сервисам, но отличаются по области видимости и жизненному циклу. **Application Context** связан с глобальным состоянием приложения и живёт столько же, сколько и сам процесс, тогда как **Activity Context** привязан к конкретной Activity и существует только на время её жизненного цикла.

#### Ключевые моменты
- **Жизненный цикл**:  
  Application Context живёт постоянно, пока запущено приложение; Activity Context уничтожается при завершении Activity, поэтому использование Activity Context вне жизненного цикла Activity может привести к утечкам памяти.  
- **Область применения**:  
  Application Context подходит для операций, не зависящих от UI и жизненного цикла, например, старт службы, получение системных сервисов, создание singleton-объектов. Activity Context используется, когда нужна привязка к UI или работу с визуальными элементами (Inflater, Toast, диалоги), которые требуют UI-связанного контекста.  
- **Утечки памяти**:  
  Использование Activity Context с долгоживущими объектами (например, синглтонами или global listeners) может привести к утечкам памяти, так как Activity не сможет быть корректно уничтожена, если на неё есть ссылки через контекст. Application Context безопаснее в таких случаях.

#### Практический контекст
В реальных проектах часто **Application Context** используют для инициализации библиотек (например, Firebase, Room), логики, не связанной с интерфейсом, а **Activity Context** — при создании UI-компонентов (AlertDialog, LayoutInflater), чтобы корректно связать ресурсы с видимым экраном и избежать сбоев в отображении. В React Native или Flutter аналогично различные контексты жизненного цикла обеспечивают правильное управление ресурсами.
