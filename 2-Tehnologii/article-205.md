# Как передаются данные в функцию: по ссылке или по значению?

Категория: Технологии

### Краткий ответ

**Как передаются данные в функцию: по ссылке или по значению?**

- зависит от языка и типа данных  
- **по значению**: передаётся копия данных (примитивы в C, Java, Python)  
- **по ссылке**: передаётся адрес объекта (объекты в C++, Java, Python)  
- смешанный подход: примитивы по значению, объекты — по ссылке или указателю  
- изменение внутри функции при передаче по ссылке влияет на оригинал  
- передача по значению даёт безопасность и изоляцию, но затратнее по памяти  
- понимание модели передачи важно для управления состоянием и оптимизацией кода

### Подробный ответ
#### Основной ответ
В языках программирования передача данных в функцию может происходить либо **по значению**, либо **по ссылке**, в зависимости от конкретного языка и типа данных. **Передача по значению** означает, что в функцию передаётся копия оригинальных данных, и изменения внутри функции не влияют на оригинал. **Передача по ссылке** — функция получает ссылку (адрес) на исходные данные, из-за чего изменения внутри функции отражаются на оригинале.

#### Ключевые моменты
- В **C/C++** для базовых типов (int, float) по умолчанию используется передача по значению, но передача по ссылке достигается через указатели или ссылки (`&`). Для сложных структур данных, чтобы избежать копирования, предпочтительна передача по ссылке.
- В **Java** всё передаётся по значению, но для объектов передаётся копия ссылки (reference), что часто путают с передачей по ссылке, хотя на самом деле копируется только ссылка, а не сам объект.
- В **Python** для всех объектов передача данных осуществляется по ссылке на объект, но при этом для неизменяемых типов (int, str, tuple) поведение выглядит как передача по значению, потому что изменяемость самой ссылки отсутствует.
- Различия влияют на **эффективность и поведение** программы: передача по значению безопаснее (оригинал не изменится), но может быть дорогой по памяти; по ссылке — быстрее, но нужно аккуратно управлять изменениями данных.

#### Практический контекст
В проектах на **C++17 и новее** часто используют **rvalue-ссылки и move-семантику**, чтобы оптимизировать передачу больших объектов. В **Java** важно понимать, что изменение полей объекта внутри функции затрагивает оригинал, а переназначение ссылки — нет. В **Python 3** хорошее понимание ссылочной модели помогает предсказать поведение mutable и immutable объектов при вызове функций.
