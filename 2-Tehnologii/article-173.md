# Где встречается боксинг в Unity?

Категория: Технологии

### Краткий ответ

**Где встречается боксинг в Unity?**  
- Unity использует C# с управляемой средой CLR  
- Боксинг возникает при преобразовании **значимых** типов (int, struct) в **ссылочные** (object, интерфейс)  
- Часто встречается при работе с коллекциями, например, при добавлении значимых типов в ArrayList или non-generic List  
- Возникает при использовании интерфейсов, если значение типа реализует их (упаковка для вызова методов интерфейса)  
- Боксинг влияет на **производительность**: лишнее выделение памяти и сборка мусора  
- Оптимизация — использовать обобщённые коллекции (List<T>), избегать interface boxing и кастов  
- В Unity важно контролировать боксинг для оптимизации **памяти и FPS** в играх

### Подробный ответ
#### Основной ответ
В **Unity** боксинг (boxing) встречается, когда значение типа **Value Type** (например, `int`, `float`, `struct`) неявно преобразуется в **reference type** `object` или интерфейс. Это происходит, т.к. value type хранится в стеке, а reference type — в куче, и для упаковки значения в объект систему Unity приходится выделять память и копировать данные. Такой процесс называется боксингом и влияет на производительность из-за аллокаций и работы сборщика мусора.

#### Ключевые моменты
- Боксинг часто возникает, например, при **использовании коллекций non-generic** (например, `ArrayList`), где `int` или `struct` вызывают упаковку в объект.
- Также типичные случаи — вызовы методов, принимающих параметры типа `object` или интерфейсов, если передаётся value type.
- В Unity, при частом боксинге, например в `Update` или в циклах, может быстро расти нагрузка на GC (garbage collector), что снижает плавность игры.

#### Практический контекст
В современном Unity рекомендуется использовать **generic коллекции** (List<T>, Dictionary<TKey,TValue>) чтобы избежать боксинга при работе с типами-значениями. Часто struct-ы для игровых данных должны быть легковесными и не передаваться как object напрямую. В профилировщике Unity можно увидеть аллокации, связанные с боксингом — оптимизировать стоит именно эти участки для улучшения производительности и снижения пропусков кадров.
