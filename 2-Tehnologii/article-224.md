# С каким фактором расширяется List при превышении capacity?

Категория: Технологии

### Краткий ответ

**Фактор расширения List при превышении capacity**

- Контекст: динамический массив (List в .NET, Java и др.)
- При переполнении capacity List увеличивает размер внутреннего массива
- Обычно размер увеличивается в 2 раза (factor = 2)
- Удвоение снижает количество аллокаций и копирований данных
- Расширение дорого по времени из-за выделения памяти и копирования данных
- Такой механизм обеспечивает амортизированную сложность вставки O(1)
- Практическое значение: быстрое добавление элементов с минимальными затратами realloc

---

**Подробности:**

В реализации динамических массивов, как List<T> в .NET или ArrayList в Java, при добавлении элемента, если текущий внутренний буфер достиг размера capacity, происходит расширение.

Алгоритм обычно следующий: выделяется новый массив вдвое большего размера, все существующие элементы копируются в новый массив, ссылочная переменная меняется на новый массив.

Удвоение capacity — баланс между затратами на перекопирование и частотой расширений. Меньший фактор приведет к слишком частым переаллокациям, больший — к избыточному выделению памяти.

Это расширение обеспечивает амортизированное время вставки O(1), несмотря на дорогую операцию копирования.

Таким образом, фактор расширения — обычно 2, что оптимизирует производительность и память при росте списка.

### Подробный ответ
#### Основной ответ
В стандартной реализации **ArrayList** в Java при превышении текущей capacity происходит расширение внутреннего массива с помощью увеличения размера примерно на 50% от текущего. Это означает, что новый размер равен текущему размеру плюс половина от него, то есть примерно в 1.5 раза больше.

#### Ключевые моменты
- В Java 8 и выше, при вызове метода `ensureCapacity` или при автоматическом расширении внутреннего массива, размер увеличивается по формуле: `newCapacity = oldCapacity + (oldCapacity >> 1)`, что эквивалентно 1.5-кратному увеличению.
- Такой подход балансирует между частотой расширений (уменьшая их количество) и расходами по памяти (чтобы не выделять сразу слишком много лишнего места).
- В других языках и реализации коллекций фактор расширения может отличаться, например, в C# List<T> расширяется в 2 раза (удваивается), но в Java для ArrayList именно 1.5.

#### Практический контекст
Такое расширение минимизирует копирование большого объема данных слишком часто, что важно для производительности. Явно задавать capacity заранее рекомендуется для оптимизации памяти и избежания лишних копирований при массовом добавлении элементов.
