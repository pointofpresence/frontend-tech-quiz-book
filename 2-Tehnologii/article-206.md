# Что такое RoadRunner?

**Что такое RoadRunner?**

- PHP-процессор и сервер приложений  
- работает как **высокопроизводительный** HTTP-сервер с поддержкой PHP  
- запускает PHP приложения как **длительные процессы** (worker'ы), избегая перезапуска при каждом запросе  
- использует **Go** для управления воркерами и балансировки нагрузки  
- поддерживает WebSockets, GRPC и TCP  
- сокращает задержки за счёт **исключения cold start** PHP исполняемых процессов  
- применяется для высоконагруженных проектов и микросервисной архитектуры  

RoadRunner — альтернатива PHP-FPM для повышения производительности и масштабируемости PHP приложений.

# Подробный ответ
## Основной ответ
**RoadRunner** — это высокопроизводительный PHP application server и процесс-менеджер, который работает как daemon на базе Golang. Он позволяет запускать PHP-приложения без необходимости старта нового процесса PHP для каждого запроса, значительно снижая задержки и повышая throughput в сравнении с классическим подходом через веб-сервер и PHP-FPM.

## Ключевые моменты
- **Многоразовый цикл жизни процесса**: RoadRunner запускает PHP worker-разработчики, которые долго живут в памяти и обрабатывают запросы последовательно. Это уменьшает накладные расходы на инициализацию PHP и снижает задержку (~50-100ms против сотен миллисекунд).
- **Поддержка протоколов и middleware**: RoadRunner может работать с HTTP, gRPC, WebSocket и queue, что позволяет строить микросервисы и event-driven архитектуры на PHP 8+.
- **Интеграция с Go**: основной сервер написан на Go, что обеспечивает параллелизм, надежность и простоту масштабирования, при этом сами бизнес-логика и сервисы реализованы на PHP.

## Практический контекст
RoadRunner часто используют для высоконагруженных API и real-time систем, где критична низкая задержка и высокая пропускная способность. Он особенно полезен в проектах на Laravel (через интеграционные пакеты), Symfony и других современных фреймворках, позволяя снижать нагрузку на инфраструктуру и упрощать deployment.
