# Какие типы данных можно положить в Intent?

**Какие типы данных можно положить в Intent?**

- Intent — контейнер для передачи данных между компонентами Android  
- Поддерживает **примитивы**: int, long, float, double, boolean, char  
- Поддерживает **строки** и массивы примитивов  
- Поддерживает объекты, реализующие **Parcelable** (быстрая сериализация)  
- Поддерживает объекты, реализующие **Serializable** (медленнее Parcelable)  
- Поддерживает **Bundle** для вложенных структур  
- Используется для передачи данных в Activity, Service, BroadcastReceiver  

---  
Этот ответ демонстрирует знание основ работы Intent и передачи данных в Android, выделяя типы и особенности по производительности.

# Подробный ответ
## Основной ответ
В **Android Intent** можно положить разнообразные **типы данных через extras** — это ключ-значение пары, реализованные на базе `Bundle`. В `Intent` можно класть простые типы данных (примитивы), сложные объекты, а также коллекции и массивы, при этом поддерживая удобную сериализацию для передачи между компонентами.

## Ключевые моменты
- **Примитивы и обёртки**: `int`, `long`, `float`, `double`, `boolean` и их классы-обёртки (`Integer`, `Boolean` и т.д.)
- **Строки и массивы**: `String`, `CharSequence`, а также массивы этих типов (`String[]`, `int[]` и т.п.)
- **Parcelable и Serializable объекты**: для передачи сложных пользовательских объектов, реализующих интерфейсы `Parcelable` (предпочтительнее за счёт производительности) либо `Serializable` (медленнее, но проще реализовать)
- **Коллекции**: списки и массивы Parcelable или Serializable объектов (например, `ArrayList<Parcelable>`, `ArrayList<String>`)
- **Специальные типы**: `Bundle` — вложенные наборы данных, `IBinder` (с API 18+), `Size` и `SizeF` (с API 21), `SparseArray<Parcelable>` (оптимизирован для Android)

## Практический контекст
Типично в `Intent` кладут ID или ключи (примитивы), `String` с параметрами, а также объекты моделей (реализующие Parcelable) — чтобы отправить данные из одной Activity в другую. В современных приложениях часто используют `Parcelable`, так как он быстр и оптимизирован под Android, например, в React Native и Jetpack Compose он тоже активно применяется. Правильный выбор типа влияет на производительность и надёжность передачи данных между контекстами.
