# Что такое CORS

Категория: Технологии

### Краткий ответ

**Что такое CORS**  
- Безопасность веба, политика браузера  
- Cross-Origin Resource Sharing — механизм  
- Разрешает или запрещает запросы с других доменов  
- Управляется HTTP-заголовками (Access-Control-Allow-Origin и др.)  
- Предотвращает атаки типа «межсайтовый скриптинг» (XSS)  
- Важен для API и веб-приложений с фронтендом и бэкендом на разных доменах  
- Позволяет легально обмениваться ресурсами между сайтами, соблюдая безопасность

### Подробный ответ
#### Основной ответ  
**CORS (Cross-Origin Resource Sharing)** — это механизм безопасности в браузерах, который контролирует, каким образом веб-страницы могут запрашивать ресурсы с другого домена (происхождения). Он нужен для предотвращения небезопасных кросс-доменных запросов и защиты пользователя от атак типа CSRF и XSS.  

#### Ключевые моменты  
- По умолчанию браузеры реализуют политику **Same-Origin Policy**, которая запрещает скриптам получать доступ к ресурсам другого домена без явного разрешения. CORS позволяет серверу настроить специальные HTTP-заголовки (`Access-Control-Allow-Origin`), которые разрешают запросы с определённых источников.  
- CORS поддерживает несколько типов запросов: **simple requests** (GET, POST с простыми заголовками) проходят сразу, а сложные (например, с методами PUT, DELETE или нестандартными заголовками) требуют **preflight-запроса OPTIONS** для проверки разрешений.  
- Важен ответ сервера с корректными заголовками: помимо `Access-Control-Allow-Origin`, часто используются `Access-Control-Allow-Methods`, `Access-Control-Allow-Headers` и `Access-Control-Allow-Credentials` для управления методами, заголовками и отправкой cookies соответственно.  

#### Практический контекст  
В веб-разработке CORS особенно актуален, когда frontend и backend размещены на разных хостах (например, React 18 на localhost:3000 и API на localhost:8080). Без корректной настройки CORS браузер блокирует запросы, что требует либо конфигурации сервера (nginx, express.js и т.д.) для отдачи нужных заголовков, либо применения proxy для обхода ограничения.
