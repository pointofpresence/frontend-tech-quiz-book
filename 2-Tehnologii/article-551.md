# Можно ли передать функцию как параметр другой функции?

Категория: Технологии

### Краткий ответ

**Можно ли передать функцию как параметр другой функции?**

- основа функционального программирования  
- функции — объекты первого класса  
- передаются как параметры и возвращаются  
- позволяют создавать **коллбэки** и **хайер-ордер функции**  
- обеспечивают гибкость и повторное использование кода  
- применяются в JS, Python, Java (через функциональные интерфейсы)  
- ключ к асинхронному коду и обработке событий

### Подробный ответ
#### Основной ответ
Да, в большинстве современных языков программирования функция может быть передана как параметр другой функции. Это один из ключевых аспектов функционального программирования и позволяет создавать более гибкие, переиспользуемые и модульные компоненты. Такие функции называют **функциями высшего порядка**.

#### Ключевые моменты
- В языках как **JavaScript, Python, Java (начиная с версий с лямбда-выражениями, например Java 8+), C#**, функция — это объект, который можно передавать в качестве параметра, возвращать из других функций и сохранять в переменные.
- При передаче функции как параметра обычно передается либо **указатель/ссылка на функцию**, либо **лямбда/анонимная функция**.
- Такой подход упрощает реализацию **колбеков, событийных обработчиков, стратегий, декораторов и middleware**.
- В некоторых языках (например, в C/C++) функции передаются через указатели, что требует осторожности с типами и временем жизни данных.
  
#### Практический контекст
В React 18 часто передают колбеки в пропсы компонентов, чтобы управлять событиями. В Python функции высшего порядка — основа для функций вроде `map()`, `filter()` и `sorted()` с пользовательскими ключами сортировки. В Java Stream API методы принимают лямбды или ссылки на методы для обработки потоков данных. Такой паттерн помогает написать чистый, читаемый и эффективный код, повышающий повторное использование логики.
