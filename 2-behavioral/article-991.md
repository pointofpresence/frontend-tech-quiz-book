# Как перенести коммит из одной ветки в другую?

### Категории

Git, Version control, Behavioral, Career development, Interview, Testing

### Краткий ответ

**Как перенести коммит из одной ветки в другую?**

- Git операция для **переноса изменений**  
- Используется команда **git cherry-pick <commit>**  
- Применяет выбранный коммит на текущую ветку  
- Автоматически создаёт новый коммит с теми же изменениями  
- Подходит для избирательного копирования изменений без слияния всей ветки  
- При конфликтах — ручное разрешение, затем **git cherry-pick --continue**  
- Вариант: использовать **git format-patch** + **git am** для сложных случаев  
- Практическая ценность — интеграция фиксированных изменений в разные ветки (например, багфиксы из dev в release)

### Подробный ответ
#### Основной ответ
Для переноса коммита из одной ветки в другую в Git обычно используют команды **git cherry-pick**, **git rebase** или **git merge** в зависимости от контекста. Чаще всего применяется **cherry-pick**, который позволяет взять конкретный коммит (или набор коммитов) и применить их поверх другой ветки, сохранив историю изменений.

#### Ключевые моменты
- **git cherry-pick <commit-hash>** — команда извлекает изменения из указанного коммита и добавляет их поверх текущей ветки, создавая новый коммит с такими же изменениями, но с новым хешем. Это удобно, если нужно выборочно перенести отдельные коммиты.
- В случае нескольких коммитов можно указать диапазон — например, git cherry-pick A..B, или списком через пробел.
- Если коммит конфликтует с текущей веткой, cherry-pick остановится и предложит решить конфликты вручную, затем выполнить git cherry-pick --continue.
- Альтернативно, если необходимо перенести целую историю, можно использовать **git rebase** или даже **git merge**, но для избирательной переноса обычно cherry-pick подходит лучше.

#### Практический контекст
В реальных проектах cherry-pick применяют для быстрого переноса исправлений багов из веток разработки в релизные или хотфиксы, не внося все изменения целиком. Например, в GitLab под cherry-pick есть UI, а в командной строке это простой и безопасный способ вынести конкретный коммит в production-ветку.
