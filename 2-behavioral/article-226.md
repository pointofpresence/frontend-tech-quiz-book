# Что такое GraphQL?

### Категории

Behavioral, API, Testing, Backend, Database, Query language

### Краткий ответ

**Что такое GraphQL?**

- спецификация для API-запросов от Facebook (2015)  
- язык запросов, позволяющий клиенту определять структуру ответа  
- клиент запрашивает только нужные данные, без избыточности  
- единственная точка входа (endpoint), а не множество ресурсов, как в REST  
- поддерживает вложенные запросы и агрегирование данных из разных источников  
- типизированная схема описывает набор типов данных и возможных запросов  
- облегчает эволюцию API без версионирования  
- применяется в web и мобильных приложениях для оптимизации работы с данными  

GraphQL — эффективный способ гибкого и экономного взаимодействия клиента с сервером при работе с данными.

### Подробный ответ
#### Основной ответ
**GraphQL** — это язык запросов и среда исполнения для API, разработанный Facebook в 2015 году. В отличие от REST, где структура данных фиксирована на стороне сервера, в GraphQL клиент сам формулирует, какие именно данные ему нужны, что существенно сокращает избыточность и повышает эффективность передачи данных.

#### Ключевые моменты
- **Гибкость запросов**: клиент может запросить только нужные поля и вложенные объекты в одной операции, уменьшая количество запросов и общее потребление трафика.
- **Типизированная схема**: сервер описывает структуру данных и типы через схему GraphQL Schema Definition Language (SDL), что даёт сильную типизацию и удобные инструменты для валидации и авто-документирования API.
- **Единая точка входа**: весь API доступен через один endpoint, что упрощает архитектуру и управление версиями.
- **Поддержка подписок и мутаций**: кроме запросов на чтение, GraphQL поддерживает мутации для изменения данных и подписки для real-time обновлений, что часто используется в React 18+ и мобильных приложениях.

#### Практический контекст
GraphQL широко применяется в крупных frontend-проектах, например, в React или Vue, где часто нужны сложные, динамические данные с множества источников. Netflix, GitHub и Shopify используют GraphQL для уменьшения количества сетевых запросов и повышения UX за счёт адаптивных запросов. Для реализации часто применяют Apollo Server/Client либо Relay.
