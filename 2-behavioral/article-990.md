# Что такое чистая функция?

### Категории

Behavioral, Self development, Async, Functional programming, Pure function, Code review, Frontend, Live coding, Code quality, Testing

### Краткий ответ

**Что такое чистая функция?**

- концепт из функционального программирования  
- функция, результат которой зависит только от входных аргументов  
- не имеет побочных эффектов: не изменяет состояние, не взаимодействует с внешним миром  
- при одинаковых входных данных всегда возвращает одинаковый результат (диапазон значений строго предсказуем)  
- позволяет легче тестировать и отлаживать код  
- способствует оптимизациям, например, мемоизации и параллелизму  
- повышает читаемость и надежность программного кода

### Подробный ответ
#### Основной ответ
**Чистая функция** — это функция, которая при одинаковых входных данных всегда возвращает одинаковый результат и не вызывает никаких побочных эффектов, то есть не изменяет внешнее состояние и не зависит от него.

#### Ключевые моменты
- **Детерминированность**: вызов чистой функции с одинаковыми аргументами гарантированно вернёт одинаковое значение, что облегчает тестирование и отладку.
- **Отсутствие побочных эффектов**: функция не изменяет глобальные переменные, не взаимодействует с I/O (файлы, сеть), что делает её предсказуемой и безопасной для параллельного выполнения.
- **Идемпотентность по входам**: функциональное программирование опирается на чистые функции, чтобы добиться удобного рефакторинга, масштабируемости и кэширования результатов (memoization).

#### Практический контекст
Чистые функции часто применяются для реализации бизнес-логики, особенно при использовании таких библиотек как **Redux** в React 18+, где редьюсеры — это именно чистые функции. Это упрощает тестирование и позволяет безопасно использовать функциональные паттерны без скрытых зависимостей.
