# Почему использовать ::before/::after для 10k элементов эффективнее, чем добавлять DOM-узлы?

### Категории

Live coding, Performance, CSS pseudo-elements, Frontend, DOM manipulation, Optimization, Behavioral

### Краткий ответ

**Почему ::before/::after эффективнее, чем добавлять 10k DOM-узлов?**

- CSS псевдоэлементы (::before/::after) — создают виртуальный контент без реальных DOM-узлов  
- Нет увеличения DOM-дерева — меньше использования памяти и ресурсов браузера  
- Отрисовка эффективнее, так как браузер не выполняет парсинг, layout и repaint новых элементов  
- Меньше операций JavaScript при динамическом обновлении, снижается нагрузка на Event Loop  
- Улучшенная производительность при большом числе элементов (10k+) за счет оптимизации рендера  
- Поддержка стилей и трансформаций через CSS без дополнительных обработчиков событий  
- Идеально для декоративных целей, когда узлы не нужны в DOM для логики приложения  

Итог: использование ::before/::after предотвращает рост DOM и снижает нагрузку на парсер и рендер, повышая масштабируемость и производительность при большом количестве элементов.

### Подробный ответ
#### Основной ответ
Использование псевдоэлементов **::before** и **::after** для большого количества элементов (например, 10 000) значительно эффективнее с точки зрения производительности, чем добавление реальных DOM-узлов. Это связано с тем, что псевдоэлементы не создают новых узлов в DOM-дереве — они лишь расширяют визуальное представление существующих элементов, снижая нагрузку на браузер при рендеринге и память.

#### Ключевые моменты
- **Меньшая DOM-структура:** 10 000 реальных элементов создают огромную иерархию DOM, которая замедляет обход, рендеринг, пересчет стилей и реакции на события. Псевдоэлементы при этом остаются CSS-слоями, не увеличивая размер DOM.
- **Быстрее Repaint и Reflow:** Изменения в псевдоэлементах зачастую вызывают только перерисовку (repaint), без перестроения структуры (reflow), что экономит ресурсы браузера.
- **Оптимизация памяти:** Меньше объектов — меньшие затраты памяти, что особенно критично при работе с большими объемами элементов на странице.
- **Управляемость:** Стили псевдоэлементов настраиваются централизованно через CSS без необходимости манипулировать DOM через JavaScript, что упрощает поддержку и снижает количество операций DOM API.

#### Практический контекст
При работе в React 18+ или аналогичных библиотеках, генерация множества однотипных элементов с помощью DOM-узлов может вызвать сильные лаги, тогда как добавление декоративных частей через **::before/::after** позволяет создавать визуальные эффекты (например, иконки, рамки, цвета) без роста виртуального DOM и реального дерева. Используют в интерфейсах с большими списками, таблицами и элементами галерей, где важно сохранить отзывчивость UI.
