# Как корутины ведут себя при использовании бесконечного цикла (например, while(true))?

### Категории

Behavioral, Async, Concurrency, Performance, Infinite loop

### Краткий ответ

**Как корутины ведут себя при использовании бесконечного цикла (например, while(true))?**

- многозадачность Go  
- корутина блокирует **поток**, если не вызывает **yield** (например, C-усыпление отсутствует)  
- бесконечный цикл без **операций ввода-вывода** или **runtime.Gosched()** полностью **захватывает ЦП**  
- другие корутины **не получают процессорное время** → **звездныйвое блокирование**  
- вызов `runtime.Gosched()` в цикле передает управление планировщику, позволяя переключаться  
- рекомендуемый подход — использовать **операции блокировки** или **yield**, чтобы не блокировать планировщик  
- практическое значение: корректное управление временем планирования в бесконечных циклах важно для производительности и отзывчивости приложения

### Подробный ответ
#### Основной ответ  
При использовании бесконечного цикла, например `while(true)`, внутри корутины поведение зависит от того, как устроена сама корутина и её механизм согласования. В языках и фреймворках с поддержкой корутин (например, Kotlin, Python asyncio, C# async-await) корутина — это легковесный поток выполнения, который при встрече с «точками приостановки» (suspension points) отдаёт управление и может быть позже возобновлён. Если же внутри корутины стоит бесконечный цикл без операций приостановки (без `yield`, `suspend`, `await`), то такая корутина просто будет постоянно занимать поток или event loop, блокируя выполнение других задач. 

То есть корутина **не приостанавливается автоматически** в бесконечном цикле, если явно не вставлены операции приостановки. Если же цикл включает операции, которые приводят к приостановке (например, `delay()` в Kotlin, `await asyncio.sleep()` в Python), то корутина корректно отдаёт управление и позволяет другому коду исполняться параллельно.

#### Ключевые моменты  
- **Без await/yield/suspend** бесконечный цикл полностью блокирует корутину и может деградировать производительность или вызвать зависание.  
- **С операциями приостановки** (await, yield и т.п.) цикл становится "кооперативным", позволяя эффективно планировать и выполнять множество корутин.  
- Коррутины — не полноценные потоки ОС, они переключаются только в «точках приостановки» (кооперативная многозадачность), что отличает их от прерываемой многозадачности в потоках.  

#### Практический контекст  
В проектах на Kotlin (Coroutine 1.6+) для бесконечных задач используют `while(true)` вместе с `delay()` или `yield()`, чтобы не блокировать event loop и поддерживать отзывчивость. Аналогично, в Python asyncio цикл с `await asyncio.sleep()` внутри не "замораживает" event loop, позволяя выполнять другие async задачи параллельно. Такие паттерны часто применяются для длительного мониторинга, polling или обработки событий.
