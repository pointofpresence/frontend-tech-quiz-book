# Какие этапы происходят после ввода URL в браузере?

### Категории

DNS, Web, HTTP, Loading, Browser, Frontend, Networking, Behavioral

### Краткий ответ

**Этапы после ввода URL в браузере**  
- DNS-запрос: преобразование домена в IP  
- Установление TCP-соединения (3-way handshake)  
- Отправка HTTP-запроса (GET/POST и др.)  
- Сервер обрабатывает запрос и отправляет ответ (HTML, код статуса)  
- Браузер парсит HTML, строит DOM  
- Загрузка ресурсов (CSS, JS, изображения) через дополнительные запросы  
- Выполнение JS, применение CSS, отрисовка страницы на экран  

Итог: последовательный процесс от разрешения адреса до рендеринга страницы.

### Подробный ответ
#### Основной ответ
После ввода URL в браузере запускается комплексный процесс загрузки страницы, включающий несколько ключевых этапов: разрешение DNS, установление соединения с сервером, обмен запросами и ответами, а также рендеринг страницы. В совокупности эти этапы обеспечивают корректное получение и отображение нужного веб-ресурса.

#### Ключевые этапы
- **Разрешение DNS** — браузер преобразует доменное имя (например, example.com) в IP-адрес сервера через DNS-серверы. Этот шаг критичен для маршрутизации запроса.
- **Установка TCP и TLS-сессии** — после получения IP происходит TCP handshake (обычно трехстороннее рукопожатие), а затем SSL/TLS handshake, если соединение защищённое (HTTPS), что обеспечивает шифрование данных.
- **Отправка HTTP-запроса** — браузер формирует и отправляет запрос, обычно GET, с заголовками, cookies и метаданными.
- **Ответ сервера** — получая HTTP-ответ с кодом состояния (напр. 200 OK), браузер начинает обработку содержимого (HTML, CSS, JS, изображения).
- **Парсинг и рендеринг** — браузер парсит HTML в DOM-дерево, загружает и применяет CSS в CSSOM, объединяет их в render tree, выполняет JavaScript. После этого создаётся layout (вычисление расположения элементов) и paint (отрисовка).
- **Загрузка ресурсов и повторное рендеринг** — асинхронные запросы (например, через JS) могут подтягивать дополнительные данные, вызывая обновления отображения.

#### Практический контекст
В React 18+ на этапе рендеринга активно используется hydration для повышения производительности. CDN и HTTP/2 существенно снижают задержки на доставку ресурсов за счёт параллельных запросов и сжатия. Инструменты вроде Chrome DevTools помогают отследить все эти шаги в реальном времени, позволяя оптимизировать загрузку и время первого рендеринга (First Contentful Paint).
