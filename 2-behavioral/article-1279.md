# Как реализовать базовое приложение на Spring Boot?

### Категории

Behavioral, API, Framework, Application, Backend

### Краткий ответ

**Как реализовать базовое приложение на Spring Boot?**

- Spring Boot — фреймворк для **быстрого старта** Java-приложений  
- Создать главный класс с аннотацией **@SpringBootApplication**  
- Использовать **Spring Initializr** для генерации проекта с зависимостями  
- Определить контроллеры с **@RestController** и методами с **@GetMapping/@PostMapping**  
- Подключить встроенный сервер (обычно **Tomcat**) — запускается автоматически  
- Конфигурировать приложение через **application.properties/yml**  
- Запустить приложение, вызвав метод **SpringApplication.run()** для автоконфигурации и запуска  
- Используется для создания **REST API**, микросервисов, веб-приложений с минимальными настройками

### Подробный ответ
#### Основной ответ
Для базового приложения на **Spring Boot** обычно создают простой REST API или веб-приложение с минимумом конфигурации. Spring Boot позволяет быстро стартовать за счет автоконфигурации и встроенного веб-сервера (например, Tomcat). Основные шаги включают создание структуры проекта, добавление зависимости Spring Boot Starter, написание контроллера и запуск приложения.

#### Ключевые моменты
- **Создание проекта**: обычно используют Spring Initializr (https://start.spring.io) для генерации скелета с нужными зависимостями — например, `spring-boot-starter-web` для REST-сервера.
- **Основной класс приложения** с аннотацией `@SpringBootApplication` — точка входа, запускается методом `SpringApplication.run()`.
- **Контроллер**: создается класс с аннотацией `@RestController`, где можно определить метод с `@GetMapping("/hello")`, который возвращает простой текст.
- **Автоматическое конфигурирование** снимает необходимость ручной настройки сервера, что позволяет фокусироваться на бизнес-логике.
- Встроенный **Tomcat** сервер запускается внутри приложения на стандартном порте 8080.
- Для сборки и запуска часто используют **Maven** или **Gradle**.

#### Практический контекст
Типичный пример — написание микросервиса, возвращающего JSON через REST, либо простого веб-приложения. Такой подход сокращает время разработки и обеспечивает масштабируемость. В реальных проектах удобно сразу добавить Spring Data, Security и Actuator для мониторинга. Также Spring Boot 3+ использует Java 17+ по умолчанию, что дает современные возможности языка.

```java
@SpringBootApplication
public class DemoApplication {
    public static void main(String[] args) {
        SpringApplication.run(DemoApplication.class, args);
    }
}

@RestController
public class HelloController {
    @GetMapping("/hello")
    public String hello() {
        return "Привет, Spring Boot!";
    }
}
```

Это базовая структура для старта любого Spring Boot приложения.
