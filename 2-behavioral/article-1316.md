# Какие существуют типы индексов в PostgreSQL и когда их использовать?

### Категории

Indexing, Behavioral, API, Microservices, PostgreSQL, Architecture, Concurrency, Node.js, Performance, Indexes, Backend, Database, Messaging, TypeScript, Optimization, Design patterns

### Краткий ответ

**Типы индексов в PostgreSQL и когда использовать**

- B-tree: основной, для **равенств и диапазонов**, универсален  
- Hash: оптимален для **точных равенств**, но ограничен в функционале  
- GIN: для **мультивалютных данных**, полнотекстовый поиск, JSONB  
- GiST: для **пространственных данных** и **нестандартных сравнений** (например, R-tree)  
- SP-GiST: для **неравномерных или иерархических данных**, альтернативный GiST  
- BRIN: для **очень больших таблиц** с локальной сортировкой, экономит место  
- Bloom: для индексации на основе **множества атрибутов** с низким качеством коллизий  

Использование зависит от задачи:  
- Если часто ищете по диапазону — B-tree  
- При точных совпадениях и большом объеме — Hash (редко)  
- Для JSONB, массивов, полнотекста — GIN  
- Для геометрии, GIS и кастомных типов — GiST/SP-GiST  
- Для больших, смежных данных — BRIN 

Индексы уменьшают полнотекстовые сканы, ускоряют запросы, но замедляют вставки, поэтому выбор критичен для баланса производительности.

### Подробный ответ
#### Основной ответ
В **PostgreSQL** существует несколько основных типов индексов, которые используются для оптимизации разных видов запросов и операций. Ключевые из них — **B-tree**, **Hash**, **GIN**, **GiST**, **SP-GiST** и **BRIN**. Выбор типа индекса зависит от природы данных и типа запросов.

#### Ключевые моменты
- **B-tree** — самый универсальный и самый распространённый тип индекса в PostgreSQL. Он отлично подходит для операций сравнения (`=`, `<`, `>`, `BETWEEN`), сортировки и поиска диапазонов. Типично используется для индексации первичных ключей и обычных столбцов с числовыми или строковыми типами. В PostgreSQL версии 14+ оптимизирован под разные сценарии.
- **Hash** — поддерживает только равенство (`=`). Раньше имел ограничения по надежности и нельзя было использовать для сортировок или диапазонов, сейчас стал более стабильным, но всё еще менее универсален, чем B-tree. Хорош для высокопроизводительных запросов на точное совпадение без сортировки.
- **GIN (Generalized Inverted Index)** — эффективен для индексации полей с множественными значениями, например, массивов, JSONB или полнотекстового поиска. Идеален для быстрого поиска, где элементы могут встречаться в любой позиции.
- **GiST (Generalized Search Tree)** — расширяемый тип индекса, который поддерживает широкий спектр пользовательских операторов, часто используется для геопространственных данных (PostGIS), полнотекстового поиска и других нестандартных запросов.
- **SP-GiST (Space-partitioned GiST)** — оптимизирован для специфических структур данных, таких как квадродеревья или префиксные деревья. Подходит для сжатия пространственных и иерархических данных с нерегулярным распределением.
- **BRIN (Block Range Index)** — лёгкий индекс с низкими затратами по диску, эффективен для больших таблиц с физически отсортированными данными (например, временные ряды). Быстро индексирует диапазоны внутри блоков, но менее точный, чем B-tree.

#### Практический контекст
- **B-tree** — используется по умолчанию для большинства задач, как в PostgreSQL 13 и 14.
- **GIN** — PostgreSQL 14+ предлагает оптимизации для JSONB и полнотекстового поиска, делая GIN обязательным для таких сценариев.
- **BRIN** — актуален для petabyte-хранилищ и аналитики, когда сортировка по времени или датам позволяет сэкономить ресурсы.
- **GiST/SP-GiST** — массово применяются в геоинформационных системах (PostGIS), поиске похожих элементов и пользовательских типах данных.

Выбор индекса — это баланс между скоростью чтения, нагрузкой на запись и использованием дискового пространства, поэтому важно тестировать под конкретные паттерны запросов и нагрузки.
