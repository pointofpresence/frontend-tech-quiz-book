# Что произойдет, если изменить неизменяемую переменную внутри функции?

### Категории

Immutable, Functions, Backend, Python, Async, Messaging, Database, Variables, Behavioral

### Краткий ответ

**Что произойдет при изменении неизменяемой переменной внутри функции**

- неизменяемая переменная (const/val) не допускает переназначения или изменения значения  
- попытка изменить вызовет **синтаксическую ошибку** (compile-time или runtime, в зависимости от языка)  
- внутри функции объявление с тем же именем создаст **новую локальную переменную** (в языках с блочной областью, например JS let/const)  
- для объектов с неизменяемой ссылкой можно менять **внутреннее состояние**, если объект мутируемый (напр., свойства объекта)  
- использование неизменяемых переменных усиливает **безопасность и предсказуемость** кода  
- в функциональном стиле такой подход предотвращает **побочные эффекты** и упрощает отладку  
- если обязательно нужно изменить, применяют **новый объект** или возвращают обновленное значение  

Итог: попытка прямого изменения неизменяемой переменной приведёт к ошибке или создаст новую локальную переменную, сохраняя исходную неизменной.

### Подробный ответ
#### Основной ответ
Если попытаться изменить **неизменяемую (immutable) переменную** внутри функции, например, в языках с неизменяемыми типами данных (как строки в Python, числа в Java или кортежи), то фактически вы не измените саму переменную, а создадите новую переменную в локальной области, либо получите ошибку, если объект действительно нельзя изменить. Неизменяемость означает, что содержимое объекта нельзя изменять после его создания. Попытка "изменить" такой объект вызовет либо создание нового объекта, либо ошибку `TypeError` (в Python при попытке изменить tuple), либо игнорируется, если операция создает новый объект.

#### Ключевые моменты
- **Неизменяемые объекты** (str, int, tuple в Python, String в Java) нельзя модифицировать — любые операции создают новый объект в памяти.
- В функции при присваивании переменной с тем же именем внутри локальной области создаётся новый локальный объект, не влияющий на внешний.
- В языках с явной неизменяемостью (например, в функциональных) попытка изменить неизменяемую переменную приведет к ошибке компиляции или runtime (например, `val` в Kotlin нельзя изменить после инициализации).

#### Практический контекст
В профессиональной практике это важно для обеспечения **безопасности данных** и **отсутствия side-effect**. Например, в функциональном программировании неизменяемость упрощает параллелизм и прогнозируемую логику, а при использовании immutable объектов в JavaScript (через const или библиотеки типа Immutable.js) изменение объекта внутри функции фактически создаст копию, предотвращая неожиданные мутации.
