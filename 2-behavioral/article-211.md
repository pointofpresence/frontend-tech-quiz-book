# Какие настройки сервера требуются для CORS?

### Категории

Configuration, HTTP headers, CORS, API, Security, Behavioral

### Краткий ответ

**Какие настройки сервера требуются для CORS?**

- HTTP-заголовки для контроля доступа  
- **Access-Control-Allow-Origin**: разрешённые домены  
- **Access-Control-Allow-Methods**: разрешённые HTTP-методы (GET, POST, etc.)  
- **Access-Control-Allow-Headers**: разрешённые кастомные заголовки  
- **Access-Control-Allow-Credentials**: поддержка cookie и авторизации  
- Обработка **OPTIONS preflight** запросов для проверки политики  
- Настройка таймаута и кэширования с **Access-Control-Max-Age**  
- Практическое значение: защищает API от несанкционированного доступа, обеспечивает междоменные запросы без ошибок безопасности

### Подробный ответ
#### Основной ответ
Для настройки **CORS (Cross-Origin Resource Sharing)** на сервере необходимо правильно сконфигурировать HTTP-заголовки, чтобы браузеры разрешали запросы с других доменов. В основе лежат заголовки, которые сервер отправляет в ответе на preflight запрос (OPTIONS) и на обычные запросы. Основные настройки включают разрешение источников (origins), методов и заголовков, которые могут использоваться клиентом.

#### Ключевые моменты
- **Access-Control-Allow-Origin** — самый критичный заголовок. Указывает, с каких источников (доменов) разрешены запросы. Можно указать конкретный домен, несколько доменов (через логику сервера) или `*` для разрешения всех.
- **Access-Control-Allow-Methods** — перечень HTTP-методов, разрешенных сервером (например, GET, POST, PUT, DELETE).
- **Access-Control-Allow-Headers** — список заголовков, которые клиент может отправлять в запросе (например, Authorization, Content-Type).
- **Access-Control-Allow-Credentials** — если требуется отправка cookies или авторизационных данных, должна быть включена с `true` и нельзя использовать `*` в Access-Control-Allow-Origin.
- **Preflight-запросы (OPTIONS)** — сервер должен корректно отвечать на OPTIONS-запросы, подтверждая разрешённые методы и заголовки.

#### Практический контекст
В современных web-приложениях, например с React 18 и backend на Node.js (Express.js) или Spring Boot, CORS обычно настраивается через middleware (например, пакет `cors` в Node.js). В продакшн рекомендуется явно указывать разрешённые домены, чтобы уменьшить безопасность риски. Также важно проверить, что сервер корректно обрабатывает preflight запросы с latency около 50ms и соблюдает требования безопасности, особенно при работе с куками и токенами авторизации.
