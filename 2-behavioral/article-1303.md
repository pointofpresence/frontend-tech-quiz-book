# Что такое реляционная база данных?

### Категории

System analysis, Database design, Theory, Behavioral, Message brokers, Architecture, Technical interview, Database, Backend, Workflow, Data modeling, Gitflow

### Краткий ответ

**Что такое реляционная база данных?**
- система управления данными на основе **таблиц**
- данные хранятся в виде **строк и столбцов**
- таблицы связаны через **ключи** (первичные, внешние)
- использует **структурированный язык SQL** для управления
- обеспечивает **целостность и согласованность** данных через ограничения
- поддерживает **транзакции** с ACID-свойствами
- применяется для сложных бизнес-приложений с отношениями между данными

### Подробный ответ
#### Основной ответ
**Реляционная база данных (РБД)** — это тип базы данных, в которой данные организованы в виде таблиц (отношений), состоящих из строк и столбцов. Каждая таблица содержит данные по определённой теме, а связи между таблицами реализуются через ключи (первичные и внешние). Концепция основана на реляционной модели, предложенной Эдгаром Коддом в 1970 году.

#### Ключевые моменты
- **Структурированность и нормализация**: данные логично разделены на таблицы для минимизации избыточности и поддержания целостности посредством нормализации (форм нормализации обычно используют 1-3NF).
- **Язык запросов SQL**: РБД используют декларативный язык SQL для манипуляций с данными — выборки, вставки, обновления, удаления.
- **Транзакционность и ACID**: надёжность достигается свойствами Атомарности, Согласованности, Изолированности и Долговечности, что критично для бизнес-приложений с высокими требованиями к консистентности.
- **Индексы и оптимизация запросов**: позволяют существенно ускорить поиск и выборку данных.
- **Примеры**: широко применяемые СУБД — PostgreSQL 14+, MySQL 8, Oracle DB, Microsoft SQL Server.

#### Практический контекст
Реляционные базы данных идеально подходят для систем с чётко структурированными данными, такими как финансовые приложения, CRM, ERP-системы и веб-сайты, где важна целостность данных и сложные запросы. Например, в интернет-магазине таблицы пользователей, заказов и товаров связаны между собой через ключи, обеспечивая непротиворечивое хранение информации.
