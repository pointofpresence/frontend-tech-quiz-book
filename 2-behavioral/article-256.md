# Как остановить всплытие события (event propagation) при клике внутри модального окна?

### Категории

Event propagation, UI, Frontend, Event handling, JavaScript, Modal, Behavioral

### Краткий ответ

**Как остановить всплытие события в модальном окне?**  
- Веб, события, DOM  
- всплытие — событие поднимается от вложенного к родительским элементам  
- для остановки вызывают `event.stopPropagation()` внутри обработчика клика модального окна  
- `stopPropagation` предотвращает передачу события дальше по дереву DOM  
- можно дополнительно вызвать `event.preventDefault()`, если нужно предотвратить стандартное поведение  
- важно применять именно в обработчике модального окна, чтобы не закрывать модал при кликах внутри  
- используется для управления интерактивностью и предотвращения закрытия модального окна при кликах внутри контента

### Подробный ответ
#### Основной ответ
Чтобы остановить всплытие события (event propagation) при клике внутри модального окна, в обработчике события нужно вызвать метод **event.stopPropagation()**. Этот метод предотвращает дальнейшее распространение события вверх по дереву DOM, благодаря чему родительские обработчики на элементы вне модального окна не срабатывают.

#### Ключевые моменты
- События в браузере обрабатываются в три фазы: захват (capture), целевой элемент (target) и всплытие (bubble). Всплытие — это когда событие поднимается по DOM от вложенного элемента к родителям.
- Для модального окна часто добавляют слушатель на оверлей (фон) с целью закрыть окно при клике вне содержимого. Чтобы предотвратить закрытие при клике внутри модального контента, внутри элемента модального окна нужно остановить всплытие.
- В React 18 и современных браузерах **event.stopPropagation()** корректно работает и в синтетических, и в нативных событиях.
  
#### Практический контекст
В типичном UI паттерне модальное окно располагается внутри оверлея с обработчиком нажатия, закрывающим окно. Без остановки всплытия клик по содержимому модального окна также "всплывет" до оверлея и вызовет закрытие. В обработчике клика на модальном контенте вызов **event.stopPropagation()** решает эту проблему эффективно и просто.
