# Можно ли после GROUP BY применить WHERE‑фильтрацию? Если нет, то как?

### Категории

Behavioral, Group by, Backend, Filtering, Database, SQL

### Краткий ответ

**Можно ли после GROUP BY применить WHERE‑фильтрацию? Если нет, то как?**

- SQL: WHERE фильтрует до группировки  
- GROUP BY создает агрегаты  
- WHERE не видит агрегированные данные  
- для фильтрации агрегатов нужен HAVING  
- HAVING применяется после GROUP BY  
- HAVING фильтрует группы по агрегатам  
- для обычных столбцов — WHERE, для агрегатов — HAVING  

Основная идея: после группировки фильтруем результаты через HAVING, а не WHERE.

### Подробный ответ
#### Основной ответ
После выполнения **GROUP BY** применять **WHERE** нельзя, потому что WHERE-фильтр действует на строки исходной таблицы, до группировки. Для фильтрации уже сгруппированных данных используется конструкция **HAVING** — он фильтрует агрегированные группы.

#### Ключевые моменты
- **WHERE** фильтрует исходные записи, еще до агрегирования и формирования групп.
- **HAVING** применяется после GROUP BY к агрегатным результатам, например, к sum, count, avg и т.д.
- Именно HAVING позволяет, например, показать только те группы, где сумма продаж больше заданного значения.

#### Практический контекст  
Типичная ситуация: нужно сгруппировать пользователей по региону и отфильтровать регионы, где количество пользователей превышает 100. Тогда запрос будет такой:

```sql
SELECT region, COUNT(*) as users_count
FROM users
GROUP BY region
HAVING COUNT(*) > 100;
```

Если попробовать использовать WHERE для COUNT(*), возникнет ошибка, так как агрегатные функции недоступны на этапе WHERE.

---

Таким образом, логика SQL строго разделяет этапы: сначала фильтрация строк через WHERE, затем группировка через GROUP BY, и после этого фильтрация групп через HAVING. Это ключевой момент при построении сложных запросов с агрегатами.
