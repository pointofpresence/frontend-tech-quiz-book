# Что такое API? Чем REST API отличается от других типов API?

### Категории

HTTP, Behavioral, API, Web, Backend, Database, REST

### Краткий ответ

**Что такое API? Чем REST API отличается от других типов API?**

- API — это **интерфейс** для взаимодействия между программами  
- Позволяет **обращаться** к функциям и данным сервиса без внутреннего доступа  
- REST API — один из видов API, базирующийся на архитектуре **REST** (Representational State Transfer)  
- Использует стандартные HTTP-методы: **GET, POST, PUT, DELETE**  
- Оперирует ресурсами через **URL**, передавая состояние через представления (JSON, XML)  
- Главное отличие REST — **статeless** (без сохранения состояния), упрощает масштабируемость  
- Другие API (SOAP, GraphQL): SOAP — более сложный протокол с WS-стандартами, GraphQL — гибкий запрос данных с клиентской стороны  
- REST проще, более легковесный, широко используется для веб-сервисов и микросервисов  

Итог: API — механизм взаимодействия приложений; REST API — современный, HTTP-ориентированный, масштабируемый и простой вариант API, отличающийся статeless-дизайном и ресурсной моделью.

### Подробный ответ
#### Основной ответ
**API (Application Programming Interface)** — это интерфейс, который позволяет программным приложениям взаимодействовать друг с другом через набор определённых методов и протоколов. API описывает, каким образом запросы должны быть сформированы, как получить доступ к функционалу или данным, и что ожидать в ответе.

**REST API** (Representational State Transfer) — один из популярных стилей построения API, основанный на архитектурных принципах REST. Он использует стандартные HTTP-методы (GET, POST, PUT, DELETE) и ориентирован на работу с ресурсами, представленными в виде URL.

#### Ключевые моменты
- **REST API vs SOAP**: REST проще, работает поверх HTTP и универсален, а SOAP — более строгий протокол с обязательным использованием XML и встроенной логикой безопасности (WS-Security), часто применяется в корпоративных системах.
- **REST API vs GraphQL**: REST ориентирован на ресурсы и фиксированные эндпоинты, где клиент получает предопределённые данные. GraphQL предоставляет гибкий запрос, где клиент определяет, какие поля нужны, снижая избыточность.
- **Основные свойства REST**: stateless (без состояния между запросами), кешируемость, клиент-серверная архитектура.

#### Практический контекст
REST API широко используют для создания web-сервисов из-за его простоты и совместимости с HTTP. К примеру, в React-приложениях общение с backend обычно происходит через REST API, возвращающий JSON, что обеспечивает легкую интеграцию и масштабируемость.
