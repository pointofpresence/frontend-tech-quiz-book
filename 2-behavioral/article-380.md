# Что такое корутина в Python?

### Категории

Async, Backend, Behavioral, Coroutine

### Краткий ответ

**Что такое корутина в Python?**  
- парадигма **асинхронного** программирования  
- функция с ключевым словом **async def**  
- выполняется **непрерывно**, может **приостанавливать** себя через await  
- позволяет **эффективно** работать с I/O без блокировки  
- управление возвращается циклу событий (**event loop**)  
- ключ к написанию **масштабируемого** сетевого и параллельного кода  
- широко используется с библиотеками: **asyncio**, aiohttp, fastapi

### Подробный ответ
#### Основной ответ
**Корутина в Python** — это специализированный тип функции, который может приостанавливать своё выполнение и позже возобновлять его, позволяя тем самым реализовывать асинхронное и конкурентное программирование. В отличие от обычных функций, корутины работают на основе механизма cooperative multitasking, где управление явно передается между задачами.

#### Ключевые моменты
- Корутины реализуются с помощью конструкции `async def` в Python 3.5+ и управляются с помощью оператора `await`, который приостанавливает выполнение корутины, ожидая завершения асинхронной операции без блокировки основного потока.
- Они являются фундаментом для асинхронного программирования в Python и тесно связаны с библиотекой **asyncio**, которая предоставляет event loop и средства для запуска, координации и управления корутинами.
- В отличие от генераторов, которые также способны приостанавливать выполнение через `yield`, корутины предназначены для управления асинхронными операциями и имеют синтаксис и семантику, оптимизированные для этого.

#### Практический контекст
В реальных проектах корутины широко применяются для работы с сетевыми I/O, базами данных и другими источниками данных с высокой задержкой, где важно эффективно использовать время ожидания без блокировки потоков. Например, в веб-серверах на базе **FastAPI** или **aiohttp** корутины позволяют обслуживать тысячи одновременных клиентов с минимальным использованием ресурсов.
