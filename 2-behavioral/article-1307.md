# Какой протокол использует RabbitMQ?

### Категории

Protocol, Behavioral, Async, Backend, Messaging

### Краткий ответ

**Какой протокол использует RabbitMQ?**  
- RabbitMQ — брокер сообщений  
- Основной протокол — **AMQP (Advanced Message Queuing Protocol)**  
- AMQP — открытый, стандартный, **надежный протокол очередей**  
- Позволяет **гарантировать доставку сообщений**, маршрутизацию и подтверждения  
- Поддерживает различные обмены, очереди, binding'и  
- RabbitMQ дополнительно поддерживает MQTT, STOMP (через плагины)  
- AMQP обеспечивает гибкую архитектуру и совместимость с многими клиентами  
- Используется для **масштабируемой, отказоустойчивой** коммуникации между сервисами

### Подробный ответ
#### Основной ответ
**RabbitMQ** использует протокол **AMQP (Advanced Message Queuing Protocol)** — открытый стандартный протокол для обмена сообщениями между приложениями. AMQP обеспечивает надежную, асинхронную доставку сообщений с поддержкой подтверждений, маршрутизации и очередей.

#### Ключевые моменты
- **AMQP 0-9-1 и AMQP 1.0**: RabbitMQ изначально реализует AMQP версии 0-9-1, который является самым популярным и широко используемым в мире. Поддержка AMQP 1.0 появилась позже, но она менее распространена в RabbitMQ.
- **Гибкая маршрутизация**: AMQP позволяет настраивать сложные пути прохождения сообщений через exchange-ы, очереди и binding-и, что помогает решать различные задачи очередей и балансировки нагрузки.
- **Поддержка нескольких протоколов**: кроме AMQP, RabbitMQ поддерживает протоколы MQTT, STOMP и HTTP (через плагины), что расширяет возможности интеграции с разными системами.
  
#### Практический контекст
При использовании RabbitMQ в проектах часто выбирают AMQP 0-9-1 для гарантированной доставки и сложной логики маршрутизации, например, разделения задач по разным worker-ам. Наличие официальных клиентских библиотек для разных языков (Java, Python, .NET) реализует этот протокол, что обеспечивает надежную и стандартизованную коммуникацию.
