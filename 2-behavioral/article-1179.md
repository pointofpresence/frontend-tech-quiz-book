# В чём отличие кластеризованного и некластеризованного индекса? Когда их стоит применять?

### Категории

Indexing, Behavioral, API, Backend, Database, Optimization

### Краткий ответ

**Отличие кластеризованного и некластеризованного индекса**

- индексы — структура для ускорения поиска в БД  
- **кластеризованный** индекс — определяет **физический порядок строк** в таблице  
- в таблице может быть **только один** кластеризованный индекс  
- кластеризованный индекс хранит данные **в порядке ключа**  
- **некластеризованный** индекс — отдельная структура с ссылками на строки  
- может быть **много некластеризованных** индексов  
- кластеризованный индекс лучше для **диапазонных запросов**  
- некластеризованный — для выборки по уникальным или неупорядоченным полям  
- нельзя кластеризовать таблицу без явного указания или по умолчанию  
- правильный выбор улучшает **скорость запросов** и **производительность** при разных сценариях использования

### Подробный ответ
#### Основной ответ
**Кластеризованный индекс** определяет физический порядок хранения данных в таблице — строки располагаются на диске в соответствии с порядком ключа индекса. В таблице может быть только один кластеризованный индекс, так как данные могут быть упорядочены только одним способом. **Некластеризованный индекс** — это отдельная структура данных, которая хранит индексированные ключи и указатели на фактические строки таблицы, не изменяя их физический порядок.

#### Ключевые моменты
- **Кластеризованный индекс** быстрее при поиске диапазонов, т.к. строки физически упорядочены — подходит для запросов с сортировкой или фильтровкой по интервалам (например, дата, ID).
- **Некластеризованный индекс** эффективен для быстрого поиска по отдельным ключам, но требует дополнительного шага для доступа к полной строке (lookup).
- Кластеризованные индексы лучше использовать для колонок с высокой селективностью и частыми запросами на диапазон, а некластеризованные — для отдельных столбцов с высокой селективностью, когда нужны частые точечные поиски.

#### Практический контекст
В **PostgreSQL** кластеризованный индекс реализуется как индекс типа B-tree, который можно связать с таблицей через `CLUSTER` команду, но физический порядок меняется вручную. В **SQL Server** кластеризованный индекс — это основная физическая организация таблицы, а некластеризованный создается отдельно как отдельная структура. В реальных проектах часто делают кластеризованный индекс по первичному ключу, а к часто фильтруемым колонкам добавляют некластеризованные индексы для ускорения точечных запросов и join-операций.
