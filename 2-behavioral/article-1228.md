# Что такое sealed-классы и для чего они применяются?

### Категории

Android, Theory, Behavioral, Mobile, OOP, Sealed classes, Kotlin, Live coding

### Краткий ответ

**Что такое sealed-классы и для чего они применяются?**  
- sealed-класс — ограниченный класс с контролируемым наследованием  
- наследоваться могут только объявленные внутри или в том же файле подклассы  
- обеспечивает **безопасность типов** и строгий контроль иерархии  
- используется для реализации закрытых иерархий (например, **алгебраические типы данных**)  
- облегчает **вычисление exhaustive when/switch** (компилятор знает все подклассы)  
- уменьшает ошибки, связанные с непредвиденным наследованием  
- применяется в языках Kotlin, Scala, Java (с Java 17) для поддержки **устойчивой архитектуры** и **чёткой модели домена**

### Подробный ответ
#### Основной ответ
**Sealed-классы** — это особый тип классов в языках программирования, таких как Kotlin или Java (начиная с версии 15), которые ограничивают иерархию наследования. Они позволяют объявить базовый класс, наследниками которого могут быть только заранее определённые подклассы, указываемые в самом sealed-классе или в том же модуле. Это помогает контролировать все возможные варианты наследования и улучшает безопасность типов.

#### Ключевые моменты
- **Контроль наследования**: sealed-класс закрывает иерархию, подклассы известны компилятору, что позволяет реализовать проверку полноты при использовании `when` или `switch` без необходимости default-веток.
- **Более безопасный код**: упрощается поддержка и развитие, так как список наследников фиксирован и не может быть расширен в других модулях, что снижает ошибки и неожиданное поведение.
- **Использование в pattern matching**: в Kotlin 1.5+ и Java 17+ становятся мощной основой для безопасного и выразительного сопоставления с образцом, что особенно актуально в обработке состояний и событий.

#### Практический контекст
В реальных проектах sealed-классы часто применяют для описания конечных множеств состояний UI (например, загрузка, успех, ошибка) или событий, где важно гарантировать, что обработка всех вариантов завершена и корректна. В Android-разработке на Kotlin такой подход уменьшает баги и упрощает поддержку кода за счёт статической проверки вариантов.
