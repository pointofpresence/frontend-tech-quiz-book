# Что такое REST? Какие принципы лежат в его основе?

### Категории

Backend, REST, Behavioral, Database, API, Principles

### Краткий ответ

**Что такое REST? Какие принципы лежат в его основе?**

- REST (Representational State Transfer) — стиль архитектуры веб-сервисов  
- Опирается на протокол HTTP  
- Использует **ресурсы**, идентифицируемые URL  
- Основные методы: **GET, POST, PUT, DELETE** (CRUD)  
- **Безсостояние** — сервер не хранит состояние клиента  
- Клиент-серверное разделение обязанностей  
- Унифицированный интерфейс (стандартизированные методы и форматы)  
- Кэшируемость для повышения производительности  
- Слои системы для масштабируемости  
- Принцип света слоев упрощает предотвращение взаимодействия между компонентами  
- Поддержка кодируемого представления ресурса (JSON, XML)  
- Широкое применение в построении масштабируемых и гибких API  

В итоге REST — это легковесная, стандартизированная архитектура для создания простых, масштабируемых и поддерживаемых веб-сервисов.

### Подробный ответ
#### Основной ответ
**REST (Representational State Transfer)** — это архитектурный стиль для проектирования сетевых приложений, в основном веб-сервисов, который опирается на стандартные методы протокола HTTP. Основная идея — сделать взаимодействие клиента и сервера простым, предсказуемым и масштабируемым за счёт использования стандартных операций и структурированных ресурсов.

#### Ключевые моменты
- **Клиент-серверная архитектура**: разделение обязанностей — сервер управляет данными, а клиент отвечает за интерфейс, что упрощает развитие и масштабирование.
- **Безсостояние (Stateless)**: каждый запрос от клиента содержит всю необходимую информацию, сервер не хранит состояние между запросами, что облегчает масштабирование и отказоустойчивость.
- **Единый интерфейс**: использование стандартных HTTP-методов (GET, POST, PUT, DELETE) и унифицированных URI для идентификации ресурсов.
- **Кэшируемость**: сервер может указывать, какие ответы можно кэшировать, что повышает производительность и снижает нагрузку.
- **Слои**: архитектура может быть многослойной, клиент не знает, с каким конкретно сервером он взаимодействует, что даёт дополнительную гибкость и безопасность.
- **Код по требованию (опционально)**: сервер может передавать клиенту код (например, JavaScript) для расширения функциональности.

#### Практический контекст
REST широко используется в разработке веб-API благодаря простоте и гибкости. Например, в **PostgreSQL 14+** или **Spring Boot**, REST позволяет быстро создавать сервисы с чётким разделением логики и легко интегрируемыми клиентами. RESTful API обычно снабжены документацией в формате OpenAPI (Swagger), что облегчает тестирование и поддержку в командах.
