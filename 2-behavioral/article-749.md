# Принципы ООП: инкапсуляция, полиморфизм, наследование. Примеры применения.

### Категории

Behavioral, Microservices, High load, Live coding, Architecture, Integration, OOP, Inheritance, Encapsulation, Backend, Polymorphism, Interview

### Краткий ответ

**Принципы ООП: инкапсуляция, полиморфизм, наследование**

- ООП — парадигма, структурирует код через объекты
- **Инкапсуляция**: скрытие данных, доступ через методы, защищает внутреннее состояние
- **Наследование**: расширение класса, повторное использование кода, иерархия типов
- **Полиморфизм**: одинаковый интерфейс, разные реализации, обеспечивает гибкость
- Пример: класс «Животное», наследники «Собака», «Кошка» с переопределением метода «издать звук»
- Инкапсуляция нужна для контроля доступа и предотвращения неправильного использования
- Полиморфизм упрощает изменение и расширение системы без правок клиентского кода

Этот набор принципов позволяет писать модульный, поддерживаемый и расширяемый код.

### Подробный ответ
#### Основной ответ
**Объектно-ориентированное программирование (ООП)** строится на трёх ключевых принципах: **инкапсуляция**, **наследование** и **полиморфизм**. 

**Инкапсуляция** — это скрытие внутренней реализации класса и предоставление доступа к данным только через публичный интерфейс (методы). Это позволяет защитить объект от некорректных изменений и облегчает поддержку кода.

**Наследование** — механизм создания нового класса на основе уже существующего, который позволяет использовать и расширять функциональность, избегая дублирования.

**Полиморфизм** — способность объектов разных классов обрабатываться через общий интерфейс. Это дает возможность вызывать методы с одинаковыми названиями, но с разной реализацией в подклассах (динамический полиморфизм).

#### Ключевые моменты
- **Инкапсуляция** обеспечивает контроль доступа с помощью областей видимости (`private`, `protected`, `public`), повышает безопасность и поддерживаемость.
- **Наследование** способствует повторному использованию кода и иерархической организации моделей (например, базовый класс `Animal` и подклассы `Dog`, `Cat`).
- **Полиморфизм** позволяет использовать метод «один интерфейс — много реализаций», что упрощает расширяемость и заменяемость компонентов (например, разные алгоритмы сортировки реализуют одинаковый интерфейс `Sort`).

#### Практический контекст
В реальных проектах, например, на Java или C#, инкапсуляция используется чтобы скрыть детали реализации сервиса; наследование — для создания специализированных классов (например, разные типы аккаунтов в банке); полиморфизм — для подстановки разных реализаций сервисов без изменения клиентского кода, что удобно при тестировании и развитии архитектуры. В React 18, например, можно считать компоненты с разной логикой, но одинаковым интерфейсом, реализуя концепцию полиморфизма через props.
