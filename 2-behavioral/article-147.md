# Что такое REST?

### Категории

Web, Backend, Design principles, Database, Architecture, API, Testing, Behavioral

### Краткий ответ

**Что такое REST?**

- Архитектурный стиль для веб-сервисов  
- Основан на **ресурсах**, которые идентифицируются URI  
- Использует стандартные HTTP-методы: **GET**, **POST**, **PUT**, **DELETE**  
- Подразумевает **безсостояние** (stateless) взаимодействие клиента и сервера  
- Клиент управляет состоянием, сервер — только обработка запросов  
- Форматы данных чаще всего: **JSON** или XML  
- Высокая масштабируемость, простота интеграции и кэширование  

REST позволяет создавать легковесные, масштабируемые API, подходящие для распределённых систем и микросервисов.

### Подробный ответ
#### Основной ответ
**REST (Representational State Transfer)** — это архитектурный стиль для проектирования распределённых систем, особо популярных в веб-разработке для создания API. REST определяет набор ограничений, которые обеспечивают масштабируемость, простоту и взаимодействие между клиентом и сервером через стандартные HTTP методы.

#### Ключевые моменты
- **Ресурсы и URI**: Всё, с чем работает REST API, представлено как ресурсы, доступные по уникальным URL (например, /users/123).
- **HTTP методы**: Чётко разграничиваются операции: GET — получить данные, POST — создать, PUT/PATCH — обновить, DELETE — удалить. Это позволяет использовать стандартные средства HTTP и кэширование.
- **Статус код ответа**: Используются стандартные коды HTTP (200, 201, 404, 500 и т.п.) для информирования клиента о результате операции.
- **Бездействие серверного состояния (stateless)**: Каждый запрос содержит всю необходимую информацию, сервер не сохраняет контекст между вызовами, что повышает масштабируемость.
- **Часто используется JSON** для передачи представлений ресурсов, хотя это не обязательное требование.

#### Практический контекст
REST — стандарт де-факто для веб-сервисов с момента появления, например, в API крупных систем (GitHub, Twitter API). Его простота и совместимость с HTTP сделали его наиболее распространённым подходом до недавнего времени, когда набирают популярность gRPC и GraphQL, особенно для более сложных сценариев. Однако для построения масштабируемых, легко расширяемых и понятных API в системах с широкой экосистемой клиентов REST остаётся ключевым выбором.
