# Перечислите все типы данных в JavaScript?

### Категории

Growth, Data types, Behavioral, JavaScript, Frontend, Interview, Testing

### Краткий ответ

**Типы данных в JavaScript**

- базовые примитивы: **Number, String, Boolean, Symbol, BigInt, Undefined, Null**  
- объект — **сложный тип** для коллекций и структур данных  
- функции — объект первого класса  
- typeof выделяет все примитивы и объекты  
- примитивы — **неизменяемы**, объекты — изменяемы  
- BigInt — новые числовые типы для больших чисел  
- Symbol — уникальные и неизменяемые идентификаторы  
- практическая значимость: типы влияют на операции, заданные преобразования и поведение кода в runtime

### Подробный ответ
#### Основной ответ
В **JavaScript** существует 7 основных типов данных, которые делятся на **примитивные** и **объектные**. Примитивные типы хранят значения напрямую и являются неизменяемыми, а объектные — это коллекции значений или более сложные структуры.

Основные типы данных:

1. **Undefined** — значение переменной не было присвоено. Изначально переменная имеет значение `undefined`.
2. **Null** — специальное значение, означающее «отсутствие значения» или «пустое значение».
3. **Boolean** — логический тип, может быть либо `true`, либо `false`.
4. **Number** — числовой тип; включает как целые, так и дробные числа. В ES6 появился также тип **BigInt** для целых чисел произвольной длинны.
5. **String** — строковый тип, последовательность символов в UTF-16.
6. **Symbol** (с ES6) — уникальный и неизменяемый идентификатор.
7. **Object** — сложный тип для хранения коллекций и более сложных структур (масивы, функции, объекты, даты и др.).

#### Ключевые моменты
- **Примитивы**: `undefined`, `null`, `boolean`, `number`, `string`, `symbol`, `bigint` — неизменяемы, копируются по значению.
- **Объекты** — копируются по ссылке, могут включать массивы, функции, экземпляры классов.
- **Особенности**: `typeof null` — «object» (историческая особенность), BigInt появился в ES2020, символы гарантируют уникальность (используются в метаданных и ключах объектов).

#### Практический контекст
Знание типов важно для корректного сравнения, например, `===` и `==` работают по-разному с примитивами и объектами. В React 18 и Node.js 18+ часто используют BigInt для уникальных идентификаторов. Symbol помогает избежать конфликтов при добавлении свойств в глобальные объекты или библиотеки. Понимание объекта как контейнера критично для работы с данными и их структурированием в реальных приложениях.
