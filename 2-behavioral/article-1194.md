# Что такое лямбда-выражение в Kotlin?

### Категории

Theory, Behavioral, Mobile, Functional programming, Java interoperability, Lambda

### Краткий ответ

**Что такое лямбда-выражение в Kotlin?**  
- функциональный тип, сокращение функции  
- анонимная функция без имени и тела, заданного inline  
- принимает параметры, возвращает результат  
- передаётся как значение и используется для упрощения кода  
- часто применяется в коллекциях и API высокого порядка  
- поддерживает замыкания (доступ к внешним переменным)  
- повышает читаемость и гибкость кода через функциональный стиль

### Подробный ответ
#### Основной ответ
**Лямбда-выражение в Kotlin** — это безымянная функция, которую можно передавать как значение, хранить в переменных и использовать как параметр функций. Оно позволяет лаконично описывать поведение, обычно в функциональном стиле, упрощая работу с коллекциями и асинхронным кодом.

#### Ключевые моменты
- **Синтаксис**: лямбда записывается в фигурных скобках `{ параметры -> тело }`, например `{ x: Int -> x * 2 }`.
- **Тип**: представляет собой объект функционального типа, например `(Int) -> Int`, благодаря чему Kotlin поддерживает высокоуровневые функции высшего порядка.
- **Контекст**: лямбда имеет доступ к переменным внешнего скоупа (замыкание), что удобно для работы с состоянием.
- В Kotlin 1.3+ появилась поддержка **inlining** лямбд для оптимизации производительности, уменьшая накладные расходы при вызовах.

#### Практический контекст
Лямбды часто применяются при работе с коллекциями: `list.filter { it > 0 }.map { it * 2 }`. Также они активно используются в корутинах (например, `launch { ... }`), при работе с RxJava и в Android для обработки событий. Благодаря им код становится более компактным и выразительным, при этом сохраняется типобезопасность.
