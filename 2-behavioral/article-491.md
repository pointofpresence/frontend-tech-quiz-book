# Почему при использовании var в цикле получаются одинаковые значения?

### Категории

Variable scoping, Behavioral, JavaScript, Event loop, Loops, Code review, Live coding

### Краткий ответ

**Почему при использовании var в цикле получаются одинаковые значения?**

- var имеет **функциональную область видимости**, не блочную  
- при цикле все итерации разделяют **одну общую переменную**  
- замыкания внутри цикла захватывают **одну и ту же ссылку** var-переменной  
- изменение переменной происходит к концу цикла, все замыкания видят **итоговое значение**  
- для сохранения значения каждой итерации используют **let** (блочная область видимости) или создают новые области видимости (например, через функции)  
- проблема возникает из-за **отсутствия блоковой области**, что приводит к одинаковым результатам в замыкании  
- показывает важность понимания **видимости переменных** и замыканий в JS для корректного кода

### Подробный ответ
#### Основной ответ
При использовании **var** в цикле, например, `for (var i = 0; i < n; i++)`, все итерации цикла фактически используют **одну общую переменную `i`**, объявленную в функциональной области видимости. В итоге, при асинхронных операциях внутри цикла все колбэки или отсроченные функции «видят» одно и то же значение переменной — ту самую, которая была изменена после завершения цикла. Поэтому при обращении к `i` внутри таких функций получаем одно и то же значение (обычно последнее).

#### Ключевые моменты
- **Область видимости var** — функциональная, а не блочная, поэтому переменная `i` одна на весь цикл и не создаётся заново на каждой итерации.
- **Асинхронные операции внутри цикла** (например, setTimeout или промисы) ссылаются на эту одну переменную, которая изменяется, поэтому все колбэки после завершения цикла получают одинаковое значение.
- **ES6 let решает проблему** — `let` создаёт новую переменную на каждую итерацию цикла (блочная область видимости), поэтому в асинхронных функциях с каждой итерацией передаётся правильное значение.

#### Практический контекст
Часто такая проблема возникает при использовании `var` в циклах с setTimeout, промисами или событиями. Например, при отложенной отрисовке элементов или асинхронных запросах нужно использовать `let` или создавать замыкания (IIFE) для сохранения правильного значения переменной в каждой итерации. В React 18+ и современных приложениях типично использовать `let` или const внутри циклов, чтобы избежать подобных ошибок.
