# Что делают аннотации @Transactional и @Retryable?

### Категории

Behavioral, Java, Backend, Annotations, Workflow, Transaction management, Gitflow, Retry

### Краткий ответ

**Что делают аннотации @Transactional и @Retryable?**  
- Контекст: Spring Framework, управление транзакциями и повторными попытками  
- **@Transactional**: помечает метод/класс для автоматического управления **транзакциями** (открытие, коммит, откат)  
- Обеспечивает **атомарность** и консистентность при работе с БД  
- Поддерживает **прокси** для контроля транзакций на уровне аспектов  
- **@Retryable**: определяет метод для автоматических **повторных вызовов** при определённых исключениях  
- Настраивается по числу попыток, задержке и типам исключений  
- Упрощает обработку временных ошибок (сетевые сбои, тайм-ауты)  
- Оба решают инфраструктурные задачи без явного кода  
- Практическое значение: стабильность работы сервисов и корректность данных без 'мусорного' бизнес-кода  

Такой ответ показывает понимание управления транзакциями и паттернов надёжности в Spring, что важно для backend-разработчика.

### Подробный ответ
#### Основной ответ
Аннотация **@Transactional** в Spring используется для управления транзакциями на уровне метода или класса, обеспечивая атомарность, консистентность, изоляцию и долговечность (ACID) операций с базой данных. Она автоматически открывает транзакцию перед выполнением метода и коммитит её после успешного завершения, либо откатывает при возникновении исключения.

Аннотация **@Retryable** позволяет автоматически повторять вызов метода при возникновении определённых исключений. Это полезно для работы с ненадёжными внешними сервисами или временными ошибками. При срабатывании механизма ретраев метод будет вызван повторно с заданными интервалами или ограничением количества попыток.

#### Ключевые моменты
- **@Transactional** поддерживает разные уровни изоляции и propagation, что даёт гибкость в сложных сценариях работы с БД, например, используется в Spring Framework 5 и выше.
- **@Retryable** интегрируется с Spring Retry, позволяет задавать параметры: maxAttempts, backoff (задержка между вызовами), исключения, на которые срабатывает повтор.
- Транзакции и ретраи часто комбинируют, но нужно быть осторожным, поскольку **@Retryable** может приводить к повторным попыткам и, соответственно, повторным транзакциям.

#### Практический контекст
В реальных проектах **@Transactional** применяется для гарантии целостности данных — например, при сохранении заказа и связанных записей. **@Retryable** полезна для обработки сетевых ошибок при вызове REST API или интеграции с очередями, когда временные сбои — обычное дело. В связке эти аннотации улучшают устойчивость и качество бизнес-логики.
