# Как работает Event Loop в JavaScript?

### Категории

Async, Frontend, Event loop, JavaScript, Behavioral

### Краткий ответ

**Как работает Event Loop в JavaScript?**

- основа: **однопоточный** цикл обработки задач  
- выделяет две основные очереди: **call stack** и **event queue**  
- call stack — стек вызовов, где выполняется код  
- event queue — очередь событий (callback'ов) от асинхронных операций  
- Event Loop проверяет, свободен ли call stack  
- если стек пуст, берет следующую задачу из event queue и помещает в стек  
- обеспечивает **асинхронное, неблокирующее** выполнение кода  
- ключ для обработки **таймеров, IO, промисов** и DOM-событий в браузере

### Подробный ответ
#### Основной ответ
**Event Loop** в **JavaScript** — это механизм, который позволяет неблокирующему однопоточному движку асинхронно обрабатывать операции. Поскольку JavaScript работает в однопоточном окружении (например, в браузере или Node.js), event loop обеспечивает выполнение кода, обработку событий и асинхронных задач без блокировки основного потока.

Принцип работы таков: движок сначала выполняет весь синхронный код, затем берёт задачи из очередей (tasks и microtasks) и выполняет их по порядку. Event Loop постоянно проверяет, нет ли в очередях задач, и если есть — извлекает их и запускает.

#### Ключевые моменты
- **Call Stack** — стек вызовов, где хранится выполняющийся синхронный код; event loop запускает задачи только когда стек пуст.
- **Task Queue (Macro-task queue)** — очередь обычных событий (setTimeout, setInterval, I/O), которые ставятся в очередь после завершения текущего стека.
- **Microtask Queue** — очередь более приоритетных задач (промисы `.then()`, `process.nextTick` в Node.js); микротаски выполняются сразу после завершения текущего стека и перед выполнением macro-tasks.
- Event Loop постоянно переключается между проверкой стека, микротасков и обычных тасков, обеспечивая плавную асинхронную обработку.
- Операции I/O и таймеры обрабатываются в фоне (через Web APIs или Node.js C++ API), а callback’и возвращаются в очередь задач.

#### Практический контекст
Например, в React 18 понимание event loop важно для оптимизации асинхронных эффектов и concurrent rendering. В Node.js event loop обеспечивает высокую пропускную способность при одновременной работе с сетью и файлами. Понимание разницы между microtasks и macrotasks помогает избежать неожиданных багов с порядком выполнения кода, например при работе с промисами и таймерами.
