# Для чего используется модификатор reified в Kotlin?

### Категории

Android, Behavioral, Mobile, Async, Reified, Generics, Kotlin, Language features

### Краткий ответ

**Для чего используется модификатор reified в Kotlin?**

- Специфично для **inline функций**  
- Позволяет иметь **доступ к типу T во время выполнения**  
- Решает проблему **стёртых типов** в дженериках JVM  
- Даёт возможность использовать **операторы is и cast с T**  
- Упрощает рефлексию и создание экземпляров с дженериками  
- Позволяет писать более безопасный и лаконичный код  
- Применяется для реализации универсальных *inline*-шаблонов на основе типов  

Итого: **runtime-доступ к обобщённым типам** через inline + reified для обхода ограничений JVM.

### Подробный ответ
#### Основной ответ
В Kotlin модификатор **`reified`** используется в контексте **inline-функций** с целью сохранить информацию о типе во время выполнения, то есть сделать тип параметра доступным на этапе runtime. Обычно из-за **статической типизации JVM** информация о параметризованных типах стирается (type erasure), и напрямую работать с типом невозможно. `reified` избавляет от этой проблемы, позволяя использовать тип как полноценный объект, например, для вызова `is` или `class.java`.

#### Ключевые моменты
- **Применяется только в inline-функциях**: Kotlin требует, чтобы функция была инлайновой, т.к. при компиляции код копируется с реальным типом.
- **Позволяет работать с типом T внутри функции**: можно получить `T::class`, `T::class.java`, или сделать проверку `x is T`.
- **Упрощает создание обобщённых утилит**: например, десериализацию JSON или получение классов для рефлексии без явного передачи `Class<T>`.

#### Практический контекст
Часто используется в функциях библиотеки, например, в `inline fun <reified T> Gson.fromJson(json: String): T`, где не нужно передавать класс десериализуемого объекта вручную, что улучшает читаемость и снижает количество boilerplate кода. В React, Retrofit или других фреймворках на Kotlin также применяют `reified` для создания удобных API с generics.
