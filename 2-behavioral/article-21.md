# Перечислите основные методы объектов?

### Категории

Promises, Async, Frontend, Templating, Interview, Behavioral

### Краткий ответ

**Основные методы объектов в JavaScript**

- **Object.keys(obj)** — возвращает массив ключей (имен свойств) объекта  
- **Object.values(obj)** — возвращает массив значений всех свойств  
- **Object.entries(obj)** — возвращает массив пар [ключ, значение]  
- **Object.assign(target, ...sources)** — копирует свойства из источников в целевой объект (поверхностное клонирование)  
- **Object.freeze(obj)** — делает объект неизменяемым (замораживает)  
- **Object.hasOwn(obj, prop)** — проверяет, есть ли собственное свойство у объекта  
- **Object.getPrototypeOf(obj)** — возвращает прототип объекта  
- **Object.create(proto, descriptors)** — создаёт новый объект с указанным прототипом и свойствами  
- **Object.defineProperty(obj, prop, descriptor)** — определяет или изменяет свойство с детальной настройкой (геттеры, сеттеры, перечислимость)

---

Эти методы покрывают основные операции: получение свойств, клонирование, проверка, изменение метаинформации и работу с прототипами, что важно при манипуляции и расширении объектов в JS.

### Подробный ответ
#### Основной ответ
В JavaScript **объекты** представляют собой коллекции пар ключ-значение, и у них есть ряд встроенных методов, которые позволяют управлять, модифицировать и манипулировать этими данными. Основные методы объектов включают методы для получения ключей, значений, проверок наличия свойств и клонирования.

#### Ключевые моменты
- **Object.keys(obj)** — возвращает массив всех собственных enumerable ключей объекта. Полезно для итерации.
- **Object.values(obj)** — возвращает массив всех собственных enumerable значений объекта.
- **Object.entries(obj)** — возвращает массив пар [ключ, значение], используется для более гибкой работы с данными.
- **Object.hasOwn(obj, key)** (с ECMAScript 2022) — проверяет, является ли свойство собственным (не из прототипа) объекта.
- **Object.defineProperty(obj, prop, descriptor)** — позволяет задать или изменить детально свойства объекта (getter/setter, writable, enumerable).
- **Object.freeze(obj)** — делает объект неизменяемым: нельзя добавлять, удалять или менять свойства.
- **Object.assign(target, ...sources)** — копирует перечислимые свойства из одного или нескольких объектов в целевой объект, используется для поверхностного клонирования и смешивания.
- **Object.getPrototypeOf(obj)** — возвращает прототип объекта, полезно для работы с наследованием.

#### Практический контекст
В реальных проектах часто используют **Object.keys** или **Object.entries** при переборе данных, работают с **Object.assign** для объединения объектов конфигураций, а **Object.freeze** — для защиты константных данных. Новая функция **Object.hasOwn** более надежна, чем устаревший метод `obj.hasOwnProperty`, особенно при работе с унаследованными объектами.
