# Создаёт ли функция-конструктор объект всегда?

### Категории

Behavioral, JavaScript, Async, Constructor, Templating, Promise, Frontend

### Краткий ответ

**Создаёт ли функция-конструктор объект всегда?**

- область JavaScript, объекты  
- функция-конструктор с `new` создаёт новый объект, связывает `this` и возвращает его по умолчанию  
- если функция-конструктор **явно возвращает объект**, будет возвращён именно он, игнорируя стандартный новый объект  
- если возвращается **примитив**, возвращается новый объект по умолчанию  
- без `new` функция-конструктор не создаёт объект автоматически — `this` ссылается на глобальный объект или `undefined` в strict mode  
- итог: с `new` объект создаётся или заменяется возвращаемым объектом; без `new` — нет гарантии создания объекта  
- важно для корректного использования конструкторов и предотвращения ошибок  

Такой ответ демонстрирует глубокое понимание поведения функций-конструкторов и области видимости `this`.

### Подробный ответ
#### Основной ответ
Функция-конструктор в JavaScript **не всегда создаёт новый объект**, если при её вызове используется ключевое слово `new`. В обычном вызове без `new` функция просто выполняется как обычная, и новый объект автоматически не создаётся. Однако при вызове с `new` функция-конструктор создаёт новый объект и связывает его с `this`.

Важно понимать, что если функция-конструктор явно возвращает объект, то этот объект будет результатом вызова вместо созданного `this`. Если же возвращается примитив или ничего не возвращается, то возвращается автоматически созданный объект `this`.

#### Ключевые моменты
- При вызове с `new`, JavaScript:
  - Создаёт пустой объект
  - Связывает его с `this` функции-конструктора
  - Если функция не возвращает объект, возвращается `this`
- Если функция-конструктор возвращает объект, то этот объект будет возвращён вместо `this`
- При вызове без `new` никакого нового объекта не создаётся, `this` связывается с глобальным объектом (в строгом режиме `undefined`), что не создаёт ожидаемый объект

#### Практический контекст
В реальных проектах важно всегда использовать `new` с функциями-конструкторами или переходить на классы ES6 для явного создания объектов. Для предотвращения ошибки часто внутри функции-конструктора проверяют `if (!(this instanceof Constructor)) { return new Constructor(); }` чтобы гарантировать создание объекта.
