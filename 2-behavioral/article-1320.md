# Что такое команда git cherry-pick?

### Категории

Git, Version control, Behavioral, High load, Code management, VCS, Backend, Database

### Краткий ответ

**Что такое команда git cherry-pick?**

- Git команда для выбора отдельных коммитов  
- копирует конкретный коммит из другой ветки  
- применяет изменения в текущей ветке  
- работает с отдельными изменениями, не слияниями  
- используется для точечного исправления или переноса фич  
- может вызвать конфликты при расхождениях  
- удобно для выборочного переноса важных изменений без полной интеграции веток

### Подробный ответ
#### Основной ответ
**git cherry-pick** — это команда в системе контроля версий Git, которая позволяет выбрать один или несколько конкретных коммитов из одной ветки и применить их изменения в текущую ветку. По сути, она «перетаскивает» отдельные коммиты, не сливая всю ветку целиком, что удобно для выборочного переноса исправлений.

#### Ключевые моменты
- **Избирательное применение коммитов**: cherry-pick позволяет включить только нужные изменения, без интеграции полного контекста ветки или истории.
- **Создает новые коммиты**: при применении изменения повторяются как новые коммиты в целевой ветке, с новой хеш-суммой, сохраняя при этом автора и сообщение.
- **Разрешение конфликтов**: при несовпадении контекста могут возникать конфликты, которые нужно решать вручную, аналогично merge.
- Часто применяется для быстрого включения багфиксов из feature-веток в стабильные релизные ветки, когда полный merge нежелателен.

#### Практический контекст
В реальных проектах с большими командами git cherry-pick позволяет быстро портировать критичные исправления между ветками без излишних изменений, например, из ветки разработки в production. В Git 2.23+ улучшена поддержка конфликтов и указание диапазонов коммитов, что повысило удобство использования этой команды.
