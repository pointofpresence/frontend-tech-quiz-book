# Расскажи про различия JOIN на уровне множеств

### Категории

Behavioral, High load, Joins, Backend, Database, Sets

### Краткий ответ

**Различия JOIN на уровне множеств**  
- JOIN — операция объединения таблиц по ключам  
- **INNER JOIN** — пересечение множеств, только общие записи  
- **LEFT JOIN** — левая часть полна, дополняется правой (NULL если нет совпадений)  
- **RIGHT JOIN** — правая часть полна, дополняется левой (NULL если нет)  
- **FULL JOIN** — объединение множеств с сохранением всех элементов обеих таблиц  
- NULL выступает вместо отсутствующих элементов во второй таблице  
- Используется для разных целей: фильтрация (INNER), сохранение контекста (LEFT/RIGHT), полный охват (FULL)

### Подробный ответ
#### Основной ответ
В терминах теории множеств, операции **JOIN** в реляционных базах данных можно рассматривать как различные способы объединения, пересечения и фильтрации множеств строк двух таблиц. Основные виды JOIN отражают классические операции множеств, варьируясь по способу сопоставления и включения элементов из двух множеств (таблиц).

#### Ключевые моменты
- **INNER JOIN** соответствует операции пересечения множеств: возвращает только те элементы, которые присутствуют в обоих множествах (таблицах), то есть строки с совпадающими значениями по условию объединения.
- **LEFT JOIN (LEFT OUTER JOIN)** — это операция объединения с сохранением всех элементов из левого множества и соответствующих совпадений из правого, где отсутствующие пары заполняются NULL. Формально это левое дополнение до объединения.
- **RIGHT JOIN (RIGHT OUTER JOIN)** симметричен LEFT JOIN, но сохраняет все элементы из правого множества, добавляя данные из левого, либо NULL при отсутствии совпадений.
- **FULL JOIN (FULL OUTER JOIN)** — операция полного объединения множеств, включающая все элементы из обеих таблиц, где отсутствующие пары заполняются NULL. Это условный аналог объединения множеств с объединением по условию.

#### Практический контекст
Понимание JOIN через призму множеств помогает правильно выбрать тип соединения в зависимости от требуемой логики запроса. Например, для фильтрации строго совпадающих записей — применяют **INNER JOIN**. Для получения всех записей из одной таблицы с дополнительной информацией из другой — используют **LEFT JOIN** и т.п. В PostgreSQL 14+ и MySQL 8.0+ все эти типы оптимизированы для эффективного выполнения с большими данными.
