# Приведи пример: как решить 10 CPU-bound и 10 IO-bound задач на Python?

### Категории

Behavioral, Async, Concurrency, Backend, Optimization, CPU-bound, Parallelism, I/O-bound

### Краткий ответ

**Пример решения 10 CPU-bound и 10 IO-bound задач на Python**

- CPU-bound: задачи, нагружающие CPU, требуют масштабирования на ядра
- IO-bound: задачи с задержками ввода-вывода, тормозятся ожиданием

Решение:

1. **CPU-bound задачи:**
   - Использовать **multiprocessing.Pool** для распараллеливания на несколько процессов
   - Каждый процесс выполняет CPU-интенсивную функцию
   - Избегать GIL (Global Interpreter Lock) ограничений потоков
   - Пример кода:

```python
from multiprocessing import Pool

def cpu_task(x):
    # CPU-интенсивная операция
    return x * x

if __name__ == '__main__':
    with Pool(10) as p:
        results = p.map(cpu_task, range(10))
    print(results)
```

2. **IO-bound задачи:**
   - Использовать **asyncio** и асинхронные библиотеки (например, aiohttp для HTTP)
   - Эффективно обрабатывать ожидание IO без блокировки
   - Пример кода:

```python
import asyncio
import aiohttp

async def io_task(url):
    async with aiohttp.ClientSession() as session:
        async with session.get(url) as resp:
            return await resp.text()

async def main():
    urls = ['http://example.com'] * 10
    results = await asyncio.gather(*(io_task(url) for url in urls))
    print(results)

asyncio.run(main())
```

Вывод:

- Для CPU-bound задач нужен **параллелизм через процессы** (multiprocessing) из-за GIL
- Для IO-bound задач — **асинхронность через asyncio** повышает эффективность за счёт неблокирующего IO
- Такое разделение позволяет максимально эффективно использовать ресурсы и оптимизировать время выполнения задачи на Python

### Подробный ответ
#### Основной ответ
Для решения **CPU-bound** задач в Python обычно используют **мультипроцессинг** (модуль `multiprocessing`), потому что глобальная блокировка интерпретатора (GIL) ограничивает параллелизм потоков именно для CPU-интенсивных вычислений. Для **IO-bound** задач разумно применять **многопоточность** (`threading`) или **асинхронный код** (`asyncio`), поскольку эти задачи часто простаивают в ожидании ввода-вывода и GIL не мешает параллельному выполнению.

В вашем случае — 10 CPU-bound и 10 IO-bound задач — оптимально разделить их обработку:

1. Запускаете пул процессов (`multiprocessing.Pool`) для обработки 10 CPU-bound задач параллельно, эффективно используя ядра CPU.
2. Для 10 IO-bound задач используете либо `concurrent.futures.ThreadPoolExecutor`, либо `asyncio` в зависимости от специфики IO (сетевые запросы, файловые операции).

#### Ключевые моменты
- **GIL и CPU-bound задачи**: Python GIL позволяет только одному потоку выполнять байт-код одновременно, поэтому потоки не ускорят CPU-интенсивную работу, здесь именно процессный параллелизм нужен.
- **IO-bound задачи не блокируют GIL**: Когда поток ожидает ввода-вывода, GIL освобождается, другие потоки могут выполняться, что даёт выигрыш в производительности.
- **Асинхронность vs Потоки**: `asyncio` более эффективен при большом количестве коротких IO-задач с ожиданием (например, множество HTTP-запросов), тогда как потоки проще для интеграции с blocking IO.

#### Практический контекст
Пример для CPU-bound задач — вычисление больших чисел в пуле процессов (`multiprocessing.Pool.map`). Для IO-bound — загрузка страниц по HTTP параллельно через `ThreadPoolExecutor` или `asyncio` + `aiohttp`.

Такой подход максимально эффективно использует ресурсы системы: CPU полностью загружается на тяжёлой вычислительной работе, а IO не становится узким местом, не блокируя вычисления. В продакшене часто комбинируют оба механизма, например, с помощью очередей задач (celery), балансируя нагрузку.
