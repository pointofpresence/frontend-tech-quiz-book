# Какие протоколы передачи данных используются в современных приложениях? Чем они отличаются?

### Категории

Security, Data transfer, Networking, API, Protocols, Behavioral

### Краткий ответ

**Протоколы передачи данных в современных приложениях**

- HTTP/HTTPS: основной веб-протокол, основан на запрос-ответ; HTTPS — защищённый с TLS  
- WebSocket: двунаправленный, постоянное соединение, для real-time приложений  
- gRPC: протокол на основе HTTP/2 с бинарным форматом, эффективен для микросервисов  
- MQTT: легковесный протокол для IoT, работает поверх TCP, оптимизирован для малого трафика  
- TCP vs UDP: TCP — надёжный, ориентирован на соединение; UDP — быстрый, без подтверждения  
- REST vs SOAP: REST — лёгкий, использует HTTP, текстовый формат; SOAP — сложный, с расширенными стандартами безопасности  
- Отличия в надёжности, скорости и области применения (например, real-time vs запрос-ответ)

### Подробный ответ
#### Основной ответ
В современных приложениях чаще всего используются **HTTP/HTTPS**, **WebSocket**, **gRPC**, а также протоколы более низкого уровня, такие как **TCP** и **UDP**. Каждый из них решает разные задачи по передаче данных в зависимости от требований к скорости, надежности и двусторонней связи.

#### Ключевые моменты
- **HTTP/HTTPS** — самый распространённый протокол на основе текстового формата (обычно JSON или XML), работает поверх TCP, обеспечивает надежную доставку с подтверждениями, подходит для REST API и веб-приложений. HTTPS добавляет шифрование TLS.
- **WebSocket** — протокол для двунаправленного постоянного соединения поверх TCP, используется в real-time приложениях (чат, игры, финансовые платформы), снижает overhead постоянных HTTP-запросов.
- **gRPC** — современный RPC-фреймворк, использующий HTTP/2 и бинарный протокол protobuf, обеспечивает быструю сериализацию, поддержку стриминга и эффективную коммуникацию между микросервисами.
- **TCP** — надежный протокол транспортного уровня с контролем ошибок и порядком доставки, используется базово во многих приложениях.
- **UDP** — менее надежный, но более быстрый протокол без гарантий доставки и порядка, востребован в медиасервисах и играх для минимизации задержек.

#### Практический контекст
В веб-разработке доминирует HTTPS для классических запросов и WebSocket для real-time. Для микросервисов используются gRPC (в частности, в Kubernetes и с сервис-мешами, например, Istio). UDP часто применяется в видеоконференциях и потоковом видео, где важна минимальная задержка, а не полная надежность доставки. Выбор протокола определяется требованиями к задержкам, безопасности и типу данных.
