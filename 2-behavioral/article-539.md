# Что такое JVM, JDK, JRE?

### Категории

Behavioral, Live coding, Backend, Database, Interview, Technical

### Краткий ответ

**Что такое JVM, JDK, JRE?**  
- JVM (Java Virtual Machine) — среда выполнения, исполняет байт-код Java; обеспечивает **платформонезависимость**  
- JRE (Java Runtime Environment) — JVM + стандартные библиотеки для запуска Java-приложений  
- JDK (Java Development Kit) — JRE + инструменты разработки (компилятор javac, отладчик, утилиты)  
- JVM обрабатывает байт-код, JRE — для запуска, JDK — для разработки и запуска  
- JDK нужен разработчикам; JRE — конечным пользователям  
- JVM реализует управление памятью, сборку мусора, безопасность  
- Важно: без JVM байт-код Java не запустится  

Этот ответ показывает понимание и четкое разграничение каждого компонента.

### Подробный ответ
#### Основной ответ
**JVM (Java Virtual Machine)** — это виртуальная машина, которая исполняет байт-код Java, обеспечивая платформонезависимость за счёт абстрагирования от аппаратного и операционного окружения.  
**JRE (Java Runtime Environment)** — среда выполнения Java-приложений, включает JVM и базовые библиотеки, необходимые для запуска уже скомпилированных программ.  
**JDK (Java Development Kit)** — полный комплект для разработки на Java, содержит JRE, компилятор `javac`, отладочные и другие инструменты для написания и сборки кода.

#### Ключевые моменты
- JVM обеспечивает **кроссплатформенность** Java, интерпретируя байт-код, созданный компилятором.
- JRE — это минимальная среда для запуска Java-приложений, в ней нет инструментов разработки.
- JDK — включает всё необходимое для **создания, компиляции и отладки** Java-приложений; например, JDK 17+ отличается добавленными инструментами и улучшениями производительности.
- Важно понимать, что без JDK невозможно скомпилировать код, а без JRE — запустить уже готовое приложение.

#### Практический контекст
При локальной разработке всегда нужен **JDK**, например, версии OpenJDK 11 или 17, которые сегодня широко используются. В production-среде часто устанавливают только **JRE**, чтобы минимизировать размер и поверхность атаки на сервере, где запускаются Java-приложения.
