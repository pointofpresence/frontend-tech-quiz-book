# Какие методы относятся к REST?

### Категории

Behavioral, API, Backend, Database, REST, Testing

### Краткий ответ

**Методы REST**

- REST — архитектурный стиль для API
- Основные HTTP методы:
  - **GET** — получение ресурсов
  - **POST** — создание ресурсов
  - **PUT** — полное обновление ресурса
  - **PATCH** — частичное обновление ресурса
  - **DELETE** — удаление ресурса
- Каждый метод обладает **семантикой и идемпотентностью** (например, PUT идемпотентен, POST — нет)
- Используются для управления состоянием **через HTTP протокол**
- Обеспечивают стандартный интерфейс для CRUD операций

Этот набор методов — основа построения RESTful сервисов, обеспечивая **простоту и предсказуемость** API.

### Подробный ответ
#### Основной ответ  
**Методы, относящиеся к REST**, — это стандартные HTTP-методы, используемые для выполнения операций над ресурсами. Основные из них — **GET, POST, PUT, PATCH, DELETE, OPTIONS, HEAD**. Каждый метод соответствует конкретной цели и принципам REST, обеспечивая четкое семантическое разделение действий над ресурсами.

#### Ключевые моменты  
- **GET** — запрос на получение представления ресурса без изменения состояния сервера; идемпотентен и безопасен. Например, получение списка пользователей.  
- **POST** — создание нового ресурса или выполнение операции, изменяющей состояние; не идемпотентен. Часто используется для создания новых записей.  
- **PUT** — полное обновление ресурса, идемпотентен: повторные запросы приводят к одному и тому же результату. Если ресурс не существует, может создавать его.  
- **PATCH** — частичное обновление ресурса, часто используется для изменения отдельных полей без пересылки всей сущности.  
- **DELETE** — удаление ресурса, идемпотентен (несмотря на то, что первый вызов удаляет, последующие не изменяют состояние).  
- **OPTIONS** — запрос сервера о поддерживаемых методах, часто используется для CORS.  
- **HEAD** — идентичен GET, но без тела ответа, полезен для проверки заголовков.

#### Практический контекст  
В реальных RESTful API, например, на базе **Node.js с Express** или **Spring Boot**, эти методы строго соответствуют CRUD-операциям: создание (POST), чтение (GET/HEAD), обновление (PUT/PATCH), удаление (DELETE). Правильное использование этих методов обеспечивает предсказуемость и совместимость API с клиентами и инструментами.
