# Как создать асинхронный контекстный менеджер?

### Категории

Async, Backend, Behavioral, Context manager

### Краткий ответ

**Как создать асинхронный контекстный менеджер?**  
- Python, асинхронность, менеджеры контекста  
- Использовать методы **__aenter__ и __aexit__**  
- __aenter__ — асинхронный вход (await внутри)  
- __aexit__ — асинхронный выход (await внутри)  
- Класс или объект должен реализовывать эти методы  
- Менеджер позволяет работать с ресурсами async/await  
- Применяется для безопасного async-управления ресурсами (файлы, сети)

### Подробный ответ
#### Основной ответ
Асинхронный контекстный менеджер в Python создается с помощью методов `__aenter__` и `__aexit__`, которые управляют входом и выходом из асинхронного контекста. В отличии от обычных менеджеров контекста (`__enter__`/`__exit__`), эти методы являются **корутинами**, то есть поддерживают `await`. Такой менеджер используется совместно с оператором `async with`, что позволяет делать асинхронные действия при входе и выходе из блока кода.

#### Ключевые моменты
- **Определение методов:**  
  `__aenter__(self)` — асинхронно запускается при входе, часто инициализирует ресурсы;  
  `__aexit__(self, exc_type, exc, tb)` — асинхронно вызывается при выходе, освобождает ресурсы и обрабатывает исключения.
- **Применение:**  
  Используют для работы с асинхронными ресурсами, например, подключение к базе данных (в `asyncpg`), работу с асинхронными файлами или сетевыми сессиями.
- **Стандарт:**  
  В Python 3.5+ поддерживается асинхронный менеджер контекста, а в стандартной библиотеке примеры — `asyncio.Lock`, `aiofiles`.

#### Практический контекст  
Типичный пример — асинхронное открытие и закрытие сетевого соединения с ожиданием завершения операций внутри блока `async with`. Это улучшает читаемость кода и управление ресурсами без блокирования потока.

```python
class AsyncContextManager:
    async def __aenter__(self):
        print("Подключение...")
        await asyncio.sleep(1)  # эмуляция async действия
        print("Подключено")
        return self

    async def __aexit__(self, exc_type, exc, tb):
        print("Отключение...")
        await asyncio.sleep(1)
        print("Отключено")

### Использование:
### async with AsyncContextManager() as manager:
### # асинхронная работа
```

Такой подход свойственен современным asyncio-приложениям, где важно не блокировать event loop и грамотно управлять жизненным циклом ресурсов.
