# Как сделать так, чтобы картинка подгружалась с сервера только тогда, когда пользователь до нее долистал?

### Категории

Lazy loading, Performance, Async, Optimization, Behavioral

### Краткий ответ

**Как сделать ленивую загрузку изображения?**

- Веб-разработка, оптимизация загрузки  
- Ленивая загрузка (lazy loading) — загрузка при близости к viewport  
- Использовать нативный атрибут **loading="lazy"** для `<img>` (современные браузеры)  
- Для сложных случаев — **Intersection Observer API**  
  - Отслеживает появление изображения в области видимости  
  - При срабатывании заменяет `data-src` на `src`  
- Уменьшает трафик и ускоряет рендеринг страницы  
- Особенно полезно на страницах с большим количеством изображений или скроллом  
- Поддержка через популярные SPA-фреймворки (React, Vue) и библиотеки (lazysizes)  

Итог: эффект достигается благодаря отложенной загрузке через `loading="lazy"` или Intersection Observer, что оптимизирует UX и производительность.

### Подробный ответ
#### Основной ответ
Для того чтобы изображение загружалось с сервера только тогда, когда пользователь доскроллил до него, применяется техника **ленивой загрузки (lazy loading)**. Основная идея в том, чтобы откладывать загрузку ресурса (в данном случае картинки) до момента, когда она становится почти видимой в области просмотра пользователя. Это экономит трафик и ускоряет начальную загрузку страницы.

#### Ключевые моменты
- **Intersection Observer API** — современный и эффективный способ реализации lazy loading. Позволяет отслеживать появление элемента в viewport без постоянных дорогостоящих событий scroll.
- Альтернативный (более старый) способ — слушать события scroll и проверять положение изображения через `getBoundingClientRect()`, но это менее производительно и требует дебаунса.
- Для удобства в HTML появился атрибут `loading="lazy"` для тегов `<img>`, который нативно поддерживается в современных браузерах (сначала Chrome 76+, Firefox 75+, Safari 15.4+), что позволяет реализовать ленивую загрузку без JS.
  
#### Практический контекст
В React 18+ обычно используют Intersection Observer для контроля загрузки изображений, особенно если нужны дополнительные кастомные действия (анимации, placeholder). Для сайтов без JS опционально можно использовать `loading="lazy"`. В крупных проектах уменьшение количества сразу загружаемых ресурсов помогает достичь **время загрузки первого контента < 2 сек**, улучшая показатели Core Web Vitals. Часто это сочетают с техникой разгрузки изображений через CDN и оптимизацией форматов (WebP, AVIF).
