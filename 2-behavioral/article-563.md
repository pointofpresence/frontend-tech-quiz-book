# Что такое only и defer в Django ORM?

### Категории

Backend, Behavioral, Database, Optimization

### Краткий ответ

**Что такое only и defer в Django ORM?**

- оптимизация **выборки полей** модели
- **only()** загружает указанные поля, остальные — отложенно (лениво)
- **defer()** исключает указанные поля из запроса, остальные загружаются сразу
- уменьшают нагрузку на БД, сокращают объем данных из базы
- важны при работе с моделями с большим числом полей/тяжёлым данными
- уменьшают **трафик** и ускоряют работу ORM
- применяются для **оптимизации** запросов при чтении данных

### Подробный ответ
#### Основной ответ
В **Django ORM** методы `only()` и `defer()` используются для оптимизации выборки данных, влияя на то, какие поля модели загружаются из базы данных в памяти. Они позволяют контролировать набор полей, загружаемых при выполнении запроса, что экономит память и уменьшает нагрузку на сеть при работе с большими моделями.

- Метод `only()` указывает ORM загружать **только указанные поля**, остальные поля будут загружаться лениво при первом обращении (через дополнительный запрос).
- Метод `defer()` наоборот исключает из первоначальной выборки **указанные поля**, загружая остальные поля сразу, а deferred-поля — лениво.

#### Ключевые моменты
- `only('field1', 'field2')` — полезно, когда нужны ограниченные данные из модели, например, часто можно экономить 50-70% данных, сокращая трафик и время ответа.
- `defer('large_text', 'blob_field')` — используется, когда из модели нужно почти всё, кроме тяжелых полей. Это снижает первоначальную нагрузку, не жертвуя доступом ко всем данным.
- Ленивое подгружение deferred полей реализовано через отдельные SQL-запросы при первом обращении, что может влиять на производительность при множественных доступа к этим полям (N+1 проблема).
- Важно понимать, что с точки зрения API `only()` и `defer()` возвращают всё ту же модель, но с разной стратегией загрузки.

#### Практический контекст
В реальных проектах `only()` часто применяют для уменьшения размера ответа при списках объектов (например, в API или админке), где нужны ключевые поля, но не вся информация. А `defer()` удобен для игнорирования полей с большими blob-объектами или редко используемыми подробностями, что особенно полезно в отчетах и аналитике с большими таблицами. В Django 4+ оба метода интегрированы с QuerySet и поддерживаются во всех основных операциях ORM.
