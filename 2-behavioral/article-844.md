# Иммутабельность String и String Pool

### Категории

Theory, Behavioral, String, Memory management, Database, Optimization, Code review, Immutability

### Краткий ответ

**Иммутабельность String и String Pool**  
- String в Java — immutable (неизменяемый) объект  
- Иммутабельность обеспечивает безопасность потоков и кеширование  
- String Pool — спец. область в памяти для хранения строковых литералов  
- При создании строки через литерал JVM ищет в пуле, чтобы переиспользовать объект  
- Новый объект создаётся только, если такой строки в пуле нет  
- String Pool экономит память и ускоряет сравнение строк (==)  
- Иммутабельность String критична для корректной работы пула и безопасности приложения

### Подробный ответ
#### Основной ответ
**String в Java** являются **иммутабельными** объектами — после создания значение строки нельзя изменить. Благодаря этому, строки безопасны для использования в многопоточной среде без синхронизации, а также позволяют эффективно реализовать **String Pool** — пул строковых литералов для переиспользования одинаковых объектов. **String Pool** — это специальное место в памяти, где хранятся уникальные экземпляры строк. При создании строкового литерала сначала проверяется пул, и если такая строка уже есть, возвращается ссылка из пула, иначе создается новая строка и попадает в пул.

#### Ключевые моменты
- **Иммутабельность String** означает, что любые операции, изменяющие строку (например, concat, replace), на самом деле создают новый объект, а не изменяют существующий.
- **String Pool** снижает потребление памяти и увеличивает производительность за счёт повторного использования одних и тех же литеральных строк, что особенно важно при большом количестве одинаковых строк в приложении.
- При использовании `new String("value")` создаётся новый объект вне пула, в то время как литералы, объявленные в коде, автоматически попадают в пул. Чтобы явно добавить строку в пул, используется метод `intern()`.
- Иммутабельность обеспечивает безопасность пула: если строки могли бы меняться, ссылки в пуле стали бы некорректными и вызывали бы баги.

#### Практический контекст
В реальных проектах **иммутабельность строк** помогает избежать сложностей с синхронизацией, а **String Pool** существенно оптимизирует работу с большими текстами и повторяющимися строками — это особенно критично в высоконагруженных системах или при работе с постоянными данными (ключи, идентификаторы). В Java версии 8+ пул строк оптимизирован для лучшей производительности и меньшего расхода памяти.
