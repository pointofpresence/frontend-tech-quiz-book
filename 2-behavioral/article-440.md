# Как в Next.js добавляются скрипты, которые должны выполняться до рендера страницы?

### Категории

Behavioral, Nested structures, Script injection, Rendering, Frontend, UI components, Page lifecycle, Forms

### Краткий ответ

**Как в Next.js добавляются скрипты, которые должны выполняться до рендера страницы?**

- Next.js использует компонент **`<Head>`** из `next/document` для вставки скриптов в `<head>`
- Для выполнения скриптов **до рендера** применяют кастомный `_document.js`
- В `_document.js` можно определить метод `getInitialProps` и добавить скрипты с `dangerouslySetInnerHTML` или через теги `<script>`
- Скрипты в `<Head>` загружаются **до** отрисовки страницы в браузере
- Можно использовать встроенный компонент **`next/script`** с атрибутом `strategy="beforeInteractive"` для раннего запуска
- Такой подход гарантирует выполнение JS до монтирования React-компонентов
- Применяется для инициализации аналитики, сторонних библиотек, polyfills

Итого, основа — кастомный `_document.js` и `<Script strategy="beforeInteractive">` для управления порядком загрузки и выполнения скриптов до рендера.

### Подробный ответ
#### Основной ответ
В **Next.js** для добавления скриптов, которые должны выполняться **до рендера страницы**, стандартным и рекомендуемым способом является использование встроенного компонента **`next/script`** с параметром `strategy="beforeInteractive"`. Этот параметр указывает, что скрипт нужно загрузить и выполнить до того, как React начнёт гидрировать или рендерить страницу на клиенте. Такой подход особенно полезен для скриптов, от которых зависит корректная работа самого приложения или сторонних библиотек, которые должны инициализироваться раньше.

#### Ключевые моменты
- **`next/script`** — компонент Next.js с механизмом автоматической оптимизации загрузки скриптов (динамическая подгрузка, дедупликация).
- Параметр `strategy="beforeInteractive"` гарантирует выполнение скрипта **сразу после загрузки**, до клиентской гидрации — примерно как скрипт в `<head>` с `defer=false`.
- Альтернативные стратегии: `"afterInteractive"` (выполняется после гидрации, подходит для не критичных скриптов) и `"lazyOnload"` (загрузка после полной загрузки страницы).
- Если нужен **классический скрипт в `<head>`**, можно вставлять теги `<script>` вручную через `next/head`, но тогда теряется контроль и оптимизации `next/script`.
- Для серверного кода можно вставлять скрипты через `_document.js`, но это статичный вариант и не поддерживает оптимизации.

#### Практический контекст
В проектах на Next.js 12+ (в том числе в React 18) часто используют `next/script` с `strategy="beforeInteractive"` для аналитики, интеграции с A/B тестами или инициализации SDK, требующих загрузки до рендера (например, Google Tag Manager). Это улучшает UX, снижая мигания или ошибки из-за отсутствия критичных скриптов при первом отображении страницы.
