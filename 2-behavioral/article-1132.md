# Что такое упаковка (boxing) и распаковка (unboxing) в .NET и зачем они нужны?

### Категории

Boxing, Behavioral, Unboxing, Performance, Memory management, Backend, Database

### Краткий ответ

**Упаковка (boxing) и распаковка (unboxing) в .NET**

- область: работа с памятью и типами в .NET
- **упаковка**: преобразование значения типа-значения (value type) в ссылочный тип (object или интерфейс)
- механизм: выделение объекта в управляемой куче, копирование значения внутрь объекта
- **распаковка**: обратное преобразование — извлечение значения из объекта обратно в тип-значение
- применяется для совместимости value types с API, работающими со ссылочными типами (например, коллекции без обобщений)
- упаковка — дорогая операция по производительности и памяти, из-за аллокации в куче и копирования
- распаковка требует проверки типа и безопасности
- оптимизация: в новых версиях .NET уменьшен накладной расход, рекомендовано минимизировать boxing/unboxing

Итог: boxing/unboxing позволяют использовать типы-значения в контекстах, где ожидаются объекты, обеспечивая типовую совместимость, но влияют на производительность.

### Подробный ответ
#### Основной ответ
**Упаковка (boxing)** в .NET — это процесс преобразования значения типа-значения (value type), например `int` или `struct`, в объект (типа-ссылки, reference type). При этом создаётся объект в куче, и значение копируется туда. **Распаковка (unboxing)** — обратное действие: извлечение значения из объекта обратно в тип-значение. Эти операции нужны для того, чтобы можно было работать с value types как с объектами, например, когда value type передаётся в метод, ожидающий `object`.

#### Ключевые моменты
- **Value types** (например, `int`, `bool`, `struct`) хранятся в стеке и имеют фиксированный размер, а **reference types** — в куче с управляемой памятью.
- Boxing создаёт новый объект в куче — это сравнительно дорогая по ресурсам операция, важно минимизировать её в производительном коде.
- Unboxing требует точного соответствия типа, иначе будет выброшено исключение `InvalidCastException`.
- В .NET начиная с версии 2.0 и выше, с появлением обобщений (generics), упаковка используется реже, так как generics позволяют работать с value types напрямую без упаковки.

#### Практический контекст
В реальных проектах упаковка часто встречалась в классических коллекциях (`ArrayList`) до .NET 2.0, где можно было хранить только объекты. Обобщения (`List<T>`) решают эту проблему и снижают накладные расходы упаковки/распаковки, что критично для производительных приложений. Оптимизация, минимизирующая boxing/unboxing — важная часть профилирования .NET-приложений.
