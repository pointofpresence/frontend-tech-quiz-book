# Как работает git rebase?

### Категории

### Краткий ответ

**Как работает git rebase?**

- Git rebase — операция для **перемотки** ветки  
- Переносит коммиты поверх другой базовой ветки  
- Выполняет **повторное применение** изменений по одному  
- Сохраняет **чистую, линейную** историю  
- Используется для **избегания слияний (merge commits)**  
- Может вызвать конфликты, решаются вручную  
- Улучшает читаемость истории, полезен для **подготовки PR**

### Подробный ответ
#### Основной ответ
**Git rebase** — это операция в Git, которая позволяет перенести или "переписать" последовательность коммитов с одной ветки на другую, изменяя их базу. По сути, это способ интегрировать изменения из одной ветки, применяя коммиты поверх другой, создавая линейную историю коммитов без merge-commit. Rebase часто используется для упрощения истории и подготовки ветки к слиянию.

#### Ключевые моменты
- **Принцип работы**: Git последовательно "отъединяет" коммиты от исходной ветки, затем повторно применяет их на базу целевой ветки. При этом коммиты получают новые хеши, поскольку меняется их родитель.
- **Варианты использования**:  
  - **Interactive rebase (`git rebase -i`)** — дает контроль над историей, позволяет менять порядок коммитов, объединять их (squash), редактировать сообщения или вовсе удалять коммиты.  
  - **Rebase на ветку upstream** — помогает держать локальную ветку актуальной без создания merge-коммитов, что упрощает историю.
- **Риски**: Переписывание публичной истории (ветки, которая уже была опубликована или используется коллегами) может привести к конфликтам и необходимости губительных force-push (push --force), что требует аккуратности и согласованности в команде.

#### Практический контекст
В проектах с гибкой ветвлением (feature branches) rebase используют перед созданием pull request, чтобы история была чистой и понятной, а командные code review — проще. В React 18+ и других активно развивающихся проектах держать «чистую» историю особенно важно для быстрого поиска багов и понимания изменений. Часто rebase сочетают с инструментами CI/CD и Git hooks для автоматизации контроля качества.
