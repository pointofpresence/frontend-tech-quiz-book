# Как интерпретатор Python обрабатывает строковые и числовые значения в JSON?

### Категории

Behavioral, UI testing, Data parsing, Backend testing, Automation, JSON, Testing

### Краткий ответ

**Как интерпретатор Python обрабатывает строковые и числовые значения в JSON?**

- **Область:** десериализация JSON в Python  
- строки JSON → тип **str** в Python  
- числа JSON:  
  - целые → тип **int**  
  - дробные → тип **float**  
- парсинг выполняет модуль **json** (функция json.loads)  
- преобразует JSON-тип → эквивалентный Python-тип  
- точность чисел зависит от формата (float — ограничена)  
- применяется для дальнейшей работы с данными в Python (в объектах, словарях)  
- поддерживает вложенные структуры (списки, словари) с элементами чисел и строк  

Итог: Python встроенно преобразует JSON строки и числа в соответствующие типы **str, int, float**, обеспечивая удобную работу с данными.

### Подробный ответ
#### Основной ответ
В Python обработка JSON-строк происходит через стандартный модуль **json**, который реализует парсинг и генерацию JSON. При десериализации (функция `json.loads`) JSON-строка преобразуется в соответствующие Python объекты. Числовые типы JSON (целые числа и числа с плавающей запятой) конвертируются в Python `int` и `float` соответственно, а строковые значения JSON – в стандартные Python строки `str`.

#### Ключевые моменты
- В JSON числа могут быть только целыми или с плавающей точкой; при загрузке, например, `42` превратится в `int`, а `3.14` — в `float`. При этом Python автоматически различает эти типы.
- Строки JSON декодируются как Unicode строки Python (`str` в Python 3), что позволяет корректно обрабатывать любые UTF-8 символы — благодаря внутренней поддержке `unicode`.
- Обработка происходит с учётом стандарта RFC 8259: числа в JSON не могут содержать NaN или Infinite, и `json` модуль в Python учитывает эти ограничения. При возникновении запрещённых значений будет выброшена ошибка при загрузке.
- При сериализации обратно в JSON (функция `json.dumps`), числа Python конвертируются в соответствующие числовые литералы JSON, строки — в JSON-строки с необходимым эскейпингом.

#### Практический контекст
В реальных проектах при интеграции с внешними API данные часто приходят в формате JSON. При использовании `json.loads` в Python мы получаем привычные `dict`, `list`, `str`, `int` и `float`, что упрощает дальнейшую работу с данными (арифметика, поиск по строкам). Например, при получении API ответа с полями "price": 123.45 и "name": "Машина" в Python мы сразу можем работать с типами `float` и `str` соответственно без дополнительного кастинга.
