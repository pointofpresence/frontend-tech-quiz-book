# В чём разница между ArrayList и LinkedList? Какие операции эффективны для каждой из них?

### Категории

Theory, Behavioral, Performance, Backend, Database, Optimization, Data structures, Collections

### Краткий ответ

**Разница между ArrayList и LinkedList, эффективные операции**

- **ArrayList**: основан на **динамическом массиве**, хранит элементы в смежных ячейках памяти  
- Быстрый доступ по индексу: **O(1)**  
- Вставка/удаление в конце: **O(1)** (амортизированно)  
- Вставка/удаление в середине: **O(n)** из-за сдвигов элементов  

- **LinkedList**: реализован как **двусвязный список**, хранит узлы с ссылками на соседей  
- Доступ по индексу: **O(n)**, требуется проход по ссылкам  
- Вставка/удаление в любом месте: **O(1)** при наличии ссылки на узел (эффективно для очередей/стеков)  
- Подходит для частых вставок/удалений в середине  

- Выбор зависит от сценария:  
  - Для частого случайного доступа — **ArrayList**  
  - Для частых вставок/удалений — **LinkedList**  

- Коллекции Java: лучше ArrayList для **быстрого чтения**, LinkedList — для **динамической модификации**  

Итог: **ArrayList — быстрый индексированный доступ, LinkedList — эффективные вставки/удаления.**

### Подробный ответ
#### Основной ответ
**ArrayList** и **LinkedList** — это две разные реализации интерфейса **List** в Java, которые используют разные внутренние структуры данных и оптимизированы под разные операции. **ArrayList** основан на динамическом массиве, а **LinkedList** — на двусвязном списке.

#### Ключевые моменты
- **ArrayList** эффективно работает с операциями доступа по индексу — get и set выполняются за **O(1)** благодаря прямому обращению к элементам массива. Однако вставка или удаление элементов в середине списка требует сдвига последующих элементов, что занимает **O(n)** по времени.
- **LinkedList** обеспечивает быструю вставку и удаление элементов в начале, в конце и в середине списка — операции add и remove выполняются за **O(1)** при наличии ссылки на нужный узел. Доступ по индексу медленнее — **O(n)**, потому что приходится последовательно обходить узлы.
- В **ArrayList** используется меньше памяти на элемент, так как в LinkedList у каждого узла хранится две дополнительные ссылки (на предыдущий и следующий элементы), что увеличивает расход памяти.
  
#### Практический контекст
В React приложениях, серверных сервисах на Java или Android чаще выбирают **ArrayList** для хранения коллекций с частым рандомным доступом и редкими модификациями, например, список пользователей. **LinkedList** актуален в ситуациях, когда часто добавляют или удаляют элементы в середине или начале коллекции, например, при реализации очередей или стэков с частыми вставками и удалениями.

--- 
**Итог:**
- Для быстрых случайных обращений — **ArrayList**
- Для эффективных вставок и удалений — **LinkedList**
