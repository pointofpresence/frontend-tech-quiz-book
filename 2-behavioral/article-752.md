# Что такое стирание типов (type erasure) в дженериках?

### Категории

Behavioral, Java, Type erasure, Backend, Generics, Type system

### Краткий ответ

**Что такое стирание типов (type erasure) в дженериках?**  
- механизм компиляции дженериков, убирающий конкретные типы  
- дженерик-код компилируется в один общий байт-код без информации о типах  
- обеспечивает обратную совместимость с ранними версиями JVM или средами без дженериков  
- конкретные типы заменяются на их **ограничения** (например, Object) в коде байт-кода  
- исключает проверку типов во время выполнения, проверка происходит **на этапе компиляции**  
- снижает размер сгенерированного кода, но усложняет рефлексию с точными типами  
- широко применяется в Java, отличается от подхода reified types в Kotlin  
   
Type erasure — ключевой принцип реализации дженериков в Java для обеспечения совместимости и эффективности.

### Подробный ответ
#### Основной ответ
**Стирание типов (type erasure)** — это техника, используемая в некоторых языках программирования с дженериками (например, в Java), при которой информация о параметрах типа удаляется (стиpается) на этапе компиляции. В рантайме остается только исходный базовый или raw-тип без конкретизации дженериков. Это позволяет обеспечить обратную совместимость с более старыми версиями виртуальной машины, где дженерики не поддерживались.

#### Ключевые моменты
- В Java, например, дженерики реализованы с помощью стирания типов, то есть **информация о типе параметра теряется после компиляции**, и во время выполнения JVM работает с обобщённым типом как с обычным классом.
- При этом компилятор вставляет **кастинги и проверки в местах, где нужны конкретные типы**, что сохраняет типобезопасность, но вызывает дополнительные операции времени исполнения.
- Главный trade-off — это **обратная совместимость** и возможность интеграции новых решений в уже существующую платформу, но цена — невозможность использовать параметры типа напрямую в рантайме (например, нельзя узнать класс параметра T через reflection без дополнительной информации).

#### Практический контекст
В реальных проектах это означает, что в Java нельзя сделать `new T()` или получить класс T напрямую из дженерика. Разработчики используют обходные пути — например, передачу `Class<T>` в конструктор, чтобы сохранить типовую информацию. В отличие от Java, в таких языках как C# состояние типов сохраняется в рантайме благодаря реализации дженериков через reified types, что расширяет возможности дженериков, но требует иной реализации рантайма.
