# Что такое Not Canvas фреймворк/подход?

### Категории

Teamwork, Game development, Process, Architecture, Behavioral

### Краткий ответ

**Что такое Not Canvas фреймворк/подход?**  
- концепция визуализации вне HTML Canvas  
- акцент на DOM и CSS для графики  
- использует векторные элементы (SVG, CSS shapes)  
- упрощает доступность и SEO  
- облегчает интеграцию с веб-инструментами  
- меньше требований к рендерингу GPU  
- подходит для интерактивных, адаптивных интерфейсов

### Подробный ответ
#### Основной ответ
**Not Canvas** — это концепция и подход к веб-разработке и графике, которая ставит своей целью отказаться от использования HTML5 `<canvas>` как основного средства для визуализации и интерактивного рендеринга. Вместо этого применяется более декларативный, компонентный и семантический подход с использованием нативных DOM-элементов, CSS и современных веб-технологий, обеспечивающих лучшую доступность, SEO и производительность.

#### Ключевые моменты
- **Декларативная визуализация через DOM**: вместо императивного рисования на canvas с API, интерфейсы строятся с помощью обычных элементов DOM, что упрощает управление состоянием и стилизацию.
- **Улучшенная интеграция с браузерами**: обычный DOM проще индексировать поисковыми системами и поддерживать вспомогательные технологии, что обычно проблематично с Canvas.
- **Лучше для UI и анимаций**: CSS трансформации и WebGL используются отдельно, а не как единственный рендер-слой, что повышает гибкость.
- Обычно данный подход применяется в **React**, **Vue** и других фреймворках через имитацию графики через SVG или CSS, а не низкоуровневое рисование.

#### Практический контекст
В крупных проектах часто избегают `<canvas>` из-за трудностей с управлением состоянием графики, тестированием и accessibility. Not Canvas подход позволяет создавать адаптивные, легко тестируемые компоненты интерфейса с прозрачной структурой DOM, при этом сохраняя высокую интерактивность через современные технологии типа SVG, WebGL и CSS-анимаций.
