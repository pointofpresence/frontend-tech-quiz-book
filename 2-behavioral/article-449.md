# Как подписаться на событие клика кнопки в DOM?

### Категории

Behavioral, JavaScript, Interactivity, Event handling, DOM, Frontend

### Краткий ответ

**Как подписаться на событие клика кнопки в DOM?**

- JavaScript, DOM-события  
- Основной метод — **addEventListener**  
- Синтаксис: `element.addEventListener('click', handler)`  
- Обработчик принимает объект события (event)  
- Позволяет навешивать несколько слушателей  
- Поддерживается в современных браузерах  
- Используется для реакции на взаимодействие пользователя

### Подробный ответ
#### Основной ответ
В браузере для подписки на событие клика по кнопке в DOM обычно используют метод **addEventListener**. Он позволяет назначить функцию-обработчик, которая будет вызвана при наступлении события `click`. Это современный и гибкий способ работы с событиями, поддерживаемый всеми актуальными браузерами.

Пример базового кода:

```js
const button = document.querySelector('button');
button.addEventListener('click', (event) => {
  console.log('Кнопка нажата', event);
});
```

#### Ключевые моменты
- Метод **addEventListener** позволяет добавлять несколько обработчиков на одно событие, не перезаписывая предыдущие, в отличие от назначения напрямую свойства `onclick`.
- В обработчике доступен объект события **Event**, содержащий полезные данные (например, target, координаты клика, информацию о клавишах).
- Можно управлять фазами событий (capture и bubble), задавая третий аргумент в addEventListener (обычно `false` — обработка в фазе всплытия).
- Для отписки от события используют **removeEventListener** с той же функцией.

#### Практический контекст
Этот подход часто применяют в React 18+ в сочетании с refs для прямого доступа к DOM-элементам или в vanilla JS для управления интерактивностью без фреймворков. Также важно учитывать нюансы делегирования событий и производительности при подписке на большое число элементов — тогда используют event delegation на родительском элементе.
