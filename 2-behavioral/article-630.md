# Как пропустить тест? (@pytest.mark.skip)

### Категории

Behavioral, Python, Automation, Skip tests, Testing

### Краткий ответ

**Как пропустить тест с @pytest.mark.skip**

- **@pytest.mark.skip** — декоратор для пропуска теста в pytest  
- позволяет **отметить тест, который не нужно запускать**  
- pytest **сообщит о пропуске** с указанием причины (если указана)  
- можно добавить параметр `reason="пояснение"`, чтобы объяснить пропуск  
- полезен для временного исключения тестов при известных проблемах или неподдерживаемых условиях  
- отличается от `skipif` тем, что пропускает всегда, а не по условию  
- используется для **чёткой коммуникации** в команде о статусе тестов  

Пример:

```python
@pytest.mark.skip(reason="Функционал временно недоступен")
def test_example():
    pass
```

### Подробный ответ
#### Основной ответ
В **pytest** для пропуска (skip) теста используется декоратор `@pytest.mark.skip`. Он позволяет пометить тест, который не должен выполняться при запуске набора тестов. Это удобно, например, когда тест временно неактуален, зависит от внешних условий или не исправлен, но при этом не хочется его удалять.

Использовать его просто: достаточно поставить перед функцией теста `@pytest.mark.skip(reason="Причина пропуска")`, где аргумент `reason` — обязательный для понимания, почему тест пропущен.

#### Ключевые моменты
- `@pytest.mark.skip` — статический пропуск, тест всегда пропускается, независимо от условий.
- Для условного пропуска существует `@pytest.mark.skipif(condition, reason="...")`, который пропускает тест, если `condition` истинно.
- В отчётах pytest помечает такие тесты как из пропущенных и выводит указанную причину, что облегчает поддержку тестов.
- `skip` полезен для игнорирования тестов при известных проблемах, без удаления из кода.

#### Практический контекст
Например, в проекте с pytest 7, если нужно временно исключить интеграционный тест из CI пайплайна из-за известного бага, ставят:
```python
import pytest

@pytest.mark.skip(reason="Fix later - известный баг #123")
def test_integration():
    ...
```
При таком запуске тест будет пропущен, а в отчёте появится явное указание на причину. Это помогает поддерживать чистоту тестового набора и избегать ложных падений.
