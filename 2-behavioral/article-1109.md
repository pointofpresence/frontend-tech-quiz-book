# В чём разница между WHERE и HAVING в SQL-запросах?

### Категории

Behavioral, Query optimization, Architecture, Database, Backend, Filtering, Query, Optimization, Data structures, Live coding, SQL

### Краткий ответ

**Разница между WHERE и HAVING в SQL-запросах**  

- SQL-запросы, фильтрация данных  
- **WHERE** фильтрует данные **до агрегации**  
- работает с строками таблицы напрямую  
- **HAVING** применяется **после агрегации** (GROUP BY)  
- фильтрует результаты агрегатных функций (COUNT, SUM)  
- WHERE нельзя использовать с агрегатами  
- HAVING нужен для условий на сгруппированные данные  
- практика: WHERE — для отдельных строк, HAVING — для групп  
- часто используются вместе для комплексной фильтрации

### Подробный ответ
#### Основной ответ
**WHERE** и **HAVING** — это два типа условий фильтрации в SQL, но они работают на разных этапах обработки данных. **WHERE** фильтрует строки **до** группировки и агрегирования, а **HAVING** — **после** группировки, применяясь к агрегированным результатам.

#### Ключевые моменты
- **WHERE** использует условие для отсеивания строк из таблицы до выполнения `GROUP BY`, применим исключительно к отдельным строкам без использования агрегатных функций (например, `COUNT()`, `SUM()`).
- **HAVING** предназначен для ограничения уже сгруппированных данных и часто содержит агрегатные функции, так как применяется к результатам после группировки.
- Вопрос оптимизации: фильтрация с помощью WHERE обычно эффективнее, потому что сокращает объем данных на раннем этапе, а HAVING — дополнительная фильтрация на агрегатном уровне.

#### Практический контекст
В реальных запросах, например, чтобы получить клиентов с общей суммой заказов больше 1000, сначала используем WHERE для фильтрации только активных клиентов, затем HAVING — чтобы отобрать группы с суммой заказов, превышающей порог. В PostgreSQL 13+, MySQL 8+ и других СУБД это стандартный и оптимальный паттерн для сложного аналитического SQL.
