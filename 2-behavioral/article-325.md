# Что такое SOAP?

### Категории

Protocol, Behavioral, API, Network, Integration, Web services, Senior

### Краткий ответ

**Что такое SOAP?**  
- протокол обмена сообщениями  
- основан на **XML** для структурирования данных  
- использует HTTP/HTTPS или SMTP как транспорт  
- поддерживает **форматирование запросов/ответов** и описание сервисов через WSDL  
- обеспечивает **надежность** и **безопасность** (WS-Security)  
- широко применяется в корпоративных интеграциях и веб-сервисах  
- хорош для сложных транзакций, но более громоздкий чем REST

### Подробный ответ
#### Основной ответ
**SOAP (Simple Object Access Protocol)** — это протокол обмена структурированными сообщениями в распределённых вычислениях, основанный на XML. Он предназначен для взаимодействия между различными приложениями по сети, обеспечивая стандартизированный способ обмена данными и вызова удалённых процедур (RPC) вне зависимости от платформы и языка программирования.

#### Ключевые моменты
- **Стандартизованный формат**: SOAP использует XML-сообщения с жёстко определённой структурой (envelope, header, body), что обеспечивает совместимость между разнородными системами.
- **Транспортная независимость**: протокол может работать поверх HTTP, SMTP, TCP и других транспортных протоколов, хотя чаще всего используется вместе с HTTP.
- **Поддержка расширенных возможностей**: SOAP поддерживает стандарты безопасности (WS-Security), транзакционности, маршрутизации и гарантированной доставки, что особенно важно для корпоративных решений.
- **Чёткая контрактность**: обычно сопряжён с WSDL, который описывает доступные операции, входные/выходные параметры и типы данных, облегчая автоматическую генерацию клиентского кода.

#### Практический контекст
В крупных корпоративных системах, таких как банковские сервисы и государственные порталы, SOAP часто используется благодаря строгим стандартам безопасности и надёжности передачи сообщений. Однако, для более лёгких или публичных API постепенно вытесняется более простым и легковесным REST, который опирается на JSON и HTTP без сложных стандартов.
