# Как выполнить глубокое копирование объекта?

### Категории

Immutable, Behavioral, Memory optimization, Algorithm, Practical task, Deep copy, Memory management, Backend, Practical, Objects, Frontend, Live coding, Object, Bugfixing

### Краткий ответ

**Как выполнить глубокое копирование объекта?**

- копирование объекта с рекурсивным копированием вложенных объектов  
- нельзя использовать простое присваивание или Object.assign (копируется только первый уровень)  
- JSON.parse(JSON.stringify(obj)) — быстрый, но не копирует функции, даты, undefined, циклы  
- рекурсивная функция с проверкой типов и обработкой массивов, объектов, Date, Map, Set  
- lodash.cloneDeep — проверенное библиотечное решение с поддержкой сложных типов  
- важно избегать копирования по ссылке для сохранения независимости данных  
- применимо при работе с состоянием в React, сериализацией, клонированием данных без мутаций

### Подробный ответ
#### Основной ответ
**Глубокое копирование объекта** — это процесс создания полной независимой копии исходного объекта, включая все вложенные объекты и структуры. В отличие от поверхностного копирования, где копируются ссылки на вложенные объекты, глубокое копирование повторно создает вложенные объекты, чтобы изменения в копии не влияли на оригинал.

#### Ключевые моменты
- **Рекурсивное клонирование**: Обычно глубокое копирование реализуют с помощью рекурсивного обхода всех свойств объекта — для каждого свойства проверяют, является ли оно примитивом или объектом, и создают новый экземпляр.
- **Обработка циклических ссылок**: Чтобы избежать бесконечной рекурсии при циклических ссылках, добавляют структуру хранения уже скопированных объектов (например, `Map`), которая позволяет повторно использовать скопированные элементы.
- **Инструменты и методы**: В простых случаях используют `JSON.parse(JSON.stringify(obj))` — он быстро работает, но пропускает функции, свойства с `undefined`, символы и не справляется с цикличными структурами. Для более сложных задач применяют специализированные библиотеки (например, Lodash с `_.cloneDeep`) или пишут кастомные функции.

#### Практический контекст
В React 18+ глубокое копирование часто нужно для работы с иммутабельностью состояния, в Node.js — при дублировании конфигураций, где важно избежать несинхронизированных изменений. При работе с большими структурами стоит учитывать производительность и поддерживать обработку циклов и нестандартных типов.
