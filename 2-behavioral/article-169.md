# Чем оператор spread (...) отличается от rest параметров?

### Категории

Bug fixing, Behavioral, JavaScript, React, Spread, Rest parameters, Debugging, TypeScript, Frontend

### Краткий ответ

**Отличие оператора spread (...) от rest параметров**

- Область: JavaScript, синтаксис функций и коллекций  
- **spread** — распаковывает элементы массива/объекта в списки аргументов или свойства  
- **rest** — собирает несколько аргументов или свойств в массив или объект  
- spread работает **при вызове** или создании литерала  
- rest применим только в **объявлении параметров функции** или для деструктуризации  
- spread расширяет, rest **агрегирует**  
- пример spread: `func(...arr)` → распаковка вызова  
- пример rest: `function(...args)` → сбор аргументов  
- ключевая разница — **направление передачи данных**: spread → расширение, rest → свертка  
- практическое значение: spread для передачи множества элементов, rest для работы с неопределенным числом аргументов или свойств  

Такое различие важно для управления аргументами и копированием коллекций.

### Подробный ответ
#### Основной ответ
Оператор **spread (...)** и **rest параметры (...)** выглядят одинаково синтаксически, но выполняют противоположные по смыслу задачи: **spread** распаковывает элементы из массива или объекта, а **rest** собирает оставшиеся элементы в массив или объект. Проще говоря, spread — это операция расширения (spread out), rest — сбор оставшихся (rest parameters) в одну сущность.

#### Ключевые моменты
- **Spread** используется для **разворачивания элементов** массива или свойств объекта в местах вызова функции или создания нового объекта/массива. Например, можно скопировать массив или передать множество аргументов функции.
- **Rest** применяется в определении функций или деструктуризации, чтобы **собрать оставшиеся** значения после сбора нескольких — это удобно для работы с переменным числом аргументов.
- В JavaScript (ES6) spread работает в местах вызова и литералах (`[...arr]`, `{...obj}`), а rest — только в параметрах функции или деструктуризации (`function(...args)`, `const {a, ...rest} = obj`).
- В React, например, spread часто используется для передачи props, а rest — для «выделения» нужных свойств и сбора остальных.

#### Практический контекст
В React 18 типично использовать spread для передачи всех пропсов компоненту `<Button {...props} />`. Rest позволяет получить в функции первые несколько параметров, а остальные сгруппировать для дальнейшей обработки: 
```js
function log(first, ...others) {
  console.log(first, others);
}
```
Это дает гибкость и читаемость, а также экономит код при работе с вариативным числом аргументов. Таким образом, оба оператора тесно связаны, но обладают разной семантикой и контекстом применения.
