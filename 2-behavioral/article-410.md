# Для чего нужны useMemo и useCallback? Как работает мемоизация?

### Категории

Behavioral, JavaScript, Memoization, Functions, Performance, React hooks, Optimization, Frontend, Live coding

### Краткий ответ

**Для чего нужны useMemo и useCallback? Как работает мемоизация?**  
React hooks, оптимизация производительности

- **useMemo** — мемоизация результата функции  
- **useCallback** — мемоизация функции (callback)  
- предотвращают ненужные пересчёты и рекомпозиции компонентов  
- мемоизация хранит вычисленные значения и возвращает их, если зависимости не изменились  
- реализуют проверку зависимостей (deps array) для обновления данных  
- снижают количество рендеров и тяжелых вычислений в React  
- полезны при передаче props и работе с контекстом, чтобы избежать лишних ререндеров  

Мемоизация — это кэширование результата функции по входным параметрам, чтобы вернуть сохранённое значение вместо повторного вычисления.

### Подробный ответ
#### Основной ответ
**useMemo** и **useCallback** — это React хуки, которые помогают оптимизировать производительность компонента, **мемоизируя** результаты вычислений или функции, чтобы избежать лишних пересчётов и ререндеров. **useMemo** мемоизирует результат вычисления значения, а **useCallback** — мемоизирует саму функцию, сохраняя её ссылку неизменной между рендерами.

Мемоизация — это техника сохранения ранее вычисленного результата функции для повторного использования, если входные данные не изменились. В React она важна для предотвращения ненужных обновлений компонентов и избежания повторных дорогостоящих вычислений.

#### Ключевые моменты
- **useMemo вычисляет и кеширует значение**: принимает функцию и массив зависимостей; пересчитывает значение только если зависимостям изменились, иначе возвращает сохранённое.
- **useCallback мемоизирует функцию**: полезен для передачи стабильно идентичных callback’ов в дочерние компоненты, что снижает их повторный рендер, особенно в сочетании с React.memo.
- **Мемоизация уменьшает нагрузку**: снижая количество повторных вычислений и рендеров, повышает производительность, особенно при сложных вычислениях или глубокой иерархии компонентов.

#### Практический контекст
Типично используют **useMemo** для тяжелых вычислений, например, фильтрации большого списка, а **useCallback** — для передачи событийных функций потомкам, чтобы избежать лишних ререндеров. В React 18 и выше эти хуки становятся ещё более актуальными для оптимизации в новых Concurrent режимах и задачах с высокой нагрузкой.
