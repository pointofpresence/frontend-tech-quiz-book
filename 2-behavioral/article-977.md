# Какие особенности имеют хуки useEffect и useLayoutEffect?

### Категории

Behavioral, React, Containerization, Async, Lifecycle, Performance, Frontend, Effect, Hooks

### Краткий ответ

**Особенности хуков useEffect и useLayoutEffect**

- React hooks для **побочных эффектов**
- **useEffect**: вызывается **после** рендера и отрисовки на экран  
- **useLayoutEffect**: вызывается **синхронно сразу после** вычисления DOM, до отрисовки  
- useEffect не блокирует отрисовку, useLayoutEffect блокирует (может замедлить UI)  
- useLayoutEffect полезен для измерений DOM и синхронизации перед отрисовкой  
- useEffect подходит для асинхронных операций и работы с API  
- Главное отличие — время вызова: useLayoutEffect приостановит рендер, useEffect — нет  

Итого: useEffect для **асинхронных** побочных эффектов после рендера, useLayoutEffect — для **синхронной** работы с DOM перед отрисовкой, чтобы избежать мерцаний и рассогласований UI.

### Подробный ответ
#### Основной ответ
**useEffect** и **useLayoutEffect** — это хуки React, позволяющие выполнять побочные эффекты в функциональных компонентах, но с разной таймингом выполнения. **useEffect** запускается после того, как React обновил DOM и отрендерил изменения, а **useLayoutEffect** срабатывает синхронно сразу после изменения DOM, но перед тем, как браузер отрисует изменения на экран.

#### Ключевые моменты
- **useEffect** оптимален для асинхронных и "побочных" задач: загрузки данных, подписок, таймеров. Выполняется асинхронно, не блокируя отрисовку, что улучшает производительность и восприятие пользователем.
- **useLayoutEffect** используется для синхронных операций, которые должны произойти до браузерной перерисовки — например, измерения размеров элементов, синхронизация скролла или манипуляции с DOM, влияющие на визуальное отображение. Задерживает перерисовку, что может приводить к блокировке UI при тяжелых операциях.
- В React 18+ во время серверного рендеринга **useLayoutEffect** не вызывается (для избежания ошибок), тогда как useEffect игнорируется при SSR, так как эффекты относятся к клиентскому рендерингу.

#### Практический контекст
Если нужно обновить внешний API или сделать асинхронный запрос — предпочтительнее использовать **useEffect**. Если нужно получить размеры DOM-элемента сразу после обновления и синхронно убедиться, что UI отрисовался корректно перед показом пользователю — useLayoutEffect помогает избежать мерцания или скачков UI. Например, анимации часто используют **useLayoutEffect** для точной синхронизации. Однако частое и долгие операции в useLayoutEffect могут ухудшить производительность и отзывчивость интерфейса.
