# Какие флаги есть у куки (например, HttpOnly, Secure)?

### Категории

Cookies, HTTP, Security, Behavioral, Web, Frontend

### Краткий ответ

**Флаги у cookie: назначение и безопасность**

- **HttpOnly**: запрещает доступ к cookie из JS, предотвращает XSS-атаки  
- **Secure**: передача cookie только по HTTPS, защищает от перехвата  
- **SameSite**: ограничивает отправку cookie между сайтами, уменьшает CSRF  
  - возможные значения: Strict, Lax, None  
- **Path**: ограничивает область видимости cookie по URL  
- **Domain**: задаёт доступность cookie для субдоменов  
- **Expires/Max-Age**: срок жизни cookie, после чего удаляется  
- **Priority** (редко): указывает приоритет хранения cookie браузером  

Эти флаги обеспечивают безопасность и контроль области действия cookie, минимизируют риски атак и утечки данных.

### Подробный ответ
#### Основной ответ
У **HTTP cookies** есть несколько важных флагов, которые контролируют их безопасность, область действия и поведение. Основные из них — **HttpOnly**, **Secure**, **SameSite**. Эти флаги помогают защитить куки от различных атак, таких как XSS и CSRF, а также настраивать, когда и как куки отправляются браузером к серверу.

#### Ключевые моменты
- **HttpOnly** — куки становится недоступным для JavaScript на стороне клиента (например, через `document.cookie`), что снижает риск кражи куки через XSS-атаки. Обычно используется для сессионных токенов.
- **Secure** — куки передаются только через защищённое соединение HTTPS, что предотвращает перехват куки в открытом трафике.
- **SameSite** — определяет, будет ли куки отправлен в кросс-доменных запросах. Имеет значения:
  - **Strict** — куки не отправляются при переходах с внешних сайтов (максимальная защита от CSRF, но может ухудшить UX),
  - **Lax** — более мягкий режим, позволяет отправлять куки при "безопасных" переходах (например, GET-запросы навигации),
  - **None** — куки всегда отправляются, требует включенного Secure.
- **Path** — задаёт путь на сервере, для которого куки действуют (например, `/app` будет действовать только для URL внутри `/app`).
- **Domain** — задаёт домен, которому куки принадлежат; по умолчанию это домен, с которого был установлен куки.
- **Expires / Max-Age** — определяет время жизни куки: `Expires` — дата истечения, `Max-Age` — время в секундах. Без этого куки считается сессионным и удаляется при закрытии браузера.

#### Практический контекст
В проектах с неотъемлемой безопасностью (финтех, админки) обязательно ставят **HttpOnly+Secure+SameSite=Strict** для сессионных куков. Для публичных сайтов часто используют **SameSite=Lax** для удобства авторизации и предотвращения CSRF. В новых спецификациях браузеров Account Service рекомендует всегда включать **SameSite** для лучшего контроля.
