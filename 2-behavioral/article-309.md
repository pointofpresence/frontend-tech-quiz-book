# Расскажи про Event-loop. Когда выполняется SetImmediate

### Категории

Behavioral, Microservices, Async, Event loop, Timers, Backend, Database, TypeScript

### Краткий ответ

**Event-loop. Когда выполняется SetImmediate**

- Event-loop — цикл обработки событий в Node.js  
- Управляет очередями: timers, I/O callbacks, idle, poll, check (SetImmediate), close callbacks  
- SetImmediate ставит колбек в очередь **check**, выполняется после всех I/O операций  
- Выполняется строго **после завершения текущей фазы poll**  
- Отличие от setTimeout(fn,0): SetImmediate запускается **быстрее и гарантированно после I/O**  
- Используется для выполнения кода сразу после I/O, обходя timers  
- Идеально подходит для разбивки тяжёлых операций и асинхронного контроля  

**Итог:** SetImmediate запускается на фазе check, после завершения poll, гарантируя выполнение после всех активных I/O операций текущего цикла event-loop.

### Подробный ответ
#### Основной ответ
**Event-loop** — это механизм, отвечающий за обработку асинхронных операций в средах вроде Node.js или браузеров. Он обеспечивает неблокирующее выполнение кода, позволяя продолжать работу приложения во время ожидания I/O, таймеров или других событий. **SetImmediate** — это функция в Node.js, которая ставит колбэк в очередь для выполнения после завершения текущей фазы event-loop, но перед следующей итерацией таймеров.

#### Ключевые моменты
- В **Node.js** event-loop состоит из нескольких фаз: timers, I/O callbacks, idle/prepare, poll, check и close callbacks.
- Колбэк, переданный в **setImmediate()**, выполняется в фазе **check**, которая идёт сразу после фазы poll.
- Если внутри текущей фазы event-loop вызвать setImmediate, его колбэк выполнится после завершения текущей фазы (обычно после операций ввода-вывода).
- **setImmediate** отличается от **setTimeout(fn, 0)** тем, что setTimeout ставится в фазу timers, и его выполнение может быть отложено, если event-loop занят.

#### Практический контекст
Типично, **setImmediate** используют для отложенного выполнения кода после I/O, чтобы не блокировать текущую фазу event-loop, например, для разбивки тяжелых вычислений или для гарантированного выполнения после завершения синхронного кода и операций ввода-вывода. В React Native или browser-окружении его нет, но в Node.js это удобный инструмент для контроля порядка асинхронных вызовов.
