# Чем HTTP отличается от HTTPS?

### Категории

Live coding, TypeScript, Network, Encryption, Protocol, API, Security, Behavioral

### Краткий ответ

**Чем HTTP отличается от HTTPS?**

- протоколы передачи данных в интернете  
- HTTP — **незащищённый** протокол  
- HTTPS — HTTP поверх **TLS/SSL** (шифрование)  
- HTTPS гарантирует **конфиденциальность** и **целостность** данных  
- HTTPS требует **сертификат** для подтверждения подлинности сервера  
- HTTPS снижает риски **перехвата** и **атаки "человек посередине"**  
- обязательный для безопасных сайтов: банки, интернет-магазины, авторизация

### Подробный ответ
#### Основной ответ  
**HTTP (HyperText Transfer Protocol)** — это протокол передачи данных между клиентом и сервером в сети без шифрования. **HTTPS** — это расширение HTTP с использованием **TLS/SSL** для шифрования соединения, обеспечивающее безопасность передачи данных. Основное отличие в том, что HTTPS гарантирует конфиденциальность, целостность и аутентичность данных, а HTTP передаёт данные в открытом виде.

#### Ключевые моменты  
- **Шифрование данных:** HTTPS использует TLS (например, TLS 1.2 или 1.3), который защищает трафик от перехвата, MITM-атак и подделки. HTTP не шифрует данные, что делает коммуникацию уязвимой.  
- **Аутентификация сервера:** HTTPS требует действующего SSL-сертификата, который удостоверяет подлинность сайта. Это помогает пользователям избежать фишинга и подмены. HTTP такой возможности не предоставляет.  
- **Порт по умолчанию:** HTTP работает через порт 80, HTTPS — через 443. Браузеры автоматически переключаются на HTTPS при наличии сертификата и настройки.  
- **Производительность:** HTTPS накладывает небольшую нагрузку из-за криптографических операций (handshake, шифрование), но в современных версиях TLS накладные расходы минимальны, и даже быстрее HTTP через HTTP/2.  

#### Практический контекст  
В реальных проектах HTTPS — стандарт для любого сайта, особенно где передаются личные данные (логины, платежи). Google и другие поисковики даже дают преимущество сайтам на HTTPS, а браузеры помечают сайты с HTTP как «Не безопасные», что влияет на доверие пользователей и SEO.
