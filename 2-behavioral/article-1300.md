# Отличия куки от session storage?

### Категории

Browser, HTTP, Security, Behavioral, Storage, State management, Frontend

### Краткий ответ

**Отличия cookie и sessionStorage**

- cookie — данные хранятся на стороне клиента в браузере, sessionStorage — в сессии вкладки  
- cookie автоматически отправляются на сервер с каждым запросом, sessionStorage — только в JS-коде браузера  
- cookie имеют ограничение по размеру (~4KB), sessionStorage — больше (~5-10MB)  
- cookie могут иметь срок жизни (expire), sessionStorage удаляется при закрытии вкладки  
- cookie используются для аутентификации и трекинга, sessionStorage — для временного хранения данных напрямую в браузере  
- cookie доступны и через JS, и серверу, sessionStorage — только через JS  
- sessionStorage изолирован по вкладкам, cookie общие для домена  

Итого: cookie — для длительного, серверно-ориентированного хранения с передачей на сервер, sessionStorage — для временного клиентского хранения на вкладке.

### Подробный ответ
#### Основной ответ
**Cookies** и **Session Storage** — это два разных способа хранения данных на стороне клиента, которые применяются в веб-разработке, но имеют принципиальные отличия по области видимости, сроку жизни и безопасности.

##### Отличия:
- **Cookies** — маленькие текстовые файлы (до 4 КБ), которые браузер отправляет вместе с каждым HTTP-запросом к серверу. Изначально созданы для идентификации сессии пользователя. Могут иметь дату истечения и отправляются на сервер автоматически.
- **Session Storage** — часть Web Storage API, где данные хранятся в сессии браузера конкретной вкладки и удаляются при её закрытии. Эти данные не отправляются на сервер, доступны только в JavaScript на клиенте.

#### Ключевые моменты
- **Область видимости:** Cookies доступны и на клиенте, и на сервере, так как автоматически прикрепляются к HTTP-запросам. Session Storage доступен только в браузере, по конкретной вкладке, что упрощает управление локальными данными без передачи на сервер.
- **Срок хранения:** Cookies могут быть как сессийными (удаляются при закрытии браузера), так и постоянными (со временем истечения). Session Storage живёт только в рамках одной сессии вкладки, исчезая при её закрытии.
- **Безопасность и производительность:** Cookies подвержены риску перехвата и атакам CSRF, поэтому для безопасности применяют HttpOnly и Secure флаги. Session Storage изолирован по origin, более защищён от межсайтовых атак, но данные доступны только для JavaScript.

#### Практический контекст
**Cookies** часто используют для аутентификации и хранения токенов с возможностью серверной проверки, а также для persistent preference. **Session Storage** подходит для временного хранения данных формы, состояний UI и других клиентских данных, когда не нужно синхронизировать с сервером или сохранять долго после закрытия вкладки.
