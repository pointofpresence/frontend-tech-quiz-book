# Как обработать ошибку в промисах без catch, используя только then?

### Категории

Async, Behavioral, Live coding, Error handling

### Краткий ответ

**Как обработать ошибку в промисах без catch, используя только then**

- Работа с **Promise** в JS, асинхронность  
- then принимает **два колбэка**: первый для успеха, второй для ошибки  
- Вместо catch можно передать второй аргумент в then  
- Пример: `promise.then(res => {...}, err => {...})`  
- Позволяет обработать ошибку внутри цепочки без отдельного catch  
- Такой подход снижает читаемость, less preferred  
- Полезно для компактной обработки ошибок без отдельного блока catch

### Подробный ответ
#### Основной ответ
Обработать ошибку в промисах без использования **catch** можно, используя второй параметр метода **then**, который служит для обработки ошибок. Метод **then** принимает два аргумента: первый — функция для успешного выполнения промиса, второй — для обработки ошибки. Такой подход эквивалентен использованию **catch**, но строгая цепочка с использованием только **then** может быть менее читаемой.

#### Ключевые моменты
- Второй параметр в **then(onFulfilled, onRejected)** обрабатывает ошибки, возникающие в самом промисе или предыдущем звене цепочки.
- Важный нюанс: если в обработчике ошибок не происходит корректной обработки или повторного бросания, ошибка считается обработанной и цепочка продолжается как успешная.
- Использование второго параметра **then** реже встречается в современных кодовых базах, так как **catch** улучшает читаемость и позволяет выделить обработку ошибок отдельно.

#### Практический контекст
В JavaScript (ES6+) для обработки исключений в промисах часто предпочитают `.then(...).catch(...)` или async/await с try/catch. Однако, если политика проекта или стиль кода требует исключительно использования **then**, можно писать так:

```js
fetch(url)
  .then(
    response => response.json(),
    error => {
      console.error('Ошибка сети или загрузки:', error);
      // Можно вернуть fallback-значение или повторно бросить ошибку
    }
  )
  .then(
    data => console.log('Данные обработаны', data),
    error => console.error('Ошибка при парсинге JSON:', error)
  );
```

Такой паттерн позволяет локально ловить ошибки на каждом шаге без **catch**, но требует внимательного управления потоком ошибок.

---

Этот подход демонстрирует хорошее знание возможностей промисов, показывает умение работать с ошибками нестандартным способом, что важно на интервью.
