# Что такое Collection Runner в Postman?

### Категории

Behavioral, API, Backend, Database, Automation, Testing

### Краткий ответ

**Что такое Collection Runner в Postman?**

- инструмент для запуска **наборов запросов** (коллекций)  
- позволяет автоматизировать **последовательное выполнение** API-запросов  
- поддерживает параметризацию через **переменные и CSV/JSON** данные  
- выводит детальные **отчёты и логи** каждого запроса  
- полезен для **тестирования API**, нагрузочного и сценарного запуска  
- ускоряет проверку бизнес-логики и интеграции  
- незаменим для **CI/CD** в автоматизации тестов API

### Подробный ответ
#### Основной ответ
**Collection Runner** в Postman — это инструмент, позволяющий запускать все запросы из коллекции последовательно или параллельно с возможностью задания переменных, параметров и повторов. Он широко используется для автоматизации тестирования API, нагрузочного тестирования и проверки различных сценариев работы сервера.

#### Ключевые моменты
- **Пакетное выполнение запросов**: Collection Runner может запускать десятки и сотни запросов в определённом порядке, что удобно для комплексного тестирования API.
- **Поддержка данных из файла**: можно подгружать CSV или JSON с данными переменных, что позволяет выполнять параметризованные запросы и тестировать разные варианты входных данных.
- **Отчеты и логирование**: после выполнения всех запросов доступен подробный лог с результатами, временем выполнения и статусами, что упрощает анализ и отладку.

#### Практический контекст
Использую Collection Runner в проектах для регрессионного тестирования API после изменений в backend (например, на PostgreSQL 14+). Это помогает убедиться, что новые версии не ломают существующий функционал, а также позволяет быстро запускать нагрузочные тесты с использованием большого объема данных через CSV.
