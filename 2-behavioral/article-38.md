# Чему будет равен this при вызове функции-конструктора?

### Категории

Promises, This keyword, Backend inquiry, Async, JavaScript, Behavioral

### Краткий ответ

**Чему будет равен this при вызове функции-конструктора?**

- Контекст: JavaScript, функция-конструктор  
- this ссылается на **новый объект**, создаваемый конструктором  
- происходит **автоматическое создание и возврат** объекта  
- this содержит поля и методы, задаваемые внутри конструктора  
- внутри конструктора this не равен глобальному объекту или undefined  
- если вернуть **явно объект**, то this заменяется на него  
- используется для инициализации экземпляров (объектов) классов или шаблонов  
- this обеспечивает создание независимых объектов с одинаковой структурой  

Итог: **this в конструкторе — это новый создаваемый объект, который будет возвращён при вызове new**.

### Подробный ответ
#### Основной ответ
При вызове функции-конструктора через оператор `new` значение `this` внутри функции будет ссылаться на **новый созданный объект**. То есть создаётся пустой объект, который затем связывается с прототипом конструктора, и `this` внутри функции указывает именно на этот объект. После выполнения тела конструктора он возвращает `this` (если не возвращено явно другое значение).

#### Ключевые моменты
- **Создание нового объекта**: вызов `new FunctionName()` создаёт пустой объект, связанный с `FunctionName.prototype`.
- **Привязка `this`**: внутри конструктора `this` указывает на этот только что созданный объект.
- **Возврат результата**: если в конструкторе явно не возвращён объект, возвращается именно `this`. Если возвращён объект — возвращается он, а не `this`; если возвращено примитивное значение — игнорируется и возвращается `this`.
- Такой подход отличает конструкторный вызов от вызова функции без `new`, где `this` зависит от контекста (например, `undefined` в строгом режиме или глобальный объект в нестрогом).

#### Практический контекст
В реальных проектах понимание контекста `this` при вызове конструктора критично для создания экземпляров классов и компонентов. Например, в ES5-конструкторах часто используем:

```js
function User(name) {
  this.name = name;
}
const user1 = new User('Alice');
```

Здесь `this` внутри `User` указывает на создаваемый объект `user1`. В ES6 синтаксис классов консистентно делает то же самое под капотом.
