# Можете описать, как работает формат соединения WebSocket?

### Категории

Communication, Backend, WebSocket, Async, Realtime, Protocol, Networking, API, Behavioral

### Краткий ответ

**Как работает формат соединения WebSocket?**

- протокол для **двунаправленного** взаимодействия по TCP  
- начинается с HTTP/HTTPS **рукопожатия** (handshake)  
- клиент отправляет HTTP-запрос с заголовком `Upgrade: websocket`  
- сервер подтверждает переход в режим WebSocket ответом 101 Switching Protocols  
- после handshake устанавливается **полно-дуплексное** соединение  
- сообщения обмениваются в формате **фреймов**, поддерживающих текст и бинарные данные  
- обеспечивает **низкую задержку** и эффективный обмен без постоянных запросов  
- используется для чатов, игры, лайв-обновлений, где нужна реактивность и постоянное соединение

### Подробный ответ
#### Основной ответ
**WebSocket** — это протокол сетевого взаимодействия, который обеспечивает двунаправленное, полнодуплексное соединение поверх одного TCP-соединения между клиентом и сервером. Это позволяет клиенту и серверу обмениваться данными в реальном времени без необходимости постоянно устанавливать новые HTTP-запросы, как в случае с традиционными запросами.

#### Ключевые моменты
- Соединение начинается с **HTTP-запроса на улучшение (upgrade request)**, где клиент посылает заголовок `Upgrade: websocket`. Сервер отвечает с подтверждением, если поддерживает WebSocket, и затем происходит переключение на WebSocket-протокол поверх того же TCP-соединения.
- После успешного установления соединения данные передаются в виде **фреймов**, которые могут быть текстовыми, бинарными или контрольными (например, для ping/pong). Формат фрейма стандартизирован в RFC 6455, и каждый фрейм содержит заголовок с длиной и маскировкой.
- Для безопасности и предотвращения злоупотреблений клиентские данные маскируются согласно протоколу (обязательная маскировка клиентом, в отличие от сервера), что препятствует атакам типа псевдокода.
- WebSocket позволяет поддерживать постоянное соединение с низкой задержкой (~10-50ms в оптимальных условиях), что критично для real-time приложений: чаты, игры, финансовые тикеры.

#### Практический контекст
В реальных проектах, например в React 18+ или Node.js, WebSocket часто используется через библиотеки типа `ws` или `Socket.IO`, которые дополнительно обеспечивают автоматический повтор соединений и обмен пользовательскими событиями поверх базового протокола. На уровне инфраструктуры часто применяют балансировщики с поддержкой WebSocket (Nginx, HAProxy) и SSL/TLS для безопасности.
