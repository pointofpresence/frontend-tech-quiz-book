# Как JavaScript работает с примитивами и объектами? В чём разница в поведении?

### Категории

Language fundamentals, Memory management, Primitives, Objects, JavaScript, Behavioral

### Краткий ответ

**Как JavaScript работает с примитивами и объектами? В чём разница в поведении?**

- Примитивы: **хранятся по значению** (string, number, boolean, null, undefined, symbol, bigint)  
- При присваивании/передаче копируется **значение** примитива  
- Объекты (Array, Function, Object): хранятся по **ссылке**  
- При присваивании/передаче копируется **ссылка**, а не сам объект  
- Изменения объекта через одну ссылку видны везде  
- Примитивы **неизменны**; операции создают новые значения  
- Это влияет на **передачу в функции**: примитивы передаются по значению, объекты — по ссылке  
- Главная разница — **мутабельность и механизм копирования**

### Подробный ответ
#### Основной ответ
В **JavaScript** есть два типа данных: **примитивы** и **объекты**. Примитивы включают `string`, `number`, `boolean`, `null`, `undefined`, `symbol` и `bigint`. Они хранятся и передаются по значению, то есть при присваивании или передаче в функцию создаётся копия значения. Объекты — это составные структуры типа `Object`, `Array`, `Function` и прочих, которые передаются по ссылке (точнее, по ссылке на объект в памяти).

При работе с примитивами, если вы присваиваете переменную другой, они независимы — изменение одной не повлияет на другую. С объектами переменная содержит указатель на объект, поэтому при копировании переменной копируется ссылка, а изменения объекта отражаются во всех ссылках.

#### Ключевые моменты
- **Передача по значению vs по ссылке**: примитивы всегда копируются (например, `let a = 5; let b = a;` — b это другая копия), а объекты передаются по ссылке (`let obj1 = {x:1}; let obj2 = obj1;` — обе переменные указывают на один объект).
- **Мутируемость**: объекты изменяемы, можно менять их свойства без создания нового объекта; примитивы иммутабельны — например, строку нельзя изменить, только создать новую.
- **Автоупаковка примитивов**: при вызове методов на примитивах (например, `"abc".toUpperCase()`) JS временно создаёт соответствующий объект-обёртку (String object), выполняет метод и уничтожает объект.

#### Практический контекст
Это отличие важно при передаче данных в функции, особенно если данные могут изменяться. Например, изменение объекта внутри функции повлияет на внешнюю переменную, а изменение примитива — нет. В React 18+ или других фреймворках часто используют **immutability** для оптимизации рендера, что требует внимательной работы с объектами и примитивами.
