# Как работает CountDownLatch?

### Категории

Multithreading, Behavioral, Async, Concurrency, Synchronization, java.util.concurrent

### Краткий ответ

**Как работает CountDownLatch?**

- синхронизатор в Java concurrency  
- инициализируется с **счетчиком**  
- метод await() блокирует поток до счетчика 0  
- метод countDown() уменьшает счетчик на 1  
- когда счетчик достигает 0 — все wait завершаются  
- полезен для **ожидания завершения** нескольких потоков  
- упрощает **согласованное** старт или завершение задач

### Подробный ответ
#### Основной ответ
**CountDownLatch** — это синхронизатор из пакета `java.util.concurrent`, который позволяет одному или нескольким потокам ждать, пока не завершится определённое количество операций в других потоках. Его основная задача — блокировать поток(и) до тех пор, пока счётчик не уменьшится до нуля, после чего все ожидающие потоки продолжают работу.

#### Ключевые моменты
- **Инициализация** происходит с заданным числом — счётчиком, который представляет количество событий/операций, которые должны произойти до продолжения.
- Потоки вызывают метод `await()` и блокируются, если счётчик не равен нулю, ожидая уведомления.
- Другие потоки вызывают `countDown()` для уменьшения счётчика на единицу после завершения своей задачи.
- Когда счётчик достигает нуля, все ожидающие потоки автоматически разблокируются.
- В отличие от **CyclicBarrier**, CountDownLatch не может быть перезапущен (одноразовый).

#### Практический контекст
Часто используется для реализации сценариев ожидания завершения нескольких потоков (например, загрузка в несколько потоков и синхронизация результата), либо для старта потоков одновременно после подготовки (стартовый барьер). В проектах с Java 8+ это удобный and лёгкий механизм для координации параллельных задач с гарантией завершения.
