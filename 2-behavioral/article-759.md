# Как реализовать graceful shutdown (defer, контекст, обработка ошибок в группах горутин)?

### Категории

HTTP, Goroutines, Behavioral, Error handling, Async, Architecture, Context, Concurrency, Database, Messaging, Graceful shutdown, Defer

### Краткий ответ

**Как реализовать graceful shutdown (defer, контекст, обработка ошибок в группах горутин)?**  
- управление завершением через **context.Context** (cancel/cancellation)  
- use **defer** для освобождения ресурсов (файлы, соединения)  
- группы горутин (например, **errgroup.Group**) для синхронизации и сбора ошибок  
- слушать системные сигналы (SIGINT, SIGTERM) для запуска отмены контекста  
- корректно завершать долго выполняющиеся операции по отмене контекста  
- ловить и обрабатывать ошибки из всех горутин централизованно  
- итог: плавное завершение без потери данных и утечек ресурсов

### Подробный ответ
#### Основной ответ
Graceful shutdown — это корректное завершение работы приложения, при котором оно закрывает все ресурсы, завершает активные операции и не оставляет процесса в неконсистентном состоянии. В языке Go для этого обычно используется комбинация **defer**, **контекста (context)** и управления группами горутин через **errgroup** или аналогичные инструменты. Такой подход позволяет аккуратно отменять фоновые задачи, ожидать их завершения и корректно обработать ошибки.

#### Ключевые моменты
- **Defer** обеспечивает порядок освобождения ресурсов: в функции main инициализируем закрытие сервисов (например, баз данных, сетевых соединений) после завершения работы, вызывая defer для закрытия.
- **Context** используется для распространения сигнала отмены: при получении системного сигнала (SIGINT/SIGTERM) вызываем cancel-функцию контекста, что позволяет завершить все горутины, слушающие его.Done().
- **Errgroup** (из пакета golang.org/x/sync/errgroup) облегчает запуск группы горутин с общей обработкой ошибок и синхронизацией завершения — благодаря контексту можно прекратить все задачи при ошибке или отмене.
- В сочетании с системным прослушиванием сигналов с помощью канала os.Signal (signal.Notify) возможно плавно отследить запрос на прерывание и инициировать shutdown.

#### Практический контекст
Стандартный паттерн в Go-приложениях (например, в микросервисах на Go 1.18+, в сервисах с HTTP сервером и воркерами) — в main создаётся root context с cancel, далее запускается errgroup с контекстом. При получении сигнала вызывается cancel, горутины слушают ctx.Done() и корректно завершают работу, после чего defer закрывает соединения. Это позволяет гарантировать, что например активные запросы к БД завершатся, HTTP сервер отдаст уже принятые запросы и только потом приложение завершится, обеспечивая uptime и целостность данных.
