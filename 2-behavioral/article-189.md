# Чем отличается скрытие элементов с помощью opacity, visibility и display: none?

### Категории

Display, UI, CSS, Frontend, Visibility, Opacity, Web, Optimization, Style, Behavioral

### Краткий ответ

**Чем отличается скрытие элементов с помощью opacity, visibility и display: none?**

- CSS-методы **скрытия элементов** с разными эффектами
- **opacity**: визуальное скрытие, элемент остаётся в потоке и занимает место, кликабелен (полупрозрачность)
- **visibility: hidden**: элемент невидим, но занимает место, не кликабелен, сохраняет пространство
- **display: none**: полностью удаляет элемент из потока, не занимает место, не кликабелен
- Влияние на **layout**: display строит заново, visibility и opacity нет
- **Перформанс**: display может быть дороже из-за перерасчёта layout'а
- Применение: display для полного скрытия/удаления, visibility для сохранения места, opacity для визуальных эффектов (анимации)
- Учет **доступности**: display и visibility скрывают элемент от экранных читалок, opacity нет (остается читабельным)

### Подробный ответ
#### Основной ответ
**opacity**, **visibility** и **display: none** — это разные CSS-свойства, которые влияют на видимость элементов на странице, но делают это по-разному и с разными последствиями для рендера и взаимодействия.

- **opacity** задаёт уровень прозрачности элемента от 0 (полностью прозрачный) до 1 (полностью непрозрачный). Элемент остаётся в дом-дереве, занимает место, реагирует на события, но становится визуально невидимым или полупрозрачным.

- **visibility: hidden** скрывает элемент визуально, но он всё ещё занимает место в раскладке страницы, и на него нельзя кликнуть. По сути, элемент невидим, но сохраняет своё пространство.

- **display: none** полностью удаляет элемент из раскладки (layout flow); элемент не занимает места, он не отображается и игнорируется при расчёте размеров и позиционирования, на него нельзя кликнуть.

#### Ключевые моменты
- **Влияние на layout:** opacity и visibility сохраняют пространство элемента, display: none убирает элемент из потока и layout.
- **Взаимодействие с пользователем:** opacity и visibility могут препятствовать взаимодействию (visibility: hidden — точно нет), но при opacity: 0 элемент всё ещё может получать события мыши; display: none не позволяет взаимодействовать.
- **Производительность:** display: none снижает нагрузку на браузер, так как элемент исключён из рендеринга, в то время как opacity и visibility требуют отрисовки, особенно при анимациях opacity.

#### Практический контекст
Часто используют **opacity** для плавных анимаций прозрачности (в React 18 или CSS transitions), **visibility: hidden** — когда необходимо скрыть элемент, сохранив место (например, скрывать элементы формы), а **display: none** — для динамического удаления компонента из DOM-потока (например, переключение вкладок, модальные окна), чтобы убрать влияние на остальной layout и улучшить производительность.
