# Event Loop, что и когда выводится в консоль

### Категории

Live coding, Memory optimization, Async, Execution order, Frontend, Event loop, Optimization, JavaScript, Algorithms, Behavioral

### Краткий ответ

**Event Loop, что и когда выводится в консоль**  
- Event Loop — механизм **асинхронного** управления очередями задач в JS  
- Задачи делятся на: **микротаски (Promises, process.nextTick)** и **макротаски (setTimeout, I/O)**  
- Сначала выполняются все микротаски текущего цикла, затем макротаски  
- Консоль вывод зависит от порядка попадания колбеков в очередь  
- Пример: сначала `console.log` из синхронного кода, затем микротаски, потом макротаски  
- Используется для оптимального **оперативного исполнения** асинхронного кода  
- На практике важно понимать, почему вывод в консоль может быть неочередным, исходя из типа задачи и стадии Event Loop  

Детальны разбор примера:

```js
console.log('1');              // Синхронно — выводится первым
setTimeout(() => console.log('2'), 0);   // Макротаска — после микротасок
Promise.resolve().then(() => console.log('3')); // Микротаска — после синхронного, перед setTimeout
console.log('4');              // Синхронно — выводится вторым
```

Вывод в консоль:  
```
1  
4  
3  
2  
```

Объяснение по этапам:  
- Синхронный код сразу выводит `1` и `4`  
- Затем выполняются микротаски (`3`)  
- И наконец макротаски (`2`)  

Понимание этой последовательности помогают предсказать вывод консоли и правильно писать асинхронный код.

### Подробный ответ
#### Основной ответ
**Event Loop** — это механизм в JavaScript (и других средах с циклом событий, например Node.js), который управляет выполнением асинхронного кода. Он позволяет обрабатывать операции ввода-вывода, таймеры, промисы и коллбэки, не блокируя главный поток выполнения. Event Loop постоянно проверяет очередь задач (task queue) и микротасков (microtask queue), чтобы определить, какой код запускать дальше.

Когда вы видите вывод в консоль, порядок зависит от того, где и в какой очереди (микротасков или макротасков) находится код:
- **Макротаски (task queue)** — включают setTimeout, setInterval, I/O операции.
- **Микротаски (microtask queue)** — это промисы (Promise.then/catch/finally), MutationObserver.

После выполнения синхронного кода Event Loop сначала обрабатывает все микротаски, а потом макротаски, что влияет на порядок вывода в консоль.

#### Ключевые моменты
- Сначала выполняется весь синхронный код, его console.log выведется первым.
- После синхронного кода выполняются все микротаски (промисы), их console.log выведется сразу после.
- Затем выполняется одна макротаска из очереди (например, setTimeout), вывод которой появится последним.
- В React 18+ и других современных средах порядок и приоритет микротасков может быть оптимизирован, но основной принцип сохраняется.
  
#### Практический контекст
Например, в коде:
```js
console.log('sync');
setTimeout(() => console.log('macro'), 0);
Promise.resolve().then(() => console.log('micro'));
```
в консоль выведется:
```
sync
micro
macro
```
Понимание Event Loop критично для отладки асинхронного кода, устранения race conditions и оптимизации производительности как во фронтенде, так и в бекенде (Node.js).
