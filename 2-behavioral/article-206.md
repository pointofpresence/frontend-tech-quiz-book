# Почему примитивные пропсы эффективны для мемоизации?

### Категории

Live coding, Performance, TypeScript, Props, Frontend, Memoization, Behavioral

### Краткий ответ

**Почему примитивные пропсы эффективны для мемоизации?**  
- контекст: React, оптимизация рендеринга  
- примитивы — строки, числа, булевы значения  
- сравниваются по значению **быстро и точно** (по ссылке не нужно)  
- мемоизация полагается на **поверхностное сравнение** пропсов  
- объекты/массивы требуют глубокого сравнения или меняют ссылку при каждом рендере  
- примитивы предотвращают ненужные повторные рендеры  
- практическая выгода — **минимизация затрат** на перерендеринг компонентов и улучшение производительности UI

### Подробный ответ
#### Основной ответ
Примитивные пропсы (например, строки, числа, булевы значения) эффективны для мемоизации, потому что сравниваются по значению, а не по ссылке. При мемоизации (например, с помощью React.memo или useMemo) важно быстро и корректно определить, изменились ли пропсы. Примитивы сравниваются очень быстро и однозначно — если значение прежнее, мемоизация срабатывает, повторный рендер не происходит.

#### Ключевые моменты
- **Сравнение по значению**: При мемоизации для пропсов с примитивными типами достаточно простого поверхностного сравнения (`===`), что выполняется мгновенно и предотвращает ненужные перерендеры.
- **Объекты и массивы сравниваются по ссылке**: Если пропсы — объекты или массивы, то сравниваются ссылки, и даже одинаковые по содержимому структуры будут восприниматься как разные, если ссылка изменилась. Это приводит к срабатыванию перерендеров и снижению эффективности мемоизации.
- **Стабильность и простота**: Примитивные значения легче предсказывать, они не имеют побочных эффектов, за их изменениями проще следить, что улучшает общую производительность и снижает риски багов.

#### Практический контекст
В React 18+ при использовании React.memo или useMemo для оптимизации компонентов рекомендуется по возможности передавать примитивные пропсы. Если необходимо передавать объекты, стоит использовать useMemo или useCallback для стабилизации ссылок, иначе мемоизация не будет эффективной, и компонент будет перерендериваться чаще, чем нужно.
