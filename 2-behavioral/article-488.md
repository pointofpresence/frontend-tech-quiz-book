# Каким способом лучше всего получить доступ к нативному DOM-элементу из React-компонента?

### Категории

DOM, UI, Behavioral, Frontend, React, Component

### Краткий ответ

**Каким способом лучше всего получить доступ к нативному DOM-элементу из React-компонента?**

- React: UI-библиотека для построения компонентов
- Лучший способ — **рефы (refs)**
- refs создаются через `React.createRef()` или `useRef()`
- позволяют получить ссылку на **DOM-узел** напрямую
- не нарушают **унидирекциональную** архитектуру React
- обеспечивают **контролируемый доступ** к элементам
- подходят для фокуса, анимаций, интеграции с библиотеками DOM
- избегай прямого доступа к DOM через селекторы (document.querySelector)
- refs — стандартный и **рекомендуемый паттерн** React

Итого: для доступа к DOM-элементу из React-компонента используют **рефы (refs)** — это правильный, безопасный и интегрированный способ.

### Подробный ответ
#### Основной ответ
Лучший способ получить доступ к нативному DOM-элементу в React — использовать **рефы (Refs)**. Рефы позволяют напрямую ссылаться на элементы DOM или компоненты и работать с ними вне обычного жизненного цикла React. Для этого обычно используется `React.createRef()` или хук `useRef()` в функциональных компонентах.

#### Ключевые моменты
- **React.createRef()** используется в классах, создавая объект ref, который прикрепляется к элементу через атрибут `ref` и позволяет получить доступ к DOM через `ref.current`.
- В **функциональных компонентах с React 16.8+** применяется хук `useRef()`, который возвращает изменяемый объект с полем `.current`, сохраняющий ссылку на DOM-элемент.
- Такой доступ позволяет вызывать нативные методы, например, `focus()`, `scrollIntoView()`, а также интегрировать нестандартные библиотеки, которые требуют прямой работы с DOM.
- Использование рефов — **исключение, а не правило** в React, где предпочтителен декларативный подход. Прямой доступ к DOM нужен в случаях анимации, работы с неуправляемыми элементами формы или интеграции с внешним кодом.
- Неправильное обращение к DOM через селекторы (например, `document.getElementById`) или манипуляции вне React могут привести к непредсказуемым багам, поэтому именно рефы считаются "реактовским" и безопасным способом.

#### Практический контекст
В React 18+ для большинства новых проектов рекомендуют использовать функциональные компоненты и `useRef()`. Например, установка фокуса после монтирования:

```jsx
function InputFocus() {
  const inputRef = React.useRef(null);

  React.useEffect(() => {
    inputRef.current?.focus();
  }, []);

  return <input ref={inputRef} />;
}
```

Такой способ обеспечивает согласованность с жизненным циклом React и предотвращает утечки памяти.
