# В чём разница между zero value слайса и мапы?

### Категории

Theory, Behavioral, Slices, Zero value, Memory management, Backend, Maps, Data structures, Live coding

### Краткий ответ

**Разница между zero value слайса и мапы**

- Контейнеры языка Go: слайс и мапа
- **Zero value слайса** — nil-срез, указывает на nil, длина и емкость 0
- Можно читать, но **записать нельзя** (без инициализации)
- **Zero value мапы** — nil-мапа, указывает на nil, при попытке записи — panic
- Для мапы нужно явно инициализировать (make) перед записью
- Чтение из nil-мапы возвращает zero value элемента без ошибки  
- Слайс можно расширять после инициализации (через append)  
- Практическое отличие: слайс zero value безопасен для чтения и append; мапа zero value безопасна только для чтения

---

**Подытоживая:**  
Zero value слайса — nil, но append работает, zero value мапы — nil, запись вызывает panic, нужно инициализировать через make перед записью.

### Подробный ответ
#### Основной ответ
**Zero value** (нулевое значение) слайса и мапы в Go означает, что переменная объявлена, но не инициализирована конкретным значением, то есть содержит значение по умолчанию для своего типа. Отличие в том, что **zero value слайса — это `nil` слайс с длиной и ёмкостью 0, а zero value мапы — это `nil` карта, которая не может быть использована для записи без явной инициализации**.

#### Ключевые моменты
- **Zero value слайса** — это nil-слайс, который можно безопасно читать, проверять на длину, и в него можно append-ить элементы, Go выделит память автоматически. Например: `var s []int` — это nil-слайс.
- **Zero value мапы** — это nil-мапа, котоая является неизменяемой для операций записи (например, `m["key"] = val` вызовет panic), но операции чтения (например, `v := m["key"]`) безопасны и просто вернут нулевое значение типа.
- Чтобы использовать мапу, необходимо её инициализировать через `make(map[keyType]valueType)` или присвоить готовую. Для слайса же append на nil-слайс работает без ошибок и создаёт новый underlying array.

#### Практический контекст
Нередко в коде `var s []int` можно сразу использовать для добавления данных через `append`, тогда как `var m map[string]int` требует явного создания мапы через `make` перед добавлением. Это отличие важно помнить, чтобы избежать runtime panic, и правильно обрабатывать zero value объектов. В больших проектах распределённой логики инициализацию мап часто делают в конструкторе, а слайсы используют более гибко.
