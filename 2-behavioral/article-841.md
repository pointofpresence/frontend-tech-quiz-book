# В чем отличие асинхронности, threading'га и мультипроцессинга?

### Категории

Multithreading, Behavioral, Multiprocessing, High load, Async, Concurrency, Parallelism

### Краткий ответ

**Отличия асинхронности, threading и мультипроцессинга**

- Асинхронность: **один поток**, неблокирующие операции, управление задачами через event loop (например, async/await в JS, Python)
- Threading: **много потоков** в одном процессе, параллельное выполнение с общей памятью, возможны гонки и блокировки
- Мультипроцессинг: **несколько процессов**, каждое с отдельной памятью, тяжелее и безопаснее в плане изоляции, обходит GIL в Python  
- Асинхронность снижает затраты на переключение контекста, эффективна для I/O задач  
- Threading полезен для CPU-bound задач без ограничений GIL или с нативной многопоточностью  
- Мультипроцессинг подходит для параллелизма с полной изоляцией и стабильностью при высоких нагрузках  
- В реальных проектах часто комбинируют для оптимальной производительности и устойчивости  

Такой набор дает глубокое понимание каждой технологии и отличия, важные для выбора подхода.

### Подробный ответ
#### Основной ответ
Асинхронность, многопоточность (threading) и мультипроцессинг — это три разных подхода к выполнению параллельных или неблокирующих операций, каждый из которых решает свои задачи и обладает специфическими особенностями. Асинхронность ориентирована на неблокирующую обработку операций ввода-вывода без выделения дополнительных системных ресурсов, threading использует несколько потоков в рамках одного процесса для параллельного выполнения, тогда как мультипроцессинг создаёт отдельные процессы, полностью изолированные друг от друга с собственной памятью.

#### Ключевые моменты
- **Асинхронность** (например, с async/await в Python 3.7+) — это кооперативное управление задачами в одном потоке: CPU не блокируется во время ожидания I/O, что снижает накладные расходы на переключение контекста. Подходит для большого количества I/O-bound задач (сетевые запросы, файловые операции) с низким потреблением ресурсов.
- **Threading** — запускает несколько потоков в рамках одного процесса, которые могут выполняться параллельно на многоядерных системах, но ограничены GIL (Global Interpreter Lock) в CPython, что снижает эффективность для CPU-bound задач. Потоки разделяют память, что упрощает обмен данными, но требует синхронизации (mutex, семафоры).
- **Мультипроцессинг** создает несколько независимых процессов с отдельной памятью, что обходит GIL и эффективно для параллельных вычислений CPU-bound. Однако обмен данными между процессами дороже (IPC), и затраты на создание/завершение процесса выше, чем у потоков.

#### Практический контекст
При разработке сетевых сервисов на Python 3.8+ часто предпочитают асинхронный подход (asyncio) для масштабируемого I/O, тогда как для тяжёлых вычислений задействуют multiprocessing. Threading же часто применяется в Java или C++ для параллелизма с разделяемой памятью и низкой задержкой. Выбор зависит от характера задачи: I/O-bound — асинхронность, CPU-bound — мультипроцессинг, комбинированные задачи — микс.
