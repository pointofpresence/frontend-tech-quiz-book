# Как избежать ConcurrentModificationException при удалении элементов из коллекции во время итерации?

### Категории

Behavioral, Concurrency, Optimization, Database, Exception handling, Code review, Data structures, Collections

### Краткий ответ

**Как избежать ConcurrentModificationException при удалении элементов из коллекции во время итерации?**

- Контекст: Java, коллекции, итерация  
- **ConcurrentModificationException** — возникает при структурных изменениях коллекции вне итератора  
- Использовать **Iterator.remove()** для безопасного удаления во время итерации  
- Другой вариант — использовать **CopyOnWriteArrayList** (потокобезопасный, копирует элементы)  
- Можно собрать элементы для удаления в отдельный список, затем удалить после итерации  
- Для потоков — применять **ConcurrentHashMap** или синхронизировать доступ  
- Итог: удалить надо через итератор или заранее подготовить элементы к удалению, чтобы избежать модификации коллекции напрямую в цикле  

Такой подход гарантирует безопасность итерации и предотвращает исключения.

### Подробный ответ
#### Основной ответ
**ConcurrentModificationException** возникает, когда мы изменяем коллекцию (например, удаляем элементы) во время её итерации с использованием стандартного итератора, не используя специально предназначенные для этого методы. Для безопасного удаления элементов во время обхода коллекции нужно применять **Iterator.remove()** или использовать структуры, поддерживающие безопасные операции параллельного изменения, например, коллекции из **java.util.concurrent**.

#### Ключевые моменты
- **Iterator.remove()** — стандартный способ удаления текущего элемента без выбрасывания исключения. Этот метод синхронизирован с итератором и корректно обновляет внутренние счетчики модификаций коллекции.
- **Использование CopyOnWriteArrayList** или других потокобезопасных коллекций из пакета java.util.concurrent, которые допускают изменение при итерации без ConcurrentModificationException, но с пониманием накладных расходов (например, копирование при каждой модификации).
- **Создание промежуточных списков** — альтернатива, когда сначала собирают элементы, которые нужно удалить, а после окончания итерации удаляют их вне цикла. Это простой, но менее эффективный метод.

#### Практический контекст
В реальных приложениях, например при фильтрации или очистке кэша, предпочтительно использовать Iterator с remove() для классических коллекций (ArrayList, HashSet) в однопоточной или контролируемой среде. Для многопоточной среды лучше использовать CopyOnWriteArrayList или ConcurrentHashMap, чтобы избежать блокировок и исключений. Такой подход обеспечивает стабильность и читаемость кода без неожиданных ошибок во время исполнения.
