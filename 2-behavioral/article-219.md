# Чем отличается controlled от uncontrolled компонентов?

### Категории

Components, UI, Uncontrolled components, Forms, Frontend, Controlled components, React, Behavioral

### Краткий ответ

**Controlled vs Uncontrolled компоненты**

- Управление состоянием: controlled — состояние в React через props и setState, uncontrolled — состояние внутри DOM  
- Источник правды: controlled — React контролирует данные, uncontrolled — DOM хранит данные  
- События: controlled компоненты обновляют state на каждый ввод, uncontrolled — полагаются на рефы для доступа к значению  
- Валидация: controlled проще валидировать и синхронизировать с UI  
- Использование: controlled — сложные формы, согласованные с состоянием, uncontrolled — простые или с минимальной логикой  
- Производительность: uncontrolled может быть чуть быстрее из-за меньшего количества рендеров  
- Резюме: controlled — React управляет вводом, uncontrolled — управление передано DOM-элементу напрямую  

Такое понимание демонстрирует владение архитектурой React-форм и баланс между удобством и контролем.

### Подробный ответ
#### Основной ответ
**Controlled** и **uncontrolled компоненты** — два подхода к управлению состоянием форм в React. **Controlled компонент** полностью управляет своим состоянием через React state и props, тогда как у **uncontrolled компонента** состояние хранится в DOM-элементе, и React не контролирует напрямую значение.

#### Ключевые моменты
- **Controlled компонент** всегда получает текущее значение через проп `value` и обновляет состояние через обработчик `onChange`. Это даёт полный контроль над вводом и упрощает валидацию, синхронизацию данных и предсказуемость поведения.
- **Uncontrolled компонент** использует DOM для хранения состояния (например, через ref и доступ к `input.value`). Такой компонент проще и имеет меньше "воды", но сложнее интегрируется в сложную логику взаимодействия и проверки.
- В React начиная с версии 16+ controlled компоненты считаются best practice для форм сложной логики или с динамическими валидациями, но uncontrolled подходит для простых форм без сложного контроля.

#### Практический контекст
В реальных проектах, например с React 18, controlled компоненты часто используют с библиотеками вроде Formik или React Hook Form для тонкой настройки форм и управления ошибками. Uncontrolled компоненты лучше применять для быстрого и простого ввода без лишнего кода, когда не нужна постоянная синхронизация значения.
