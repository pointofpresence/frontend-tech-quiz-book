# Как передавать данные между горутинами (каналы, sync-пакеты)?

### Категории

Theory, Behavioral, Async, Concurrency, Communication, Synchronization

### Краткий ответ

**Как передавать данные между горутинами (каналы, sync-пакеты)?**
- Go — язык с **конкурентностью** через goroutine
- **Каналы (channels)** — структурированный способ обмена данными
- Каналы передают **типизированные** значения, обеспечивают **синхронизацию**
- Операции отправки/приёма по каналу блокируют горутину, обеспечивая **согласованность**
- Пакеты sync (например, **WaitGroup, Mutex, Cond**) — для более низкоуровневой **синхронизации** и управления доступом
- WaitGroup ждёт завершения нескольких горутин, Mutex — защищает **разделяемые данные**
- Хорошая практика — использовать каналы для передачи данных, sync — для блокировок и ожидания
- Выбор зависит от задачи: каналы удобны для логики потоков, sync — для контроля состояния

Это обеспечивает **безопасное**, эффективное взаимодействие между горутинами и предотвращает гонки данных.

### Подробный ответ
#### Основной ответ
В Go передачу данных между горутинами обычно осуществляют при помощи **каналов (channels)** и средств из пакета **sync** для синхронизации. Каналы предоставляют типобезопасный и удобный способ обмена данными, обеспечивая одновременный обмен (синхронизацию) между горутинами. Пакет **sync** предлагает низкоуровневые механизмы, такие как **Mutex**, **WaitGroup**, **Cond**, которые позволяют управлять доступом к данным и координировать выполнение, но требуют аккуратного подхода из-за большей явной сложности.

#### Ключевые моменты
- **Каналы**: позволяют одной горутине отправлять данные, а другой — получать. Они могут быть буферизированными и неблокирующими, но по умолчанию блокируют отправителя до получения значения. Это обеспечивает безопасный обмен и предотвращает состояние гонок без явных блокировок.
- **sync.Mutex** и **sync.RWMutex** удобны для защиты общих ресурсов (например, мапы или среза). В отличие от каналов, они не передают данные, а просто синхронизируют доступ.
- **sync.WaitGroup** — для ожидания завершения группы горутин, часто используется вместе с каналами или мьютексами для координации.
- Использование каналов более идиоматично в Go и способствует "share memory by communicating", а пакеты sync подходят, когда нужно управлять состоянием напрямую.

#### Практический контекст
В типичных проектах каналы применяются для передачи сообщений, событий и для построения pipeline из горутин (например, обработка данных из внешних источников с параллельной фильтрацией). Пакеты sync полезны, если нужно обновлять общие счетчики, метрики или управлять циклом жизни горутин с высокой степенью контроля. В Go с версии 1.18+ активно рекомендуют комбинировать каналы с контекстом (context.Context) для отмены и тайм-аутов.
