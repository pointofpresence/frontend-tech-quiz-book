# Как в Go передаются слайсы — по значению или по ссылке?

### Категории

Behavioral, Context, Concurrency, Backend, Channels, Data structures, Memory, Go

### Краткий ответ

**Как в Go передаются слайсы — по значению или по ссылке?**

- Слайс в Go — структура из трёх полей: **указатель на массив**, **длина**, **ёмкость**
- Передаётся как **значение**, копируя структуру слайса (не весь массив)
- При копировании **указатель остаётся тем же**, потому изменяемые данные доступны обоим экземплярам
- Модификации элементов слайса внутри функции отражаются вне её
- Изменения длины или ёмкости скопированного слайса не влияют на оригинал
- Такой механизм оптимален по производительности и гибкости
- Практическое значение: можно эффективно передавать большие объёмы без копирования данных, но следить за изменением содержимого и структуры слайса

### Подробный ответ
#### Основной ответ
В Go **слайсы передаются по значению**, что означает копирование структуры слайса (header), но не данных, на которые он указывает. Структура слайса содержит указатель на underlying array, длину и емкость. Таким образом, когда вы передаете слайс в функцию, копируется только этот небольшой header, а не весь массив.

#### Ключевые моменты
- Слайс в Go — это оболочка над массивом: содержит указатель на данные, длину и емкость.
- При передаче слайса по значению копируется структура слайса, но все элементы существуют в памяти один раз и передаются по ссылке.
- Изменения элементов слайса внутри функции влияют на исходный массив; однако изменения структуры слайса (например, присваивание нового слайса) не меняют слайс вызывающей функции.

#### Практический контекст
Такой механизм позволяет эффективно работать с подмножествами массивов без копирования больших объемов данных, что снижает накладные расходы. Например, в Go 1.18+ этот подход активно применяется для работы с generic-функциями, где слайсы передаются для оптимального доступа и мутабельности данных.
