# Какой роутер использовался на проекте? (App Router vs Pages Router)

### Категории

SSR, Web, Routing, SPA, Frontend, ISR, Behavioral

### Краткий ответ

**App Router vs Pages Router**

- React/Next.js маршрутизация
- Pages Router: **файловая структура** → маршруты
- App Router: **React Server Components**, вложенные маршруты
- Pages Router: простой, стабильный, старый API
- App Router: современный, гибкий, поддержка async/streaming
- В проекте — зависит от технологии Next.js **версии** (App Router с Next.js 13+)
- Practical: App Router для новых проектов, Pages Router для legacy/простых случаев

### Подробный ответ
#### Основной ответ
В современных проектах на React с использованием Next.js выбор между **App Router** и **Pages Router** зависит от конкретных требований и стадии миграции. На последних версиях Next.js (начиная с 13) активно используется **App Router** — он основан на концепции React Server Components и предоставляет более гибкую архитектуру, включая поддержку серверных компонентов, новую маршрутизацию с layouts и улучшенный data fetching. В то же время, **Pages Router** (классический подход) проще и стабильнее, использовался в Next.js до версии 13 и подходит для статических сайтов и менее сложных приложений.

#### Ключевые моменты
- **App Router**:
  - Поддерживает **React Server Components**, что улучшает производительность и масштабируемость.
  - Использует **файловую маршрутизацию с вложенными layout'ами**, упрощая организацию UI.
  - Позволяет встроенный async data fetching через `fetch` или `server actions`.
  - В React 18+ и Next.js 13+ стал рекомендуемым, но требует адаптации к новым концепциям.
- **Pages Router**:
  - Работает по привычной системе файловой маршрутизации в `pages/`, поддерживает getStaticProps, getServerSideProps.
  - Более зрелый и стабильный, проще для быстрого старта и базовых сайтов.
  - Менее гибок для новых возможностей React 18+.
- Часто проекты начинают с Pages Router и постепенно мигрируют на App Router, чтобы не сломать текущий функционал.

#### Практический контекст
В моём последнем проекте на Next.js 13 использовался **App Router** для реализации комплексного UI с динамическими layout'ами и серверным рендерингом. Это позволило значительно ускорить загрузку страниц (LCP ~1.2s) и улучшить UX, используя новые возможности React Server Components и встроенные async actions вместо API роутов для более чистой архитектуры. В тоже время мы поддерживали часть Pages Router для legacy страниц, что обеспечивало мягкую миграцию без сбоев.
