# Можно ли создать сущность через PUT-запрос, если её не существует?

### Категории

Testing, HTTP, Onboarding, QA, REST, Interviews, Architecture, API, HTTP methods, Behavioral

### Краткий ответ

**Можно ли создать сущность через PUT-запрос, если её не существует?**

- REST, HTTP методы  
- PUT предназначен для **идемпотентного** создания или обновления ресурса  
- если ресурс отсутствует, сервер может создать его с URI из запроса  
- отличается от POST: PUT работает по конкретному URI, POST — создаёт по серверным правилам  
- поддержка создания через PUT зависит от API и сервера  
- важно, что повторные PUT-запросы не изменяют результат после первого  
- практическое применение: обновление/создание конфигурации или файла по известному пути

### Подробный ответ
#### Основной ответ
Да, согласно спецификации HTTP, метод **PUT** можно использовать для создания сущности, если она ещё не существует по указанному URI. Метод PUT предназначен для записи или замены ресурса по определённому адресу — если ресурс отсутствует, сервер может создать его с содержимым запроса. Это отличает PUT от POST, который обычно используется для создания ресурсов с серверно-сгенерированным идентификатором.

#### Ключевые моменты
- В **RESTful API** PUT идемпотентен: повторные запросы по одному URI с одинаковыми данными не меняют состояние сервера, что удобно для повторных операций создания или обновления.
- Если сущность отсутствует, сервер может создать её и вернуть код **201 Created**. Если же сущность существует, PUT обновляет её и обычно возвращает **200 OK** или **204 No Content**.
- В отличие от PUT, метод **POST** часто используется для создания ресурса без указания его конкретного URL, а URI генерируется автоматически (например, нумерация ID).

#### Практический контекст
В реальных системах PUT чаще используется для обновления ресурсов с определённым идентификатором, но поддержка создания через PUT удобна, когда клиент контролирует URI — например, в S3 или некоторых REST API. Это позволяет синхронизировать состояние клиента и сервера надёжно, не завися от генерации ID сервером.
