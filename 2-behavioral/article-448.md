# Как определить, что элемент виден во viewport?

### Категории

Behavioral, Algorithms, Arrays, Viewport, Visual detection, DOM, Frontend, Live coding

### Краткий ответ

**Как определить, что элемент виден во viewport?**

- фронтенд, видимость элемента  
- проверка позиции относительно **окна просмотра браузера**  
- используем метод **getBoundingClientRect()**: возвращает координаты элемента  
- проверяем: top ≥ 0 && left ≥ 0 && bottom ≤ viewport height && right ≤ viewport width  
- для частичной видимости достаточно проверять пересечение координат  
- альтернатива — **Intersection Observer API** для отслеживания видимости асинхронно и производительно  
- важно для **ленивой загрузки**, анимаций, оптимизации производительности  
   
Эффективно выявляет, виден ли элемент и насколько, с учетом динамического скролла и ресайза.

### Подробный ответ
#### Основной ответ
Определить, что элемент виден во **viewport** (области видимости страницы) можно с помощью различных методик, учитывающих положение элемента относительно окна браузера. Самый современный и удобный способ — использовать **Intersection Observer API**, который асинхронно отслеживает появление элемента во viewport. Альтернативный способ — проверять расположение элемента через методы `getBoundingClientRect()` и сравнивать координаты с размерами окна (`window.innerHeight`, `window.innerWidth`).

#### Ключевые моменты
- **Intersection Observer API** (начиная с Chrome 51+, Firefox 55+) позволяет эффективно наблюдать за пересечением элемента с viewport или любым другим ancestor-элементом — минимизирует нагрузку, не требует постоянных перерасчетов при скролле.
- В классическом подходе `element.getBoundingClientRect()` возвращает координаты относительно viewport: элемент виден, если его границы пересекаются с видимой областью, то есть `rect.top >= 0 && rect.bottom <= window.innerHeight`.
- Нужно учитывать частичную видимость и перекрытие другими элементами, а также ситуацию с горизонтальным скроллом.
- Кроме того, Intersection Observer позволяет контролировать порог видимости (threshold), например, срабатывать при 50% видимости элемента.

#### Практический контекст
В реальных проектах Intersection Observer часто используют для ленивой загрузки изображений, реализации бесконечной прокрутки или аналитики видимости рекламных объявлений с оптимальной производительностью. Для старых браузеров возможны fallback на `getBoundingClientRect()` с слушателями событий scroll и resize, хотя это менее эффективно.
