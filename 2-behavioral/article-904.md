# Какие клиенты используются для отправки HTTP-запросов в Java-приложениях (например, RestTemplate, WebClient)?

### Категории

HTTP, Behavioral, API, Async, Client, Backend, REST

### Краткий ответ

**Клиенты для HTTP-запросов в Java**

- RestTemplate: синхронный HTTP-клиент Spring, устаревающий с переходом к реактивности  
- WebClient: реактивный, асинхронный клиент из Spring WebFlux  
- Apache HttpClient: мощный низкоуровневый клиент с расширенными возможностями  
- OkHttp: легковесный, удобный HTTP/2 и WebSocket клиент  
- HttpURLConnection: встроенный Java API, низкоуровневый, устаревающий  
- Отличия RestTemplate (блокирующий) vs WebClient (неблокирующий) — выбор по архитектуре  
- WebClient предпочтительнее для микросервисов с высокой нагрузкой и реактивным стеком  

Этот набор покрывает основные популярные клиенты для отправки HTTP в Java, их ключевые особенности и выбор в зависимости от задачи и архитектуры приложения.

### Подробный ответ
#### Основной ответ
В Java-приложениях для отправки HTTP-запросов наиболее часто используются клиенты **RestTemplate** и **WebClient** из экосистемы Spring, а также сторонние библиотеки, такие как **Apache HttpClient** и **OkHttp**. RestTemplate — это синхронный HTTP-клиент, который долгое время был стандартом в Spring, однако начиная с Spring 5 рекомендуется переходить на WebClient для поддержки реактивного и асинхронного программирования.

#### Ключевые моменты
- **RestTemplate** (Spring Framework до версии 5) — простой и понятный синхронный клиент. Применяется для класcических blocking-запросов, поддерживает типизацию и настройку через `RestTemplateBuilder`. Основное ограничение — блокирующая модель, что неэффективно при масштабировании под высоконагруженные сервисы.
- **WebClient** (введён в Spring 5, часть Spring WebFlux) — реактивный, не блокирующий клиент с поддержкой асинхронности и backpressure. Позволяет эффективно работать с потоковыми и неблокирующими сценариями, особенно полезен в микросервисных архитектурах с высокой нагрузкой.
- **Apache HttpClient** и **OkHttp** — универсальные, детально настраиваемые HTTP-клиенты. OkHttp часто используется в Android-разработке и современных Java-проектах за счет производительности и простоты. Apache HttpClient подходит для сложных сценариев, где нужна тонкая настройка протоколов и безопасности.
  
#### Практический контекст
Сегодня в новых Spring Boot 2+ проектах стандартом становится WebClient из-за его неблокирующей модели и интеграции с Reactor. RestTemplate по-прежнему поддерживается, но находится в режиме «maintenance only». Если необходим синхронный стиль — выбирают RestTemplate, но для реактивных приложений и микросервисов WebClient обеспечивает лучшее масштабирование и обработку ошибок. В отдельных случаях, когда нужен низкоуровневый контроль, берут Apache HttpClient или OkHttp.
