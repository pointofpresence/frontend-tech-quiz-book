# Как организовать выполнение группы запросов с использованием DispatchGroup?

### Категории

Behavioral, Async, Architecture, Concurrency, Performance

### Краткий ответ

**Как организовать выполнение группы запросов с использованием DispatchGroup?**

- DispatchGroup — механизм группировки асинхронных задач в GCD  
- Создаём DispatchGroup, вызываем enter() перед каждым запросом  
- По окончании каждого запроса вызываем leave()  
- Метод notify() уведомляет об окончании всех запросов  
- wait() можно использовать для блокировки до завершения всех задач  
- Помогает ждать и синхронизировать множественные параллельные операции  
- Используется для объединения результатов или обновления UI после всех запросов

### Подробный ответ
#### Основной ответ  
**DispatchGroup** в iOS/Swift — это механизм из GCD (Grand Central Dispatch), который позволяет синхронизировать выполнение группы асинхронных задач и получить уведомление, когда все они завершатся. Это удобно, когда нужно параллельно запустить несколько запросов (например, сетевых), а затем объединить результаты.  

Основная идея — создать **DispatchGroup**, вызвать `.enter()` для каждой задачи, а по окончании задачи вызвать `.leave()`. В итоге с помощью `.notify(queue:)` можно выполнить код после того, как все задачи завершатся. Это помогает избежать callback hell и управлять асинхронностью в удобном синхронном стиле.

#### Ключевые моменты  
- Контроль: DispatchGroup позволяет контролировать сразу несколько асинхронных процессов с разными сроками выполнения.  
- Безопасность: Важно соблюдать баланс вызовов `.enter()` и `.leave()`, иначе блокировка не снимется или программа зациклится.  
- Использование `.wait()` — блокирует текущий поток до завершения всех задач, но в UI-потоке этого часто стоит избегать, лучше использовать `.notify()`.

#### Практический контекст  
Типично используют DispatchGroup при выполнении нескольких API-запросов одновременно: например, загрузка профиля пользователя, списка сообщений и настроек, чтобы после их завершения обновить UI одним блоком. В React Native или backend с SwiftNIO аналогичные задачи решают схожим образом с промисами или группами задач.

Пример Swift-кода упрощённо:  
```swift
let group = DispatchGroup()
var results = [String]()

group.enter()
fetchData1 { data in
    results.append(data)
    group.leave()
}

group.enter()
fetchData2 { data in
    results.append(data)
    group.leave()
}

group.notify(queue: .main) {
    print("Все запросы завершены: \(results)")
}
```

Такой подход позволяет эффективно управлять зависимостями между несколькими асинхронными операциями, сохраняя читаемость и контроль.
