# Как запустить HTTP-запросы асинхронно в Go?

### Категории

HTTP, Behavioral, Async, Architecture, Concurrency, Pattern, Database, Messaging, Outbox

### Краткий ответ

**Как запустить HTTP-запросы асинхронно в Go?**  
- контекст: конкурентность, net/http, goroutines  
- используем **goroutines** для выполнения запросов параллельно  
- каждый HTTP-запрос внутри отдельной goroutine (go func())  
- для синхронизации и ожидания результатов - **sync.WaitGroup** или каналы  
- контроль ошибок через каналы или структуры результатов  
- net/http клиент **потокобезопасен**, можно использовать общий client  
- подходит для масштабируемых и быстрых запросов к API или сервисам  

---

**Детальный ответ для подготовки:**

В Go асинхронное выполнение HTTP-запросов достигается за счет легковесных потоков — goroutines. Чтобы запустить несколько запросов одновременно, достаточно запустить функцию с вызовом http.Get или client.Do внутри отдельной goroutine. Поскольку нужно дождаться завершения всех запросов, обычно используют sync.WaitGroup: перед каждым запуском добавляют счетчик wg.Add(1), в конце каждой goroutine вызывают wg.Done(), а главный поток ждет wg.Wait().

Также можно использовать каналы для сбора результатов или ошибок. Net/http клиент является потокобезопасным, поэтому один экземпляр client можно безопасно использовать в нескольких горутинах.

Такой подход позволяет масштабировать запросы, быстро запускать множество параллельных операций без блокировки основного потока и эффективно использовать ресурсы.

---

**Пример кода:**

```go
var wg sync.WaitGroup
client := &http.Client{}

urls := []string{"http://example.com/a", "http://example.com/b"}

for _, url := range urls {
    wg.Add(1)
    go func(u string) {
        defer wg.Done()
        resp, err := client.Get(u)
        if err != nil {
            log.Println("Error:", err)
            return
        }
        defer resp.Body.Close()
        // обработка ответа
    }(url)
}

wg.Wait()
```

---

Такой ответ демонстрирует понимание конкурентности Go, потокобезопасности http.Client, синхронизации результатов и практическое использование goroutines, что производит впечатление на интервьюера.

### Подробный ответ
#### Основной ответ
В Go для запуска HTTP-запросов асинхронно обычно используют **горутины** — легковесные потоки, управляемые рантаймом Go. Запускаешь каждый запрос в отдельной горутине, что позволяет параллельно выполнять множество запросов без блокировки основного потока. Для контроля их завершения часто применяют `sync.WaitGroup` или каналы. Это стандартный и эффективный способ добиться асинхронного поведения в Go без сложных callback- или promise-механизмов.

#### Ключевые моменты
- **Горутины** создаются с помощью ключевого слова `go`. Например, `go doRequest()`.
- Для ожидания завершения нескольких запросов удобно использовать `sync.WaitGroup`, который позволяет основному потоку дождаться окончания всех горутин.
- В HTTP-клиенте Go (`net/http`) запросы выполняются синхронно, поэтому оборачиваем вызов `http.Client.Do(req)` в горутину для параллельной отправки.
- Если нужно собирать ответы, обычно используют каналы для передачи результатов обратно в основной поток.
- Важный аспект — **ограничение количества одновременно выполняемых запросов** (например, с помощью семафора или буферизированных каналов), чтобы не перегрузить систему или внешние серверы.

#### Практический контекст
В реальных проектах с микросервисами или парсерами часто требуется выполнять сотни параллельных запросов с низкой задержкой. В Go 1.18+ при этом можно использовать контекст (`context.Context`) для отмены долгих запросов, а также кастомные настройки `http.Client` (таймауты, количество соединений) для повышения устойчивости и производительности. Типичный паттерн — запуск множества горутин с запросами и сбор результатов через каналы + обработка ошибок для каждого.

---

Пример простейшего асинхронного запуска HTTP-запросов:

```go
var wg sync.WaitGroup
urls := []string{"http://example.com", "http://golang.org"}

for _, url := range urls {
    wg.Add(1)
    go func(u string) {
        defer wg.Done()
        resp, err := http.Get(u)
        if err != nil {
            log.Println("Error:", err)
            return
        }
        defer resp.Body.Close()
        // обработка resp
    }(url)
}
wg.Wait()
```
