# Как работает TypeScript в браузере?

### Категории

TypeScript, Browser, Runtime, Compilation, Frontend, Behavioral

### Краткий ответ

**Как работает TypeScript в браузере?**

- TypeScript — это **супермножество JavaScript** с типами  
- **Браузеры не поддерживают** TypeScript напрямую  
- Исходный TS-код **транспилируется в JS** с помощью tsc или сборщика (Webpack, Vite)  
- В браузер загружается только **чистый JavaScript**  
- TS компиляция выявляет ошибки на этапе **разработки (build time)**, не во время выполнения  
- Позволяет использовать более строгую типизацию и новые фичи ES  
- Упрощает масштабируемость и поддержку проекта, снижая ошибки в рантайме

### Подробный ответ
#### Основной ответ
**TypeScript** — это язык программирования, который расширяет JavaScript, добавляя статическую типизацию. Однако браузеры напрямую **не понимают TypeScript**, они работают только с JavaScript. Для использования TypeScript в браузере исходный код сначала компилируется (транспилируется) в совместимый с браузером JavaScript. Этот процесс обычно выполняется в разработческой среде с помощью компилятора `tsc` или сборщиков вроде Webpack, Vite, которые интегрируют `ts-loader` или `esbuild`.

#### Ключевые моменты
- **Транспиляция**: TypeScript-код преобразуется в чистый JavaScript, который понятен браузеру. При этом типы удаляются, так как они нужны только на этапе компиляции.
- **Source Maps**: для удобства отладки создаются карты исходников (*.map-файлы), которые позволяют увидеть исходный TypeScript-код прямо в девтулз браузера.
- **Инструменты**: обычно используют сборщики (Webpack 5+, Vite, Esbuild) с плагинами для быстрой компиляции и «горячей» перезагрузки.
- **Run-time**: Нет TypeScript-рантайма в браузере — все проверки типов на этапе сборки, в runtime выполняется уже JavaScript.

#### Практический контекст
В реальных проектах TypeScript помогает ловить ошибки на этапе разработки и улучшает автодополнение в IDE. Но для запуска в браузере всегда нужно сначала скомпилировать в JavaScript. Например, в React с TypeScript (React 18+) и Vite процесс компиляции очень быстрый, и на продакшене в браузере вообще не будет файлов с TypeScript — только JS.
