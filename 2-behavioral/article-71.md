# Как оптимизировать количество рендеров в React

### Категории

Performance, Frontend, Rendering, Optimization, Behavioral

### Краткий ответ

**Как оптимизировать количество рендеров в React**

- оптимизация React-компонентов  
- минимизировать ненужные обновления  
- использовать React.memo для мемоизации функциональных компонентов  
- применять shouldComponentUpdate или PureComponent в классовых компонентах  
- использовать useMemo и useCallback для мемоизации значений и функций, чтобы не создавать новые при каждом рендере  
- избегать анонимных функций и объектов в props, чтобы предотвратить лишние рендеры дочерних компонентов  
- следить за правильным использованием ключей (key) в списках для оптимального обновления DOM  
- разбивать большой компонент на мелкие, чтобы локализовать ререндеры  
- использовать lazy loading и code-splitting для уменьшения нагрузки  
- применять профилирование (React DevTools Profiler) для выявления причин лишних рендеров  

Эти приёмы помогают сократить количество ненужных рендеров, повысить производительность и отзывчивость приложения.

### Подробный ответ
#### Основной ответ
Оптимизация количества рендеров в **React** — важная задача для повышения производительности приложения, особенно при сложных интерфейсах. Основная идея — минимизировать ненужные повторные рендеры компонентов, которые не получили существенных изменений в пропсах или состоянии.

#### Ключевые моменты
- **React.memo** — оборачивает функциональный компонент и предотвращает повторные рендеры, если пропсы не изменились поверхностно (shallow comparison). Это аналогично `PureComponent` для классовых компонентов.
- **Хук useMemo и useCallback** — позволяют мемоизировать вычисления и функции, которые передаются в дочерние компоненты. Это особенно важно для предотвращения изменения ссылок (reference equality), из-за которой React воспринимает пропсы как новые и снова рендерит компонент.
- **Разделение состояний и компонентов** — локализация state в самых нижних компонентах или разделение на маленькие компоненты снижает количество обновлений, поскольку изменение одного локального состояния не заставляет рендерить всю цепочку.
- **Оптимизация контекста** — использование `React.Context` может приводить к массовым рендерам при любом обновлении контекста, поэтому важно либо мемоизировать значения, либо ограничивать область распространения.
- **Lazy loading и virtualized lists** (например, с react-window) — для списков с большим количеством элементов рендерятся только видимые в данный момент части, что значительно снижает нагрузку.

#### Практический контекст
В React 18+ с автоматическим batching и строгим режимом (`StrictMode`) важно комбинировать эти методы. Например, в крупных проектах часто используют **React.memo** вместе с хуками `useCallback` и `useMemo` для контроля рендеров, а оптимальное разделение состояний позволяет добиться 99.9% аптайма без лагов UI с latency ~50ms. Также нередко используют профайлер React DevTools для выявления узких мест и целевой оптимизации, а для списков — react-window или react-virtualized.
