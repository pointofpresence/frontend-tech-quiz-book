# Что такое EXPLAIN в PostgreSQL?

### Категории

Behavioral, Query planning, Query optimization, Performance, Backend, Database, Optimization, EXPLAIN PLAN, SQL

### Краткий ответ

**Что такое EXPLAIN в PostgreSQL?**

- инструмент для анализа запроса  
- показывает план выполнения запроса  
- отображает порядок операций (Seq Scan, Index Scan и т.п.)  
- включает оценку стоимости и количество строк  
- помогает выявлять узкие места и оптимизировать запросы  
- можно использовать с ANALYZE для точных метрик реального выполнения  
- критичен для улучшения производительности и устранения проблем с индексами

### Подробный ответ
#### Основной ответ
**EXPLAIN** в PostgreSQL — это команда, которая показывает план выполнения SQL-запроса, раскрывая, как именно база данных будет его обрабатывать. Она помогает понимать порядок операций, используемые индексы, методы соединений и оценочные затраты, что критично для оптимизации запросов и повышения производительности.

#### Ключевые моменты
- **План выполнения**: EXPLAIN отображает последовательность операций (например, Seq Scan, Index Scan, Join types), через которые PostgreSQL собирается пройти, чтобы получить результат.
- **Оценка стоимости**: показывает предварительные оценки затрат в условных единицах (стоимость CPU и диска) и предполагаемое количество обработанных строк.
- **Опция ANALYZE**: при использовании `EXPLAIN ANALYZE` запрос реально выполняется и показываются реальные времена выполнения, что существенно помогает для точного анализа.
- **Визуализация**: результаты EXPLAIN удобно смотреть в текстовом формате, но можно генерировать графические планы с помощью external tools (например, `pgAdmin`, `EXPLAIN.depesz.com`).

#### Практический контекст
В реальных проектах EXPLAIN — основной инструмент для диагностики медленных запросов. Например, при наличии задержек команда с этим инструментом выявляет узкие места (отсутствие индексов, неэффективные join’ы) и принимает решения по оптимизации, например, добавлению индексов или переработке запроса.  
Типично используют EXPLAIN в PostgreSQL 13+ с `BUFFERS` и `VERBOSE` опциями для максимально детальной информации.
