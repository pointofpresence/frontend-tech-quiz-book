# Где хранятся строки в Java?

### Категории

Backend, Strings, Behavioral, Memory

### Краткий ответ

**Где хранятся строки в Java?**

- память для строк выделяется в **heap**  
- строки — объекты класса **String**  
- литералы строк хранятся в **String Pool** (специальный пул строк в permgen/metaspace)  
- **String Pool** помогает **экономить память** через повторное использование литералов  
- строки в пуле — **immutable** (неизменяемы), обеспечивая безопасность и оптимизацию  
- операции `new String()` создают новую строку в heap вне пула  
- пул строк ускоряет сравнение строк через `==` для литералов  

В итоге: строки — объекты в heap, литералы — в String Pool, что экономит память и ускоряет работу с неизменяемыми строками.

### Подробный ответ
#### Основной ответ
В Java строки хранятся в объекте класса **java.lang.String**, который представляет собой неизменяемую (immutable) последовательность символов. Фактические символы внутри строки хранятся в массиве символов — в старых версиях Java (до Java 9) это был **char[]**, а начиная с Java 9 внутрь строки добавлена оптимизация с использованием **byte[]** и дополнительного флага для поддержки компактного хранения ASCII-символов (Compact Strings). Сам объект строки расположен в куче (heap), а в более ранних версиях (до Java 7u40) пул строк (String Pool) размещался в области памяти PermGen, а с Java 8 — в Metaspace.

#### Ключевые моменты
- **Heap и String Pool**: Все строки — объекты в куче, но литералы и Interned строки хранятся в специальном пуле строк (**String Pool**) для экономии памяти и переиспользования.
- **Immutable**: Строка не меняется после создания, что дает безопасность многопоточности и возможность использования строк в кешах.
- **Оптимизация Compact Strings (Java 9+)**: для строк, состоящих из ASCII символов, используется массив байт вместо массива char, что экономит память без потери производительности.
- **Отличие от char[]**: строка не просто обертка над массивом символов, она контролирует доступ, реализует кэширование хеш-кода и обеспечивает безопасность.

#### Практический контекст
В реальных проектах понимание хранения строк важно для оптимизации памяти и производительности — например, использование метода **intern()** для явного добавления строк в пул при большом числе повторяющихся значений или правильный выбор между конкатенацией через StringBuilder и "+" в циклах. Кроме того, новая модель Compact Strings помогает снизить потребление памяти для большого количества коротких ASCII-строк, что особенно актуально для серверных приложений на Java 11+.
