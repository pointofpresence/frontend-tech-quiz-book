# Как минимизировать перерендеры компонента при изменении пропсов?

### Категории

State management, Behavioral, React, Props, Performance, TypeScript, Optimization, Rendering, Live coding

### Краткий ответ

**Как минимизировать перерендеры компонента при изменении пропсов?**

- React: оптимизация ререндеров  
- Использовать **React.memo** для функциональных компонентов — мемоизация пропсов  
- В классовых компонентах — реализовать **shouldComponentUpdate** или использовать **PureComponent**  
- Передавать пропсы примитивных типов или **иммутабельные** объекты для корректного сравнения  
- Использовать коллбеки с **useCallback** и значения с **useMemo** для пропсов функций и объектов  
- Разбивать компоненты на мелкие, чтобы ререндерить только нужные  
- Избегать ненужных новых ссылок при передаче пропсов (константы вне рендера)  
- Практическое значение — улучшение **производительности** и отзывчивости UI, снижение нагрузки на DOM и React reconciliation

### Подробный ответ
#### Основной ответ
Минимизация **перерендеров компонента** при изменении пропсов — ключ к повышению производительности в React и похожих UI-библиотеках. Главная идея — избегать лишних обновлений, если пропсы не изменились по сути. Для этого обычно используют мемоизацию компонентов и оптимизацию сравнения пропсов.

#### Ключевые моменты
- В React можно применить **React.memo** — обёртку, которая по умолчанию делает поверхностное сравнение пропсов и предотвращает перерендер, если пропсы не изменились. Для сложных структур можно передать свою функцию сравнения (custom comparator).
- При передаче функций или объектов в пропсах полезно использовать **useCallback** или **useMemo**, чтобы сохранять ссылочную стабильность и не инициировать лишние ререндеры дочерних компонентов.
- В классических компонентах стоит реализовать метод **shouldComponentUpdate**, чтобы самостоятельно контролировать необходимость обновления, основываясь на сравнении предыдущих и новых пропсов.
- Важно избегать создания новых объектов или функций inline в JSX, так как каждый новый объект изменяет ссылку и провоцирует перерендер.
  
#### Практический контекст
В React 18+ типично оптимизируют компоненты с помощью React.memo, особенно на больших списка данных или сложных UI. Также применяют useMemo/useCallback, чтобы контролировать стабильность пропсов. Часто анализируют причины ререндеров с помощью React DevTools Profiler, чтобы локализовать "тяжёлые" обновления и оптимизировать их. Такой подход позволяет повысить отзывчивость интерфейса и снизить нагрузку на CPU.
