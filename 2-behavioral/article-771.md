# В чём разница между диспетчерами Dispatchers.IO и Dispatchers.Default? Когда использовать каждый?

### Категории

Multithreading, Behavioral, Async, Concurrency, Performance

### Краткий ответ

**Dispatchers.IO vs Dispatchers.Default: разница и применение**  
- **Назначение:** Dispatchers.IO — для блокирующих операций ввода-вывода, Dispatchers.Default — для CPU-интенсивных задач  
- **Пул потоков:** IO расширяет пул потоков по мере нагрузки, Default ограничен числом CPU (обычно равным числу ядер)  
- **Примеры IO:** чтение файлов, сетевые запросы, доступ к БД  
- **Примеры Default:** парсинг, вычисления, алгоритмы  
- **Почему:** IO оптимизирован для долговременной блокировки потоков, Default — для оптимальной загрузки CPU без лишних переключений  
- **Использование:** выбирайте IO для операций, которые ждут внешние ресурсы; Default — для вычислений без блокировки  
- **Практическое:** помогает избежать проблем с производительностью и блокировками в Kotlin Coroutines  

Такое разделение улучшает эффективность и отзывчивость приложений при работе с многозадачностью.

### Подробный ответ
#### Основной ответ
**Dispatchers.IO** и **Dispatchers.Default** — это предопределённые корутинные диспетчеры в Kotlin, которые управляют пулом потоков для выполнения задач. Разница между ними состоит в предназначении и характере задач, которые они оптимально обрабатывают.

**Dispatchers.Default** предназначен для выполнения CPU-интенсивных задач, таких как сложные вычисления, алгоритмы, обработка данных. Он использует пул потоков, размер которого обычно соответствует количеству доступных процессорных ядер, что обеспечивает максимальную загрузку CPU без избыточного контекста переключения.

**Dispatchers.IO**, напротив, оптимизирован для операций ввода-вывода — чтения или записи файлов, сетевых запросов, работы с базами данных. Он выделяет гораздо больший пул потоков (по сравнению с Default), чтобы избежать блокировок при долгих операциях I/O, которые тратят время в ожидании, а не на выполнение CPU-вычислений.

#### Ключевые моменты
- **Dispatchers.Default**: лучший выбор для вычислительных задач; держит количество потоков около числа CPU-ядер (обычно coreCount).
- **Dispatchers.IO**: масштабируется до большего количества потоков (до 64*coreCount в последних версиях Kotlin), чтобы параллельно обрабатывать множество блокирующих I/O операций.
- Если не использовать IO-диспетчер для блокирующих операций, можно «застопорить» пул потоков Default, что приведёт к деградации производительности.
- В Kotlin 1.6+ Dispatchers.IO адаптивен и динамически регулирует размер пула, учитывая нагрузку.

#### Практический контекст
В реальных проектах для работы с сетью, файлами или базами данных используют **Dispatchers.IO** — например, запросы к API, чтение из файла или запись в БД. Для CPU-интенсивной обработки данных (парсинг, математические операции) — **Dispatchers.Default**. Этот баланс помогает эффективно использовать ресурсы и избегать искусственного ограничивания числа потоков.
