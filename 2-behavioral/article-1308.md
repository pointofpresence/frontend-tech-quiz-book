# Как создать генератор с помощью yield?

### Категории

Generator, Behavioral, Async, Backend, Yield

### Краткий ответ

**Как создать генератор с помощью yield?**  
- Python-функция с ключевым словом **yield**  
- приостанавливает выполнение, возвращая значение  
- состояние функции сохраняется между вызовами  
- создает объект-генератор (итератор)  
- позволяет эффективно работать с большими данными  
- используется для ленивой (отложенной) генерации элементов  
- улучшает память и производительность в циклах и потоках данных

### Подробный ответ
#### Основной ответ
В Python **генератор с помощью yield** — это функция, которая при вызове возвращает итератор, по которому можно проходить в цикле. В отличие от обычной функции, которая возвращает конечный результат и завершает работу, генератор с `yield` приостанавливает своё выполнение и сохраняет текущее состояние, позволяя возвращать значения по одному. При следующем вызове продолжается с места, на котором остановились.

#### Ключевые моменты
- **yield** — ключевое слово, которое заменяет return в генераторе, но не завершает функцию, а "выдаёт" текущее значение и ставит функцию на паузу.
- Генераторы экономят память: не строят всю коллекцию сразу, а вычисляют элементы по мере необходимости (ленивые вычисления).
- Можно использовать для обработки больших потоков данных или бесконечных последовательностей, где хранить весь результат сразу невозможно (например, чтение логов, потоковое API).

#### Пример:
```python
def count_up_to(n):
    count = 1
    while count <= n:
        yield count
        count += 1
```
При итерировании по `count_up_to(5)` последовательно получим числа от 1 до 5.

#### Практический контекст
Генераторы — стандартный подход в Python 3.x для работы с потоками данных, асинхронными операциями (в связке с asyncio), а также для реализации собственных итераторов без создания промежуточных коллекций, что существенно повышает производительность и снижает потребление памяти. В крупных проектах типично используют генераторы для обработки CSV или логов по строчно, чтобы избежать загрузки всего файла в память.
