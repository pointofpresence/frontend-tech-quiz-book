# В чём разница между POST (создание) и PUT (обновление) в REST API?

### Категории

HTTP, REST, Methods, API, Testing, Behavioral

### Краткий ответ

**Разница между POST и PUT в REST API**  
- оба метода HTTP для изменения ресурсов в REST  
- **POST**: создание нового ресурса, необязательно идемпотентен  
- **PUT**: полное обновление существующего ресурса, идемпотентен  
- при POST сервер генерирует URI нового ресурса  
- при PUT клиент указывает точный URI ресурса  
- POST добавляет, PUT заменяет полностью данные по указанному URI  
- практическое: POST для создания, PUT для обновления/перезаписи ресурса

### Подробный ответ
#### Основной ответ
В **REST API** методы **POST** и **PUT** имеют различные семантические значения: **POST** используется преимущественно для создания новых ресурсов, а **PUT** — для обновления или замены существующих ресурсов. Ключевая разница в идемпотентности и способе идентификации ресурса.

#### Ключевые моменты
- **POST** — неидемпотентный метод, обычно отправляется на коллекцию (например, `/users`), сервер создает новый ресурс, генерируя уникальный идентификатор, и возвращает его. Повторный POST обычно создаёт новые уникальные ресурсы.
- **PUT** — идемпотентный, клиент явно указывает URL ресурса (например, `/users/123`), и запрос заменяет весь ресурс по этому адресу; повторение PUT с теми же данными не меняет результат.
- **Различия в использовании**: POST подходит для создания, когда клиент не знает заранее идентификатор; PUT — когда идентификатор известен и ресурс нужно полностью заменить или создать с заданным ID (upsert).
- **Влияние на API-дизайн**: POST зачастую служит для действий с эффектом создания и вызывает новые состояния сервера, PUT — для точного целевого обновления.

#### Практический контекст
В реальных проектах, например, в REST API для пользователей, POST /users создаёт нового пользователя с новым ID, а PUT /users/123 заменит информацию пользователя с ID 123. Это позволяет надежно различать операции создания и обновления, а также удобно реализовать повторяемость запросов и кэширование.
