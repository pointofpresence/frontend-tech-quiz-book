# Что такое gRPC?

### Категории

Behavioral, API, Communication protocol, Backend, Database, Remote procedure call

### Краткий ответ

**Что такое gRPC?**

- высокопроизводительный **RPC-фреймворк** от Google  
- использует **HTTP/2** для транспорта, обеспечивая мультиплексирование и сжатие  
- сообщения сериализуются через **Protocol Buffers** (protobuf) — компактный, бинарный формат  
- поддерживает **би- и унидиrectional streaming**, кроме классических вызовов  
- автоматическая генерация клиентских и серверных стубов на разных языках  
- гарантирует **низкую задержку** и эффективность на сетевом уровне  
- востребован для микросервисной архитектуры и межсервисного взаимодействия высокого уровня  

gRPC решает задачи масштабируемой, быстродействующей и типобезопасной коммуникации между распределёнными системами.

### Подробный ответ
#### Основной ответ
**gRPC** — это современный фреймворк для удалённого вызова процедур (RPC), разработанный Google, который позволяет создавать высокопроизводительные и масштабируемые распределённые системы. Он использует протокол HTTP/2 для эффективной коммуникации и протокол буферов (Protocol Buffers) как формат сериализации данных, обеспечивая лёгкость и скорость передачи сообщений между клиентом и сервером.

#### Ключевые моменты
- **HTTP/2** обеспечивает мультиплексирование потоков, сжатие заголовков и двунаправленные стримы, что значительно снижает задержки и повышает пропускную способность по сравнению с HTTP/1.1.
- **Protocol Buffers (protobuf)** используют бинарный формат данных, который меньше по размеру и быстрее десериализуется, чем JSON или XML, что критично для микросервисов и мобильных приложений.
- gRPC поддерживает разные виды взаимодействия: Unary (один запрос — один ответ), Server Streaming, Client Streaming и Bi-directional Streaming, что даёт гибкость для разных сценариев обмена данными.
- Кроме того, gRPC интегрируется с современными инструментами безопасности — TLS для шифрования, аутентификацией с помощью токенов и включает встроенную поддержку deadline/timeouts.

#### Практический контекст
В реальных проектах gRPC часто используют для связи между микросервисами (service-to-service communication), где важна высокая производительность и строго типизированные контракты API. В версиях, например, gRPC на Java и Go, latency может достигать порядка 10-20ms в локальной сети с 99.9% uptime. Также gRPC удобен для мобильных приложений и IoT, где экономия трафика и время отклика критичны.
