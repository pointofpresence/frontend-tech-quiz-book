# Какие типы индексов существуют?

### Категории

SOLID, Indexing, Behavioral, API, Microservices, Security, Technical interview, Architecture, Indexes, Backend, Database, Principles, Optimization, Code review

### Краткий ответ

**Типы индексов в БД**

- B-дерево: сбалансированное, для диапазонных запросов  
- Хеш-индексы: быстрый точечный поиск (равенство)  
- Bitmap-индексы: эффективны для колонок с малым числом уникальных значений  
- Полнотекстовые: поиск по тексту, с анализом слов  
- Пространственные: для геоданных (R-дерево, Quad-tree)  
- Сложные/композитные: на несколько колонок для многокритериальных запросов  
- Партиционированные: разбивают индекс по сегментам для масштабируемости  

Индексы ускоряют поиск, но замедляют вставки и обновления.

### Подробный ответ
#### Основной ответ
В базах данных существует несколько типов **индексов**, которые оптимизируют скорость поиска и сортировки, снижая время отклика запросов. Основные типы индексов — это **B-Tree (или B+Tree)**, **Hash-индексы**, **Bitmap-индексы**, **GiST**, **GIN** и **Full-Text** индексы. Каждый из них имеет свою область применения и особенности.

#### Ключевые моменты
- **B-Tree индекс** — самый распространённый тип, подходит для диапазонных запросов, сравнений, сортировки. В PostgreSQL 14+ и MySQL используется по умолчанию. Поддерживает равенства и неравенства, эффективен для уникальных и неуникальных ключей.
- **Hash-индекс** — оптимален для операций точного совпадения (равенства). В PostgreSQL, например, historically был менее распространён из-за ограничений и надежности, но в версии 10+ стал более стабильным. Не подходит для range queries.
- **Bitmap-индексы** — эффективны для столбцов с низкой кардинальностью (например, пол или категория). Используются в аналитических системах, например, в Oracle. Могут комбинироваться для эффективных пересечений условий.
- **GiST и GIN** — специализированные индексы для полнотекстового поиска, географических данных и массивов (например, в PostgreSQL). GiST позволяет создавать индекс для сложных структур, GIN оптимален для поиска по множественным значениям в одном поле.
- **Full-Text индекс** — обеспечивает быстрый поиск по тексту с поддержкой морфологии и релевантности. Активно применяется в PostgreSQL (tsvector), Elasticsearch и других поисковых движках.

#### Практический контекст
В реальных проектах B-Tree — универсальное решение для большинства OLTP систем, Hash подойдет для быстрых equality lookup, а GIN/GiST — для сложных запросов с JSONB и геоданными. Bitmap-индексы чаще встречаются в хранилищах данных для аналитики, где данные с низкой кардинальностью. Оптимальный выбор индекса критичен для снижения нагрузки и повышения отзывчивости приложений.
