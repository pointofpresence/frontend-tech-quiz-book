# Как отлавливать события на фазе погружения?

### Категории

Behavioral, Async, Event handling, Frontend, Live coding, Interview question

### Краткий ответ

**Как отлавливать события на фазе погружения?**

- обработка событий в DOM  
- фазы: погружение (capturing), цель, всплытие  
- отлавливать на фазе погружения помогает параметр useCapture = true  
- addEventListener(type, handler, true) — слушатель на погружении  
- событие сначала проходит фазу погружения сверху вниз по дереву  
- полезно для ранней обработки, предотвращения всплытия  
- применяется для управления порядком обработки сложных событий

### Подробный ответ
#### Основной ответ
Отлавливание событий на **фазе погружения** (capture phase) в DOM — это реализация одного из этапов модели всплытия событий в браузерах. По умолчанию события сначала происходят на верхних узлах DOM и направляются вниз, пока не достигнут целевого элемента (фаза погружения), затем происходит фаза цели и, наконец, фаза всплытия обратно вверх. Чтобы ловить события именно на фазе погружения, при добавлении обработчика события нужно явно указать опцию `{ capture: true }` в методе `addEventListener`.

#### Ключевые моменты
- **Использование addEventListener с опцией capture**:  
  Методу `addEventListener` передаётся третий аргумент — объект с настройками, либо булево значение (устаревшее). Например:  
  ```js
  element.addEventListener('click', handler, { capture: true });
  ```
  — это заставит обработчик вызываться при фазе погружения.
- **Разница с фазой всплытия (bubble phase)**: по умолчанию обработчики работают на этапе всплытия, когда событие идет от целевого элемента вверх по дереву.
- **Практическое применение**: захват событий используется, если нужно гарантировать, что обработчик сработает раньше, чем другие обработчики на фазе цели или всплытия. Например, для глобальной отмены события или предотвращения конфликтов с дочерними элементами.

#### Практический контекст
В реальных проектах на React 18+ или при vanilla JS сложных интерфейсах часто используют `capture` для глобальных слушателей, чтобы перехватить события до их обработки вложенными компонентами. Это помогает корректно контролировать поведение сложных всплывающих меню, drag-n-drop и подобных интерактивных компонентов.
