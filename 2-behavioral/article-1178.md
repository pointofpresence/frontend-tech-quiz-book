# Чем отличаются интерфейсы IEnumerable<T> и IQueryable<T> и для чего нужен QueryProvider?

### Категории

Behavioral, LINQ, Backend, Query, Interface, Patterns, Design patterns, Collections

### Краткий ответ

**Различия IEnumerable<T> и IQueryable<T>, роль QueryProvider**  
- **IEnumerable<T>** — интерфейс коллекции, выполняет запросы в памяти (LINQ to Objects)  
- **IQueryable<T>** — расширяет IEnumerable<T>, поддерживает отложенное формирование запросов к удалённым источникам данных (LINQ to SQL, Entity Framework)  
- **IQueryable<T>** строит выражение в виде дерева Expression Tree  
- **QueryProvider** — преобразует Expression Tree в конкретный запрос (SQL, API и т.д.)  
- Благодаря QueryProvider запросы выполняются на стороне источника, экономя трафик и время  
- IEnumerable<T> — для обработки в памяти, IQueryable<T> — для оптимизации запросов к БД  
- Использование IQueryable<T> позволяет комбинировать фильтры и трансформации без немедленного выполнения запроса (отложенное выполнение)

### Подробный ответ
#### Основной ответ
**IEnumerable<T>** и **IQueryable<T>** — это интерфейсы, которые служат для работы с коллекциями данных, но имеют разные цели и механизмы выполнения запросов. **IEnumerable<T>** выполняет перебор коллекции в памяти и используется для локальных данных. **IQueryable<T>** строит выражение запроса, которое может быть преобразовано и выполнено удаленно (например, в БД). **QueryProvider** — это компонент, реализующий логику преобразования выражения запроса в конкретный механизм его исполнения, например, SQL для БД.

#### Ключевые моменты
- **IEnumerable<T>** — предоставляет перебор элементов в памяти, запросы выполняются сразу на стороне клиента, хорошо подходит для коллекций в памяти (List, Array).
- **IQueryable<T>** — инкапсулирует выражения с отложенным выполнением, запросы формируются в древовидную структуру Expression Tree, которую **QueryProvider** преобразует в оптимальный для источника данных запрос, например SQL в Entity Framework.
- **QueryProvider** реализует метод Execute и CreateQuery, позволяя провайдеру (например, EF Core 6 или LINQ to SQL) превратить LINQ-выражения в запросы к API БД, оптимизировать и выполнять их.

#### Практический контекст
Использование **IQueryable<T>** позволяет не загружать все данные из базы в память, а отправлять только необходимые данные, что существенно повышает производительность в крупномасштабных приложениях. **IEnumerable<T>** более простой инструмент для обработки данных уже после выборки. Понимание разницы критично при оптимизации запросов и проектировании слоев доступа к данным.
