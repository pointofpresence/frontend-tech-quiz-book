# Что такое горутина? Как ее остановить?

### Категории

Behavioral, Async, Concurrency, Backend, Goroutine

### Краткий ответ

**Что такое горутина? Как ее остановить?**
- примитив **конкурентности** в Go  
- **легковесный поток**, выполняющий функцию асинхронно  
- планируется планировщиком Go runtime  
- нет прямого метода остановки горутины  
- контроль через **контекст (context.Context)** с отменой  
- можно остановить через сигналы по каналам (chan)  
- важен корректный выход из функции по сигналу  
- остановка — управление на уровне **логики** горутины  

**Итого:** горутина — легковесный поток без прямой остановки, управление прекращением через контекст или каналы с отзывом изнутри.

### Подробный ответ
#### Основной ответ
**Горутина** — это легковесный поток исполнения в языке **Go**, который позволяет запускать функции параллельно с минимальными накладными расходами. В отличие от классических потоков ОС, горутины управляются рантаймом Go и используют гораздо меньше памяти (около 2KB стек изначально), что позволяет одновременно запускать тысячи горутин. Горутины планируются на системные потоки через M:N scheduler (множество горутин на несколько потоков ОС), что обеспечивает высокую производительность и масштабируемость.

Остановить горутину напрямую невозможно через стандартный API. Для контролируемого завершения используются шаблоны кооперативной отмены, например, через **контекст (context.Context)** или каналы, по которым горутина получает сигнал завершения и корректно прекращает работу.

#### Ключевые моменты
- Горутины создаются ключевым словом `go` перед вызовом функции и запускаются асинхронно.
- Для безопасного и контролируемого останова рекомендуют использовать `context.Context` с отменой (`context.WithCancel`) или каналы, чтобы горутина сама завершилась при получении сигнала.
- Принудительной остановки горутины из вне, как в потоках Java (`Thread.stop()`), в Go нет, что помогает избегать гонок и зависаний.

#### Практический контекст
В реальных проектах в Go обычно создают горутины для обработки запросов, фоновых задач и параллельной работы с сетью или I/O. Для отмены длительных операций передают `context.Context` в функцию горутины и вызывают `cancel()`, чтобы аккуратно завершить все ресурсы и избежать утечек. Такая практика характерна для HTTP-серверов в Go (например, в Go 1.18+).
