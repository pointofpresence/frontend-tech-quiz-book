# Что такое "абстрактный класс" (или "абстрактная сущность") и для чего он используется в моделировании?

### Категории

System analysis, OOP, Abstraction, Modeling, Technical

### Краткий ответ

**Что такое абстрактный класс и зачем он нужен в моделировании?**

- ООП, базовый концепт  
- класс с частично реализованным или не реализованным интерфейсом  
- содержит абстрактные методы без тела  
- нельзя создать экземпляр напрямую  
- служит для **общего интерфейса/контракта** в иерархии  
- заставляет наследников реализовать ключевые методы  
- помогает задать общий каркас и стандарты для классов-наследников  
- упрощает поддержку и расширение кода через полиморфизм и повторное использование  

Итог: абстрактный класс — инструмент для **задачи шаблонов и обязательств** в проектировании, обеспечивающий гибкость и структурированность кода.

### Подробный ответ
#### Основной ответ
**Абстрактный класс** — это класс в объектно-ориентированном программировании, который не предназначен для создания непосредственных экземпляров, а служит шаблоном или базой для других классов. Он может содержать как реализованные методы, так и абстрактные — без реализации, которые должны быть определены в наследниках. В моделировании абстрактный класс используется для выделения общих свойств и поведения группы сущностей, помогая структурировать код и повысить переиспользуемость.

#### Ключевые моменты
- **Нельзя создать объект напрямую:** такие классы помечаются ключевым словом `abstract` (в Java, C#) и служат только как основа для конкретных подклассов.
- **Определяет интерфейс и общее поведение:** позволяет задать обязательные методы (например, `calculate()`), которые будут реализованы в дочерних классах с разной логикой.
- **Повышает уровень абстракции и упрощает поддержку кода:** вместо дублирования общих атрибутов в каждой конкретной реализации достаточно описать их один раз.
- В моделировании помогает выразить концепции, которые существуют в предметной области, но не имеют самостоятельных конкретных инстансов, например, "Транспортное средство" или "Фигура" — общий базовый тип.

#### Практический контекст
При проектировании ПО на основе предметной области обычно создают абстрактные классы для базовых сущностей (например, в финансовых приложениях `Account` как базовый класс для `SavingsAccount` и `CheckingAccount`). Это обеспечивает единообразие API и облегчает расширение функциональности без изменения существующего кода. Особенно актуально в больших системах на Java 17 или C# 10, где строгая типизация и принципы SOLID обязательны к соблюдению.
