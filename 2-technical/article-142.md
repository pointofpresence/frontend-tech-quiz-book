# При выполнении UniTask создаётся ли новый поток?

### Категории

Async, Multithreading, Technical, Game development

### Краткий ответ

**При выполнении UniTask создаётся ли новый поток?**

- **UniTask** — это библиотека для эффективной **асинхронной** работы в Unity и C#  
- **UniTask не создаёт новый поток** по умолчанию  
- Асинхронность достигается через **использование существующего потока и планировщика задач**  
- Выполнение может быть **синхронным**, если результат уже готов  
- Важно: новый поток создаётся, только если явно вызвать операцию с **Task.Run** или аналогом  
- UniTask минимизирует **накладные расходы на переключение потоков** по сравнению с Task  
- Применяется для **оптимизации производительности** в игровых проектах на Unity  

Итого: UniTask — это **обёртка для асинхронности без автоматического создания потоков**, что снижает затраты и повышает производительность.

### Подробный ответ
#### Основной ответ
**UniTask** — это библиотека для асинхронного программирования в Unity, которая предоставляет более лёгкую и производительную альтернативу стандартному `Task` и `async/await`. При вызове UniTask по умолчанию **новый поток не создаётся**. Выполнение асинхронных операций происходит в контексте существующего потока (обычно основного потока Unity), пока не задействуется явное переключение на другой поток или операция ввода-вывода.

#### Ключевые моменты
- **UniTask оптимизирован для Unity** и ориентирован на минимизацию накладных расходов, связанных с созданием потоков и сборкой мусора.
- Асинхронность внутри UniTask достигается через **кооперативную многозадачность (coroutines-like), использующую планировщик Unity или своё внутреннее переключение контекста**, а не через создание новых потоков или пул потоков.
- **Если нужно выполнять тяжёлые вычисления параллельно, нужно вручную использовать `ThreadPool` или `Task.Run`**, а затем обернуть результат в UniTask для удобного async/await синтаксиса. UniTask сам по себе не форсирует новый поток.

#### Практический контекст
В реальных Unity проектах UniTask часто применяется для работы с асинхронной загрузкой ресурсов, ожиданием таймеров или внешних событий с минимальной нагрузкой на систему. Код остаётся в основном потоке, что критично для безопасного взаимодействия с Unity API, который не потокобезопасен.

---
Таким образом, UniTask отвечает за удобное управление асинхронностью, но не за многопоточность — создание новых потоков происходит только при явном использовании механизмов .NET, а не самой библиотеки UniTask.
