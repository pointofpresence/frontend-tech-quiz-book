# Какие существуют реализации коллекций в Kotlin?

### Категории

Programming languages, Data structures, Technical, Collections

### Краткий ответ

**Реализации коллекций в Kotlin**

- Коллекции делятся на **мутабельные** и **немутируемые**
- Немутируемые: **List**, **Set**, **Map** — только для чтения
- Мутабельные: **MutableList**, **MutableSet**, **MutableMap** — поддерживают изменение данных
- List — упорядоченный индексированный набор элементов  
- Set — уникальные элементы, неупорядоченный (HashSet), или упорядоченный (LinkedHashSet, TreeSet)
- Map — пары ключ-значение, ключи уникальны (HashMap, LinkedHashMap, TreeMap)
- Перечисленные реализации базируются на Java Collections, адаптированных для Kotlin
- Используются для эффективной работы с данными в зависимости от требований к изменяемости и скорости доступа

### Подробный ответ
#### Основной ответ
В **Kotlin** для работы с коллекциями предусмотрены несколько основных интерфейсов и их реализаций — это стандартные структуры данных, предназначенные для хранения и управления множествами элементов. Основные коллекции — это **List**, **Set** и **Map**, каждая из которых имеет мутабельные (mutable) и немутируемые (immutable) варианты. Kotlin сделан поверх **Java Collections**, но предоставляет более удобный и безопасный API.

#### Ключевые реализации
- **List** — упорядоченная коллекция элементов, допускающая дубликаты.
  - `List` (немутируемый) — только для чтения, например, `listOf()`.
  - `MutableList` — изменяемый список, примеры: `ArrayList`, `LinkedList` (в Java-реализации).
- **Set** — коллекция уникальных элементов, без определённого порядка.
  - `Set` — неизменяемый набор, например, `setOf()`.
  - `MutableSet` — изменяемый набор, чаще всего основан на `LinkedHashSet` для сохранения порядка вставки.
- **Map** — отображение ключей в значения (ключ-значение).
  - `Map` — Read-only отображение, например, `mapOf()`.
  - `MutableMap` — изменяемая карта, часто реализована на основе `HashMap` или `LinkedHashMap`.

#### Особенности и дополнения
- Kotlin имеет **двойную иерархию**: read-only интерфейсы (например, `List`) и mutable (например, `MutableList`), что повышает безопасность и явность изменений.
- Для неизменяемых коллекций Kotlin часто возвращает обертки над Java-коллекциями, предотвращающие модификацию.
- В Kotlin 1.4+ появились **persistent collections** (из kotlinx.collections.immutable) — неизменяемые структуры с эффективными копированиями.
- Более специализированные коллекции, как `Sequence` (ленивые коллекции) для обработки потоков данных с отложенным вычислением.

#### Практический контекст
При разработке на Kotlin часто используют немутируемые списки и карты для обеспечения безопасной многопоточности и предсказуемости кода. Если нужны модификации — применяют `MutableList` или `MutableMap`. Для производительности и оптимизации памяти можно выбирать конкретные реализации, например, `ArrayList` для частого произвольного доступа или `LinkedHashSet` для сохранения порядка и быстрых проверок на уникальность.
