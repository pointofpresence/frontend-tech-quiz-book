# Что такое оператор нулевого слияния?

### Категории

Live coding, Syntax, Backend, Technical, Nullish coalescing, Interview, JavaScript, Operator

### Краткий ответ

**Что такое оператор нулевого слияния?**  
- оператор для проверки **null** и **undefined**  
- возвращает правый операнд, если левый — null или undefined  
- записывается как **??**  
- отличается от || тем, что не считает false, 0, '' за "пустые"  
- удобен для установки **значений по умолчанию**  
- широко используется в JavaScript и TypeScript  
- повышает **читаемость** кода и предотвращает ошибки при проверке значений

### Подробный ответ
#### Основной ответ
**Оператор нулевого слияния** (nullish coalescing operator) — это специальный логический оператор, обычно обозначаемый как `??`, который возвращает первый операнд, если он не является `null` или `undefined`. Если первый операнд равен `null` или `undefined`, возвращается второй операнд. Оператор полезен для установки значений по умолчанию именно тогда, когда переменная не определена или отсутствует.

#### Ключевые моменты
- В отличие от логического оператора `||` (ИЛИ), который также используется для выбора значения по умолчанию, `??` игнорирует значения вроде `0`, пустой строки `""` или `false`. Эти значения считаются валидными и не вызывают переход ко второму операнду.
- Оператор введён в стандарте **ECMAScript 2020** и широко поддерживается в современных версиях JavaScript и TypeScript.
- Важно не использовать оператор `??` совместно с логическими операторами (`&&`, `||`) без скобок, так как это может привести к синтаксическим ошибкам — требуется явная расстановка приоритетов.

#### Практический контекст
Например, при работе с пользовательскими настройками, когда значение может быть 0 или пустой строкой, но нужно задать дефолт только если значение отсутствует, оператор нулевого слияния идеально подходит:

```javascript
const userValue = 0;
const defaultValue = 10;
const value = userValue ?? defaultValue; // value будет 0, а не 10, как при использовании ||
```

Таким образом, `??` позволяет избежать ошибок при проверке «пустых», но валидных значений, улучшая читабельность и надёжность кода.
