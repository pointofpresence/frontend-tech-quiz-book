# Какие существуют асинхронные библиотеки в Python, помимо FastAPI?

### Категории

API, Async, Web, Backend, Technical

### Краткий ответ

**Асинхронные библиотеки Python, кроме FastAPI**

- контекст: **асинхронное программирование в Python**
- **asyncio** — стандартная библиотека для async IO, **event loop**
- **aiohttp** — клиент и сервер для HTTP с поддержкой async/await
- **Trio** — современный, простой и безопасный async фреймворк
- **Tornado** — веб-сервер с поддержкой асинхронности, экстенсивный стек
- **Sanic** — веб-фреймворк, ориентированный на производительность
- **Curio** — минималистичный async runtime, упор на понятность
- практически все дают поддержку **конкурентных network IO** и масштабируемость

Эти библиотеки используются для разработки высокопроизводительных, масштабируемых сетевых приложений, веб-сервисов и микросервисов на Python.

### Подробный ответ
#### Основной ответ
В Python существует множество асинхронных библиотек, которые используются для разных целей помимо FastAPI, который, в первую очередь, является асинхронным веб-фреймворком. Среди популярных асинхронных библиотек можно выделить **asyncio** как стандартный модуль для работы с асинхронным программированием, а также вспомогательные инструменты и фреймворки, такие как **aiohttp**, **Trio**, **Curio**, и **Tornado**.

#### Ключевые моменты
- **asyncio** — встроенный модуль с Python 3.4+, обеспечивающий событийный цикл, корутины и механизмы для разработки асинхронного кода на базовом уровне. Это основа многих других библиотек.
- **aiohttp** — асинхронный HTTP клиент и сервер, построенный поверх asyncio, удобен для создания веб-сервисов и запросов с низкой задержкой.
- **Trio** — альтернативный асинхронный фреймворк, который предлагает более простой и безопасный API для работы с конкурентностью, акцентируется на удобстве и отменяемости задач.
- **Curio** — библиотека от того же автора, что и Trio, сосредоточена на минимализме и производительности, тоже на базе корутин.
- **Tornado** — старый, но проверенный веб-фреймворк и сетевой сервер с поддержкой асинхронности, применялся до широкого распространения asyncio.
- Также в экосистеме есть **Sanic**, похожий на FastAPI сервер с асинхронной обработкой HTTP запросов.

#### Практический контекст
В реальных проектах часто используют **asyncio** как фундамент для построения асинхронных приложений, aiohttp идеально подходит для простых API или клиентов, где нужна асинхронность без дополнительной обвязки. Trio и Curio применяются, когда важна более строгая модель параллелизма и простота кода, например, в научных и исследовательских проектах. Tornado остаётся актуальным в проектах с наследием и там, где нужна высокая производительность на одном процессе. FastAPI же — это современное решение с удобным синтаксисом и автоматической валидацией данных, но вы должны знать и другие инструменты для выбора оптимального набора библиотек.
