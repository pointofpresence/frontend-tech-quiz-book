# Какие основные хуки в React и для чего они нужны?

### Категории

State management, React, Async, React hooks, Optimization, Functional components, Frontend, Hooks, Technical

### Краткий ответ

**Основные хуки в React и их назначение**

- Хуки — функции для работы с состоянием и жизненным циклом в функциональных компонентах
- **useState** — управление локальным состоянием, хранит значения и позволяет их обновлять
- **useEffect** — отслеживает побочные эффекты (запросы, таймеры) и синхронизирует их с рендером
- **useContext** — доступ к контексту для глобального или сквозного состояния без пропсов
- **useReducer** — сложная логика обновления состояния, альтернативный способ вместо useState, похож на Redux по подходу
- **useMemo** — мемоизация вычислений для оптимизации производительности, пересчет только при изменении зависимостей
- **useCallback** — мемоизация функций, чтобы избежать лишних перерисовок дочерних компонентов
- Используются для улучшения читаемости кода, управления состоянием и оптимизации реакт-приложений без классовых компонентов

### Подробный ответ
#### Основной ответ
**Основные хуки в React** — это функции, которые позволяют использовать состояние и другие возможности React внутри функциональных компонентов, без необходимости писать классы. Они появились с React 16.8 и существенно упростили управление логикой и состоянием компонентов.

#### Ключевые моменты
- **useState** — управляет локальным состоянием компонента. Позволяет создавать и обновлять переменные состояния с перерисовкой UI при изменениях. Очень прост и универсален.
- **useEffect** — отвечает за побочные эффекты, такие как запросы к API, подписки и таймеры. Этот хук позволяет синхронизировать компонент с внешними системами или глобальным состоянием. В React 18 улучшено управление эффектами для конкурентного режима.
- **useContext** — позволяет получить доступ к контексту React, минуя пропсы. Полезен для глобальных настроек, темы, авторизации и других shared data.
- **useMemo** и **useCallback** — оптимизируют производительность, мемоизируя вычисления и функции соответственно, чтобы их не создавать заново при каждом рендере.
- **useRef** — хранит мутабельные объекты между рендерами без триггера обновления, часто используют для доступа к DOM-элементам или сохранения предыдущего значения.

#### Практический контекст
В реальных проектах React 18+ часто используют **useState** для управления формами, **useEffect** для загрузки данных, **useContext** для теминга и авторизации. Оптимизации с **useMemo** и **useCallback** помогают избежать лишних расчетов и ререндеров, что критично в больших приложениях с динамичным UI.
