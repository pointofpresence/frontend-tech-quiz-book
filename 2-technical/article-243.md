# Почему в Unity используют UniTask вместо стандартных .NET Task?

### Категории

Game development, Async, Performance, Optimization, Technical

### Краткий ответ

**Почему в Unity используют UniTask вместо стандартных .NET Task?**

- Unity — очень чувствителен к **производительности** и **памяти**  
- Стандартный Task генерирует много **heap-аллоций** и **боксинга**, тормозя GC  
- UniTask — **легковесная альтернатива** без аллокаций для async/await  
- Поддерживает **корутины**, работает совместно с Unity main thread и жизненным циклом объектов  
- Снижает **нагрузку на сборщик мусора** (GC), что критично для игр  
- Позволяет писать **асинхронный код с минимальными затратами** на мобильных и слабых платформах  
- Практическое применение — кастомный async без просадки FPS в Unity-проектах  

UniTask — оптимизация асинхронности под особенности Unity, улучшая производительность и управляемость памяти по сравнению с обычными .NET Task.

### Подробный ответ
#### Основной ответ
В **Unity** часто используют **UniTask** вместо стандартных **.NET Task** из-за особенностей внутри игрового движка и требований к производительности. Стандартные Task работают на основе TPL и выделяют значительное количество памяти, что может привести к частым вызовам сборщика мусора (GC), а это критично в играх и real-time приложениях. UniTask оптимизирован под Unity, минимизирует аллокации, работает быстрее и лучше интегрируется с циклом обновления движка.

#### Ключевые моменты
- **Отсутствие или минимизация аллокаций**: UniTask реализован с учётом особенностей Unity’s managed environment, что снижает давление на GC и помогает избежать фреймдропа.
- **Интеграция с Unity API**: UniTask поддерживает асинхронные операции Unity (например, ожидание `WaitForSeconds`, `AsyncOperation`, `CancellationToken` в Unity) напрямую, что упрощает синхронизацию с жизненным циклом объектов и сцен.
- **Легковесность и производительность**: UniTask — это value-type структура (struct), что значительно эффективнее heap-allocated классов Task, особенно в Unity 2018-2021, где у Task высокая нагрузка на сборщик мусора.

#### Практический контекст
В крупных игровых проектах на Unity с высокой частотой кадров (60+ FPS) переход на UniTask позволяет снизить задержки вызова GC, улучшить отзывчивость UI и стабильность поведения асинхронных операций. Стандартные Task подойдёт для backend-сервисов на.NET, UniTask — для game loop и взаимодействия с Unity API. Например, в Reactivity или DOTS архитектурах UniTask — де-факто стандарт для асинхронности.
