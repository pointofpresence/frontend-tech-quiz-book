# Какие инструменты можно использовать для анализа производительности?

### Категории

Teamwork, Architecture, Optimization, Performance analysis, Technical

### Краткий ответ

**Инструменты для анализа производительности**

- профилировщики CPU и памяти (например, **perf**, **VisualVM**, **Chrome DevTools**)  
- трассировщики (например, **Jaeger**, **Zipkin**) для распределённой диагностики  
- мониторинг метрик (например, **Prometheus**, **Grafana**) для метрик в реальном времени  
- инструмент логирования с анализом запросов (**ELK stack**, **Splunk**)  
- специализированные APM-системы (**New Relic**, **Datadog**, **Dynatrace**)  
- стеки трассировки стека (stack traces) для выявления узких мест  
- нагрузочное тестирование (**JMeter**, **Locust**) для стресс-тестов  
 
Используются для выявления **узких мест**, оптимизации ресурсов и улучшения отклика.

### Подробный ответ
#### Основной ответ
Для анализа производительности приложений и систем существует широкий спектр инструментов, которые позволяют измерять метрики, выявлять узкие места и оптимизировать работу. Выбор инструмента зависит от технологии, типа нагрузки и уровня детализации, который требуется. Часто пользуются комбинацией инструментов для мониторинга в реальном времени и глубокого профилирования.

#### Ключевые моменты
- **Профайлеры кода**: инструменты вроде **perf** (Linux), **VisualVM** (Java), **dotTrace** (.NET) дают детальный разбор потребления CPU, памяти, времени выполнения функций. Это помогает локализовать "горячие" участки кода и сделать оптимизацию целенаправленной.
- **Системный мониторинг**: **Prometheus** с **Grafana** для агрегации метрик (CPU, RAM, IO, latency) в продакшене, что обеспечивает визуализацию и alerting. Для Windows часто используют **PerfMon**, для Linux - **top**, **htop**, **iotop**.
- **Инструменты нагрузочного тестирования**: **JMeter**, **Gatling**, **Locust** позволяют создать искусственную нагрузку, измерить throughput, p99-латентность и обнаружить проблемы масштабирования.
- **APM-системы (Application Performance Monitoring)**: сервисы типа **New Relic**, **Datadog**, **Elastic APM** фиксируют транзакции, трассируют запросы по микросервисам и собирают логи ошибок, что позволяет быстро выявить причины деградации.
- **Специализированные средства**: для фронтенда — **Lighthouse** (анализ производительности вэб-страниц), для баз данных — **pg_stat_statements** в PostgreSQL, **EXPLAIN ANALYZE** для разбора запросов.

#### Практический контекст
В реальных проектах часто используют Prometheus + Grafana для мониторинга инфраструктуры и New Relic для глубокого APM с трассировкой распределенных систем. Для анализа горячих точек CPU в backend сервисах на Java — VisualVM или Flight Recorder. Нагрузочное тестирование на этапе предрелиза делают через JMeter, чтобы понять узкие места под пиковыми нагрузками.
