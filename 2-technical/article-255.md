# Как можно заменить position: sticky, если бы его не было?

### Категории

Performance, Technical, Async, Debugging, Error handling, CSS, Frontend, Problem solving

### Краткий ответ

**Как заменить position: sticky без поддержки**

- CSS fallback: нет прямого аналога без JS  
- Использовать **JS отслеживание прокрутки** (scroll event)  
- Вычислять позицию элемента относительно viewport  
- При достижении нужной точки менять стиль на **position: fixed**  
- Обратное изменение при прокрутке вверх  
- Контролировать плавность — дебаунсинг и оптимизация производительности  
- Практическое применение: кроссбраузерная поддержка и кастомный UI-контроль

### Подробный ответ
#### Основной ответ
Если бы **position: sticky** не существовал, его поведение можно было бы эмулировать с помощью **JavaScript** и обработки событий прокрутки (scroll). Идея в том, чтобы отслеживать положение элемента относительно viewport и переключать его стили между обычным и фиксированным позиционированием при достижении определённой точки прокрутки.

#### Ключевые моменты
- **Отслеживание scroll**: с помощью события `window.onscroll` или `Intersection Observer` вычисляем положение элемента относительно верхней части окна браузера.
- **Переключение position**: когда элемент достигает верхней границы viewport, меняем его стиль с `position: static/relative` на `position: fixed` с нужным `top`, чтобы он "прилипал" к экрану.
- **Учитывание оригинального пространства**: чтобы не "прыгал" контент, обычно создают placeholder с высотой оригинального элемента, который появляется, когда элемент становится fixed. Это сохраняет поток документа.
- **Оптимизация производительности**: для плавности используют **throttle/debounce** или `requestAnimationFrame` для обработки scroll, предотвращая просадки FPS.
- **Кроссбраузерность**: такой подход работает везде, даже где отсутствует поддержка `position: sticky`, но требует поддержки JavaScript.

#### Практический контекст
В реальных проектах подобное вручную реализуют, если нужна поддержка старых браузеров, например, IE11. Сейчас `position: sticky` поддерживается почти во всех современных браузерах (Chrome 56+, Firefox 32+, Safari 6.1+), поэтому JS fallback нужен редко. В сложных интерфейсах с кастомными поведениями ("липкие" сайдбары, шапки) часто комбинируют CSS и JS для гибкого контроля.

Таким образом, эмуляция `position: sticky` — это классический пример прогрессивного улучшения с помощью оптимизированного JS-кода, который повторяет ключевые визуальные эффекты CSS-свойства.
