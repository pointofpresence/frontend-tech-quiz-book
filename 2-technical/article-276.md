# Как работает useReducer и когда его использовать вместо useState или Redux?

### Категории

State management, Technical, Frontend, Optimization, React hooks

### Краткий ответ

**useReducer: работа и применение**

- React hook для **управления сложным состоянием**
- работает через **редьюсер**: функция (state, action) → newState
- облегчает управление **множественными и связанными** обновлениями состояния
- альтернативы: useState — для простых состояний, Redux — для глобального состояния
- когда использовать:
  - логика обновления **сложнее** setState
  - переходы состояния с четкой **последовательностью**
  - локальное состояние с **многочисленными действиями**
- useReducer проще Redux, не требует дополнительного трекинга и middleware
- идеален для UI-компонентов с **множественными состояниями** и "переходами"

Итог: useReducer — локальный, декларативный менеджер сложного состояния, между useState и глобальным Redux.

### Подробный ответ
#### Основной ответ
**useReducer** — это React-хук, который управляет состоянием через редьюсер (функцию, принимающую текущее состояние и action, и возвращающую новое состояние). Он полезен, когда логика обновления состояния сложная, включает множественные подтипы действий или когда новое состояние зависит от предыдущего. В отличие от **useState**, где обычно обновляется простое state-значение, **useReducer** позволяет централизовать всю логику обновления в одном месте, что облегчает поддержку и масштабирование, особенно для сложных форм или нескольких взаимосвязанных состояний.

#### Ключевые моменты
- **useState** подойдет для простых или изолированных состояний с небольшой логикой обновления (например, toggle, input).
- **useReducer** полезен при наличии сложных и вложенных правил изменения состояния, когда есть много действий с разным эффектом на стейт, либо когда нужно предотвращать избыточные ререндеры, так как обновления контролируются через dispatch.
- **Redux** — это глобальное управление состоянием с middleware, side effects и DevTools, подходит для крупных приложений с разделённым и масштабируемым глобальным состоянием. В React 18+ часто используют Redux Toolkit для упрощения.
- useReducer можно использовать локально внутри компонента или с контекстом для создания собственного аналога простой глобальной стейт-машины, но без всех возможностей Redux.

#### Практический контекст
В реальных приложениях **useReducer** часто применяется для управления сложными формами, фильтрами, шагами мультистеп-процессов или при работе с локальным состоянием, где нужна качественная изоляция бизнес-логики внутри компонента. Redux же используют, когда требуется глобальный, кросс-компонентный доступ к состоянию и мощный инструментарий для разработки и отладки.
