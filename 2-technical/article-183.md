# Что такое React Context? В чём его отличие от Redux?

### Категории

GroupBy, React context, State management, Technical, Design system, Frontend, Redux, Practical tasks, Event loop

### Краткий ответ

**React Context vs Redux**

- React Context — **встроенный API React** для передачи данных
- Позволяет избежать **прокидывания props** через дерево компонентов (prop drilling)
- Подходит для **локального** или редко меняющегося состояния (тема, язык)
- Redux — **отдельная библиотека** для управления состоянием приложения
- Имеет **централизованное** хранилище с **иммутабельностью** и предсказуемыми изменениями через действия (actions)
- Redux удобен для **сложных** состояний и бизнес-логики, а Context — для простых случаев
- В Redux легче делать **отладку и тестирование** благодаря middleware и devtools

Итог: Context — для простого обмена данными, Redux — мощный менеджер состояний для сложных приложений.

### Подробный ответ
#### Основной ответ
**React Context** — это встроенный механизм React для передачи данных по всему дереву компонентов без необходимости передавать props вручную на каждом уровне. Он решает проблему "прокидывания пропсов" (prop drilling) через множество вложенных компонентов, предоставляя глобальный доступ к определённому состоянию или функциям. Контекст удобен для простых случаев глобального состояния, например, тема приложения или язык интерфейса.

В отличие от Context, **Redux** — это отдельная библиотека для управления состоянием приложения на более глубоком уровне. Redux предлагает централизованный стор, строгую архитектуру с actions, reducers и middleware, что позволяет более детально контролировать логику изменений, а также масштабировать состояние в крупных приложениях.

#### Ключевые моменты
- **Область применения**: Context хорош для передачи конфигурационных или редко меняющихся данных. Redux — для сложных состояний с множеством изменений и взаимодействий.
- **Механизм обновлений**: Context вызывает переработку всех подписанных компонентов при смене значения, что может привести к снижению производительности без оптимизаций. Redux использует селекторы и мемоизацию (reselect) для ограниченного обновления компонентов.
- **Архитектура и масштабируемость**: Redux предлагает шаблоны для организации кода, логику сайд-эффектов через middleware (redux-thunk, redux-saga). Context не предоставляет таких инструментов из коробки.

#### Практический контекст
В React 18 и выше Context чаще используют для theming или i18n, где состояние статично или меняется редко. В больших приложениях с множеством разных сущностей и сложной логикой — предпочтительнее Redux или похожие state-менеджеры, чтобы избежать проблем с производительностью и поддержкой кода. Часто можно использовать их вместе: Context для UI-настроек, Redux для бизнес-логики.
