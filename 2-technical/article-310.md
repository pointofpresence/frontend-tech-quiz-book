# Как избежать утечек памяти в js?

### Категории

Browser, Performance, Memory management, Optimization, Frontend, Technical

### Краткий ответ

**Как избежать утечек памяти в js?**

- управление ссылками: освобождать объекты, когда не нужны  
- использовать **WeakMap/WeakSet** для слабых ссылок  
- избегать глобальных переменных — они живут всё время  
- следить за **замыканиями**: не захватывать лишние данные  
- удалять обработчики событий при ненадобности  
- оптимально управлять таймерами и асинхронными вызовами  
- использовать профайлеры памяти (Chrome DevTools) для поиска утечек

### Подробный ответ
#### Основной ответ
Утечки памяти в **JavaScript** — это ситуации, когда объекты остаются в памяти, несмотря на то, что они больше не нужны приложению, что приводит к постепенному росту используемой памяти и ухудшению производительности. Чтобы избежать утечек, важно понимать механизмы сборки мусора и держать ссылки на ненужные объекты минимальными.

#### Ключевые моменты
- **Разорвать ненужные ссылки:** например, очищать таймеры (`clearTimeout`, `clearInterval`), отменять слушатели событий, обнулять переменные и замыкания, чтобы сборщик мусора мог удалить объекты.
- **Избегать незаметных замыканий:** часто утечки возникают, когда замыкания сохраняют ссылки на большие объекты, которые больше не нужны; контролируйте области видимости и освобождайте ресурсы.
- **Работа с DOM:** удаляя элементы из DOM, также нужно убирать все связанные обработчики событий, иначе элементы останутся в памяти из-за живых ссылок.
- **WeakMap и WeakSet:** если необходимо хранить ссылки на объекты, которые не должны препятствовать сборке мусора, используйте слабые коллекции. Они позволяют сборщику мусора убирать объекты, если нет других сильных ссылок.
- **Профилирование памяти:** использовать инструменты, например, Chrome DevTools, для мониторинга heap snapshots и выявления утечек (объекты, которые постоянно увеличиваются без освобождения).

#### Практический контекст
При работе с большими одностраничными приложениями (SPA) на React 18+ или Vue 3, важно корректно управлять состоянием компонентов, чтобы при их размонтировании все связанные ресурсы освобождались — это снижает риски утечек и обеспечивает стабильность приложения даже после долгой работы. Также в Node.js важно закрывать соединения и очищать таймеры на сервере.

---
Таким образом, предотвращение утечек — это системная практика: управление ссылками, очищение ресурсов и регулярное профилирование памяти.
