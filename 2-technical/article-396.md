# Как узнать текущий фрейм вьюхи во время анимации?

### Категории

Mobile, Animation, View, UI, Technical, iOS

### Краткий ответ

**Как узнать текущий фрейм вьюхи во время анимации?**

- iOS-анимация, UIView
- При анимациях через UIView.animate фрейм меняется постепенно
- Чтобы узнать текущий фрейм, нужно использовать **presentation layer** CALayer
- presentationLayer содержит *текущие* значения анимируемых свойств в реальном времени
- пример: `view.layer.presentation()?.frame` вернёт актуальный фрейм во время анимации
- основной слой (`model layer`) всегда содержит конечное значение
- важно для синхронизации логики с визуальным состоянием анимации

Итого: **presentation layer** — источник текущего фрейма во время анимации UIView.

### Подробный ответ
#### Основной ответ
Текущую **фрейм-вьюху (frame of a view)** во время анимации в iOS можно получить, обратившись к свойству `presentationLayer` соответствующего слоя (`CALayer`). Именно в `presentationLayer` хранится текущее визуальное состояние слоя в процессе анимации, тогда как `modelLayer` содержит конечные значения свойств. Следовательно, во время анимации нужно использовать `view.layer.presentationLayer()?.frame`, чтобы получить актуальные координаты и размеры вью.

#### Ключевые моменты
- **presentationLayer** — это копия слоя с текущим отображаемым состоянием, обновляющаяся на каждом цикле рендеринга, отражая progress анимации.
- Свойство `view.frame` или `view.layer.frame` в процессе анимации не изменяется сразу, а обновляется только после её завершения, поэтому не подходит для получения текущих значений.
- Использование `presentationLayer` безопасно только на главном потоке и при наличии запущенной анимации, иначе может быть `nil`.

#### Практический контекст
Например, для распознавания позиции вьюхи при скролле с кастомной анимацией или для отслеживания коллизий в движущемся UI удобно брать `presentationLayer.frame` — это стандартный способ в UIKit и Core Animation с момента iOS 8+. В реальных проектах это часто используется при реализации draggable объектов с анимацией, когда нужно точно знать, где именно находится элемент на экране прямо сейчас, а не где он должен быть в конце.
