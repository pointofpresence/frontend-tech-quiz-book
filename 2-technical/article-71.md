# Какие типы данных в JSON вы знаете?

### Категории

Data formats, Serialization, Types, Technical, Data types, Data format

### Краткий ответ

**Типы данных JSON**

- формат обмена данными, текстовый
- **string** — строка в кавычках
- **number** — число (целое или с плавающей точкой)
- **boolean** — true или false
- **null** — отсутствие значения
- **object** — набор пар ключ-значение (объект)
- **array** — упорядоченный список значений
- используется для передачи структурированных данных между клиентом и сервером

### Подробный ответ
#### Основной ответ
**JSON (JavaScript Object Notation)** поддерживает ограниченный набор типов данных, чтобы обеспечить простоту и легкость сериализации/десериализации. Основные типы данных в JSON — это **объекты, массивы, строки, числа, логические значения и null**. Эти типы универсальны и легко парсятся большинством языков программирования.

#### Ключевые моменты
- **Объект (object)** — набор ключ-значение, где ключ всегда строка, а значение может быть любым JSON-типом. Обеспечивает структурированное хранение данных.
- **Массив (array)** — упорядоченный список значений одинаковых или разных типов. Позволяет хранить коллекции элементов.
- **Строка (string)** — последовательность символов Unicode, всегда в двойных кавычках, поддерживается escape-последовательности (например, `\n`, `\"`).
- **Число (number)** — может быть целым или с плавающей точкой, поддерживает как положительные, так и отрицательные значения, но неявно ограничен стандартом IEEE 754.
- **Логический тип (boolean)** — всего два значения: `true` и `false`.
- **Null** — специальное значение для обозначения отсутствия данных.

#### Практический контекст
В реальных проектах JSON используют для передачи данных между клиентом и сервером, хранения конфигураций и логов. Например, в React 18+ при работе с REST API JSON чаще всего содержит объекты и массивы с вложенными примитивами. Важно помнить, что JSON не поддерживает такие типы, как даты или функции, поэтому их приходится сериализовать отдельно.
