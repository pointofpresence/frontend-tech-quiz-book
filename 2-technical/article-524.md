# Как работает return в лямбде внутри inline-функции?

### Категории

Theory, Coding interview, Functional programming, Return, Lambda, Inline functions, Live coding, Technical

### Краткий ответ

**Как работает return в лямбде внутри inline-функции?**

- inline-функция выполняется **встроением** кода компилятором  
- **return** из лямбды внутри inline-функции — это **non-local return**  
- означает, что возврат происходит не из лямбды, а из внешней функции, содержащей вызов inline  
- поведение работает только в inline-функциях, чтобы избежать накладных расходов и обеспечить прямой выход  
- для локального возврата из лямбды используют **return@label** или **anonymous functions**  
- non-local return удобен для прерывания выполнения внешней функции из лямбды  
- применяется для чистого, лаконичного кода без лишних вложенных условий и флагов  

Итог: **return в лямбде в inline-функции прерывает исполнение самой внешней функции, не только лямбды.**

### Подробный ответ
#### Основной ответ
В Kotlin, когда мы используем **return** внутри лямбда-выражения, вложенного в **inline** функцию, механизм возврата ведёт себя иначе, чем в обычных функциях. В таком случае **return** производит **non-local return** — оно не просто покидает лямбду, а возвращает управление из самой вызывающей inline-функции. Это возможно благодаря тому, что inline-функции "встраивают" тело лямбды прямо в тело вызывающей функции, что позволяет такому `return` досрочно завершать внешний контекст.

#### Ключевые моменты
- **Non-local return** в inline-функциях: обычно внутри лямбды `return` уходит из неё, но в inline-функциях `return` выкидывает управление из всей inline-функции, что невозможно в не-inline-контексте.
- Чтобы вернуть из лямбды **только из неё**, нужно использовать **локальный return** с меткой, например `return@lambda`, что прервёт только выполнение лямбды, а не внешней функции.
- В нем-inline функциях подобное поведение невозможно: `return` внутри лямбды работает как обычный возврат из неё (local return). Это ограничение связано с тем, что тело лямбды не встраивается в вызывающую функцию.
  
#### Практический контекст
Типично `return` внутри лямбд в inline-функциях применяется в таких функциях как `forEach`, `filter`, `map` из стандартной библиотеки Kotlin, где можно досрочно прервать проход с помощью обычного `return`. Например, в `inline fun forEach(action: (T) -> Unit)`, если внутри action использовать `return`, выполнение forEach прервётся целиком.

Таким образом, понимание разницы между **local** и **non-local** return критично для корректной работы с лямбдами и inline-функциями в Kotlin, особенно для контроля потока и предотвращения неожиданных преждевременных выходов из функций.
