# В чём разница между Serializable и Parcelable?

### Категории

Android, Data transfer, Performance, Serialization, Technical

### Краткий ответ

**Разница между Serializable и Parcelable**

- **Serializable** — Java стандартный интерфейс для превращения объекта в поток байтов.
- **Parcelable** — Android-интерфейс для передачи объектов между компонентами через Binder.
- Serializable проще в реализации, требует минимум кода (маркеров интерфейс).
- Parcelable быстрее по производительности, так как оптимизирован для Android.
- Parcelable требует вручную описывать процесс сериализации (более сложен).
- Serializable подходит для сохранения в файл и сети, Parcelable — для IPC и Intents.
- В Android рекомендован Parcelable из-за эффективности и скорости обмена данными.

### Подробный ответ
#### Основной ответ
**Serializable** и **Parcelable** — это два интерфейса в Android для сериализации объектов, то есть преобразования их в поток байтов для передачи между компонентами (например, через Intent). Главное различие в том, что **Serializable** — стандартный Java-интерфейс, простой в использовании, но медленный по производительности, а **Parcelable** — собственный Android-интерфейс, значительно быстрее, но требует ручной реализации методов.

#### Ключевые моменты
- **Serializable** — реализуется автоматически с помощью рефлексии, не требует дополнительного кода, но из-за рефлексии и создания промежуточных объектов тормозит, это может привести к задержкам и повышенному потреблению памяти.
- **Parcelable** — оптимизирован для Android, работает напрямую с Parcel (контейнером для упакованных данных), требует реализации методов `writeToParcel()` и `CREATOR`, что повышает скорость передачи в 2-3 раза по сравнению с Serializable.
- **Trade-off** — Serializable проще использовать и подходит для небольших и простых объектов, а Parcelable рекомендуется для высокопроизводительных приложений с ограничениями по памяти и скорости (например, передача сложных моделей данных между Activity или Service в Android).

#### Практический контекст
В реальных проектах с Android 11+ и многопоточными приложениями часто выбирают **Parcelable** для обмена данными между компонентами, потому что это снижает задержки и предотвращает OutOfMemoryError при больших объемах. В то же время, если нужно быстро сохранить объект в файл или отправить через сеть без Android-зависимости, используют **Serializable**.
