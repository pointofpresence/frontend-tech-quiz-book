# В чем различия forEach и map?

### Категории

Live coding, Iteration, TypeScript, Technical, Array methods, Frontend, Functional programming, Array, JavaScript

### Краткий ответ

**forEach vs map**

- обе работают с массивами (итерирование)
- forEach: перебирает элементы, **не возвращает значения**
- map: создаёт новый массив одинаковой длины
- map применяется для **преобразования данных**
- forEach — побочные эффекты, действия без возврата
- map иммутабелен, forEach — нет
- выбор зависит от: нужен **результат** (map) или **операция** (forEach)

### Подробный ответ
#### Основной ответ
**forEach** и **map** — методы массива в JavaScript, но они имеют ключевые различия в назначении и поведении. **forEach** служит для перебора массива и выполнения побочных эффектов (например, вывод в консоль, изменение внешних данных), не возвращая новый массив. **map** же используется для преобразования каждого элемента массива, возвращая новый массив того же размера, но с изменёнными значениями.

#### Ключевые моменты
- **Возвращаемое значение**: forEach возвращает `undefined`, map возвращает новый массив, позволяя цепочку вызовов и функциональное программирование.
- **Иммутабельность**: map создаёт новый массив, не изменяя исходный, что соответствует принципам чистых функций, forEach обычно применяется для мутаций или side-effect’ов.
- **Performance**: разница минимальна, но map предпочтителен, если нужно сохранить и использовать результат, forEach — когда результат не нужен.
- В React 18+ часто используют map для генерации JSX списков, что требует возвращаемого массива элементов.

#### Практический контекст
В реальных проектах map используется для создания новых массивов с преобразованными данными, например, при фильтрации или форматировании API-ответов. forEach удобен для логирования, обновления внешних состояний или вызовов функций без возврата результата. Понимание разницы помогает писать чистый, читаемый и функциональный код.
