# Есть компонент, внутри него падает какой-то из дочерних компонентов, из-за этого ломается приложение, как можно этого избежать?

### Категории

Browser, JavaScript, Error handling, Stability, Component, Optimization, Frontend, Technical

### Краткий ответ

**Изоляция ошибок в React**

- React Error Boundaries: класс с методом `componentDidCatch`
- Ловят ошибки в дочерних компонентах, не ломают дерево
- Позволяют рендерить запасной UI при ошибке
- Работают только с ошибками в рендере, lifecycle и конструкторах дочерних компонентов
- Для асинхронных ошибок нужны другие методы (try/catch, внешний обработчик)
- Использование повышает **устойчивость** интерфейса
- Применяется для критичных секций, чтобы локализовать сбои и улучшить UX

### Подробный ответ
#### Основной ответ
Чтобы избежать полного краха приложения из-за ошибки в дочернем компоненте, в React (начиная с версии 16) используется концепция **Error Boundaries** — специальные компоненты, которые ловят ошибки в дереве дочерних компонентов и предотвращают их всплытие выше. Такой подход позволяет заменить сломанный компонент на запасной UI (например, fallback UI) и сохранить работоспособность всего приложения.

#### Ключевые моменты
- **Error Boundary** — это класс-компонент с методами `componentDidCatch(error, info)` и `static getDerivedStateFromError(error)`, в котором можно отловить ошибки в рендеринге, lifecycle методах или конструкторах дочерних компонентов.
- Они **не ловят ошибки в асинхронных коллбэках**, событиях или внутри самого Error Boundary.
- Использование Error Boundaries позволяет показывать пользователю запасной интерфейс, логировать ошибки во внешние сервисы (Sentry, LogRocket) и при этом избежать падения всего UI.

#### Практический контекст
В крупных React-приложениях стандартной практикой является оборачивать критичные части UI (например, крупные страницы или сложные виджеты) в Error Boundary. Это дает высокий уровень устойчивости: даже если один виджет/страница падает из-за ошибки, пользователь получает понятное сообщение об ошибке или возможность обновить только эту часть, а не все приложение.
