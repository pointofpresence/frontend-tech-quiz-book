# Что такое Promise?

### Категории

Promises, Sorting, Validation, Backend, Technical, Async, TypeScript, Frontend, Database, Microservices, Concurrency, API, JavaScript

### Краткий ответ

**Что такое Promise?**  
- объект для работы с **асинхронными** операциями в JS  
- представляет **отложенный** результат, доступный в будущем  
- имеет 3 состояния:  
  - **pending** (ожидание)  
  - **fulfilled** (выполнено успешно)  
  - **rejected** (отклонено с ошибкой)  
- обеспечивает **цепочку** обработки через then/catch  
- упрощает управление асинхронным кодом, избегая callback hell  
- используется для работы с API, таймерами, файлами и сетевыми запросами  
- позволяет **композицию** асинхронных действий (Promise.all, Promise.race)

### Подробный ответ
#### Основной ответ
**Promise** в JavaScript — это объект, который представляет результат асинхронной операции, который может быть выполнен (fulfilled), отклонён (rejected) или находиться в состоянии ожидания (pending). Promises позволяют писать асинхронный код, избегая «ад колбэков», улучшая читаемость и структурированность.

#### Ключевые моменты
- **Состояния:** Promise начинается в состоянии *pending* (ожидание), затем может перейти либо в *fulfilled* (завершён с результатом), либо в *rejected* (завершён с ошибкой).
- **Методы:** Основные методы — `then` для обработки успешного результата, `catch` для ошибок и `finally` для кода, который выполняется в любом случае.
- **Цепочки и композиция:** Промисы поддерживают цепочки, что позволяет последовательно обрабатывать результаты и ошибки, упрощая сложные асинхронные последовательности.
- **Отличие от колбэков:** Promises решают проблемы вложенности и непредсказуемого порядка выполнения, давая валидный и предсказуемый паттерн работы с async.

#### Практический контекст
В React 18 и Node.js 16+ Promises — стандарт de-facto для работы с асинхронностью, например, при запросах к API (fetch возвращает Promise), работе с файлами (fs.promises), или таймерах (Promise-based delay). Они чаще используются совместно с корутинами `async/await` для ещё более чистого синтаксиса и управления ошибками.
