# Отличие анимации при использовании requestAnimationFrame и SetInterval

### Категории

Performance, Animation, Timers, Technical, Browser, Frontend, Event loop, Optimization

### Краткий ответ

**Отличие анимации requestAnimationFrame и setInterval**

- область: веб-анимация, таймеры  
- requestAnimationFrame — **браузерный API** для анимации  
- синхронизируется с **обновлением кадра** (обычно 60fps)  
- автоматически **подавляет** вызовы при свёрнутом табе (экономия ресурсов)  
- setInterval — **регулярное выполнение** кода по интервалу, без синхронизации с кадрами  
- setInterval может вызвать **дрожание/заикание**, если не совпадает с частотой обновления экрана  
- requestAnimationFrame обеспечивает **плавность** и лучшее энергопотребление для анимаций

Итого: для качественной анимации лучше использовать **requestAnimationFrame** из-за синхронизации с перерисовкой и оптимизации браузером.

### Подробный ответ
#### Основной ответ
Основное отличие анимации на основе **requestAnimationFrame** и **setInterval** заключается в том, что **requestAnimationFrame** синхронизирован с частотой обновления дисплея, тогда как **setInterval** запускает функцию через фиксированные интервалы времени без учета перерисовки экрана. Это делает requestAnimationFrame более плавным и эффективным для анимаций в браузере.

#### Ключевые моменты
- **Синхронизация с рендером**: requestAnimationFrame вызывает callback примерно перед следующим перерисовыванием экрана (~60 FPS на большинстве мониторов), что минимизирует мерцание и джиттер. SetInterval же запускает функцию строго по таймеру, что может привести к "рассинхронизации" с частотой обновления и дерганой анимации.
- **Оптимизация ресурсов**: requestAnimationFrame автоматически приостанавливается, если вкладка не активна или окно свернуто, экономя CPU и энергию. SetInterval продолжает работать, зачастую бесполезно потребляя ресурсы.
- **Гибкость и точность**: с requestAnimationFrame можно использовать временные метки (timestamp), передаваемые в callback, для более точного расчета прогресса анимации, в то время как setInterval требует ручного управления временем, что сложнее и менее надежно.

#### Практический контекст
В React 18+ и современных SPA requestAnimationFrame — стандарт для анимаций UI (например, при кастомных переходах или canvas анимациях). SetInterval иногда применяется для задач с фиксированной периодичностью, не зависящих от визуаьного обновления, например, для таймеров или опроса данных, но не для плавной анимации.
