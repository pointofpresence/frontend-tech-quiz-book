# В чём разница между select_related и prefetch_related?

### Категории

Query optimization, Performance, Django, Backend, Database, ORM, Query, Optimization, Technical

### Краткий ответ

**Разница между select_related и prefetch_related**  
- оба — методы оптимизации запросов Django ORM для связанных моделей  
- **select_related**: использует SQL JOIN, загружает связанные объекты одной сложной выборкой  
- эффективно для связей **ForeignKey** и **OneToOne** (один к одному/многие к одному)  
- **prefetch_related**: выполняет отдельные SQL-запросы и объединяет результат на уровне Python  
- подходит для связей **ManyToMany** и обратных ForeignKey (многие ко многим, один ко многим)  
- select_related быстрее при простой вложенности, prefetch_related гибче для сложных связей и больших наборов  
- правильный выбор минимизирует количество запросов и улучшает производительность приложения

### Подробный ответ
#### Основной ответ
**select_related** и **prefetch_related** — это два оптимизационных механизма в Django ORM для уменьшения количества запросов к базе данных при работе с связанными моделями, но они используют разный подход и подходят для разных типов связей.

**select_related** использует SQL JOIN и подгружает связанные объекты одним объединённым SQL-запросом. Это оптимально для **ForeignKey** и **OneToOne** связей, когда нужно получить связанные модели вместе с основными.

**prefetch_related** выполняет отдельный запрос для связанных объектов и затем связывает их уже в Python. Он предпочтителен для **ManyToMany** и обратных связей, где использовать JOIN неэффективно или невозможно.

#### Ключевые моменты
- **select_related** уменьшает количество запросов до 1, объединяя таблицы через SQL JOIN, но увеличивает объём результирующего набора (удобно для жёстких связей один к одному или один ко многим).
- **prefetch_related** всегда делает минимум два запроса: один для основных записей и один (или несколько) для связанных, затем объединяет данные на уровне Python. Это эффективнее при работе с большими множествами связанных данных и сложными отношениями.
- В Django 3.2+ есть улучшения в **prefetch_related**, поддерживающие сложные кастомные queryset и вложенный prefetch, что облегчает работу с глубоко вложенными связями.

#### Практический контекст
Типично в проектах с Django использовать **select_related** для того, чтобы агрегировать данные из связанных записей, например, у модели `Book` подгружать данные автора (`author = ForeignKey`) сразу через join, а для сборов связанных множеств, например, комментариев или тегов (ManyToMany), применять **prefetch_related**, чтобы избежать N+1 проблемы и не перегружать базу сложными join.
