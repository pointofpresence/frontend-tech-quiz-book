# Чем отличаются функции sort и sorted?

### Категории

API, Python, Functions, Backend, Sorting, Technical

### Краткий ответ

**Чем отличаются функции sort и sorted?**

- Оба сортируют коллекции в Python  
- **sort()** — метод списка, сортирует **на месте** (in-place)  
- **sorted()** — функция, возвращает **новый отсортированный список**  
- sort() меняет исходный список, возвращает None  
- sorted() не меняет исходные данные, работает с любыми итерируемыми объектами  
- Оба принимают ключ (key) и порядок (reverse) для кастомизации  
- Выбор: sort() — быстрее, если порядок в исходнике важен; sorted() — для неизменяемых данных или копии  

Итого: sort() — in-place сортировка списка; sorted() — универсальная, возвращает новый объект.

### Подробный ответ
#### Основной ответ
Функции **`sort`** и **`sorted`** в Python предназначены для сортировки, но существенно отличаются по применению и поведению. `sort` — это метод списков, который **изменяет исходный список на месте** (in-place), не возвращая нового объекта. В то время как `sorted` — это встроенная функция, которая **возвращает новый отсортированный список**, оставляя исходный неизменным.

#### Ключевые моменты
- **`list.sort()`** работает только с объектами типа `list` и выполняет сортировку на месте, что экономит память, но ведет к изменению оригинальных данных.
- **`sorted()`** принимает любой итерируемый объект (списки, кортежи, словари по ключам, строки), возвращая новый отсортированный список, поэтому подходит для неизменяемых коллекций.
- Оба поддерживают параметры `key` и `reverse` для кастомизации порядка сортировки.
- Использование `sort` предпочительно, когда нужно отсортировать список эффективно без создания копии, а `sorted` — когда нужно сохранить исходные данные или сортировать другие итерируемые объекты.

#### Практический контекст
В производственных проектах часто выбирают `sort()` для сортировки больших списков с целью минимизации памяти (например, списки из миллионов элементов). `sorted()` применяют, когда данные нужно сохранить неизменными — например, при цепочке обработки, где оригинальный порядок важен для других операций или при работе с кортежами и другими итерируемыми объектами. В React-подобных библиотеках или функциональном стиле программирования предпочтительна именно `sorted`, т.к. избегается мутация состояния.
