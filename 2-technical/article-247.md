# REST: что это и на каких принципах основан?

### Категории

API, Architecture, Backend, Database, Messaging, Queues, Technical

### Краткий ответ

**REST: что это и на каких принципах основан?**  
- архитектурный стиль для **веб-сервисов**  
- ресурсы представлены **URI**  
- использование ограниченного набора **HTTP-методов** (GET, POST, PUT, DELETE)  
- принцип **stateless** — каждое обращение независимо  
- клиент-серверная архитектура, отделение UI и хранения данных  
- поддержка **кэширования** для повышения производительности  
- универсальный интерфейс через стандартные протоколы HTTP  
- применяется для **масштабируемых и простых** API в интернете

### Подробный ответ
#### Основной ответ
**REST (Representational State Transfer)** — это архитектурный стиль проектирования распределённых систем, особенно веб-сервисов, который ориентирован на использование стандартных HTTP-методов и ресурсов с уникальными URI. Его ключевая идея — представлять данные и операции над ними через понятия ресурсов и их состояний, доступных через унифицированный интерфейс.

REST основан на шести основных принципах (дизайн-ограничениях), формализованных в диссертации Роя Филдинга (2000):

1. **Клиент-сервер** — разделение ответственности, что позволяет развивать UI и сервер независимо.
2. **Отсутствие состояния (stateless)** — каждый запрос содержит всю необходимую информацию, сервер не хранит состояние между запросами.
3. **Кэширование** — ответы могут быть явно кэшируемыми, что повышает производительность и масштабируемость.
4. **Единообразие интерфейса** — унифицированные операции (GET, POST, PUT, DELETE) для взаимодействия с ресурсами через их URL.
5. **Слои системы** — архитектура может иметь промежуточные серверы (прокси, балансировщики), которые не видны клиенту.
6. **Код по требованию (опционально)** — сервер может передавать исполняемый код клиенту (например, JavaScript), расширяя функциональность.

#### Ключевые моменты
- **Ресурсы** — это основные сущности, которые идентифицируются URI, а состояние ресурса представлено в виде документов (JSON, XML и др.).
- **Статeless** обеспечивает лёгкое масштабирование и надёжность, поскольку нет необходимости сохранять сессии на сервере.
- **Использование стандартных HTTP методов** соответствует CRUD: GET — чтение, POST — создание, PUT — обновление, DELETE — удаление.
- Важным аспектом является соблюдение **конвенций и idempotency** — например, PUT должен быть идемпотентным, чтобы сторонние вызовы не приводили к непредсказуемым изменениям.

#### Практический контекст
REST получил широкое распространение в веб-разработке благодаря простоте и совместимости с HTTP. Многие современные API (Twitter API v1.1, GitHub API, Google APIs) построены на REST, часто с форматом JSON для передачи состояния. В React/Angular проектах REST API обычно служит источником данных для frontend-приложений, а инструменты вроде Postman или Swagger облегчают тестирование и документирование таких сервисов.
