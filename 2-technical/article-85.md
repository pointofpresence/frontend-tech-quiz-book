# Как написать SQL-запрос для выборки данных с группировкой и условием на количество уникальных связанных записей?

### Категории

Count, Database, Grouping, Query, Unique, Aggregation, Technical, SQL

### Краткий ответ

**SQL: группировка с условием на уникальные связанные записи**  
- выборка с **GROUP BY** по основным полям  
- подсчёт уникальных связанных записей через **COUNT(DISTINCT related_column)**  
- фильтрация групп через **HAVING** (нельзя в WHERE)  
- пример:  
```sql
SELECT main.id, COUNT(DISTINCT related.id) AS unique_count  
FROM main  
JOIN related ON related.main_id = main.id  
GROUP BY main.id  
HAVING COUNT(DISTINCT related.id) > n;
```  
- практическое использование: фильтрация по количеству связей (например, пользователей с >=3 заказами)

### Подробный ответ
#### Основной ответ
Для выборки данных с группировкой и условием на количество уникальных связанных записей в SQL обычно используют агрегатную функцию `COUNT(DISTINCT ...)` вместе с `GROUP BY` и `HAVING`. Основная идея — сгруппировать записи по нужному признаку, посчитать уникальные связанные элементы и отфильтровать группы, у которых количество таких элементов соответствует условию.

#### Ключевые моменты
- **Группировка (`GROUP BY`)** используется для объединения записей по одному или нескольким полям, например, по идентификатору пользователя или категории.
- Функция **`COUNT(DISTINCT column_name)`** подсчитывает количество уникальных значений в группе, что важно для определения количества связанных записей без дублирования.
- **Клаузула `HAVING`** применяется для фильтрации уже агрегированных групп по условию количества уникальных записей: например, отбрать только группы с более чем N уникальными связанными элементами.

#### Пример
Если у вас есть таблицы `orders` (заказы) с полями `(order_id, customer_id)` и вы хотите получить клиентов, у которых более 3 уникальных заказов, запрос может выглядеть так:

```sql
SELECT customer_id, COUNT(DISTINCT order_id) AS unique_orders
FROM orders
GROUP BY customer_id
HAVING COUNT(DISTINCT order_id) > 3;
```

#### Практический контекст
Такой запрос часто используется для анализа поведения пользователей — например, выбрать клиентов, сделавших несколько уникальных покупок, или авторов с определённым количеством уникальных публикаций. В PostgreSQL 14+ или MySQL 8+ этот подход отлично масштабируется и эффективно работает, если есть подходящие индексы (например, по `customer_id` и `order_id`).
