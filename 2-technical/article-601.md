# Что делают preventDefault и stopPropagation?

### Категории

stopPropagation, Promises, Error handling, Async, Closures, Debugging, Event handling, preventDefault, Frontend, Technical

### Краткий ответ

**Что делают preventDefault и stopPropagation?**

- **preventDefault** — отменяет стандартное поведение браузера для события (например, переход по ссылке, отправка формы).  
- Используется, чтобы заменить или предотвратить действие по умолчанию.  
- **stopPropagation** — прерывает дальнейшее распространение (всплытие/погружение) события по DOM.  
- Останавливает обработчики на родительских элементах, чтобы событие не "всплывало" выше.  
- Часто применяются вместе для тонкого управления обработкой событий в сложном UI.  
- Главное отличие: preventDefault влияет на дефолт браузера, stopPropagation — на поток события внутри DOM.  
- Ключевые для кастомного поведения и предотвращения конфликтов при взаимодействии с элементами.

### Подробный ответ
#### Основной ответ
**preventDefault()** и **stopPropagation()** — это методы объекта события (Event) в JavaScript, которые управляют поведением и распространением событий в DOM.

- **preventDefault()** отменяет стандартное действие браузера, связанное с событием. Например, при клике на ссылку по умолчанию происходит переход по URL, но вызов preventDefault() предотвратит это.
- **stopPropagation()** останавливает дальнейшее распространение события вверх по DOM-дереву (фазу bubbling), что позволяет избежать вызова обработчиков событий на родительских элементах.

#### Ключевые моменты
- **preventDefault()** полезен, когда нужно переопределить поведение по умолчанию, например, предотвратить отправку формы, открытие контекстного меню или переход по ссылке.
- **stopPropagation()** важно использовать, если нужно «задержать» событие на конкретном элементе, не давая родителям на него реагировать, что особенно актуально в сложных UI с вложенными компонентами.
- Можно встретить также **stopImmediatePropagation()**, которая дополнительно предотвращает вызов других обработчиков события на том же самом элементе.

#### Практический контекст
В React 18 и чистом JavaScript эти методы часто применяются при управлении формами, модальными окнами и кастомными виджетами, чтобы обеспечить контроль над поведением UI и избежать конфликтов между обработчиками событий. Например, при реализации выпадающих меню stopPropagation предотвращает закрытие меню при клике внутри него, а preventDefault отменяет перезагрузку страницы при сабмите формы.
