# Фазы EventLoop

### Категории

Async, Backend, Event loop, Technical

### Краткий ответ

**Фазы Event Loop в Node.js**

- управление асинхронным JS-событиями
- события обрабатываются циклом по фазам
- основные фазы:  
  - **timers**: выполнение setTimeout/setInterval  
  - **pending callbacks**: завершение ошибок I/O  
  - **idle, prepare**: внутренние операции  
  - **poll**: получение новых событий I/O (key phase)  
  - **check**: выполнение setImmediate  
  - **close callbacks**: обработка закрытых ресурсов  
- фазы выполняются последовательно, каждая обрабатывает очередь callback
- Event Loop обеспечивает **неблокирующую** работу и параллелизм

---

##### Подробности:

Event Loop — это механизм Node.js для обработки асинхронных операций без блокировки. Каждая фаза — это очередь колбеков определённого типа. Сначала выполняются таймеры (setTimeout), затем завершения I/O, затем чтение событий (poll), потом сразу setImmediate (check), и так далее. Если в poll очереди нет, Event Loop падает в ожидание, пока не появятся события. Это гарантирует эффективное управление ресурсами и порядок выполнения callback'ов.

### Подробный ответ
#### Основной ответ
**Event Loop** в Node.js — это механизм, который обрабатывает асинхронные операции, позволяя выполнять неблокирующий ввод-вывод. Он состоит из нескольких фаз, каждая из которых отвечает за обработку определённых типов задач и колбеков. Важные фазы Event Loop: **timers**, **idle, prepare**, **poll**, **check**, и **close callbacks**. Понимание их последовательности и особенностей помогает эффективно управлять асинхронным кодом и оптимизировать производительность.

#### Ключевые моменты
- **Timers** — выполнение колбеков setTimeout и setInterval, если истёк заданный таймер.
- **Pending Callbacks** — обработка колбеков системных операций, например, ошибок в сетевых запросах.
- **Poll** — главная фаза для получения новых событий ввода-вывода; в ней обрабатываются колбеки, полученные от операционной системы.
- **Check** — выполнение колбеков setImmediate, которые запускаются после poll-фазы, независимо от таймеров.
- **Close Callbacks** — вызов обработчиков события закрытия (close), например, сокетов.
  
Порядок фаз определён и повторяется циклично, что даёт гарантии по выполнению асинхронных операций. На практике, **setImmediate** и **setTimeout** работают с разной приоритетностью в Event Loop, что важно учитывать в высоконагруженных системах.

#### Практический контекст
В реальных проектах, например, в API на Node.js 18+, правильное использование фаз Event Loop критично для минимизации задержек (latency ~10-20ms), особенно при большом количестве сетевых запросов и таймеров. Знание приоритетов фаз помогает избежать "starvation" — когда отдельные колбеки постоянно откладываются, и улучшить загрузку CPU. Часто для отладки используют `--trace-event-loop` или профилирование через `clinic.js`.
