# У нас на сцене тысяча 3D-объектов с одинаковым мешем, но разными материалами — сколько будет draw calls и как это оптимизировать?

### Категории

Game development, Architecture, Performance, Optimization, Rendering, Technical

### Краткий ответ

**Draw calls для 1000 3D-объектов с одинаковым мешем, разными материалами и оптимизация**

- Каждый материал требует отдельный **draw call** → 1000 draw calls, т.к. материалы разные  
- Draw call — это команда графическому API для отрисовки одной группы объектов с одним материалом  
- Основная проблема: **CPU-GPU bottleneck** из-за большого количества переключений состояний (материалов)  
- Оптимизация: **батчинг** невозможен напрямую из-за разных материалов  
- Варианты оптимизации:  
  - **Material atlasing** (сбор текстур в один атлас) + **динамический батчинг**  
  - **Instancing** с передачей параметров материала через **shader uniforms** или **texture arrays**  
  - **Deferred Rendering** — позволяет отложить работу с материалами и сэкономить draw calls  
  - Использовать **GPU-driven rendering** для группировки и минимизации вызовов  
- Итог: без оптимизации — 1000 draw calls; с материалами на атласе/instancing — можно снизить до 1–10 draw calls, значительно улучшая производительность

### Подробный ответ
#### Основной ответ
Если на сцене тысяча 3D-объектов с одинаковым мешем, но разными материалами, обычно будет **по одному draw call на каждый уникальный материал для каждого объекта**. В худшем случае — до 1000 draw calls, если у каждого объекта свой уникальный материал. Это связано с тем, что для изменения материала GPU требует отдельного вызова отрисовки, так как материалы обычно меняют шейдеры, текстуры и рендер-стейты.

#### Ключевые моменты
- **Draw call зависит от переключения материала** — даже при одном и том же меше, разные материалы заставляют делать разные draw calls.
- **Оптимизация через material batching невозможна**, если материалы действительно уникальны, но можно применять **texture atlasing** или **instancing с uniform-параметрами**.
- **GPU Instancing** — наиболее часто используемое решение: отправлять меш и материал один раз, а различия в материалах передавать через параметры instancing (например, цвет или дополнительные текстурные координаты), уменьшая draw calls до одного.
- Если материалы сильно отличаются (разные шейдеры), то можно сгруппировать объекты по материалам, минимизируя количество draw calls, но не избегая их всех.

#### Практический контекст
В таких случаях в игровых движках (Unity, Unreal Engine) используют **GPU Instancing с параметризацией материалов** для оптимальной отрисовки тысяч одинаковых мешей с вариациями. Если вариации слишком сильны, применяют **цветовой атлас текстур**, чтобы несколько материалов занимали один texture array или atlas, позволяя загружать все с меньшим количеством draw calls (~10-50 вместо 1000) при минимальном визуальном ущербе.
