# Как реализовать транзакции через SQL-запросы?

### Категории

API, Backend, Database, Transactions, Technical

### Краткий ответ

**Как реализовать транзакции через SQL-запросы?**

- Транзакция — набор операций, объединённых в единое целое  
- Основные команды:  
  - **BEGIN TRANSACTION** — начало транзакции  
  - **COMMIT** — подтверждение изменений  
  - **ROLLBACK** — откат изменений при ошибках  
- Гарантирует **атомарность, согласованность, изолированность, стабильность** (ACID)  
- Позволяет защититься от частичных обновлений данных  
- Используется для сложных операций с несколькими запросами  
- Поддерживается большинством СУБД (PostgreSQL, MySQL, MS SQL)  
- Практически обязателен при финансовых и критичных бизнес-операциях

### Подробный ответ
#### Основной ответ
Транзакции в SQL реализуются с помощью команд **BEGIN (START) TRANSACTION**, **COMMIT** и **ROLLBACK**, которые позволяют группировать несколько операций чтения и записи в атомарную единицу работы. Транзакция гарантирует, что все операции либо успешно применятся вместе, либо в случае ошибки будут отменены, сохраняя целостность данных.

#### Ключевые моменты
- **BEGIN TRANSACTION** — стартует транзакцию, переключая сессию в режим, где изменения не фиксируются автоматически.
- **COMMIT** фиксирует все изменения, сделанные в транзакции, делая их видимыми другим сессиям.
- **ROLLBACK** отменяет изменения, выполненные внутри транзакции, возвращая базу в состояние до ее начала.
- Важно учитывать уровень изоляции транзакции (READ COMMITTED, REPEATABLE READ, SERIALIZABLE) — это влияет на видимость данных и возможные блокировки.
- При использовании сложной бизнес-логики транзакции могут быть вложенными (SAVEPOINT), что позволяет частично откатывать операции.

#### Практический контекст
На практике в PostgreSQL 14+ или MySQL 8 чаще всего применяют следующий подход:  
```sql
BEGIN;
UPDATE accounts SET balance = balance - 100 WHERE id = 1;
UPDATE accounts SET balance = balance + 100 WHERE id = 2;
COMMIT;
```
Если происходит ошибка в одном из запросов, вызывается `ROLLBACK` для отмены изменений, что исключает частичные обновления и поддерживает консистентность. Такой подход стандартен для банковских операций, изменения инвентаря и других критичных данных.
