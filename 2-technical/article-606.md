# Какие существуют способы сохранения состояния Activity?

### Категории

Mobile, Activity, Android lifecycle, State management, Technical

### Краткий ответ

**Способы сохранения состояния Activity**

- жизненный цикл Activity (основа понимания сохранения)
- onSaveInstanceState(Bundle) — сохранение в Bundle (короткоживущие данные)
- onRestoreInstanceState(Bundle) / onCreate(Bundle) — восстановление из Bundle
- ViewModel — хранение состояния при повороте, не уничтожая Activity
- savedStateHandle (в ViewModel) — удобное сохранение промежуточных данных
- persistency (SharedPreferences, БД, файлы) — для долгоживущих данных
- фреймворки типа Jetpack Navigation — помогают сохранять стейт навигации
- преимущество Bundle — автоматическое восстановление после уничтожения системой
- ViewModel — не хранит данные при полной смерти процесса (нужно комбинировать)
- комбинирование методов обеспечивает надежность и UX

Ответ адаптирован под senior уровень с упором на реальные механизмы и ограничения.

### Подробный ответ
#### Основной ответ
Сохранение состояния **Activity** в Android — ключевой момент для корректной работы приложения при смене конфигурации (повороте экрана, смене языка) или при уничтожении Activity системой для освобождения ресурсов. Основные способы сохранения состояния включают использование **onSaveInstanceState()**, сохранение с помощью **ViewModel**, а также сохранение данных в **Persistent Storage** (например, SharedPreferences или базу данных).

#### Ключевые моменты
- **onSaveInstanceState(Bundle)** — стандартный и обязательный метод для сохранения небольших данных (например, состояние UI элементов). Данные помещаются в Bundle и восстанавливаются в onCreate() или onRestoreInstanceState(). Однако Bundle ограничен по объему и не подходит для больших или сложных данных.
- **ViewModel (из архитектурных компонентов Android Jetpack)** — хранит состояние во время пересоздания Activity, как при повороте экрана. ViewModel привязан к жизненному циклу Activity и уничтожается только при завершении Activity. Отлично подходит для хранения бизнес-логики и состояния UI без необходимости сериализации.
- **Persistent Storage** — SharedPreferences, SQLite/Room, файлы — используются для долгосрочного сохранения данных, которые нужно сохранить между сессиями приложения, например настройки пользователя или кэш сложных данных. Такие данные могут быть загружены при создании Activity.
- **onRetainNonConfigurationInstance()** — устаревший способ передачи данных между пересозданиями Activity, сейчас почти не используется, замещён ViewModel.

#### Практический контекст
В практике часто комбинируют onSaveInstanceState для мелких transient-данных (например, позиция скролла), ViewModel для удержания состояния UI и бизнес-логики при смене конфигурации, а Persistent Storage для долгосрочного сохранения. В React Native и Flutter, например, аналогичные задачи решаются через state management библиотеки, но в чистом Android подходы именно такие. В React Native основной паттерн — сохранение через Redux или Context, в Android Jetpack ViewModel — правильный современный способ для управления состоянием Activity.
