# Что такое вебхуки?

### Категории

API, Network, Async, Integration, Event-driven, Technical

### Краткий ответ

**Что такое вебхуки?**  
- механизм **обратного вызова** по HTTP  
- сервер отправляет данные на URL клиента после события  
- работает **push**, а не pull  
- позволяет **автоматически** уведомлять внешние системы в реальном времени  
- используется для интеграций, синхронизации, триггеров  
- требует настройки **безопасности**: подписи, токены, фильтрация  
- повышает **эффективность** обмена данными без постоянного опроса сервера

### Подробный ответ
#### Основной ответ
**Вебхуки** (webhooks) — это механизм, с помощью которого один сервер или приложение может автоматически информировать другое о событии в режиме реального времени, отправляя HTTP-запрос (обычно POST) на заранее настроенный URL. Это альтернативный способ обмена данными в сравнении с периодическим опросом (polling), позволяющий получать обновления мгновенно и эффективно.

#### Ключевые моменты
- Вебхуки позволяют реализовать **асинхронное взаимодействие между сервисами**: например, когда платежная система сообщает вашему приложению об успешной оплате.
- Обычно на стороне получателя настраивается публичный endpoint, который **принимает данные в определённом формате** (чаще JSON или XML) и подтверждает получение HTTP-статусом 2xx.
- В отличие от API, где клиент постоянно запрашивает данные, вебхуки — **push-модель**, что уменьшает нагрузку и задержки. Но при этом требуется надёжная обработка ошибок и безопасность (подписи, проверка источника).
- Часто используются в SaaS-платформах, системах CI/CD, платежных шлюзах и уведомлениях о событиях.

#### Практический контекст
В реальных проектах вебхуки помогают интегрировать разные системы без лишней нагрузки — например, в React-приложениях с backend на Node.js с помощью Express настраивают роут `/webhook`, который принимает события от Stripe. Важен мониторинг и механизмы повторных попыток на случай недоступности сервера.
