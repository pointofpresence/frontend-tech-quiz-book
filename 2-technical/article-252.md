# Какие типы индексов есть и для чего они нужны?

### Категории

Architecture, Indexes, Backend, Database, Messaging, Queues, Technical

### Краткий ответ

**Типы индексов и их назначение**

- Индекс — структура данных для быстрой выборки из БД  
- **B-tree индекс**: универсальный для поиска, сортировки, диапазонов  
- **Hash индекс**: быстрый точечный поиск по равенству, без диапазонов  
- **Bitmap индекс**: эффективен для низкокардинальных полей, аналитика  
- **Full-text индекс**: поддержка полнотекстового поиска, специфично для текстовых данных  
- Используются для уменьшения времени запроса, ускорения операций чтения  
- Индексы занимают место и замедляют операции вставки/обновления  
- Выбор индекса зависит от типа данных, характера запросов и нагрузки  

Итог: индексы оптимизируют поиск, улучшая производительность чтения, адаптируясь к задачам и структурам данных.

### Подробный ответ
#### Основной ответ
**Индексы** в базах данных — это специализированные структуры данных, которые ускоряют поиск и сортировку по столбцам, минимизируя полный скан таблицы. Основные типы индексов ориентированы на разные сценарии запросов, обеспечивая баланс между скоростью чтения и затратами на запись.

#### Ключевые моменты
- **B-tree (B+ tree) индекс** — самый распространённый, хорошо подходит для поиска по точному значению и диапазонных запросов. Используется в PostgreSQL, MySQL InnoDB. Поддерживает быстрый поиск, вставки и удаление с логарифмической сложностью.
- **Hash индекс** — оптимален для быстрых точечных запросов равенства (WHERE column = value), но не поддерживает диапазонные запросы. В PostgreSQL применяется для определённых задач, но часто менее универсален.
- **Bitmap индекс** — эффективен для столбцов с низкой кардинальностью (небольшим числом уникальных значений), широко используется в аналитических СУБД (например, Oracle, ClickHouse). Быстро объединяется при сложных условиях.
- **Gin и GiST индексы (PostgreSQL)** — специализированные для полнотекстового поиска, работы с массивами, геоданными и прочими сложными структурами. Позволяют индексировать неструктурированные данные.
- **Полнотекстовые индексы** — предназначены для быстрого поиска по тексту с поддержкой лемматизации, ранжирования и стоп-слов, используются в MySQL, PostgreSQL (через GIN).

#### Практический контекст
На практике выбирают индекс по задаче: для OLTP систем часто B-tree, для аналитики — bitmap, для текстового поиска — полнотекстовые. Например, в PostgreSQL 14+ для JSONB полей применяют GIN индексы, чтобы ускорить операции поиска по ключам и значениям внутри JSON. Индексы снижают время ответа запросов до долей секунды, но увеличивают время вставки и размер диска, поэтому баланс важен.
