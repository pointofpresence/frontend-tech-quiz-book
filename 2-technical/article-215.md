# Как запустить PHP-приложение кроме PHP-FPM?

### Категории

Web server, Backend, Technical, Deployment, PHP

### Краткий ответ

**Как запустить PHP-приложение кроме PHP-FPM?**

- Запуск с помощью встроенного **веб-сервера PHP** (php -S), подходит для разработки  
- Использование **Apache с модулем mod_php** — интеграция с веб-сервером напрямую  
- Запуск через **CGI/FastCGI** — отдельный процесс для обработки запросов  
- Использование **Docker-контейнеров** с любым из вышеперечисленных методов  
- CLI-скрипты для фоновых задач без веб-сервера  
- Можно интегрировать с **nginx + FastCGI** без PHP-FPM, но менее эффективно  
- Выбор зависит от цели: разработка, продакшен, производительность

---

**Подробно:**  
1. Встроенный сервер PHP (`php -S localhost:8080`) — простой способ для локальной отладки.  
2. `mod_php` для Apache — классический способ до PHP-FPM, работает в процессе веб-сервера, но хуже масштабируется.  
3. CGI/FastCGI — внешний обработчик, можно обойтись без PHP-FPM, например, использованием spawn-fcgi.  
4. CLI-режим применяется для скриптов и задач вне HTTP.  
5. Docker позволяет изолировать и запускать любое из решений без конфликта.  

Выбор альтернатива PHP-FPM зависит от требований по производительности и архитектуре.

### Подробный ответ
#### Основной ответ  
Кроме использования **PHP-FPM** для запуска PHP-приложения, можно воспользоваться встроенным **веб-сервером командной строки PHP** (PHP built-in web server), а также альтернативными методами, такими как запуск с помощью **Apache модулей (mod_php)** или через **FastCGI с другим процесс-менеджером**. Встроенный сервер особенно полезен для разработки и тестирования, а mod_php — более традиционный способ запуска в продуктивной среде, хотя он менее масштабируемый и гибкий по сравнению с PHP-FPM.

#### Ключевые моменты  
- **Встроенный сервер PHP (`php -S`)** — появился с PHP 5.4, позволяет быстро запускать минимальный HTTP-сервер на локальной машине без дополнительной настройки, но не рекомендуется для продакшена из-за ограниченной производительности и безопасности.  
- **Apache с mod_php** — модуль, который интегрирует PHP прямо в процесс веб-сервера Apache, устраняя необходимость в отдельном демоне (PHP-FPM), однако каждый Apache-процесс будет весить больше, что снижает масштабируемость.  
- **FastCGI + альтернативные менеджеры процессов** — например, запуск PHP через **spawn-fcgi** или другие frontends, что позволяет организовать процесс в разных условиях, но требует более сложной настройки.

#### Практический контекст  
Для локальной разработки и отладки удобно использовать встроенный сервер PHP (`php -S localhost:8000`), что экономит время. На реальных серверах чаще выбирают PHP-FPM из-за управления процессами и высокой производительности, но в legacy-системах или с Apache может использоваться mod_php. Альтернативные FastCGI-стартеры встречаются реже и требуют специфической настройки под окружение.
