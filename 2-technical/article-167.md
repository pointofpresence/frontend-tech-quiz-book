# Что такое боксинг и анбоксинг?

### Категории

Memory management, Performance, Data types, Technical

### Краткий ответ

**Что такое боксинг и анбоксинг?**  
- Концепция из **языков программирования** (Java, C#)  
- **Боксинг** — упаковка примитивного значения (int, bool) в объект (обертку)  
- **Анбоксинг** — извлечение примитивного значения из объекта-обертки  
- Боксинг — **автоматическая или явная** конвертация в объектный тип  
- Анбоксинг — **обратное преобразование** объекта в примитив  
- Влияет на **производительность** из-за аллокаций и копирований  
- Практическое значение — позволяет использовать примитивы в структурах, где нужны объекты (коллекции, API)

### Подробный ответ
#### Основной ответ
**Боксинг (boxing)** и **анбоксинг (unboxing)** — это термины из области программирования, относящиеся к преобразованию между примитивными типами данных и объектными типами-обёртками. В языках, таких как Java и C#, боксинг — это процесс упаковки примитивного значения (например, `int`, `bool`) в объект-обёртку (`Integer`, `Boolean`), чтобы работать с ним как с объектом. Анбоксинг — обратный процесс: извлечение примитивного значения из объекта-обёртки.

#### Ключевые моменты
- **Автоматический боксинг и анбоксинг** появились в популярных версиях языков (Java с версии 5, C# уже с ранних версий) и значительно упрощают код, устраняя необходимость ручного создания объектов и извлечения значений.
- **Производительность:** боксинг создаёт дополнительные объекты в куче, что может увеличить нагрузку на сборщик мусора и снизить производительность, особенно в горячих участках кода с интенсивными операциями.
- **Безопасность типов:** боксинг превращает примитив в ссылочный объект, позволяя использовать их в коллекциях (например, `List<Integer>`), которые принимают только объекты, а не примитивы.
- **Нюансы:** при анбоксинге возможны `NullPointerException`, если объект неожиданно равен `null`, что требует осторожности.

#### Практический контекст
В реальных приложениях типично использовать боксинг при работе с коллекциями или API, ожидающими объекты, например, в `Generics` Java. В высокопроизводительных подсистемах стараются минимизировать боксинг, чтобы снизить аллокации и уменьшить нагрузку на GC, используя примитивные коллекции или специализированные структуры (например, `IntStream` в Java 8+ или `Span<T>` в C#).
