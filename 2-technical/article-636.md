# Какие основные методы жизненного цикла Activity?

### Категории

Android, Mobile, Activity, Lifecycle, Android lifecycle, Technical

### Краткий ответ

**Основные методы жизненного цикла Activity**

- Android UI-компонент
- onCreate(): инициализация, загрузка UI и данных
- onStart(): Activity становится видимой
- onResume(): начало взаимодействия с пользователем, активный режим
- onPause(): частичная потеря фокуса, сохранение данных
- onStop(): Activity невидима, освобождение ресурсов
- onDestroy(): финальная очистка, удаление Activity
- onRestart(): повторный запуск после onStop

Каждый метод отражает этап управления состоянием, важен для правильного распределения ресурсов и UX.

### Подробный ответ
#### Основной ответ
**Основные методы жизненного цикла Activity** в Android — это набор callback-методов, которые система вызывает для управления состояниями Activity от создания до уничтожения. Они помогают управлять ресурсами и взаимодействием с пользователем, поддерживая корректное поведение приложения при смене конфигураций, переключении между активностями и привозврате с фонового режима.

Ключевые методы жизненного цикла Activity:
- `onCreate()` — инициализация Activity, установка UI, начальные настройки.
- `onStart()` — Activity становится видимой, но не интерактивной.
- `onResume()` — Activity в фокусе, пользователь может взаимодействовать.
- `onPause()` — подготовка к уходу с переднего плана, сохранение изменений.
- `onStop()` — Activity теперь невидима, освобождает ресурсы.
- `onDestroy()` — финальная очистка перед уничтожением.

#### Ключевые моменты
- **Разделение onPause и onStop:** onPause всегда вызывается при частичной или полной потере фокуса, здесь сохраняют данные; onStop — когда Activity полностью скрыта.
- **onCreate — точка инициализации:** задаются layout, подключаются ViewModel, восстанавливаются state через Bundle.
- **onRestart:** вызывается при повторном старте Activity из stopped-состояния, обычно для возобновления UI.

#### Практический контекст
В реальных проектах корректное использование этих методов важно для избежания утечек памяти (например, освобождение в onStop), сохранения состояния (onPause/onSaveInstanceState) и плавного UX — например, загрузка данных в onStart или onResume, что снижает время отклика и удерживает пользовательский интерфейс актуальным. В React Native или Flutter они маппятся к своим жизненным циклам, но нативное Android-приложение требует понимания именно этих callback'ов.
