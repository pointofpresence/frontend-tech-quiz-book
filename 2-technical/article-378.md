# Механизмы защиты API — токены, rate limiting, права доступа.

### Категории

SOLID, Authentication, Security, API, Architecture, Authorization, Database, Rate limiting, Technical

### Краткий ответ

**Механизмы защиты API — токены, rate limiting, права доступа**

- аутентификация через **токены**: JWT или OAuth2, защищают от неавторизованного доступа  
- **rate limiting** ограничивает число запросов, предотвращая DDoS и злоупотребления  
- **права доступа** (RBAC/ABAC) разграничивают действия пользователей и сервисов  
- токены часто **подписаны**, обеспечивают целостность и срок действия  
- rate limiting реализуется на уровне API Gateway или сервера  
- права доступа помогают минимизировать риск **эскалации привилегий**  
- вместе обеспечивают **безопасность, производительность и контроль** API  

Этот набор охватывает криптографию, масштабируемость и управление безопасностью, критичные для надежной защиты API.

### Подробный ответ
#### Основной ответ  
Защита API — важная часть архитектуры современных сервисов, она обеспечивает безопасность и устойчивость системы. Основные механизмы — **аутентификация через токены**, **rate limiting** и **ограничение прав доступа** (authorization). Токены (например, JWT) позволяют удостоверить пользователя без постоянного запроса к базе, обеспечивая stateless аутентификацию. **Rate limiting** предотвращает перегрузки и злоупотребления, ограничивая число запросов за определённый период. **Права доступа** реализуют контроль, кто и что может делать с ресурсами API, часто через роли и политики.

#### Ключевые моменты  
- **Токены аутентификации** часто используют JWT (JSON Web Tokens) или OAuth 2.0 Bearer tokens; JWT удобен тем, что включает claims и может проверяться без обращения к серверу. Важно обеспечить их безопасность — использовать HTTPS, выставлять короткий срок жизни, возможность отзыва.  
- **Rate limiting** реализуют на уровне API gateway или внутри сервисов: фиксированное окно, скользящее окно, token bucket или leaky bucket алгоритмы. Обычно хранят счётчики в Redis для масштабируемости с поставленной целью — защита от DDoS и перебоев.  
- **Права доступа** строятся на RBAC (Role-Based Access Control) или ABAC (Attribute-Based Access Control), где определяются роли пользователей и права на операции над ресурсами. Это предотвращает неавторизованный доступ даже при корректной аутентификации.

#### Практический контекст  
В реальных проектах применяют связку: API gateway (например, Kong, AWS API Gateway) реализует rate limiting и аутентификацию токенами, а backend сервисы обеспечивают fine-grained авторизацию через middleware. В React-приложениях с REST API часто в заголовках передают JWT, а backend (например, на Node.js с Express) проверяет token, права и учитывает rate limits через Redis. Это позволяет достичь 99.9% uptime и безопасности без излишних задержек.
