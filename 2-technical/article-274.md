# Что такое кастомные исключения и зачем они нужны?

### Категории

Error handling, Architecture, PHP, Python, Backend, Database, Exceptions, Technical, Message queues

### Краткий ответ

**Что такое кастомные исключения и зачем они нужны?**  
- ООП и обработка ошибок  
- пользовательские классы ошибок  
- расширяют базовые Exception/Throwable  
- дают семантически точные ошибки  
- упрощают отладку и поддержку кода  
- улучшают обработку специфичных кейсов  
- повышают читабельность и контроль логики  
- применяются для бизнес-правил и API-ошибок

### Подробный ответ
#### Основной ответ
**Кастомные исключения** — это пользовательские классы ошибок, которые разработчик создаёт, наследуясь от базовых типов исключений в языке программирования. Они позволяют более явно и чётко обозначать конкретные ошибки бизнес-логики или технических условий, характерных именно для приложения. Это обеспечивает более понятную и структурированную обработку ошибок.

#### Ключевые моменты
- **Чёткая семантика**: кастомные исключения информируют о конкретной проблеме, например, `UserNotFoundException` вместо общего `Exception`. Это облегчает диагностику и поддержку.
- **Разделение логики обработки**: позволяет писать разные обработчики для разных типов ошибок, улучшая читаемость и надежность кода.
- **Улучшенная отладка и логирование**: благодаря кастомным типам легче фильтровать и анализировать исключения в логах и мониторинге, например, в Sentry или Prometheus.
- В современных приложениях (например, на Java 17, Python 3.10+, C# 10) кастомные исключения являются стандартной практикой для построения поддерживаемых архитектур.

#### Практический контекст
Например, в REST API вместо общего 500 Internal Server Error можно бросать `ResourceNotFoundException`, чтобы вернуть 404 код, а `ValidationException` — для 400 Bad Request. Это упрощает клиентам понимание ошибок и улучшает UX. В крупных проектах это помогает разделять ответственность и создавать устойчивый к ошибкам код.
