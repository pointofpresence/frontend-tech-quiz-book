# Как сделать поле обязательным в JSON-схеме?

### Категории

Validation, Data structures, Technical, Schema

### Краткий ответ

**Как сделать поле обязательным в JSON-схеме?**

- JSON Schema — стандарт описания структуры JSON
- Для обязательных полей используется ключ **"required"**
- "required" — массив с именами обязательных свойств объекта
- Поля в "properties" описывают структуру и типы данных
- Если поле в "required" отсутствует — валидация не пройдена
- Пример: `"required": ["name", "age"]` — поля name и age обязательны
- Используется для гарантии наличия данных при валидации JSON

---

**Пример минимальной схемы:**
```json
{
  "type": "object",
  "properties": {
    "name": { "type": "string" },
    "age": { "type": "integer" }
  },
  "required": ["name", "age"]
}
```

---

**Итог:**  
Обязательность определяется массивом ключей в поле **"required"**, указывающим, какие свойства объекта должны обязательно присутствовать.

### Подробный ответ
#### Основной ответ
В **JSON Schema** для обозначения обязательного (required) поля используется ключевое свойство `"required"`, которое принимает массив строк с именами свойств, обязательных для объекта. По умолчанию все поля считаются необязательными, и чтобы сделать поле обязательным, нужно явно перечислить его имя в массиве `"required"`.

#### Ключевые моменты
- Свойство `"required"` применяется только к объектам, оно указывается на уровне схемы, описывающей объект.
- Например, чтобы сделать поле `"name"` обязательным, схема будет содержать `"required": ["name"]`.
- При этом само поле в `"properties"` описывается отдельно — `"required"` не задаёт тип или дополнительные ограничения, а только сигнализирует валидатору, что это поле должно присутствовать.
- Если в `"required"` указать имя поля, которого нет в `"properties"`, валидаторы могут ругаться — хорошей практикой считается описывать все обязательные поля в `"properties"`.
- Для вложенных объектов правило действует аналогично — для каждого уровня указывается свой `"required"`.

#### Практический пример
```json
{
  "type": "object",
  "properties": {
    "name": { "type": "string" },
    "age": { "type": "integer" }
  },
  "required": ["name"]
}
```
Здесь поле `name` обязательно, а `age` может отсутствовать.

#### Практический контекст
Такой подход широко используется валидации API на основе OpenAPI (например, версии 3.0+), где JSON Schema задаёт контракт и гарантирует, что клиент или сервер получит минимально необходимый набор данных. В React-формах или Node.js (с ajv) эта проверка помогает отсеивать некорректные данные ещё до бизнес-логики.
