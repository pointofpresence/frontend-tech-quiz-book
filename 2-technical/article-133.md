# Где могут быть полезны mapped types?

### Категории

Static typing, TypeScript, Technical, Browser, Mapped types, Optimization

### Краткий ответ

**Где могут быть полезны mapped types?**  
- Типы в TypeScript для **создания новых типов на основе существующих**  
- Автоматическое **преобразование свойств** интерфейсов или типов (например, readonly, optional)  
- Упрощают **DRY:** повторное использование и генерация схожих структур  
- Позволяют создавать **типовые утилиты** для манипуляций со свойствами  
- Идеальны для **интерфейсов API**, где поля наследуются с изменениями  
- Помогают строго типизировать динамические или частично изменяемые объекты  
- Повышают безопасность и удобство поддержки кода с типизированным интерфейсом

### Подробный ответ
#### Основной ответ
**Mapped types** в TypeScript — это мощный инструмент для создания новых типов на основе существующих с помощью перебора ключей. Они позволяют динамически трансформировать свойства исходного типа (например, менять их модификаторы или типы значений), что существенно упрощает создание вариаций типов без дублирования кода.

#### Ключевые моменты
- **Гибкая генерация типов**: mapped types позволяют создавать типы вроде `Partial<T>`, `Readonly<T>`, `Required<T>`, изменяя свойства на основе существующего интерфейса, что повышает переиспользуемость и безопасность типов.
- **Автоматизация трансформаций**: при работе с большим количеством похожих интерфейсов (например, мутация состояния в Redux, валидация форм или DTO) mapped types помогают избежать ручного описания каждого варианта типа и уменьшают риск ошибок.
- **Поддержка keyof и условных типов**: комбинация mapped types с `keyof` и условными типами позволяет создавать очень точные и контекстно-зависимые модели, например, выбирать только обязательные или опциональные поля, изменять типы на основе условий.

#### Практический контекст
В реальных проектах mapped types часто используют для описания API-ответов, где из полного типа данных выделяют только необходимые поля, или при создании универсальных утилит для работы с формами, где можно автоматически сделать все поля опциональными для частичного обновления (patch), либо наоборот — требовать все свойства. В React 18+ mapped types помогают точно типизировать пропсы компонентов с динамически изменяемыми ключами и состояниями, что снижает количество багов и увеличивает скорость разработки.
