# Что такое коллекции в Kotlin?

### Категории

Android, Data structures, Technical, Collections

### Краткий ответ

**Что такое коллекции в Kotlin?**

- основа для работы с группами данных  
- разделяются на **мутабельные** (изменяемые) и **имутабельные** (только для чтения)  
- включают: List (упорядоченный), Set (уникальные элементы), Map (ключ→значение)  
- поддерживают функциональные операции: map, filter, reduce, flatMap  
- оптимизированы для **безопасности типов** и **лексической выразительности**  
- позволяют работать с последовательностями для ленивых вычислений  
- важны для структурирования данных и упрощения бизнес-логики  

Итог: мощный, типобезопасный API для эффективной и удобной работы с данными

### Подробный ответ
#### Основной ответ
**Коллекции в Kotlin** — это специализированные интерфейсы и классы для хранения и обработки групп объектов. Коллекции упрощают работу с наборами данных, предоставляя удобные методы для фильтрации, преобразования, поиска и сортировки. В Kotlin коллекции разделены на **immutable (неизменяемые)** и **mutable (изменяемые)**, что повышает безопасность и предсказуемость кода.

#### Ключевые моменты
- **Immutable vs Mutable**: Базовые интерфейсы — `Collection`, `List`, `Set`, `Map`. Для неизменяемых коллекций (`List`, `Set`, `Map`) доступны только операции чтения, а для изменяемых — дополнительно методы добавления, удаления, изменения элементов (`MutableList`, `MutableSet`, `MutableMap`).
- **Расширенные функциональные возможности**: Kotlin предоставляет богатый набор функций высшего порядка (`map`, `filter`, `reduce`, `groupBy` и т.д.) с поддержкой лямбд и последовательностей (`Sequence`), что помогает писать выразительный и лаконичный код.
- **Интероперабельность с Java API**: Kotlin коллекции совместимы с Java Collections Framework, позволяя работать с java.util.List или HashMap без явных преобразований при использовании JVM, сохраняя при этом безопасность типов.

#### Практический контекст
Типично в проектах на Kotlin используют неизменяемые коллекции для бизнес-логики, что снижает баги, связанные с случайными мутациями. Mutable-коллекции применяются в слоях, где нужен изменяемый state, например, при построении UI или работе с кешем. Также последовательности (`Sequence`) полезны при работе с большими потоками данных, чтобы минимизировать издержки по памяти и повысить производительность за счёт отложенных вычислений.
