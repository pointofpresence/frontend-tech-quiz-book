# Как работает position: sticky? Когда элемент "отклеивается"?

### Категории

UI, Sticky, CSS, Positioning, Layout, Live coding, Algorithms

### Краткий ответ

**Как работает position: sticky? Когда элемент "отклеивается"?**  
- CSS-свойство для частично фиксированного позиционирования  
- Элемент ведёт себя как **relative** до достижения заданной границы скролла  
- При пролистывании фиксируется относительно контейнера на заданном отступе (top, bottom, left, right)  
- "Отклеивается", когда достигает края родительского блока (контейнера с overflow, position)  
- Работает внутри своего контейнера, **не выходит за его пределы**  
- Используется для фиксации заголовков, меню при прокрутке страницы  
- Требует корректной структуры DOM и родителя с допустимым overflow для корректной работы

### Подробный ответ
#### Основной ответ  
Свойство **`position: sticky`** — это гибрид между **relative** и **fixed** позиционированием. Такой элемент ведёт себя как обычный **relative** (относительно своего нормального положения) до тех пор, пока не достигнет заданного смещения (например, `top: 0`) относительно области прокрутки, после чего начинает вести себя как **fixed** и "прилипает" к этому месту в пределах ближайшего прокручиваемого родителя или viewport. Элемент "отклеивается" (то есть перестаёт быть фиксированным) когда пользователь прокручивает дальше, и sticky-область заканчивается, либо когда элемент полностью выходит за границы контейнера.

#### Ключевые моменты  
- **Sticky работает внутри ближайшего scrolling container** — если родитель задаёт ограничивающую область прокрутки (`overflow: auto/scroll`), то "прилипание" происходит только внутри неё, а не по всему viewport.  
- Элемент с `position: sticky` и, например, `top: 0` прилипает к верхнему краю области прокрутки, но дальше "отклеивается", если прокрутка поднимает контейнер вверх настолько, что элемент выходит из видимой части контейнера или сталкивается с соседними элементами.  
- Чтобы работало sticky, у родительского контейнера не должно быть `overflow: visible` по всем осям, иначе браузер не вычислит область ограничения корректно.  
- Поведение зависит от указанных свойств `top`, `left`, `bottom` или `right`, которые задают точку "приклеивания" элемента.

#### Практический контекст  
В реальных веб-приложениях часто используют `position: sticky` для **фиксирования заголовков таблиц** или **плавающих меню**, которые должны оставаться видимыми пока пользователь не пролистает весь контейнер. Пример: в React 18+ для сложных интерфейсов sticky помогает избежать лишних событий scroll и повышает производительность по сравнению с JS-реализациями фиксированных элементов.
