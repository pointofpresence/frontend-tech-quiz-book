# Как создать Mock-сервер для эмуляции ответов API в тестах?

### Категории

UI, Backend, Automation, API, Python, Selenium, Mock, Testing

### Краткий ответ

**Как создать Mock-сервер для эмуляции ответов API в тестах?**

- тестирование API без реального сервера  
- создает имитацию реальных HTTP-запросов/ответов  
- настройка маршрутов с предопределенными ответами  
- можно использовать фреймворки: **json-server**, **WireMock**, **MockServer**  
- работает локально или в CI/CD среде  
- помогает изолировать тесты, ускорить их и повысить стабильность  
- важна автоматизация запуска и синхронизация с тестами  

Итого: mock-сервер — быстрый, контролируемый эмулятор реального API для стабильных интеграционных тестов.

### Подробный ответ
#### Основной ответ  
Для создания **Mock-сервера** в тестах, который эмулирует ответы API, обычно используют специализированные библиотеки или standalone инструменты для имитации HTTP-запросов и ответов. Основная задача — предоставить предсказуемое поведение внешнего API без настоящих сетевых вызовов, что ускоряет и стабилизирует тесты.  

#### Ключевые моменты  
- **Выбор инструмента:** Например, для JavaScript/Node.js популярны **json-server** (для быстрого создания REST API с фейковыми данными), **nock** (перехватывает HTTP-запросы на уровне клиента) и **msw (Mock Service Worker)** — современный инструмент, эмулирующий реальные вызовы на уровне Service Worker и поддерживающий браузерные и Node.js тесты.  
- **Настройка ответов:** Mock-сервер конфигурируется для возврата статичных или динамически сгенерированных ответов, включая коды статуса, заголовки и тело запроса. Обычно настраивают разные сценарии — успешные ответы, ошибки, задержки для тестирования таймаутов.  
- **Где размещать:** Mock-сервер запускается либо как отдельный процесс локально в тестах, либо интегрируется непосредственно в тестовый фреймворк (например, jest + msw). Это позволяет изолировать тестируемую логику и избежать влияния нестабильности реального API.  

#### Практический контекст  
В крупных проектах с микросервисной архитектурой часто используют **WireMock** (Java) или **MockServer**, которые позволяют эмулировать сложные HTTP-интеракции. В фронтенде с React 18+ часто применяют **msw** для энд-ту-энд или интеграционных тестов, обеспечивая latency ~50ms и 99.9% консистентность ответов. Это критично для CI/CD, чтобы тесты были надежными и быстрыми.
